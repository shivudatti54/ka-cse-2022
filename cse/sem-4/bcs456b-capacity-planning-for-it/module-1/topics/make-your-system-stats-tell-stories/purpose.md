# Learning Objectives

After studying this topic, you should students able to:

1. Identify and explain the key system metrics (CPU, memory, disk I/O, network) used in capacity planning and describe what each metric indicates about system health.

2. Establish baselines for system performance by collecting and analyzing metrics over appropriate time periods to distinguish normal from abnormal behavior.

3. Apply trend analysis techniques to predict future resource needs based on historical system statistics.

4. Interpret data visualizations including time-series graphs, heat maps, and scatter plots to extract meaningful patterns from system metrics.

5. Distinguish between correlation and causation in system metric relationships and apply this understanding to identify actual performance bottlenecks.

6. Detect anomalies in system statistics and explain the process of investigating root causes of unusual patterns.

7. Translate technical system statistics into clear, understandable narratives suitable for communication with non-technical stakeholders.
