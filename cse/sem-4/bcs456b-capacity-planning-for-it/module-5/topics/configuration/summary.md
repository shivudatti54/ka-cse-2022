# Configuration Management - Summary

## Key Definitions and Concepts

- **Configuration Management**: A process that identifies, controls, maintains, and verifies the versions of all configuration items (CIs) in an organization's IT infrastructure
- **Configuration Item (CI)**: Any component that needs to be managed to deliver an IT service, including hardware, software, documentation, and people
- **CMDB (Configuration Management Database)**: The central repository storing all information about CIs and their relationships
- **Configuration Management System (CMS)**: The overall system comprising tools, processes, and procedures for managing configurations
- **Baseline**: A snapshot of configuration items at a specific point in time for comparison purposes

## Important Formulas and Theorems

There are no specific formulas in this topic, but the following frameworks are essential:

- **Five Configuration Management Activities**: Planning → Identification → Control → Status Accounting → Verification
- **Three Types of Baselines**: Functional Baseline, Allocated Baseline, Product Baseline
- **Three CI Categories**: Business CIs, Technical CIs, Supporting CIs

## Key Points

- Configuration Management provides the foundation for all other ITSM processes including Change Management, Incident Management, and Capacity Planning
- The CMDB must contain accurate, complete, and current data to be effective
- Relationships between CIs are crucial for impact analysis and understanding service dependencies
- CIs can be hardware (servers, network devices), software (applications), or documentation (procedures)
- Automated discovery tools improve CMDB accuracy and reduce manual effort
- Configuration management supports both tactical (incident resolution) and strategic (capacity planning) decisions
- Regular audits are necessary to verify CMDB accuracy against physical infrastructure

## Common Mistakes to Avoid

1. Confusing CMS with CMDB - CMS is the entire system, CMDB is the database component
2. Treating configuration management as merely data storage without understanding its strategic importance
3. Ignoring the relationships between CIs when analyzing capacity needs
4. Failing to update the CMDB in real-time, leading to data becoming outdated
