# Learning Objectives

After studying this topic, you should students able to:

1. Define Configuration Management and explain its significance in IT Service Management
2. Identify and classify Configuration Items (CIs) with appropriate examples
3. Describe the structure and components of a Configuration Management Database (CMDB)
4. Explain the relationship between Configuration Management and Capacity Planning
5. Understand the five key activities of the Configuration Management Process
6. Differentiate between CMS and CMDB with practical examples
7. Analyze how configuration data supports capacity planning decisions
8. Explain the concept of baselines and their importance in configuration management
