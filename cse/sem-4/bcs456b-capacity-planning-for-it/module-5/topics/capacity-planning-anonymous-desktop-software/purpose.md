# Learning Objectives

After studying this topic, you should students able to:

1. Explain the concept of capacity planning and its importance for a desktop software company

2. Identify the key components of IT infrastructure that require capacity planning in a software development organization

3. Apply capacity calculation methods to determine resource requirements for development, testing, and support infrastructure

4. Differentiate between lead, lag, and match capacity planning strategies and select appropriate approaches for different scenarios

5. Analyze the trade-offs between cloud-based and on-premise infrastructure for capacity provisioning

6. Evaluate capacity requirements using quantitative methods including queueing theory and utilization calculations

7. Design a capacity planning framework appropriate for software development and release management

8. Assess the cost implications of different capacity planning decisions and recommend cost-effective solutions
