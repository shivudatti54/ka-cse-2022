# Learning Objectives

After studying this topic, you should students able to:

1. Explain the fundamental concepts of cloud computing service models (IaaS, PaaS, SaaS) and their implications for capacity planning responsibilities.

2. Identify and describe various cloud resource types including compute, storage, network, and database resources that require capacity planning.

3. Differentiate between vertical and horizontal scaling approaches and explain why horizontal scaling is preferred in cloud environments.

4. Analyze the three main auto-scaling mechanisms (metric-based, scheduled, and predictive) and determine appropriate scenarios for their application.

5. Calculate resource requirements for given workload scenarios using appropriate capacity planning formulas and techniques.

6. Evaluate different cloud capacity models (rightsizing, reserved, spot, on-demand) and recommend cost-effective strategies based on workload characteristics.

7. Design a basic capacity planning strategy for cloud-based applications considering scalability, cost optimization, and performance requirements.

8. Understand the considerations and challenges involved in multi-cloud and hybrid cloud capacity planning for enterprise environments.
