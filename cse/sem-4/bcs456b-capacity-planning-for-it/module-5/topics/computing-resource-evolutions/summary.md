# Computing Resource Evolutions - Summary

## Key Definitions and Concepts

- **Mainframe Computing:** Centralized computing paradigm using large, powerful single machines to serve entire organizations; capacity planning focused on sizing for peak requirements over 3-5 year horizons.

- **Client-Server Computing:** Decentralized paradigm distributing processing between client machines and servers; introduced tiered architectures and the need for multi-tier capacity planning.

- **Distributed Computing:** Spreading computational tasks across interconnected machines; enabled horizontal scalability and introduced planning for network latency and data consistency.

- **Grid Computing:** Aggregating distributed resources for collective problem-solving; allowed resource sharing across organizations but created deterministic planning challenges.

- **Cloud Computing:** Virtualized computing resources delivered as metered services with five essential characteristics: on-demand self-service, broad network access, resource pooling, rapid elasticity, and measured service.

- **Edge Computing:** Bringing computation and data storage closer to data sources; addresses latency and bandwidth concerns for IoT and real-time applications.

## Important Formulas and Theorems

- **Cloud Cost Calculation:** Monthly Cost = Instance Hours × Rate per Instance Hour

- **Cost Savings from Elasticity:** Savings = (Traditional Annual Cost) - (Cloud Annual Cost)

- **Edge Data Reduction:** Reduced Data = Original Data Volume × (1 - Alert Ratio)

- **Utilization Rate:** Utilization = Actual Usage Hours / Available Hours × 100%

## Key Points

- Computing evolved through distinct paradigms: mainframes → client-server → distributed → grid → cloud → edge

- Each paradigm introduced new capacity planning challenges and shifted the balance between capital and operational expenditure

- Cloud computing revolutionized capacity planning through elastic scaling and pay-as-you-go pricing

- IaaS retains infrastructure planning responsibility; PaaS shifts to application-level; SaaS eliminates infrastructure planning

- Edge computing reduces latency and bandwidth costs by processing data locally, sending only processed results to cloud

- Modern enterprises operate hybrid environments combining multiple computing paradigms

- Capacity planning evolved from static long-term provisioning to dynamic, demand-responsive resource management

## Common Mistakes to Avoid

- Confusing cloud service models (IaaS, PaaS, SaaS) and their capacity planning implications

- Assuming edge computing replaces cloud computing rather than complementing it

- Overlooking the hybrid nature of modern IT environments when planning capacity

- Failing to consider both direct costs (infrastructure) and indirect costs (management, training) in comparisons

- Underestimating the complexity of capacity planning in distributed and cloud environments

## Revision Tips

1. Create a timeline chart showing computing paradigms, their time periods, and key capacity planning characteristics.

2. Memorize the five essential characteristics of cloud computing as defined by NIST.

3. Practice cost calculation problems comparing traditional and cloud infrastructure models.

4. Review case studies of organizations that transitioned between computing paradigms, understanding their capacity planning evolution.

5. Focus on understanding why each computing evolution occurred—cost, scalability, latency, agility—rather than just memorizing facts.
