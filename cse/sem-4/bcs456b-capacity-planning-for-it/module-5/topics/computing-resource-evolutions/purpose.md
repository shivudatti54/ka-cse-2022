# Learning Objectives

After studying this topic, you should students able to:

1. Describe the evolution of computing resources from mainframes through modern cloud and edge computing, identifying key characteristics of each paradigm.

2. Explain how capacity planning approaches have transformed across different computing eras, from static mainframe provisioning to dynamic cloud elasticity.

3. Compare the three primary cloud service models (IaaS, PaaS, SaaS) and their implications for capacity planning responsibilities.

4. Analyze scenarios where edge computing provides advantages over centralized cloud architectures, considering latency, bandwidth, and cost factors.

5. Evaluate capacity planning strategies for modern hybrid IT environments combining multiple computing paradigms.

6. Calculate cost comparisons between traditional dedicated infrastructure and cloud-based elastic computing for given workload scenarios.

7. Apply understanding of computing evolution trends to anticipate future capacity planning requirements in enterprise IT environments.
