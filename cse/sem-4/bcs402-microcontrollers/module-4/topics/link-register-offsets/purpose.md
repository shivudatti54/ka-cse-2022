# Learning Objectives

After studying this topic, you should be able to:

1. Explain the role of the Link Register (LR/R14) in ARM Cortex-M subroutine calls and exception handling mechanisms.

2. Identify and interpret various EXC_RETURN values used in ARM Cortex-M processors for different execution modes and stack pointer selections.

3. Calculate the memory offset of the saved Link Register within standard and extended exception stack frames.

4. Analyze bit fields within EXC_RETURN values to determine processor state before an exception occurred.

5. Describe the differences between Main Stack Pointer (MSP) and Process Stack Pointer (PSP) in Thread and Handler modes.

6. Implement proper exception return sequences using the appropriate EXC_RETURN values in embedded applications.

7. Apply knowledge of link register offsets for stack unwinding, debugging, and custom exception handling in microcontroller firmware development.
