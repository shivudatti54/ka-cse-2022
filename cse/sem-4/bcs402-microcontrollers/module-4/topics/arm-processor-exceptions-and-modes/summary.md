# Arm Processor Exceptions And Modes

The ARM processor architecture implements a sophisticated exception handling system comprising seven operational modes: User, FIQ, IRQ, Supervisor, Abort, Undefined, and System modes. Exception handling involves a precise sequence of operations: saving the return address in the banked Link Register, copying CPSR to SPSR, modifying CPSR to reflect the new mode, and vectoring to the appropriate exception handler. The exception vector table is located at either low addresses (0x00000000) or high addresses (0xFFFF0000), with seven distinct vector entries for Reset, Undefined Instruction, SWI, Prefetch Abort, Data Abort, IRQ, and FIQ. The priority scheme ensures deterministic handling of simultaneous exceptions, with Reset having highest priority followed by Data Abort, FIQ, IRQ, Prefetch Abort, and finally SWI/Undefined instructions. The banked register architecture provides dedicated registers for each exception mode, minimizing interrupt latency and simplifying handler implementation. The assessment questions test application-level understanding through latency calculations, mode transition analysis, and system design evaluation.
