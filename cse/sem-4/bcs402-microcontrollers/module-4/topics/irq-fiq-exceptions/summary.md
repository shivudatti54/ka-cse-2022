# Irq Fiq Exceptions

ARM processors implement two primary interrupt mechanisms: IRQ for general-purpose peripheral handling and FIQ for time-critical applications. IRQ operates through the standard exception vector at 0x00000018 with configurable priority and nesting support, while FIQ achieves minimum latency through dedicated register banking (R8-R12) and strategic vector placement at 0x0000001C. The CPSR controls interrupt masking through its I and F bits, with automatic state preservation via the SPSR mechanism. Modern Cortex-M processors enhance this model through the NVIC, providing hardware priority levels, automatic state saving, and vector table relocation. Understanding these mechanisms is fundamental for developing real-time embedded systems meeting strict timing constraints.
