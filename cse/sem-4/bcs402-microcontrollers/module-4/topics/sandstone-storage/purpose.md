# Learning Objectives

After studying this topic, you should students able to:

1. Explain the layered architecture of Example Sandstone firmware framework and its components (HAL, Core Services, Application Interface)

2. Describe the bootloader initialization sequence in ARM Cortex-M microcontrollers, including stack configuration and vector table setup

3. Analyze the ARM exception handling mechanism and explain how Example Sandstone integrates with vector table structures

4. Compare IRQ and FIQ interrupt types, including their priority relationships and latency characteristics

5. Evaluate interrupt latency factors in ARM firmware including tail-chaining and late-arrival optimizations

6. Design interrupt priority schemes for embedded applications using the Nested Vectored Interrupt Controller (NVIC)

7. Implement exception handlers following ARM AAPCS calling conventions with proper register preservation

8. Explain vector table relocation using the Vector Table Offset Register (VTOR) and its applications in bootloader design
