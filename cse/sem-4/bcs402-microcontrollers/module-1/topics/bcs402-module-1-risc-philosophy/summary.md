# Bcs402 Module 1 Risc Philosophy

RISC (Reduced Instruction Set Computer) architecture emerged in the 1980s as a response to CISC complexity, emphasizing simplicity, regularity, and efficiency. Key principles include: (1) small, uniform instruction sets executing in single cycles; (2) load-store architecture where memory operations are limited to specific instructions; (3) large register files reducing memory access. Pipelining achieves near-one instruction per cycle throughput through overlapping execution stages, though hazards (data, control) require forwarding paths and branch prediction. RISC-V represents the modern open-source RISC implementation, while ARM exemplifies RISC principles in embedded microcontrollers. Mathematical analysis using CPI and execution time formulas demonstrates RISC's performance advantages over CISC architectures.
