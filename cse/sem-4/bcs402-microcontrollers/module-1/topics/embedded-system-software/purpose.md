# Learning Objectives

After studying this topic, you should be able to:

1. Distinguish between monolithic and layered software architectures in embedded systems and identify appropriate use cases for each in ARM-based applications.

2. Explain the ARM Cortex-M startup sequence, including vector table initialization, stack pointer setup, and memory section configuration.

3. Analyze linker scripts and memory maps for ARM microcontrollers to optimize code placement and memory utilization.

4. Evaluate the schedulability of periodic task sets using Rate Monotonic Scheduling (RMS) theory and the fundamental utilization bound equation.

5. Design interrupt service routines (ISRs) for ARM Cortex-M processors following proper conventions for context preservation.

6. Compare cross-compilation toolchains (ARM GCC, Keil MDK, IAR) and explain their roles in the embedded software development workflow.

7. Apply the RISC design philosophy principles to write efficient embedded software exploiting ARM's instruction set characteristics.

8. Calculate CPU utilization for real-time task sets and determine feasibility under different scheduling algorithms.
