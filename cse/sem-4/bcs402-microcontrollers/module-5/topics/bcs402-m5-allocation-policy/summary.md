# Bcs402 M5 Allocation Policy

Cache miss allocation policies govern whether fetched data blocks enter the cache structure upon misses. Three primary policies exist: **fetch-on-miss** (allocate all fetched blocks), **write-allocate** (allocate on write misses), and **no-allocate** (write-around, bypassing cache on write misses). These policies interact with write mechanisms (write-through/write-back) and replacement algorithms (LRU, FIFO, Random) to determine cache behavior. Performance analysis using Average Memory Access Time (AMAT) quantifies policy impact: AMAT = t_hit + M × t_miss. For ARM microcontrollers, Coprocessor 15 provides configuration registers (SCTLR, CTR, CCSIDR) enabling software control of cache allocation behaviors. Policy selection depends on workload characteristics—fetch-on-miss suits read-intensive programs with good temporal locality, while no-allocate benefits write-heavy streams where cache pollution must be minimized.
