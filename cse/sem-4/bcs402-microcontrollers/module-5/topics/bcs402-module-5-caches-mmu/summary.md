# Bcs402 Module 5 Caches Mmu

Cache memory resolves the processor-memory speed disparity by exploiting temporal and spatial locality through hierarchical storage. Three mapping techniques (direct-mapped, fully-associative, set-associative) represent hardware complexity versus hit rate trade-offs. Write policies (write-through, write-back) and replacement algorithms (LRU, FIFO, Random) critically affect performance, with AMAT = Hit Time + (Miss Rate × Miss Penalty) as the fundamental analytical formula. Coprocessor 15 provides ARM processor control via system control registers (c1), cache operations (c7), and cache lockdown (c9). The MMU implements virtual memory through TLB caching and multi-level page tables, providing address translation, memory protection via access permissions, and privilege separation. Integration in Cortex-M7/Cortex-A systems enables optimized firmware development with configurable cache policies and memory protection.
