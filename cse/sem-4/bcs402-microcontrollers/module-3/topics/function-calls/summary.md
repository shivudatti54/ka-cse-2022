# Function calls in ARM microcontrollers are governed by the ARM Procedure Call Standard (AAPCS), which defines how arguments are passed (first 4 in R0-R3, rest on stack), which registers are caller-saved (R0-R3, R12) vs callee-saved (R4-R11), and stack frame organization. The distinction between leaf (doesn't call other functions) and non-leaf (calls other functions) functions determines whether LR must be saved. Compiler optimizations include inlining (replacing calls with function body) and tail call optimization (converting tail recursion to jumps). Stack frame size calculation requires accounting for saved registers, return address, stacked arguments, and local variables, with 8-byte alignment requirements.
