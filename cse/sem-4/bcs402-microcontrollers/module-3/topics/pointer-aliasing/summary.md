# Pointer Aliasing

This module examined pointer aliasing in C from theoretical and practical perspectives. Key topics included formal definitions and classification of aliasing (direct, indirect, self), the strict aliasing rule and its optimization implications, and proof demonstrations showing how aliasing inhibits common compiler optimizations. The restrict keyword was introduced as a C99 mechanism for programmers to assert non-aliasing guarantees, with detailed examples of its impact on loop vectorization. Type punning techniques (union-based and memcpy-based) were covered for compliant embedded data conversion. The module concluded with embedded systems applications including volatile pointers for memory-mapped I/O and aliasing considerations in microcontroller register access patterns.
