# Learning Objectives

After studying this topic, you should be able to:

1. Explain the fundamental differences between NoSQL and relational databases, including their respective advantages and limitations in modern application development.

2. Describe the MongoDB architecture including the roles of mongod, mongos, config servers, and sharded cluster components.

3. Create, read, update, and delete documents in MongoDB collections using appropriate query operators and modification methods.

4. Design efficient MongoDB schemas by understanding when to use embedded documents versus normalized references.

5. Create and utilize various index types including single-field, compound, multikey, and text indexes to optimize query performance.

6. Build aggregation pipelines to perform complex data analysis and transformations using multiple pipeline stages.

7. Explain how MongoDB achieves high availability through replica sets and horizontal scalability through sharding mechanisms.
