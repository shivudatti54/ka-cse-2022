# Learning Objectives

After studying this topic, you should be able to:

1. Explain the concept of data item granularity in database management systems and its impact on concurrency control.

2. Identify the different levels of granularity in a database hierarchy (database, file, page, record, field) and their respective advantages and disadvantages.

3. Describe the Multiple Granularity Locking (MGL) protocol and explain why it is necessary for efficient database concurrency control.

4. Differentiate between various lock types: S, X, IS, IX, and SIX, and explain their purposes in the MGL protocol.

5. Apply the lock compatibility matrix to determine whether concurrent lock requests can be granted in given scenarios.

6. Demonstrate the correct lock acquisition and release sequence in a hierarchical database structure using MGL protocol.

7. Analyze the trade-offs between fine-grained and coarse-grained locking in terms of concurrency, overhead, and deadlock probability.

8. Solve exam problems involving lock compatibility, lock acquisition sequences, and deadlock detection in multiple granularity environments.
