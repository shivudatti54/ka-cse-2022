# Learning Objectives

After studying this topic, you should students able to:

1. Define functional dependency and explain its significance in relational database design
2. Differentiate between trivial, non-trivial, and completely non-trivial functional dependencies
3. Apply Armstrong's axioms (reflexivity, augmentation, transitivity) to derive new functional dependencies
4. Compute attribute closure (X⁺) using the closure algorithm for any given set of attributes
5. Identify super keys, candidate keys, and primary keys from functional dependencies
6. Find the minimal cover (canonical cover) of a given set of functional dependencies
7. Use derived inference rules (decomposition, union, pseudotransitivity) to simplify and solve FD-related problems
8. Apply the knowledge of functional dependencies to understand normalization concepts in subsequent modules
