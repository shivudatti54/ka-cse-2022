# Learning Objectives

After studying this topic, you should students able to:

1. Explain the purpose and importance of schema change statements in database management systems.

2. Use ALTER TABLE statements to add, modify, and drop columns from existing tables.

3. Apply ALTER TABLE to add and remove constraints including primary keys, foreign keys, and unique constraints.

4. Distinguish between DROP, TRUNCATE, and DELETE statements and select the appropriate command for different scenarios.

5. Create and drop indexes to improve database query performance.

6. Manage views including creating, querying, and dropping views using DDL commands.

7. Apply best practices for schema changes including using IF EXISTS clauses and handling dependencies with CASCADE/RESTRICT options.

8. Demonstrate understanding of how schema changes affect existing data and application functionality.
