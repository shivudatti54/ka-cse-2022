# Learning Objectives

After studying this topic, you should be able to:

1. Explain the purpose and importance of SQL Data Definition Language (DDL) in database management systems.

2. Identify and describe various SQL data types including numeric, character, date/time, and binary types with their appropriate use cases.

3. Create database tables using the CREATE TABLE statement with appropriate column definitions and constraints.

4. Modify existing table structures using ALTER TABLE statements including adding, modifying, and dropping columns and constraints.

5. Apply different types of constraints (PRIMARY KEY, FOREIGN KEY, UNIQUE, NOT NULL, CHECK, DEFAULT) to enforce data integrity.

6. Distinguish between DROP, TRUNCATE, and DELETE commands and understand their appropriate usage scenarios.

7. Design and implement relational database schemas with proper referential integrity through foreign key constraints.

8. Optimize database storage by selecting appropriate data types based on the nature of data to be stored.
