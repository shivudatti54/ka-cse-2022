# Learning Objectives

After studying this topic, you should students able to:

1. Explain the three fundamental database update operations: INSERT, DELETE, and UPDATE, including their syntax and semantics.

2. Describe the concept of a transaction in DBMS and explain why transactions are necessary for maintaining data integrity.

3. Enumerate and explain the four ACID properties of transactions (Atomicity, Consistency, Isolation, Durability) with examples.

4. Identify different types of integrity constraints (key, referential, domain, check) that must students enforced during database operations.

5. Explain how constraint violations are handled during INSERT, DELETE, and UPDATE operations.

6. Describe the various referential actions (NO ACTION, CASCADE, SET NULL, SET DEFAULT) for handling foreign key violations during deletions.

7. Distinguish between immediate and deferred constraint checking and explain their implications for transaction processing.

8. Understand the transaction states (Active, Partially Committed, Committed, Aborted) and the state transitions.
