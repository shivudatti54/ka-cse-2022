# Learning Objectives

After studying this topic, you should students able to:

1. **Understand the need for ER-to-relational mapping** in the database design process and explain how it bridges conceptual and logical database design.

2. **Map regular (strong) entity sets to relations** by creating relational schemas with appropriate primary keys and handling composite attributes.

3. **Map weak entity sets to relations** by including the owner entity's primary key as both foreign key and part of the composite primary key.

4. **Apply correct mapping rules for different cardinality ratios**: 1:1, 1:N, and N:M relationships, selecting appropriate strategies for each case.

5. **Handle special attribute types** including multi-valued attributes (separate relation), composite attributes (decompose), and derived attributes (compute, not store).

6. **Map N-ary relationships** (ternary and higher) to relational schemas by creating junction tables with composite primary keys.

7. **Enforce participation constraints** in the relational model using NOT NULL constraints on foreign keys to represent total participation.

8. **Verify mapping correctness** by ensuring all ER diagram components are represented and relationship constraints are preserved in the relational schema.
