# Equivalence Relations and Partitions - Summary

## Key Definitions and Concepts

- **Equivalence Relation**: A relation R on set A that is reflexive, symmetric, and transitive.
- **Equivalence Class**: [a] = {x ∈ A | xRa}, the set of all elements related to a.
- **Partition**: Collection of non-empty, pairwise disjoint subsets whose union equals the original set.
- **Quotient Set**: A/R = {[a] | a ∈ A}, the set of all equivalence classes.
- **Congruence Modulo n**: a ≡ b (mod n) if n divides (a - b).

## Important Formulas and Theorems

- **Fundamental Theorem**: Equivalence relations ↔ Partitions (bijection)
- **Equivalence Class Properties**:
  - Either [a] = [b] or [a] ∩ [b] = ∅
  - aRb ⟺ [a] = [b]
- **Modular Arithmetic**: [a] + [b] = [a+b], [a] × [b] = [a×b] in ℤ/nℤ

## Key Points

- An equivalence relation partitions its domain into disjoint equivalence classes.
- Every partition defines a unique equivalence relation (and vice versa).
- Congruence modulo n creates n distinct equivalence classes.
- The representative of an equivalence class is arbitrary—any element can represent the class.
- ℤ/nℤ (the integers modulo n) is a fundamental algebraic structure based on equivalence classes.
- Equivalence classes are always non-empty since each contains its representative.

## Common Mistakes to Avoid

- Confusing "symmetric" with "antisymmetric"—they are opposites.
- Forgetting to check all three properties when proving an equivalence relation.
- Assuming equivalence classes must be infinite—they can be finite.
- Incorrectly stating that a ≡ b (mod n) means a/b leaves remainder n (should be remainder less than n).

## Revision Tips

1. Practice proving relations are equivalence relations by checking all three properties.
2. Draw partition diagrams to visualize how equivalence classes split the set.
3. Memorize the Fundamental Theorem—it connects two major concepts.
4. Work through modular arithmetic examples to strengthen understanding of quotient sets.
5. Solve at least 3-4 problems involving both directions: relation → partition and partition → relation.
