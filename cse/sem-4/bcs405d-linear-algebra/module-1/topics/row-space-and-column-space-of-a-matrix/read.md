# Row Space and Column Space of a Matrix

## Introduction

In linear algebra, understanding the structure of a matrix requires more than just performing computations; we must understand the subspaces generated by its rows and columns. The row space and column space of a matrix are fundamental concepts that appear extensively in solving linear systems, understanding linear transformations, and analyzing the dimension of vector spaces. These concepts form the foundation for the rank of a matrix, which is one of the most important quantities in linear algebra.

The row space of a matrix A is the set of all possible linear combinations of its row vectors, forming a subspace of ℝⁿ. Similarly, the column space (also called the range or image) is the set of all linear combinations of its column vectors, forming a subspace of ℝᵐ. Remarkably, despite being defined differently, both spaces share the same dimension—this common dimension is precisely the rank of the matrix. This profound result connects seemingly disparate aspects of matrix theory and has significant practical implications in engineering applications, including control systems, data science, and signal processing.

## Key Concepts

### Definitions

**Row Space**: Given an m×n matrix A, the row space of A, denoted as R(A), is the subspace of ℝⁿ spanned by the row vectors of A. Each row is an n-dimensional vector, and any linear combination of these rows produces another vector in ℝⁿ. The row space captures all information about how the rows of A relate to each other.

**Column Space**: The column space of A, denoted as C(A), is the subspace of ℝᵐ spanned by the column vectors of A. Each column is an m-dimensional vector, and the column space represents all possible linear combinations of these columns. In the context of linear equations Ax = b, the column space determines whether a particular vector b is in the range of A—that is, whether Ax = b has a solution.

### Rank of a Matrix

The **rank** of a matrix A, denoted as rank(A), is defined as the dimension of its row space (or equivalently, the dimension of its column space). This equality is not accidental; it is a fundamental theorem in linear algebra that the row rank always equals the column rank. For an m×n matrix:

- 0 ≤ rank(A) ≤ min(m, n)
- If rank(A) = min(m, n), the matrix is said to have **full rank**
- If rank(A) < min(m, n), the matrix is **rank deficient**

### Finding a Basis for Row Space and Column Space

To find a basis for the row space, we transform the matrix to its **row echelon form** (REF) or **reduced row echelon form** (RREF) using elementary row operations. The non-zero rows in the echelon form form a basis for the row space. Importantly, elementary row operations preserve the row space—they do not change the set of all linear combinations of the rows.

For the column space, we can similarly use elementary column operations, but a more elegant approach exists: after obtaining the RREF, the pivot columns in the original matrix correspond to a basis for the column space. This is because elementary row operations preserve linear dependencies among columns.

### Echelon Forms

A matrix is in **row echelon form** if:

1. All non-zero rows are above any rows of all zeros
2. Each leading entry (first non-zero entry) of a non-zero row is in a column to the right of the leading entry in the row above it
3. All entries in a column below a leading entry are zeros

A matrix is in **reduced row echelon form** if it satisfies the row echelon conditions plus:

1. The leading entry in each non-zero row is 1
2. Each leading 1 is the only non-zero entry in its column

### Relationship Between Row and Column Space

The equality of row rank and column rank is one of the most important theorems in linear algebra. This can be proven using the concept of linear transformations or through elementary matrix operations. As a consequence:

- The dimension of R(A) = dimension of C(A) = rank(A)
- The orthogonal complement of the row space is the null space (for real matrices)
- The orthogonal complement of the column space is the left null space

### Elementary Operations and Space Preservation

Elementary row operations include:

- Row swapping (interchanging two rows)
- Row scaling (multiplying a row by a non-zero scalar)
- Row addition (adding a multiple of one row to another)

These operations do not change:

- The rank of the matrix
- The row space (though the rows themselves change)
- The null space

However, they do change the column space. Conversely, elementary column operations preserve the column space but may change the row space.

## Examples

### Example 1: Finding Row Space Basis

Find a basis for the row space of the matrix:

```
A = [1 2 3]
 [2 4 6]
 [1 1 1]
```

**Solution:**

Step 1: Write the matrix and perform row reduction to REF:

```
[1 2 3] → R₂ ← R₂ - 2R₁ → [1 2 3]
[2 4 6] [0 0 0]
[1 1 1] → R₃ ← R₃ - R₁ → [0 -1 -2]
```

Step 2: Continue to get echelon form:

```
[1 2 3]
[0 -1 -2]
[0 0 0]
```

Swap R₂ and R₃ for proper echelon form:

```
[1 2 3]
[0 -1 -2]
[0 0 0]
```

The non-zero rows are [1, 2, 3] and [0, -1, -2]. These form a basis for the row space.

**Basis for R(A)**: {[1, 2, 3], [0, -1, -2]}
**Dimension (rank)**: 2

### Example 2: Finding Column Space Basis

Find a basis for the column space of:

```
A = [1 2 1]
 [2 4 2]
 [1 1 0]
```

**Solution:**

Step 1: Reduce to RREF:

```
[1 2 1] R₂ ← R₂ - 2R₁ [1 2 1]
[2 4 2] → → [0 0 0]
[1 1 0] R₃ ← R₃ - R₁ [0 -1 -1]

Continuing: R₃ ← -R₃ [1 2 1]
 [0 0 0]
 [0 1 1]

R₁ ← R₁ - 2R₃ [1 0 -1]
 [0 0 0]
 [0 1 1]
```

Step 2: The pivot columns in the RREF are columns 1 and 2. Therefore, the corresponding columns in the original matrix form the basis.

**Basis for C(A)**: {[1, 2, 1]ᵀ, [2, 4, 2]ᵀ} or equivalently {[1, 2, 1]ᵀ, [2, 4, 2]ᵀ}

Actually, let's verify: The pivot columns in RREF are columns 1 and 2. In the original matrix A, these correspond to the first two columns: [1, 2, 1]ᵀ and [2, 4, 2]ᵀ.

**Dimension**: 2

### Example 3: Verifying Rank Equality

For matrix B = [1 2; 2 4], find both row space and column space dimensions.

**Solution:**

The rows are [1, 2] and [2, 4]. Note that [2, 4] = 2 × [1, 2], so only one row is linearly independent.

**Row space dimension**: 1 (basis: {[1, 2]})

The columns are [1, 2]ᵀ and [2, 4]ᵀ. Similarly, [2, 4]ᵀ = 2 × [1, 2]ᵀ.

**Column space dimension**: 1 (basis: {[1, 2]ᵀ})

Thus, rank(B) = 1, confirming row rank = column rank.

## Exam Tips

1. **Remember the fundamental theorem**: Row rank always equals column rank—this is guaranteed and often tested directly.

2. **Elementary row operations preserve rank**: When solving problems about rank, remember that row operations don't change the rank, making them useful for rank computation.

3. **Identify pivots correctly**: In echelon form, the number of non-zero rows equals the rank. Count pivots carefully in RREF.

4. **Basis correspondence**: For column space, the pivot columns in the RREF indicate which original columns form a basis—not the columns in the echelon form.

5. **Full rank condition**: A square matrix is invertible if and only if it has full rank (rank = n).

6. **Dimension relationships**: For an m×n matrix with rank r: nullity (dimension of null space) = n - r. This is the Rank-Nullity Theorem.

7. **Space notation**: Know that R(A) ⊆ ℝⁿ and C(A) ⊆ ℝᵐ—the ambient spaces are different and depend on the matrix dimensions.

8. **Quick rank test**: If you can find r linearly independent rows (or columns), the rank is at least r. Use this for quick estimates.

9. **Zero matrix**: A zero matrix has rank 0—no non-zero rows or columns exist.

10. **Transpose property**: rank(A) = rank(Aᵀ)—the rank remains unchanged under transposition.
