## Introduction

Counting trees is a fundamental topic in graph theory that deals with determining the number of distinct tree structures that can be formed on a given set of vertices. Trees are acyclic connected graphs, and they appear extensively in computer science applications such as network design, hierarchical data structures, and algorithm analysis. The ability to count trees precisely has significant theoretical and practical importance.

In this module, we explore various techniques for counting trees, with particular emphasis on Cayley's Formula, which provides an elegant solution for counting labeled trees. We also examine the Matrix-Tree Theorem, which offers a powerful combinatorial approach using linear algebra. These concepts form the backbone of tree enumeration and have numerous applications in network reliability, phylogenetic analysis, and coding theory.

## Key Concepts

### 1. Basic Definitions

A **tree** is a connected acyclic graph. A **spanning tree** of a graph G is a subgraph that is a tree and contains all vertices of G. A **labeled tree** is a tree where each vertex is assigned a unique label from 1 to n. The number of edges in any tree with n vertices is always n-1.

### 2. Cayley's Formula

**Theorem (Cayley, 1889):** The number of labeled trees on n vertices is n^(n-2).

This remarkable formula states that for any positive integer n ≥ 1, there are exactly n^(n-2) different trees that can be constructed using n distinctly labeled vertices. The formula is surprisingly simple despite the combinatorial complexity of counting all possible tree structures.

**Proof Sketch using Prüfer Sequences:**
Every labeled tree on n vertices can be uniquely encoded by a Prüfer sequence of length n-2. Each position in the sequence can contain any of the n labels, giving n^(n-2) possible sequences. Since there's a bijection between trees and Prüfer sequences, the count follows.

### 3. Prüfer Sequences

A **Prüfer sequence** is a sequence of length n-2 constructed from a labeled tree with n vertices. Given a tree, the Prüfer sequence is generated by repeatedly removing the leaf with the smallest label and recording its neighbor. This creates a unique encoding.

**Key Properties:**

- Every sequence of length n-2 with elements from {1, 2, ..., n} corresponds to a unique labeled tree
- The sequence encodes the degree of each vertex: vertex i appears deg(i) - 1 times in the sequence
- There's a one-to-one correspondence between labeled trees and Prüfer sequences

### 4. Kirchhoff's Matrix-Tree Theorem

**Theorem:** Let G be a connected graph with vertex set {v₁, v₂, ..., vₙ}. Let L be the Laplacian matrix where:

- Lᵢᵢ = degree of vertex vᵢ
- Lᵢⱼ = -1 if vertices vᵢ and vⱼ are adjacent
- Lᵢⱼ = 0 otherwise

Then any cofactor of L (determinant of L with one row and column deleted) equals the number of spanning trees in G.

This theorem provides a computational method to count spanning trees in any graph using determinant calculation.

### 5. Counting Spanning Trees in Special Graphs

**Complete Graph Kₙ:**
The number of spanning trees in Kₙ is n^(n-2), which equals the number of labeled trees (since every labeled tree is a spanning tree of Kₙ).

**Complete Bipartite Graph Kₘ,ₙ:**
The number of spanning trees in Kₘ,ₙ is m^(n-1) × n^(m-1), where the graph has partition sizes m and n.

**Cycle Graph Cₙ:**
The number of spanning trees in Cₙ is n (each edge removal creates a different spanning tree).

### 6. Minimum Spanning Trees

While counting trees considers all possible trees, **Minimum Spanning Trees (MST)** deal with weighted graphs. Algorithms like Prim's, Kruskal's, and Boruvka's are used to find MSTs. The number of MSTs in a graph can be counted using more advanced techniques involving the Tutte polynomial.

## Examples

### Example 1: Applying Cayley's Formula

**Problem:** Find the number of labeled trees on 5 vertices.

**Solution:**
Using Cayley's Formula: Number of labeled trees = n^(n-2) = 5^(5-2) = 5³ = 125

Therefore, there are 125 different labeled trees on 5 vertices.

### Example 2: Constructing Tree from Prüfer Sequence

**Problem:** Given the Prüfer sequence (2, 2, 4) for a tree on 5 vertices, construct the tree.

**Solution:**
**Step 1:** Initialize n = 5, so vertices are {1, 2, 3, 4, 5}. Prüfer sequence: (2, 2, 4)

**Step 2:** Count appearances in sequence:

- Vertex 1: appears 0 times → degree = 1
- Vertex 2: appears 2 times → degree = 3
- Vertex 3: appears 0 times → degree = 1
- Vertex 4: appears 1 time → degree = 2
- Vertex 5: appears 0 times → degree = 1

**Step 3:** Build tree iteratively:

- Smallest leaf = 1, neighbor = 2 (first element). Remove 1, connect 2-1.
- Remaining degrees: 2(2), 3(1), 4(1), 5(1). Smallest leaf = 3, neighbor = 2. Remove 3, connect 2-3.
- Smallest leaf = 4, neighbor = 2. Remove 4, connect 2-4.
- Smallest leaf = 5, neighbor = 4 (last element). Remove 5, connect 4-5.

**Tree edges:** {1-2, 2-3, 2-4, 4-5}

### Example 3: Using Matrix-Tree Theorem

**Problem:** Count spanning trees in the graph below (a triangle with a pendant vertex):

Vertices: A, B, C form a triangle; vertex D attached to A.

**Solution:**
Construct Laplacian matrix L:

```
 A B C D
A [ 3 -1 -1 -1 ]
B [-1 2 -1 0 ]
C [-1 -1 2 0 ]
D [-1 0 0 1 ]
```

Delete row and column D (or any vertex):

```
 A B C
A [ 3 -1 -1 ]
B [-1 2 -1 ]
C [-1 -1 2 ]
```

Calculate determinant:
det = 3(2×2 - (-1)(-1)) - (-1)(-1×2 - (-1)(-1)) + (-1)(-1×(-1) - 2×(-1))
= 3(4-1) - (-1)(-2-1) + (-1)(1+2)
= 3(3) - (-1)(-3) + (-1)(3)
= 9 - 3 - 3 = 3

Number of spanning trees = 3

## Exam Tips

1. **Memorize Cayley's Formula:** Remember n^(n-2) for counting labeled trees on n vertices—this is frequently tested.

2. **Understand Prüfer Sequences:** Know the algorithm to convert between trees and Prüfer sequences; this proves Cayley's formula.

3. **Matrix-Tree Theorem Application:** Be able to construct Laplacian matrices and compute cofactors for counting spanning trees.

4. **Special Graph Formulas:** Remember that spanning trees in Kₙ = n^(n-2), in Kₘ,ₙ = m^(n-1) × n^(m-1), and in Cₙ = n.

5. **Tree Properties:** Always remember that a tree with n vertices has exactly n-1 edges and is acyclic.

6. **Degree from Prüfer:** A vertex with k appearances in the Prüfer sequence has degree k+1.

7. **Verification:** After computing, verify results make sense—counts should be positive integers.
