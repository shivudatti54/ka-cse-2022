Selection Sort and Bubble Sort represent fundamental comparison-based sorting algorithms with O(n²) average-case time complexity. Selection Sort operates by iteratively selecting the minimum element from the unsorted portion and placing it in the sorted region, performing exactly n(n-1)/2 comparisons regardless of input. Its correctness is established through loop invariant methodology, proving that after k iterations, the first k elements are sorted and contain the k smallest values. Bubble Sort progressively exchanges adjacent out-of-order pairs, with larger elements "bubbling" to their final positions. Optimized Bubble Sort achieves O(n) best-case complexity through early termination when no swaps occur. Selection Sort performs exactly n-1 swaps (optimal for comparison sorts), while Bubble Sort may perform up to O(n²) swaps in worst case. Bubble Sort maintains stability (preserving relative order of equal elements), whereas Selection Sort is unstable. Both algorithms require O(1) auxiliary space, making them in-place sorting methods suitable for memory-constrained environments.
