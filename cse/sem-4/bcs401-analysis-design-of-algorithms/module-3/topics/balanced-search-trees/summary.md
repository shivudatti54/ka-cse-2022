# Balanced search trees maintain sorted data with O(log n) operations. AVL trees enforce height difference ≤1 (height < 1.44 log₂n), providing optimal search but more rotations. Red-Black trees use color properties achieving height ≤2 log₂(n+1) with O(1) amortized rotations. B-Trees use multi-way nodes for disk optimization with height ~log\_⌈m/2⌉(n). Selection depends on operation mix: AVL for read-heavy, Red-Black for balanced workloads, B-Trees for disk-based storage.
