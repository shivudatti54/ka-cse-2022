# Decision trees model comparison-based algorithms as binary trees where internal nodes represent comparisons and leaves represent final answers. The key theorem proves that any comparison-based sorting algorithm requires Ω(n log n) worst-case comparisons, using the fact that n! possible orderings require at least ⌈log₂(n!)⌉ comparisons. This bound is tight since algorithms like MergeSort achieve O(n log n). Similarly, binary search achieves the optimal Θ(log n) bound for searching in sorted arrays. These results establish fundamental limits: Ω(n log n) for sorting and element uniqueness, Θ(log n) for binary search. The comparison model excludes algorithms making non-comparison operations (like Counting Sort), which can achieve linear time under certain conditions.
