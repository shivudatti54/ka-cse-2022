# Learning Objectives

After studying this topic, you should students able to:

1. Explain the concept of formal requirement extraction and its importance in UI/UX design
2. Identify and describe various requirement gathering techniques including interviews, questionnaires, observation, and prototyping
3. Classify requirements into functional, non-functional, and domain categories with appropriate examples
4. Create user personas, user scenarios, and use cases for typical design problems
5. Document requirements using appropriate formats including SRS, user requirements document, and user stories
6. Apply requirements prioritization techniques such as MoSCoW to manage project scope
7. Analyze gathered requirements to identify gaps, inconsistencies, and missing information
8. Understand the connection between formal requirements and successful user-centered design outcomes
