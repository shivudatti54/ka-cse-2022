# Learning Objectives

After studying this topic, you should students able to:

1. Distinguish between user needs and requirements, and explain why this distinction is important in UI/UX design.

2. Identify and describe various user requirements gathering methods suitable for UI/UX projects.

3. Classify requirements into functional, non-functional, user, and system requirements with appropriate examples.

4. Create user personas based on research data to represent target user groups.

5. Write user stories in the standard format and convert them into actionable requirements.

6. Understand the components and purpose of requirements documentation in UI/UX projects.

7. Apply requirements gathering techniques to given scenarios and produce structured requirement specifications.
