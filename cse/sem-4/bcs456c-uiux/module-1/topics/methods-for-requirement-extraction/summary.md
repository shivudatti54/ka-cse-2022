# Methods for Requirement Extraction in UI/UX Design - Summary

## Key Definitions and Concepts

- **Requirement Extraction**: The process of gathering, analyzing, and documenting user needs, business goals, and technical constraints to inform design decisions.

- **Qualitative Methods**: Research approaches that provide in-depth, descriptive insights through direct user engagement (interviews, observations, focus groups).

- **Quantitative Methods**: Research approaches that collect numerical data from large samples enabling statistical analysis (surveys, analytics).

- **Contextual Inquiry**: Observing users in their natural environment while performing tasks to understand real-world context and behaviors.

- **Card Sorting**: A participatory technique where users organize content items into meaningful categories revealing mental models.

- **Task Analysis**: Breaking down user tasks into detailed steps to understand workflows and cognitive processes.

- **Mixed Methods Approach**: Combining qualitative and quantitative methods for comprehensive user understanding.

## Important Formulas and Techniques

- **User Interview Structure**: Preparation → Introduction → Main Questions → Probing → Conclusion → Analysis
- **Survey Design Principles**: Clear objectives → Simple language → Logical flow → Pre-testing → Proper distribution
- **Card Sorting Analysis**: Frequency matrix → Cluster analysis → Dendrogram creation → Information architecture mapping

## Key Points

1. Requirement extraction forms the foundation of user-centered design, determining project success or failure.

2. User interviews provide rich qualitative data through direct conversation but are time-intensive and have limited sample sizes.

3. Surveys efficiently collect quantitative data from large populations but lack depth in exploring complex behaviors.

4. Contextual inquiry reveals actual user behavior in natural settings versus self-reported behavior in interviews.

5. Card sorting directly informs information architecture by revealing user mental models for content organization.

6. Task analysis identifies workflow inefficiencies and cognitive load, guiding interaction design decisions.

7. Competitive analysis provides market context but should not lead to mere imitation of competitors.

8. Stakeholder input balances user desires with business feasibility and technical constraints.

9. Mixed methods approach combining multiple techniques yields most comprehensive understanding.

10. Research findings must be systematically analyzed and translated into actionable design requirements.

## Common Mistakes to Avoid

- **Relying solely on one method**: Each technique has blind spots; multiple methods provide triangulation and validation.
- **Skipping stakeholder interviews**: Focusing only on users while ignoring business requirements leads to impractical designs.
- **Collecting data without clear objectives**: Research without defined goals produces irrelevant or overwhelming information.
- **Leading questions in interviews**: Biased questions skew responses and invalidate findings.
- **Ignoring edge cases**: Focusing only on majority user patterns may exclude important user segments.
