# Extracting Interaction Design Requirements & Needs - Summary

## Key Definitions and Concepts

- **User Needs**: Requirements that users have for an interactive product, categorized into functional (what the product must do), emotional (how users should feel), and social (how it affects relationships) needs

- **Design Requirements**: Measurable, specific criteria derived from user needs that the design must satisfy

- **Personas**: Fictional representations of target users created from research data, encapsulating characteristics, goals, and pain points

- **User Scenarios**: Narrative descriptions of how users achieve specific goals using a product

- **Task Analysis**: Process of breaking down user tasks into component steps to understand interaction requirements

- **Stakeholders**: Individuals or groups with interest in the product, categorized as primary, secondary, and tertiary

## Important Formulas and Theorems

- **User Story Format**: "As a [user type], I want [goal] so that [benefit]"
- **SMARTER Requirements**: Specific, Measurable, Achievable, Relevant, Time-bound, Extensible, Reviewable

## Key Points

- Requirement extraction forms the foundation for successful interaction design
- User research methods should be selected based on project needs (qualitative vs. quantitative)
- Personas help maintain user focus throughout the design process
- Scenarios bridge the gap between user goals and design specifications
- Task analysis reveals cognitive and physical activities in task completion
- Both functional and non-functional requirements must be considered
- Stakeholder analysis ensures diverse perspectives are included
- Multiple research methods should be triangulated for comprehensive understanding

## Common Mistakes to Avoid

- Confusing user needs with design requirements (needs are what users want; requirements are measurable specifications)
- Creating personas without sufficient user research data
- Designing for "everyone" instead of specific user segments
- Relying solely on reported behavior instead of observing actual behavior
- Focusing only on functional requirements and ignoring emotional/social needs

## Revision Tips

1. Practice writing user stories in the standard format with various scenarios
2. Create sample personas from given user research data
3. Remember the three levels of user needs and be able to provide examples of each
4. Know the appropriate use cases for each user research method
5. Review task analysis techniques and be able to apply them to simple scenarios
6. Understand stakeholder categories and practice identifying stakeholders for given products
