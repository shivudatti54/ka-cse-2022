# Learning Objectives

After studying this topic, you should students able to:

1. Explain the importance of extracting interaction design requirements and needs in the UI/UX design process

2. Differentiate between user needs (functional, emotional, and social) and design requirements with appropriate examples

3. Apply various user research methods (interviews, surveys, observations, contextual inquiry, focus groups) to gather user requirements

4. Create effective personas that represent target users based on research data

5. Develop user scenarios that describe how users accomplish specific goals with interactive systems

6. Conduct task analysis to break down user tasks into component steps and understand interaction requirements

7. Write user stories and use cases to communicate design requirements clearly to development teams

8. Identify and analyze stakeholders to ensure diverse perspectives are considered in requirement extraction
