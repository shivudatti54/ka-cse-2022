# UX Measures - Summary

## Key Definitions and Concepts

- **Usability (ISO 9241-11)**: The extent to which a product enables users to achieve goals with effectiveness, efficiency, and satisfaction in a specified context.
- **Heuristic Evaluation**: A discount usability inspection method where evaluators examine interfaces against established usability principles.
- **SUS (System Usability Scale)**: A 10-item questionnaire providing a score from 0-100; scores above 68 indicate above-average usability.
- **Effectiveness**: Accuracy and completeness of goal achievement (measured by task success rate).
- **Efficiency**: Resources expended relative to results achieved (measured by time on task).
- **Satisfaction**: User's subjective response (measured by questionnaires like SUS, CSAT, NPS).

## Important Formulas and Theorems

- **Task Success Rate** = (Successful completions / Total attempts) × 100
- **Error Rate** = Total errors / Total tasks
- **Efficiency Ratio** = Optimal clicks or steps / Actual clicks or steps
- **SUS Score** = (Sum of odd items - 5) × 2.5 + (25 - Sum of even items) × 2.5
- **Nielsen's 10 Heuristics**: Visibility of status, Match with real world, User control, Consistency, Error prevention, Recognition over recall, Flexibility, Minimalist design, Error recovery, Help

## Key Points

- UX measurement combines quantitative metrics with qualitative insights for comprehensive evaluation
- Nielsen's heuristics provide a systematic framework for identifying usability problems
- SUS is the industry standard for quick usability assessment with a benchmark of 68 points
- A/B testing enables data-driven design decisions through controlled experiments
- Task-based metrics (success rate, time, errors) directly measure user performance
- Self-reported metrics capture user perception and satisfaction
- Formative evaluation occurs during design; summative evaluation occurs post-deployment
- Thinking aloud protocol reveals user mental models and emotional responses

## Common Mistakes to Avoid

1. **Confusing usability with UX**: Usability is about task performance; UX is broader, including emotion and experience.
2. **Relying solely on one method**: Combining multiple methods provides more comprehensive insights.
3. **Ignoring sample size**: Small sample sizes in usability testing may not represent real user behavior.
4. **Misinterpreting SUS scores**: Remember that SUS scores above 68 are above average, not necessarily "good."

## Revision Tips

1. Memorize Nielsen's 10 heuristics with concrete examples of each violation
2. Practice calculating SUS scores from sample questionnaire responses
3. Review the differences between qualitative and quantitative UX methods
4. Understand when to apply formative vs. summative evaluation in development cycles
5. Be able to design a basic usability test with appropriate tasks and metrics
