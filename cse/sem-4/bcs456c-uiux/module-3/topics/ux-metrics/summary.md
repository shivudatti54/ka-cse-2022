# UX Metrics - Summary

## Key Definitions and Concepts

- **UX Metrics**: Quantifiable measures used to evaluate the quality of user interactions with a product or system
- **Behavioral Metrics**: Objective measurements of user actions (clicks, time spent, task completion)
- **Attitudinal Metrics**: Subjective measurements of user perceptions (satisfaction, trust, loyalty)
- **Composite Metrics**: Combined measurements providing holistic scores (SUS, NPS)
- **Usability**: The degree to which a product can be used by specified users to achieve specified goals effectively, efficiently, and with satisfaction

## Important Formulas and Theorems

- **SUS Score** = (Sum of converted responses) × 2.5; Scores above 68 are above average
- **NPS** = % Promoters (9-10) - % Detractors (0-6); Range: -100 to +100
- **CSAT** = (Satisfied customers / Total responses) × 100
- **Task Completion Rate** = (Successful completions / Total attempts) × 100
- **Conversion Rate** = (Conversions / Total visitors) × 100

## Key Points

- UX metrics provide objective, data-driven insights for design decisions
- SUS is the industry standard for perceived usability with scores above 80 indicating excellent usability
- NPS measures customer loyalty through recommendation likelihood
- TIME metrics evaluate task performance across Time, Errors, Efficiency, and Memorability
- Nielsen's ten heuristics form the foundation for heuristic evaluations
- Behavioral and attitudinal metrics should be used together for comprehensive evaluation
- Business metrics like conversion rate connect UX to organizational outcomes
- WCAG provides accessibility compliance standards with three levels (A, AA, AAA)

## Common Mistakes to Avoid

- Confusing NPS with CSAT—they measure different things (loyalty vs. satisfaction)
- Interpreting SUS scores the same as NPS—SUS is about usability, NPS is about loyalty
- Relying solely on quantitative metrics without qualitative context
- Using the same metrics for all evaluation contexts—different situations require different measures
- Ignoring accessibility metrics—WCAG compliance is essential for inclusive design

## Revision Tips

- Practice calculating SUS and NPS with sample data to ensure you understand the formulas
- Memorize Nielsen's ten heuristics as they frequently appear in exams
- Create a comparison chart of when to use each metric type (behavioral vs. attitudinal)
- Remember that SUS scores multiply by 2.5 to get the final 0-100 score
- Focus on understanding interpretation thresholds (e.g., SUS > 68 is above average)
