# Prototyping - Summary

## Key Definitions and Concepts

- **Prototyping**: Creating preliminary versions of a product to test and validate design decisions before full development
- **Low-Fidelity Prototype**: Simple representations focusing on structure and layout (paper prototypes, wireframes)
- **High-Fidelity Prototype**: Polished representations with visual details and interactivity resembling the final product
- **Medium-Fidelity Prototype**: Balance between low and high fidelity with basic visuals and limited interactivity
- **Wizard of Oz Prototyping**: Using human operators to simulate automated functionality
- **Clickable Prototypes**: Interactive mockups allowing navigation through clicking elements

## Important Formulas and Theorems

Prototyping does not involve complex formulas, but understanding these principles is essential:

- **Iterative Design Cycle**: Prototype → Test → Analyze → Redesign → Repeat
- **Fidelity Trade-off**: Higher fidelity provides more realistic feedback but requires more time and investment to create and modify

## Key Points

- Prototyping bridges the gap between abstract concepts and functional products
- Low-fidelity prototypes are faster to create and encourage creative thinking
- High-fidelity prototypes are better for stakeholder presentations and final user testing
- Paper prototyping remains valuable for rapid ideation and early feedback
- Modern tools like Figma and Adobe XD enable rapid interactive prototype creation
- Prototype testing should involve representative users completing realistic tasks
- Think-aloud protocols provide valuable insights into user mental models
- Prototyping reduces development costs by identifying issues early
- The choice of prototype fidelity depends on project stage, goals, and resources

## Common Mistakes to Avoid

1. **Testing too late**: Waiting until the prototype is too polished before gathering feedback defeats the purpose of prototyping
2. **Over-investing in prototypes**: Remember that prototypes are meant to be modified and discarded, not preserved as final artifacts
3. **Testing with wrong users**: Using colleagues or stakeholders instead of target users provides misleading feedback
4. **Ignoring negative feedback**: Negative user reactions are valuable learning opportunities that should inform design iterations

## Revision Tips

1. Create a comparison table of prototype types (low/medium/high fidelity) with examples, advantages, and disadvantages
2. Practice creating quick paper prototypes for common app interfaces to reinforce learning
3. Review case studies of successful prototyping implementations in real products
4. Memorize the iterative prototyping process: Design → Build → Test → Learn → Repeat
5. Understand that the goal of prototyping is learning, not creating perfect designs
