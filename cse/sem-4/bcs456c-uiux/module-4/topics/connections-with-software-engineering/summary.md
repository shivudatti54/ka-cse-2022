# Connections with Software Engineering - Summary

## Key Definitions and Concepts

- **Design-Development Handoff**: The process of transferring design specifications, mockups, and assets from the design team to the development team, including style guides, component specifications, and interaction details.

- **Design System**: A comprehensive collection of design principles, UI components, pattern libraries, and documentation that ensures consistency between design and implementation.

- **Design Tokens**: Visual design attributes (colors, typography, spacing) stored as variables that can be shared between design tools and code implementations.

- **Responsive Design**: An approach to web design that makes pages render well across different devices and screen sizes through flexible layouts, media queries, and adaptive content.

- **Accessibility (WCAG)**: Web Content Accessibility Guidelines that ensure digital products are usable by people with diverse abilities, requiring both design and technical implementation.

## Important Formulas and Theorems

While this topic is primarily conceptual rather than formula-based, key frameworks include:

- **Agile UX Integration Model**: Distribution of UX activities (research, design, testing) throughout Agile sprints rather than in isolated phases.

- **Design Feasibility Assessment**: Evaluating design proposals against technical constraints (performance, compatibility, platform capabilities) before finalization.

## Key Points

- Design decisions directly impact engineering implementation complexity, development timelines, and code maintainability.

- Effective design handoffs require comprehensive documentation including style guides, component specs, and interactive prototypes.

- Design systems create a shared language between designers and engineers through reusable components implemented in both design tools and code.

- Agile methodologies integrate UX activities throughout the development process rather than treating design as a separate phase.

- Technical constraints including browser support, performance requirements, and platform guidelines must inform design decisions.

- Accessibility compliance requires collaboration between design (color contrast, typography) and engineering (semantic HTML, ARIA, keyboard navigation).

- Responsive design implementation requires coordinated efforts between designers creating breakpoint layouts and developers implementing flexible code.

- Modern tools like Figma enable developers to inspect CSS properties and export assets directly, streamlining the design-to-code workflow.

## Common Mistakes to Avoid

- Treating design and development as separate silos rather than integrated collaborative activities.

- Creating designs without considering technical feasibility, leading to implementation difficulties and scope changes.

- Neglecting accessibility considerations during design, which results in expensive retrofitting during development.

- Incomplete design handoffs lacking essential specifications for spacing, typography, or interaction behaviors.

- Assuming design approval is final without ongoing collaboration throughout implementation.

## Revision Tips

- Review the design-development workflow diagram and understand each handoff stage.

- Study real examples of design system implementations like Material Design or Carbon Design System.

- Practice identifying technical constraints that would affect given design proposals.

- Memorize key accessibility requirements (WCAG contrast ratios, semantic HTML, ARIA roles).

- Understand how Agile ceremonies (sprint planning, stand-ups, retrospectives) include UX activities.

- Review how responsive design uses media queries and flexible grids from both design and code perspectives.
