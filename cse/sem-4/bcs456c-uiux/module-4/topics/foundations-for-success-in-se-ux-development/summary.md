# Foundations for Success in SE-UX Development - Summary

## Key Definitions and Concepts

- **User-Centered Design (UCD)**: A design philosophy placing end users at the center of all design decisions through iterative analysis, design, evaluation, and refinement cycles.
- **Design Thinking**: A five-stage problem-solving methodology (Empathize, Define, Ideate, Prototype, Test) combining empathy, creativity, and rationality.
- **Information Architecture**: The structural design of information spaces, including organization systems, labeling, navigation, and search functionality.
- **Wireframe**: A low-fidelity static representation showing basic layout and content placement without detailed styling.
- **Prototype**: An interactive representation simulating user interactions, ranging from low to high fidelity.
- **WCAG**: Web Content Accessibility Guidelines with four principles - Perceivable, Operable, Understandable, Robust.

## Important Formulas and Theorems

- **Nielsen's 10 Usability Heuristics**: Evaluation framework for assessing UI designs covering visibility, match to real world, user control, consistency, error prevention, recognition, flexibility, aesthetic design, error recovery, and help.
- **Design Thinking Cycle**: Empathize → Define → Ideate → Prototype → Test (iterative non-linear process)
- **UCD Process**: Understand context of use → Specify user requirements → Design solutions → Evaluate against requirements → Iterate

## Key Points

- SE-UX integration is essential - technically sound products fail without good user experience.
- User research combines qualitative (interviews, observations) and quantitative (surveys, analytics) methods.
- Wireframes focus on structure; prototypes focus on interaction and feedback.
- Accessibility (WCAG) benefits all users and is often legally required.
- Agile UX integrates design into sprint-based development through continuous, incremental deliverables.
- Effective Information Architecture reduces cognitive load and enables efficient task completion.
- Usability testing with actual users is crucial - assumptions about user needs must be validated empirically.

## Common Mistakes to Avoid

- Designing based on assumptions rather than user research findings.
- Skipping low-fidelity prototyping and jumping directly to high-fidelity designs.
- Treating accessibility as an afterthought instead of integrating it from the beginning.
- Confusing wireframes (static layouts) with prototypes (interactive simulations).
- Focusing only on visual design while ignoring information architecture and navigation.

## Revision Tips

1. Practice evaluating interfaces using Nielsen's heuristics - this is frequently tested in university exams.
2. Memorize the five stages of Design Thinking in order and understand when each is applied.
3. Know the difference between qualitative and quantitative research methods and their appropriate use cases.
4. Review WCAG's four principles (POUR) and remember that accessibility benefits all users.
5. Understand how UX integrates into Agile - work in iterations, continuous feedback, incremental delivery.
