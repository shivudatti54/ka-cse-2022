# Design Influencing User Behaviour - Summary

## Key Definitions and Concepts

- **Persuasive Design**: Creating digital experiences that encourage specific user actions through thoughtful design choices
- **Dark Patterns**: Deceptive design techniques that trick users into unintended actions
- **Nudge Theory**: Using positive reinforcement to influence behaviour while maintaining freedom of choice
- **Cognitive Biases**: Systematic patterns affecting user judgment and decision-making
- **Visual Hierarchy**: Arrangement of elements to guide user attention in order of importance
- **Micro-interactions**: Small, focused moments that provide feedback during user interactions

## Important Formulas and Theorems

- **Hick's Law**: Decision time increases with number/complexity of choices
- **Fitts's Law**: Target acquisition time depends on distance and size of target
- **Miller's Law**: Working memory holds approximately 7 (±2) items
- **Gestalt Principles**: Similarity, proximity, continuity, and closure in visual perception

## Key Points

- Design elements serve as powerful triggers for user behaviour through psychological principles
- Ethical persuasion aligns user goals with business objectives; dark patterns deceive and damage trust
- Cognitive biases like scarcity, social proof, and anchoring significantly influence decisions
- Visual hierarchy guides attention through size, color, contrast, whitespace, and typography
- Micro-interactions provide immediate feedback and reinforce desired behaviours
- Effective CTAs use action-oriented language, strategic placement, and prominent visual design
- Nudges guide users toward beneficial actions without restricting choice
- Color psychology affects emotional responses and can influence action (red for urgency, green for success)
- Reducing cognitive load improves user experience and increases conversion rates
- Feedback and confirmation are essential for shaping behaviour and reducing user uncertainty

## Common Mistakes to Avoid

- Confusing ethical persuasion with manipulative dark patterns
- Overloading users with too many choices (violating Hick's Law)
- Using deceptive CTAs or misleading language
- Ignoring accessibility when implementing behaviour-influencing elements
- Applying techniques without considering ethical implications

## Revision Tips

1. Create a mind map connecting psychological principles to specific design applications
2. Practice identifying dark patterns in real websites and apps
3. Memorize key laws (Hick, Fitts, Miller) and their practical implications
4. Review at least 3 examples of behaviour-influencing design for each principle
5. Focus on the balance between user benefit and business goals in ethical design
