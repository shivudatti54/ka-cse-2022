# Storyboards in UI/UX Design - Summary

## Key Definitions and Concepts

- **Storyboard**: A visual sequence of panels depicting how users interact with a digital product, showing screens, user actions, and system responses in narrative form.
- **User Flow**: The path a user follows through an application to accomplish a specific goal.
- **Panel/Frame**: Individual screen or moment captured in a storyboard sequence.
- **Happy Path**: The ideal user journey without errors or exceptions.

## Important Formulas and Theorems

There are no specific formulas in storyboarding, but the structure typically follows:

- **Panel Content = Screen View + User Action + System Response + Annotations**
- **Storyboard Quality = Clarity + Completeness + Accessibility + Iteration**

## Key Points

- Storyboarding originated in the film industry (1930s, Disney Studios) and was adapted for UX design.
- Storyboards differ from wireframes: storyboards show sequential flows while wireframes show static layouts.
- Four main types: Linear (sequential), Non-linear (branching), Thumbnail (rough sketches), and Detailed (polished).
- Each panel should include: screen representation, user action, system response, and annotations/notes.
- Storyboards are created in the early design phases to visualize user journeys before detailed implementation.
- They serve as effective communication tools between designers, developers, and stakeholders.
- Good storyboards include both happy paths and alternative/error flows.
- Storyboarding is an iterative process that benefits from feedback and refinement.
- Tools range from pen and paper for thumbnails to digital tools like Figma, Balsamiq, or Marvel for detailed versions.

## Common Mistakes to Avoid

- Creating storyboards that are too detailed too early in the design process; start with rough sketches.
- Focusing only on the happy path and ignoring error conditions and alternative flows.
- Including too much text in panels; storyboards should be visual and easy to understand.
- Skipping the iteration phase; storyboards should evolve based on team feedback.

## Revision Tips

- Practice creating simple storyboards for common app flows like login, registration, or checkout.
- Remember the key elements of each panel: screen, action, response, annotations.
- Review the differences between storyboard types and know when to apply each.
- Study the relationship between storyboards and other UX deliverables like user personas and journey maps.
- Focus on understanding why storyboards matter in the design process, not just how to create them.
