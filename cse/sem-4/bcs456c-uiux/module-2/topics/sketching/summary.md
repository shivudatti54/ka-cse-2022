# Sketching in UI/UX Design - Summary

## Key Definitions and Concepts

- **Sketching**: A rapid, low-fidelity visualization technique used to explore, communicate, and iterate on design ideas before committing to digital implementation
- **Thumbnail Sketches**: Small, quick drawings used for rapid ideation and exploring multiple layout variations
- **Wireframe Sketches**: Detailed layout drawings showing spatial arrangement of UI elements without aesthetic details
- **High-Fidelity Sketches**: Detailed drawings incorporating visual elements closer to final design
- **Design Studio Methodology**: Collaborative sketching approach involving silent individual work, group critique, and selection

## Important Formulas and Principles

- **Convergent-Divergent Thinking**: Explore many ideas (divergent) first, then select and refine (convergent)
- **Fidelity Spectrum**: Thumbnail → Wireframe → High-fidelity → Digital mockup
- **Communication over Aesthetics**: Sketches prioritize clear communication over visual beauty

## Key Points

- Sketching enables rapid iteration and "failing fast to succeed sooner"
- Different sketch types serve different purposes in the design process
- Sticky notes are valuable for information architecture and user flow visualization
- The design studio methodology democratizes design through collaborative sketching
- Low-fidelity sketches encourage candid feedback by avoiding "precious" designs
- Standard conventions exist for representing common UI elements (mountains for images, lines for text)
- Digital sketching tools offer undo functionality and easy sharing compared to paper
- Sketching bridges abstract thinking and concrete design solutions

## Common Mistakes to Avoid

- Spending too much time on individual sketches instead of exploring multiple variations
- Creating overly detailed sketches too early in the process before user validation
- Confusing sketching with detailed visual design—sketches should communicate structure and function
- Using sketching solely for final presentations instead of as an ideation tool

## Revision Tips

1. Practice creating thumbnail sketches for common app screens (login, profile, settings) to build speed
2. Remember the fidelity progression: idea → sketch → wireframe → prototype → product
3. Review the design studio methodology steps before exams
4. Focus on understanding when to use each sketch type rather than memorizing definitions
5. Remember that sketching is a communication tool—the audience should understand the sketch without explanation
