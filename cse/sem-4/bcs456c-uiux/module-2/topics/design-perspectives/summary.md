# Design Perspectives in UI/UX - Summary

## Key Definitions and Concepts

- **Design Perspectives**: Different approaches and viewpoints that designers adopt when creating user interfaces and experiences, including user-centered, activity-centered, systems, and goal-directed design.

- **User-Centered Design (UCD)**: Design approach placing users at the center, involving them throughout the design process to create intuitive and satisfying products.

- **Activity-Centered Design (ACD)**: Focuses on the activities users perform rather than users themselves, designing tools that support specific tasks and workflows.

- **Goal-Directed Design (GDD)**: Focuses on designing for user goals, using personas to represent distinct user types and creating experiences that help achieve objectives efficiently.

- **Five Dimensions (Garrett)**: Strategy → Scope → Structure → Skeleton → Surface—five interconnected planes that guide comprehensive UX design.

## Important Formulas and Theorems

- **GDD Process**: Research → Modeling → Requirements Definition → Framework Definition
- **UCD Cycle**: Understand → Specify → Design → Prototype → Test → Iterate
- **Five Dimensions Framework**:
  - Strategy Plane: Business objectives + User needs
  - Scope Plane: Functional requirements + Content needs
  - Structure Plane: Information architecture + Interaction design
  - Skeleton Plane: Interface + Navigation + Information design
  - Surface Plane: Visual + Audio + Haptic design

## Key Points

1. Multiple design perspectives exist to address different aspects of the design problem.
2. UCD emphasizes empathy and user involvement throughout development.
3. ACD is ideal for complex systems with structured, goal-oriented tasks.
4. Systems perspective considers organizational context and technical integration.
5. GDD creates emotional resonance by designing for user motivations.
6. Garrett's five dimensions provide a structured UX design framework.
7. Stakeholder perspectives (business, technical, marketing, legal) must be balanced.
8. Modern design often integrates multiple perspectives for optimal results.
9. Each perspective has specific techniques: personas, journey maps, wireframes, card sorting.
10. The choice of perspective depends on project type, user base, and business goals.

## Common Mistakes to Avoid

1. Confusing user-centered design with activity-centered design—remember users vs. activities as the focus.
2. Ignoring technical feasibility when advocating for ideal user experiences.
3. Overlooking stakeholder perspectives other than users in the design process.
4. Treating design perspectives as mutually exclusive rather than complementary approaches.
5. Memorizing frameworks without understanding their practical application in real scenarios.

## Revision Tips

1. Create a comparative table distinguishing all four major design perspectives.
2. Practice mapping Garrett's five dimensions to actual UI/UX project examples.
3. Review past university exam questions on design perspectives to understand the question patterns.
4. Memorize the GDD and UCD process cycles in correct sequence.
5. Prepare short notes on stakeholder perspectives and why balancing them matters.
6. Understand the relationships between different perspectives rather than studying them in isolation.
