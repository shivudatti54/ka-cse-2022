# Mental Models and Conceptual Design - Summary

## Key Definitions and Concepts

- **Mental Model**: An internal mental representation users create about how a system works, based on their beliefs, experiences, and understanding of similar systems
- **Design Model**: The conceptual framework built into a product by designers, representing how the system is intended to function
- **System Image**: The visible interface, documentation, and feedback that users interact with, which shapes their mental model
- **Mapping**: The relationship between design controls and their effects in the real world
- **Conceptual Design**: High-level design that defines core concepts, user goals, functional overview, and information hierarchy
- **Card Sorting**: A research method where users organize content items into categories to reveal their mental models
- **User Flow**: A visualization of the path users take to complete specific tasks
- **Persona**: A fictional user profile created from research to represent a typical user type

## Important Formulas and Theorems

- **Mental Model Alignment**: Success = Design Model ≈ User's Mental Model (the smaller the gap, the more intuitive the interface)
- **Five Characteristics**: Mental models are Incomplete, Unstable, Imprecise, Personal, and Adaptive

## Key Points

- Mental models are shaped by previous experience, analogical reasoning, cultural norms, and system feedback
- The gap between design model and mental model determines user interface intuitiveness
- Good mapping uses natural relationships where controls correspond intuitively to their effects
- Card sorting (open and closed) helps discover how users naturally organize information
- Conceptual design precedes UI design and establishes the foundation for the product structure
- Personas are derived from user research and represent distinct user types with specific goals
- User flows help identify potential friction points in user journeys before detailed design begins

## Common Mistakes to Avoid

- Assuming all users share the same mental model without conducting user research
- Confusing conceptual design (high-level structure) with visual design (UI details)
- Overlooking the importance of system image in shaping user mental models
- Designing features based on assumptions rather than validated user needs

## Revision Tips

1. Remember the five characteristics of mental models using the acronym I-U-P-A (Incomplete, Unstable, Precise—note: this should be "Imprecise," so use I-U-I-P-A instead)
2. Practice identifying mental models in everyday interfaces (email, social media, banking apps)
3. Review card sorting methodology as it's frequently tested in exams
4. Focus on the relationship between design model, mental model, and system image
5. Be able to sketch a simple user flow for a common application scenario
