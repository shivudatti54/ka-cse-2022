# Mental Models and Conceptual Design - Storyboards - Summary

## Key Definitions and Concepts

- **Mental Model**: A psychological representation of how a user believes a system works, based on past experiences and expectations. Mental models shape how users interact with interfaces and determine their expectations.

- **Conceptual Design**: The high-level design phase where designers translate user research findings into product concepts, including features, information architecture, and user flows.

- **Storyboard**: A visual sequence of panels depicting a user's journey through a product, showing actions, interface elements, thoughts, and emotions at each stage.

## Important Formulas and Theorems

Donald Norman's Gulfs:

- **Gulf of Execution**: The gap between user goals and system capabilities
- **Gulf of Evaluation**: The gap between system representation and user understanding

These gulfs represent the friction users experience when interfaces don't align with their mental models.

## Key Points

- Mental models are personal and vary based on user background, culture, and experience
- Designers must align interfaces with user mental models rather than forcing users to learn new paradigms
- Conceptual design bridges user research and detailed UI design
- Effective storyboards include context, user actions, interface elements, thoughts, and emotions
- Storyboards help identify pain points before development, saving time and resources
- Empathy storyboards focus on emotional journey, helping create emotionally resonant experiences
- The conceptual design process includes concept development, feature prioritization, and user flow mapping
- Storyboarding facilitates communication between designers, developers, and stakeholders
- Understanding mental models reduces learning curves and increases user satisfaction
- Iterative refinement based on storyboard feedback improves overall design quality

## Common Mistakes to Avoid

1. **Assuming universal mental models**: Not all users share the same mental models; designs should accommodate diverse user perspectives.

2. **Skipping conceptual design**: Jumping directly to UI design without proper conceptual planning often leads to inconsistent and confusing products.

3. **Overcomplicating storyboards**: Including too much detail in storyboard panels defeats the purpose of quick visualization and communication.

4. **Ignoring emotional aspects**: Focusing only on functional aspects while neglecting user emotions misses crucial aspects of user experience.

## Revision Tips

1. Create your own mental model diagram for a familiar application, mapping how you think it works versus how it actually functions.

2. Practice sketching simple storyboards for common user scenarios like logging into an app or making an online purchase.

3. Review Donald Norman's fundamental concepts of execution and evaluation gulfs as these frequently appear in exams.

4. Prepare 2-3 examples of how mental model misalignment causes user frustration in popular applications.

5. Understand the sequence: User Research → Mental Model Analysis → Conceptual Design → Storyboarding → Detailed Design.
