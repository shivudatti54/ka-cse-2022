# UX Design Guidelines - Summary

## Key Definitions and Concepts

- **UX Design Guidelines**: Established principles, standards, and best practices for creating user-friendly interfaces
- **Fitts' Law**: Time to acquire a target increases with distance and decreases with target size (T = a + b log2(2D/W))
- **Hicks' Law**: Decision time increases with the number and complexity of choices
- **Miller's Law**: People can hold approximately 7 (±2) items in working memory
- **Affordance**: Perceived properties of an object that suggest how it can be used
- **Mapping**: Relationship between controls and their effects in the interface

## Important Formulas and Theorems

- **Fitts' Law Formula**: T = a + b log2(2D/W), where T = time, D = distance, W = width of target
- **Gestalt Principles**: Proximity, Similarity, Continuity, Closure, Figure-Ground
- **Nielsen's 10 Heuristics**: Visibility of status, Match with real world, User control, Consistency, Error prevention, Recognition vs recall, Flexibility, Minimalist design, Error recovery, Help

## Key Points

- Design for touch targets minimum 44px height on mobile devices
- Follow platform conventions for consistency (iOS vs Android patterns)
- WCAG requires minimum 4.5:1 color contrast ratio for normal text
- Feedback should be immediate, clear, and appropriate to the action
- Group related information using Gestalt's proximity principle
- Error messages must be constructive and suggest solutions
- Reduce cognitive load by chunking information into 5-9 items
- External consistency builds on users' prior experience with similar products

## Common Mistakes to Avoid

1. **Ignoring Accessibility**: Failing to provide keyboard navigation, text alternatives, or sufficient color contrast
2. **Inconsistent Design**: Using different terminology, colors, or interaction patterns across screens
3. **Poor Error Handling**: Displaying cryptic error codes without explaining what went wrong
4. **Excessive Cognitive Load**: Presenting too many options or too much information at once

## Revision Tips

1. Create a checklist of Nielsen's 10 heuristics and practice evaluating sample interfaces against each one
2. Memorize the WCAG four principles (POUR) and at least two success criteria under each
3. For Gestalt principles, find real UI examples in apps you use daily and identify how each principle is applied
4. Practice explaining Fitts' Law and Hicks' Law in the context of mobile design specifically
5. When answering exam questions, always support your design recommendations with specific guideline references
