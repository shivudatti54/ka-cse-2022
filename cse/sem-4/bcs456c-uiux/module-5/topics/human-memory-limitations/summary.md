# Human Memory Limitations in UI/UX Design - Summary

## Key Definitions and Concepts

- **Sensory Memory**: Brief storage of sensory information lasting fractions of a second; forms first impression of interface
- **Short-term/Working Memory**: Active processing memory holding 5-9 items (7±2) for 15-30 seconds without rehearsal
- **Long-term Memory**: Nearly unlimited capacity storage requiring meaningful encoding for retrieval
- **Miller's Law**: The principle that human short-term memory can hold approximately 7 (±2) items
- **Cognitive Load**: Total mental effort required to process information; categorized as intrinsic, extraneous, and germane
- **Chunking**: Grouping information into meaningful units to increase effective memory capacity
- **Recognition vs. Recall**: Users can recognize previously seen items more easily than generating them from memory

## Important Formulas and Theorems

- **Miller's Law**: Short-term memory capacity = 7 ± 2 items
- **Cognitive Load Theory**: Total Load = Intrinsic Load + Extraneous Load + Germane Load
- **Memory Decay**: Information in short-term memory decays within 15-30 seconds without rehearsal

## Key Points

1. Short-term memory has severe capacity limits—design interfaces that present 5-9 options at a time
2. Group related items using chunking to work within memory constraints
3. Minimize extraneous cognitive load through clear layouts, consistent patterns, and inline help
4. Design for recognition by providing labels, menus, and visual cues rather than requiring recall
5. Account for memory decay by showing progress, saving work automatically, and providing confirmation
6. Break complex tasks into manageable steps with clear progress indicators
7. Consistent navigation and design patterns help build long-term memory schemas

## Common Mistakes to Avoid

- Presenting too many options simultaneously (exceeding 7±2 limit)
- Requiring users to remember information across screens without cues
- Using vague labels that require users to recall what they mean
- Creating unnecessary complexity that adds extraneous cognitive load
- Ignoring the difference between recognition (easy) and recall (difficult) in interface elements

## Revision Tips

1. Memorize Miller's Law exactly: 7 ± 2 items in short-term memory
2. Practice applying memory principles to existing websites—identify violations and solutions
3. Remember the three cognitive load types with clear examples of each
4. Focus on the recognition vs. recall principle as it's frequently tested
5. Review the three-stage memory model with capacity and duration for each type
6. Practice drawing connections between memory limitations and specific design guidelines
