# Learning Objectives

After studying this topic, you should be able to:

1. **Define** static hashing and explain its significance in data storage and retrieval systems.
2. **List** the key characteristics of a good hash function, including determinism, uniform distribution, and fast computation.
3. **Identify** the different types of collision resolution methods used in static hashing, including separate chaining, linear probing, quadratic probing, and double hashing.
4. **Explain** the concept of load factor and its impact on the performance of static hashing, including the degradation of linear probing beyond a load factor of 0.75.
5. **Describe** the advantages and disadvantages of static hashing, including its simplicity, excellent average-case performance, and limitations due to fixed table size.
6. **Apply** the division method formula to calculate the hash value for a given key and table size, and explain why the table size should be prime.
7. **Analyze** the tradeoffs between different collision resolution methods, including separate chaining, linear probing, quadratic probing, and double hashing, in terms of storage, cache performance, and complexity.
8. **Design** a static hash table with a suitable collision resolution method for a given application, taking into account the expected data size, distribution, and performance requirements.
