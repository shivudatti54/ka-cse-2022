# Learning Objectives

After studying this topic, you should be able to:

1. Define a queue and explain the First-In-First-Out (FIFO) principle that governs its operation.
2. Identify and describe the five basic operations performed on a queue: enqueue, dequeue, peek, isEmpty, and isFull.
3. Implement a basic linear queue using an array in the C programming language, including proper overflow and underflow checks.
4. Analyze the limitation of memory wastage in a simple linear queue implementation and explain why it occurs.
5. Compare and contrast queue and stack data structures with respect to their ordering principles, insertion and deletion points, and practical applications.
6. Classify the various types of queues (linear, circular, double-ended (deque), and priority queues) based on their characteristics and applications.
7. Demonstrate the step-by-step process of tracing the state of a queue after a given sequence of enqueue and dequeue operations, showing array contents and front/rear indices.
8. Evaluate the suitability of queues for real-world and computer science applications, such as CPU scheduling, BFS, and print spooling, and justify their use in these contexts.
9. Design a queue-based solution to a given problem, taking into account the trade-offs between different queue implementations (e.g., linear vs. circular) and the need for overflow and underflow condition checks.
