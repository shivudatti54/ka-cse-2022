# Classifications: Primitive and Non-Primitive Data Structures

## Overview

Data structures are classified into two categories: primitive and non-primitive. Primitive data structures are the basic data types provided by programming languages, while non-primitive data structures are derived from primitive types and are used to store collections of data. Understanding the classification of data structures is fundamental to choosing the right structure for a specific problem.

## Key Points

- Primitive data structures are predefined, simple, and have a fixed size.
- Non-primitive data structures are user-defined, complex, and have a variable size.
- Primitive data structures include integers, floats, characters, booleans, and pointers.
- Non-primitive data structures include arrays, linked lists, stacks, queues, trees, and graphs.
- Choosing the right data structure depends on the type of operations, frequency of operations, memory constraints, access patterns, and time complexity requirements.

## Important Definitions

- **Primitive Data Structures**: Basic data types provided by programming languages.
- **Non-Primitive Data Structures**: Derived from primitive types to store collections of data.
- **Linear Data Structures**: Elements are arranged in a sequential manner.
- **Non-Linear Data Structures**: Elements are not arranged sequentially.

## Key Formulas / Syntax

- Array declaration: `int marks[5] = {85, 90, 78, 92, 88};`
- Linked list node: `struct Node { int data; struct Node* next; };`

## Comparisons

| Aspect     | Primitive              | Non-Primitive           |
| ---------- | ---------------------- | ----------------------- |
| Complexity | Simple                 | Complex                 |
| Definition | Predefined by language | User-defined or library |
| Values     | Single value           | Multiple values         |
| Size       | Fixed                  | Variable                |

## Exam Tips

- Understand the characteristics of each data structure.
- Know the trade-offs between different data structures.
- Practice implementing common data structures.
- Analyze the time and space complexity of algorithms.
- Be prepared to explain your choices and reasoning.
