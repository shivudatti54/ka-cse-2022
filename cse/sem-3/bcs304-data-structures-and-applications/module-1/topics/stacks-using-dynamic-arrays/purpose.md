# Learning Objectives

After studying this topic, you should be able to:

1. Define the limitations of a fixed-size array stack and explain how a dynamic array stack addresses these limitations.
2. Describe the doubling strategy used in dynamic array stacks and analyze its amortized time complexity.
3. Implement a dynamic array stack in C using realloc and demonstrate its usage.
4. Compare and contrast fixed-size array stacks with dynamic array stacks in terms of memory usage, push/pop complexity, and use cases.
5. Justify the "halve at quarter" strategy for shrinking the array in a dynamic array stack to avoid thrashing.
6. Assess the trade-offs between different growth factors and initial capacities in dynamic array stacks.
7. Design and implement a destroy function to properly free memory in a dynamic array stack.
8. Explain the concept of amortized analysis and its application to dynamic array stacks and other data structures.
