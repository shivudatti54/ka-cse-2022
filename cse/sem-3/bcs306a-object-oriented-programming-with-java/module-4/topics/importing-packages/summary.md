# Importing Packages

## Overview

In Java, importing packages allows you to use classes and interfaces from other packages in your program without having to use their fully qualified names (FQN) every time. This makes your code more readable and easier to maintain. Java provides various ways to import packages, including using import statements and FQN.

## Key Points

- Import statements are used to import packages or classes into a Java program.
- Fully Qualified Name (FQN) can be used to access a class or interface without importing its package.
- Importing packages makes the code more readable and easier to maintain.
- Java's built-in packages are imported using the `java` keyword.
- Import statements are not required for classes in the same package.

## Important Definitions

- **Fully Qualified Name (FQN)**: The complete name of a class or interface, including its package name.
- **Import Statement**: A statement used to import a package or class into a Java program.

## Key Syntax

- `import package_name.ClassName;` - Import a specific class from a package.
- `java.util.ArrayList<>();` - Using FQN to access a class without importing its package.

## Exam Tips

- Be prepared to identify and explain the use of import statements and FQN in Java code.
- Understand how to import packages and classes correctly in Java.
