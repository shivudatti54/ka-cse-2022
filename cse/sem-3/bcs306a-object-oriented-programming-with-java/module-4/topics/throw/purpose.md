# Learning Objectives

After studying this topic, you should students able to:

1. Explain the purpose and behavior of the `throw` statement in Java exception handling, including how it transfers control when executed.

2. Differentiate between the `throw` and `throws` keywords, understanding that `throw` actively throws an exception while `throws` declares exception propagation in a method signature.

3. Apply the correct syntax for throwing both built-in Java exceptions and custom exception objects using the `throw new ExceptionType("message")` pattern.

4. Determine whether a thrown exception requires declaration in the `throws` clause based on whether it is checked (compile-time) or unchecked (runtime).

5. Design and implement custom exception classes appropriate for specific application domains by extending either `Exception` or `RuntimeException`.

6. Demonstrate proper exception propagation through multiple method calls, including catching, transforming, and rethrowing exceptions appropriately.

7. Analyze and fix common errors related to throwing exceptions, such as throwing non-Throwable objects or incorrectly handling checked exceptions.
