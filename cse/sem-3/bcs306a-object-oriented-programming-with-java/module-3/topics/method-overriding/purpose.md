# Learning Objectives

After studying this topic, you should students able to:

1. Define method overriding in Java and explain its purpose in object-oriented programming.
2. List the rules for method overriding, including same method signature, return type, and access modifier constraints.
3. Identify the benefits of using the `@Override` annotation in method overriding.
4. Explain how method overriding enables runtime polymorphism in Java.
5. Describe the relationship between method overriding and other object-oriented concepts, such as inheritance and dynamic method dispatch.
6. Apply the rules of method overriding to create a subclass that overrides a method from its superclass.
7. Analyze the impact of access modifier changes on method overriding and explain why reducing access visibility is not allowed.
8. Justify the use of method overriding in real-world applications, such as framework callbacks and event handling.
