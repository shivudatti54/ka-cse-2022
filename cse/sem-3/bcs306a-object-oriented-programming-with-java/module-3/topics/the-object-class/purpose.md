# Learning Objectives

After studying this topic, you should students able to:

1. Recall the position of the Object class in Java's class hierarchy and its significance.
2. List the common methods provided by the Object class, such as toString(), equals(), and hashCode().
3. Explain the purpose and default behavior of the toString(), equals(), and hashCode() methods in the Object class.
4. Describe how to properly override the toString(), equals(), and hashCode() methods in a custom class to achieve desired behavior.
5. Implement the clone() method to create a copy of an object and understand its limitations.
6. Compare and contrast the uses of the getClass() method and the instanceof operator in Java.
7. Justify the importance of overriding the equals() and hashCode() methods in custom classes when using Java's Collections Framework.
8. Design and implement a custom class that correctly overrides the toString(), equals(), and hashCode() methods to demonstrate understanding of the Object class's key methods.
