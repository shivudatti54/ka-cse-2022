# Learning Objectives

After studying this topic, you should students able to:

1. Explain the concept of garbage collection in Java and its importance in memory management.

2. Identify when an object becomes eligible for garbage collection in Java programs.

3. Describe the working of the mark-and-sweep algorithm and generational garbage collection approach.

4. Compare different garbage collection algorithms available in Java (Serial, Parallel, CMS, G1, ZGC).

5. Demonstrate the use of System.gc() and Runtime.gc() methods for requesting garbage collection.

6. Explain the purpose and limitations of the finalize() method in Java.

7. Analyze common causes of memory leaks in Java applications and suggest solutions.

8. Apply garbage collection tuning parameters to optimize Java application performance.
