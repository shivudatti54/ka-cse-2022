# Learning Objectives

After studying this topic, you should be able to:

1. Explain the three access specifiers in C++ (public, private, protected) and their differences in terms of accessibility.

2. Identify which class members are accessible under each access specifier in various contexts (within the class, in derived classes, and from outside the class).

3. Declare and define friend functions, understanding their unique position as non-member functions with privileged access.

4. Design and implement friend classes where two classes need shared access to each other's private members.

5. Distinguish between member functions and friend functions, recognizing when each approach is appropriate in software design.

6. Apply access control principles to create well-encapsulated classes while providing necessary interfaces for object interaction.

7. Analyze code snippets to determine access violations and suggest corrections to maintain proper encapsulation.

8. Evaluate the trade-offs between strict encapsulation and the need for flexibility when deciding whether to grant friendship.
