# Learning Objectives

After studying this topic, you should be able to:

1. Define inheritance and explain its significance in object-oriented programming with respect to code reusability and extensibility.

2. Differentiate between the various types of inheritance: single, multiple, multilevel, hierarchical, and hybrid inheritance with appropriate examples.

3. Explain the purpose and behavior of the protected access specifier in C++ and contrast it with private and public specifiers.

4. Implement inheritance in C++ programs using different modes of inheritance (public, protected, private) and understand their effects on member accessibility.

5. Analyze the behavior of constructors and destructors in inheritance hierarchies, including the order of execution.

6. Resolve ambiguity issues arising from multiple inheritance using scope resolution operator and virtual base classes.

7. Apply inheritance concepts to design class hierarchies that model real-world relationships appropriately.

8. Evaluate when to use inheritance versus composition based on the IS-A and HAS-A relationships in software design.
