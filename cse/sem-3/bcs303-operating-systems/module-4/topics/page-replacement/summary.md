# Page Replacement - Summary

## Key Definitions

- **Page Replacement**: The process of selecting a victim page to evict from physical memory when a page fault occurs and no free frames are available.

- **Page Fault**: An interrupt generated by hardware when a process accesses a page that is not currently in physical memory.

- **Victim Page**: The page selected for eviction from physical memory to free a frame for a new page.

- **Belady's Anomaly**: The counter-intuitive phenomenon where increasing the number of frames can increase page faults under certain algorithms like FIFO.

- **Reference String**: A sequence of page numbers representing the order in which a process accesses pages.

- **Stack Algorithm**: An algorithm where pages in memory with n frames are always a subset of pages in memory with n+1 frames, preventing Belady's anomaly.

## Important Formulas

There are no specific formulas for page replacement algorithms themselves. The primary metric is:

**Page Fault Rate = Number of Page Faults / Total Page References**

The goal is to minimize this rate through appropriate algorithm selection.

## Key Points

1. Page replacement is essential when physical memory is full and a page fault occurs for a page not in memory.

2. FIFO is simple to implement but performs poorly and can exhibit Belady's anomaly.

3. Optimal (OPT) algorithm provides the minimum possible page faults but requires impossible future knowledge.

4. LRU closely approximates OPT and is a stack algorithm, but requires hardware support for exact implementation.

5. Clock (Second Chance) is a practical LRU approximation using reference bits, widely used in real systems.

6. The choice of replacement algorithm significantly impacts system performance and throughput.

7. Real systems use hybrid approaches combining multiple strategies (reference bits, modified bits, aging).

8. No single algorithm is optimal for all access patterns—context and workload matter.

## Common Mistakes

1. Confusing page replacement with page allocation—replacement decides which page to evict, allocation decides how many frames each process gets.

2. Forgetting that dirty (modified) pages must be written to disk before eviction, adding I/O overhead.

3. Believing FIFO always improves with more frames—it can worsen due to Belady's anomaly.

4. Assuming LRU is always optimal—it approximates OPT but is not always the best choice in practice.

5. Overlooking the implementation cost of algorithms—some theoretically better algorithms have prohibitive overhead.
