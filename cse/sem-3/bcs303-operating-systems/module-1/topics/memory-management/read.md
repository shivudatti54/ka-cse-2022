# Memory Management Basics

## Memory Hierarchy

```
Registers → Cache → Main Memory → Disk
(fastest) (slowest)
```

## Address Binding

Binding logical addresses to physical addresses can occur at:

| Stage          | Time             | Flexibility               |
| -------------- | ---------------- | ------------------------- |
| Compile time   | Before execution | Fixed, absolute code      |
| Load time      | When loaded      | Relocatable code          |
| Execution time | During run       | Requires hardware support |

## Logical vs Physical Address

- **Logical Address**: Generated by CPU, also called virtual address
- **Physical Address**: Actual memory location
- **Memory Management Unit (MMU)**: Translates logical to physical

## Contiguous Allocation

### Fixed Partitions

- Memory divided into fixed-size partitions
- Internal fragmentation (wasted space within partition)

### Variable Partitions

- Partitions created dynamically
- External fragmentation (scattered free space)

### Allocation Strategies

| Strategy  | Description                     | Performance           |
| --------- | ------------------------------- | --------------------- |
| First Fit | First hole that's big enough    | Fast                  |
| Best Fit  | Smallest hole that's big enough | Less waste, slow      |
| Worst Fit | Largest hole                    | Fastest fragmentation |

**First Fit** is generally preferred - fast and comparable to best fit.

## Fragmentation

### Internal Fragmentation

```
Partition: 10KB | Process: 8KB → 2KB wasted inside
```

### External Fragmentation

```
| P1 | Free | P2 | Free | P3 | Free |
Total free = 6KB, but largest contiguous = 2KB
```

### Solutions

1. **Compaction**: Shuffle memory to create large contiguous block (expensive)
2. **Paging**: Eliminate external fragmentation
3. **Segmentation**: Logical division of memory

## Memory Protection

- Base register: Starting address
- Limit register: Size of region
- Address must satisfy: base ≤ address < base + limit
