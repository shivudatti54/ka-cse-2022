Interrupts enable asynchronous CPU notification from hardware devices, transforming polling-based I/O to event-driven processing. Hardware interrupts originate from peripherals via IRQ lines managed by interrupt controllers (8259A PIC/APIC). The CPU acknowledges interrupts, fetches vector numbers from the Interrupt Vector Table/IDT, and executes corresponding ISRs. Key hardware concepts include edge vs level triggering, daisy-chain and priority-encoded arbitration, vectored interrupt mechanisms, and nested interrupt handling. Quantitative analysis involves calculating interrupt latency (device delay + controller latency + CPU acknowledgment + vector fetch + ISR entry) and determining break-even points between polling and interrupt-driven I/O based on interrupt frequency and overhead.
