Cache memory employs three primary mapping functions: direct mapping (each block maps to one line via modular arithmetic), fully associative mapping (any line), and set-associative mapping (compromise with sets and ways). Direct mapping offers simple hardware but suffers conflict misses; fully associative eliminates conflicts but requires expensive CAM hardware. Set-associative balances these concerns and dominates modern processors. Address decomposition follows $n = \text{tag} + \text{index} + \text{offset}$, where index bits = $\log_2$(cache lines), offset bits = $\log_2$(block size), and tag bits = remainder. Replacement policies (LRU, FIFO, Random) determine block eviction when cache is full, with LRU providing optimal miss rate at highest hardware cost.
