Encoders are combinational logic circuits that convert 2^n input lines into n output lines, representing the inverse function of decoders. A simple encoder requires exactly one active input at a time and produces binary output corresponding to that input. However, when multiple inputs are active simultaneously, ambiguity arises—multiple inputs compete to determine the output, resulting in incorrect or meaningless codes. This fundamental limitation necessitates the priority encoder, which establishes a hierarchical priority ordering where the highest-priority active input determines the output.

The 4-to-2 priority encoder demonstrates the core design methodology: deriving minimized Boolean expressions through K-map minimization yields Y₁ = I₂ + I₃ and Y₀ = I₁·I₂' + I₃. The valid output V = I₀ + I₁ + I₂ + I₃ indicates whether any input is active, distinguishing between "no input" (V=0) and specific input conditions (V=1).

Extended encoders like the 8-to-3 follow the same principles with additional complexity. Practical implementations include enable inputs for controlled operation and cascading for handling larger input counts. Timing analysis reveals critical path delays and potential glitch hazards that must be considered in high-speed applications. Modern digital design employs HDL (VHDL/Verilog) for synthesis and implementation.

Applications span keyboard encoding, interrupt priority resolution in microprocessors, analog-to-digital conversion, and position sensing. The encoder-decoder pair forms a fundamental building block in digital system architecture, enabling efficient data routing, memory addressing, and input processing across computing and embedded systems.
