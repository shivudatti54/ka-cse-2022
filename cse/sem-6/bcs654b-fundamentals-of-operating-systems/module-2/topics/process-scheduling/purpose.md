# Learning Objectives

After studying this topic, you should be able to:

1. Define a process and differentiate it from a program.
2. Identify and describe all components of a Process Control Block (PCB) and explain their purpose.
3. Illustrate and explain the state transition diagram for a process, including the events that cause each transition.
4. Compare and contrast the roles of the long-term, short-term, and medium-term schedulers.
5. Describe the mechanism and purpose of a context switch and explain why it is considered overhead.
6. Explain the process creation mechanism in a UNIX-like system using `fork()` and `exec()`.
7. Differentiate between the shared memory and message passing models of interprocess communication (IPC).
