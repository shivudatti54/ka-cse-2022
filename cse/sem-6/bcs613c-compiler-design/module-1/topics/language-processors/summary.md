# Language Processors

## Overview

Language processors are software utilities that translate or process programs written in programming languages. The three main types are compilers, interpreters, and assemblers, each serving different translation needs from high-level languages to machine code.

## Key Points

- **Compilers**: Translate entire source code into machine code before execution; faster execution but slower compilation
- **Interpreters**: Translate and execute source code line by line; slower execution but easier debugging
- **Assemblers**: Translate assembly language (low-level) into machine code with one-to-one instruction mapping
- **Front-End Analysis**: Includes lexical, syntax, and semantic analysis to understand source code structure
- **Back-End Synthesis**: Includes intermediate code generation, optimization, and code generation for target machine
- **Separation Benefits**: Simplicity, efficiency, portability, and ability to use specialized tools

## Important Concepts

- Compilation vs interpretation trade-offs
- Two-part compiler structure (analysis and synthesis)
- Symbol table management across phases
- Platform-dependent vs platform-independent processing

## Notes

- Common exam question: Compare and contrast compilers, interpreters, and assemblers
- Understand why lexical and syntax analysis are separated
- Remember that front-end is language-dependent, back-end is machine-dependent
