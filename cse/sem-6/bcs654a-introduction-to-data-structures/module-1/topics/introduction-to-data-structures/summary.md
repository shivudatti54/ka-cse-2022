# Introduction to Data Structures

## Overview

Data structures are specialized formats for organizing, processing, and storing data efficiently in computer memory. They provide the foundation for implementing algorithms and managing large amounts of data, enabling efficient access, modification, and program performance optimization.

## Key Points

- **Definition**: A data structure is a particular way of organizing data in a computer so it can be used effectively
- **Classification by Organization**: Divided into primitive (int, float, char) and non-primitive (arrays, linked lists, trees, graphs)
- **Linear vs Non-Linear**: Linear structures have sequential arrangement (arrays, stacks, queues), while non-linear have hierarchical relationships (trees, graphs)
- **Static vs Dynamic**: Static structures have fixed size at compile time (arrays), dynamic can change during execution (linked lists)
- **Abstract Data Types**: Define data types by their behavior rather than implementation, hiding implementation details
- **Time Complexity**: Measures algorithm execution time as a function of input size (O(1), O(n), O(log n), etc.)
- **Common Operations**: Include traversing, searching, insertion, deletion, sorting, merging, and updating elements

## Important Concepts

- Primitive data types include integers, floats, characters, and booleans
- Non-primitive structures are derived from primitive types and can store multiple values
- Linear structures allow sequential traversal in a single run
- Non-linear structures represent hierarchical or network relationships
- Selection criteria include data type, operations frequency, memory constraints, and time complexity requirements

## Notes

- Understand the complete classification hierarchy of data structures
- Be able to compare and contrast static vs dynamic allocation
- Know the characteristics that distinguish linear from non-linear structures
- Memorize common time complexities and their meanings
- Practice drawing diagrams for different data structure classifications
