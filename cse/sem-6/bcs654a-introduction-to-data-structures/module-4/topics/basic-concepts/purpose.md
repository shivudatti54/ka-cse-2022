# Learning Objectives

After studying this topic, you should be able to:

1. Define and explain the fundamental terminology of trees including root, parent, child, sibling, leaf, and ancestor nodes.

2. Distinguish between tree and non-tree structures by identifying cycles and verifying tree properties.

3. Calculate and interpret key tree metrics such as degree of a node, depth of a node, height of a tree, and number of levels.

4. Explain the hierarchical nature of trees and how parent-child relationships differ from linear data structures.

5. Compare trees with other data structures (arrays, linked lists, stacks, queues) in terms of organization and access patterns.

6. Identify real-world applications where tree structures are used to represent hierarchical relationships.

7. Apply tree terminology to analyze and describe given tree diagrams, identifying all nodes, edges, root, leaves, and internal nodes.

8. Prepare foundational concepts necessary for understanding binary trees, binary search trees, and expression trees covered in subsequent topics.
