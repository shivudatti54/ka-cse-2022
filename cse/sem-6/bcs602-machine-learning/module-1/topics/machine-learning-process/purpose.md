# Why Study the Machine Learning Process?

## Big Picture Foundation

The ML process is the backbone of every ML project. Understanding each step — from problem definition to deployment — ensures you build models systematically rather than haphazardly. This is a **foundational framework** tested in every ML exam.

## Exam Relevance

| Aspect         | Details                                                          |
| -------------- | ---------------------------------------------------------------- |
| Frequency      | Very common — appears in almost every ML exam                    |
| Marks          | 8-10 marks (describe all steps with diagram)                     |
| Question Types | Flowchart diagrams, step descriptions, sub-activity explanations |

## Practical Value

- **Structured approach** prevents skipping critical steps like data cleaning.
- **Better project planning** — know what to expect at each stage.
- **Debugging guide** — when a model fails, trace back to which step went wrong.

## Learning Outcomes

After this topic, you should be able to:

1. List and explain all 7-8 steps of the ML process
2. Draw the ML process pipeline diagram
3. Describe sub-activities within data preparation
4. Explain train/validation/test split rationale
5. Differentiate evaluation metrics for different problem types
