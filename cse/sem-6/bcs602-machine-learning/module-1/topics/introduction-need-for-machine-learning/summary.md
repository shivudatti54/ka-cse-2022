# Introduction and Need for Machine Learning

## Overview

Machine Learning enables computers to learn from data and improve performance without explicit programming. It addresses limitations of traditional programming by automatically discovering patterns and adapting to changing environments, making it essential for complex modern problems.

## Key Points

- **Traditional Programming Limitations**: Cannot handle complex problems like image recognition, constantly changing rules, or massive problem spaces
- **ML Paradigm Shift**: Reverses traditional programming by deriving rules from answers and data rather than coding explicit rules
- **Data-Driven Learning**: Relies on pattern recognition and generalization to apply learned patterns to new, unseen data
- **Adaptive Systems**: Continuously improves performance with more experience and data
- **Real-World Impact**: Powers healthcare diagnosis, fraud detection, personalized recommendations, self-driving cars, and content systems
- **Scale and Automation**: Processes information beyond human capacity and automates complex decision-making

## Important Concepts

- Machine learning is a subfield of AI that provides the primary methodology for achieving intelligent systems
- Formal definition by Tom Mitchell: Learning from experience E with respect to tasks T and performance measure P
- Four main types: supervised learning (labeled data), unsupervised learning (unlabeled data), semi-supervised learning (mixed), and reinforcement learning (reward-based)
- ML process includes problem definition, data collection, data preparation, model selection, training, evaluation, and deployment

## Notes

- Focus on understanding WHY ML is needed rather than just how it works for exam questions
- Be able to compare ML with traditional programming using the formula: Traditional Programming (Rules + Data → Answers) vs ML (Answers + Data → Rules)
- Memorize real-world application examples across different domains (healthcare, finance, retail, transportation)
