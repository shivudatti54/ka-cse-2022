Virtualization enables multiple isolated VMs to share physical hardware resources. CPU virtualization employs binary translation, para-virtualization, or hardware-assisted virtualization (Intel VT-x, AMD-V) to safely multiplex physical CPUs. Memory virtualization maps guest virtual addresses to machine addresses through shadow page tables or hardware-accelerated nested page tables (EPT/NPT). I/O virtualization offers three models: device emulation for compatibility, para-virtualized I/O (Virtio) for balanced performance, and direct passthrough (SR-IOV) for near-native throughput. Each technique represents a distinct trade-off between compatibility, performance, and isolation in virtualized environments.
