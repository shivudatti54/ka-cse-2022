A Virtual Cluster is a logical collection of interconnected VMs deployed on shared physical infrastructure, forming the foundation of IaaS/PaaS cloud offerings. The architecture comprises VMs, virtual networks, virtual storage, and cluster managers (Kubernetes, Mesos, vSphere), layered above hypervisors and physical infrastructure. VM provisioning involves request specification, placement optimization (bin-packing algorithms), instantiation, configuration, and integration. Resource management encompasses static/dynamic allocation, share-based policies, overcommitment, load balancing, server consolidation with energy models, live migration with downtime calculation, and performance isolation mechanisms. Comparative analysis reveals Kubernetes excels in container orchestration with VM support via KubeVirt, Mesos offers flexible offer-based scheduling, while vSphere DRS provides enterprise-grade live migration and SLA enforcement. Assessment includes quantitative MCQs requiring calculation (FFD placement, energy savings, overcommitment implications) and proof-level questions on bin-packing approximation bounds.
