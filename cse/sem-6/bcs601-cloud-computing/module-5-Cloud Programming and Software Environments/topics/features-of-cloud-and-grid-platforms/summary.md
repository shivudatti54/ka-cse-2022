This module covered the fundamental distinguishing features of Cloud and Grid computing platforms. Grid computing operates on a federated, decentralized model with distributed ownership across virtual organizations, emphasizing high-throughput batch processing and OGSA-based standards. Cloud computing provides on-demand, elastic resources through a service-oriented architecture (IaaS/PaaS/SaaS) with centralized provider control. Key theoretical concepts include the NIST essential characteristics, OGSA layered architecture, resource allocation as constraint satisfaction, and elasticity proofs. The comparative analysis highlighted differences in ownership, pricing, latency, security models, and use case focus. Students developed analytical skills through application-level MCQs involving cost-benefit analysis and platform selection under constraints.
