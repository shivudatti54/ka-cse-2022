Scalable computing over the internet enables systems to handle growing workloads through vertical (scale-up) and horizontal (scale-out) approaches. Amdahl's Law establishes theoretical speedup limits: $S(N) = \frac{1}{(1-P) + \frac{P}{N}}$, demonstrating that sequential bottlenecks constrain parallelization benefits. Gustafson's Law complements this by showing that scaled problems can achieve near-linear speedup. The CAP Theorem mandates trade-offs between Consistency, Availability, and Partition Tolerance in distributed systems—architects must select appropriate guarantees based on application requirements. Load balancing algorithms (Round Robin, Least Connections, Weighted variants) distribute traffic with varying complexity ($O(1)$ to $O(N)$). Modern implementations leverage virtualization, container orchestration (Kubernetes), and distributed storage to achieve elastic scaling. Key challenges include network latency, consistency maintenance, security, and cost optimization. Hard-level assessment requires applying theoretical models to numerical problems, analyzing CAP trade-offs in architectural decisions, and evaluating algorithmic choices for specific scenarios.
