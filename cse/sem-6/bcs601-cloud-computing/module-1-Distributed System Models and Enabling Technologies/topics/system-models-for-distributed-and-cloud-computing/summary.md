System models for cloud computing evolved from cluster computing (tightly coupled homogeneous nodes, centralized management) through grid computing (loosely coupled heterogeneous resources, cross-organizational virtual organizations) to modern cloud computing (virtualized resources, pay-per-use utility model). The five NIST characteristics—on-demand self-service, broad network access, resource pooling, rapid elasticity, and measured service—define cloud systems. Cloud service models (IaaS, PaaS, SaaS) represent different responsibility trade-offs between provider and user. Formal foundations include the CAP theorem (consistency-availability-partition tolerance trade-off), FLP impossibility (asynchronous consensus impossibility), and virtualization theory for resource isolation. Quantitative comparison reveals cloud computing's advantages in elasticity, startup time, and OpEx model versus traditional models. Students should understand how to apply these models to architectural decisions in distributed systems design.
