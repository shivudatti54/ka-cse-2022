Distributed Intrusion Detection Systems (DIDS) address the fundamental limitations of centralized IDS in cloud environments by deploying coordinated agents across hypervisors, VMs, and networks. Key architectural components include LIDAs (host-based, hypervisor-based, network-based), Regional Correlation Engines, and a Global Intrusion Detection Manager. Detection techniques include signature-based detection using pattern matching and anomaly-based detection employing statistical models (Gaussian distributions, multivariate analysis), machine learning algorithms (Isolation Forest, One-Class SVM), and clustering methods. Distributed correlation through temporal analysis, causal attack graphs, and Dempster-Shafer evidence theory enables detection of sophisticated multi-stage attacks. Implementation challenges encompass data overhead, synchronization, scalability, and cloud-specific threats including VM migration vulnerabilities and multi-tenant side-channel attacks. Evaluation relies on TPR, FPR, precision, F1-score, and detection latency metrics.
