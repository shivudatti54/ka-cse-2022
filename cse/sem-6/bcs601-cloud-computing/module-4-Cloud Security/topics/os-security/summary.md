OS Security in Cloud Computing addresses hardening and protection of both Host OS (with highest privilege, controlling all VMs) and Guest OS (tenant responsibility under shared responsibility model). Key hardening strategies include minimal installation, MAC deployment (SELinux/AppArmor), file integrity monitoring (AIDE/Tripwire), UEFI Secure Boot, and automated patch management. Guest OS hardening follows CIS benchmarks: service reduction, strong authentication, firewall configuration, and endpoint protection. Critical cloud-specific vulnerabilities include shared kernel issues in containers, VM escape attacks breaching VM isolation, and side-channel attacks (Spectre/Meltdown) exploiting CPU speculative execution. Formal security models (DAC, MAC with Bell-LaPadula/Biba, RBAC) provide theoretical foundations for access control implementation. Quantitative metrics enable attack surface calculation and patch latency risk assessment, supporting evidence-based security decision-making in cloud environments.
