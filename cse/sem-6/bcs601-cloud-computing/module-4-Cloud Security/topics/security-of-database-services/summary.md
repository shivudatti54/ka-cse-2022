The security of database services in cloud computing operates under a shared responsibility model where providers secure the infrastructure while customers secure their data and configurations. Key security mechanisms include: (1) Network isolation through VPCs, private subnets, security groups, and NACLs; (2) Cryptographic protection through encryption at rest (AES-256 TDE) and in transit (TLS 1.3); (3) Identity and access management implementing least privilege principles with MFA; (4) Comprehensive auditing through cloud-native logging services. Common threats include SQL injection, data exfiltration, denial of service, and misconfiguration. Students must understand formal threat models (STRIDE), compliance frameworks (GDPR, HIPAA), and be able to calculate security metrics such as encryption brute-force complexity and SLA-based availability.
