# Collections Overview

## Overview

The Java Collections Framework provides a unified architecture introduced in Java 1.2, replacing limited collection support from legacy classes. It defines interfaces, implementations, and algorithms to efficiently store, retrieve, and manipulate groups of objects with type safety through generics.

## Key Points

- **Framework Components**: Interfaces (Collection, List, Set, Map), Implementations (ArrayList, HashSet, HashMap), Algorithms (sort, search)
- **Reduces Programming Effort**: Standard interfaces and ready-to-use implementations
- **Increases Performance**: Optimized data structures for different use cases
- **Interoperability**: Collections work together through common interfaces
- **Type Safety**: Generics provide compile-time type checking
- **Interface Hierarchy**: Collection → List/Set/Queue, separate Map hierarchy
- **Benefits**: Reusability, performance, reduced errors, easier maintenance

## Important Concepts

- List maintains insertion order and allows duplicates
- Set stores unique elements only
- Map stores key-value pairs (not part of Collection)
- Queue supports FIFO processing
- Enhanced for-loop works with Iterable collections

## Notes

- Know that Collections Framework was introduced in Java 1.2
- Understand three main components: interfaces, implementations, algorithms
- Remember Map is separate from Collection hierarchy
