# Why Study Using Cookies?

## Client-Side State Management

Cookies are one of the **most important mechanisms** for maintaining state in web applications. Understanding the Java Servlet Cookie API is essential for implementing user preferences, session tracking, and personalized web experiences.

## Exam Relevance

| Aspect             | Details                                                     |
| ------------------ | ----------------------------------------------------------- |
| **Frequency**      | Commonly asked in servlet-related questions                 |
| **Marks**          | Typically 8-10 marks                                        |
| **Question Types** | Cookie CRUD operations, servlet programs, cookie vs session |
| **Must Know**      | Cookie constructor, addCookie, getCookies, setMaxAge        |

## Prerequisites

- HTTP protocol (headers, request/response cycle)
- Servlet basics (request, response objects)
- Session tracking concepts

## What You Will Learn

- Creating cookies with the Cookie class
- Reading cookies from HTTP requests
- Deleting cookies by setting maxAge to 0
- Cookie properties (path, domain, maxAge, secure, httpOnly)
- Session cookies vs persistent cookies
- Cookie limitations and best practices
