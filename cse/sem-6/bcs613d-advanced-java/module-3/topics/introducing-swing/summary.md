# Introducing Swing

## Overview

Swing is Java's advanced GUI toolkit providing rich, portable user interface components that are lightweight, customizable, and platform-independent. Introduced in Java 1.2, Swing builds on AWT while offering enhanced features including pluggable look-and-feel, MVC architecture, and extensive component library.

## Key Points

- **Lightweight Components**: Written entirely in Java, no native peer dependencies
- **Rich Component Set**: Buttons, tables, trees, text areas, menus, dialogs, and more
- **Platform Independent**: Consistent behavior across operating systems
- **Pluggable Look-and-Feel**: Change UI appearance at runtime
- **MVC Architecture**: Separates data model from view and controller
- **javax.swing Package**: Contains main Swing classes
- **JComponent Base Class**: Parent class for most Swing components
- **Enhanced Features**: Double buffering, borders, tooltips, keyboard shortcuts

## Important Concepts

- Swing vs AWT: lightweight vs heavyweight components
- JFrame, JPanel, JButton are core Swing classes
- All Swing component names start with "J" prefix
- Event handling uses delegation event model
- Thread safety requires Event Dispatch Thread

## Notes

- Remember Swing components are lightweight (no native peers)
- For exams, know advantages: portability, rich components, look-and-feel
- Understand Swing builds on AWT but provides enhanced functionality
