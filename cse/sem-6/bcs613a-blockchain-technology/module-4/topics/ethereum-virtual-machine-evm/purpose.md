# Learning Objectives

After studying this topic, you should be able to:

1.  Define the Ethereum Virtual Machine (EVM) and explain its role as the runtime environment for smart contracts.
2.  Describe the stack-based architecture of the EVM and how it processes operations.
3.  Differentiate between the EVM's memory, storage, and stack, including their purpose and associated costs.
4.  Explain the concept of gas and its critical function in managing network resources and preventing abuse.
5.  Identify key variables available in the EVM's execution context (e.g., `msg.sender`, `block.number`).
6.  Understand what EVM-compatibility means and its significance for the broader blockchain ecosystem.
