# Forks (Ethereum)

## Overview

Ethereum forks occur through protocol upgrades (hard forks, soft forks) or chain splits from community disagreement. Hard forks create incompatible protocol changes requiring all nodes to upgrade, while soft forks maintain backward compatibility. Notable forks include The DAO fork creating Ethereum Classic and major upgrades like Constantinople, London (EIP-1559), and The Merge.

## Key Points

- **Hard Fork**: Protocol change breaking backward compatibility, requires unanimous upgrade (Berlin, London, The Merge)
- **Soft Fork**: Backward-compatible change allowing gradual adoption, old nodes still function
- **Contentious Fork**: Community disagreement causing permanent chain split (DAO fork → ETH/ETC)
- **Planned Upgrades**: Coordinated hard forks implementing protocol improvements on schedule
- **EIP Process**: Ethereum Improvement Proposals propose, discuss, and implement protocol changes
- **Major Forks**: Homestead (2016), DAO Fork (2016), Byzantium (2017), Constantinople (2019), Istanbul (2019), Berlin (2021), London (2021), The Merge (2022)
- **Coordination**: Social consensus required for successful fork adoption across nodes, miners/validators, exchanges

## Important Concepts

- Hard forks require all nodes upgrade or lose consensus
- Contentious forks create two separate chains (ETH/ETC example)
- London fork (EIP-1559) fundamentally changed fee market mechanism
- The Merge hard fork transitioned from PoW to PoS consensus
- EIP process democratizes protocol development through community review

## Notes

- Know difference between hard fork (incompatible) and soft fork (compatible)
- Understand DAO fork context and ETH/ETC split
- Be familiar with major Ethereum upgrades and their key features
- Remember EIP-1559 (London) introduced base fee burning and improved UX
