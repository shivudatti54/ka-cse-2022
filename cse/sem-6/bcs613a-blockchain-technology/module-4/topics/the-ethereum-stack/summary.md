# The Ethereum Stack

## Overview

The Ethereum stack consists of multiple layers from the blockchain foundation to user-facing applications: the Ethereum protocol layer, smart contract layer, development frameworks, and application layer. This layered architecture enables developers to build decentralized applications leveraging blockchain infrastructure.

## Key Points

- **Layer 1 (Protocol)**: Ethereum mainnet blockchain with consensus, networking, and state management
- **Layer 2 (Scaling)**: Rollups (Optimistic, ZK), sidechains, state channels improving throughput and reducing costs
- **Smart Contract Layer**: Solidity/Vyper contracts deployed on blockchain providing application logic
- **Development Frameworks**: Truffle, Hardhat, Foundry for contract development, testing, deployment
- **Web3 Libraries**: Web3.js, Ethers.js enabling JavaScript interaction with blockchain
- **Node Providers**: Infura, Alchemy providing API access to Ethereum nodes
- **Application Layer**: dApp frontends (React, Vue) interfacing with contracts through Web3 libraries

## Important Concepts

- Layered stack enables separation of concerns and modularity
- L2 solutions inherit L1 security while improving scalability
- Smart contracts form middleware between blockchain and applications
- Development tools abstract complexity, accelerating dApp creation
- Web3 libraries bridge traditional web development and blockchain

## Notes

- Understand each layer's role and how they interact
- Know major L2 solutions: Optimistic Rollups, ZK-Rollups
- Be familiar with development tools: Hardhat, Web3.js, Ethers.js
- Remember application layer is where end users interact with dApps
