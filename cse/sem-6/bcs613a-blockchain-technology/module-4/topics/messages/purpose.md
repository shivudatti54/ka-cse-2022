# Learning Objectives

After studying this topic, you should be able to:

1. Define what a message is in the context of Ethereum.
2. Identify the components of a message, including sender, recipient, value, data, and gas.
3. Explain the difference between transaction messages and call messages.
4. Describe the concept of gas and its role in message execution.
5. Understand how message call transactions work and how they are executed by the EVM.
6. Provide examples of sending messages to smart contracts using the `eth_sendTransaction` method.
