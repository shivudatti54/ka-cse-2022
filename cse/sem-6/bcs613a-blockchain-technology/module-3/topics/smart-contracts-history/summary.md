# Smart Contracts History

## Overview

Smart contracts, first conceptualized by Nick Szabo in the 1990s as computerized transaction protocols, became practically viable with Ethereum's 2015 launch. They are self-executing programs on blockchain that automatically enforce agreement terms when predetermined conditions are met, eliminating intermediaries through code-based automation.

## Key Points

- **Nick Szabo's Vision (1990s)**: Defined smart contracts as computerized protocols executing contract terms
- **Ethereum Launch (2015)**: Provided secure decentralized environment for smart contract deployment and execution
- **Self-Executing Logic**: "If/when...then..." conditional statements encoded in blockchain code
- **Contract Lifecycle**: Creation (write code) → Deployment (to blockchain) → Execution (by transactions) → Finalization (recorded on chain)
- **Core Properties**: Autonomous, deterministic, immutable, transparent, trustless, distributed
- **Cryptographic Foundation**: Relies on digital signatures (ECDSA), hash functions, and public-private key cryptography
- **Programming Languages**: Solidity (Ethereum), Rust (Solana), platform-specific languages

## Important Concepts

- Smart contracts inherit security and immutability from underlying blockchain
- Immutability means bugs are permanent, requiring extensive testing and auditing
- Oracle problem: contracts cannot natively access off-chain data without trusted feeds
- Applications: DeFi, supply chain, digital identity, voting, real estate, insurance
- Comparison with traditional contracts: faster, cheaper, automated, transparent vs manual, costly, slow
