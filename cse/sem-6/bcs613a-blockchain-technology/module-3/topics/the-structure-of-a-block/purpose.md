# Learning Objectives

After studying this topic, you should be able to:

1. Describe the overall structure of a block in a blockchain, identifying its two main components: the block header and the block body.
2. List and explain the key fields in a block header including version, previous block hash, Merkle root, timestamp, difficulty target, and nonce.
3. Explain how blocks are cryptographically chained together through the previous block hash field.
4. Describe the role of the Merkle root in efficiently summarizing all transactions within a block.
5. Analyze how the block structure contributes to blockchain immutability and data integrity.
6. Differentiate between block header and block body and their respective roles.
