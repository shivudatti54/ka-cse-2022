A software process is a structured framework transforming user requirements into working software products. The software crisis of the 1960s-70s led to formalization of software engineering disciplines. Key process models include Waterfall (sequential, low flexibility), Incremental (delivers in portions), Spiral (risk-driven iterations), V-Model (parallel development and testing), Agile (flexible, collaborative, iterative), RAD (rapid prototyping), Unified Process (use-case driven), and Concurrent Development (multiple simultaneous states). Core activities encompass requirements engineering, design, implementation, verification/validation, and maintenance (consuming 50-80% of lifecycle cost). CMMI defines five maturity levels from Initial to Optimizing. Model selection depends on requirements stability, project risk, time constraints, and organizational context.
