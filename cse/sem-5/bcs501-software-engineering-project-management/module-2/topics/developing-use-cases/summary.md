Use cases constitute a fundamental requirements engineering technique for capturing functional specifications from a user-centered perspective. Developed by Ivar Jacobson and standardized in UML, use cases describe how external actors interact with a system to achieve specific goals. A complete use case specification includes the use case name, actors (classified as primary or secondary), preconditions (required system state), postconditions (guaranteed outcomes), basic flow (primary success scenario), alternative flows (variations and error handling), and extension points (locations for conditional extensions). Use case relationships—include (mandatory behavior incorporation), extend (conditional behavior addition), and generalization (inheritance)—enable behavioral reuse and modular composition. The use case diagram provides UML-standardized graphical representation of system boundaries, actors, use cases, and their relationships. Effective use cases adhere to principles of user goal orientation, essential description, completeness, consistency, and testability, serving as critical artifacts for stakeholder communication, test case generation, and system design guidance.
