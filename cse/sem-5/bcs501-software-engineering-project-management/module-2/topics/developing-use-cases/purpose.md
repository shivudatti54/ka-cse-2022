This module enables students to understand the theoretical foundations of use case modeling, including formal definitions, components, relationships, and UML notation. Students will learn to analyze system requirements from a user-centered perspective and develop comprehensive use case specifications that capture functional requirements with appropriate depth. The content supports practical application through detailed examples and established guidelines, preparing students for requirements engineering tasks in software development projects. Upon completion, students should be able to construct use case diagrams, write fully dressed use cases, identify appropriate relationships between use cases, and evaluate use case specifications for completeness and correctness.
