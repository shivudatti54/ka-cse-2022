# Learning Objectives

After studying this topic, you should be able to:

1. Define risk and explain its significance in software engineering projects
2. Distinguish between different categories of risks: project, product, business, and technical risks
3. Apply the risk evaluation process including identification, analysis, and prioritization
4. Calculate risk exposure using probability and impact values
5. Perform Expected Monetary Value (EMV) analysis for risk decision-making
6. Analyze project performance using Earned Value Management (EVM) metrics to identify risks
7. Select appropriate risk response strategies based on risk evaluation results
8. Interpret risk priority matrix and allocate management attention accordingly
