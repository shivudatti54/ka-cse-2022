# Risk Evaluation in Software Engineering - Summary

## Key Definitions and Concepts

- **Risk**: An uncertain event or condition that, if it occurs, positively or negatively affects project objectives (scope, schedule, cost, quality)
- **Risk Evaluation**: The process of analyzing and prioritizing identified risks based on probability of occurrence and potential impact
- **Risk Exposure**: The quantitative measure of risk calculated as Probability × Impact
- **Expected Monetary Value (EMV)**: The average outcome when future events are uncertain, calculated as sum of (probability × impact) for all possible outcomes

## Important Formulas and Theorems

- **Risk Exposure** = Probability × Impact
- **Expected Monetary Value (EMV)** = Σ (Probability of outcome × Value of outcome)
- **Cost Performance Index (CPI)** = Earned Value (EV) / Actual Cost (AC)
- **Schedule Performance Index (SPI)** = Earned Value (EV) / Planned Value (PV)
- **Cost Variance (CV)** = EV - AC
- **Schedule Variance (SV)** = EV - PV

## Key Points

- Risk evaluation follows risk identification in the risk management lifecycle
- Risks are categorized into project, product, business, and technical risks
- Qualitative analysis uses ratings (High/Medium/Low); Quantitative analysis uses numerical values
- Risk prioritization focuses management attention on highest exposure risks
- CPI < 1 indicates cost overrun; SPI < 1 indicates schedule delay
- Risk response strategies: Avoid, Mitigate, Transfer, Accept (and Exploit for opportunities)
- EVM metrics serve as early warning indicators for project risks
- Monte Carlo simulation provides probabilistic project outcome estimates

## Common Mistakes to Avoid

- Confusing risk identification with risk evaluation - identification finds risks, evaluation analyzes them
- Ignoring positive risks (opportunities) in risk evaluation
- Failing to update risk registers as project progresses
- Overlooking indirect impacts of risks (reputation, customer satisfaction)
- Using only qualitative analysis when quantitative would provide better decision support

## Revision Tips

- Practice calculating risk exposure for multiple scenarios
- Memorize EVM formulas as they frequently appear in exams
- Understand when to apply each risk response strategy
- Review case studies showing real software project failures due to poor risk evaluation
- Create a mental flowchart: Identify → Analyze → Prioritize → Respond → Monitor
