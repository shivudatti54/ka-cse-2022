# Project Management Life Cycle - Summary

## Key Definitions and Concepts

- **Project Management Life Cycle**: A structured framework consisting of five phases that guide a project from initiation to closure.
- **Project Charter**: A formal document that authorizes the existence of a project and grants authority to the project manager.
- **Work Breakdown Structure (WBS)**: A hierarchical decomposition of project scope into manageable work packages.
- **Earned Value Management (EVM)**: A technique that integrates scope, schedule, and cost measurements to assess project performance.

## Important Formulas and Theorems

- **Schedule Variance (SV)**: SV = EV - PV (Positive indicates ahead of schedule)
- **Cost Variance (CV)**: CV = EV - AC (Positive indicates under budget)
- **Schedule Performance Index (SPI)**: SPI = EV / PV (Greater than 1 indicates good schedule performance)
- **Cost Performance Index (CPI)**: CPI = EV / AC (Greater than 1 indicates good cost performance)
- **Estimate at Completion (EAC)**: EAC = BAC / CPI (Based on current performance)

## Key Points

1. The five phases are: Initiation, Planning, Execution, Monitoring and Controlling, and Closing.

2. Initiation establishes project authority through the project charter and identifies initial stakeholders.

3. Planning is the most critical phase that creates the roadmap for project success.

4. The WBS decomposes project scope into smaller, manageable components.

5. Monitoring and Controlling runs concurrently with Execution to track performance.

6. Earned Value Management provides quantitative measures of project health.

7. Closing phase ensures formal acceptance and documents lessons learned.

8. Each phase produces specific deliverables that mark phase completion.

## Common Mistakes to Avoid

- Confusing Monitoring and Controlling as a sequential phase rather than concurrent with Execution.
- Treating project closure as merely handing over the product without formal acceptance.
- Underestimating the importance of documentation in the closing phase.
- Ignoring change control processes leading to scope creep.

## Revision Tips

1. Create a table comparing all five phases with their key activities and deliverables.

2. Practice EVM calculations with different scenarios to master the formulas.

3. Remember that monitoring and controlling continues throughout the execution phase.

4. Focus on understanding the interrelationships between phases rather than memorizing in isolation.
