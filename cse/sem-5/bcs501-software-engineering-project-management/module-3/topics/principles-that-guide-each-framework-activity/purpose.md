# Learning Objectives

After studying this topic, you should be able to:

1. Explain how the five framework activities (Communication, Planning, Modeling, Construction, Deployment) are reinterpreted through agile principles.

2. Analyze how the Agile Manifesto values (individuals over processes, collaboration over contracts, responding to change over following a plan) influence each framework activity.

3. Compare the principles guiding each framework activity in agile methodologies versus traditional software engineering approaches.

4. Apply specific agile practices (such as TDD, CI/CD, time-boxing, WIP limits) to the appropriate framework activity.

5. Distinguish how principles manifest differently across Scrum, XP, and Kanban frameworks.

6. Evaluate the impact of agile principles on project outcomes, including reduced risk, increased customer satisfaction, and improved team productivity.

7. Design communication, planning, and deployment strategies appropriate for an agile project based on given scenario requirements.
