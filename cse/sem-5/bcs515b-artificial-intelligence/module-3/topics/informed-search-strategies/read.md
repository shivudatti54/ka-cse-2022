# Informed Search Strategies

## Introduction

Informed search strategies, also known as heuristic search, represent a significant advancement over uninformed search methods by incorporating domain-specific knowledge to guide the search process toward goal states more efficiently. While techniques like Breadth-First Search (BFS) and Depth-First Search (DFS) explore search spaces systematically without any guidance, informed search leverages heuristic functions to estimate the cost from any given state to the goal, dramatically reducing the number of nodes explored.

In artificial intelligence applications, search problems arise in various domains including puzzle solving (8-puzzle, Rubik's cube), pathfinding in maps, game playing, and automated planning. The fundamental challenge is finding optimal or near-optimal solutions efficiently within reasonable time and memory constraints. Informed search strategies address this challenge by using problem-specific knowledge to prioritize promising paths, making them essential for practical AI applications.

This module covers two primary informed search algorithms: Greedy Best-First Search and A\* Search, along with their properties, variants, and practical considerations for implementation in 's Artificial Intelligence curriculum.

## Key Concepts

### Heuristic Function

A heuristic function h(n) estimates the cost of the cheapest path from node n to a goal state. It provides domain-specific knowledge about the problem and is critical to the performance of informed search algorithms. For example, in the 8-puzzle problem, common heuristics include:

- **h1 (Misplaced Tiles)**: Count the number of tiles not in their goal position
- **h2 (Manhattan Distance)**: Sum of distances of each tile from its goal position

The heuristic function must satisfy certain properties depending on the search algorithm used. A heuristic is denoted as h(n) where n is any node in the search tree or graph.

### Greedy Best-First Search

Greedy Best-First Search expands the node that appears to be closest to the goal based on the heuristic value h(n). The algorithm selects the node with the lowest heuristic cost, regardless of the path cost incurred to reach that node.

**Algorithm:**

```
function GREEDY-BEST-FIRST-SEARCH(problem) returns a solution node
 node ← MAKE-NODE(problem.INITIAL-STATE)
 frontier ← a priority queue ordered by h(node)
 explored ← an empty set

 loop do
 if frontier is empty then return failure
 node ← POP(frontier)
 if problem.GOAL-TEST(node.STATE) then return node
 add node.STATE to explored
 for each successor in EXPAND(node, problem) do
 if successor.STATE not in explored or frontier then
 frontier ← INSERT(successor, frontier)
```

**Properties:**

- Not complete in infinite state spaces (can get stuck in loops)
- Not optimal; may not find the shortest path
- Time and space complexity: O(b^m) where m is the maximum depth
- Very fast but can be misled by poor heuristics

### A\* Search

A\* Search combines the advantages of Uniform-Cost Search and Greedy Best-First Search by considering both the path cost g(n) and the heuristic estimate h(n). The evaluation function is f(n) = g(n) + h(n), where:

- g(n) = cost from start to node n
- h(n) = estimated cost from n to goal
- f(n) = estimated total cost of the solution through n

**Algorithm:**

```
function A*-SEARCH(problem) returns a solution node
 node ← MAKE-NODE(problem.INITIAL-STATE)
 g-cost[node] ← 0
 frontier ← a priority queue ordered by f(n) = g(n) + h(n)
 explored ← an empty hash table

 loop do
 if frontier is empty then return failure
 node ← POP(frontier) // node with lowest f-value
 if problem.GOAL-TEST(node.STATE) then return node
 add node.STATE to explored
 for each successor in EXPAND(node, problem) do
 successor.g-cost ← node.g-cost + action.cost
 successor.f-cost ← successor.g-cost + h(successor)
 if successor.STATE not in explored then
 if successor.STATE not in frontier with lower g-cost then
 frontier ← INSERT(successor, frontier)
```

**Properties:**

- Complete if b is finite
- Optimal if h(n) is admissible
- Time complexity: O(b^d) where d is solution depth
- Space complexity: Explores all nodes within the f-cost bound, can be expensive in memory

### Admissible Heuristic

A heuristic h(n) is **admissible** if it never overestimates the true cost to reach the goal from any node. Formally: 0 ≤ h(n) ≤ h*(n), where h*(n) is the true minimum cost. An admissible heuristic is optimistic—it may underestimate but never overestimate costs.

**Example:** In pathfinding, straight-line distance is admissible because it's always shorter than or equal to the actual road distance.

### Consistent (Monotone) Heuristic

A heuristic h(n) is **consistent** (or monotone) if for every node n and every successor n' generated by an action with cost c:
h(n) ≤ c + h(n')

Consistency is a stronger condition than admissibility. For A\* search on graphs, a consistent heuristic guarantees optimality and prevents re-expansion of nodes.

### Dominance

If heuristic h1(n) ≥ h2(n) for all non-goal states, then h1 dominates h2. A dominant heuristic explores fewer nodes and is therefore better for A\* search. The more accurate the heuristic, the more efficient the search.

## Examples

### Example 1: 8-Puzzle Problem using A\* Search

**Problem:** Start state to goal state in 8-puzzle using Manhattan Distance heuristic.

**Given:**

- Start:

```
1 2 3
4 5 6
7 8 _
```

- Goal:

```
1 2 3
4 5 6
7 8 _
```

- Actually, let's use a different start to make it interesting.

**Start:**

```
1 2 3
4 _ 6
7 5 8
```

**Goal:**

```
1 2 3
4 5 6
7 8 _
```

**Step-by-step with Manhattan Distance:**

Initial node:

- State:

```
1 2 3
4 _ 6
7 5 8
```

- g(n) = 0
- h(n) = Manhattan distance = |1-1|+|1-1|+|1-1| + |2-2|+|2-3|+|2-3| + |3-3|+|3-2|+|3-3| = 0 + 1 + 1 = 2
- f(n) = 0 + 2 = 2

Expand node, generate successors (blank moves up, down, left, right), calculate f for each, continue until goal is reached.

The algorithm will find the optimal solution with minimal node expansions due to the heuristic guidance.

### Example 2: Comparing Greedy vs A\* Search

**Problem:** Finding route from Arad to Bucharest in Romania map.

**Greedy Best-First Search:**

- Expands based only on straight-line distance to Bucharest
- Path: Arad → Sibiu → Fagaras → Bucharest (total: 450 km)
- Fast but not optimal

**A\* Search:**

- Uses g(n) (distance traveled) + h(n) (straight-line distance)
- Finds optimal path: Arad → Sibiu → Rimnicu → Pitesti → Bucharest (total: 418 km)

This demonstrates how A\* guarantees optimality while Greedy may miss better solutions.

### Example 3: Verifying Admissibility

**Given Heuristic for 8-puzzle:**

- h1(n) = Number of misplaced tiles
- h2(n) = Sum of Manhattan distances

**Analysis:**

- h1 is admissible: Maximum misplaced tiles is 8, but actual minimum moves to solve is at least 8
- h2 is admissible: Each tile must move at least its Manhattan distance, total moves ≥ sum of distances

**Dominance:** Since h2(n) ≥ h1(n) for all states, h2 dominates h1 and will explore fewer nodes.

## Exam Tips

1. **Remember the f(n) formula**: f(n) = g(n) + h(n), where g(n) is path cost and h(n) is heuristic estimate. This is the core of A\* search.

2. **Know the difference between Greedy and A\***: Greedy uses only h(n), while A* uses both g(n) and h(n). Greedy is faster but not optimal; A* is optimal with admissible heuristics.

3. **A\* optimality condition**: Remember that A\* finds optimal solution if and only if the heuristic is admissible (never overestimates). This is a frequently tested concept.

4. **Consistency requirement**: For graph search (not tree search), A\* requires consistent heuristics to guarantee optimality and avoid re-expanding nodes.

5. **Space complexity awareness**: A* can consume significant memory as it stores all generated nodes in memory. Be prepared to discuss memory-bounded variants like SMA*.

6. **Heuristic properties**: Know that admissibility means h(n) ≤ h\*(n), and consistency requires h(n) ≤ c + h(n') for all successors.

7. **Trade-offs**: More accurate heuristics explore fewer nodes but require more computation to calculate. Balance is essential in practical applications.

8. **Complete vs Optimal**: Greedy is neither complete nor optimal. A\* is both complete and optimal with admissible heuristics.
