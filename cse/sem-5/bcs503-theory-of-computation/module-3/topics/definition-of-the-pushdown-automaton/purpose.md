# Learning Objectives

After studying this topic, you should be able to:

1. Define a Pushdown Automaton (PDA) using its formal 7-tuple representation: M = (Q, Σ, Γ, δ, q₀, Z₀, F)

2. Explain the purpose and function of each component in the PDA definition (states, alphabets, transition function, start state, initial stack symbol, accepting states)

3. Differentiate between deterministic PDA (DPDA) and non-deterministic PDA (NPDA) based on their transition functions

4. Understand and represent PDA configurations using the triple notation (q, w, α)

5. Explain the two methods of acceptance in PDAs: acceptance by final state and acceptance by empty stack

6. Design simple PDAs to recognize context-free languages like {0ⁿ1ⁿ | n ≥ 1} and balanced parentheses

7. Trace the execution of a PDA on a given input string, showing step-by-step configuration changes
