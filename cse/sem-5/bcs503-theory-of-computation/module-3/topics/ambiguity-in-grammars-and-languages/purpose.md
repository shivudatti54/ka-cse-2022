# Learning Objectives

After studying this topic, you should be able to:

1. Define and explain the concept of ambiguity in context-free grammars with precise mathematical formulation.

2. Identify whether a given grammar is ambiguous by constructing multiple parse trees for the same string.

3. Distinguish between ambiguous and unambiguous grammars with specific examples from programming language constructs.

4. Explain the concept of inherent ambiguity and provide examples of inherently ambiguous languages.

5. Convert ambiguous grammars into unambiguous grammars by introducing precedence levels and restructuring productions.

6. Analyze common sources of ambiguity in programming language grammars, such as the if-else problem and operator precedence.

7. Understand the practical implications of ambiguity in compiler design and parsing algorithms.

8. Apply techniques like left factoring and left recursion removal to resolve grammar ambiguities.
