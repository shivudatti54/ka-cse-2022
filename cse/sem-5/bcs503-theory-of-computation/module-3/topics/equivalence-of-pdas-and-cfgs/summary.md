# Equivalence of Pushdown Automata and Context-Free Grammars - Summary

## Key Definitions and Concepts

- **Equivalence Theorem:** A language L is context-free if and only if there exists a pushdown automaton that accepts L. This establishes the exact correspondence between CFGs and PDAs.

- **Pushdown Automaton (PDA):** An automaton with a stack memory that enables recognition of languages requiring matching pairs (like balanced parentheses).

- **Context-Free Grammar (CFG):** A grammar where productions have a single nonterminal on the left side, capable of generating recursive structures.

- **Acceptance by Final State:** PDA accepts if after consuming input, it reaches a designated final state.

- **Acceptance by Empty Stack:** PDA accepts if after consuming input, the stack becomes empty.

## Important Formulas and Theorems

- **CFG to PDA Construction:**
  - δ(q, ε, Z₀) = {(p, S)} - Initialize
  - δ(p, ε, A) = {(p, α) | A → α ∈ P} - Apply productions
  - δ(p, a, a) = {(p, ε)} - Match terminals

- **PDA to CFG Construction:**
  - Create variables [pXq] for states p, q and stack symbol X
  - Productions derived from each transition δ(q, a, X) = {(r, Y₁...Yₖ)}
  - Start variable S generates [q₀Z₀q] for final states q

- **Equivalence of Acceptance Modes:** Languages accepted by empty stack PDAs = languages accepted by final state PDAs

- **Knuth-Thompson Theorem:** A grammar is LR(k) if and only if its corresponding PDA is deterministic

## Key Points

- The equivalence between CFGs and PDAs is fundamental: every context-free language can be generated by a CFG and recognized by a PDA.

- CFG to PDA conversion uses the stack to simulate leftmost derivations by replacing nonterminals with production right-hand sides.

- PDA to CFG conversion creates variables that track the relationship between states and stack symbols during computation.

- Both acceptance criteria (final state and empty stack) for PDAs are equivalent in expressive power.

- Deterministic PDAs correspond to deterministic context-free languages, which are exactly LR(k) grammars.

- The conversion procedures are constructive and can be algorithmically performed.

- The equivalence demonstrates that generative (grammar) and recognition (automata) perspectives describe the same class of languages.

## Common Mistakes to Avoid

- Confusing the roles of terminals and nonterminals in the PDA stack during CFG→PDA conversion.

- Forgetting the initialization step that pushes the start symbol onto the stack.

- Not handling ε-productions correctly when converting from CFG to PDA.

- In PDA to CFG conversion, failing to create all necessary state-pair variables for each stack symbol.

- Assuming that all PDAs can be easily converted to deterministic PDAs; determinism is a special property.

## Revision Tips

- Practice both conversions (CFG→PDA and PDA→CFG) with at least 2-3 examples each to build confidence.

- Focus on understanding the conceptual mapping rather than memorizing all formal details.

- Remember that the stack is the key component that gives PDAs power beyond finite automata.

- Review related concepts: CFL pumping lemma, normal forms (CNF, GNF), and closure properties of CFLs.
