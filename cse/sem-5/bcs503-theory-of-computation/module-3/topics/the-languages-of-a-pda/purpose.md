# Learning Objectives

After studying this topic, you should be able to:

1. Define a Pushdown Automaton (PDA) and explain its seven components: states, input alphabet, stack alphabet, transition function, initial state, initial stack symbol, and final states.

2. Explain the concept of acceptance by final state and determine whether a given string is accepted by a PDA using this criterion.

3. Explain the concept of acceptance by empty stack and determine whether a given string is accepted by a PDA using this criterion.

4. Construct a PDA that accepts a given context-free language by final state or empty stack, demonstrating proficiency in stack operations (push, pop, replace).

5. Prove the equivalence between acceptance by final state and acceptance by empty stack for pushdown automata.

6. Distinguish between deterministic and non-deterministic PDAs and explain the implications for language recognition power.

7. Analyze the relationship between pushdown automata and context-free languages, understanding that PDAs accept exactly the CFLs.
