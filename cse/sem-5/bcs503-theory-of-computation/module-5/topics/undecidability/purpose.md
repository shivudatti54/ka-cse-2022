# Learning Objectives

After studying this topic, you should be able to:

1. Define decidability, undecidability, recursive languages, and recursively enumerable languages with precise technical accuracy.

2. Explain Alan Turing's proof of the undecidability of the halting problem using diagonalization and self-reference arguments.

3. Apply Rice's Theorem to determine whether given language properties are decidable or undecidable.

4. Use the reduction technique to prove new problems undecidable by reducing known undecidable problems to them.

5. Identify and explain at least five well-known undecidable problems in computer science.

6. Distinguish between the three possible outcomes for a Turing machine on an input: accept, reject, and loop.

7. Understand the practical implications of undecidability for software engineering, compiler design, and program verification.
