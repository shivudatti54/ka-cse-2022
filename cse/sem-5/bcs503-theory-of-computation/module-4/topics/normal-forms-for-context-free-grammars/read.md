# Normal Forms for Context-Free Grammars

## 1. Introduction and Theoretical Background

Normal forms for context-free grammars (CFGs) represent standardized structural configurations that facilitate the theoretical analysis and practical processing of grammars. A **normal form** is a canonical form to which any grammar in a certain class can be equivalent transformed without altering the language it generates. In the theory of computation, context-free grammars serve as the formal foundation for programming language syntax and compiler design, making their normal forms essential tools for both theoretical and applied computer science.

The significance of normal forms in formal language theory cannot be overstated. They provide a uniform structure that simplifies the proof of theoretical properties, enables the design of efficient parsing algorithms, and facilitates the construction of equivalent automata. Two particularly important normal forms are **Chomsky Normal Form (CNF)** and **Greibach Normal Form (GNF)**, each with distinct characteristics and applications.

The Chomsky Normal Form, named after Noam Chomsky who formalized it in 1957, restricts production rules to either generate single terminal symbols or pairs of non-terminal symbols. This restriction creates a binary tree structure for derivations, which is fundamental to various parsing algorithms and complexity analyses. The Greibach Normal Form, named after Sheila Greibach who defined it in 1965, requires all productions to begin with a terminal symbol, making it particularly suitable for top-down parsing and the construction of pushdown automata.

## 2. Chomsky Normal Form (CNF)

### 2.1 Formal Definition

A context-free grammar G = (V, Σ, P, S) is said to be in **Chomsky Normal Form** if all production rules in P satisfy one of the following conditions:

1. **Terminal productions**: A → a, where A ∈ V is a non-terminal, a ∈ Σ is a terminal symbol
2. **Binary productions**: A → B C, where A, B, C ∈ V are non-terminals, and neither B nor C is the start symbol S (unless S appears on the left-hand side)

Additionally, the start symbol S should not appear on the right-hand side of any production rule. The only exception to the ε-free requirement is the optional production S → ε, which may be retained if the empty string belongs to the language generated by the grammar.

**Definition 2.1 (CNF)**: A CFG G = (V, Σ, P, S) is in CNF if every production in P has one of the forms:

- A → a, where a ∈ Σ
- A → B C, where B, C ∈ V \ {S}

### 2.2 Key Properties of CNF

The Chomsky Normal Form exhibits several important properties that make it theoretically and practically valuable:

**Property 2.1 (Derivation Tree Structure)**: In a grammar in CNF, every internal node in a derivation tree has exactly two children, creating a perfect binary tree structure. This property is crucial for the CYK parsing algorithm and for establishing the O(n³) complexity bound for the membership problem for context-free languages.

**Property 2.2 ( pumping lemma corollary)**: The pumping lemma for context-free languages can be more directly applied to grammars in CNF, as the binary structure of productions simplifies the analysis of variable repetitions in derivations.

**Property 2.3 (Chomsky Hierarchy)**: Grammars in CNF are context-free grammars by definition, and they generate exactly the same class of languages as all context-free grammars (except possibly the empty string).

### 2.3 Theorem: Existence of CNF

**Theorem 2.1 (Chomsky's Normal Form Theorem)**: Every context-free grammar that does not generate the empty string (i.e., L(G) ⊄ {ε}) can be converted to an equivalent grammar in Chomsky Normal Form.

**Proof Outline**: The conversion proceeds through a series of transformations, each preserving language equivalence:

1. **Elimination of ε-productions**: For each nullable non-terminal A (where A ⇒\* ε), remove all occurrences of A from the right-hand side of productions and add new productions accordingly.

2. **Elimination of unit productions**: For each pair of non-terminals (A, B) where A ⇒\* B through a sequence of unit productions, add productions A → α for each production B → α where α is not a single non-terminal.

3. **Removal of useless symbols**: Eliminate non-terminals that either cannot derive any terminal string (unreachable) or are not reachable from the start symbol.

4. **Conversion to binary form**: For productions A → X₁ X₂ ... Xₙ where n > 2, introduce new non-terminals and replace the production with a chain of binary productions.

Each step preserves the language generated by the grammar while approaching the CNF structure. ∎

## 3. Conversion Algorithm for CNF

The transformation of an arbitrary CFG to CNF involves four distinct phases. Understanding each phase thoroughly is essential for both theoretical comprehension and practical application.

### 3.1 Step 1: Removal of ε-Productions

An ε-production (also called a nullable production) is a production of the form A → ε. The presence of ε-productions complicates many algorithms, so their elimination is the first step in CNF conversion.

**Algorithm 3.1 (ε-Production Elimination)**:

1. Identify the set NULL of all nullable non-terminals:
   - Initialize NULL = {A | A → ε ∈ P}
   - For each production B → α, if all symbols in α are in NULL, add B to NULL
   - Repeat until no new non-terminals are added

2. For each production A → X₁X₂...Xₖ (where k ≥ 1):
   - For each subset of positions where nullable non-terminals appear
   - Generate a new production with those nullable symbols removed
   - Do not generate the empty production

3. Remove all original ε-productions from P

**Example 3.1**: Given G: S → AB, A → aA | ε, B → bB | ε

- NULL = {A, B, S} (since S → AB and both A, B are nullable)
- From A → aA, generate: aA, a (by removing A)
- From B → bB, generate: bB, b (by removing B)
- From S → AB, generate: AB, A, B (by removing nullable symbols)
- Final productions: S → AB | A | B, A → aA | a, B → bB | b

**Complexity**: The ε-elimination algorithm runs in O(|P| × 2^|V|) time in the worst case, where |P| is the number of productions and |V| is the number of non-terminals.

### 3.2 Step 2: Removal of Unit Productions

A **unit production** (also called a rename) is a production of the form A → B, where both A and B are non-terminals. These productions create chains in derivations that complicate analysis.

**Algorithm 3.2 (Unit Production Elimination)**:

1. Compute the relation UNIT(A, B) = true if A ⇒\* B through a sequence of unit productions only

2. For each pair (A, B) where UNIT(A, B) is true:
   - For each production B → α where α is not a single non-terminal
   - Add production A → α to P

3. Remove all unit productions from P

**Example 3.2**: Given G: S → A | B, A → a, B → b

- UNIT(S, A) = true (S → A)
- UNIT(S, B) = true (S → B)
- Add: S → a, S → b
- Final: S → a | b, A → a, B → b (unit productions removed)

### 3.3 Step 3: Removal of Useless Symbols

A non-terminal is **useless** if it either cannot derive any terminal string (it is non-generating) or cannot be reached from the start symbol (it is non-reachable).

**Algorithm 3.3 (Useless Symbol Removal)**:

1. **Remove non-generating symbols**:
   - Initialize GENERATING with terminals
   - For each production A → α, if all symbols in α are in GENERATING, add A
   - Repeat until fixed point
   - Remove all non-generative non-terminals

2. **Remove unreachable symbols**:
   - Initialize REACHABLE = {S}
   - For each production A → α, if A ∈ REACHABLE, add all non-terminals in α to REACHABLE
   - Repeat until fixed point
   - Remove all non-reachable symbols

### 3.4 Step 4: Conversion to Binary Form

This step ensures that all productions have at most two symbols on the right-hand side.

**Algorithm 3.4 (Binary Form Conversion)**:

For each production A → X₁X₂...Xₙ where n > 2:

1. Create new non-terminals Y₁, Y₂, ..., Yₙ₋₂
2. Replace the production with: A → X₁Y₁, Y₁ → X₂Y₂, ..., Yₙ₋₃ → Xₙ₋₂Xₙ₋₁Yₙ₋₂, Yₙ₋₂ → Xₙ₋₁Xₙ
3. Add productions for any terminal appearing as Xᵢ by introducing new non-terminals

**Example 3.3**: Given production S → ABCD (n = 4)

- Introduce: Y₁, Y₂
- Replace with: S → A Y₁, Y₁ → B Y₂, Y₂ → C D

## 4. Greibach Normal Form (GNF)

### 4.1 Formal Definition

A context-free grammar G = (V, Σ, P, S) is in **Greibach Normal Form** if all production rules have the form:

**A → a α**

where 'a' is a terminal symbol (the leftmost symbol on the right-hand side) and α is a (possibly empty) string of non-terminals from V. Thus, every production begins with a terminal symbol followed by zero or more non-terminals.

**Definition 4.1 (GNF)**: A CFG G = (V, Σ, P, S) is in GNF if every production in P has the form A → aα, where a ∈ Σ and α ∈ V\*.

**Note**: The empty string ε is permitted only in the form S → ε if S is the start symbol and ε ∈ L(G).

### 4.2 Key Properties of GNF

**Property 4.1 (Top-Down Parsing)**: GNF is particularly suited for recursive descent parsing and LL(k) parsers because each production uniquely determines the next step in derivation based on the next input symbol.

**Property 4.2 (PDA Construction)**: Given a grammar in GNF, a pushdown automaton recognizing the same language can be constructed directly. Each production A → aα corresponds to a transition that pops A and pushes α on the stack.

**Property 4.3 (Left-Recursion Elimination)**: GNF requires the elimination of left recursion, which makes parsing more efficient and prevents infinite loops in top-down parsers.

### 4.3 Theorem: Existence of GNF

**Theorem 4.1 (Greibach's Normal Form Theorem)**: Every context-free grammar can be converted to an equivalent grammar in Greibach Normal Form.

**Proof Outline**: The conversion involves:

1. First convert the CFG to CNF (eliminating ε-productions, unit productions, and making productions binary)

2. For each non-terminal ordered as A₁, A₂, ..., Aₙ, modify productions to ensure they start with terminals:
   - For productions Aᵢ → Aⱼα where i ≤ j, substitute the definitions of Aⱼ
   - This process continues iteratively until all productions start with terminals

3. Eliminate any remaining left recursion through standard techniques

The key insight is that by properly ordering non-terminals and substituting definitions, we can always ensure productions begin with terminal symbols. ∎

### 4.4 Conversion Algorithm for GNF

**Algorithm 4.1 (CFG to GNF Conversion)**:

1. **Eliminate ε-productions** using Algorithm 3.1

2. **Eliminate unit productions** using Algorithm 3.2

3. **Convert to CNF** using Algorithms 3.3 and 3.4

4. **Arrange non-terminals** in some order A₁, A₂, ..., Aₙ

5. **For i from 1 to n**:
   - For each production Aᵢ → Aⱼα with j < i:
     - Substitute the right-hand side of each production Aⱼ → β
   - For each production Aᵢ → aα (already starting with terminal):
     - Keep as is

6. **Eliminate any ε-productions** that may have been introduced

**Example 4.1**: Convert S → AB, A → aA | a, B → b to GNF

- Already in CNF: S → AB, A → aA | a, B → b
- Order: S, A, B

- Processing A: A → aA | a (already starts with terminal)

- Processing B: B → b (already starts with terminal)

- Processing S: S → AB
  - Substitute A: S → aAB | aB
  - Substitute B: S → aAb | ab | aB

- Final GNF: S → aAB | aB | aAb | ab, A → aA | a, B → b

## 5. Detailed Worked Examples

### Example 5.1: Complete CNF Conversion

**Given Grammar G:**
S → aSb | ab | ε

**Solution:**

**Step 1: Remove ε-productions**

The start symbol S is nullable (S → ε exists).

For S → aSb: removing the nullable S gives aab (new production)
For S → ab: no nullable symbols to remove
For S → ε: remove this production

New productions: S → aSb | ab | aab

**Step 2: Remove unit productions**

Check for unit productions A → B. In this case, there are none.

**Step 3: Remove useless symbols**

All symbols can derive terminal strings and are reachable from S. No removal needed.

**Step 4: Convert to binary form**

Introduce non-terminals for terminals:

- Let A → a
- Let B → b

Now convert each production:

- S → aSb becomes S → A S B
- S → ab becomes S → A B
- S → aab becomes S → A A B

Now ensure RHS has at most 2 symbols:

- For S → A S B: introduce X₁ → S B
  S → A X₁
- For S → A B: already binary
- For S → A A B: introduce X₂ → A B
  S → A X₂

**Final CNF:**
S → A B | A X₁ | A X₂
X₁ → S B
X₂ → A B
A → a
B → b

### Example 5.2: Complete GNF Conversion

**Given Grammar G:**
S → ASB | ε
A → aA | a
B → bB | b

**Solution:**

**Step 1: Remove ε-productions**

S is nullable. For S → ASB, removing S gives AB.

New productions: S → ASB | AB, A → aA | a, B → bB | b

**Step 2: Remove unit productions**

No unit productions exist in this grammar.

**Step 3: CNF Conversion**

Introduce terminals as non-terminals:

- Let A₁ → a
- Let B₁ → b

Convert S → ASB:

- S → A S B | A S B₁ | A₁ S B | A₁ S B₁

Convert S → AB:

- S → A B | A₁ B | A B₁ | A₁ B₁

Convert A → aA: A → A₁ A
Convert A → a: A → A₁
Convert B → bB: B → B₁ B
Convert B → b: B → B₁

Now convert all RHS with more than 2 symbols to binary form.

**Step 4: Convert to GNF**

Order non-terminals: S, A, B, A₁, B₁, X₁, X₂, ...

Process each production to ensure it starts with a terminal:

For A₁ → a (already in GNF)
For B₁ → b (already in GNF)

For A → A₁: substitute A₁ → a: A → a (in GNF)
For B → B₁: substitute B₁ → b: B → b (in GNF)

For S → A S B: substitute A → a: S → a S B (in GNF)
For S → A S B₁: S → a S B₁
For S → A₁ S B: A₁ → a: S → a S B
For S → A₁ S B₁: S → a S B₁

Continue this substitution for all productions until all start with terminals.

**Final GNF Productions (partial):**
S → aSB | aSB₁ | aS | aSB | aSB₁
A → aA | a
B → bB | b

## 6. Applications and Significance

### 6.1 CYK Parsing Algorithm

The CYK algorithm (Cocke-Younger-Kasami) is a classic parsing algorithm for context-free languages that runs in O(n³) time. It requires the input grammar to be in CNF to function efficiently.

**Algorithm 6.1 (CYK)**:
Given a string w of length n and a grammar in CNF:

1. Initialize a triangular table T where T[i][j] contains non-terminals that derive the substring w[i:j]
2. For each position i, add A if A → w[i] is a production
3. For each length l from 2 to n, for each position i, for each split k:
   - Add A to T[i][j] if there exists B ∈ T[i][k] and C ∈ T[k][j] such that A → BC
4. The string is accepted if S ∈ T[0][n-1]

The CNF restriction ensures that each derivation step combines exactly two substrings, making the algorithm straightforward to implement.

### 6.2 Pushdown Automata Construction

Given a grammar in GNF, a pushdown automaton can be constructed directly:

- For each production A → aα in GNF, create a transition:
  - δ(q, a, A) = {(q, α)}
- The start state has transition δ(q₀, ε, S) = {(q₁, ε)}
- Accept by empty stack or final state

This direct correspondence between GNF productions and PDA transitions demonstrates the equivalence between context-free grammars and pushdown automata.

## 7. Complexity Analysis

The conversion of CFGs to normal forms involves polynomial-time algorithms:

| Step                        | Time Complexity | Space Complexity |
| --------------------------- | --------------- | ---------------- |
| ε-production elimination    | O(P × 2^V)      | O(V + P)         |
| Unit production elimination | O(V³ + VP)      | O(V²)            |
| Useless symbol removal      | O(VP)           | O(V + P)         |
| Binary form conversion      | O(P × L)        | O(P + L)         |
| GNF conversion              | O(n³ × P)       | O(n × P)         |

Where V = |V|, P = |P|, n = number of non-terminals, L = maximum length of RHS
