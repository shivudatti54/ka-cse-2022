# Learning Objectives

After studying this topic, you should be able to:

1. Define Border Gateway Protocol (BGP) and explain its role as an exterior gateway protocol in Internet routing between autonomous systems.

2. Distinguish between External BGP (eBGP) and Internal BGP (iBGP), including their operational differences, default behaviors, and appropriate use cases in network design.

3. Describe the four BGP message types (OPEN, KEEPALIVE, UPDATE, NOTIFICATION) and explain the state machine transitions during BGP session establishment.

4. Analyze BGP path attributes (AS_Path, Local Preference, Origin, MED, Next Hop) and explain how they influence route selection and enable policy-based routing.

5. Apply the BGP route selection process to determine which path BGP will choose when multiple routes to the same destination exist.

6. Configure basic BGP peering relationships between routers in different autonomous systems and verify session establishment.

7. Implement basic traffic engineering techniques using BGP attributes such as Local Preference and AS_Path prepending.

8. Evaluate BGP stability mechanisms including route flap damping and describe considerations for BGP convergence in enterprise networks.
