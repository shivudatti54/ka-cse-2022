# Open Shortest Path First (OSPF) Protocol

## Introduction

Open Shortest Path First (OSPF) is a widely-used interior gateway protocol (IGP) designed for IP networks within a single autonomous system. Developed by the Internet Engineering Task Force (IETF) as a successor to RIP, OSPF implements the Dijkstra's shortest path first algorithm to calculate the optimal routing paths. It is classified as a link-state routing protocol, where each router maintains a complete map of the network topology and computes the best path to each destination independently.

OSPF became an industry standard due to its superior scalability, faster convergence times, and support for variable-length subnet masking (VLSM). Unlike distance-vector protocols, OSPF routers exchange link-state advertisements (LSAs) to build a consistent view of the network. This approach eliminates routing loops and provides more accurate path selection based on metrics like bandwidth. OSPF is extensively deployed in enterprise networks, service provider backbones, and data centers where hierarchical network design and rapid convergence are critical requirements.

## Key Concepts

### OSPF Areas and Hierarchical Design

OSPF implements a two-level hierarchical structure using areas to control routing updates and reduce overhead. The backbone area (Area 0) serves as the central hub connecting all other areas. All non-backbone areas must connect to Area 0 through area border routers (ABRs). This hierarchical design minimizes the size of the link-state database (LSDB) on each router, reduces LSA flooding, and localizes routing changes within specific areas.

**Area Types:**

- **Backbone Area (Area 0):** The core area that connects all other OSPF areas; cannot be subdivided
- **Stub Area:** An area that does not receive external (Type 5 LSAs); uses default routes for external destinations
- **Totally Stub Area:** Cisco-specific area that blocks both external routes and summary routes (except default)
- **Not-So-Stubby Area (NSSA):** Allows injection of external routes while maintaining stub area properties

### OSPF Router Types

OSPF defines several router roles based on their position in the network topology:

- **Internal Router (IR):** All interfaces belong to the same OSPF area
- **Backbone Router (BR):** At least one interface in Area 0
- **Area Border Router (ABR):** Connects two or more OSPF areas; maintains separate LSDB for each connected area
- **Autonomous System Boundary Router (ASBR):** Connects OSPF domain to external networks (different AS); performs route redistribution

### OSPF Packet Types

OSPF uses five distinct packet types for neighbor discovery, adjacency formation, and routing information exchange:

1. **Hello Packets:** Discover neighbors and maintain neighbor relationships; sent to multicast address 224.0.0.5 every 10 seconds (30 seconds on NBMA networks)
2. **Database Description (DBD) Packets:** Summarize the router's LSDB for synchronization
3. **Link-State Request (LSR) Packets:** Request specific LSA records from neighboring routers
4. **Link-State Update (LSU) Packets:** Respond to LSR packets; contain complete LSA information
5. **Link-State Acknowledgment (LSAck) Packets:** Acknowledge receipt of LSU packets

### Link-State Advertisements (LSAs)

OSPF routers generate different LSA types to describe various network elements:

- **Type 1 (Router LSA):** Generated by every OSPF router; describes the router's interfaces and neighbors within the same area
- **Type 2 (Network LSA):** Generated by designated routers (DR); describes all routers attached to a broadcast network
- **Type 3 (Summary LSA):** Generated by ABRs; describes routes to other areas (inter-area routes)
- **Type 4 (Summary LSA):** Generated by ASBRs; describes the location of ASBRs to other areas
- **Type 5 (External LSA):** Generated by ASBRs; describes routes to external destinations

### OSPF Network Types

OSPF adapts its behavior based on underlying network media:

- **Broadcast Multi-Access:** Ethernet networks; elects DR and BDR to reduce adjacencies
- **Point-to-Point:** Serial links between two routers; no DR/BDR election
- **Non-Broadcast Multi-Access (NBMA):** Frame Relay, ATM networks; requires manual neighbor configuration
- **Point-to-Multipoint:** Treats NBMA as multiple point-to-point links; no DR/BDR needed
- **Virtual Links:** Tunnels that non-backbone areas to connect isolated Area 0 segments

### OSPF Metric (Cost)

OSPF uses cost as its metric, calculated based on interface bandwidth. The default cost formula is: **Cost = Reference Bandwidth / Interface Bandwidth**. The default reference bandwidth is 100 Mbps. Common interface costs include: FastEthernet (1), Serial 1544 Kbps (64), and 10 Mbps Ethernet (10).

### Designated Router (DR) and Backup DR

In multi-access networks, OSPF elects a Designated Router (DR) and Backup Designated Router (BDR) to optimize adjacency formation. All other routers form full adjacencies only with DR and BDR, reducing OSPF traffic from O(n×n) to O(n). The router with highest OSPF priority (0-255, default 1) becomes DR; ties broken by highest router ID.

## Examples

### Example 1: Calculating OSPF Cost and Path Selection

**Problem:** Given a network with three routers connected as follows:

- Router A <--> Router B via FastEthernet (100 Mbps)
- Router B <--> Router C via Serial link (1.544 Mbps)
- Router A <--> Router C via Serial link (1.544 Mbps)

Calculate the cost of each path from Router A to Router C and determine which path OSPF will choose.

**Solution:**

**Step 1: Calculate individual link costs**

- FastEthernet (100 Mbps): Cost = 100/100 = 1
- Serial (1.544 Mbps): Cost = 100/1.544 ≈ 64 (rounded)

**Step 2: Calculate path costs**

- Path 1 (A→B→C): Cost = 1 + 64 = 65
- Path 2 (A→C direct): Cost = 64

**Step 3: Determine chosen path**
OSPF chooses the path with lowest cumulative cost. Therefore, OSPF will select the direct path from Router A to Router C with total cost 64.

### Example 2: OSPF Adjacency State Machine

**Problem:** Two OSPF routers are connected via a point-to-point link. Describe the adjacency states they will transition through until reaching Full state.

**Solution:**

**Step 1: Down State**
Initial state; no Hello packets received yet.

**Step 2: Init State**
Router receives Hello packet from neighbor but its own router ID is not in the neighbor's Hello packet.

**Step 3: 2-Way State**
Routers exchange Hello packets and see each other's router IDs. On point-to-point links, they proceed to build adjacency. On broadcast networks, this is the final state unless elected as DR/BDR.

**Step 4: ExStart State**
Routers establish master/slave relationship for DBD exchange; determine initial sequence number.

**Step 5: Exchange State**
Routers exchange DBD packets describing their LSDB contents.

**Step 6: Loading State**
Routers send LSR packets for missing or outdated LSAs; receive LSU responses.

**Step 7: Full State**
LSDBs synchronized; routers maintain full adjacency; OSPF routing can begin.

### Example 3: Configuring OSPF and Verifying Adjacency

**Problem:** Router A has Router ID 1.1.1.1 and is connected to Router B (Router ID 2.2.2.2) via FastEthernet interface. Configure OSPF process 10 in Area 0 and verify the adjacency.

**Solution:**

**Configuration Commands:**

```
RouterA# configure terminal
RouterA(config)# router ospf 10
RouterA(config-router)# network 192.168.1.0 0.0.0.255 area 0
RouterA(config-router)# exit
RouterA(config)# interface fastethernet 0/0
RouterA(config-if)# ip ospf 10 area 0
```

**Verification Commands:**

```
RouterA# show ip ospf neighbor
RouterA# show ip ospf interface fastethernet 0/0
RouterA# show ip ospf database
```

**Expected Output:**

- Neighbor state should show "FULL" indicating successful adjacency
- Interface should show DR/BDR election results
- Database should display Type 1 and Type 2 LSAs

## Exam Tips

1. **OSPF vs RIP Differences:** Remember OSPF is link-state (uses Dijkstra's algorithm, faster convergence, supports VLSM) while RIP is distance-vector (uses Bellman-Ford, slower convergence, no VLSM).

2. **LSA Type Functions:** Type 1 describes router's own links; Type 2 describes multi-access network members; Type 3 summarizes inter-area routes; Type 4 locates ASBRs; Type 5 carries external routes.

3. **Area Types and Their Characteristics:** Stub areas block Type 5 LSAs and use default routes; NSSA allows limited external route injection; Totally Stub blocks both Type 3 and Type 5 except default.

4. **DR/BDR Election Criteria:** Highest OSPF priority (0-255, default 1) wins; Router ID breaks ties; Priority 0 makes router ineligible.

5. **Hello Timer Values:** 10 seconds for broadcast and point-to-point; 30 seconds for NBMA networks; Dead timer is four times the Hello interval.

6. **OSPF Packet Types in Order:** Remember sequence: Hello → DBD → LSR → LSU → LSAck for adjacency establishment.

7. **Router ID Selection:** Highest loopback IP; highest active physical interface IP; must be manually configured if no loopback exists.

8. **Cost Calculation:** Remember the formula Cost = Reference Bandwidth (default 100 Mbps) / Interface Bandwidth. Higher bandwidth means lower cost.

9. **Adjacency Requirements:** Must match Hello/Dead timers, area ID, authentication, and OSPF network type for adjacency to form.

10. **Virtual Links:** Used to connect an area to Area 0 when physical connectivity is impossible; must be configured on both endpoints.
