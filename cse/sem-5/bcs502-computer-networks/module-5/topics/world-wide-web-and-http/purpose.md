# Learning Objectives

After studying this topic, you should be able to:

1. Explain the client-server architecture of the World Wide Web and the roles of web browsers and web servers.

2. Describe the structure and components of a Uniform Resource Locator (URL), including scheme, hostname, port, path, query string, and fragment.

3. Understand the HTTP protocol fundamentals, including the request-response model and stateless nature of HTTP transactions.

4. Identify and explain different HTTP request methods (GET, POST, PUT, DELETE, HEAD, OPTIONS, PATCH) and their appropriate use cases.

5. Interpret HTTP status codes and categorize them into informational, success, redirection, client error, and server error classes.

6. Analyze HTTP request and response message formats, including headers and message bodies.

7. Explain how cookies enable session management in stateless HTTP and describe cookie attributes and types.

8. Compare HTTP and HTTPS, understanding the security benefits provided by SSL/TLS encryption.
