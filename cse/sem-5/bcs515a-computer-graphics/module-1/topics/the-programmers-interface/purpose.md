# Learning Objectives

After studying this topic, you should be able to:

1. Explain the purpose and role of graphics APIs in providing an interface between application software and graphics hardware
2. Identify and describe the various coordinate spaces in the graphics transformation pipeline (object, world, view, clip, NDC, screen)
3. Describe the complete transformation pipeline from 3D object coordinates to 2D screen pixels, including the matrices involved at each stage
4. Compare and contrast the fixed-function pipeline with the programmable pipeline model
5. Identify the different programmable shader stages and their roles in the rendering pipeline
6. Explain the purpose and usage of various buffer objects (VBOs, IBOs, UBOs) in graphics programming
7. Understand graphics state management and its impact on rendering performance
8. Write simple shader programs (vertex and fragment) demonstrating understanding of input/output variables and uniform parameters
