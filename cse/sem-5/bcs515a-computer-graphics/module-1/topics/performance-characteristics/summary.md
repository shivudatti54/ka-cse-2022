# Performance Characteristics - Summary

## Key Definitions

- **Frame Rate**: The number of complete images rendered per second, measured in frames per second (fps) or Hertz (Hz)
- **Refresh Rate**: The frequency at which a display device updates the visible image
- **Resolution**: The number of pixels in each dimension (width × height) that the graphics system renders
- **Color Depth**: The number of bits used to represent each pixel's color (bit depth)
- **Fill Rate**: The rate at which a GPU can process and display pixels, measured in pixels or samples per second
- **Memory Bandwidth**: The data transfer rate between GPU and video memory, measured in GB/s
- **Latency**: The delay between user input and corresponding visual feedback
- **Polygon Throughput**: The rate at which a GPU can process geometric primitives, measured in triangles per second

## Important Formulas

- **Frame Time**: T = 1000 / fps (where T is in milliseconds)
- **Pixels per Frame**: Resolution = Width × Height
- **Memory per Frame**: Bytes = Pixels × (Color Depth / 8)
- **Fill Rate**: Samples/sec = Resolution × Samples per Pixel × Frame Rate
- **Bandwidth**: Bytes/sec = Bytes per Frame × Frame Rate

## Key Points

1. Frame rate directly determines the time available per frame—60 fps provides only 16.67ms for all rendering operations.

2. Resolution has a quadratic relationship with workload—doubling resolution quadruples the number of pixels to process.

3. Color depth affects both memory storage and bandwidth requirements proportionally to bit depth.

4. Fill rate becomes critical when rendering large visible polygons or applying per-pixel post-processing effects.

5. Memory bandwidth frequently limits graphics performance more than processing power, especially with high-resolution textures.

6. Latency in graphics pipelines includes input processing, simulation, rendering, and display update time.

7. Target frame rates vary by application: 30 fps minimum for basic games, 60 fps for smooth interaction, 90+ fps for VR to prevent motion sickness.

8. Modern GPUs use parallel architectures to achieve fill rates exceeding 10 Giga-samples per second.

9. Performance characteristics are interdependent—a system might be polygon-bound or fill-rate-bound depending on the scene.

10. Anti-aliasing techniques like MSAA multiply effective resolution requirements by the number of samples.

## Common Mistakes

1. **Confusing frame rate with refresh rate**: Frame rate is generated by the GPU while refresh rate is a property of the display—these are independent but should be matched for optimal results.

2. **Ignoring the impact of anti-aliasing**: Multi-sampling significantly increases computational requirements, often by 2-4x.

3. **Underestimating bandwidth requirements**: Many students calculate only framebuffer bandwidth without considering textures, depth buffers, and intermediate buffers.

4. **Not accounting for memory latency**: High bandwidth doesn't always mean low latency; memory access patterns matter significantly.

5. **Assuming polygon count is the only metric**: A scene with few visible triangles but heavy overdraw can be more fill-rate limited than a scene with many triangles.
