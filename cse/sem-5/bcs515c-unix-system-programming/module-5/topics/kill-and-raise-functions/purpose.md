# Learning Objectives

After studying this topic, you should be able to:

1. Explain the concept of signals in Unix/Linux operating systems and their role in inter-process communication.

2. Describe the purpose and functionality of the kill() function and its parameters (pid and sig).

3. Explain how raise() function sends signals to the current process and its relationship with kill().

4. Implement signal handlers using the signal() function to handle asynchronous events in C programs.

5. Understand process groups and their role in broadcast signal delivery using kill() with negative pid values.

6. Analyze the return values and error conditions (ESRCH, EPERM, EINVAL) associated with kill() and raise() functions.

7. Write complete C programs demonstrating inter-process signal communication using kill() and self-signal generation using raise().

8. Apply knowledge of kill/raise functions to solve real-world process management problems in Unix-like systems.
