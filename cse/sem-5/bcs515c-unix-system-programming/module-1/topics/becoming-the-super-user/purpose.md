# Learning Objectives

After studying this topic, you should be able to:

1. Define the concept of the super user (root) in Linux and explain why such an account is necessary in operating systems.

2. Explain the difference between regular user privileges and root privileges in terms of system access and security implications.

3. Use the `su` command to switch to root user or other user accounts with proper syntax and options.

4. Execute commands with elevated privileges using the `sudo` command and understand its authentication mechanism.

5. Configure user sudo privileges by editing the `/etc/sudoers` file using the `visudo` command.

6. Compare and contrast the `su` and `sudo` commands with respect to security, password requirements, and logging capabilities.

7. Apply best practices for obtaining and using super user privileges in Linux system administration tasks.

8. Demonstrate practical examples of administrative tasks that require super user privileges such as software installation, service management, and user account creation.
