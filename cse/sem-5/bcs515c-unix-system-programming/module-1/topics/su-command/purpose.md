# Learning Objectives

After studying this topic, you should be able to:

1. Understand the purpose and functionality of the `su` command in Linux operating systems

2. Explain the basic syntax and common options of the `su` command including `-`, `-c`, `-l`, and `-m`

3. Demonstrate the ability to switch to root user and other user accounts using appropriate `su` commands

4. Differentiate between `su` and `sudo` commands with respect to authentication and security

5. Execute single commands as other users using the `-c` option without starting interactive shells

6. Understand how environment variables are handled when switching users with and without the login shell option

7. Explain the role of PAM (Pluggable Authentication Modules) in securing the `su` command

8. Apply best practices for using `su` in practical system administration and development scenarios
