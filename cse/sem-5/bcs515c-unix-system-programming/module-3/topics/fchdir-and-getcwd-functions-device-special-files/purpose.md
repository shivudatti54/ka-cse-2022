# Learning Objectives

After studying this topic, you should be able to:

1. Explain the concept of current working directory in Unix/Linux processes and its role in path resolution.

2. Demonstrate the use of the fchdir() system call to change the working directory using an open file descriptor.

3. Implement the getcwd() function to retrieve the absolute pathname of the current working directory with proper error handling.

4. Distinguish between character device files and block device files, explaining their differences and use cases.

5. Understand device major and minor numbers and their significance in device identification.

6. Create device special files programmatically using the mknod() system call.

7. Apply appropriate error handling techniques when working with directory and device file operations.
