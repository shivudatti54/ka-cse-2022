# Learning Objectives

After studying this topic, you should be able to:

1. Describe the complete memory layout of a C program and identify all the sections from lowest to highest memory addresses.

2. Explain the purpose and characteristics of each memory segment: Text Segment, Data Segment, BSS Segment, Heap, and Stack.

3. Differentiate between stack and heap memory allocation, including their growth directions, allocation methods, and lifetime of stored data.

4. Identify where different types of variables (global, local, static, automatic) are stored in the memory layout.

5. Analyze C programs and determine which memory segment each variable belongs to.

6. Understand the concept of stack frames and how function calls use the stack for parameters, return addresses, and local variables.

7. Explain common memory-related problems such as memory leaks, buffer overflows, segmentation faults, and dangling pointers.

8. Apply this knowledge to write efficient and correct C programs, particularly in the context of dynamic memory allocation and pointer manipulation.
