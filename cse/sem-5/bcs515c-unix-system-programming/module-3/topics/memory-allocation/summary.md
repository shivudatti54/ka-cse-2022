Memory allocation in operating systems encompasses techniques for assigning physical memory to processes. Contiguous allocation methods include fixed partitioning (with equal or unequal sizes) and variable partitioning, each presenting trade-offs between simplicity and memory efficiency. Allocation algorithms—First Fit, Best Fit, Worst Fit, and Next Fit—determine how free memory holes are selected to satisfy process requests. Fragmentation manifests as internal waste within allocated blocks or external scattering of usable memory across non-contiguous holes. Paging eliminates external fragmentation by dividing memory into fixed-size frames and pages, using page tables and TLBs for address translation. Segmentation provides logically meaningful variable-sized memory blocks, often combined with paging in modern systems. Understanding these concepts is essential for appreciating operating system memory management design decisions.
