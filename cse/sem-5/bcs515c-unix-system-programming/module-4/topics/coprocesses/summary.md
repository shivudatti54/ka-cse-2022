Coprocesses are background processes with bidirectional communication channels established through unnamed pipes. Introduced in ksh and adopted by Bash 4.0+, they provide file descriptors `${NAME[0]}` for reading and `${NAME[1]}` for writing. Key operations include pipe creation via fork(), file descriptor duplication, and process synchronization using `wait` and `read -t`. Applications range from continuous calculators (bc) to data processors (awk) and interactive server processes. Proper descriptor management and process reaping are essential for writing robust coprocess-based scripts.
