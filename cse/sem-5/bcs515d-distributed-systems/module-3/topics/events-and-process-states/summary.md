Events and process states form the basic analytical units in distributed systems. An event is an instantaneous atomic action classified as internal, send, or receive. A process maintains state comprising program counter, local memory, stack, and channel buffers. Process execution forms a total order of events internally, but across processes only a partial order exists due to the absence of global clocks. Concurrent events lack causal ordering and are incomparable. The happened-before relation (→), introduced by Lamport, defines causal precedence through local ordering, message ordering (send precedes receive), and transitivity. This strict partial order underlies all reasoning about time, causality, and synchronization in distributed systems.
