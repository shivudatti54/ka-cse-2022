SNMP is an application-layer protocol enabling management information exchange between network devices through a manager-agent architecture. The protocol utilizes a hierarchical MIB structure with OID-based object identification. Key operations include GET (retrieve values), SET (modify configuration), GETBULK (efficient bulk retrieval), TRAP (unacknowledged notifications), and INFORM (reliable notifications). SNMPv1 uses plaintext community strings, SNMPv2c adds bulk operations while maintaining weak security, and SNMPv3 implements USM/VACM with authentication and encryption. For IoT deployments, SNMP addresses scale management through GETBULK optimization, handles heterogeneity via standard MIBs, and accommodates resource constraints through protocol flexibility. Comparison with NETCONF/YANG reveals distinct advantages: SNMP's polling model suits operational monitoring while NETCONF's transactional configuration orientation supports zero-touch provisioning. Understanding these protocols enables appropriate management strategy design for diverse IoT ecosystems.
