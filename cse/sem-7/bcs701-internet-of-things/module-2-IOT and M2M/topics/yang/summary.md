NETCONF is a session-based network management protocol operating over secure transports (mandatory SSH) with a four-layer architecture (Transport, RPC, Operations, Content). YANG (RFC 7950) is a data modeling language that defines hierarchical data structures with nodes including leaf, leaf-list, container, list, choice, and grouping. For IoT management, NETCONF-YANG addresses heterogeneity through YANG augmentations, scale through candidate/startup datastores, security through mandatory SSH/TLS, and automation through programmatic RPC operations. A complete YANG module for IoT sensors includes configuration data (config true), operational state (config false), RPC definitions for device operations, and notifications for event-driven management. YANG validation ensures XML instances conform to type constraints, key uniqueness, must conditions, and uniqueness expressions. Advanced YANG features include identity statements for extensible enumerations, feature flags for conditional modeling, and augment statements for modular extensions to standard models.
