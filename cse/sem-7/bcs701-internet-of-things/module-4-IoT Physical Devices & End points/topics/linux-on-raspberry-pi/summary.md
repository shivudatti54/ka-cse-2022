This chapter examined Linux on Raspberry Pi for IoT applications, covering kernel architecture including CFS scheduler, memory management, and device driver frameworks. Real-time capabilities via PREEMPT_RT patches were analyzed for time-critical IoT applications. Systemd service management enables robust IoT daemon deployment with dependency resolution and automatic recovery. Shell scripting provides automation for sensor data collection and preprocessing pipelines. GPIO programming through sysfs and libgpiod enables hardware interfacing. Network configuration including static IP addressing and firewall rules ensures secure IoT connectivity. A complete environmental monitoring system demonstrated integration of sensors, SQLite storage, and MQTT communication. Assessment through application-based MCQs validated understanding of systemd dependencies, real-time kernel configuration, and modern GPIO interfaces.
