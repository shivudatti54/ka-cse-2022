Logical design in IoT systems represents the platform-independent specification of functional components, data flows, and control structures. This module covered: (1) Mathematical foundations including formal definitions of data flow models with latency bounds and the perception-network-application architectural model; (2) Finite state machine theory with safety and liveness proofs, implemented through an ExtendedStateMachine class supporting guards and actions; (3) Industrial IoT application through a smart manufacturing case study demonstrating state transitions, timing constraints, and verification; (4) Event-driven architecture formalization with event bus implementation for pub-sub patterns; (5) Communication protocol abstraction for IoT messaging; (6) Numerical problems analyzing transition timing, event latency, and state reachability; (7) Analysis-based assessment questions evaluating safety properties, throughput calculation, and security vulnerabilities. The Python implementations provide verifiable logical models that can execute across Raspberry Pi, edge devices, and cloud platforms, enabling rapid prototyping while maintaining formal correctness guarantees.
