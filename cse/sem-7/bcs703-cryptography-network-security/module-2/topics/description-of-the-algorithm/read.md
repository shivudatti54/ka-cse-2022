# Description of the Algorithm - Pseudorandom Number Generators

## Introduction

Pseudorandom Number Generators (PRNGs) constitute fundamental cryptographic primitives that produce sequences of numbers appearing statistically random while being generated by deterministic algorithms. Unlike True Random Number Generators (TRNGs) that derive randomness from physical phenomena such as thermal noise or radioactive decay, PRNGs employ mathematical formulas to generate number sequences from an initial seed value. The security of modern cryptographic systems critically depends on the quality of random number generation, as weak PRNGs have historically led to catastrophic security breaches in encryption keys, digital signatures, and authentication protocols.

A PRNG can be formally modeled as a deterministic polynomial-time algorithm G that takes as input a seed s from {0,1}^n and produces an output string G(s) of arbitrary length L(n). The fundamental security requirement for cryptographically secure PRNGs is **indistinguishability** from true random strings—a computationally bounded adversary who observes polynomially many outputs from G should be unable to distinguish them from equally many truly random strings except with negligible probability.

## Theoretical Foundations

### Formal Security Definitions

**Definition 1 (Pseudorandomness):** A function G: {0,1}^_ → {0,1}^_ is a PRNG if there exists a polynomial L(·) such that:

1. **Expansion:** For every n, |G(s)| = L(n) where L(n) > n
2. **Pseudorandomness:** For every probabilistic polynomial-time distinguisher D, there exists a negligible function negl(·) such that:
   |Pr[D(G(s)) = 1] - Pr[D(r) = 1]| ≤ negl(n)
   where s ← {0,1}^n and r ← {0,1}^{L(n)}

This definition, known as the **distinguisher-based definition**, captures the intuition that no efficient algorithm can tell whether it is receiving output from G or a truly random source.

**The Next-Bit Test:** An equivalent formulation states that G passes the next-bit test if no polynomial-time algorithm can predict the (i+1)-th bit of the output given the first i bits, with probability better than 1/2 + negl(n). Yao's theorem (1982) established the equivalence between these two formulations, proving that passing all next-bit tests is necessary and sufficient for pseudorandomness.

### Seed and Internal State Architecture

The operation of any PRNG begins with a **seed value** s ∈ {0,1}^n, which serves as the initial input containing sufficient entropy. For cryptographic applications, the seed must be drawn from a high-entropy source to ensure unpredictability—the seed should contain at least n bits of min-entropy for security parameter n.

The seed initializes the **internal state** S(i) ∈ {0,1}^n of the PRNG at iteration i. The internal state represents the memory that the algorithm maintains between output generations, and its size directly impacts security. A fundamental requirement is that the state space must be exponentially large (2^n states for security parameter n) to prevent exhaustive search attacks where an adversary attempts to guess the internal state.

**State Transition Function f:** The state evolves according to a deterministic transition function f: {0,1}^n → {0,1}^n, expressed as S(i+1) = f(S(i)). For cryptographically secure PRNGs, this function must exhibit **one-wayness**—computing forward is computationally trivial, but inverting the transition to recover previous states is computationally infeasible. This asymmetry provides **forward security**, ensuring that compromising current outputs does not reveal past outputs.

**Output Function g:** The output extraction function g: {0,1}^n → {0,1}^m produces the pseudorandom output bits, typically with m < n. The output function often employs cryptographic primitives such as hash functions or block ciphers. We express this as O(i) = g(S(i)). Critically, the output reveals only partial information about the state, preventing state recovery attacks.

### Period and State Space Structure

The **period** of a PRNG refers to the length τ such that S(i+τ) = S(i) for some i, after which the sequence necessarily repeats. For a PRNG with n-bit state, the maximum theoretical period is 2^n. The state transition function induces a directed graph on the state space consisting of cycles and transient paths leading to cycles. A well-designed PRNG should have a single large cycle of length approximately 2^n, minimizing the probability of short periods.

**Theorem:** If the state transition function f: {0,1}^n → {0,1}^n is a permutation (bijection), then the period of any state is at most 2^n, with equality holding if and only if f consists of a single n-cycle.

This theorem motivates the use of bijective (permutation-based) state transitions in high-quality PRNGs, as permutations guarantee maximal period by ensuring every state has exactly one predecessor and one successor.

## Algorithmic Description and Examples

### Example 1: Linear Congruential Generator (LCG)

The Linear Congruential Generator represents the simplest classical PRNG, defined by the recurrence:

**X(i+1) = (a × X(i) + c) mod m**

**Parameters:**

- Modulus m = 2^31 - 1 (Mersenne prime)
- Multiplier a = 48271 (primitive root modulo m)
- Increment c = 0 (multiplicative LCG)

**Algorithm (Pseudocode):**

```
INPUT: seed X₀, parameters (a, m, c)
OUTPUT: sequence of pseudorandom numbers

1. X ← X₀ mod m
2. FOR i = 1 TO ∞ DO
3.     X ← (a × X + c) mod m
4.     U(i) ← X / m    // normalized to [0,1)
5.     RETURN U(i)
```

**Numerical Example (seed X₀ = 1):**

- X₁ = (48271 × 1) mod 2147483647 = 48271
  U₁ = 48271 / 2147483647 ≈ 0.0000225
- X₂ = (48271 × 48271) mod 2147483647 = 957
  U₂ = 957 / 2147483647 ≈ 0.0000004
- X₃ = (48271 × 957) mod 2147483647 = 4626172
  U₃ = 4626172 / 2147483647 ≈ 0.00215

**Security Analysis:** LCGs are **NOT cryptographically secure** due to their linear structure. Given n consecutive outputs, one can recover the parameters (a, c, m) using only O(n) outputs via lattice reduction techniques (Knuth-Shanks algorithm). The output sequence exhibits detectable correlations, failing the next-bit test. LCGs remain useful for simulation and non-cryptographic applications due to their efficiency.

### Example 2: Blum-Blum-Shub (BBS) Generator

The BBS generator provides provable security based on the quadratic residue hardness assumption.

**Mathematical Formulation:**
Let n = p × q where p, q are primes ≡ 3 (mod 4). The generator is:

- **State:** S(i) = X(i)² mod n
- **Output:** O(i) = least significant bit of S(i)

**Algorithm:**

```
INPUT: n = p×q (Blum integer), seed X₀ coprime to n
OUTPUT: sequence of pseudorandom bits

1. X ← X₀² mod n    // Initialize state
2. FOR i = 1 TO ∞ DO
3.     X ← X² mod n    // Compute next state
4.     b ← least significant bit of X
5.     RETURN b
```

**Security Proof Sketch:** The security of BBS reduces to the **Quadratic Residuosity Problem** (QRP). Specifically, predicting the next output bit requires solving the QRP: given X(i) = s² mod n, compute the least significant bit of √X(i) mod n. Since no polynomial-time algorithm is known for solving QRP without knowing p and q, BBS achieves logarithmic security (outputting O(log log n) bits per iteration provides maximal security).

### Example 3: CTR-DRBG (Counter-mode Deterministic Random Bit Generator)

Modern cryptographic standards (NIST SP 800-90A) specify CTR-DRBG using block ciphers in counter mode.

**Mathematical Specification:**

- **Key:** K ∈ {0,1}^128
- **Block Cipher:** AES-128
- **State:** (Key, Counter, V) where V is the working state

**Algorithm:**

```
INPUT: Key K, entropy input entropy, nonce
OUTPUT: pseudorandom bits

1. (Key, Counter) ← KeyDerivationFunction(entropy, nonce)
2. V ← 0
3. FOR i = 1 TO ∞ DO
4.     Block ← AES_ENC(Key, Counter)
5.     Output ← Block XOR V
6.     V ← Output
7.     Counter ← Counter + 1
8.     RETURN Output
```

**Security Reduction:** The security of CTR-DRBG reduces to the **pseudorandom permutation** property of the underlying block cipher. If AES is a PRP, then the output is indistinguishable from random. The counter ensures non-repeating inputs, guaranteeing that no two encryption calls produce the same block cipher input.

## Security Properties and Requirements

### Forward and Backward Security

**Forward Security:** Compromising the current state S(t) does not reveal previous outputs O(1), O(2), ..., O(t-1). This is achieved by using one-way state transitions—while S(t+1) = f(S(t)) is easy to compute, inverting this relationship is computationally infeasible.

**Backward Security (Prediction Resistance):** Given outputs O(1), O(2), ..., O(t), it is computationally infeasible to predict future outputs O(t+1), O(t+2), ... without knowing the seed. This requires that the state transition function be a hard-core predicate generator.

### Entropy Source and Seeding

A critical requirement for cryptographic PRNGs is proper **seeding from entropy sources**. The seed must contain genuine randomness—using predictable seeds (timestamps, process IDs) catastrophically weakens security. The entropy rate should exceed the security parameter: at least n bits of min-entropy for n-bit security.

**Definition (Entropy):** The min-entropy of a random variable X is H∞(X) = -log₂(max_x Pr[X = x]). For security parameter n, we require H∞(seed) ≥ n.

## Summary

PRNGs transform a short random seed into long pseudorandom sequences through deterministic algorithms. Cryptographically secure PRNGs must satisfy indistinguishability from true random strings, pass the next-bit test, and provide forward/backward security. While simple generators like LCG are efficient, they lack security properties. Modern cryptographically secure PRNGs rely on hard mathematical problems (quadratic residuosity) or provably secure symmetric primitives (block ciphers, hash functions). Understanding the formal security definitions and algorithmic structure is essential for implementing secure cryptographic systems.
