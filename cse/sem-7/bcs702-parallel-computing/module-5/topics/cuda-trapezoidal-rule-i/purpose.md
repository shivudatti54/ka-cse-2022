# Learning Purpose: CUDA Trapezoidal Rule I

**1. Why this topic matters**
The CUDA trapezoidal rule is the first complete CUDA application that solves a real numerical problem, bridging the gap between understanding CUDA syntax and writing practical GPU-accelerated code. It demonstrates how a well-known sequential algorithm -- the trapezoidal rule for numerical integration -- maps onto the GPU thread model. More importantly, this first version using atomicAdd deliberately exposes a critical performance pitfall: serialization at atomic operations. Understanding why this simple approach fails to achieve good speedup is essential for appreciating the optimized techniques (shared-memory reduction) that follow.

**2. What you will learn**
You will learn how to map the trapezoidal rule's independent function evaluations to individual CUDA threads, compute each thread's unique subinterval point using the global thread index formula, handle the half-weight boundary terms for f(a) and f(b), and use atomicAdd to accumulate partial results into a single global variable. You will also understand the host-side workflow: allocating and initializing device memory with cudaMalloc and cudaMemset, configuring the grid and block dimensions, launching the kernel, and retrieving the result with cudaMemcpy. Critically, you will learn why atomicAdd creates a serialization bottleneck and causes floating-point non-determinism.

**3. How it connects to other topics**
This topic directly extends the Threads, Blocks, and Grids topic by applying thread indexing to a concrete numerical problem. It parallels the OpenMP trapezoidal rule (Module 4) and MPI trapezoidal rule (Module 3), completing the course's recurring theme of implementing the same algorithm across three parallel programming models. The atomicAdd serialization issue here is analogous to the OpenMP critical section bottleneck. The limitations exposed in this topic motivate the CUDA Trapezoidal Rule II topic, which introduces shared-memory parallel reduction as the solution.

**4. Real-world relevance**
GPU-accelerated numerical integration is widely used in computational finance (option pricing via Monte Carlo), physics simulations (field integrals), engineering analysis (stress and thermal calculations), and signal processing (convolution and filtering). The atomicAdd pattern, despite its limitations for pure reduction, is still used in real applications for histogram computation, sparse matrix assembly, and collision detection. Understanding when atomicAdd is acceptable and when structured reduction is necessary is a practical skill for any CUDA programmer.
