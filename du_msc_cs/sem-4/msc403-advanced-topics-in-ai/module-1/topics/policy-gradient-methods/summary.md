# Policy Gradient Methods - Summary

## Key Definitions and Concepts
- **Policy**: Probability distribution π(a|s) over actions given state
- **Trajectory**: Sequence (s0,a0,r0,...,sT) generated by policy
- **Advantage Function**: A(s,a) = Q(s,a) - V(s) measures action's relative value
- **Trust Region**: Constraint to prevent destructive policy updates (PPO/TRPO)

## Important Formulas and Theorems
1. **Policy Gradient Theorem**:  
   ∇J(θ) = 𝔼_π[∇logπ(a|s;θ) * Q^π(s,a)]
2. **REINFORCE Update**:  
   Δθ = αG_t∇logπ(a_t|s_t;θ)
3. **PPO-Clip Objective**:  
   L(θ) = 𝔼[min(r(θ)A, clip(r(θ),1-ε,1+ε)A)]
4. **TD Error**: δ = r + γV(s') - V(s)

## Key Points
- PG methods optimize policies directly via gradient ascent on expected returns
- Actor-Critic architectures decouple policy (actor) and value estimation (critic)
- Advantage normalization is crucial for training stability
- PPO achieves SOTA by limiting policy update steps through clipping
- Entropy regularization prevents premature convergence to deterministic policies
- Monte Carlo rollouts have high variance but unbiased; TD methods trade variance for bias
- Parallel environment sampling (A3C) improves data efficiency

## Common Mistakes to Avoid
- Forgetting to disable gradient tracking for old policies in PPO
- Using Q-values instead of advantages in gradient updates
- Ignoring advantage normalization leading to unstable gradients
- Setting PPO's ε > 0.3 causing ineffective policy updates
