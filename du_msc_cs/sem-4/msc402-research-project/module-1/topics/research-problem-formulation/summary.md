# Research Problem Formulation - Summary

## Key Definitions and Concepts
- **Research Problem**: Clear articulation of an issue requiring systematic investigation
- **PICOS Framework**: Defines Population, Intervention, Comparison, Outcomes, Setting
- **Type I/II Errors**: False positive/negative rates in hypothesis testing
- **DSRM**: Design Science Research Methodology for applied projects

## Important Formulas and Theorems
- **Sample Size Calculation**: n = (Z² * p(1-p)) / e² (for empirical studies)
- **Algorithmic Complexity**: O(f(n)) bounds for proposed solutions
- **Statistical Power**: 1 - β ≥ 0.8 for valid hypothesis testing

## Key Points
- Problem statements must address "why" (significance) and "how" (methodology)
- Literature gaps can be identified through citation chain analysis
- Interdisciplinary problems require balancing technical depth and domain knowledge
- Hypothesis should be falsifiable using computational methods
- Scope limitations prevent resource overutilization in 2-year MSc projects
- Validation plans must specify datasets (e.g., ImageNet), tools (e.g., PyTorch), and metrics (e.g., F1-score)
- Ethical considerations are mandatory for AI/ML problems under EU AI Act guidelines

## Common Mistakes to Avoid
- Confusing technical challenges with research problems (e.g., "Implementing X" vs "Why X fails in Y context")
- Omitting baseline comparisons with state-of-the-art methods
- Overlooking temporal validity (e.g., IoT security problems pre/post quantum computing)
- Using vague success criteria ("improve performance" vs "achieve 95% accuracy")

## Revision Tips
1. Practice rewriting published paper abstracts into problem statements
2. Create gap analysis matrices for 3 recent IEEE conference papers
3. Use mind maps to trace problem derivation from broader societal needs
4. Participate in DU's research colloquiums to critique peer problem formulations
