# Literature Survey Tools - Summary

## Key Definitions and Concepts
- Boolean Search: Using AND/OR/NOT operators to refine database queries
- PRISMA: Preferred Reporting Items for Systematic Reviews and Meta-Analyses
- Citation Chaining: Tracking references backward/forward through citation links
- H-index: Metric measuring researcher productivity and impact
- Predatory Journals: Low-quality publications prioritizing profit over peer review

## Important Formulas and Theorems
- Bradford's Law: z = 2log(n/1.781) (n=articles in core journals)
- Journal Impact Factor: IF = Citations in Year Y to items published in Y-1+Y-2 / Total citable items Y-1+Y-2
- Co-citation Strength: wij = Σ ci * cj where ci,cj are citation counts

## Key Points
- Tool selection depends on research phase: Exploration (Semantic Scholar) vs Validation (Scopus)
- Systematic reviews require explicit inclusion/exclusion criteria
- Citation metrics should be normalized by research field
- Preprint servers enable access to cutting-edge work but require quality checks
- AI tools can introduce bias through opaque recommendation algorithms
- Institutional access often needed for full database capabilities
- Version control is crucial when collaborating on literature surveys

## Common Mistakes to Avoid
- Over-reliance on single database (e.g., only Google Scholar)
- Ignoring grey literature (technical reports, theses)
- Confusing correlation with impact in citation analysis
- Failing to document search strategy reproducibility

## Revision Tips
- Create cheat sheets for database-specific search syntax
- Practice with DU's institutional access to IEEE/ACM portals
- Use Anki flashcards for PRISMA flowchart components
- Compare tool outputs using same seed paper
