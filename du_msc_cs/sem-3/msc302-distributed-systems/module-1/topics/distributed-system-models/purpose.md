# Learning Objectives

After studying this topic, you should be able to:

1. Classify distributed systems using architectural and interaction models
2. Analyze tradeoffs in consistency, availability, and partition tolerance
3. Design algorithms considering synchronous/asynchronous timing assumptions
4. Evaluate failure handling strategies for crash-stop vs Byzantine scenarios
5. Apply vector clocks to resolve causal relationships in event ordering
6. Contrast consensus protocols (Paxos, Raft, PBFT) under different models
7. Propose model adaptations for emerging paradigms (edge computing, serverless)
8. Critically assess real-world systems using formal model specifications
