**Purpose**  
Amortized analysis provides a fundamental tool for reasoning about the long‑term performance of data structures and algorithms, enabling engineers to guarantee efficient worst‑case behaviour over sequences of operations rather than isolated calls. This approach is critical in real‑world systems such as databases, network routers, and real‑time scheduling where consistent throughput and predictable latency matter more than occasional spikes.

**Learning Objectives**  
- **Explain** the difference between worst‑case, average‑case, and amortized analysis.  
- **Apply** the aggregate, accounting, and potential methods to compute amortized costs.  
- **Analyze** classic data structures (e.g., dynamic arrays, Fibonacci heaps, splay trees) using amortized bounds.  
- **Design** new data structures and prove amortized performance guarantees for their operations.  
- **Compare** amortized analysis with probabilistic and competitive analysis in appropriate contexts.  
- **Evaluate** real‑world algorithms where amortized performance influences system design (e.g., hash tables, B‑trees, sliding‑window buffers).  
- **Synthesize** amortized results to make informed trade‑offs between time, space, and implementation complexity.
