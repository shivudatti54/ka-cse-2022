# Learning Objectives

After studying this topic, you should be able to:

1. Analyze the architecture of NewSQL systems and their advantages over traditional RDBMS
2. Evaluate trade-offs between in-memory storage and persistence mechanisms
3. Design sharding strategies for distributed NewSQL deployments
4. Compare different concurrency control mechanisms (OCC vs MVCC)
5. Implement HTAP solutions using vectorized query execution
6. Assess the impact of emerging hardware (NVM, RDMA) on database performance
7. Critique research papers on distributed consensus in NewSQL systems
8. Develop disaster recovery plans for in-memory databases
