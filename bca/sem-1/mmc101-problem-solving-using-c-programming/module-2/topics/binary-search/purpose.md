# Learning Objectives

After studying this topic, you should be able to:

1. Understand the fundamental principle of binary search and how it achieves logarithmic time complexity through divide-and-conquer.

2. Identify the prerequisite condition (sorted array) and explain why binary search fails on unsorted arrays.

3. Implement binary search iteratively in C programming, using proper boundary conditions and loop termination criteria.

4. Implement binary search recursively in C programming, understanding the role of recursion stack in space complexity.

5. Analyze and calculate time complexity O(log n) and space complexity O(1) for iterative, O(log n) for recursive implementations.

6. Perform step-by-step dry runs of binary search on sample arrays to trace algorithm execution manually.

7. Compare binary search with linear search in terms of time complexity, space requirements, and suitable use cases.

8. Apply binary search concepts to solve practical problems like searching in sorted databases, dictionary lookups, and custom record searches.
