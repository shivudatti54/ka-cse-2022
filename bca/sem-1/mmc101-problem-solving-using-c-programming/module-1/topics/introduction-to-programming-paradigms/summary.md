# Introduction To Programming Paradigms - Summary

## Key Definitions and Concepts

- **Programming Paradigm**: A classification of programming languages based on their style, features, and computational model. It represents the approach or methodology used to structure and write computer programs.

- **Procedural Programming**: A paradigm based on procedures (functions) that operate on data. Programs are organized as a sequence of instructions. C is a procedural language.

- **Imperative Programming**: A paradigm where programs consist of statements that modify program state. Focuses on describing HOW to accomplish a task through explicit control flow.

- **Declarative Programming**: A paradigm focusing on WHAT should be accomplished rather than HOW. Examples include SQL and HTML.

- **Functional Programming**: A paradigm treating computation as evaluation of mathematical functions, avoiding mutable data and state changes.

- **Object-Oriented Programming**: A paradigm organizing code around objects that contain data and methods. Emphasizes encapsulation, inheritance, and polymorphism.

## Important Formulas and Theorems

There are no specific formulas in this topic, but the conceptual framework of paradigms can be summarized as:

- Paradigm Selection Criteria: Problem type → Appropriate paradigm → Suitable language
- C's Paradigm Position: Procedural/Imperative → System programming → Performance-critical applications

## Key Points

- C is a procedural programming language, not an object-oriented language

- Procedural programming organizes code around functions that operate on data structures

- The imperative paradigm focuses on explicit state changes through assignment statements

- Declarative programming describes desired results without specifying control flow

- Understanding paradigms helps in selecting the right approach for different problems

- Many modern languages support multiple paradigms (multi-paradigm languages)

- C's procedural nature makes it ideal for system-level programming and applications requiring direct memory access

## Common Mistakes to Avoid

- Confusing C with object-oriented languages - C does NOT support classes or objects

- Mixing up paradigms with languages - a paradigm is a concept, a language is a tool

- Thinking paradigms are mutually exclusive - many languages support multiple paradigms

- Assuming newer paradigms are always better - each has appropriate use cases

## Revision Tips

- Create a comparison table of all paradigms with their key characteristics

- Practice identifying which paradigm a given code snippet represents

- Remember that C's historical context (developed for Unix system programming) influenced its procedural design

- Focus on the fundamental distinction: imperative (HOW) vs declarative (WHAT)
