# Learning Objectives

After studying this topic, you should be able to:

1.  Define a Request-Reply protocol and describe its fundamental two-message exchange pattern.
2.  Explain the critical role of marshalling in enabling communication between heterogeneous systems in a Request-Reply interaction.
3.  Differentiate between "at-least-once" and "at-most-once" invocation semantics and identify which type of operation (idempotent vs. non-idempotent) each is suited for.
4.  Describe the relationship between basic Request-Reply protocols and higher-level abstractions like Remote Procedure Call (RPC).
5.  Identify key challenges such as timeouts, retries, and partial failures inherent in this communication model.
6.  Diagram a simple client-server interaction using the Request-Reply pattern.
