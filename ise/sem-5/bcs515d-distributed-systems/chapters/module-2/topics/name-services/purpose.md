# Learning Objectives

After studying this topic, you should be able to:

1.  Define a name service and explain its critical role in enabling resource sharing and location transparency in distributed systems.
2.  Differentiate between the concepts of names, addresses, and identifiers, and describe the process of name resolution.
3.  Explain the hierarchical structure of the DNS namespace, including the roles of the root, TLD, and authoritative name servers.
4.  Trace and describe the step-by-step process of iterative DNS resolution for a given domain name.
5.  Identify common DNS record types (A, AAAA, CNAME, MX, NS) and state their specific purposes.
6.  Contrast a simple name service with a more comprehensive directory service, highlighting the differences in their data models and query capabilities.
