# **Distributed Systems: Chapter 12.1 Revision Notes**

## **Introduction**

- A distributed system is a collection of independent computers that appear to be a single cohesive system to the user.
- Distributed systems are used to solve complex problems that cannot be solved by a single computer.

## **File Service Architecture**

- A file service architecture is a type of distributed system that provides file sharing and management capabilities.
- Components:
  - Client: requests files from the server
  - Server: provides files to clients
  - File System: stores and manages files
  - Network: connects clients and servers

## **Key Concepts**

- **Distributed File System (DFS)**: a type of file system that stores files across multiple servers.
- **File Transfer Protocol (FTP)**: a protocol for transferring files over a network.
- **Network File System (NFS)**: a protocol for sharing files across multiple computers.

## **Definitions**

- **Replication**: the process of storing multiple copies of a file in different locations.
- **Consistency**: the property of a distributed system that ensures that data is consistent across all nodes.
- **Availability**: the property of a distributed system that ensures that data is accessible to all users.

## **Theorems**

- **Consistency Theorem**: a distributed system is consistent if and only if it satisfies the consistency condition.
- **Availability Theorem**: a distributed system is available if and only if it satisfies the availability condition.

## **Important Formulas**

- **Availability Formula**: (1 - f) \* R \* A = U, where f is the failure rate, R is the recovery rate, A is the availability, and U is the utilization.

## **Revision Tips**

- Understand the different components of a file service architecture.
- Familiarize yourself with common protocols such as FTP and NFS.
- Review the key concepts of replication, consistency, and availability.
- Practice solving problems related to distributed systems and file services.
