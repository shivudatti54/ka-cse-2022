# Name Services and the Domain Name System

### Overview

- **Definition:** A name service is a database that maps names to addresses or IP addresses.
- **Domain Name System (DNS):** A global system for translating human-readable domain names into IP addresses.

### Key Concepts

- **Name Service:** A database that maps names to addresses or IP addresses.
- **Domain Name System (DNS):** A global system for translating domain names into IP addresses.
- **Resolver:** A software component that queries the DNS to resolve domain names to IP addresses.
- **Recursion:** A technique used by resolvers to query DNS recursively to resolve complex domain names.

### Important Formulas and Definitions

- **DNS Query:** A request for information about a domain name.
- **DNS Record:** A data entry in the DNS database that maps a name to an IP address.
- **MX Record (Mail Exchanger Record):** Maps a domain name to a mail server or server that handles email.
- **A Record (Address Record):** Maps a domain name to an IP address.

### Key Theorems

- **Theorem:** The DNS is a decentralized system that uses caching and recursion to resolve domain names.
- **Theorem:** The DNS is a hierarchical system, with top-level domains (TLDs) at the top and subdomains below.

### Important Domain Names and TLDs

- **TLDs:** .com, .org, .net, etc.
- **Top-Level Domain (TLD):** The highest level of the domain name hierarchy.
- **Subdomain:** A domain name that is part of a larger domain name.

### Revision Tips

- **Understand the DNS hierarchy:** From TLDs to subdomains.
- **Know the different types of DNS records:** A, MX, NS, etc.
- **Practice DNS queries and recursion:** To improve your understanding and retention.
