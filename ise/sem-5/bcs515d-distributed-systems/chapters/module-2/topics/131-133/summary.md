# **Distributed Systems: 13.1-13.3 Revision Notes**

### Introduction

- **Definition:** A distributed system is a collection of independent computers that appear to be a single, cohesive system to the user.
- **Characteristics:**
  - Geographically dispersed
  - Multiple computers
  - Distributed data and resources

### File Service Architecture

- **Definition:** A file service architecture is a design pattern for organizing and managing data in a distributed system.
- **Components:**
  - **Client:** The user-facing component that interacts with the file system.
  - **Server:** The component that manages the file system.
  - **Storage:** The physical devices that store the files.

### Key Concepts

- **File System Hierarchy Standard (FSHS):** A standard for organizing files in a distributed system.
- **Network File System (NFS):** A protocol for sharing files over a network.
- **Caching:** Storing frequently accessed files in a faster, local storage.

### Important Formulas and Theorems

- **Thompson's Network File System:** A theoretical model for a file system that provides fault-tolerance and scalability.
- **GFS (Google File System):** A distributed file system that provides high availability and scalability.

### Key Definitions

- **Fault tolerance:** The ability of a system to continue functioning even if one or more components fail.
- **Scalability:** The ability of a system to handle increased load or demand without compromising performance.

### Important Terms

- **Distributed file system:** A file system that stores and manages files across multiple computers.
- **Distributed storage:** The storage of files across multiple computers in a distributed system.
