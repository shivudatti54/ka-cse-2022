# Learning Objectives

After studying this topic, you should be able to:

1.  Identify and describe the five primary segments of a C program's memory layout: Text, Initialized Data, BSS, Heap, and Stack.
2.  Differentiate between the stack and heap segments in terms of purpose, allocation/deallocation mechanism, size, growth direction, and variable lifetime.
3.  Correctly predict the memory segment where a given variable (global, static, local, dynamic) will be stored based on its declaration and scope.
4.  Explain the purpose and optimization benefit of the BSS segment.
5.  Describe the process of a function call and how it affects the stack segment (stack frame creation).
6.  Link common runtime errors (e.g., segmentation fault, memory leak, stack overflow) to their underlying causes in specific memory segments.
