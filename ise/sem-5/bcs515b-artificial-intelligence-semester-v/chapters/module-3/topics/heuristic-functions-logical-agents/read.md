# Module 3: Heuristic Functions & Logical Agents

## Introduction

Welcome,  Engineers! In your journey through Artificial Intelligence, you've learned about problem-solving as a state-space search. Module 3 bridges the gap between simple search algorithms and more sophisticated, knowledge-based systems. This section focuses on two pivotal concepts: **Heuristic Functions**, which make search algorithms like A\* smart and efficient, and **Logical Agents**, which use knowledge representation and reasoning to act rationally in complex environments.

## Core Concepts

### 1. Heuristic Functions

A heuristic function, often denoted as `h(n)`, is a function that estimates the cost of the cheapest path from a given node `n` to a goal state. It is a crucial component of **informed search** algorithms, providing a "hint" or an educated guess to guide the search towards the goal more efficiently than blind search methods.

**Key Properties:**

- **Admissibility:** A heuristic is admissible if it _never overestimates_ the true cost to reach the goal. Admissibility ensures that algorithms like A\* are guaranteed to find an optimal solution. For example, the straight-line distance is an admissible heuristic for pathfinding on a map, as the shortest path "as the crow flies" is always less than or equal to the actual road distance.
- **Consistency (Monotonicity):** A heuristic is consistent if for every node `n` and every successor `n'` generated by action `a`, the estimated cost of reaching the goal from `n` is no greater than the step cost of getting to `n'` plus the estimated cost from `n'` to the goal: `h(n) ≤ c(n, a, n') + h(n')`. Consistency implies admissibility and ensures that A\* is optimal and efficient.

**Example:** In the 8-puzzle problem (sliding tiles on a 3x3 board), common heuristics are:

- **Misplaced Tiles:** `h(n)` = number of tiles that are not in their goal position. (Admissible)
- **Manhattan Distance:** `h(n)` = sum of the horizontal and vertical distances each tile is from its goal position. (Also admissible, and more informed than misplaced tiles)

A more informed heuristic (one that gives a closer estimate to the true cost) will generally lead to expanding fewer nodes during the search, making the algorithm faster.

### 2. Logical Agents

A logical agent is an agent that uses the tools of **knowledge representation** and **logical reasoning** to derive actions from its knowledge base (KB). Instead of perceiving the world as raw data, it builds a symbolic representation of what it knows and uses inference to make decisions.

**Key Components:**

- **Knowledge Base (KB):** A set of sentences expressed in a formal language that represents what the agent knows about the world.
- **Inference Engine:** The algorithm that allows the agent to derive new sentences (conclusions) that are entailed by the KB. This process is called **theorem proving**.

**The Wumpus World:** A classic example for logical agents is the Wumpus World—a cave grid where the agent must find gold while avoiding bottomless pits and a deadly monster. The agent's sensors give limited information (e.g., a "Breeze" percept near a pit).

The agent can use **propositional logic** to represent knowledge.

- Let `P_{1,1}` be the proposition "There is a Pit in square [1,1]".
- Let `B_{1,2}` be the proposition "There is a Breeze in square [1,2]".

After moving to square [1,1] and not feeling a breeze, the agent can add to its KB: `¬B_{1,1}`. A fundamental rule in the Wumpus world is that a breeze occurs if and only if there is a pit in a neighboring square. For square [1,2], this rule is: `B_{1,2} ⇔ (P_{1,1} ∨ P_{1,3} ∨ P_{2,2})`.

If the agent then perceives a breeze in [1,2] (`B_{1,2}` is true), the logical equivalence tells it that one of the adjacent squares `{P_{1,1}, P_{1,3}, P_{2,2}}` must be true. Since it already knows `¬P_{1,1}` (from `¬B_{1,1}`), it can infer that the pit must be in `P_{1,3} ∨ P_{2,2}`. This logical inference allows the agent to avoid those squares until it gathers more information.

## Key Points & Summary

| Concept                       | Description & Purpose                                                                                                   |
| :---------------------------- | :---------------------------------------------------------------------------------------------------------------------- |
| **Heuristic Function `h(n)`** | An estimate of the cost from node `n` to the goal. It makes informed search (e.g., A\*) efficient and scalable.         |
| **Admissibility**             | A property of a heuristic guaranteeing it never overestimates the true cost. Essential for A\* to find an optimal path. |
| **Consistency**               | A stronger property than admissibility that ensures the heuristic's estimates are monotonically non-decreasing.         |
| **Logical Agent**             | An agent that uses a knowledge base of logical sentences and an inference engine to make rational decisions.            |
| **Knowledge Base (KB)**       | The central component of a logical agent, storing everything the agent knows about its world.                           |
| **Inference**                 | The process of deriving new, logically entailed sentences from the KB to make informed actions.                         |

**In essence,** heuristic functions provide the computational "intelligence" for efficient search in large state spaces, while logical agents provide the "reasoning" capability for making decisions based on represented knowledge. Together, they form the foundation for many advanced AI systems you will encounter.
