# **UNIX SYSTEM PROGRAMMING Revision Notes: Text Book 2, 2**

### Introduction

- UNIX is a multi-user, multi-tasking operating system
- Developed by Bell Labs in 1971
- Designed to be portable, reliable, and efficient

### Unix Standardization

- UNIX is a portable operating system, designed to run on different hardware platforms
- Standardization ensures compatibility and portability
- Key standardization documents:
  - POSIX (Portable Operating System Interface)
  - Unix Standard

### UNIX

- UNIX (Unix) is a popular operating system, based on UNIX System V
- Derived from UNIX System III, released in 1983
- Key features:
  - Multi-user, multi-tasking capabilities
  - File system hierarchy (e.g., /bin, /etc, /usr)
  - Shell (e.g., Bourne Shell, Korn Shell)

### Important Formulas and Definitions

- **Shell**: a program that reads commands from the user and executes them
- **Process**: a program in execution
- **Job**: a sequence of processes
- **Shell Variables**: variables stored in the shell's memory
- ** wildcards** `*`, `?`, `[`, `]`
- ** Pipes** `|`
- **Redirectors** `<`, `>`, `>>`

### Theorems

- **The Shell's Command-Line Interface**: a standardized interface for interacting with the shell
- **Process Scheduling**: the discipline of assigning processes to CPUs for execution

### Key Terms

- **Open File**: a file that can be accessed or modified by multiple processes
- **Close File**: a file that is no longer needed and can be released from the system's memory
- **File System**: a collection of files and directories that are stored on a storage device
