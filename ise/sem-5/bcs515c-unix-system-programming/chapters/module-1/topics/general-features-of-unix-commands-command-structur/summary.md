# **General Features of Unix Commands/ Command Structure**

### Overview

- Unix commands are used to interact with the operating system and perform various tasks.
- Unix commands follow a specific structure and syntax.

### Key Features of Unix Commands

- **Command Line Interface (CLI)**: Unix commands are entered through the command line interface.
- **Syntax**: Unix commands follow a specific syntax, which includes the command name, options, and arguments.
- ** piping**: Unix commands can be piped together to perform complex operations.
- **Redirection**: Unix commands can redirect output to files or other commands.
- **File Permissions**: Unix commands can change file permissions using the `chmod` command.

### Command Structure

- **Command Name**: The command name is always the first word of the command.
- **Options**: Options are used to modify the command's behavior.
- **Arguments**: Arguments are used to provide data to the command.
- **Flags**: Flags are used to specify options.
- **Options and Arguments**: Options and arguments are separated by spaces.

### Important Formulas and Definitions

- **Theorem**: Unix commands follow the "Command Line" format: `command [options] [arguments]`
- **Definition**: A Unix command is a program that takes commands as input and produces output.

### Key Commands

- **cd**: Change directory
- **ls**: List files and directories
- **mkdir**: Make a directory
- **rm**: Remove a file or directory
- **cp**: Copy a file or directory
- **mv**: Move or rename a file or directory

### Quick Revision Tips

- Understand the command line interface and syntax.
- Learn the basic commands and their functions.
- Practice piping and redirection to perform complex operations.
- Familiarize yourself with file permissions and the `chmod` command.
