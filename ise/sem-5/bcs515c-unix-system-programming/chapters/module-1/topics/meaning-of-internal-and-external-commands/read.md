# **Meaning of Internal and External Commands**

## **Introduction**

In the UNIX operating system, commands are used to interact with the system and perform various tasks. These commands can be executed either from within a program or from a terminal. There are two types of commands in UNIX: internal and external commands.

## **Internal Commands**

Internal commands are built-in commands that are executed by the shell itself. These commands are typically used for tasks such as file manipulation, process management, and control of the shell.

**Examples of Internal Commands:**

- `cd`: Changes the current directory.
- `pwd`: Prints the current working directory.
- `set`: Sets the shell variables and options.
- `export`: Exports shell variables to the environment.
- `alias`: Creates an alias for a command.

**Characteristics of Internal Commands:**

- They are built-in to the shell and do not require an external program to be executed.
- They do not produce any output or error messages.
- They can be used for file manipulation, process management, and control of the shell.
- They can be overridden by other commands with the same name.

## **External Commands**

External commands, on the other hand, are programs that are stored on disk and executed by the shell. These commands are typically used for tasks such as file management, system administration, and application-specific tasks.

**Examples of External Commands:**

- `ls`: Lists the files and directories in a given directory.
- `cp`: Copies files and directories.
- `mv`: Moves or renames files and directories.
- `rm`: Removes files and directories.
- `grep`: Searches for patterns in files.

**Characteristics of External Commands:**

- They are separate programs that are stored on disk.
- They produce output or error messages.
- They are typically used for file management, system administration, and application-specific tasks.
- They can be used in conjunction with internal commands to perform more complex tasks.

**Key Points:**

- Internal commands are built-in to the shell and do not require an external program to be executed.
- External commands are separate programs that are stored on disk and executed by the shell.
- Internal commands are typically used for file manipulation, process management, and control of the shell, while external commands are used for file management, system administration, and application-specific tasks.

## **Conclusion:**

In conclusion, internal and external commands are two types of commands used in UNIX. Internal commands are built-in to the shell, while external commands are separate programs that are executed by the shell. Understanding the difference between internal and external commands is essential for effective use of the UNIX operating system.

**Key Concepts:**

- Internal commands: built-in to the shell, do not produce output or error messages.
- External commands: separate programs stored on disk, produce output or error messages.
- Internal commands: used for file manipulation, process management, and control of the shell.
- External commands: used for file management, system administration, and application-specific tasks.
