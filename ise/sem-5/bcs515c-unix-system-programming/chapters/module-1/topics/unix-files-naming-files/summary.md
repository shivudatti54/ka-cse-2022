# Unix Files: Naming Files

### Overview

- Unix file system is hierarchical and organized using a tree-like structure.
- Files are identified by a unique name and path.

### Key Concepts

- **Path**: A sequence of directories that lead to a file.
- **Filename**: The name of the file itself.
- **Directory**: A container that holds files and subdirectories.
- **Subdirectory**: A directory within another directory.

### File Naming Rules

- **Name Length**: Maximum 256 characters, including the dot (.)
- **Character Set**: Only allowed characters are letters (a-z, A-Z), digits (0-9), underscores (\_), and dots (.)
- **Reserved Characters**: Not allowed in filenames: /, \, :

### File Extension

- **File Extension**: Denotes the file type or format.
- **Common File Extensions**:
  - Text files: .txt, .doc, .pdf
  - Image files: .jpg, .png, .gif
  - Executable files: .exe, .sh, .bash

### Important Formulas and Definitions

- **File Descriptor**: A unique integer used to identify an open file.
- **File Type**: The category of a file (e.g., regular file, directory, symbolic link).

### Important Theorems

- **File System Hierarchy Standard (FHS)**: A set of guidelines for organizing files in a Unix system.
- **POSIX Standard**: A set of guidelines for Unix-like operating systems, including file naming conventions.

### Revision Tips

- Practice creating and using file paths.
- Familiarize yourself with common file extensions and naming conventions.
- Understand the differences between files, directories, and subdirectories.
