# **Command Arguments and Options**

**Definition:** Command arguments and options are used to provide additional information to a Unix command to modify its behavior.

**Key Points:**

- **Arguments:**
  - Required input to a command
  - Must be specified in the correct order
  - Can be a single word or phrase
- **Options:**
  - Optional input to a command
  - Indicated by a hyphen (-) or double hyphen (--) followed by a letter
  - Can be used to modify the behavior of a command

**Common Options:**

- `-a` or `--all`: Shows all files or processes
- `-l` or `--long`: Displays long listing or detailed output
- `-h` or `--help`: Displays help information for a command
- `-v` or `--verbose`: Increases the verbosity of a command

**Formulas:**

- `command [option] argument`
- `command [argument] [option]`

**Theorems:**

- The order of arguments and options matters
- Options can be combined using the `-` or `--` symbol

**Revision Tips:**

- Practice writing commands with arguments and options
- Pay attention to the order of arguments and options
- Use the `man` command to learn about options and their usage
