# **7.1 - Data Communication**

### Definition

Data communication refers to the process of sharing data between two or more devices over a communication channel. This process involves encoding, transmitting, and decoding data to ensure its accurate and timely delivery.

### Components of Data Communication

- **Sender**: The device that sends data to the receiver.
- **Channel**: The medium through which data is transmitted, such as a wireless network or a physical medium like a cable.
- **Receiver**: The device that receives and decodes the data.

### Key Concepts

- **Data Encoding**: The process of converting data into a format that can be transmitted over a communication channel.
- **Data Transmission**: The process of sending data from the sender to the receiver over the communication channel.
- **Data Decoding**: The process of converting the transmitted data back into its original format at the receiver.

### Examples

- Sending an email from a user's computer to the recipient's computer via the internet.
- Transferring files from one device to another using a USB cable.

# **7.2 - Network Topology**

### Definition

Network topology refers to the physical and logical arrangement of devices in a network. It defines how devices are connected and communicate with each other.

### Types of Network Topologies

- **Bus Topology**: A single cable connects all devices in a linear sequence.
- **Star Topology**: All devices are connected to a central device or hub.
- **Ring Topology**: Devices are connected in a circular configuration, and data travels in one direction around the ring.
- **Mesh Topology**: Each device is connected to every other device, providing multiple paths for data transmission.

### Advantages and Disadvantages of Different Topologies

| Topology      | Advantages                                                      | Disadvantages                                         |
| ------------- | --------------------------------------------------------------- | ----------------------------------------------------- |
| Bus Topology  | Easy to install and maintain, cost-effective                    | Difficult to troubleshoot, single point of failure    |
| Star Topology | Easy to manage and maintain, provides redundancy                | More expensive than bus topology                      |
| Ring Topology | Provides high-speed data transmission, secure                   | Difficult to add new devices, single point of failure |
| Mesh Topology | Provides high reliability, multiple paths for data transmission | Most expensive topology, complex to manage            |

### Key Concepts

- **Node**: A device connected to a network.
- **Hub**: A central device that connects multiple nodes in a star topology.
- **Ring**: A circular configuration of devices in a ring topology.

### Examples

- A home network with a router, modem, and multiple devices connected using Ethernet cables.
- A corporate network with a hub, switches, and multiple devices connected using fiber-optic cables.

# **7.3 - Network Models**

### Definition

Network models refer to the theoretical frameworks used to design, analyze, and optimize computer networks. They provide a structured approach to understanding network behavior and performance.

### Types of Network Models

- **Physical Layer Model**: Focuses on the physical properties of the network, such as cable quality and distance.
- **Data Link Layer Model**: Focuses on the logical arrangement of devices in the network, such as MAC addresses and protocols.
- **Network Layer Model**: Focuses on the routing and forwarding of data between devices, including protocols such as IP and ICMP.
- **Transport Layer Model**: Focuses on the reliable transfer of data between devices, including protocols such as TCP and UDP.

### Key Concepts

- **Layered Architecture**: A network is composed of multiple layers, each with its own set of protocols and functions.
- **Protocols**: Standardized communication languages used between devices in a network.
- **Network Metrics**: Performance metrics used to evaluate network behavior and optimization, such as throughput, latency, and packet loss.

### Examples

- The OSI model, which provides a layered architecture for network design and analysis.
- The TCP/IP model, which provides a layered architecture for network design and analysis in the internet.
