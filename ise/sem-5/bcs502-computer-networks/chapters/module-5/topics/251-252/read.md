# **25.1 - Network Topology**

### Definition

Network topology refers to the physical or logical arrangement of devices in a network. It describes how devices are connected to each other and can affect the performance and scalability of the network.

### Types of Network Topology

#### 1. **Bus Topology**

In a bus topology, all devices are connected to a single cable or backbone, called the bus. This allows for easy installation and maintenance, but it can be prone to interference and makes it difficult to add new devices.

- Advantages:
  - Simple to install and maintain
  - Low cost
- Disadvantages:
  - Prone to interference
  - Difficult to add new devices

#### 2. **Star Topology**

In a star topology, all devices are connected to a central device, called the hub or switch. This allows for easy management and troubleshooting, but it can be prone to single-point failure.

- Advantages:
  - Easy to manage and troubleshoot
  - Redundant connections can be added
- Disadvantages:
  - Prone to single-point failure
  - More expensive than bus topology

#### 3. **Ring Topology**

In a ring topology, devices are connected in a circular configuration, and data travels in one direction around the ring. This allows for fast data transfer, but it can be prone to congestion and single-point failure.

- Advantages:
  - Fast data transfer
  - Redundant connections can be added
- Disadvantages:
  - Prone to congestion and single-point failure
  - Difficult to add new devices

#### 4. **Mesh Topology**

In a mesh topology, each device is connected to every other device, allowing for multiple paths for data transfer. This provides high reliability and redundancy, but it can be expensive and difficult to install.

- Advantages:
  - High reliability and redundancy
  - Multiple paths for data transfer
- Disadvantages:
  - Expensive
  - Difficult to install

### 25.2 - Network Protocols

=========================

### Definition

Network protocols are sets of rules that govern how devices communicate with each other on a network. They define the format of data, how data is transmitted, and how errors are handled.

### Types of Network Protocols

#### 1. **Application Layer Protocols**

Application layer protocols are used for communication between applications running on different devices. Examples include HTTP, FTP, and SMTP.

- Examples:
  - HTTP (Hypertext Transfer Protocol)
  - FTP (File Transfer Protocol)
  - SMTP (Simple Mail Transfer Protocol)

#### 2. **Transport Layer Protocols**

Transport layer protocols are used for reliable data transfer between devices. Examples include TCP and UDP.

- Examples:
  - TCP (Transmission Control Protocol)
  - UDP (User Datagram Protocol)

#### 3. **Internet Layer Protocols**

Internet layer protocols are used for routing data between devices on different networks. Examples include IP and ICMP.

- Examples:
  - IP (Internet Protocol)
  - ICMP (Internet Control Message Protocol)

#### 4. **Link Layer Protocols**

Link layer protocols are used for framing and error-free transfer of data frames between devices on the same network. Examples include Ethernet and Wi-Fi.

- Examples:
  - Ethernet
  - Wi-Fi

### Key Concepts

---

- **Packet switching**: A method of data transmission where data is broken into small packets and transmitted independently.
- **Routing**: The process of determining the best path for data to travel between devices on different networks.
- **Network architecture**: The overall design and structure of a network, including the devices, protocols, and topologies used.
