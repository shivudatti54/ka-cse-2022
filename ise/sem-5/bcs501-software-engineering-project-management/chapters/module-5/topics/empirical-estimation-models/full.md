# **Empirical Estimation Models**

## **Introduction**

Empirical estimation models are a type of software estimation technique that relies on historical data and real-world experience to estimate the complexity and effort required for a software project. These models are widely used in software engineering and project management because they provide a more accurate estimate of project costs and durations compared to traditional models that rely on personal opinions and anecdotal evidence.

## **Historical Context**

The concept of empirical estimation models dates back to the 1960s, when the first software estimation techniques were developed. However, it wasn't until the 1980s that empirical estimation models began to gain popularity. The development of the Software Engineering Body of Knowledge (SWEBOK) in 1993 marked a significant milestone in the history of empirical estimation models. The SWEBOK recognized the importance of empirical estimation models and provided a framework for their development and application.

## **Types of Empirical Estimation Models**

There are several types of empirical estimation models, each with its strengths and weaknesses. Some of the most popular empirical estimation models include:

### 1. **Program Estimation using the COCOMO Model**

The COCOMO (Construction Cost and Development Time) model is a widely used empirical estimation model that was developed in the 1970s. The model estimates the effort required for a software project based on several factors, including the program's size, complexity, and the experience of the development team.

#### Formula:

Effort (E) = 22.5 \* KC \* S^1.05

Where:

- E = effort required for the project
- K = software development rate (KLOCs/day)
- C = calendar months in the development period
- S = software size (KLOCs)

### 2. **Program Estimation using the Software Engineering Institute's (SEI) Project Risk Model**

The SEI's Project Risk Model is an empirical estimation model that estimates the effort required for a software project based on the project's risk factors. The model takes into account factors such as the project's size, complexity, and the experience of the development team.

#### Formula:

Effort (E) = 800 \* (size + complexity) \* risk factor

Where:

- E = effort required for the project
- size = software size (KLOCs)
- complexity = software complexity (1-10)
- risk factor = project risk factor (1-10)

### 3. **Program Estimation using the Agile Estimation Model**

The Agile Estimation Model is an empirical estimation model that is based on the Agile software development methodology. The model estimates the effort required for a software project based on the project's size, complexity, and the experience of the development team.

#### Formula:

Effort (E) = 40 \* KC \* S^1.15

Where:

- E = effort required for the project
- K = software development rate (KLOCs/day)
- C = calendar months in the development period
- S = software size (KLOCs)

### 4. **Program Estimation using the Parametric Estimation Model**

The Parametric Estimation Model is an empirical estimation model that estimates the effort required for a software project based on historical data and real-world experience. The model takes into account factors such as the project's size, complexity, and the experience of the development team.

#### Formula:

Effort (E) = b \* k \* log(n)

Where:

- E = effort required for the project
- b = base cost (KLOCs)
- k = productivity factor (KLOCs/day)
- log(n) = logarithmic function of the project size (KLOCs)
