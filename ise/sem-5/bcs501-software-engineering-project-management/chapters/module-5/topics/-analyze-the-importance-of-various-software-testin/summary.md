# **Software Testing Methods and Agile Methodology**

## **Importance of Software Testing Methods**

- **Equivalence Partitioning**: Divides the input data into subsets to ensure comprehensive testing
- **Boundary Value Analysis**: Tests the input values at the boundaries of the partition
- **State Transition Testing**: Verifies the software's behavior during different states
- **Black Box Testing**: Tests the software without knowledge of its internal workings
- **White Box Testing**: Tests the software with knowledge of its internal workings
- **Grey Box Testing**: Tests the software with partial knowledge of its internal workings
- **Test-Driven Development (TDD)**: Writing tests before writing code
- **Behavior-Driven Development (BDD)**: Tests the software's behavior using natural language

## **Agile Methodology**

- **Manifesto for Agile Software Development**: Emphasizes individualism, collaboration, and iterative progress
- **Scrum Framework**: Iterative development, sprints, and daily stand-ups
- **Kanban Method**: Visual management, limiting work in progress, and continuous improvement
- **Lean Software Development**: Eliminate waste, maximize value, and continuous improvement

## **Key Formulas and Definitions**

- **Test Efficiency**: The ratio of passing tests to total tests executed
- **Test Effectiveness**: The ratio of defects found to total defects
- **Defect Density**: The number of defects per unit of code
- **Defect Rate**: The number of defects per unit of time

## **Important Theorems**

- **Dijkstra's Algorithm**: Finds the shortest path in a graph
- **Bellman's Algorithm**: Finds the shortest path in a weighted graph
