# **Summary: Techniques to Enhance Software Quality**

### Understanding Quality Metrics

- **Definition:** Software quality refers to the degree to which a software system meets its requirements and user expectations.
- **Key Quality Metrics:**
  - Code coverage
  - Test coverage
  - Defect density
  - Defect leakage

### Testing Techniques

- **Unit Testing:**
  - Definition: Testing individual components or units of code.
  - Formula: TTT = (Number of test cases) / (Number of test cases + 1)
- **Integration Testing:**
  - Definition: Testing interactions between components or units of code.
  - Formula: ITT = (Number of test cases) / (Number of test cases + 1)
- **Equivalence Partitioning:**
  - Definition: Dividing input data into partitions based on equivalence.
  - Formula: EP = (Number of partitions) \* (Number of test cases per partition)

### Design Techniques

- **Separation of Concerns (SoC):**
  - Definition: Separating logical concerns or modules in software design.
- **Single Responsibility Principle (SRP):**
  - Definition: Assigning a single responsibility to a module or component.

### Best Practices

- **Pair Programming:**
  - Definition: Collaborating on code writing between two programmers.
- **Code Review:**
  - Definition: Reviewing code written by others to ensure quality.

### Additional Techniques

- **Test-Driven Development (TDD):**
  - Definition: Writing tests before writing code.
- **Behavior-Driven Development (BDD):**
  - Definition: Defining software behavior through natural language.

### Important Formulas and Theorems

- **Cockburn's Law:** "Good design makes that which is easily broken, easily fixed."

### Key Terms

- **Testability:** The ability to test a software component or module.
- **Maintainability:** The ease with which a software system can be modified or updated.
