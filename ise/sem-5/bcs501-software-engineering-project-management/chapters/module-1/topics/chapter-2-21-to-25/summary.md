# **Chapter 2: 2.1 to 2.5 Revision Notes**

## **2.1 Software Project Management**

- Definition: The application of knowledge, skills, methods, and tools to execute an IT project effectively.
- Importance of project management: Ensures project success, meets stakeholder expectations, and delivers value to the organization.
- Key aspects:
  - Scope, schedule, budget, quality, and resources
  - Project life cycle: initiation, planning, execution, monitoring and control, closure

## **2.2 Project Life Cycle**

- Definition: A series of stages through which a project progresses from initiation to closure.
- Phases:
  - Initiation: Defines project scope, goals, and stakeholders.
  - Planning: Develops project plan, including schedule, budget, and resources.
  - Execution: Carries out the project work.
  - Monitoring and Control: Tracks progress, identifies issues, and takes corrective action.
  - Closure: Formalizes project completion and evaluates lessons learned.

## **2.3 Project Management Processes**

- Definition: A set of tasks and activities that support project management.
- Key processes:
  - Initiating: Defining project scope, goals, and stakeholders.
  - Planning: Developing project plan, including schedule, budget, and resources.
  - Executing: Carrying out project work.
  - Monitoring and Controlling: Tracking progress, identifying issues, and taking corrective action.
  - Closing: Formalizing project completion and evaluating lessons learned.

## **2.4 Project Management Knowledge Areas**

- Definition: A specific area of knowledge that contributes to project management.
- Key knowledge areas:
  - Integration Management: Ensures that all aspects of the project are properly integrated.
  - Scope Management: Defines project scope, goals, and stakeholders.
  - Time Management: Develops project schedule, including milestones and deadlines.
  - Cost Management: Establishes project budget and monitors costs.
  - Quality Management: Ensures that project deliverables meet customer requirements.

## **2.5 Project Management Body of Knowledge (PMBOK)**

- Definition: A comprehensive framework for project management that provides a set of guidelines and best practices.
- Key aspects:
  - Five process groups: Initiating, Planning, Executing, Monitoring and Controlling, and Closing.
  - Ten knowledge areas: Integration Management, Scope Management, Time Management, Cost Management, Quality Management, Resource Management, Communications Management, Risk Management, Procurement Management, and Stakeholder Management.

**Formulas and Definitions:**

- **Germann's Law:** "The best project manager is the one who leaves the project when it's finished."
- **Project Management Maturity Model (PMM):** A framework for measuring project management maturity levels.
- **Earned Value Management (EVM):** A method for measuring project performance based on earned value, cost variance, and schedule performance index.

**Important Theorems:**

- **The Project Management Triangle:** A triangle model that illustrates the interdependence of scope, time, and cost in project management.
- **The Five-Step Method:** A structured approach to project management that involves defining project scope, developing a project plan, executing the project, monitoring and controlling, and closing the project.
