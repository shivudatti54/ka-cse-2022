# **Principles that Guide Practice: Software Engineering Knowledge**

### Key Principles

- **Modularity**: Break down large systems into smaller, independent modules for easier maintenance and modification.
- **Abstraction**: Hide complex implementation details and show only necessary information to the user or other components.
- **Encapsulation**: Bundle data and methods that operate on that data within a single unit (class, module, etc.) to improve code organization and security.
- **Scalability**: Design systems to grow and adapt to changing requirements without compromising performance or functionality.
- **Reusability**: Develop components and code that can be used in multiple contexts to reduce development time and improve efficiency.

### Design Patterns

- **Single Responsibility Principle (SRP)**: A class should have only one reason to change.
- **Open-Closed Principle (OCP)**: A class should be open for extension but closed for modification.
- **Liskov Substitution Principle (LSP)**: Derived classes should be substitutable for their base classes.

### Software Engineering Theorems

- **Abstraction Theorem**: Abstraction helps reduce complexity and improve modularity.
- **Modularity Theorem**: Modularity improves the maintainability and scalability of software systems.

### Important Formulas and Definitions

- **Friedman-Nijenhuis Theorem**: Provides a lower bound for the complexity of a system.
- **Wirth's Theorem**: States that a good programming language should have a minimal set of features.

### Important Theorems (continued)

- **Church-Rosser Theorem**: Helps prove the existence of a Turing complete language.
- **Jacobsen's Theorem**: Demonstrates the importance of encapsulation in software design.

### Additional Key Points

- **Separation of Concerns (SoC)**: Divide a system into distinct components that handle specific tasks to improve maintainability.
- **Don't Repeat Yourself (DRY)**: Avoid duplicated code to reduce maintenance time and improve code quality.
