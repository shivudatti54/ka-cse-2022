# **Software Engineering & Project Management**

## **Module: Chapter 4: 4.1 to 4.3**

### 4.1: Introduction to Software Engineering

---

#### Definition:

Software engineering is the application of engineering principles and techniques to the development of software systems. It involves the careful planning, design, development, testing, and maintenance of software applications.

#### Key Concepts:

- **Software development lifecycle (SDLC):** A series of phases that a software project goes through, from initiation to delivery.
- **Software engineering principles:** Guiding principles that help software engineers develop high-quality software systems.
- **Software engineering tools and techniques:** Various tools and techniques used by software engineers to develop software systems.

#### Example:

A software engineer uses the SDLC to develop a new mobile app. The project goes through the following phases: requirements gathering, design, development, testing, and deployment.

### 4.2: Project Management

---

#### Definition:

Project management is the process of planning, organizing, and controlling resources to achieve specific project goals and objectives.

#### Key Concepts:

- **Project management framework:** A structured approach to managing projects, including tools and techniques.
- **Project scope:** The specific goals and objectives of a project.
- **Project schedule:** The timeline for completing a project.
- **Project budget:** The financial resources allocated to a project.

#### Example:

A project manager uses a project management framework to plan and execute a new software development project. The project scope includes developing a new web application, the project schedule includes a 6-month timeline, and the project budget includes $100,000.

### 4.3: Software Engineering Process

---

#### Definition:

The software engineering process is a set of activities that software engineers use to develop software systems.

#### Key Concepts:

- **Phases of the software engineering process:** Requirements gathering, design, development, testing, and maintenance.
- **Software engineering process models:** Various models that describe the software engineering process, including waterfall, agile, and iterative.
- **Software engineering process models:** Various models that describe the software engineering process.

#### Example:

A software engineer uses a software engineering process model to develop a new software application. The process includes requirements gathering, design, development, testing, and maintenance, and the model used is an iterative approach.

## **Key Takeaways**

- Software engineering is the application of engineering principles and techniques to software development.
- Project management is the process of planning, organizing, and controlling resources to achieve specific project goals and objectives.
- The software engineering process is a set of activities used to develop software systems.
