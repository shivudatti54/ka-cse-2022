# **Principles that Guide Each Framework Activity**

## **Software Engineering Principles**

- **Abstraction**: Separate essential features from non-essential details.
- **Encapsulation**: Hide internal implementation details from external interference.
- **Inheritance**: Build upon existing components to create new ones.
- **Polymorphism**: Treat different types of objects uniformly.

## **Project Management Principles**

- **Scope**: Define project boundaries and deliverables.
- **Time**: Establish realistic milestones and deadlines.
- **Cost**: Set budget constraints and track expenses.
- **Quality**: Ensure project meets stakeholder expectations.

## **Key Formulas and Definitions**

- **Waterfall Model**: Input → Analysis → Design → Implementation → Testing → Deployment
- **Agile Model**: Iterative → Incremental → Flexible → Adaptive
- **V-Model**: Requirements → Design → Implementation → Testing → Review

## **Important Theorems**

- **Kanban Theorem**: Limit work in progress to improve flow and reduce waste.
- **Pareto Principle**: 80% of results come from 20% of efforts.

## **Revision Tips**

- Review key principles and formulas regularly.
- Practice applying principles to real-world scenarios.
- Focus on understanding the underlying concepts rather than memorizing formulas.
