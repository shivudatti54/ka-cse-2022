# **Textbook 1: Chapter 3: 3.1 to 3.6 Revision Notes**

## **3.1: Introduction to Software Engineering**

- Definition: Software engineering is the application of engineering principles and techniques to the development of software.
- Key concepts:
  - Software engineering as a profession
  - Software engineering discipline
- Important formulas: None

## **3.2: Types of Software**

- Definition: Software can be classified into four categories:
  1.  Application software
  2.  System software
  3.  Firmware
  4.  Device software
- Key concepts:
  - Characteristics of each category
  - Examples of each type of software

## **3.3: Software Requirements**

- Definition: Software requirements are the specifications that define what the software should do.
- Key concepts:
  - Functional requirements
  - Non-functional requirements
  - User requirements
- Important formulas: None

## **3.4: Software Design**

- Definition: Software design is the process of creating a detailed plan for the software.
- Key concepts:
  - Object-oriented design
  - Systems design
  - Data design
- Important formulas: None

## **3.5: Software Construction**

- Definition: Software construction is the process of coding the software.
- Key concepts:
  - Programming languages
  - Programming paradigms
  - Software development life cycles
- Important formulas: None

## **3.6: Software Maintenance**

- Definition: Software maintenance is the process of modifying and updating the software.
- Key concepts:
  - Types of maintenance: maintenance, repair, and upgrade
  - Software maintenance life cycle
- Important formulas: None

**Key Definitions:**

- **Engineering discipline**: the application of scientific and mathematical principles to solve real-world problems
- **Software engineering**: the application of engineering principles and techniques to the development of software
