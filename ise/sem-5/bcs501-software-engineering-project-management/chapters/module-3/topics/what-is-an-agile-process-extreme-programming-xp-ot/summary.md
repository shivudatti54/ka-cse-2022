# **Agile Processes and Software Engineering**

### What is an Agile Process?

- Definition: An iterative and adaptive approach to project management that emphasizes flexibility and collaboration.
- Key characteristics:
  - Incremental development
  - Continuous improvement
  - Flexibility and adaptability
  - Customer and team involvement
- Importance: Helps teams respond to change, deliver working software, and satisfy customer needs.

### Extreme Programming (XP)

- Definition: A software development methodology that emphasizes technical practices such as pair programming, continuous integration, and refactoring.
- Key practices:
  - Pair programming
  - Continuous integration
  - Refactoring
  - Test-driven development
  - Continuous feedback
- Benefits:
  - Improved code quality
  - Increased productivity
  - Enhanced team collaboration

### Other Agile Process Models

- **Scrum**:
  - Framework for managing and completing complex projects using iterative and incremental practices.
  - Key roles: Product Owner, Scrum Master, Development Team.
  - Key ceremonies: Sprint Planning, Daily Scrum, Sprint Review.
- **Kanban**:
  - Visual system for managing work, emphasizing continuous flow and limit of workflow capacity.
  - Key principles: Visualize workflow, Limit work in progress, Continuously improve.
- **Lean Software Development**:
  - Approach to software development that emphasizes elimination of waste, continuous improvement, and delivery of value to customers.

**Formulas, Definitions, and Theorems**

- **Agile Manifesto**:
  - Individuals and interactions over processes and tools
  - Working software over comprehensive documentation
  - Customer collaboration over contract negotiation
  - Responding to change over following a plan
- **Kanban Cycle Time**:
  - The time it takes for a task to move through the workflow.
