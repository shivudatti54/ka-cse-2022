# **Requirements Modeling Scenarios, Information and Analysis Classes**

### Overview

- Software engineering and project management module covering requirements modeling and scenario-based analysis
- Focus on requirement analysis, scenario-based modeling, and UML models that supplement the Use Case (Use C)

### Key Points

---

#### Requirement Analysis

- Definition: The process of identifying, documenting, and prioritizing functional and non-functional requirements
- Importance: Ensures that the software meets the needs and expectations of stakeholders
- Techniques:
  - Use Cases
  - Use Case Diagrams
  - Use Case Scenarios
  - Use Case Models

#### Scenario-Based Modeling

- Definition: A design technique that models the system's behavior using scenarios to describe the interactions between actors and the system
- Techniques:
  - Use Case Scenarios
  - Act-Actor Dialogue
  - Use of Actors and Use Cases
  - Scenario-Based Modeling Tools

#### UML Models that Supplement the Use Case

- Definition: UML models that provide additional information and insights to support the Use Case model
- Types of Models:
  - Class Diagrams
  - Object Diagrams
  - State Machine Diagrams
  - Activity Diagrams
  - Sequence Diagrams

### Important Formulas, Definitions, and Theorems

---

- **Use Case Diagram Formula**: `U = {Actors, Use Cases, Associations}` (where `U` is the set of Use Cases)
- **Use Case Model Theorem**: "The Use Case model is valid if it satisfies the following conditions:
  1.  Completeness: All possible use cases are included.
  2.  Consistency: The use cases are consistent with each other.
  3.  Completeness: All actors are correctly associated with their use cases."

### Quick Revision Notes

---

- Requirements analysis is the foundation of software engineering

- UML models can supplement the Use Case model by providing additional information and insights
