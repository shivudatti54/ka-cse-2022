# Chapter 7: 7.1 to 7.3 Revision Notes

=====================================

### Overview

- This chapter covers the following topics:
  - 7.1: Software Requirements Analysis
  - 7.2: Software Design
  - 7.3: Software Implementation

### Software Requirements Analysis (7.1)

---

- Definition: The process of defining, identifying, and documenting the functional and non-functional requirements of a software system.
- Importance: Ensures that the software meets the needs of the users and stakeholders.
- Key Points:
  - Functional requirements: what the software should do
  - Non-functional requirements: constraints on the software's behavior (e.g., performance, security)
  - User stories: a natural language description of a software feature from the user's perspective
  - Use cases: a description of how the user interacts with the software

### Software Design (7.2)

---

- Definition: The process of creating a detailed and coherent plan for the software's architecture, components, and interactions.
- Importance: Ensures that the software is scalable, maintainable, and meets the functional and non-functional requirements.
- Key Points:
  - Architecture patterns: reuse of existing designs to solve common software problems
  - Design patterns: solutions to specific design problems
  - Component-based design: breaking down the software into smaller, independent components
  - Interfaces: the boundaries between components

### Software Implementation (7.3)

---

- Definition: The process of translating the software design into code.
- Importance: Ensures that the software is implemented correctly and meets the requirements.
- Key Points:
  - Programming languages: languages used to write the software code
  - Data structures: the way data is organized and stored in the software
  - Algorithms: the steps taken to solve a specific problem
  - Testing: verifying that the software meets the requirements

### Important Formulas, Definitions, and Theorems

---

- Definition of a requirement: "A requirement is a statement of what the software should do or be."
- Definition of a use case: "A use case describes how a user interacts with the software."
- Theorem of Decomposition: "Any complex system can be broken down into smaller, simpler components."

### Key Concepts

---

- Requirements engineering: the process of defining, identifying, and documenting the requirements.
- Design patterns and architecture patterns: reuse of existing designs to solve common software problems.
- Testing: verifying that the software meets the requirements.
