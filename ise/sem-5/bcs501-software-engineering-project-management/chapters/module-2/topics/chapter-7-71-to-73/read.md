# **Chapter 7: 7.1 to 7.3**

## **7.1: Software Requirements**

### Definition

Software requirements are the specifications of what the software should do, including functional, performance, and usability requirements.

### Types of Software Requirements

- Functional requirements: describe what the software should do (e.g., process orders)
- Non-functional requirements: describe how the software should behave (e.g., response time, security)
- Performance requirements: describe the capabilities of the software (e.g., throughput, memory usage)
- Usability requirements: describe how the software should be used (e.g., user interface, navigation)

### Example

A retail company wants to develop a software application for managing customer orders. The functional requirements of the software might include:

- Process orders
- Update inventory levels
- Send order confirmations to customers

The non-functional requirements might include:

- Response time: the software should respond to user input within 2 seconds
- Security: the software should ensure that customer data is encrypted and secure

### Key Concepts

- Requirements gathering: identifying and documenting software requirements
- Requirements analysis: evaluating and prioritizing software requirements
- Requirements specification: creating a detailed document of software requirements

## **7.2: Software Design**

### Definition

Software design is the process of creating a detailed plan for the software architecture, including the components, interactions, and data structures.

### Types of Software Designs

- Object-oriented design (OOD): emphasizes modularity, reusability, and abstraction
- Component-based design: emphasizes modularity and reuse
- Layered design: emphasizes separation of concerns and modularity

### Example

A software company wants to develop a web-based email client. The software design might include:

- A user interface layer that displays the email inbox and allows users to compose new emails
- A business logic layer that processes email requests and updates the inbox
- A data storage layer that stores email messages and user data

### Key Concepts

- Architectural patterns: reusable design solutions for common software problems
- Design patterns: reusable solutions for specific design problems
- Software architecture: the overall structure and organization of the software

## **7.3: Software Construction**

### Definition

Software construction is the process of translating software design into working software.

### Phases of Software Construction

1.  **Translation**: translating design specifications into code
2.  **Compilation**: converting source code into machine code
3.  **Linking**: combining object files into a single executable file
4.  **Testing**: verifying that the software meets requirements and works correctly

### Example

A software company wants to develop a mobile app for tracking fitness goals. The software construction process might involve:

- Writing source code to implement the user interface and business logic
- Compiling the source code into object files
- Linking the object files into a single executable file
- Testing the app to ensure it meets requirements and works correctly

### Key Concepts

- Programming languages: languages used to write software code
- Development tools: software used to support the software construction process
- Debugging: identifying and fixing errors in the software
