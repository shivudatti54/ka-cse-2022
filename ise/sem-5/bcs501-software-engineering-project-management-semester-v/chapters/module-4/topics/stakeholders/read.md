# Project Planning and Stakeholders

## Stakeholders

### Who Are Stakeholders?

Stakeholders are individuals or organizations actively involved in the project or whose interests may be affected by project execution or completion.

### Stakeholder Categories

| Category      | Examples                                    |
| ------------- | ------------------------------------------- |
| **Internal**  | Project team, management, other departments |
| **External**  | Customers, suppliers, regulators, end users |
| **Primary**   | Directly affected by project outcomes       |
| **Secondary** | Indirectly affected                         |

### Key Stakeholders

1. **Project Sponsor** - Provides resources, authority
2. **Project Manager** - Leads day-to-day management
3. **Customers/Users** - Will use the product
4. **Development Team** - Builds the product
5. **Functional Managers** - Provide resources
6. **Steering Committee** - Governance oversight

### Stakeholder Analysis

**Steps:**

1. Identify all stakeholders
2. Assess their interests and influence
3. Determine their requirements
4. Plan engagement strategies

**Power/Interest Grid:**

```
        High │ Keep        │ Manage
      Power  │ Satisfied   │ Closely
             ├─────────────┼───────────
        Low  │ Monitor     │ Keep
      Power  │ (Minimum)   │ Informed
             └─────────────┴───────────
              Low Interest  High Interest
```

## Setting Objectives

### SMART Objectives

- **S**pecific - Clear and well-defined
- **M**easurable - Quantifiable success criteria
- **A**chievable - Realistic and attainable
- **R**elevant - Aligned with business goals
- **T**ime-bound - Has deadline

### Types of Objectives

| Type            | Focus                |
| --------------- | -------------------- |
| **Strategic**   | Business alignment   |
| **Tactical**    | Project deliverables |
| **Operational** | Day-to-day execution |

## Business Case

### Purpose

The business case justifies the project investment by demonstrating:

- Expected benefits
- Estimated costs
- Return on investment
- Alignment with strategy

### Business Case Components

1. **Executive Summary**
2. **Problem/Opportunity Statement**
3. **Analysis of Options**
4. **Benefits Analysis**
5. **Cost Analysis**
6. **Risk Assessment**
7. **Implementation Timeline**
8. **Recommendation**

### Benefits Realization

**Tangible Benefits:**

- Cost savings
- Revenue increase
- Time savings
- Productivity improvement

**Intangible Benefits:**

- Customer satisfaction
- Brand reputation
- Employee morale
- Competitive advantage

## Project Success and Failure

### Traditional Success Criteria (Iron Triangle)

```
        Scope
          /\
         /  \
        /    \
       /      \
      /________\
    Time      Cost
```

**Success = Delivered on:**

- **Scope** - All required features
- **Time** - Within schedule
- **Cost** - Within budget

### Modern Success Factors

Beyond the iron triangle:

- **Quality** - Meets standards
- **Stakeholder Satisfaction** - Happy customers
- **Team Satisfaction** - Healthy team
- **Strategic Alignment** - Business value
- **Sustainability** - Maintainable solution

### Common Failure Reasons

| Category          | Examples                            |
| ----------------- | ----------------------------------- |
| **Planning**      | Poor estimates, unclear scope       |
| **Resources**     | Insufficient skills, understaffing  |
| **Communication** | Poor stakeholder engagement         |
| **Technical**     | Wrong technology choices            |
| **Management**    | Weak leadership, no risk management |
| **External**      | Market changes, regulatory changes  |

### Standish Group CHAOS Report Findings

**Project Outcomes:**

- Successful (on time, budget, scope): ~30%
- Challenged (partial success): ~50%
- Failed (cancelled): ~20%

**Success Factors:**

1. Executive support
2. User involvement
3. Clear requirements
4. Proper planning
5. Realistic expectations

## Project Planning Process

### Planning Steps

1. **Define scope** - What's included/excluded
2. **Create WBS** - Break down work
3. **Estimate effort** - Resources needed
4. **Develop schedule** - Timeline
5. **Plan resources** - People, tools
6. **Budget costs** - Financial plan
7. **Identify risks** - What could go wrong
8. **Define quality** - Standards
9. **Plan communication** - Information flow

### Work Breakdown Structure (WBS)

Hierarchical decomposition of project deliverables.

```
Project
├── Phase 1
│   ├── Task 1.1
│   ├── Task 1.2
│   └── Task 1.3
├── Phase 2
│   ├── Task 2.1
│   └── Task 2.2
└── Phase 3
    └── Task 3.1
```

**WBS Principles:**

- 100% rule - All work included
- Mutually exclusive - No overlap
- Outcome-oriented - Deliverable focus
- Manageable chunks - 8-80 hours

## Key Takeaways

1. **Stakeholder analysis** identifies interests and influence
2. **SMART objectives** ensure clarity and measurability
3. **Business case** justifies project investment
4. Success includes **scope, time, cost** plus quality and satisfaction
5. **WBS** breaks project into manageable pieces
6. Most projects face **challenges** - planning reduces risk
