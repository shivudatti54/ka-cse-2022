### **Activities Covered by Software Project Management**

Software Project Management (SPM) is the application of knowledge, skills, tools, and techniques to project activities to meet the project requirements. It is a structured framework that guides a software project from its initial conception to its final deployment and maintenance. The primary goal of SPM is to deliver a high-quality software product on time and within budget. This involves a series of interconnected activities that can be broadly categorized into the following phases.

#### **1. Project Planning**

This is the foundational activity where the project's roadmap is created. It involves:

- **Feasibility Study:** Analyzing technical, operational, and economic feasibility to determine if the project should be undertaken.
- **Resource Planning:** Identifying and allocating human resources, hardware, software, and tools needed.
- **Cost Estimation:** Predicting the effort (in person-months) and cost required to complete the project using techniques like **COCOMO** (Constructive Cost Model).
- **Scheduling:** Creating a timeline for the project using tools like **Gantt Charts** and **PERT/CPM** (Program Evaluation and Review Technique/Critical Path Method) to track progress and identify critical tasks.
- **Risk Management:** Identifying potential risks (technical, managerial, organizational) and planning mitigation strategies.

_Example: Before building a new mobile app, a project manager creates a plan detailing the team size, required APIs, a 6-month timeline with milestones, and a list of potential risks like changing third-party library dependencies._

#### **2. Project Scope Management**

This involves defining and controlling what is included _in_ the project and, just as importantly, what is _not_. It includes:

- **Requirement Gathering:** Eliciting functional and non-functional requirements from stakeholders.
- **Defining Scope:** Creating a clear and detailed Statement of Work (SOW) or scope document.
- **Managing Scope Changes:** Implementing a formal change control process to handle "scope creep," which is the uncontrolled addition of new features.

#### **3. Project Execution**

This is the phase where the actual development work takes place. Key activities include:

- **Team Coordination:** Assigning tasks, conducting meetings (daily stand-ups, reviews), and facilitating communication.
- **Quality Assurance:** Implementing processes to ensure the software meets quality standards. This includes activities like **testing**, **code reviews**, and adhering to coding standards.
- **Procurement:** Acquiring necessary software, hardware, or services from external vendors.

#### **4. Project Monitoring & Controlling**

This activity runs concurrently with project execution. It involves tracking, reviewing, and regulating the progress to meet performance objectives.

- **Tracking Progress:** Comparing actual progress against the planned schedule and budget using milestones and deliverables.
- **Performance Measurement:** Using techniques like **Earned Value Management (EVM)** to objectively measure project performance.
- **Taking Corrective Actions:** If the project is off track, the manager must identify the cause and take steps to correct the course (e.g., reallocating resources, revising estimates).

#### **5. Project Closure**

This final phase marks the formal completion of the project. Activities include:

- **Client Acceptance:** Obtaining formal sign-off from the customer that the delivered product meets the requirements.
- **Post-Project Review:** Conducting a retrospective analysis to document lessons learned—what went well and what could be improved for future projects.
- **Releasing Resources:** Reassigning team members and closing out all contracts and financial accounts related to the project.
- **Archiving:** Creating a final project report and storing all project documentation for future reference.

### **Key Points & Summary**

| Activity                     | Primary Focus                         | Key Outcome                        |
| :--------------------------- | :------------------------------------ | :--------------------------------- |
| **Planning**                 | Creating the project blueprint        | Project Plan, Schedule, Budget     |
| **Scope Management**         | Defining and controlling deliverables | Clear Scope Statement              |
| **Execution**                | Building the software product         | Working Software, Test Reports     |
| **Monitoring & Controlling** | Tracking progress and performance     | Status Reports, Corrective Actions |
| **Closure**                  | Formalizing project completion        | Client Sign-off, Lessons Learned   |

In essence, Software Project Management is a continuous cycle of planning, doing, checking, and acting. These activities are not strictly sequential; they often overlap and interact throughout the project's lifecycle. Effective SPM ensures that a software project is completed successfully, satisfying all stakeholders.
