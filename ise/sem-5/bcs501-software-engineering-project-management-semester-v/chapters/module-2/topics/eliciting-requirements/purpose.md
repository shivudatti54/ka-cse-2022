# Learning Objectives

After studying this topic, you should be able to:

1. Define Security Requirements Engineering and explain its importance in the Secure SDLC
2. Differentiate between functional, non-functional, and policy-based security requirements
3. Apply various elicitation techniques such as abuse cases, threat modeling, and security use cases
4. Document security requirements using appropriate templates and traceability matrices
5. Identify common categories of security requirements for authentication, authorization, data protection, and input validation
6. Integrate security requirements into agile methodologies and DevSecOps practices
