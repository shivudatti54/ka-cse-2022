# Learning Objectives

After studying this topic, you should be able to:

1. Define requirements analysis and explain its importance in the software development lifecycle
2. Differentiate between functional and non-functional requirements with clear examples
3. Apply various requirements analysis techniques including DFDs, ERDs, and use case diagrams
4. Evaluate requirements using quality criteria such as clarity, consistency, and verifiability
5. Prioritize requirements using appropriate methods like MoSCoW and the Kano model
6. Identify common challenges in requirements analysis and propose mitigation strategies
