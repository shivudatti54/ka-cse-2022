# Learning Objectives
After studying this topic, you should be able to:
1.  Define Apache Hive and explain its primary role in the Hadoop ecosystem as a data warehousing solution.
2.  Describe the core components of Hive's architecture (Metastore, Driver, Execution Engine) and their interactions.
3.  Differentiate between Managed and External tables and explain the implications of creating/dropping each.
4.  Explain the concept of partitioning and its significance for query performance optimization (partition pruning).
5.  Compare and contrast common Hive file formats (e.g., TextFile, ORC, Parquet) and justify the use of columnar formats for analytics.
6.  Write basic HiveQL DDL and DML statements to create tables and query data.
7.  Contrast Hive's 'Schema-on-Read' approach and batch-oriented nature with the 'Schema-on-Write' and transactional nature of traditional RDBMS.
