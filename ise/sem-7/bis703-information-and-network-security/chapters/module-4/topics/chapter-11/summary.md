# Chapter 11: Information and Network Security

### Key Concepts

- **Definition:** Information and network security refers to the practices, technologies, and policies designed to safeguard information and computer networks from unauthorized access, use, disclosure, disruption, modification, or destruction.
- **Types of Security:**
  - Confidentiality: Protecting sensitive information from unauthorized access.
  - Integrity: Ensuring the accuracy and completeness of data.
  - Availability: Ensuring access to authorized users.

### Threats and Vulnerabilities

- **Threats:**
  - Malware (viruses, worms, trojans)
  - Social engineering
  - Physical attacks (hacking, break-in)
  - Insider threats (employees, contractors)
- **Vulnerabilities:**
  - Weak passwords
  - Unpatched software
  - Insufficient access controls
  - Insecure protocols (HTTP, FTP)

### Security Controls

- **Access Control:**
  - Authentication (login, password)
  - Authorization (role-based access control)
  - Accounting (logging, auditing)
- **Cryptography:**
  - Symmetric encryption (AES, DES)
  - Asymmetric encryption (RSA, Diffie-Hellman)
  - Hash functions (MD5, SHA-256)

### Risk Management

- **Risk Assessment:**
  - Identify threats and vulnerabilities
  - Evaluate potential impact
  - Prioritize mitigation actions
- **Risk Mitigation:**
  - Implement security controls
  - Monitor and audit
  - Continuously review and update

### Important Formulas and Theorems

- **CIA Triad:**
  - Confidentiality (C): Protecting sensitive information
  - Integrity (I): Ensuring data accuracy and completeness
  - Availability (A): Ensuring access to authorized users
- **Bell-LaPadula Model:**
  - Sensitive data is handled by a single process
  - Access is controlled based on classify Cleartext Data (CCD) matrix
- **Biba Model:**
  - Sensitive data is handled by multiple processes
  - Access is controlled based on Read/Write Access Control Matrix (RWACM)
