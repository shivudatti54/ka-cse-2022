### Introduction to Hadoop

- Hadoop is a distributed computing framework used for processing large data sets.
- It was first introduced in 2004 by Doug Cutting and Mike Cafarella.
- Hadoop is not a relational database management system (RDBMS), but rather a data processing engine.

### Why Hadoop?

- Scalability: Hadoop can handle large amounts of data and scale horizontally.
- Fault tolerance: Hadoop can handle node failures and continue processing data.
- Cost-effective: Hadoop is open-source and can be run on commodity hardware.

### Why Not RDBMS?

- RDBMS is designed for transactions and may not be efficient for large-scale data processing.
- RDBMS requires a fixed schema, which can be inflexible for big data.

### RDBMS vs Hadoop

- **Hadoop**:
  - Distributed computing framework
  - Handles large amounts of data
  - Scalable and fault-tolerant
- **RDBMS**:
  - Relational database management system
  - Designed for transactions
  - Limited scalability and fault tolerance

### History of Hadoop

- 2004: Doug Cutting and Mike Cafarella introduce Hadoop.
- 2006: Hadoop is open-sourced and released under the Apache License.
- 2009: Hadoop gains popularity and is adopted by several large-scale data processing projects.

### Important Formulas and Definitions

- **MapReduce**: A programming model used for processing large data sets in parallel.
- **Shuffle**: A step in the MapReduce process where data is sorted and re-partitioned.
- **Reduce**: A step in the MapReduce process where data is aggregated and processed.

### Key Concepts

- **Distributed computing**: A computing model where multiple processing nodes work together to achieve a common goal.
- **Big data**: A term used to describe large, complex data sets that are difficult to process using traditional data processing tools.
- **Hadoop ecosystem**: A collection of tools and technologies that are used with Hadoop, including HDFS, MapReduce, and Pig.
