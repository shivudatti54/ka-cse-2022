## Introduction

Map Reduce is a programming model used for processing large data sets in parallel across a cluster of computers. It is a key component of the Hadoop ecosystem, which is a distributed computing framework that stores and processes large data sets. Map Reduce provides a flexible and scalable way to process data, making it an ideal choice for big data analytics.

## What is Map Reduce?

Map Reduce is a programming model that consists of two main components: the Mapper and the Reducer. The Mapper is responsible for breaking down the data into smaller chunks, while the Reducer is responsible for aggregating the results from the Mapper.

Here's a high-level overview of the Map Reduce process:

1. **Input**: The input data is fed into the Map Reduce system.
2. **Mapper**: The Mapper breaks down the input data into smaller chunks, called keys and values. The Mapper applies a user-defined function to each chunk of data.
3. **Shuffle**: The output from the Mapper is stored in a temporary storage area called the Shuffle.
4. **Reducer**: The Reducer receives the output from the Shuffle and aggregates the results using a user-defined function.
5. **Output**: The final output is written to disk.

## History of Map Reduce

The Map Reduce programming model was first developed for the Google search engine. The original Map Reduce system, called the Google File System (GFS), was designed to handle the massive amounts of data generated by the Google search engine.

In 2005, the Apache Hadoop project was founded to develop a more scalable and flexible version of the Map Reduce system. The first release of Hadoop, version 0.6, was released in 2006.

## Why Map Reduce?

Map Reduce provides several benefits, including:

- **Scalability**: Map Reduce can handle large data sets by distributing the processing across multiple machines.
- **Flexibility**: Map Reduce allows users to define their own processing functions, making it a flexible and customizable framework.
- **Fault tolerance**: Map Reduce provides built-in fault tolerance, allowing the system to recover from machine failures.

## Why not RDBMS?

Relational Database Management Systems (RDBMS) are designed for small to medium-sized data sets and are not well-suited for big data analytics. RDBMS require a pre-defined schema, which can be inflexible and limiting for large data sets. Additionally, RDBMS are not designed for parallel processing, making it difficult to scale to large data sets.

## RDBMS vs Hadoop

Hadoop is a distributed computing framework that stores and processes large data sets. Hadoop provides a scalable and flexible way to process data, making it an ideal choice for big data analytics. Here are some key differences between RDBMS and Hadoop:

- **Data storage**: RDBMS store data in a fixed schema, while Hadoop stores data in a schema-less format.
- **Scalability**: RDBMS are designed for small to medium-sized data sets, while Hadoop is designed to handle large data sets.
- **Parallel processing**: RDBMS are not designed for parallel processing, while Hadoop provides built-in support for parallel processing.

## Mapper

The Mapper is the first stage of the Map Reduce process. The Mapper breaks down the input data into smaller chunks, called keys and values. The Mapper applies a user-defined function to each chunk of data, producing a series of key-value pairs.

Here's an example of a simple Mapper that calculates the sum of a list of numbers:

```python
def mapper(key, value):
    # Calculate the sum of the numbers
    sum = 0
    for num in value:
        sum += num
    return (key, sum)
```

## Reducer

The Reducer is the second stage of the Map Reduce process. The Reducer receives the output from the Mapper and aggregates the results using a user-defined function.

Here's an example of a simple Reducer that calculates the average of a list of numbers:

```python
def reducer(key, values):
    # Calculate the sum of the numbers
    sum = 0
    for value in values:
        sum += value
    # Calculate the average
    average = sum / len(values)
    return (key, average)
```

## Partitioner

A Partitioner is used to divide the output from the Mapper into smaller chunks, called partitions. The Partitioner is used to optimize the performance of the Reducer.

Here's an example of a simple Partitioner that divides the output into 10 partitions:

```python
def partitioner(key, values):
    # Divide the output into 10 partitions
    partition = key % 10
    if partition == 0:
        partition = 10
```
