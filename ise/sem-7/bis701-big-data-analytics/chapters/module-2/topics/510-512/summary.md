# **Revision Notes: 5.10-5.12 - BIG DATA ANALYTICS**

### Introduction to Hadoop

- **Hadoop**: A distributed computing framework for processing large data sets.
- **Why Hadoop**: Handles massive data volumes, low cost, and flexible data processing.

### RDBMS vs Hadoop

- **RDBMS (Relational Database Management System)**:
  - Structured data storage
  - Query optimization
  - ACID compliance
- **Hadoop**:
  - Unstructured or semi-structured data processing
  - Decentralized data storage
  - No ACID compliance

### History of Hadoop

- **2005**: Doug Cutting and Mike Cafarella develop Nutch, a web crawler that uses MapReduce.
- **2006**: Google patents MapReduce.
- **2009**: Hadoop 0.9.0 is released by the Apache Software Foundation.

### Important Formulas and Definitions

- **MapReduce**: A programming model for processing large data sets.
  - Map: Distributes data across nodes
  - Reduce: Combines output from Map
- **Hadoop Distributed File System (HDFS)**: A scalable, fault-tolerant storage system.
- **YARN (Yet Another Resource Negotiator)**: A resource management layer for Hadoop.

### Important Theorems

- **Shannon's Source Coding Theorem**: Establishes the limit on compression ratio for a lossless source code.
- **Hadamard's Inequality**: Provides a lower bound for the mutual information between two random variables.
