# **BIG DATA ANALYTICS: REVISION NOTES - CHAPTER 10**

### 10.1: What is Hive?

- Hive is a data warehousing and SQL-like query language for Hadoop.
- It allows users to analyze and process large datasets using SQL-like queries.
- Hive is a way to simplify and standardize data analysis on top of Hadoop.

### 10.2: Hive Architecture

- Hive architecture consists of:
  - Hive Metastore: stores metadata about the data in Hadoop
  - Hive Query Compiler: compiles Hive queries into MapReduce jobs
  - Hive Execution Engine: executes the compiled MapReduce jobs
  - Hive Manager: manages the Hive environment

### 10.3: Hive Data Types

- Hive supports the following data types:
  - Integer
  - Double
  - String
  - Boolean
  - Array
  - Map
  - Struct
  - Date
  - Timestamp

### 10.4: Hive File Formats

- Hive supports the following file formats:
  - Text

  - Avro

  - Parquet

### 10.5: Hive Query

- Hive queries use SQL-like syntax to retrieve data from Hadoop.
- Hive supports various query types:
  - SELECT
  - INSERT
  - UPDATE
  - DELETE
  - CREATE TABLE
  - DROP TABLE

### 10.6: Hive Query Language

- Hive query language is based on SQL and supports:
  - SELECT clauses
  - FROM clauses
  - WHERE clauses
  - GROUP BY clauses
  - HAVING clauses

### 10.7: Hive Query Optimization

- Hive optimizes queries using:
  - Pushdown optimization
  - Join optimization
  - Filtering optimization

### 10.8: Hive Query Execution

- Hive executes queries using:
  - MapReduce jobs
  - Pig
  - Impala
