# Parallel Computing Revision Notes

### Definitions

- **Parallel Computing**: The use of multiple processing units or cores to solve a problem simultaneously, increasing processing speed and efficiency.
- **Parallel Algorithm**: A algorithm that can be parallelized, meaning it can be executed by multiple processors or cores simultaneously.

### Important Formulas

- **Amdahl's Law**: Limits the maximum theoretical speedup that can be achieved by parallel processing, based on the fraction of the program that cannot be parallelized.
- **Malmquist's Interpolation Formula**: Used to calculate the maximum theoretical speedup of parallel processing.

### Theorems

- **Master-Theorem**: A mathematical framework for solving recurrence relations that describe the complexity of algorithms.
- **Amdahl's Theorem**: States that the maximum theoretical speedup of parallel processing is limited by the fraction of the program that cannot be parallelized.

### Key Points

- **Types of Parallelism**:
  - **Data Parallelism**: Executing the same operation on different data.
  - **Task Parallelism**: Executing different tasks concurrently.
  - **Pipelining**: Breaking down a process into a series of stages, each executed concurrently.
- **Parallel Computational Models**:
  - **Distributed Computing**: Using multiple processors or nodes to solve a problem.
  - **Multithreading**: Using multiple threads within a single processor to solve a problem.
  - **GPU Computing**: Using Graphics Processing Units (GPUs) to solve computational problems.
- **Parallel Algorithms**:
  - **Divide and Conquer**: Breaking down a problem into smaller sub-problems and solving them concurrently.
  - **Dynamic Programming**: Breaking down a problem into smaller sub-problems and solving them recursively.

### Important Concepts

- **Synchronization**: Coordinating the execution of multiple processes or threads to avoid conflicts.
- **Communication**: Exchanging data between processes or threads to coordinate their execution.
- **Load Balancing**: Distributing the workload evenly among multiple processors or nodes to avoid bottlenecks.

This summary provides a concise overview of key concepts, formulas, and theorems in parallel computing, covering definitions, important formulas, theorems, and key points, as well as important concepts.
