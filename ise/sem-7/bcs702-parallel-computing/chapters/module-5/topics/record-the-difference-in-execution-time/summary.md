# **Record the difference in execution time**

**Definition:**

- Execution time: The time taken by a program to complete its execution.
- Parallel computing: A technique of solving a problem by using multiple processors or cores to speed up the execution time.

**Key Points:**

- **Parallel Computing**: Divide a task into smaller sub-tasks and execute them concurrently using multiple processors or cores.
- **Synchronization**: Ensure that all threads or processes complete their tasks before proceeding to the next step.
- **Global Synchronization**: Synchronize all threads or processes using a global lock or barrier.

**Important Formulas:**

- **Amdahl's Law**: A theoretical limit on the speedup that can be achieved by parallel processing. (1/f(1-p) + p)
- **Gotoh's Law**: A theoretical limit on the speedup that can be achieved by parallel processing for certain types of problems. (1 + p)

**Theorems:**

- **Master Theorem**: A mathematical framework for solving recurrence relations that arise in algorithms.
- **Amdahl's Lower Bound**: A theoretical limit on the speedup that can be achieved by parallel processing.

**Key Concepts:**

- **Parallelization**: Breaking down a task into smaller sub-tasks that can be executed concurrently.
- **Synchronization**: Ensuring that all threads or processes complete their tasks before proceeding to the next step.
- **Load Balancing**: Distributing the workload evenly among multiple processors or cores.

**Revision Tips:**

- Understand the key concepts of parallel computing and synchronization.
- Familiarize yourself with Amdahl's Law and Gotoh's Law.
- Practice solving problems that involve parallelization and synchronization.
