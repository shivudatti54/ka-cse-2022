# Learning Objectives

After studying this topic, you should be able to:

1.  Define decompilation and distinguish it from related techniques like disassembly.
2.  Describe the key stages of the decompilation process at a high level.
3.  Explain the major challenges and limitations that prevent perfect decompilation.
4.  Compare and contrast the decompilation features of primary tools like Ghidra and IDA Pro.
5.  Identify common code obfuscation techniques (e.g., control flow flattening) in decompiler output.
6.  Apply knowledge of decompilation to analyze a sample of pseudo-C code and infer program functionality.
