# Learning Objectives

After studying this topic, you should be able to:

1.  Define log parsing and log normalization and distinguish between the two processes.
2.  Explain the critical importance of parsing and normalization for effective SIEM correlation and security analysis.
3.  Identify common techniques used for parsing (e.g., Regex, Grok) and normalization (e.g., field mapping, time standardization).
4.  Recognize examples of common normalized field names in a SIEM (e.g., `src_ip`, `dest_ip`, `user`, `timestamp`).
5.  Describe the challenge of multi-line logs and the need for timezone normalization to UTC.
6.  Analyze a sample raw log entry and identify the key fields that would be extracted during parsing.
