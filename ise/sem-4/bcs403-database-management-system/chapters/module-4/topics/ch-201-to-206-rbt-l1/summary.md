### Key Concepts

---

### Chapter 20.1: Introduction to Database Management Systems

- Definition: A database management system (DBMS) is a software application that allows users to define, create, maintain, and manipulate databases.
- DBMS functions:
  - Data definition: creating and modifying database schema
  - Data control: controlling access to data
  - Data processing: performing queries and transactions
  - Data integrity: ensuring data consistency and accuracy

### Chapter 20.2: Types of Databases

- Relational databases:
  - Structured and hierarchical
  - Examples: MySQL, PostgreSQL
- Non-relational databases:
  - No structuring: key-value, document, graph
  - Examples: MongoDB, Cassandra
- Time-series databases:
  - Optimized for high-volume, high-velocity data
  - Examples: InfluxDB, TimescaleDB

### Chapter 20.3: Database Design

- Entity-relationship modeling:
  - Entities and attributes
  - Relationships between entities
  - Examples: ER diagrams, data flow diagrams
- Normalization:
  - Reducing data redundancy and improving data integrity
  - Examples: 1NF, 2NF, 3NF

### Chapter 20.4: Data Modeling

- Object-relational mapping (ORM):
  - Mapping objects to relational tables
  - Examples: Hibernate, Entity Framework
- Data warehousing:
  - Storing historical data for reporting and analysis
  - Examples: Star schema, snowflake schema

### Chapter 20.5: Database Security

- Authentication and authorization:
  - Verifying user identity and access rights
  - Examples: username/password, token-based authentication
- Encryption and access control:
  - Protecting data from unauthorized access
  - Examples: symmetric and asymmetric encryption, access control lists

### Chapter 20.6: Database Performance Optimization

- Indexing:
  - Improving query performance by indexing columns
  - Examples: B-tree index, hash index
- Caching:
  - Storing frequently accessed data in memory
  - Examples: database cache, application cache
