# **DATABASE MANAGEMENT SYSTEM Revision Notes**

**Topic: Textbook 1, Ch 7.1 to 7.3**

### Definitions

- **Database**: A collection of organized data, using a common language, and managed using various techniques and tools.
- **Database Management System (DBMS)**: A software system that manages and controls access to a database.

### Key Concepts

- **Entity-Relationship Model (ERM)**:
  - Entity: A real-world entity or concept.
  - Relationship: A connection between two or more entities.
- **Entity Classes**: A set of entities with similar characteristics.
- **Attributes**: Characteristics or features of an entity.
- **Primary Key**: A unique identifier for an entity.
- **Foreign Key**: A reference to the primary key of another entity.

### Important Formulas and Theorems

- **Data Independence**: The ability to modify the database without affecting the application.
- **Entity Integrity**: The database ensures that each entity has a unique primary key.
- **Referential Integrity**: The database ensures that foreign keys reference existing primary keys.

### Key Points

- **Database Design Principles**:
  - First Normal Form (1NF)
  - Second Normal Form (2NF)
  - Third Normal Form (3NF)
- **Data Types**:
  - Atomic: A single value or field.
  - Compound: A combination of atomic values or fields.
- **Database Normalization**:
  - Removing redundancy and improving data integrity.

### Theorems

- **Closure Property**: A set is closed under a specific operation (e.g., union, intersection).
- **Distributive Law**: A set operation can be distributed over a set of elements.

This summary provides key concepts, definitions, formulas, and theorems related to database management systems, covering chapters 7.1 to 7.3. It is designed to serve as a quick revision guide before exams.
