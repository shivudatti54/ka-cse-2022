### Employee Table

- Table name: EMPLOYEE
- Attributes:
  - EMPNO (Employee Number): Unique identifier for each employee
  - ENAME (Employee Name)
  - JOB (Job Title)
  - DEPT (Department)
  - SAL (Salary)
  - HIREDATE (Date of Hire)

### EMPNO

- Definition: EMPNO is a unique identifier assigned to each employee.
- Formula: EMPNO = (DEPT No. x 100) + EMPLOYEE No.
- Theorem: EMPNO is an example of a composite key, as it is composed of two attributes (DEPT No. and EMPLOYEE No.).

### Definitions and Formulas

- **Composite Key**: A composite key is a combination of two or more attributes that uniquely identify each record in a table.
- **Primary Key**: A primary key is a unique identifier for each record in a table, used to ensure data integrity.
- **Unique Identifier**: A unique identifier is a value that uniquely identifies each record in a table.

### Theorems

- **Data Integrity**: Theorem stating that the data in a database is consistent and accurate.
- **Consistency**: Theorem stating that the data in a database remains consistent even after changes are made.
