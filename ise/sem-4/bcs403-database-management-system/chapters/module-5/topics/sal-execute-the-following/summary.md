# **DATABASE MANAGEMENT SYSTEM (DMS) REVISION NOTES**

## **I. Introduction**

- A database management system (DMS) is a software system that allows for the creation, modification, and manipulation of data in a database.
- DMS provides a framework for organizing, storing, and retrieving data.

## **II. Entity-Relationship Model (ERM)**

- A conceptual model that represents the structure and relationships between entities in a database.
- Entities: real-world objects or concepts that have attributes or properties.
- Attributes: characteristics of an entity, such as name, age, or address.
- Relationships: connections between entities, such as one-to-one, one-to-many, or many-to-many.

## **III. Database Design**

- Database design: the process of creating a database schema and database structure.
- Database schema: a visual representation of the database structure.
- Database structure: the organization of data in the database.

## **IV. Database Management System (DBMS)**

- A DBMS provides a set of tools for managing and querying a database.
- Common features of a DBMS include:
  - Data definition language (DDL): used to create, modify, and drop database objects.
  - Data manipulation language (DML): used to insert, update, and delete data.
  - Query language (SQL): used to retrieve and manipulate data.

## **V. Database Normalization**

- Database normalization: the process of organizing data in a database to minimize data redundancy and improve data integrity.
- Normalization rules:
  - First normal form (1NF): each cell in the table must contain a single value.
  - Second normal form (2NF): each non-key attribute must depend on the entire primary key.
  - Third normal form (3NF): if a table is in 2NF, and a non-key attribute depends on another non-key attribute, then it should be moved to a separate table.

## **VI. Database Modeling**

- Database modeling: the process of creating a logical and physical model of a database.
- Common database modeling techniques include:
  - Entity-relationship diagrams (ERDs)
  - Object-oriented modeling (OOM)
  - Relational modeling

## **VII. Database Security**

- Database security: the practices and technologies used to protect a database from unauthorized access.
- Common database security measures include:
  - Authentication: verifying the identity of users.
  - Authorization: controlling access to data and functions.
  - Encryption: protecting data from unauthorized access.

## **VIII. Database Performance**

- Database performance: the measure of how efficiently a database can respond to queries and requests.
- Common factors that affect database performance include:
  - Indexing
  - Caching
  - Query optimization
