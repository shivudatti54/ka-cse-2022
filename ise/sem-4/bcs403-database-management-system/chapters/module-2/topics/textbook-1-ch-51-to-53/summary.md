### Database Management System Revision Notes

#### Textbook 1: Ch 5.1 to 5.3

**Definition and Purpose**

- A database management system (DBMS) is a software system that allows users to define, create, maintain, and manipulate databases.
- DBMS provides a standard interface for accessing, modifying, and managing data in a database.

**Database Types**

- Relational databases: Organize data into tables with well-defined relationships.
- Non-relational databases: Store data in a variety of formats, such as XML, JSON, or key-value pairs.

**Database Management Techniques**

- Normalization: Organize data to minimize data redundancy and dependency.
- Denormalization: Organize data to improve query performance.

**Database Design**

- Entity-Relationship Diagram (ERD): Visual representation of database entities and relationships.
- Entity-Attribute-Value (EAV) model: Storing data as attributes of entities.

**Important Formulas and Theorems**

- **Data Independence Theorem**: A database system is data independent if it can be modified without affecting the physical storage devices.

**Important Definitions**

- **Entity**: A person, place, or thing in a database.
- **Attribute**: A characteristic or feature of an entity.
- **Relationship**: A connection between two or more entities.
- **Primary Key**: A unique identifier for an entity.

**Key Terminology**

- **Primary key**: A unique identifier for an entity.
- **Foreign key**: A field in one table that references the primary key of another table.
- **Join**: Combining two or more tables based on a common attribute.
- **Subquery**: A query nested inside another query.
