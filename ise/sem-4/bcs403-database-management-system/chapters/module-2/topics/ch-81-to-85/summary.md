# **DATABASE MANAGEMENT SYSTEM (Ch 8.1 to 8.5) Revision Notes**

### Introduction

- Definition: A database management system (DBMS) is a software application that allows users to define, create, maintain, and manipulate databases.
- Purpose: To manage data efficiently and effectively.

### Types of DBMS

- **Relational DBMS (RDBMS)**: Organizes data into tables with well-defined relationships between them.
- **Object-Oriented DBMS**: Organizes data into objects that contain both data and methods to manipulate the data.
- **Hierarchical DBMS**: Organizes data into a tree-like structure.
- **Network DBMS**: Organizes data into a network of interconnected records.

### Characteristics of DBMS

- **Autonomy**: Ability to operate independently.
- **Integration**: Ability to integrate with other applications.
- **Support**: Provides support for various data types.
- **Security**: Ensures data security and access control.
- **Scalability**: Ability to handle large amounts of data.

### SQL (Structured Query Language)

- **Definition**: A standard language for managing relational databases.
- **Commands**: SELECT, INSERT, UPDATE, DELETE, CREATE, DROP
- **Functions**: SUM, AVG, COUNT, MIN, MAX

### Database Normalization

- **Definition**: The process of organizing data into a structured format to minimize data redundancy.
- **Rules**: First Normal Form (1NF), Second Normal Form (2NF), Third Normal Form (3NF)

### Data Model

- **Entity-Relationship Model**: A model that describes the relationships between entities in a database.
- **Conceptual Model**: A high-level model that describes the overall structure of the database.

### Important Formulas and Theorems

- **Data Integrity**: The concept of ensuring data accuracy and consistency.
- **Data Consistency**: The concept of ensuring that data remains consistent across the database.
