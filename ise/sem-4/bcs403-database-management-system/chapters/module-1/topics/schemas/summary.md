# Schemas

================

### Definition

A schema is a blueprint or a conceptual structure that defines the organization and relationships of data in a database.

### Key Points

- **Schema Design**: A schema defines the structure of the data, including the relationships between different entities.
- **Entity-Relationship Model**: A schema is represented using an entity-relationship model, which describes the relationships between entities.
- **Database Schema**: A database schema is a set of rules that define the structure and organization of data in a database.
- **Types of Schemas**: There are two main types of schemas:
  - **Physical Schema**: The physical structure of the database, including the storage locations and data types.
  - **Logical Schema**: The conceptual structure of the data, including the relationships between entities.

### Important Formulas and Theorems

- **First Normal Form (1NF)**: Each row in the table must contain a unique combination of values.
- **Second Normal Form (2NF)**: Each non-key attribute in the table must depend on the entire primary key.
- **Third Normal Form (3NF)**: If a table is in 2NF, and a non-key attribute depends on another non-key attribute, then it should be moved to a separate table.

### Other Key Concepts

- **Data Independence**: The ability to modify the schema of a database without affecting the application that uses it.
- **Schema Evolution**: The process of modifying a schema without disrupting the application that uses it.

### Important Definitions

- **Schema**: A blueprint or conceptual structure that defines the organization and relationships of data in a database.
- **Entity**: A thing or concept that has attributes and relationships.
- **Attribute**: A characteristic or feature of an entity.
- **Relationship**: A connection between entities.
