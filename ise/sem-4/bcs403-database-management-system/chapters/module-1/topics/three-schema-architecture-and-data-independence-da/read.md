# **Three Schema Architecture and Data Independence, Database Languages, and Interfaces, The Database System Environment**

## **Overview**

In this module, we will explore the three-schema architecture, data independence, database languages, and interfaces, and the database system environment. We will delve into the concepts, definitions, and examples to understand how these components work together to create a robust and efficient database system.

## **Three Schema Architecture**

The three-schema architecture is a conceptual model that separates the database into three distinct schemas: the external schema, the application schema, and the data schema.

### External Schema

The external schema is the user-facing interface of the database. It is the schema that users interact with, and it defines the structure of the database in terms of tables, columns, and relationships.

### Application Schema

The application schema is the schema that is used by the application to interact with the database. It is typically defined by the application developer and is specific to the application's requirements.

### Data Schema

The data schema is the underlying schema of the database, which defines the structure of the data in the database. It is the most fundamental schema and is common to all applications that use the database.

## **Data Independence**

Data independence is the ability of a database system to be independent of the underlying data storage or retrieval mechanism. It allows the database system to be modified or replaced without affecting the applications that use it.

### Types of Data Independence

There are two types of data independence:

- **Physical Data Independence**: This refers to the ability of the database system to be independent of the physical storage or retrieval mechanism.
- **Logical Data Independence**: This refers to the ability of the database system to be independent of the logical structure of the data.

## **Database Languages and Interfaces**

Database languages and interfaces are used to interact with the database system. There are several types of database languages and interfaces, including:

### Database Languages

There are several types of database languages, including:

- **SQL (Structured Query Language)**: A standard language for managing relational databases.
- **Procedural Languages**: Languages that allow the user to define procedures to perform tasks.
- **Object-Oriented Languages**: Languages that allow the user to define objects to perform tasks.

### Database Interfaces

There are several types of database interfaces, including:

- **Application Programming Interfaces (APIs)**: Interfaces that allow the application to interact with the database.
- **Database Management Interfaces**: Interfaces that allow the database administrator to interact with the database.
- **User Interfaces**: Interfaces that allow the end-user to interact with the database.

## **Database System Environment**

The database system environment refers to the overall context in which the database system operates. It includes the hardware, software, and network components that make up the database system.

### Hardware Components

The hardware components of the database system include:

- **Database Server**: The computer that hosts the database.
- **Database Client**: The computer that interacts with the database.

### Software Components

The software components of the database system include:

- **Database Management System (DBMS)**: The software that manages the database.
- **Database Language**: The language used to interact with the database.

### Network Components

The network components of the database system include:

- **Network Interface Card (NIC)**: The hardware component that allows the database to communicate with other devices.
- **Network Protocol**: The protocol used to communicate between devices.

### Key Concepts

---

Here are some key concepts to remember:

- **Three-schema architecture**: A conceptual model that separates the database into three distinct schemas: the external schema, the application schema, and the data schema.
- **Data independence**: The ability of a database system to be independent of the underlying data storage or retrieval mechanism.
- **Database languages**: A set of commands used to interact with the database, including SQL, procedural languages, and object-oriented languages.
- **Database interfaces**: A set of interfaces that allow the application to interact with the database, including APIs, database management interfaces, and user interfaces.
