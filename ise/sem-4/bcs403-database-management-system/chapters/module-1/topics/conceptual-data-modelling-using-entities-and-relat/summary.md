# **Conceptual Data Modelling using Entities and Relationships**

## **Entity Types and Entity Sets**

- **Entity Type**: A conceptual entity that has an identity and can exist independently.
- **Entity Set**: A collection of instances of an entity type.
- **Key Attribute**: A single attribute that uniquely identifies an entity instance.

## **Structural Constraints**

- **Composite Attribute**: An attribute that consists of multiple attributes.
- **Derived Attribute**: An attribute that is derived from one or more other attributes.
- **Inconsistent Data**: Data that violates the structural constraints of the entity.

## **Weak Entity Types**

- **Composite Attribute**: An attribute that consists of multiple attributes.
- **Partial Key**: A composite attribute that does not uniquely identify an entity instance.
- **Transitive Property**: A property that holds between entities that are related through a weak entity.

## **ER Diagrams**

- **Entity**: Represented as a box with a name and attributes.
- **Relationship**: Represented as a line connecting two entities.
- **Attribute**: Represented as an arrow within an entity box.

## **Formulas and Definitions**

- **Entity-Relationship Model (ERM)**: A conceptual model that represents entities and relationships.
- **Data Flow Diagram**: A diagram that shows the flow of data through an information system.

## **Important Theorems**

- **Entity Integrity Theorem**: States that each entity must have a unique identifier.
- **Functional Dependency Theorem**: States that if certain attributes determine another attribute, then the first set of attributes determines the second set.

## **Revision Tips**

- Understand the difference between entity types and entity sets.
- Be able to identify key attributes and structural constraints.
- Learn to create ER diagrams and understand the concepts of weak entity types and transitive properties.
