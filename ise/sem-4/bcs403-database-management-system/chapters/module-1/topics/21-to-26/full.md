In this module, we will delve into the world of DBMS, focusing on the fundamentals of database design, data modeling, normalization, and database administration. This module will cover topics 2.1 to 2.6, providing a comprehensive understanding of the subject.

## **2.1: Introduction to Database Management Systems**

A DBMS is a software system that provides a way to store, organize, and retrieve data in a database. It acts as an intermediary between the user and the database, providing a layer of abstraction and making it easier to manage and maintain the data.

### Characteristics of a DBMS

- Provides a way to store and manage data
- Supports various data types, such as text, numbers, and dates
- Provides data integrity and consistency
- Supports data security and access control
- Supports data backup and recovery

### Types of DBMS

- Relational DBMS (RDBMS): Organizes data into tables with defined relationships
- Object-Oriented DBMS (OODB): Organizes data into objects that represent real-world entities
- Hierarchical DBMS: Organizes data into a hierarchical structure
- Network DBMS: Organizes data into a network of interconnected tables

### Advantages of DBMS

- Improved data management and retrieval
- Enhanced data security and access control
- Increased data consistency and integrity
- Supports scalability and flexibility

## **2.2: Data Modeling**

Data modeling is the process of defining the structure and relationships of data in a database. It involves creating a conceptual model of the database, which is then translated into a logical model and finally into a physical model.

### Data Modeling Techniques

- Entity-Relationship Modeling (ERM)
- Object-Role Modeling (ORM)
- Data Flow Diagrams (DFDs)

### Data Modeling Tools

- Entity-Relationship Diagrams (ERDs)
- UML (Unified Modeling Language)
- Data Modeling Languages (DMLs)

### Case Study: Data Modeling for a University Database

Suppose we want to create a database for a university. We can use data modeling techniques to define the structure and relationships of the data. We can create an ERD that shows the relationships between different entities, such as students, courses, and grades.

## **2.3: Normalization**

Normalization is the process of organizing data in a database to minimize data redundancy and dependency. It involves dividing large tables into smaller tables and linking them through relationships.

### Normalization Rules

- First Normal Form (1NF): Each table cell must contain a single value
- Second Normal Form (2NF): Each table must be in 1NF, and each non-key attribute must depend on the entire primary key
- Third Normal Form (3NF): Each non-key attribute must depend on the entire primary key

### Normalization Techniques

- Horizontal Normalization
- Vertical Normalization

### Case Study: Normalization for a Sales Database

Suppose we want to create a database for a company that sells products online. We can use normalization to organize the data in a way that minimizes redundancy and dependency. We can create a 3NF table that shows the relationships between products, customers, and orders.

### Database Administration Tools

- SQL (Structured Query Language)
- DBMS (Database Management System)
- DBA (Database Administrator)

### Case Study: Database Administration for a Banking Website

Suppose we want to create a database for a banking website. We can use database administration tasks to manage and maintain the database. We can create a schema that includes tables for customers, accounts, and transactions.
