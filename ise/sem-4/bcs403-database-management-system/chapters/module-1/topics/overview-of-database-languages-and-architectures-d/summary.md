# Database Management System

### Overview of Database Languages and Architectures: Data Models

#### Key Concepts

- **Data Models**: A representation of the database's structure and organization.
- **Entity-Relationship Model (ERM)**: A data model that describes relationships between entities.
- **Object-Relational Mapping (ORM)**: A technique to map objects to relational databases.
- **Data Warehousing**: A type of database designed for reporting and analytics.

#### Data Models

- **Entity Model**: A data model that focuses on entities and their attributes.
- **Class Model**: A data model that uses object-oriented concepts.
- **Extended Entity Model**: A data model that extends the entity model with additional information.
- **Data Flow Diagrams**: A graphical representation of data flows and processing steps.

#### Notations and Symbols

- **Entity Set**: A set of entities in the database.
- **Relationship Set**: A set of relationships between entities.
- **Attribute**: A characteristic of an entity.
- **Primary Key**: A unique identifier for an entity.
- **Foreign Key**: A reference to a primary key in another entity.

#### Important Formulas and Theorems

- **First Normal Form (1NF)**: A data model that eliminates repeating groups.
- **Boyce-Codd Normal Form (BCNF)**: A data model that minimizes transitive dependencies.
- **Third Normal Form (3NF)**: A data model that eliminates transitive dependencies.
- **Data Integrity Theorem**: A theorem that ensures data consistency.

#### Important Definitions

- **Database Language**: A programming language used to interact with a database.
- **Database Architecture**: The design and organization of a database.
- **Data Model**: A representation of the database's structure and organization.
- **Entity**: A real-world object or concept in the database.
- **Relationship**: A connection between two or more entities.

#### Key Architectures

- **Relational Database Architecture**: A database that uses tables and relationships.
- **Object-Oriented Database Architecture**: A database that uses objects and classes.
- **Graph Database Architecture**: A database that uses graph theory to store data.
