# **Database Management System - 3.1 to 3.10 RBT: L1 Revision Notes**

### Overview

- Database Management System (DBMS) is a software system that helps in managing and storing data in a database.
- DBMS provides a way to organize, control, and retrieve data using various techniques and tools.

### Key Concepts

- **Definition of DBMS**: A software system that allows data to be stored, modified, and retrieved in an organized and efficient manner.
- **Characteristics of DBMS**:
  +Autonomy
  +Integrated
  +Controlled access
  +Support for transfer of data (data independence)
  +Support for data definition (data abstraction)
- **Types of DBMS**:
  +Relational DBMS
  +Object-Oriented DBMS
  +Hierarchical DBMS
  +Network DBMS

### Relational Database Management System (RDBMS)

- **Definition**: A type of DBMS that uses a relational model to store data in tables.
- **Key Features**:
  +Tables
  +Relations
  +Primary keys
  +Foreign keys
  +Views
  +Stored procedures
- **RDBMS Formulas**:
  +Normal form (NF): A relation is in NF if it is in NF1, NF2, and NF3.
  +Denormalization: A relation is denormalized if it is not in NF1, NF2, or NF3.

### Important Formulas, Definitions, and Theorems

- **First Normal Form (1NF)**: A table is in 1NF if each cell contains a single value.
- **Second Normal Form (2NF)**: A table is in 2NF if it is in 1NF and all non-key attributes depend on the entire primary key.
- **Third Normal Form (3NF)**: A table is in 3NF if it is in 2NF and there are no transitive dependencies.
- **Boyce-Codd Normal Form (BCNF)**: A table is in BCNF if it is in 3NF and there are no transitive dependencies.
- **Data Independence**: The ability to change the physical storage of data without affecting the application that uses it.

### Important Terms

- **Entity-Relationship Model**: A graphical representation of the relationships between entities in a database.
- **Entity**: A real-world object or concept that has a unique identifier.
- **Relationship**: A connection between two or more entities.
