**Module:** No. of Hours: 8
**Topic:** Textbook 1: Ch 1.1 to 1.8

### Definitions

- **Database**: A collection of organized data stored in a way that allows for efficient retrieval and manipulation.
- **Database Management System (DBMS)**: Software that manages and controls access to a database.
- **Entity**: A concept or object in a database that has its own set of attributes.
- **Relationship**: A connection between two or more entities in a database.

### Key Concepts

- **Database Entities**: Attributes,Primary Keys, Foreign Keys
- **Database Relationships**: One-to-one, One-to-many, Many-to-many
- **Database Normalization**: 1NF, 2NF, 3NF

### Important Formulas

- **Codd's 12 Rules**: Not applicable
- **Relational Model**: Not applicable
- **Database Normalization Rules**:
  - 1NF: Each cell must contain a single value
  - 2NF: Each non-key attribute must depend on the entire primary key
  - 3NF: If a table is in 2NF, and a non-key attribute depends on another non-key attribute, then it should be moved to a different table

### Theorems

- **Entity-Relationship Model (ERM)**: Not applicable
- **Database Design Theorem**: Not applicable

### Important Terms

- **Primary Key**: A unique identifier for each entity
- **Foreign Key**: A field that references the primary key of another entity
- **Normalization**: The process of organizing data in a database to minimize data redundancy and improve data integrity
