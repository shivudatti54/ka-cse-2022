# **Functions – Plain and One-to-One**

## **Introduction**

In the realm of discrete mathematical structures, functions play a crucial role in describing relationships between sets of objects. In this section, we will delve into the world of functions, exploring the concepts of plain functions and one-to-one functions, and examining their properties, examples, and applications.

## **What is a Function?**

A function is a relation between a set of inputs, called the domain, and a set of possible outputs, called the range. It assigns to each element in the domain exactly one element in the range. In other words, a function is a rule that takes an input and produces an output.

## **Plain Functions**

A plain function, also known as a regular function, is a function that assigns to each element in the domain exactly one element in the range. In other words, a plain function is a function where each input corresponds to exactly one output.

## **Notation**

Plain functions are often denoted using the following notation:

f: A → B

where A is the domain, B is the range, and f is the function.

## **Examples of Plain Functions**

1. **Addition**: f(x) = x + 1, where A = Z and B = Z
2. **Multiplication**: f(x) = x × 2, where A = Z and B = Z
3. **Exponential Function**: f(x) = 2^x, where A = Z and B = R

## **Properties of Plain Functions**

- **Domain**: The set of all possible inputs in the domain.
- **Range**: The set of all possible outputs in the range.
- **Codomain**: The set of all possible outputs in the codomain (i.e., the range of the domain).
- **Surjection**: A surjection is a function where every element in the codomain is the image of at least one element in the domain.
- **Injection**: An injection is a function where every element in the range is the image of at most one element in the domain.

## **One-to-One Functions**

A one-to-one function, also known as an injective function, is a function where every element in the range is the image of at most one element in the domain. In other words, a one-to-one function is a function where each output corresponds to at most one input.

## **Notation**

One-to-one functions are often denoted using the following notation:

f: A → B

where A is the domain, B is the range, and f is the function, and the notation "f is injective" or "f is one-to-one" is used to indicate that the function is one-to-one.

## **Examples of One-to-One Functions**

1. **Linear Function**: f(x) = 2x + 1, where A = Z and B = Z
2. **Square Root Function**: f(x) = √x, where A = Z and B = R
3. **Exponential Function**: f(x) = 2^x, where A = Z and B = R

## **Properties of One-to-One Functions**

- **Injectivity**: A one-to-one function is injective by definition.
- **Surjectivity**: A one-to-one function is not necessarily surjective. However, if a one-to-one function is also onto (i.e., surjective), it is called a bijection.
- **Injectivity**: A one-to-one function is injective if and only if it has a one-to-one correspondence between the domain and range.

## **Historical Context**

The concept of a function dates back to ancient civilizations, with the earliest recorded mention of a function being the Babylonian mathematical text "YBC 7289", which dates back to around 1800 BCE. The concept of a function was further developed by ancient Greek mathematicians such as Euclid and Archimedes.

In the modern era, the concept of a function was formalized by René Descartes, who introduced the notation f: A → B. The concept of one-to-one functions was also developed by mathematicians such as Leonhard Euler and Augustin-Louis Cauchy.

## **Modern Developments**

In modern mathematics, the concept of functions is used extensively in various fields, including algebra, analysis, and topology. The development of computer science and programming also relies heavily on the concept of functions, with functions being used to describe algorithms and data structures.

## **Applications of Functions**

Functions have numerous applications in various fields, including:

- **Computer Science**: Functions are used to describe algorithms and data structures, and are a fundamental concept in programming languages.
- **Finance**: Functions are used to model financial systems and predict stock prices.
- **Biology**: Functions are used to model population growth and disease spread.
- **Optimization**: Functions are used to optimize solutions to problems in fields such as logistics and supply chain management.

## **Case Studies**

1. **Linear Regression**: A linear regression model is a function that predicts the value of a continuous variable based on a linear combination of a set of independent variables.
2. **Logistic Regression**: A logistic regression model is a function that predicts the probability of a binary outcome based on a linear combination of a set of independent variables.
3. **Decision Trees**: A decision tree is a function that describes the decision-making process for a binary classification problem.
