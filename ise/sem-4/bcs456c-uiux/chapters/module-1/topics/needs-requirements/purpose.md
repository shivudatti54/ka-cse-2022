# Why Study Consensus Requirements?

## Design Principles Foundation

Understanding consensus requirements helps you **evaluate and compare** different blockchain systems. When choosing or designing a consensus mechanism, you need to know what properties are essential and what trade-offs are acceptable.

## Exam Relevance

| Aspect         | Details                                                |
| -------------- | ------------------------------------------------------ |
| Frequency      | Asked in 4 out of 6 blockchain exams                   |
| Marks          | 6-12 marks typically                                   |
| Question Types | List requirements, explain trade-offs, compare systems |

## Real-World Applications

1. **Protocol Selection** - Choosing the right blockchain for a use case
2. **Security Auditing** - Evaluating if a system meets requirements
3. **System Design** - Building new consensus mechanisms
4. **Investment Analysis** - Assessing blockchain project viability
5. **Regulatory Compliance** - Finality requirements for legal recognition

## Connection to Other Topics

Understanding requirements helps with:

- **PoW/PoS comparison** - Different requirement trade-offs
- **Finality** - Deep dive into safety guarantees
- **Scalability solutions** - Layer 2, sharding trade-offs
- **Attack analysis** - How attacks violate requirements

## Learning Outcomes

After studying this topic, you should be able to:

1. Define safety and liveness requirements
2. Explain the blockchain trilemma
3. Compare fault tolerance levels
4. Describe Sybil resistance mechanisms
5. Analyze trade-offs between requirements
