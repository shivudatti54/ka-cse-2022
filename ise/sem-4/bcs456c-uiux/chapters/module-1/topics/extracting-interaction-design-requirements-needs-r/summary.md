# **Extracting Interaction Design Requirements: Needs & Requirements**

## **Key Concepts**

- **Needs**: User needs are the fundamental requirements that drive the design process.
- **Requirements**: User requirements are the specific needs that are identified and documented.
- **Formula**: Needs → Requirements → Design

## **Defining Needs**

- **Definition**: Needs are the user's goals, behaviors, and motivations.
- **Types of Needs**:
  - Functional needs: basic functionality
  - Aesthetic needs: visual appeal
  - Emotional needs: emotional experience

## **Extracting User Requirements**

- **User Requirements Analysis (URA)**: a systematic approach to identify user needs and requirements.
- **Steps**:
  1.  Identify users and stakeholders
  2.  Analyze user needs and behaviors
  3.  Document user requirements
- **Notation**: User Requirements Specification (URS)

## **Formulas and Theorems**

- **Moore's Law**: The number of transistors on a microchip doubles approximately every two years, leading to exponential improvements in computing power and reductions in cost.
- **Heuristic Evaluation**: a method for evaluating user interfaces based on a set of established principles and guidelines.

## **Important Definitions**

- **Usability**: the degree to which a system is easy to use.
- **User Experience (UX)**: the overall experience a user has when interacting with a system.

## **Quick Revision Tips**

- Identify user needs and behaviors.
- Analyze user requirements using URA.
- Document user requirements in URS format.
- Apply heuristic evaluation to evaluate user interfaces.
