# Learning Purpose: Connections with Software Engineering

## 1. Why is this topic important?

This topic is critical because UI/UX design does not exist in a vacuum; it is a core component of the software development lifecycle. A design that is not feasible to build or is misaligned with engineering constraints is ultimately a failure. Understanding this connection ensures designs are practical, implementable, and contribute to a cohesive, high-quality final product.

## 2. What will students learn?

Students will learn how UX research and UI design workflows integrate with agile and waterfall development methodologies. They will explore concepts like design systems, component libraries, and developer handoff processes using tools like Figma. The goal is to foster effective collaboration, clear communication, and a shared language between design and engineering teams.

## 3. How does it connect to other concepts?

This module directly builds on prior knowledge of design principles (e.g., consistency, accessibility) and prototyping by grounding them in technical reality. It connects the "why" (user research from Module 2) and the "what" (wireframing from Module 3) to the "how" of software construction, ensuring designs are both user-centric and technically sound.

## 4. Real-world applications

This knowledge is applied daily when UX designers create developer-friendly annotations, participate in sprint planning, and use version-controlled design systems to ensure consistency across a scalable product. It bridges the gap between a pixel-perfect mockup and a functional, shippable feature, reducing costly rework and fostering a more efficient product team.
