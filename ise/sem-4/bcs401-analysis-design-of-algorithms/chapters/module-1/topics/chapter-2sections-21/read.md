## 2.1: What is an Algorithm?

---

An algorithm is a well-defined procedure that takes some input and produces a corresponding output. It is a set of instructions that are used to solve a specific problem or perform a particular task. Algorithms are the building blocks of computer programs, and they are used in a wide range of applications, from simple calculators to complex operating systems.

### Characteristics of an Algorithm

---

An algorithm typically has the following characteristics:

- **Well-defined**: An algorithm should be easy to understand and follow.
- **Unambiguous**: An algorithm should be clear and concise, with no ambiguity or confusion.
- **Efficient**: An algorithm should be designed to solve the problem in the most efficient way possible.
- **Reusable**: An algorithm should be able to be used in a variety of different contexts.
- **Testable**: An algorithm should be able to be tested and verified to ensure that it is working correctly.

### Types of Algorithms

---

There are several types of algorithms, including:

- **Deterministic algorithms**: These algorithms always produce the same output for the same input.
- **Non-deterministic algorithms**: These algorithms may produce different outputs for the same input.
- **Recursive algorithms**: These algorithms use recursion to solve a problem.
- **Iterative algorithms**: These algorithms use iteration to solve a problem.

### Example of an Algorithm

---

A simple algorithm for calculating the area of a rectangle is:

1.  Read the length and width of the rectangle from the user.
2.  Multiply the length and width together to get the area.
3.  Print out the area.

This algorithm is well-defined, unambiguous, efficient, reusable, and testable.

## Key Concepts

---

- **Input**: The data that an algorithm uses to solve a problem.
- **Output**: The result that an algorithm produces as a solution to a problem.
- **Pseudo-code**: A high-level representation of an algorithm that is easy to read and understand.
- **Flowchart**: A visual representation of an algorithm that shows the sequence of steps.
- **Time complexity**: The amount of time that an algorithm takes to run.
- **Space complexity**: The amount of memory that an algorithm uses.

### Time Complexity

---

Time complexity is a measure of how long an algorithm takes to run. It is usually expressed in terms of the number of operations that the algorithm performs. Common time complexities include:

- **O(1)**: Constant time complexity - the algorithm takes the same amount of time regardless of the input size.
- **O(log n)**: Logarithmic time complexity - the algorithm takes time proportional to the logarithm of the input size.
- **O(n)**: Linear time complexity - the algorithm takes time proportional to the input size.
- **O(n log n)**: Linearithmic time complexity - the algorithm takes time proportional to the product of the input size and its logarithm.
- **O(n^2)**: Quadratic time complexity - the algorithm takes time proportional to the square of the input size.

### Space Complexity

---

Space complexity is a measure of how much memory an algorithm uses. It is usually expressed in terms of the amount of memory that the algorithm uses as a function of the input size. Common space complexities include:

- **O(1)**: Constant space complexity - the algorithm uses a constant amount of memory regardless of the input size.
- **O(n)**: Linear space complexity - the algorithm uses a amount of memory proportional to the input size.
- **O(n log n)**: Linearithmic space complexity - the algorithm uses a amount of memory proportional to the product of the input size and its logarithm.
- **O(n^2)**: Quadratic space complexity - the algorithm uses a amount of memory proportional to the square of the input size.
