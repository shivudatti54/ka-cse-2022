### Introduction

---

- An algorithm is a well-defined procedure that takes some input and produces a corresponding output.
- It is a set of instructions that is used to solve a specific problem.

### Key Concepts

---

- **Algorithmic Problem Solving**:
  - Breaking down a problem into smaller sub-problems
  - Identifying the problem's constraints and requirements
  - Developing a solution using a combination of algorithms and techniques
- **Computational Complexity**:
  - Measures the amount of time or space required by an algorithm
  - Can be expressed using Big O notation

### Important Formulas and Definitions

---

- **Big O Notation**:
  - O(1) - constant time complexity
  - O(log n) - logarithmic time complexity
  - O(n) - linear time complexity
  - O(n log n) - linearithmic time complexity
  - O(n^2) - quadratic time complexity
  - O(2^n) - exponential time complexity
- **Time Complexity**:
  - Analyzes the amount of time an algorithm takes to complete
  - Can be expressed using Big O notation
- **Space Complexity**:
  - Analyzes the amount of memory an algorithm requires
  - Can be expressed using Big O notation

### Important Theorems

---

- **Greedy Algorithm Theorem**:
  - A greedy algorithm is optimal for a problem if and only if the optimal substructure property holds
- **Divide and Conquer Algorithm Theorem**:
  - A divide and conquer algorithm can be solved using a recursive approach

### Other Key Points

---

- **Algorithm Design Techniques**:
  - Divide and Conquer
  - Dynamic Programming
  - Greedy Algorithm
- **Common Algorithmic Problems**:
  - Sorting algorithms (e.g. Bubble Sort, Selection Sort)
  - Searching algorithms (e.g. Linear Search, Binary Search)
