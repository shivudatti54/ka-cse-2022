## Introduction

---

The concept of exhaustive search, also known as brute force search, is a simple yet effective algorithmic technique used to solve optimization problems. It involves examining every possible solution to a problem and selecting the best one. In this section, we will delve into the world of exhaustive search and explore its applications, particularly in the context of the Travelling Salesman Problem (TSP).

## Historical Context

---

The concept of exhaustive search dates back to the 19th century, when mathematicians such as Pierre-Simon Laplace and Leonhard Euler used brute force methods to solve complex problems. However, it wasn't until the mid-20th century that exhaustive search became a widely recognized algorithmic technique, particularly in the field of computer science.

## Exhaustive Search

---

Exhaustive search is a simple yet effective algorithmic technique that involves examining every possible solution to a problem. The basic steps involved in exhaustive search are:

1.  **Define the search space**: Identify the set of possible solutions to the problem.
2.  **Generate all possible solutions**: Create a list of all possible solutions to the problem.
3.  **Evaluate each solution**: Assess each solution based on a set of criteria or constraints.
4.  **Select the best solution**: Choose the solution that meets the specified criteria or constraints.

Exhaustive search is often used to solve problems that have a finite number of possible solutions.

## Travelling Salesman Problem (TSP)

---

The Travelling Salesman Problem (TSP) is a classic problem in computer science and operations research that involves finding the shortest possible tour that visits a set of cities and returns to the starting point. TSP is an NP-hard problem, which means that the number of possible solutions grows exponentially with the number of cities.

TSP is a prime example of a problem that can be solved using exhaustive search. The basic steps involved in solving TSP using exhaustive search are:

1.  **Define the search space**: Identify the set of possible tours that can be taken by the salesman.
2.  **Generate all possible tours**: Create a list of all possible tours that can be taken by the salesman.
3.  **Evaluate each tour**: Assess each tour based on the total distance traveled.
4.  **Select the best tour**: Choose the tour that has the shortest total distance.
