# **Chapter 11 (Section 11.2) Revision Notes**

- **Decision Trees**
  - Definition: A decision tree is a tree-like model of a decision-making process.
  - Components:
    - **Decision node**: A node that represents a decision.
    - **Leaf node**: A node that represents the outcome of the decision-making process.
    - **Path**: A sequence of nodes that represent the decision-making process.
  - **Example**: Identifying whether a person is eligible for a loan based on their credit score.

- **P, NP, and NP-Complete Problems**
  - **P**:
    - Definition: The set of decision problems that can be solved in polynomial time.
    - **Polynomial time**: A time complexity of O(n^k), where n is the input size and k is a constant.
  - **NP**:
    - Definition: The set of decision problems that can be verified in polynomial time.
    - **Polynomial time**: A time complexity of O(n^k), where n is the input size and k is a constant.
  - **NP-Complete Problems**:
    - Definition: A set of decision problems that are at least as hard as the hardest problems in NP.
    - **Example**: The traveling salesman problem, which is NP-complete.

- **Cook-Levin Theorem**
  - Definition: A theorem that states that P = NP if and only if every problem in NP can be solved in polynomial time.
  - **Important implication**: If P = NP, then every problem in NP has a polynomial-time algorithm.

- **Formulas and Definitions**
  - **Turing Machine**: A theoretical model for computation that consists of a tape divided into cells, each of which can hold a symbol.
  - **Time complexity**: A measure of the amount of time an algorithm takes to complete, typically expressed as a function of the input size.

- **Important Theorems**
  - **Cook-Levin Theorem**: A theorem that states that P = NP if and only if every problem in NP can be solved in polynomial time.
  - **Karp's Reduction** (Reduction by Reduction): A technique for reducing the running time of an algorithm by reducing the size of the input.
