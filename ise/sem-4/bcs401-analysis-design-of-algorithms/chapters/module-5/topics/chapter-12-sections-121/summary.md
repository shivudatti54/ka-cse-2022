## Introduction

- Limitations of algorithmic power
- Decision Trees
- P, NP, and NP-Complete Problems

## Definitions

- **Decision Tree**: A tree-like data structure where each internal node represents a test, each branch represents a possible outcome, and each leaf node represents a classification (0 or 1)
- **P (Polynomial Time)**: A class of decision problems that can be solved in polynomial time on a non-deterministic Turing machine
- **NP (Nondeterministic Polynomial Time)**: A class of decision problems that can be solved in polynomial time on a non-deterministic Turing machine, where a solution can be verified in polynomial time
- **NP-Complete**: A class of decision problems that are at least as hard as the hardest problems in NP

## Important Formulas and Theorems

- **Cook-Levin Algorithm**: A polynomial-time algorithm for solving SAT problems
- **Karp's Reduction**: A technique for reducing the complexity of a problem to a known NP-complete problem
- **P vs NP Problem**: The question of whether every problem in NP can be solved in polynomial time

## Key Points

- Decision Trees are a simple and efficient way to solve decision problems, but they can be limited by the number of branches
- P, NP, and NP-Complete problems are important classes in computational complexity theory
- NP-Complete problems are the hardest problems in NP, and if P=NP, every problem in NP can be solved in polynomial time
