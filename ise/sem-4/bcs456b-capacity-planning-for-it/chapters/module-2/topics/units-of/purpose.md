# Learning Purpose: Units of Measurement in Capacity Planning

**1. Why is this topic important?**
Understanding units of measurement is fundamental because they are the common language of IT capacity. Without standardized units, quantifying resources like storage, processing power, and memory is ambiguous, leading to costly errors in procurement, under-provisioning (causing performance issues), or over-provisioning (wasting capital).

**2. What will students learn?**
Students will learn the core units used to measure IT resource consumption and capacity, including bytes (KB, MB, GB, TB) for storage, hertz (GHz) and cores for processing (CPU), and bytes for memory (RAM). They will understand how to accurately interpret and apply these units to quantify current utilization and forecast future needs.

**3. How does it connect to other concepts?**
This knowledge directly connects to forecasting (Module 3), where units are used to model future demand, and to financial planning (Module 4), where they are translated into budgetary requirements. It provides the essential quantitative foundation upon which all subsequent capacity planning analysis is built.

**4. Real-world applications**
This is applied when specifying requirements for new server hardware, allocating virtual machine resources in a cloud environment (e.g., AWS, Azure), purchasing network-attached storage, or justifying IT budget requests to management with clear, quantifiable metrics.
