# **Ecosystem – Android Versions – Android Activity – Features of Android – Android Architecture Stack Linux Kernel**

## **Overview**

- The Android ecosystem refers to the collection of hardware, software, and content used to develop and run Android applications.
- The Open Handset Alliance (OHA) is a consortium of companies that develop and promote the Android operating system.

## **Android Versions**

- Android versions are major releases of the Android operating system, e.g. Android 1.0, Android 2.0, etc.
- Key versions:
  - Android 1.0 (2008)
  - Android 2.0 (Eclair, 2009)
  - Android 3.0 (Honeycomb, 2011)
  - Android 4.0 (Ice Cream Sandwich, 2011)
  - Android 5.0 (Lollipop, 2014)
  - Android 6.0 (Marshmallow, 2015)
  - Android 7.0 (Nougat, 2016)
  - Android 8.0 (Oreo, 2017)
  - Android 9.0 (Pie, 2018)
  - Android 10 (Q, 2019)
  - Android 11 (2020)
  - Android 12 (2021)
  - Android 13 (2022)

## **Android Activity**

- An Android activity is a single screen or view in an Android application.
- Key features:
  - Lifecycle methods (e.g. onCreate, onResume, onPause)
  - Intent system for inter-process communication
  - Layout inflation and resource customization

## **Features of Android**

- Key features:
  - Touchscreen interface
  - Multi-touch support
  - GPS and sensor integration
  - Virtual keyboard and voice recognition
  - Expandable storage and file system
  - Android Pay and other payment systems

## **Android Architecture Stack**

- The Android architecture stack consists of:
  - Android Runtime (ART)
  - Dalvik Virtual Machine (DVVM)
  - Java Virtual Machine (JVM)
  - Android NDK for native code development
  - Android SDK for software development

## **Linux Kernel**

- Android is built on top of the Linux kernel.
- Key features:
  - Process management and scheduling
  - Memory management and virtualization
  - File system and storage management
  - Network and device management

## **Important Formulas and Definitions**

- **Intent**: A message that requests an action be performed on an app component.
- **Activity**: A single screen or view in an Android application.
- **Service**: A component that runs in the background and provides a service to other components.
- **BroadcastReceiver**: A component that listens for broadcast messages.
- **Thread**: A separate flow of execution within a process.

Note: This summary is not exhaustive, but it covers the key points for quick revision.
