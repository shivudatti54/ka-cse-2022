# **Mobile Application Development: Revision Notes (Chapters 3 & 4)**

### Overview

- Chapters 3 & 4 cover the creation of the first Android application, focusing on directory structure and Android User Interface (UI).

### Key Concepts

- **Directory Structure**
  - Project structure:
    - `app/` (main directory)
    - `src/` (source code directory)
    - `res/` (resource directory)
  - Folder organization:
    - `java/` (Java source files)
    - `res/layout/` (XML layout files)
    - `res/values/` (XML values files)
- **Android User Interface (UI)**
  - UI components:
    - `Activity`
    - `Fragment`
    - `View`
  - Layout files:
    - XML layout files (`res/layout/`)
  - Thematic design:
    - Material Design
    - Theme.xml

### Important Formulas, Definitions, and Theorems

- **Activity**: A single screen or window in an Android app.
- **Fragment**: A self-contained piece of UI.
- **View**: A UI component that can be displayed on the screen.
- **XML**: Extensible Markup Language, used for layout files.

### Key Takeaways

- Create a clean and organized project structure.
- Understand the different UI components and layout files.
- Use Material Design principles for a visually appealing UI.

### Revision Tips

- Review the directory structure and project organization.
- Practice creating UI components and layout files.
- Familiarize yourself with Material Design principles.
