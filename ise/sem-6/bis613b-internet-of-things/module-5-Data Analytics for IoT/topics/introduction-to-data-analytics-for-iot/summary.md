# Introduction to Data Analytics for IoT

=====================================

### Overview

Data analytics for IoT is the process of examining, processing, and extracting valuable insights from the massive volumes of data generated by connected devices. It encompasses specialized techniques to handle continuous data streams, real-time processing, diverse formats, and distributed sources.

### Key Points

- **Four Types of Analytics:** Descriptive (what happened), Diagnostic (why it happened), Predictive (what will happen), and Prescriptive (what should be done)
- **Three V's of IoT Data:** Volume (petabytes from sensors), Velocity (high-speed continuous streams), and Variety (structured, semi-structured, unstructured)
- **Additional Data Characteristics:** Veracity (data quality concerns), Variability (inconsistent flow rates), and Value (extracting actionable insights)
- **Analytics Pipeline Stages:** Data Generation, Collection, Storage, Processing, Analysis, Visualization, and Action
- **Batch vs Stream Processing:** Batch for historical analysis using Hadoop/Spark; Stream for real-time using Storm/Flink
- **Edge vs Cloud Analytics:** Edge provides low-latency local processing; Cloud enables complex analytics with vast resources; Hybrid balances both
- **Key Technologies:** Hadoop (batch), Spark (fast batch/stream), Storm (real-time), Oozie (workflow), Kafka (messaging)

### Important Concepts

- Descriptive analytics summarizes historical data; Diagnostic drills into root causes
- Predictive analytics uses ML and statistical models for forecasting; Prescriptive recommends optimal actions
- IoT analytics pipeline flows from device data generation through to automated actions and feedback
- Lambda/Kappa architectures combine batch and stream processing for hybrid IoT analytics

### Notes

- Remember all four analytics types with their key questions for exam answers
- The Three V's (Volume, Velocity, Variety) are frequently asked; know specific IoT scale examples
- Understand the distinction between batch processing (high throughput, high latency) and stream processing (low latency, continuous)
