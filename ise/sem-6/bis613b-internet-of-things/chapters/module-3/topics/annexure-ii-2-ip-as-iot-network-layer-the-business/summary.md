# **Annexure-II 2 IP as IoT Network Layer**

## **Business Case for IP**

- **Definition:** IP (Internet Protocol) is a fundamental protocol for the Internet that enables communication between devices.
- **Business Case:**
  - Scalability: IP supports a large number of devices, making it suitable for IoT applications.
  - Flexibility: IP allows for various network topologies and can be used for both wired and wireless networks.
  - Interoperability: IP enables communication between devices from different manufacturers.

## **Need for Optimization**

- **Challenges:**
  - Device density: Increasing number of devices can lead to network congestion and decreased performance.
  - Limited bandwidth: IoT devices often require low-bandwidth connections, which can lead to reduced network performance.
  - Security: IoT devices are vulnerable to cyber-attacks, compromising network security.

## **Optimizing IP for IoT**

- **Techniques:**
  - **Network Slicing:** Partitioning the network into multiple virtual networks with dedicated resources.
  - **Low-Power Wide-Area Networks (LPWANs):** Using low-power communication protocols for IoT devices.
  - **Network Function Virtualization (NFV):** Virtualizing network functions to improve efficiency and scalability.
- **Profiles and Compliances:**
  - **IEEE 802.15.4:** Standard for LPWANs.
  - **CoAP (Constrained Application Protocol):** Protocol for constrained networks and devices.
  - **IPv6:** Next-generation IP protocol that supports large address spaces.

## **Important Formulas and Definitions**

- **Theorem:** The OSI Model (Open Systems Interconnection Model) is a 7-layered model that describes the architecture of a communication network.
- **Formula:** The formula for calculating the bandwidth of a network is: Bandwidth = Throughput / Latency.
