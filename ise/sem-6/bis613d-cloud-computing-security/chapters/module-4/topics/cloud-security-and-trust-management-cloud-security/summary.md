# **Cloud Security and Trust Management**

## **Revision Notes**

### Cloud Security Defense Strategies

- **Multi-layered security approach**: Implementing multiple security measures to prevent unauthorized access
  - Network security
  - Data encryption
  - Identity and access management
- **Security governance**: Establishing policies, procedures, and standards for cloud security
- **Continuous monitoring**: Regularly monitoring and analyzing security logs to detect threats

### Distributed Intrusion/Anomaly Detection

- **Distributed Intrusion Detection Systems (DIDS)**: Using multiple sensors to detect anomalies and intrusions
- **Machine Learning (ML) and Artificial Intelligence (AI)**: Using ML and AI algorithms to identify patterns and anomalies
- **Behavioral analysis**: Analyzing user behavior to identify suspicious activity

### Data and Software Protection Techniques

- **Data encryption**: Protecting data in transit and at rest using encryption algorithms
  - AES (Advanced Encryption Standard)
  - RSA (Rivest-Shamir-Adleman)
- **Access control**: Controlling access to data and resources using authentication and authorization mechanisms
- **Data loss prevention (DLP)**: Preventing unauthorized data exfiltration

### Key Formulas and Theorems

- **Shannon's Source Coding Theorem**: A fundamental theorem in information theory that forms the basis of data compression algorithms
- **Bell's Inequality**: A mathematical inequality that is used to detect anomalies and intrusions
- **Bayes' Theorem**: A mathematical formula used in machine learning and AI to classify data

### Important Definitions

- **Cloud security**: The practice of protecting cloud-based resources and data from unauthorized access, use, disclosure, disruption, modification, or destruction
- **Cloud trust management**: The process of managing and maintaining trust in cloud computing environments
- **Security-as-a-Service (SaaS)**: A cloud-based security service that provides security controls and monitoring
