# Cloud Security: Revision Notes

### Encryption

- **Definition:** Protecting data from unauthorized access by converting it into an unreadable format.
- **Types:**
  - Symmetric encryption (e.g., AES)
  - Asymmetric encryption (e.g., RSA)
- **Importance:** Essential for securing data in the cloud.
- **Key concept:** Data Encryption Standard (DES)

### Security of Database Services

- **Definition:** Protecting database data from unauthorized access, modification, or deletion.
- **Security measures:**
  - Access control lists (ACLs)
  - Row-level security (RLS)
  - Data masking
- **Key concept:** Database Security Model (DSM)

### OS Security

- **Definition:** Protecting an operating system from unauthorized access, modification, or deletion.
- **Security measures:**
  - Firewalls
  - Intrusion detection systems (IDS)
  - Secure boot mechanisms
- **Key concept:** Security Zones

### VM Security

- **Definition:** Protecting virtual machines from unauthorized access, modification, or deletion.
- **Security measures:**
  - Virtualization-based security (VBS)
  - Hypervisor-based security (HBS)
  - VM monitoring and logging
- **Key concept:** Virtual Machine Security (VMS)

### Security Risks Posed by Shared Images and Management OS

- **Definition:** Risks associated with shared images and management OS in the cloud.
- **Security concerns:**
  - Malicious code in images
  - Management OS vulnerabilities
  - Unintended changes to configurations
- **Key concept:** Image Vulnerability Assessment (IVA)

### XOAR (Xen Orchestra for Access and Runtime)

- **Definition:** A security framework for virtualized environments.
- **Key concept:** Secure Virtualization Architecture (SVA)

### A Trusted Hypervisor (ATH)

- **Definition:** A trusted hypervisor that ensures the security and integrity of virtualized environments.
- **Key concept:** Trusted Execution Environment (TEE)
- **Important Formula:** None (theoretical framework)

## Key Concepts and Definitions

- **Cloud Security**: Protecting data, applications, and infrastructure in the cloud.
- **Security Risks**: Threats to the security and integrity of cloud resources.
- **Security Measures**: Techniques and technologies used to mitigate security risks.

Note: This summary is not exhaustive, but it covers the key points for quick revision.
