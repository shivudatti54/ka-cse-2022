# Cloud Computing & Security

**Revision Notes: 11.10 to 11.14, Chapter 4: 4.6**

### Key Concepts

- **Cloud Security**: The practice of ensuring the confidentiality, integrity, and availability of data in cloud computing.
- **Cloud Security Risks**: Cyber attacks, data breaches, unauthorized access, and data loss.
- **Privacy Impact Assessment (PIA)**: A process to identify and mitigate potential privacy risks associated with cloud services.
- **Cloud Data**: Data stored or processed in the cloud.

### Important Formulas and Definitions

- **Cloud Security Framework**:
  - NIST Cybersecurity Framework (CSF): A comprehensive framework for managing cloud security risks.
  - ISO/IEC 27001:2013: An international standard for information security management systems (ISMS).
- **Cloud Security Theorems**:
  - Church-Turing thesis: A theorem that states any effectively calculable function can be computed by a Turing machine.
  - Curry-Howard correspondence: A theorem that establishes a connection between proof theory and programming.

### Key Points

- Cloud security risks can be categorized into three main types:
  - Confidentiality risks: Unauthorized access to sensitive data.
  - Integrity risks: Unauthorized modification or alteration of data.
  - Availability risks: Denial of access to data or services.
- PIA involves:
  - Identifying potential privacy risks
  - Assessing the likelihood and impact of these risks
  - Developing mitigating controls to reduce the risk
- Cloud providers are responsible for:
  - Implementing security measures to protect customer data
  - Ensuring the confidentiality, integrity, and availability of data
