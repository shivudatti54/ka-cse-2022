# **Cloud Computing & Security Revision Notes (Chapter 6: 6.1 to 6.5)**

### 6.1 Service Models

- **IaaS (Infrastructure as a Service)**: Provides virtualized computing resources over the internet.
  - Key characteristics: Customization, control, and management of infrastructure.
- **PaaS (Platform as a Service)**: Provides a platform for developing, running, and managing applications over the internet.
  - Key characteristics: Focus on application development, deployment, and management.
- **SaaS (Software as a Service)**: Provides software applications over the internet.
  - Key characteristics: Focus on application usage, with minimal configuration and maintenance.

### 6.2 Service Level Agreements (SLAs)

- Definition: A contract that specifies the service level and quality of service for a cloud service.
- Key components:
  - Service availability and uptime
  - Response time and resolution time
  - Data security and backup

### 6.3 Cloud Security Architecture

- Definition: The overall structure and organization of security controls in a cloud environment.
- Key components:
  - Network security
  - Identity and access management
  - Data security and encryption
  - Incident response and management

### 6.4 Cloud Security Threats

- Definition: Risks and attacks on cloud-based systems and data.
- Key threats:
  - Data breaches and unauthorized access
  - Denial-of-service (DoS) and distributed denial-of-service (DDoS) attacks
  - Insider threats and privilege escalation
  - Malware and ransomware attacks

### 6.5 Cloud Security Best Practices

- Definition: Guidelines and recommendations for securing cloud-based systems and data.
- Key best practices:
  - Implement a cloud security strategy and governance
  - Use secure protocols and encryption
  - Monitor and analyze logs and security events
  - Conduct regular security audits and risk assessments
