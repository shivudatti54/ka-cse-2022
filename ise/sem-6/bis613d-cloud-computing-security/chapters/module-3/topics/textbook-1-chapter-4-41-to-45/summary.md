# **Cloud Computing & Security Revision Notes: Chapter 4**

### 4.1: Cloud Service Models

- **Service Models:**
  - IaaS (Infrastructure as a Service)
  - PaaS (Platform as a Service)
  - SaaS (Software as a Service)
- **Key Characteristics:**
  - IaaS: Customizable infrastructure, resource allocation
  - PaaS: Pre-configured platform, resource allocation
  - SaaS: Pre-configured application, resource allocation

### 4.2: Cloud Service Providers

- **Cloud Service Providers:**
  - Public Cloud (AWS, Azure, Google Cloud)
  - Private Cloud (on-premises, owned by an organization)
  - Hybrid Cloud (combination of public and private clouds)

### 4.3: Cloud Security Architecture

- **Cloud Security Architecture:**
  - Network Security
  - Identity and Access Management (IAM)
  - Data Security
  - Compliance and Governance

### 4.4: Cloud Security Threats

- **Cloud Security Threats:**
  - Insider Threats
  - Malware and Ransomware
  - Denial of Service (DoS) and Distributed Denial of Service (DDoS)
  - Data Breaches and Unauthorized Access

### 4.5: Cloud Security Controls

- **Cloud Security Controls:**
  - Firewalls and Network Segmentation
  - Access Control Lists (ACLs) and IAM
  - Encryption and Key Management
  - Monitoring and Incident Response

## Important Formulas, Definitions, and Theorems:

- **Cloud Security Architecture Diagram:** Not explicitly defined, but can be a combination of network, identity, and data security components.
- **Risk Assessment Matrix:** Not explicitly defined, but can be used to evaluate the likelihood and impact of cloud security threats.
- **Cloud Security Framework:** NIST Cybersecurity Framework, ISO 27017 Cloud Security, and PCI-DSS Cloud Security are some examples.
