# File System Interface: File Concept

### Definitions and Notations

- **File**: A collection of data stored in a specific location on a storage device.
- **File System Interface (FSI)**: An interface between the operating system and file systems.
- **File Handle**: A unique identifier for a file.
- **File Status**: The state of a file (e.g., open, closed, read-only).

### File System Concepts

- **File Type**: The type of data stored in a file (e.g., text, image, executable).
- **File Size**: The amount of storage space allocated to a file.
- **File Permissions**: The access rights granted to users for a file (e.g., read, write, execute).
- **File Attributes**: Properties of a file, such as creation date, last modified date, and owner.

### File System Operations

- **Open**: Establishes a connection to a file.
- **Create**: Creates a new file.
- **Read**: Retrieves data from a file.
- **Write**: Inserts data into a file.
- **Delete**: Removes a file from the file system.

### Important Formulas and Theorems

- **File Size Formula**: `file_size = total_storage_space - free_storage_space`
- **File Allocation Formula**: `file_allocation = (total_storage_space \* file_size) / total_storage_space`

### Key Points

- A file system interface is necessary for efficient file management.
- Files have different types, sizes, and permissions.
- File systems support various operations, including open, create, read, write, and delete.
- Understanding file system concepts and operations is essential for operating system development and management.
