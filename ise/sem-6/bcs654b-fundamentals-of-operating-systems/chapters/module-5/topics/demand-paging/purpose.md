# Learning Objectives
After studying this topic, you should be able to:
1.  Define demand paging and explain its role in implementing virtual memory.
2.  Describe the function of the valid-invalid bit in a page table entry and the circumstances that cause a page fault.
3.  Trace the complete sequence of steps executed by the hardware and operating system to handle a page fault.
4.  Calculate the Effective Access Time (EAT) for a memory system given page fault rates and service times.
5.  Explain the Copy-on-Write (COW) technique, its purpose, and how it optimizes the `fork()` system call.
6.  Understand the necessity of page replacement algorithms and the basic steps involved when no free frame is available.
