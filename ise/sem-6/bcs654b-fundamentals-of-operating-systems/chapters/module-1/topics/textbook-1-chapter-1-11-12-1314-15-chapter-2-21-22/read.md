# **FUNDAMENTALS OF OPERATING SYSTEMS**

## **Module: Introduction**

### What are Operating Systems?

An operating system (OS) is a software that manages computer hardware resources and provides common services to computer programs. It acts as an intermediary between computer hardware and user-level applications.

### Key Functions of Operating Systems:

- Process Management: Creates, schedules, and terminates processes
- Memory Management: Allocates and deallocates memory for running programs
- File System Management: Provides a file system interface for storing and retrieving files
- Input/Output Management: Handles input/output operations between devices and programs
- Security: Ensures the security and integrity of system resources
- Networking: Provides networking capabilities for communication between devices

### Computer System Organization

A computer system consists of the following components:

- **Hardware Components:**
  - Central Processing Unit (CPU)
  - Memory (RAM)
  - Input/Output Devices (Keyboard, Mouse, Monitor)
  - Storage Devices (Hard Disk Drive, Solid-State Drive)
- **Software Components:**
  - Operating System
  - Application Software
  - Firmware

### Types of Operating Systems

- **Single-User, Single-Tasking OS:** Allows only one user and one task to run at a time. Examples: MS-DOS, old versions of Windows.
- **Single-User, Multi-Tasking OS:** Allows one user to run multiple tasks simultaneously. Examples: Windows 3.1, old versions of macOS.
- **Multi-User, Multi-Tasking OS:** Allows multiple users and tasks to run simultaneously. Examples: Unix, Linux, modern versions of Windows.

### Operating System Hierarchy

The operating system hierarchy consists of the following levels:

1.  **Device Driver Level:** Provides a interface between the operating system and hardware devices.
2.  **System Call Level:** Provides a interface between the operating system and application software.
3.  **Kernel Level:** The core part of the operating system that manages hardware resources and provides basic services to application software.

**Chapter 2: Operating System Concepts**

### 2.1: Process Management

Process management is the ability of an operating system to create, schedule, and terminate processes. A process is a program in execution.

### 2.2: Process Scheduling

Process scheduling is the ability of an operating system to allocate system resources to a running process. There are two types of process scheduling:

- **First-Come-First-Served (FCFS) Scheduling:** Schedules processes based on the order in which they arrive at the system.
- **Prioritized Scheduling:** Schedules processes based on their priority levels.

### 2.2.1: Process Scheduling Algorithms

There are several process scheduling algorithms:

- **FCFS Scheduling Algorithm:** Schedules processes based on the order in which they arrive at the system.
- **Shortest Job First (SJF) Scheduling Algorithm:** Schedules processes based on their remaining execution time.
- **Round Robin (RR) Scheduling Algorithm:** Schedules processes in a circular order.

### 2.2.2: Process Scheduling Techniques

There are several process scheduling techniques:

- **Preemptive Scheduling:** Schedules processes based on their priority levels and allocates system resources accordingly.
- **Non-Preemptive Scheduling:** Schedules processes based on their priority levels, but does not allocate system resources.

### 2.3: Memory Management

Memory management is the ability of an operating system to allocate and deallocate memory for running programs. There are two types of memory management:

- **Static Memory Allocation:** Allocates memory from the beginning of the program execution.
- **Dynamic Memory Allocation:** Allocates memory on demand.

### 2.3.2: Memory Management Algorithms

There are several memory management algorithms:

- **First-Fit (FF) Algorithm:** Allocates memory to a program based on the first available allocation unit.
- **Best-Fit (BF) Algorithm:** Allocates memory to a program based on the smallest allocation unit.
- **Worst-Fit (WF) Algorithm:** Allocates memory to a program based on the largest allocation unit.
