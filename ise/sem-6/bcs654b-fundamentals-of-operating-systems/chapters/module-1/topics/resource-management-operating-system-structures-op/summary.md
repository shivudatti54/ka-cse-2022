# **Resource Management Operating System Structures: Operating System Services**

## **Revision Notes**

### Key Concepts

- **Operating System Services**: Provide basic services to computer hardware components
- **System Calls**: Interface between user application and OS kernel
- **Process Management**: Creation, scheduling, synchronization, and termination of processes
- **Memory Management**: Allocation, deallocation, and protection of memory
- **File Management**: Creation, deletion, and manipulation of files
- **Input/Output Management**: Handling of input/output operations between devices and applications

### Important Formulas and Definitions

- **CPU Time**: Measure of time spent by CPU in executing a process
  - Formula: CPU Time = Total CPU Time / Number of Processes
- **I/O Time**: Measure of time spent by I/O devices in transferring data
  - Formula: I/O Time = Total I/O Time / Number of I/O Operations
- **Process Scheduling**: Algorithm for allocating CPU time to processes
  - First-Come-First-Served (FCFS), Shortest Job First (SJF), Priority Scheduling (PS)
- **Memory Allocation**: Allocation of memory to processes
  - First-Come-First-Served (FCFS), Best Fit, Worst Fit
- **File Allocation**: Allocation of file space to files
  - Contiguous Allocation, Linked Allocation

### Important Theorems

- **Memory Theorem**: States that computer memory is divided into two parts: primary memory and secondary memory
- **Factoring Theorem**: States that the total CPU time is the sum of the CPU times for each process

### Key Points

- Operating system services are essential for computer system organization
- System calls provide a interface between user application and OS kernel
- Process management is crucial for efficient use of computer resources
- Memory management is essential for preventing memory-related errors
- File management is important for efficient storage and retrieval of files
- Input/output management is essential for handling input/output operations between devices and applications
