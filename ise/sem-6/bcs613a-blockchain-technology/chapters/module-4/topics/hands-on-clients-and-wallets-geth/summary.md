# Hands-on: Clients and Wallets – Geth

### Introduction

- Geth is a popular Ethereum client used for development and testing.
- It is written in Go and provides a command-line interface for interacting with the Ethereum network.

### Key Features of Geth

- **Wallet creation**: Geth allows users to create new wallets and manage existing ones.
- **Transaction signing**: Geth supports signing transactions using the Ethereum account's private key.
- **Transaction sending**: Geth enables users to send transactions to the Ethereum network.

### Geth Configuration

- **Data directory**: The location where Geth stores its data.
- **Network ID**: The ID of the Ethereum network to connect to.
- **Account management**: Options for managing Ethereum accounts.

### Geth Commands

- **`geth`**: The main Geth command for starting the node.
- **`geth --wallet`**: Command for creating a new wallet.
- **`geth --account`**: Command for managing Ethereum accounts.

### Important Formulas and Definitions

- **Public key**: A unique identifier for an Ethereum account.
- **Private key**: The secret key used to sign transactions.
- **Keystore file**: A file used to store the private key for an Ethereum account.

### Theorems and Concepts

- **Blockchain**: A distributed ledger technology used for secure and transparent data storage.
- **Consensus algorithm**: A mechanism for achieving agreement among nodes on the Ethereum network.
- **Ethereum stack**: The combination of the Ethereum protocol, client software, and operating system.
