# **Revision Notes: Chapter 3 of Blockchain Technology**

### Decentralization using Blockchain

- **Definition:** Decentralization - a system where data is not controlled by a single entity, but is distributed among many nodes.
- **Types of Decentralization:**
  - Horizontal Decentralization: multiple nodes have equal control
  - Vertical Decentralization: power is distributed among multiple levels
- **Blockchain Decentralization:**
  - A distributed ledger that records transactions across a network of nodes
  - Each node has a copy of the ledger, ensuring data integrity and security

### Methods of Decentralization

- **P2P (Peer-to-Peer) Networks:**
  - Direct communication between nodes, eliminating the need for intermediaries
  - Examples: BitTorrent, Ethereum
- **Distributed Hash Tables (DHTs):**
  - A decentralized storage system for data, allowing for efficient lookup and retrieval
  - Examples: Freenet, BitTorrent

### Important Formulas and Theorems

- **Hash Function:** A one-way function that maps input data to a fixed-size output (e.g., SHA-256)
- **Public-Key Cryptography:**
  - A method of encrypting data using a pair of keys: public and private
  - Examples: RSA, Elliptic Curve Cryptography (ECC)

### Important Definitions

- **Consensus Mechanism:** A protocol that ensures all nodes agree on the state of the blockchain
- **Node:** A computer or device participating in the blockchain network
- **Block:** A group of transactions recorded in the blockchain ledger

### Important Theorems

- **Shannon's Theorem:** A fundamental limit on the information that can be transmitted through a communication channel
- **Gödel's Incompleteness Theorem:** A result that shows any formal system is either incomplete or inconsistent
