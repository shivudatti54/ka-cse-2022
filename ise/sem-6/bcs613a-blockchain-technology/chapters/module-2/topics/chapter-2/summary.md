# Chapter 2: Decentralization using Blockchain

### Introduction

- Definition: A decentralized system is a network that allows nodes to share information and make decisions without the need for a central authority.
- Blockchain technology provides a decentralized method for achieving decentralization.

### Decentralization using Blockchain

- **Decentralized Network**: A network of nodes that work together to achieve a common goal, without relying on a central authority.
- **Consensus Mechanism**: A mechanism that allows nodes to agree on the state of the network, ensuring that all nodes have the same view of the network.
- **Block Chain**: A distributed database that records all transactions and data in a decentralized network.

### Methods of Decentralization

- **Peer-to-Peer (P2P) Networks**: Networks where nodes act as both clients and servers, allowing for decentralized communication.
- **Distributed Hash Tables (DHTs)**: A data storage system that allows nodes to share and retrieve data in a decentralized manner.

### Important Formulas and Definitions

- **Hash Function**: A one-way function that maps data to a fixed-size string of characters (e.g. SHA-256).
- **Public Key Cryptography**: A method of encryption that uses a pair of keys: a public key for encryption and a private key for decryption.
- **Smart Contracts**: Self-executing contracts with the terms of the agreement written directly into lines of code.

### Theorems

- **Byzantine Generals' Problem**: A theoretical problem that proves that it is impossible to achieve consensus in a decentralized network without a leader.
