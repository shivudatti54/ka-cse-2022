# Chapter 9 Revision Notes

### Hyperledger Fabric Overview

- Hyperledger Fabric is a distributed ledger technology that enables the creation of a blockchain network.
- It is an open-source framework that allows for the development of enterprise-grade blockchain applications.
- Fabric uses a consensus protocol called the Byzantine Fault Tolerance (BFT) algorithm.

### Hyperledger Fabric Architecture

- The Fabric architecture consists of four main components:
  - **Peer**: Each peer node in the network maintains a local copy of the distributed ledger.
  - **Orderer**: The orderer node is responsible for maintaining the order of transactions and ensuring the integrity of the ledger.
  - **Client**: The client node is where users interact with the blockchain network.
  - **Schema**: The schema defines the structure and organization of the data stored in the ledger.

### Hyperledger Fabric Consensus Protocol

- **Byzantine Fault Tolerance (BFT) algorithm**: A BFT algorithm is used to ensure that the network remains consistent in the presence of faults.
- **Raft protocol**: Fabric uses the Raft protocol to achieve consensus.

### Important Formulas and Definitions

- **Byzantine Generals Problem (BGP)**: A problem that describes the scenario in which a group of generals must agree on a course of action despite the possibility of traitors among them.
- **Byzantine Fault Tolerance (BFT)**: A fault-tolerance protocol that ensures the network remains consistent in the presence of faults.
- **Consensus protocol**: A protocol that ensures the network reaches a consensus on the state of the ledger.

### Important Theorems

- **Godel's Incompleteness Theorem**: A theorem that states that any formal system that is powerful enough to describe basic arithmetic is either incomplete or inconsistent.
- **Church-Turing Thesis**: A thesis that states that any effectively calculable function can be computed by a Turing machine.
