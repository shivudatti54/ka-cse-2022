# **Stacks: Introduction, Stack Operations, Stack Implementation using Arrays, Applications of Stacks**

### Introduction

- A stack is a linear data structure that follows the LIFO (Last In First Out) principle.
- It is a collection of elements that can be added or removed from the top.
- The top element is the first one to be added and the last one to be removed.

### Stack Operations

- **Push**: Adds an element to the top of the stack.
- **Pop**: Removes an element from the top of the stack.
- **Peek**: Returns the top element of the stack without removing it.
- **IsEmpty**: Checks if the stack is empty.
- **Size**: Returns the number of elements in the stack.

### Stack Implementation using Arrays

- **Top Index**: The index of the top element in the array.
- **Base Address**: The starting address of the array.
- **Stack Size**: The maximum number of elements the array can hold.
- **Array Initialization**: The array is initialized with a specific size.
- **Push Operation**: The top element is stored at the top index of the array.
- **Pop Operation**: The top element is removed and stored at the previous top index.

### Important Formulas and Theorems

- **Stack Depth**: The number of elements in the stack is equal to the number of elements that can be pushed onto the stack.
- **Stack Overflow**: The stack is full and cannot hold any more elements.
- **Stack Underflow**: The stack is empty and cannot perform any operations.

### Applications of Stacks

- **Evaluating Postfix Expressions**: Stacks can be used to evaluate postfix expressions.
- **Implementing Recursive Functions**: Stacks can be used to implement recursive functions iteratively.
- **Parsing and Syntax Analysis**: Stacks can be used in parsing and syntax analysis.
- **Backtracking Algorithms**: Stacks can be used in backtracking algorithms.
- **Depth-First Search**: Stacks can be used in depth-first search algorithms.
