## Revision Notes for 17.3.2

### Introduction

- **Definition:** A data structure is a way to organize and store data in a computer so that it can be efficiently accessed and manipulated.
- **Importance:** Understanding data structures is crucial for any programming endeavor.

### Sorting Algorithms

- **Definition:** A sorting algorithm is a method for arranging elements in a specific order.
- **Types of Sorting Algorithms:**
  - **Comparison-based sorting:** Bubble Sort, Selection Sort, Insertion Sort, Merge Sort, Quick Sort
  - **Non-comparison-based sorting:** Radix Sort, Counting Sort

### Bubble Sort

- **Algorithm:** Bubble Sort is a simple sorting algorithm that repeatedly steps through the list, compares adjacent elements, and swaps them if they are in the wrong order.
- **Formula:**
  - **Best-case scenario:** O(n) time complexity
  - **Worst-case scenario:** O(n^2) time complexity
  - **Average-case scenario:** O(n^2) time complexity
- **Example:** Bubble Sort is used to sort a list of numbers in ascending order.

### Selection Sort

- **Algorithm:** Selection Sort is a simple sorting algorithm that selects the smallest (or largest) element from the unsorted portion of the list and swaps it with the first unsorted element.
- **Formula:**
  - **Best-case scenario:** O(n^2) time complexity
  - **Worst-case scenario:** O(n^2) time complexity
  - **Average-case scenario:** O(n^2) time complexity
- **Example:** Selection Sort is used to sort a list of numbers in ascending order.

### Insertion Sort

- **Algorithm:** Insertion Sort is a simple sorting algorithm that builds the final sorted array (or list) one item at a time.
- **Formula:**
  - **Best-case scenario:** O(n) time complexity
  - **Worst-case scenario:** O(n^2) time complexity
  - **Average-case scenario:** O(n^2) time complexity
- **Example:** Insertion Sort is used to sort a list of numbers in ascending order.

### Important Formulas and Theorems

- **Big O notation:** A way to describe the performance or complexity of an algorithm.
- **Time complexity:** The amount of time an algorithm takes to complete as a function of the input size.
- **Space complexity:** The amount of memory an algorithm uses as a function of the input size.
