# **The Target Language Chapter 6: 6.1**

## **Key Points**

- **Definition:** The Target Language (TL) is the language that the compiler is designed to generate code for.
- **Key Features:**
  - A simple, minimal language with minimal dependencies
  - Designed to be easy to compile and optimize
- **Properties:**
  - Small, compact code
  - Low overhead
  - Fast execution

## **Symbolic Representation**

- **Symbolic Representation:** The TL is represented using a set of symbols, including:
  - **Variables:** `x`, `y`, `z`, etc.
  - **Constants:** `0`, `1`, `2`, etc.
  - **Operators:** `+`, `-`, `*`, `/`, etc.
- **Symbolic Formulas:** The TL can be represented using symbolic formulas, including:
  - **Arithmetic Expressions:** `a + b`, `a * b`, etc.
  - **Logical Expressions:** `a && b`, `a || b`, etc.

## **Terminology and Notation**

- **Notation:** The TL uses a specific notation, including:
  - **Variables:** `x[i]`, `y[j]`, etc.
  - **Arrays:** `a[i]`, `b[j]`, etc.
- **Terminology:** The TL uses specific terminology, including:
  - **Addressing Modes:** `immediate`, `register`, etc.
  - **Data Types:** `int`, `float`, etc.

## **Important Formulas and Theorems**

- **Addressing Formula:** `a = p + i`, where `a` is the address, `p` is the page, and `i` is the index.
- **Data Type Formula:** `t = a \* b`, where `t` is the data type, `a` is the size of the operand, and `b` is the size of the result.

- Review the notation and terminology used in the TL.
