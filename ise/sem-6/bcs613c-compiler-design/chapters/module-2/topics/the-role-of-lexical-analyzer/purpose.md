# Learning Objectives: The Role of the Lexical Analyzer

After studying this topic, you should be able to:

1.  Describe the fundamental purpose and function of the lexical analyzer within the compiler pipeline.
2.  Differentiate clearly between the concepts of a 'token' and a 'lexeme' and provide examples of each.
3.  Explain the key responsibilities of the lexical analyzer, including stripping comments/whitespace and interacting with the symbol table.
4.  Justify the design decision to separate lexical analysis from syntax analysis, citing at least three advantages.
5.  Trace the sequence of tokens generated by a lexical analyzer for a given simple source code statement.
6.  Identify common types of lexical errors and describe a simple strategy a lexical analyzer might use to recover from them.
