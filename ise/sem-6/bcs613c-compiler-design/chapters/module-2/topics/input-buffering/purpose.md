# Learning Objectives

After studying this topic, you should be able to:

1. Explain the need for input buffering in lexical analysis and its impact on compiler performance
2. Describe the working of the two-buffer (double buffering) scheme with appropriate diagrams
3. Understand the role of sentinels in simplifying buffer boundary checks
4. Implement basic input buffering techniques for lexical analysis
5. Analyze the performance implications of different buffering approaches
6. Handle special cases like token boundaries across buffers and very long tokens
