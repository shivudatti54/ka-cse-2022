# Learning Objectives

After studying this topic, you should be able to:

1.  Explain the fundamental principle behind recursive descent parsing and how it differs from bottom-up techniques.
2.  Identify whether a given grammar is suitable for recursive descent parsing, recognizing issues like left recursion and common prefixes.
3.  Transform a simple left-recursive grammar into a right-recursive, LL(1)-compatible form suitable for recursive descent.
4.  Manually trace the execution of a recursive descent parser for a given input string, showing the sequence of function calls and token consumption.
5.  Compute FIRST and FOLLOW sets for non-terminals in a grammar and use them to explain the predictive parsing decisions.
6.  Compare and contrast the advantages and disadvantages of recursive descent parsing with other parsing methods, such as LR parsing.
