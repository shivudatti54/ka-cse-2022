# Learning Objectives

After studying this topic, you should students able to:

1. Explain the role of the Host OS in cloud security and why its compromise is catastrophic.
2. Describe Host OS hardening strategies including minimal installation, CIS benchmarks, and mandatory access controls.
3. Differentiate between Host OS and Guest OS security responsibilities under the shared responsibility model.
4. Identify OS-level vulnerabilities unique to cloud environments (shared kernel, management OS exposure, side-channel attacks).
5. Compare DAC, MAC, and RBAC access control models and explain their use in cloud OS security.
6. Explain the role of process isolation mechanisms (namespaces, cgroups) and integrity monitoring (AIDE, TPM, Secure Boot).
