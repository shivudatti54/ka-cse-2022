# Features of Cloud and Grid Platforms

Cloud and Grid computing are distinct distributed computing paradigms with different architectures, philosophies, and feature sets. Grid computing coordinates and shares computing power across dynamic, multi-institutional virtual organizations to solve large-scale computational problems, embodying "collaboration for complex problem-solving" primarily in scientific and academic research. Cloud computing provides ubiquitous, on-demand network access to shared, configurable computing resources that can be rapidly provisioned with minimal management effort, representing "IT as a utility delivered as a service" driven by commercial providers.

Grid platforms feature distributed ownership across participating organizations, resource sharing as a core principle, decentralized federated control, high throughput computing for maximizing job completion, standards-based architecture (OGSA), and job-oriented submission to scheduling systems. Cloud platforms are defined by five NIST essential characteristics (BRNOE): Broad network access, Resource pooling, Measured service, On-demand self-service, and Rapid Elasticity, along with service models (IaaS, PaaS, SaaS) and deployment models (public, private, hybrid, community). Modern platforms blur these distinctions through cloud bursting, HPC on cloud, and data-intensive grids.

## Key Takeaways

- Grid computing focuses on collaborative problem-solving while cloud computing delivers IT as a utility service
- Grid features distributed ownership and decentralized control; cloud features centralized ownership and on-demand provisioning
- Five NIST characteristics (BRNOE) define cloud computing: Broad access, Resource pooling, Measured service, On-demand service, Rapid elasticity
- Modern platforms converge with HPC in cloud and cloud bursting blending grid and cloud paradigms
