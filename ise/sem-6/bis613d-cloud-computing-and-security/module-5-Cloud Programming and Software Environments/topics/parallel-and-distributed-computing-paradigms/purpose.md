# Learning Objectives

After studying this topic, you should students able to:

1. Differentiate between parallel and distributed computing paradigms and identify appropriate use cases for each
2. Explain the fundamental principles of data parallelism, task parallelism, and pipeline parallelism
3. Describe the architecture and operation of Client-Server, Peer-to-Peer, MapReduce, and Actor models
4. Analyze the trade-offs between different paradigms in terms of scalability, fault tolerance, and complexity
5. Map cloud computing services to appropriate programming paradigms (e.g., AWS Lambda to serverless, Hadoop to MapReduce)
6. Apply the CAP theorem to evaluate distributed system design choices and limitations
