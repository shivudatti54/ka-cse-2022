# Scalable Computing Over the Internet

## Overview

Scalable computing refers to a system's ability to handle growing workloads by being enlarged to accommodate growth. With exponential growth of internet users and data, scalable computing has become essential for modern applications, evolving from client-server architecture to cloud computing paradigms.

## Key Points

- **Elasticity**: Dynamic provisioning and deprovisioning of computing resources to meet changing demands automatically
- **Vertical Scaling (Scale-Up)**: Adding more resources to a single node; simpler but has physical limits and single point of failure
- **Horizontal Scaling (Scale-Out)**: Adding more nodes to a system; virtually unlimited scaling with fault tolerance using commodity hardware
- **Resource Pooling**: Provider's computing resources are pooled to serve multiple consumers using multi-tenant models
- **Load Balancing**: Distributes network traffic across multiple servers to prevent any single server from becoming overwhelmed
- **Virtualization Technology**: Allows multiple virtual machines to run on a single physical machine for efficient resource utilization
- **Content Delivery Networks (CDNs)**: Distribute content geographically to reduce latency and improve performance

## Important Concepts

- Evolution timeline: Standalone Computers → Client-Server → Cluster Computing → Grid Computing → Cloud Computing
- Five key characteristics: Elasticity, On-Demand Self-Service, Broad Network Access, Resource Pooling, Measured Service
- Horizontal scaling is preferred for cloud environments despite increased complexity
- Emerging trends include edge computing, serverless computing, and AI-driven autoscaling
