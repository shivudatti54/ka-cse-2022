# System Models for Distributed and Cloud Computing

## Overview

System models provide abstract representations of distributed computing systems, helping understand their structure, behavior, and characteristics. Cloud computing builds upon concepts from cluster computing, grid computing, and utility computing, with models categorizing different cloud approaches and architectural foundations.

## Key Points

- **Cluster Computing Model**: Tightly coupled systems with homogeneous hardware, centralized management, and high-speed interconnects for HPC applications
- **Grid Computing Model**: Loosely coupled systems with heterogeneous resources, cross-organizational boundaries, and decentralized control for resource sharing
- **Cloud Computing Model**: On-demand access to shared computing resources with five essential characteristics (on-demand self-service, broad network access, resource pooling, rapid elasticity, measured service)
- **IaaS (Infrastructure as a Service)**: Provides virtualized computing resources over the internet (e.g., Amazon EC2, Azure VMs)
- **PaaS (Platform as a Service)**: Provides development and deployment platforms in the cloud (e.g., Google App Engine, Heroku)
- **SaaS (Software as a Service)**: Provides complete software solutions accessible via web browsers (e.g., Google Workspace, Salesforce)
- **Deployment Models**: Public cloud (open to anyone), Private cloud (single organization), Hybrid cloud (combination), Community cloud (shared by multiple organizations)

## Important Concepts

- Fundamental distributed system characteristics: Concurrency, Lack of global clock, Independent failures, Resource sharing, Openness, Transparency, Scalability
- Evolution represents shift from localized, dedicated resources to distributed, on-demand resource provisioning
- Enabling technologies include virtualization, web services/APIs, distributed storage systems, and automation/orchestration tools
- Challenges include security and privacy, performance and reliability, interoperability and portability, and resource management

## Notes

- Be prepared to draw and explain layered diagrams for IaaS, PaaS, and SaaS models in exams
- Understand differences between cluster, grid, and cloud computing in tabular format
- Focus on the five NIST essential characteristics of cloud computing
- Remember real-world examples for each service and deployment model
