# Distributed System Models and Enabling Technologies

## Overview

Cloud Computing is an evolution of distributed systems, which are collections of independent computers appearing as a single coherent system. Understanding distributed system models (clusters, grids, and clouds) and their enabling technologies (virtualization, SOA, utility computing) is fundamental to grasping how modern cloud platforms function.

## Key Points

- **Clusters**: Groups of homogeneous computers in one location, designed for high performance and availability through parallel processing and redundancy
- **Grids**: Federation of heterogeneous resources from multiple administrative domains, enabling large-scale resource sharing across organizations
- **Clouds**: Virtualized, service-oriented systems offering on-demand utility computing with elasticity, scalability, and pay-as-you-go pricing
- **Virtualization**: Core technology that creates an abstraction layer over physical hardware, allowing multiple Virtual Machines to run on a single physical machine
- **Service-Oriented Architecture (SOA)**: Design pattern using web services and APIs to enable communication and form the foundation for XaaS models
- **Utility Computing**: Vision of providing computing resources as a metered service, basis for cloud's pay-as-you-go pricing model
- **Autonomic Computing**: Self-managing computer systems crucial for cloud elasticity and scalability features like auto-scaling

## Important Concepts

- Evolution from clusters to grids to clouds represents a shift towards greater flexibility and accessibility
- Tight coupling in clusters vs. loose coupling in grids vs. elastic coupling in clouds
- High-speed interconnects in clusters (InfiniBand) vs. wide-area networks in grids vs. internet-based cloud access
- Centralized management in clusters vs. decentralized in grids vs. provider-managed in clouds
