# **Methods & Objects**

### Key Points

- **Method**: A function that belongs to an object and can be called on that object.
- **Object**: An unordered collection of key-value pairs.
- **Methods**: Can be used to perform operations on an object's properties.
- **Properties**: Can be accessed using dot notation (e.g., `obj.property`).
- **Methods are functions**: Can be called using parentheses (e.g., `obj.method()`).
- **Methods can be defined inside objects**: Using the `function` keyword.
- **Methods can be defined outside objects**: Using the `function` keyword and then assigning to the object.

### Important Formulas and Definitions

- **Prototype chain**: A chain of objects that an object inherits from.
- **Inheritance**: The ability of an object to inherit properties and methods from another object.
- **Scope**: The region of the code where a variable or function is defined.

### Key Theorems

- **Method binding**: The process of binding a method to an object.
- **Method overriding**: When a method with the same name is defined in a subclass.

### Revision Notes

- Methods can be defined using the `function` keyword.
- Methods can be passed as arguments to other functions.
- Methods can be used to manipulate an object's properties.
- Objects can inherit properties and methods from other objects.
- The prototype chain is used to access inherited properties and methods.
