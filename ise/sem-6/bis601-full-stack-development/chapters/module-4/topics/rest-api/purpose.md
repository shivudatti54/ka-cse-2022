### Learning Purpose: REST API

**1. Why is this topic important?**
REST APIs are the fundamental building blocks of modern web and mobile applications. They enable the frontend (client-side) and backend (server-side) to communicate seamlessly, making them essential for any full stack developer. Understanding REST APIs is critical for creating scalable, decoupled, and interoperable systems.

**2. What will students learn?**
Students will learn the core principles of REST (Representational State Transfer), how to design clear and predictable API endpoints, and the use of HTTP methods (GET, POST, PUT, DELETE) for CRUD operations. They will also learn to handle HTTP status codes, requests, and responses, and implement a RESTful API using a backend framework.

**3. How does it connect to other concepts?**
This topic directly connects to frontend development (Module 2/3), where students will use JavaScript (e.g., Fetch API) to consume the APIs they build. It relies on backend logic (Node.js/Express from Module 3), server-side routing, and databases (Module 5) to persist and retrieve data, forming the crucial link between all layers of the stack.

**4. Real-world applications**
REST APIs are used everywhere: social media feeds (fetching posts), e-commerce sites (processing orders), weather apps (pulling data), and banking systems. Mastering them allows developers to build applications that can interact with countless external services and empower frontend frameworks like React or Angular.
