# **Add a new city at the end**

## **Introduction**

In this study material, we will cover how to add a new city to an existing database in MongoDB. We will explore the concept of documents, collections, and databases, and understand the query language used in MongoDB.

## **What is a Document?**

A document in MongoDB is a collection of key-value pairs that represent a single entity. Each document is an independent unit and can contain a mix of data types such as strings, integers, dates, and more. Documents are stored in collections, and each collection represents a table in a relational database.

## **What is a Collection?**

A collection in MongoDB is a group of related documents. Each collection has a unique name and can contain multiple documents. Collections are similar to tables in relational databases.

## **What is a Database?**

A database in MongoDB is a container that holds multiple collections. Each database has a unique name and can contain multiple collections.

## **Adding a new city**

To add a new city to an existing database in MongoDB, we need to follow these steps:

### Step 1: Create a new document

To add a new city, we need to create a new document. Each document should have a unique `_id` field, which is automatically generated by MongoDB.

### Step 2: Add the city data

The city data can be added as a field in the document. For example, we can add the city name, country, and population.

### Step 3: Insert the document into the collection

Once we have created the new document, we can insert it into the collection using the `insertOne` or `insertMany` method.

## **Example Code**

```javascript
const MongoClient = require('mongodb').MongoClient;

const url = 'mongodb://localhost:27017';
const dbName = 'cities';

MongoClient.connect(url, function (err, client) {
  if (err) {
    console.log(err);
    return;
  }
  console.log('Connected to MongoDB');

  const db = client.db(dbName);
  const collection = db.collection('cities');

  const newCity = {
    _id: ObjectId(),
    name: 'New City',
    country: 'USA',
    population: 100000,
  };

  collection.insertOne(newCity, function (err, result) {
    if (err) {
      console.log(err);
    } else {
      console.log('New city added successfully');
    }
  });

  client.close();
});
```

## **Querying the data**

Once we have inserted the new city, we can query the data using the query language in MongoDB. We can use the `find` method to retrieve all cities, or use the `$eq` operator to retrieve cities with a specific condition.

## **Example Query**

```javascript
const MongoClient = require('mongodb').MongoClient;

const url = 'mongodb://localhost:27017';
const dbName = 'cities';

MongoClient.connect(url, function (err, client) {
  if (err) {
    console.log(err);
    return;
  }
  console.log('Connected to MongoDB');

  const db = client.db(dbName);
  const collection = db.collection('cities');

  collection.find({ name: { $eq: 'New City' } }).toArray(function (err, cities) {
    if (err) {
      console.log(err);
    } else {
      console.log(cities);
    }
  });

  client.close();
});
```

## **Key Concepts**

- **Documents**: A collection of key-value pairs that represent a single entity.
- **Collections**: A group of related documents.
- **Databases**: A container that holds multiple collections.
- **InsertOne**: A method to insert a single document into a collection.
- **InsertMany**: A method to insert multiple documents into a collection.
- **Find**: A method to retrieve documents from a collection.
- **$eq**: An operator to compare values.
- **ObjectId**: A unique identifier for each document.
- **MongoShell**: A command-line tool to interact with MongoDB.
