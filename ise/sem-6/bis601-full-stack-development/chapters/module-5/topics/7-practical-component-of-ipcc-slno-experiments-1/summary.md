## MongoDB Concepts

### Key Points

- **Documents**: Similar to JSON or XML files, they are self-contained, readable, and maintainable.
- **Collections**: Groups one or more documents together based on similar characteristics.
- **Databases**: A collection of related data stored in a single file.
- **Query Language**: Used to find specific data within a database.
- **Installation**: MongoDB can be installed on most operating systems.
- **Mongo Shell**: A command-line tool for interacting with MongoDB.

## Practical Components

### Key Points

- **Document-based database**: MongoDB uses a document-based approach to store and retrieve data.
- **Flexible schema**: MongoDB's schema-less design allows for flexibility and scalability.
- **Scalability**: MongoDB's distributed architecture makes it suitable for large-scale applications.
- **Real-time data analytics**: MongoDB's query language and indexing capabilities enable real-time data analysis.
- **Elasticity**: MongoDB's horizontal scaling capabilities allow for easy addition of new nodes.

### Important Formulas, Definitions, Theorems

- **NoSQL Database**: A database that does not use the traditional relational database management system (RDBMS) structure.
- **Schema-less Design**: The ability to store data without a predefined schema.
- **Document-oriented Database**: A database that stores data in self-contained documents.
