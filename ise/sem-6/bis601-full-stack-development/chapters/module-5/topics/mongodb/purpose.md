# Learning Objectives

After studying this topic, you should be able to:

1. Explain the document data model and how it differs from the relational model
2. Describe MongoDB's architecture components including replica sets and sharding
3. Write basic CRUD operations using MongoDB query language
4. Construct aggregation pipelines with multiple stages for data analysis
5. Implement appropriate indexing strategies for different query patterns
6. Evaluate when MongoDB is suitable for a given use case compared to other database types
