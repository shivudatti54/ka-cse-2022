# Text Book 2: Chapter 6

### Key Points

- **What is MongoDB?**
  - NoSQL document-based database
  - Distributed, scalable, and flexible
  - Supports modern web and mobile applications
- **MongoDB Document Structure**
  - JSON-like documents with embedded arrays and objects
  - Fields are optional and can be nested
  - Documents can contain multiple data types (strings, numbers, dates, etc.)
- **MongoDB Collection**
  - A collection is a group of related documents
  - Each collection has its own unique name
  - A collection can contain multiple documents
- **MongoDB Database**
  - A database is a container for multiple collections
  - Each database has its own unique name
  - Databases can be used to organize related collections
- **MongoDB Query Language**
  - Query language is used to retrieve data from MongoDB
  - Queries can be used to filter, sort, and aggregate data
  - Query language supports various operators (e.g., ==, >, $regex)
- **MongoDB Installation**
  - Download and install MongoDB from official website
  - Choose between Community Server, Enterprise Server, or Stitch

### Important Formulas, Definitions, and Theorems

- **Formula:** `db.collection.find()`
  - Retrieves a cursor to a collection
  - Returns a set of documents that match the query
- **Definition:** **Document** - a single piece of data stored in MongoDB
- **Definition:** **Collection** - a group of related documents stored in MongoDB
- **Definition:** **Database** - a container for multiple collections stored in MongoDB
- **Theorem:** **The MongoDB Query Language is flexible and powerful**
  - Supports various operators and query methods
