# Learning Objectives

After studying this topic, you should be able to:

1. Define an operating system and explain its dual role as a resource manager and virtual machine
2. Identify and describe the major components of an operating system architecture
3. Compare and contrast different types of operating systems including batch, multiprogrammed, time-sharing, distributed, and real-time systems
4. Explain the purpose and types of system calls and their role in user-kernel interaction
5. Describe the process model including process states and the process control block structure
6. Explain the memory model and the distinction between user space and kernel space
7. Trace the boot sequence from power-on to user login
8. Analyze how the system model provides the foundation for understanding process synchronization and deadlock handling topics
