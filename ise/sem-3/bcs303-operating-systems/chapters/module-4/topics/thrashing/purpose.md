# Learning Objectives

After studying this topic, you should be able to:

1. Define thrashing and explain why it occurs in virtual memory systems with demand paging
2. Analyze the relationship between CPU utilization and degree of multiprogramming, including the thrashing region of the curve
3. Apply the working set model to determine whether a system will experience thrashing
4. Use the page fault frequency model to control thrashing through frame allocation adjustments
5. Calculate effective memory access time during thrashing using the formula EAT = (1-p) × Ma + p × Mt
6. Identify the causes of thrashing and explain how each contributes to performance degradation
7. Evaluate different strategies for preventing and controlling thrashing in operating systems
8. Solve numerical problems related to working set sizes, page fault rates, and memory allocation decisions
