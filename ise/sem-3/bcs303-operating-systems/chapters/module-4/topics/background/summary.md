# Background - Summary

## Key Definitions and Concepts

- MEMORY MANAGEMENT: The operating system function that controls how memory is allocated to processes and how the physical memory is mapped to processes
- LOGICAL ADDRESS: An address generated by the CPU during program execution, independent of physical memory (also called virtual address)
- PHYSICAL ADDRESS: The actual location in RAM where data is stored
- ADDRESS BINDING: The process of mapping a logical address to a physical address, which can occur at compile time, load time, or execution time
- MEMORY MANAGEMENT UNIT (MMU): Hardware component that translates logical addresses to physical addresses
- INTERNAL FRAGMENTATION: Wasted memory within an allocated block due to the block being larger than requested
- EXTERNAL FRAGMENTATION: Scattered free memory blocks that cannot satisfy a large request despite total free memory being sufficient

## Important Formulas and Theorems

- Logical Address = Page Number × Page Size + Offset
- Physical Address = Frame Number × Page Size + Offset
- Page Number = Logical Address DIV Page Size
- Offset = Logical Address MOD Page Size
- Physical Address with Base Register = Base Register Value + Logical Address

## Key Points

- Memory hierarchy exists because fastest memory (registers) is most expensive and smallest in capacity
- The principle of locality explains why memory management techniques work: programs tend to access the same data repeatedly
- Address binding at execution time is required for virtual memory and swapping support
- Base and limit registers provide both relocation (allowing programs to load anywhere) and protection (preventing unauthorized access)
- Contiguous memory allocation suffers from external fragmentation and internal fragmentation
- First-fit is generally fastest but can lead to more fragmentation over time; best-fit minimizes wasted space but searches longer
- Paging eliminates external fragmentation by dividing both logical and physical memory into fixed-size blocks (pages and frames)
- The MMU is central to modern memory management, performing address translation on every memory access

## Common Mistakes to Avoid

- Confusing logical and physical addresses; remember that CPU generates logical addresses, MMU translates them to physical
- Forgetting that offset remains unchanged during address translation in paging systems
- Mixing up internal and external fragmentation; internal occurs within allocated blocks, external occurs between free blocks
- Assuming that more physical memory always improves performance; effective management matters more than raw size
