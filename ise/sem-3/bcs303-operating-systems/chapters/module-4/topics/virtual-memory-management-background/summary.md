# Virtual Memory Management: Background

=====================================

## Key Concepts

---

- **Virtual Memory**: An extension of physical memory that provides a large address space, allowing multiple programs to run simultaneously.
- **Page Replacement Algorithms**: Used to select pages to replace when the physical memory is full.
- **Paging**: A technique to divide a program into fixed-size blocks called pages, which are stored in physical memory.
- **Segmentation**: A technique to divide a program into fixed-size blocks called segments, which are stored in physical memory.

## Definitions

---

- **Page Table**: A data structure that maps virtual page numbers to physical frame numbers.
- **Frame**: A unit of physical memory.
- **Page Fault**: An event that occurs when a page is accessed that is not in physical memory.
- **Page Reference Bit**: A bit that indicates whether a page is in physical memory or not.

## Formulas

---

- **Page Replacement Formula**: `P = (page fault rate) / (page replacement rate)`
- **Optimal Page Replacement Formula**: `P = (page fault rate) / (page replacement rate + 1)`

## Theorems

---

- **Theorem of Memory Hierarchy**: "The performance of a memory hierarchy is determined by the size of its components and the order in which they are accessed."
- **Theorem of Page Replacement**: "The optimal page replacement algorithm is the one that minimizes the number of page faults."
