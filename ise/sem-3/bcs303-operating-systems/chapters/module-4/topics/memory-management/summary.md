# Memory Management - Summary

## Key Definitions and Concepts

- MEMORY MANAGEMENT: The OS function controlling the allocation and deallocation of physical memory to processes
- LOGICAL ADDRESS: Address generated by CPU, relative to program's address space
- PHYSICAL ADDRESS: Actual address in RAM hardware
- MMU (Memory Management Unit): Hardware component translating logical to physical addresses
- PAGE FAULT: Exception when referenced page is not in physical memory
- SWAPPING: Moving entire processes between memory and disk

## Important Formulas and Theorems

- Logical address = Page Number × Page Size + Offset
- Physical address = Frame Number × Frame Size + Offset
- Number of pages = 2^(logical address bits - offset bits)
- Page table size = Number of entries × Entry size
- Effective access time = Hit ratio × Memory access + (1 - Hit ratio) × Page fault cost

## Key Points

- Memory management must satisfy relocation, protection, sharing, and logical organization requirements
- Contiguous allocation suffers from external fragmentation; paging eliminates external but has internal fragmentation
- Paging uses fixed-size pages and frames, enabling any page to be placed in any frame
- Segmentation provides logical organization with variable-sized segments; can be combined with paging
- Virtual memory extends physical memory using disk, allowing programs larger than RAM
- Demand paging loads pages only when referenced (lazy loading)
- FIFO is simplest replacement but may evict useful pages; LRU approximates optimal
- Thrashing occurs when excessive paging degrades performance; working set model prevents it

## Common Mistakes to Avoid

- Confusing page number with frame number in address translation calculations
- Forgetting that offset remains unchanged during address translation
- Believing optimal page replacement can be implemented (it requires future knowledge)
- Overlooking internal fragmentation in paging systems
- Not considering that page fault handling time is dominated by disk I/O

## Revision Tips

1. Practice address translation problems with different page sizes and logical address formats
2. Trace through page replacement algorithms with sample reference strings until you can do them quickly
3. Memorize the key differences between contiguous allocation, paging, and segmentation
4. Understand why thrashing occurs and how the working set model addresses it
5. Remember that offset bits = log2(page size) and remaining bits are for page number
