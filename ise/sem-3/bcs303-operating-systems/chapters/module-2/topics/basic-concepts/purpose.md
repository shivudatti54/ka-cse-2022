# Learning Objectives

After studying this topic, you should be able to:

1. Define a process and explain its components including program code, stack, heap, data section, and Process Control Block
2. Describe the five process states and explain the transitions between them with appropriate examples
3. Distinguish between user-level threads and kernel-level threads, including their advantages and disadvantages
4. Explain the relationship between processes and threads in modern operating systems
5. Understand the fork-exec model of process creation in Unix-like systems
6. Analyze the context switching mechanism and compare switching overhead between processes and threads
7. Apply thread creation and synchronization concepts to solve basic concurrent programming problems
8. Evaluate the performance implications of using processes versus threads in different scenarios
