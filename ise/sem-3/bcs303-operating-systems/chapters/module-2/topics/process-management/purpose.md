# Learning Objectives

After studying this topic, you should be able to:

1. Define what a process is and explain the difference between a program and a process in operating systems

2. Identify and explain the five process states (new, ready, running, waiting, terminated) and describe valid state transitions

3. Describe the structure and components of a Process Control Block (PCB) and explain its role in process management

4. Differentiate between long-term, short-term, and medium-term schedulers and explain their functions in the scheduling hierarchy

5. Explain the process creation mechanism including parent-child relationships and the use of fork() and exec() system calls

6. Describe what context switching is, what information is saved/restored, and why it represents overhead in CPU operations

7. Analyze process behavior in real-world scenarios by tracing state transitions and understanding how operating systems manage concurrent execution
