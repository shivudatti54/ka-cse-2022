# Learning Objectives

After studying this topic, you should be able to:

1. Explain the thread lifecycle and distinguish between thread states (ready, running, blocked) in the context of scheduling decisions.

2. Analyze the various scheduling criteria including CPU utilization, throughput, turnaround time, waiting time, and response time, and evaluate trade-offs between them.

3. Compare and contrast thread scheduling with process scheduling, highlighting the advantages of thread-level scheduling in terms of context switch overhead and resource sharing.

4. Apply different scheduling algorithms (FCFS, SJF, Round Robin, Priority) to solve numerical problems and calculate average waiting time and turnaround time.

5. Evaluate the convoy effect in FCFS and starvation in priority scheduling, explaining how different algorithms address these performance issues.

6. Describe the distinction between Process Contention Scope (PCS) and System Contention Scope (SCS) in multi-threaded environments.

7. Analyze how modern operating systems implement thread scheduling, specifically explaining the Completely Fair Scheduler in Linux and priority-based scheduling in Windows.
