# Implementing File System: File System Structure

### Overview

- A file system is a collection of files and directories that are stored on a storage device
- It provides a way to organize, manage, and access files

### Key Components

- **File**: a collection of data stored in a specific format
- **Directory**: a container that holds files and subdirectories
- **Root Directory**: the topmost directory in the file system hierarchy
- **File System Structure**: the organization of files and directories

### File System Structure

- **Hierarchical Structure**: files and directories are organized in a tree-like structure
- **Nodes**: the basic units of the file system structure
- **Files**: nodes that contain data
- **Directories**: nodes that contain subdirectories and files

### File System Types

- **Simple File System**: each file has a unique identifier
- **Hierarchical File System**: files and directories are organized in a tree-like structure

### Relationships Between Components

- **Parent-Child Relationship**: a directory can contain multiple files and subdirectories
- **File-Directory Relationship**: a file is contained within a directory

### Important Formulas and Theorems

- **Disk Block Size**: the number of bytes stored on a disk block (e.g. 512 bytes)
- **Free Space**: the amount of space available on a disk

### Definitions

- **File System Interface**: a standard interface for interacting with the file system
- **File System Call**: a system call that interacts with the file system

### Important Concepts

- **File System Cache**: a cache that stores frequently accessed files
- **File System Integrity**: the ability to ensure the integrity of files and directories
