# **Operating System Structure Revision Notes**

**Definition:** An operating system (OS) is a software that manages computer hardware and software resources, providing a platform for running applications.

**Key Components:**

- **Process Management:**
  - Process creation and termination
  - Process scheduling
  - Context switching
- **Memory Management:**
  - Virtual memory
  - Page replacement algorithms
  - Memory protection
- **File System:**
  - File organization
  - File retrieval
  - File storage
- **Input/Output (I/O) Management:**
  - I/O device management
  - I/O scheduling
  - I/O buffering
- **Security:**
  - User authentication
  - Access control
  - Encryption

**Important Formulas and Definitions:**

- **Bus Structure:**
  - Address bus
  - Data bus
  - Control bus
- **Paging:**
  - Page table
  - Page size
  - Frame size
- **Multiprogramming:**
  - Process synchronization
  - Deadlock prevention
- **Threading:**
  - Thread scheduling
  - Thread synchronization

**Important Theorems:**

- **Amdahl's Law:** Limits the speedup of a parallel program
- **Golomb's Curve:** Estimates the number of processors required for a given speedup
- **Karp-Miller Algorithm:** Solves the n-Queens problem optimally

**Revision Tips:**

- Review process management and memory management concepts
- Understand the different types of file systems (e.g., hierarchical, network-based)
- Familiarize yourself with I/O management concepts
- Study security concepts and protocols (e.g., encryption, access control)

**Key Terms:**

- Multitasking
- Multiprocessing
- Multithreading
- Process
- Thread
- Kernel
- Device driver
- Boot loader
