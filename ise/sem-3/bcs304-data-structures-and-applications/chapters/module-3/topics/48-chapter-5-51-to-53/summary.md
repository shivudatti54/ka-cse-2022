# **4.8 Chapter-5: 5.1 to 5.3 - Quick Revision Notes**

## **Introduction**

- Definition: Data structures are used to organize and store data in a way that allows for efficient access, modification, and retrieval.
- Importance: Data structures are essential in programming as they enable efficient processing and analysis of large datasets.

## **5.1: Introduction to Stacks**

- Definition: A stack is a data structure that follows the Last-In-First-Out (LIFO) principle.
- Operations:
  - Push: Add an element to the top of the stack.
  - Pop: Remove an element from the top of the stack.
  - Peek: Return the top element without removing it.
- Example: Evaluating postfix expressions.

## **Formulas/Definitions/Theorems**

- Stack: A set of elements with a last-in-first-out order.
- Stack Operations:
  - Time complexity:
    - Push: O(1)
    - Pop: O(1)
    - Peek: O(1)

## **5.2: Introduction to Queues**

- Definition: A queue is a data structure that follows the First-In-First-Out (FIFO) principle.
- Operations:
  - Enqueue: Add an element to the end of the queue.
  - Dequeue: Remove an element from the front of the queue.
  - Peek: Return the front element without removing it.
- Example: Job scheduling.

## **Formulas/Definitions/Theorems**

- Queue: A set of elements with a first-in-first-out order.
- Queue Operations:
  - Time complexity:
    - Enqueue: O(1)
    - Dequeue: O(1)
    - Peek: O(1)

## **5.3: Introduction to Trees**

- Definition: A tree is a data structure composed of nodes with a root and child nodes.
- Types of Trees:
  - Binary Tree: A tree with at most two child nodes.
  - N-ary Tree: A tree with at most n child nodes.
- Operations:
  - Insert: Add a new node to the tree.
  - Delete: Remove a node from the tree.
  - Search: Find a node with a given key.

## **Formulas/Definitions/Theorems**

- Tree: A data structure composed of nodes with a root and child nodes.
- Tree Operations:
  - Time complexity:
    - Insert: O(log n)
    - Delete: O(log n)
    - Search: O(log n)
