# Learning Objectives

After studying this topic, you should be able to:

1. Define a queue data structure and explain its FIFO (First-In-First-Out) ordering principle

2. Identify and describe all fundamental queue operations: enqueue, dequeue, front, rear, isEmpty, isFull, and size

3. Implement a queue using array-based storage with proper index management

4. Analyze and handle overflow and underflow conditions in queue implementations

5. Trace through queue operations step-by-step and determine the state of the queue after sequences of operations

6. Compare and contrast queues with stacks, highlighting the key differences in element access and removal order

7. Apply queue data structures in real-world scenarios including CPU scheduling, print queues, and breadth-first search algorithms

8. Evaluate the time complexity of queue operations and understand their computational efficiency
