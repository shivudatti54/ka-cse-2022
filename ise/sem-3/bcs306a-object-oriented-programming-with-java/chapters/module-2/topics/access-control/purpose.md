# Learning Objectives

After studying this topic, you should be able to:

1. Define access control and explain its role in achieving encapsulation in object-oriented programming

2. Identify and differentiate between the four access modifiers in Java: public, private, protected, and default (package-private)

3. Apply appropriate access modifiers to class members based on design requirements and encapsulation principles

4. Analyze how access control affects visibility of members within the same package, in different packages, and in inheritance hierarchies

5. Design classes that properly encapsulate data using private variables with public getter and setter methods

6. Explain the rules governing access control when a subclass inherits from a parent class, including the rule against weakening access

7. Debug compilation errors related to inappropriate access to class members
