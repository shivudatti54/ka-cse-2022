# **Chapter 6 Summary**

### Class Fundamentals

- **Definition:** A class is a blueprint or template that defines the properties and behavior of an object.
- **Keyword:** `class`
- **Example:** `public class MyClass { ... }`

### Declaring Objects

- **Object Declaration:** `Object obj;`
- **Reference Variables:** Declare a variable to store the reference of an object, e.g. `Object ref;`
- **Initializing Reference Variables:** `ref = new MyClass();`

### Assigning Object Reference Variables

- **Assignment:** `ref = obj;`
- **Null References:** `Object obj = null;`
- **NullPointerException:** Throws an exception when trying to access or manipulate a null reference.

### Important Formulas and Definitions

- **Inheritance:** A form of polymorphism where a class inherits properties and behavior from another class.
- **Polymorphism:** The ability of an object to take on multiple forms.
- **Encapsulation:** The concept of bundling data and methods that operate on that data within a single unit.

### Theorems

- **Single Inheritance Theorem:** A class can inherit from only one parent class.
- **Method Overriding Theorem:** A subclass can provide a different implementation of a method that is already defined in its superclass.

### Key Points

- A class defines a blueprint for an object.
- Objects are created using the `new` keyword.
- Reference variables are used to store the reference of an object.
- Null references can cause `NullPointerException`.
