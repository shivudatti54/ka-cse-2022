# **Chapter 2: Object-Oriented Programming (OOP) with Java**

### Key Concepts

- **Abstraction**: Definition, Types, and Examples
  - Definition: The practice of showing only necessary information and hiding irrelevant details.
  - Types:
    - Data Abstraction
    - Functional Abstraction
  - Examples:
    - Data Hiding
    - Encapsulation
- **Three Principles of Object-Oriented Programming (OOP)**
  - **Abstraction**: Hiding implementation details and showing only necessary information.
  - **Encapsulation**: Bundling data with methods that operate on that data.
  - **Inheritance**: Creating a new class based on an existing class.
- **Object-Oriented Programming Paradigms**
  - **Imperative Programming**: Focus on procedures and statements.
  - **Object-Oriented Programming**: Focus on objects and their interactions.

### Important Formulas, Definitions, and Theorems

- **Abstraction Theorem**: A program that exhibits abstraction will be more efficient, readable, and maintainable.
- **Encapsulation Theorem**: A program that exhibits encapsulation will be more modular, flexible, and scalable.
- **Inheritance Theorem**: A program that exhibits inheritance will be more reusable, flexible, and maintainable.

### Key Terms

- **Class**: A blueprint or template for creating objects.
- **Object**: An instance of a class, with its own set of attributes and methods.
- **Inheritance**: A relationship between classes where a subclass inherits properties and behavior from a parent class.
- **Polymorphism**: The ability of an object to take on multiple forms, depending on the context.

### Key Java Concepts

- **Constructors**: Special methods used to initialize objects.
- **Methods**: Blocks of code that perform a specific task, often used to interact with objects.
- **Attributes**: Data members of a class, which can be accessed and modified through methods.
