# Chapter 9: Packages

### Overview

- A package is a collection of related classes, interfaces, and other types.
- Packages are used to organize code and provide a way to avoid naming conflicts.

### Key Points

- **Importing Packages**:
  - Use `import` statement to bring packages into scope.
  - Can import specific classes or all classes in a package.
  - Can import packages with wildcard (`*`) or use `as` keyword.
- **Package Declaration**:
  - Use `package` keyword followed by package name.
  - Package name must be unique and follow naming conventions.
- **Member Access**:
  - Use `public`, `protected`, `default`, and `private` access modifiers.
  - Access modifiers determine the scope of class members.
- **Package Names**:
  - Follow Java naming conventions.
  - Use dots to separate package names.

### Important Formulas and Definitions

- **Package Declaration Formula**: `package packageName;`
- **Import Statement Formula**: `import packageName;` or `import packageName.*;`

### Important Theorems and Principles

- **Single Responsibility Principle (SRP)**: A class should have one reason to change.
- **Open-Closed Principle (OCP)**: A class should be open for extension but closed for modification.

### Key Terms

- **Package**: A collection of related classes, interfaces, and other types.
- **Import**: A statement that brings packages into scope.
- **Access Modifier**: A keyword that determines the scope of class members.
