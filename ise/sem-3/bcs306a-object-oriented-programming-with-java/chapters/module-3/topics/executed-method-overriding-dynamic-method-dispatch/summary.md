# **Inheritance and Object Oriented Programming in Java**

### Inheritance Basics

- Inheritance: A mechanism in which one class can inherit the properties and behavior of another class.
- Superclass (Parent): The class being inherited from.
- Subclass (Child): The class doing the inheriting.
- Inheritance Relationship: A -> B (A is-a B)

### Executed

- **Method Overriding**: When a subclass provides a different implementation of a method that is already defined in its superclass.
- **Dynamic Method Dispatch**: The method call is determined at runtime, rather than at compile time.
- **Method Overloading**: When multiple methods with the same name can be defined, but with different parameters.

### Abstract Classes and Methods

- **Abstract Class**: A class that cannot be instantiated and can only be inherited from.
- **Abstract Methods**: Methods declared in an abstract class that must be implemented by any non-abstract subclass.
- Definition: A class without a body is called abstract.

### Using Final with Inheritance

- **Final Method**: A method declared as final cannot be overridden in a subclass.
- **Final Class**: A class declared as final cannot be subclassed.
- Definition: A class or method that cannot be modified or changed once created.

### Local Variable Type Inference

- **Local Variable Type Inference (LVTI)**: A feature in Java that allows the compiler to infer the type of a local variable from its assignment.
- Definition: A feature that allows the compiler to determine the type of a variable without explicit type declaration.

### Key Formulas and Theorems

- Inheritance Relationship: A -> B (A is-a B)
- Method Overloading: Same name, different parameters
- Method Overriding: Subclass provides different implementation
- Abstract Class: Cannot be instantiated, can only be inherited from
- Final Method: Cannot be overridden
- Final Class: Cannot be subclassed
