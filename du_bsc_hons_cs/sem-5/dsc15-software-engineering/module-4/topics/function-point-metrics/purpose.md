# Learning Objectives

After studying this topic, you should be able to:

1. Define Function Point Metrics and explain its purpose and importance in software engineering.
2. Identify and classify different types of functions, including inputs, outputs, and inquiries.
3. Apply the counting rules to a software system to determine its size in function points.
4. Understand the concept of the Value Adjustment Factor and how it is applied to the raw function point count.
5. Assess the 14 General System Characteristics and determine a score for each one.
6. Use Function Point Metrics to estimate the effort required to develop a software system.
7. Explain the limitations and criticisms of Function Point Metrics, including its subjectivity and lack of precision.
8. Apply Function Point Metrics to real-world software systems and case studies.
