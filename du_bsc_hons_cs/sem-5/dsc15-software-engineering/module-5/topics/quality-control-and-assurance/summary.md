# Quality Control and Assurance — Software Engineering

## Introduction

Quality Control and Assurance (QC/QA) are fundamental aspects of software engineering that ensure software products meet specified requirements and user expectations. In the context of Delhi University's BSc (Hons) Computer Science (NEP 2024 UGCF), this topic covers the systematic processes, techniques, and standards used to deliver reliable, defect-free software.

---

## Key Concepts

- **Quality Assurance (QA)**: Process-oriented approach focused on preventing defects through systematic activities across the software development lifecycle (SDLC). Emphasizes process improvement.

- **Quality Control (QC)**: Product-oriented approach focused on identifying defects in the final product through testing and inspection. Concerned with the outcome.

- **Software Quality**: Degree to which software meets specified requirements, including functionality, reliability, efficiency, usability, maintainability, and portability.

- **Verification vs Validation**:
  - *Verification*: "Are we building the product right?" — checking specifications
  - *Validation*: "Are we building the right product?" — checking against user needs

- **Software Quality Metrics**:
  - Defect density (defects per thousand lines of code)
  - Mean Time Between Failures (MTBF)
  - Code coverage percentage
  - Cyclomatic complexity

- **Testing Methodologies**:
  - White-box testing (structural, internal logic)
  - Black-box testing (functional, input-output behavior)
  - Grey-box testing (combination)

- **Levels of Testing**:
  - Unit Testing — individual components
  - Integration Testing — component interfaces
  - System Testing — complete system
  - Acceptance Testing — user validation

- **Types of Testing**:
  - Static Testing (reviews, walkthroughs, inspections)
  - Dynamic Testing (execution-based)
  - Regression Testing (re-testing after changes)
  - Performance Testing (load, stress)

- **Software Process Improvement Models**:
  - Capability Maturity Model (CMM/CMMI)
  - ISO 9001 and ISO/IEC 25010 standards

- **Review Process**:
  - Peer Reviews and Inspections — formal technique to detect defects early
  - Walkthrough — informal group review
  - Technical Review — formal evaluation by experts

---

## Conclusion

Quality Control and Assurance are essential for delivering software that meets user requirements and industry standards. While QA focuses on preventing defects through process improvement, QC identifies defects through systematic testing. Understanding verification, validation, testing levels, and quality standards is crucial for developing reliable software and succeeding in exams.
