## Purpose & Learning Objectives  

In real‑world computer‑science tasks such as fraud detection, network‑security monitoring, and healthcare diagnostics, datasets are often heavily skewed and contain rare anomalies. Mastering the combined techniques of class balancing and outlier detection is essential for building fair, accurate, and robust machine‑learning models that perform well on minority classes while correctly identifying true anomalies.

- **Explain** the impact of class imbalance and the role of outliers in predictive modeling.  
- **Apply** class‑balancing methods (e.g., oversampling, undersampling, SMOTE) to create balanced training data.  
- **Implement** outlier‑detection algorithms (e.g., statistical tests, density‑based approaches, Isolation Forest, Autoencoders).  
- **Evaluate** model performance using metrics that address both class balance (precision, recall, F1‑score) and anomaly detection (ROC‑AUC, PR‑AUC, detection rate).  
- **Compare** the strengths and limitations of class‑balancing techniques versus outlier‑detection methods across various domains.  
- **Design** end‑to‑end pipelines that integrate class‑balancing preprocessing with anomaly‑detection modules.  
- **Analyze** ethical considerations such as bias, privacy, and the consequences of false positives when handling rare events.  
- **Utilize** popular Python libraries (e.g., imbalanced‑learn, scikit‑learn, PyOD) to prototype real‑world solutions.
