# Recursive and Recursively Enumerable Languages

## Introduction

In the study of formal language theory, we classify languages based on the computational power required to recognize them. After exploring regular languages and context-free languages, we now venture into the realm of computability theory, where we examine what can be decided by algorithmic processes. This topic introduces two fundamental classes of languages: **Recursive languages** (also called decidable languages) and **Recursively Enumerable languages** (also called Turing-recognizable or semi-decidable languages).

Understanding these language classes is crucial because they represent the boundary between what computers can and cannot do. The distinction between decidable and undecidable problems forms the foundation of theoretical computer science and has profound implications for every area of computing. As DU students preparing for competitive examinations, mastering these concepts will not only help you ace your exams but also develop a deep understanding of computational limitations that every computer scientist must appreciate.

## Key Concepts

### 1. Recursively Enumerable Languages (RE)

A language L is **recursively enumerable** (RE) or **Turing-recognizable** if there exists a Turing machine M such that:

- If w ∈ L, then M eventually halts and accepts w
- If w ∉ L, then M either rejects w or runs forever (never halts)

In other words, an RE language is one for which there exists a Turing machine that will **list out all strings in the language** or **accept every string in the language** (but may loop forever on strings not in the language). The key characteristic is that when the machine accepts, we know the answer is "yes," but we may never get an answer for strings not in the language.

**Important Properties of RE Languages:**
- If L is RE, its complement L̅ may or may not be RE
- The class of RE languages is closed under union, intersection, and concatenation
- RE languages are not closed under complementation
- Every regular language is RE
- Every context-free language is RE (since CFLs are decidable, they are also RE)

### 2. Recursive Languages (Decidable)

A language L is **recursive** (or **decidable**) if there exists a Turing machine M such that:

- If w ∈ L, then M halts and accepts w
- If w ∅ L, then M halts and rejects w

The critical difference from RE languages is that a recursive language recognizer **always halts** on every input. We get a definite "yes" or "no" answer for every string.

**Important Properties of Recursive Languages:**
- If L is recursive, then L̅ (complement) is also recursive
- The class of recursive languages is closed under union, intersection, complementation, concatenation, and Kleene star
- Every recursive language is recursively enumerable
- Every regular language is recursive
- Every context-free language is recursive (since CFL membership is decidable)

### 3. The Relationship Between RE and Recursive Languages

The fundamental relationship is: **Recursive ⊂ Recursively Enumerable**

This means every recursive language is also recursively enumerable, but the converse is not true. There exist languages that are RE but not recursive (not decidable).

### 4. Decidable Problems

A problem is **decidable** if there exists an algorithm that always halts and gives the correct answer. Key decidable problems include:

- **Emptiness Problem for Regular Languages:** Given a DFA, does it accept any strings?
- **Emptiness Problem for Context-Free Languages:** Given a CFG, does it generate any strings?
- **Membership Problem for Regular Languages:** Is a given string in the language of a DFA?
- **Membership Problem for Context-Free Languages:** Is a given string in the language of a CFG?
- **Equivalence Problem for DFAs:** Do two DFAs accept the same language?
- **Finiteness Problem:** Does a given DFA accept a finite or infinite language?

### 5. Undecidable Problems

Problems that are not recursive (not decidable) are called **undecidable**. Famous undecidable problems include:

- **Halting Problem:** Given a program and input, will the program halt?
- **Emptiness Problem for Turing Machines:** Given a TM, does it accept any strings?
- **Equivalence Problem for Turing Machines:** Do two TMs accept the same language?
- **Regularity Problem:** Is the language generated by a given TM regular?
- **Rice's Theorem:** Any non-trivial property of the language recognized by a TM is undecidable

### 6. Computation History Method

To prove undecidability, we often use the **computation history method**. A **computational history** is a sequence of configurations showing how a Turing machine processes an input. We can encode this history as a string and use it to reduce the Halting Problem to other problems, proving their undecidability.

### 7. Mapping Reducibility

A language A is **mapping reducible** to language B (A ≤ₘ B) if there exists a computable function f such that for all strings w: w ∈ A if and only if f(w) ∈ B. If A ≤ₘ B and B is decidable, then A is also decidable. Conversely, if A is undecidable and A ≤ₘ B, then B is undecidable.

## Examples

### Example 1: Proving a Language is Recursively Enumerable

**Problem:** Show that the language L = {⟨M⟩ | Turing machine M accepts at least one string} is recursively enumerable.

**Solution:**

We need to construct a Turing machine that halts and accepts if ⟨M⟩ encodes a TM that accepts at least one string. The construction:

1. Input: Encoding ⟨M⟩ of a Turing machine
2. Enumerate all possible strings over the input alphabet of M (e.g., in lexicographic order: ε, a, b, aa, ab, ba, ...)
3. For each string w in the enumeration:
   - Simulate M on w
   - If M accepts w, accept and halt
   - If M rejects w, continue to the next string

This TM will accept if and only if M accepts at least one string (i.e., M has at least one accepting computation). If M accepts no strings, this TM will run forever, which is allowed for RE languages. ∎

### Example 2: Proving a Language is Not Recursive

**Problem:** Prove that the language L = {⟨M⟩ | Turing machine M accepts all strings} is not recursive (undecidable).

**Solution using Reduction from Halting Problem:**

We prove this by contradiction. Assume L is decidable. Then there exists a TM H that decides L.

Construct a new TM D that takes input ⟨M, w⟩ (encoding of TM M and string w):

1. Construct TM N that ignores its input and simulates M on w
2. If M halts on w, N accepts its input
3. If M loops on w, N loops forever
4. Run H on ⟨N⟩
5. If H accepts (meaning N accepts all strings), then M halts on w → output "yes"
6. If H rejects (meaning N doesn't accept all strings), then M loops on w → output "no"

This construction would solve the Halting Problem, which is known to be undecidable. Therefore, our assumption must be false, and L is not recursive. ∎

### Example 3: Closure Under Complement

**Problem:** Show that if a language L is recursive, then its complement L̅ is also recursive.

**Solution:**

Since L is recursive, there exists a TM M that decides L:
- Accepts if input w ∈ L
- Rejects if input w ∅ L

To decide L̅, construct a new TM M' that:
1. Runs M on input w
2. If M accepts, reject
3. If M rejects, accept

Since M always halts, M' also always halts. If w ∈ L̅ (i.e., w ∅ L), M rejects, so M' accepts. If w ∉ L̅ (i.e., w ∈ L), M accepts, so M' rejects. Therefore, L̅ is recursive. ∎

## Exam Tips

1. **Remember the key distinction:** Recursive languages always halt (decidable), while RE languages may loop forever on rejected inputs.

2. **Closure properties are crucial:** Know which operations preserve each language class. Recursive languages are closed under all operations including complementation; RE languages are not closed under complementation.

3. **Every recursive language is RE:** But the converse is false. This is the most important relationship between these classes.

4. **The complement of RE may not be RE:** A language L is recursive if and only if both L and its complement are RE.

5. **Know Rice's Theorem:** It states that any non-trivial property of the language recognized by a TM is undecidable. This is essential for proving many problems undecidable.

6. **Reduction technique:** To prove undecidability, reduce a known undecidable problem (like Halting Problem) to the problem in question.

7. **Diagonalization argument:** Understand how diagonalization is used to prove the existence of undecidable problems.

8. **Vocabulary matters:** RE = Turing-recognizable = Semi-decidable; Recursive = Decidable = Recursive enumerability with complement also RE.

9. **Emptiness testing:** The emptiness problem for TMs is undecidable, but for DFAs and CFGs, it is decidable. This contrast frequently appears in exams.

10. **Enumeration property:** A language is RE if and only if some Turing machine can enumerate its strings in finite time.
