# Context-Free Grammars - Summary

## Key Definitions and Concepts

- **Context-Free Grammar (CFG)**: A 4-tuple G = (V, Σ, R, S) where V is non-terminals, Σ is terminals, R is production rules, S is start symbol.

- **Derivation**: Sequence of replacing non-terminals using production rules to generate a string of terminals.

- **Parse Tree**: Tree representation of a derivation with start symbol at root, terminals at leaves.

- **Ambiguous Grammar**: A grammar where some string has two or more distinct parse trees.

- **Chomsky Normal Form (CNF)**: Productions are only A → BC or A → a (plus S → ε).

- **Greibach Normal Form (GNF)**: Productions are A → aα where a is terminal and α is string of non-terminals.

## Important Formulas and Theorems

- **Equivalence**: CFLs = Languages generated by CFGs = Languages accepted by PDAs

- **Pumping Lemma for CFLs**: For any CFL L, there exists pumping length p such that any string s ∈ L with |s| ≥ p can be written as s = uvxyz where |vxy| ≤ p, |vy| > 0, and uvⁿxyⁿz ∈ L for all n ≥ 0.

- **Closure Properties**: CFLs closed under union, concatenation, Kleene star, homomorphism, inverse homomorphism; NOT closed under intersection or complementation.

## Key Points

- Every CFG generates exactly one language, but multiple grammars can generate the same language.

- Leftmost and rightmost derivations yield the same parse tree structure.

- CNF conversion requires: remove ε-productions → remove unit productions → remove useless symbols → binarize.

- A language is context-free if and only if a pushdown automaton accepts it.

- To prove non-context-freedom, use pumping lemma for CFLs—show the string cannot be pumped while staying in the language.

- Programming language syntax is typically specified using CFGs in BNF or EBNF notation.

## Common Mistakes to Remember

1. Confusing terminals and non-terminals—terminals appear in final strings, non-terminals are replaced during derivation.

2. Forgetting that ε-productions can only be removed from the start symbol in CNF.

3. In CNF conversion, ensuring the start symbol never appears on the right-hand side of productions.

4. When using pumping lemma, remember vxy can span across different symbol types or be confined to one type—both lead to contradiction.

5. Ambiguity is a property of the grammar, not the language—a language can have ambiguous and unambiguous grammars.

## Revision Tips

1. Practice CNF conversion with at least 3-4 examples to master each transformation step.

2. For ambiguity questions, try deriving the same string in multiple ways—draw the parse trees to visualize.

3. Memorize the pumping lemma statement and work through classic non-CFL proofs like aⁿbⁿcⁿ.

4. Connect theory with practice: relate CFGs to parsers in compilers and to BNF notation you've seen in language manuals.
