# Context-Free Grammars

## Introduction

A Context-Free Grammar (CFG) is a formal grammatical structure that plays a fundamental role in the theory of computation, programming language design, and compiler construction. Unlike regular grammars which can only generate regular languages, context-free grammars possess the expressive power to describe nested structures, making them essential for modeling programming language syntax and natural language syntax alike.

The significance of context-free grammars in computer science cannot be overstated. Every programming language uses some form of CFG to define its syntax—students familiar with BNF (Backus-Naur Form) notation have already encountered CFGs in disguise. The theory of context-free grammars provides the mathematical foundation for parsing algorithms, compiler design, and formal language processing. In the University of Delhi's Theory of Computation curriculum, this topic forms a critical bridge between regular languages (studied in earlier modules) and more complex computational models.

## Key Concepts

### Formal Definition of Context-Free Grammar

A context-free grammar is a 4-tuple G = (V, Σ, R, S) where:

- **V** is a finite set of variables (non-terminals) - these represent syntactic categories
- **Σ** is a finite set of terminals (alphabet) - these are the actual symbols of the language
- **R** is a finite set of production rules of the form A → α, where A ∈ V and α ∈ (V ∪ Σ)*
- **S** is the start symbol (distinguished variable), where S ∈ V

The term "context-free" refers to the fact that the left-hand side of each production rule consists of a single non-terminal, independent of the surrounding "context" in which it appears.

### Derivation Process

A derivation is a sequence of applications of production rules that transforms the start symbol into a string of terminals. Consider the grammar G where:
- V = {S, A, B}
- Σ = {a, b}
- S is the start symbol
- R includes: S → AB, A → aA | ε, B → b

The string "aab" can be derived as: S ⇒ AB ⇒ aAB ⇒ aaAB ⇒ aaB ⇒ aab

**Leftmost Derivation**: At each step, replace the leftmost non-terminal
**Rightmost Derivation**: At each step, replace the rightmost non-terminal

### Parse Trees (Derivation Trees)

A parse tree is a hierarchical tree representation of a derivation where:
- The root is the start symbol
- Each internal node is a non-terminal
- Each leaf is a terminal or ε
- If a node labeled A has children labeled X₁, X₂, ..., Xₙ, then A → X₁X₂...Xₙ is a production rule

Parse trees provide a visual representation of how a string is generated and are particularly useful in understanding ambiguity.

### Ambiguity in Grammars

A grammar is said to be **ambiguous** if there exists at least one string in the language generated by the grammar that has two or more distinct parse trees (or equivalently, two or more different leftmost derivations).

**Example**: Consider the grammar G with rules:
- S → S + S | S * S | (S) | id

The string "id + id * id" has two different parse trees, making this grammar ambiguous. This is precisely why most programming languages require unambiguous grammars for their syntax.

### Chomsky Normal Form (CNF)

A grammar is in Chomsky Normal Form if every production is one of:
1. A → BC (where B and C are non-terminals, and C ≠ S)
2. A → a (where a is a terminal)
3. S → ε (only for the start symbol)

**Conversion to CNF**:
1. Remove ε-productions (productions generating empty string)
2. Remove unit productions (A → B)
3. Eliminate useless symbols
4. Convert terminals in productions with more than two symbols to new non-terminals
5. Break productions with more than two non-terminals into binary productions

### Greibach Normal Form (GNF)

A grammar is in Greibach Normal Form if every production is of the form:
A → aα where a ∈ Σ (a terminal) and α ∈ V* (string of zero or more non-terminals)

GNF is particularly useful because it directly corresponds to the construction of pushdown automata, establishing the equivalence between CFGs and PDAs.

### Closure Properties

Context-free languages are closed under:
- Union
- Concatenation
- Kleene Star
- Homomorphism
- Inverse Homomorphism

However, they are NOT closed under intersection or complementation.

## Examples

### Example 1: Constructing a CFG for Balanced Parentheses

**Problem**: Design a CFG that generates all strings of balanced parentheses.

**Solution**:
- V = {S}
- Σ = {(, )}
- S → ( ) | ( S ) | SS | S S

**Derivation of "(()())"**:
S ⇒ SS ⇒ ( )SS ⇒ ( )( )S ⇒ ( )( )( ) ⇒ ( ( ) )( ) ⇒ ( ( ) )( )

Actually, let's use: S → ( S ) | SS | ε

S ⇒ SS ⇒ ( )S ⇒ ( )SS ⇒ ( )( )S ⇒ ( )( )( ) ⇒ ( ( ) )( ) — wait, let me redo:

S ⇒ SS ⇒ (S)S ⇒ ((S))S ⇒ (() )S ⇒ (() )ε

More clearly: S ⇒ SS ⇒ ( )SS ⇒ ( )( )S ⇒ ( )( )( ) — using S → ε

### Example 2: Converting to Chomsky Normal Form

**Problem**: Convert the following grammar to CNF:
G: S → aAb, A → aA | b | ε

**Solution**:

**Step 1**: Add new start symbol
S₀ → S
S → aAb
A → aA | b | ε

**Step 2**: Remove ε-productions
- A → ε is removed
- Find all positions where A can be ε: S → aAb becomes S → ab and S → aAb (keeping both)
- A → aA becomes A → aA | a

Current: S₀ → S, S → aAb | ab, A → aA | a | b

**Step 3**: Remove unit productions
- S₀ → S (unit) → replace S with its productions
- S₀ → aAb | ab

Now: S₀ → aAb | ab, S → aAb | ab, A → aA | a | b

**Step 4**: Remove useless symbols - none here

**Step 5**: Convert to CNF
- Create new non-terminals: X → a, Y → b
- Replace terminals in longer productions:
  - S → aAb becomes S → XAY
  - S → ab becomes S → XY
  - A → aA becomes A → XA
  - A → a becomes A → X
  - S₀ → aAb becomes S₀ → XAY
  - S₀ → ab becomes S₀ → XY

**Final CNF**:
S₀ → XAY | XY
S → XAY | XY
A → XA | X | b
X → a
Y → b

### Example 3: Proving Language is Context-Free

**Problem**: Show that L = {aⁿbⁿcⁿ | n ≥ 1} is not context-free.

**Proof using pumping lemma for CFLs**:
Assume L is context-free. By pumping lemma, there exists pumping length p.
Consider string s = aᵖbᵖcᵖ.

For any decomposition s = uvxyz with |vxy| ≤ p and |vy| > 0:
- Since |vxy| ≤ p, vxy cannot contain all three types of symbols
- Cases:
  1. vxy contains only a's and b's → uvvxxyyz contains wrong order
  2. vxy contains only b's and c's → similar problem
  3. vxy contains only one type → pumping changes count of one symbol but not others

In all cases, pumped string is not in L. Contradiction. Therefore L is not context-free.

## Exam Tips

1. **Remember the formal definition**: Be able to write a CFG as G = (V, Σ, R, S) and explain each component clearly in exams.

2. **Draw parse trees for derivations**: Many questions ask you to derive a string and draw its parse tree—practice both leftmost and rightmost derivations.

3. **CNF conversion is examinable**: Know all steps of conversion to CNF—removing ε-productions, unit productions, useless symbols, and then binarizing.

4. **Ambiguity vs unambiguity**: Be able to identify if a grammar is ambiguous and, if so, construct an unambiguous grammar for the same language when possible.

5. **Pumping lemma for CFLs**: Understand the statement and be able to apply it to prove languages are not context-free.

6. **Connection with PDA**: Remember that context-free grammars generate exactly the languages accepted by pushdown automata (this is a key equivalence proof).

7. **Applications matter**: Relate CFGs to programming language parsers and BNF notation—questions often test this application aspect.
