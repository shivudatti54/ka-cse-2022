# CFG-PDA Relationship: Theory of Computation

## Introduction

The relationship between Context-Free Grammars (CFGs) and Pushdown Automata (PDA) is fundamental in automata theory. This relationship establishes that **CFGs and PDAs are equivalent in computational power** — every context-free language can be generated by a CFG and recognized by a PDA. This equivalence is a core topic in the Delhi University BSc (Hons) Computer Science syllabus (NEP 2024 UGCF), specifically under Unit III: Pushdown Automata and Context-Free Grammars.

---

## Key Concepts

### 1. Context-Free Grammar (CFG)
- A CFG is a 4-tuple: **G = (V, Σ, R, S)**
  - **V**: Set of variables (non-terminals)
  - **Σ**: Set of terminals
  - **R**: Production rules (e.g., A → α)
  - **S**: Start symbol
- Generates languages called **Context-Free Languages (CFL)**
- Useful for describing syntax of programming languages

### 2. Pushdown Automaton (PDA)
- A PDA is a 7-tuple: **M = (Q, Σ, Γ, δ, q₀, Z₀, F)**
  - **Q**: Finite states
  - **Σ**: Input alphabet
  - **Γ**: Stack alphabet
  - **δ**: Transition function
  - **q₀**: Initial state
  - **Z₀**: Initial stack symbol
  - **F**: Final/accepting states
- Uses a **stack** for memory (LIFO structure)
- Two acceptance methods:
  - **Acceptance by Final State**: Reach a final state after reading input
  - **Acceptance by Empty Stack**: Stack becomes empty after reading input

### 3. Equivalence Theorem
- **Theorem**: A language L is context-free **iff** some PDA recognizes L
- Two parts:
  - **CFG → PDA**: Convert grammar to automaton
  - **PDA → CFG**: Convert automaton to grammar

### 4. Converting CFG to PDA
- Use **rightmost derivation** simulation
- Procedure:
  1. Create start state and final state
  2. For each production A → α, add transition: (q₀, ε, A) → (q_f, α)
  3. For each terminal a, add transition: (q₀, a, a) → (q₀, ε)
- Results in a **non-deterministic PDA**

### 5. Converting PDA to CFG
- Use **state elimination method** or **construction method**
- Steps:
  1. Create variables for each pair (p, X, q) — state X state
  2. For each transition δ(p, a, X) = {(q, γ)}, add productions
  3. Generate start variable: [q₀, Z₀, q_f] for some final state q_f
- Produces equivalent CFG

### 6. Deterministic vs Non-Deterministic PDA
- **Deterministic PDA (DPDA)**: At most one move for each state, input, stack symbol
- **NPDA** is more powerful — can recognize all CFLs
- **DPDA** recognizes **Deterministic Context-Free Languages (DCFL)**, a proper subset of CFL

---

## Conclusion

The CFG-PDA relationship is essential for understanding language recognition and generation. Since every CFG can be converted to an equivalent PDA and vice-versa, this equivalence proves that **pushdown automata precisely characterize context-free languages**. This concept is crucial for compiler design, parsing, and understanding the limitations of regular languages. For exams, remember: CFGs are **generative** devices, while PDAs are **recognitive** devices — both describe the same class of languages.
