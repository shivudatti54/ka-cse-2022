# Chomsky Normal Form (CNF)

## Introduction

Chomsky Normal Form (CNF) is a standardized form of context-free grammars that plays a fundamental role in the theory of computation, particularly in parsing algorithms and decidability problems. Named after Noam Chomsky, the renowned linguist and mathematician, CNF was developed as part of his work on formal language theory in the 1950s.

The importance of CNF in computer science cannot be overstated. It simplifies the analysis of context-free grammars and is essential for the CYK (Cocke-Younger-Kasami) parsing algorithm, which is a classic dynamic programming approach for determining whether a string can be generated by a given grammar. Additionally, CNF is crucial in proving the Cook-Levin theorem, which establishes the NP-completeness of the Boolean Satisfiability problem, connecting automata theory to computational complexity theory.

In the context of the University of Delhi's BSc (H) Computer Science curriculum, understanding CNF is essential for students preparing for competitive examinations like GATE and for those pursuing advanced studies in compilers, natural language processing, and formal verification.

## Key Concepts

### Definition of Chomsky Normal Form

A context-free grammar is said to be in Chomsky Normal Form if all its production rules satisfy one of the following conditions:

1. **A → BC** (where A, B, C are non-terminals and B, C are not the start symbol)
2. **A → a** (where a is a terminal symbol)

In other words, every production either:
- Produces exactly two non-terminals (binary productions), or
- Produces exactly one terminal (terminal productions)

The start symbol, however, may have an ε-production (S → ε) in some definitions, though many treatments exclude this to maintain strict CNF.

### Why Convert to CNF?

The conversion to CNF serves several important purposes:

1. **Simplified Grammar Structure**: CNF imposes a uniform structure on productions, making theoretical analysis straightforward.

2. **Efficient Parsing**: The CYK algorithm requires grammars to be in CNF to work efficiently with time complexity O(n³ × |G|).

3. **Theoretical Applications**: CNF is essential in proving the pumping lemma for context-free languages and in establishing the equivalence between pushdown automata and context-free grammars.

4. **Normal Form Theory**: Understanding CNF helps students appreciate the concept of normal forms in formal language theory.

### Steps for Conversion to CNF

Converting an arbitrary context-free grammar to CNF involves systematic elimination of "problematic" productions. The algorithm typically follows these steps:

**Step 1: Eliminate ε-productions**
- Identify all nullable non-terminals (those that can derive ε)
- For each production A → α, add productions where ε replaces any nullable non-terminal in α
- Remove all ε-productions (except possibly S → ε if S is nullable)

**Step 2: Eliminate unit productions**
- A unit production is of the form A → B where both A and B are non-terminals
- Find all pairs (A, B) such that A ⇒* B using only unit productions
- For each such pair, if B → β is a non-unit production, add A → β
- Remove all unit productions

**Step 3: Remove useless symbols**
- A symbol (terminal or non-terminal) is useful if it appears in some derivation of a terminal string from the start symbol
- Eliminate symbols that cannot derive any terminal string or cannot be reached from the start symbol

**Step 4: Convert to binary form**
- For productions with more than two symbols on the right-hand side, introduce new non-terminals
- Replace A → X₁X₂...Xₖ (where k > 2) with:
  - A → X₁A₁
  - A₁ → X₂A₂
  - ...
  - Aₖ₋₂ → Xₖ₋₁Xₖ
- For mixed productions (terminals and non-terminals), replace terminals in RHS with corresponding non-terminals

## Examples

### Example 1: Converting a Simple Grammar to CNF

**Given Grammar G:**
S → aSb | ε

**Step 1: Eliminate ε-productions**
- S is nullable (S ⇒ ε)
- For S → aSb, replace nullable S:
  - S → aSb | ab
- Remove S → ε
- Result: S → aSb | ab

**Step 2: Eliminate unit productions**
- Check for unit productions: None exist in S → aSb, ab

**Step 3: Remove useless symbols**
- All symbols are useful

**Step 4: Convert to binary form**
- Introduce new non-terminals:
  - Let A₁ be a new non-terminal
- S → aSb becomes S → aA₁
- A₁ → Sb
- Now replace terminal 'a' and 'b' with non-terminals:
  - Introduce B → a and C → b
  - S → B A₁
  - A₁ → S C
- Final CNF productions:
  - S → B A₁ | B C (Note: S → BC represents terminal strings)
  - A₁ → S C
  - B → a
  - C → b

### Example 2: Comprehensive CNF Conversion

**Given Grammar G:**
S → ASB | A
A → a
B → b

**Step 1: Eliminate ε-productions**
- No ε-productions exist

**Step 2: Eliminate unit productions**
- S → A is a unit production
- A → a is non-unit
- Add S → a (replacing A with its production)
- Remove S → A

**Step 3: Remove useless symbols**
- All symbols are useful

**Step 4: Convert to binary form**
- S → ASB has 3 symbols on RHS
- Introduce new non-terminal C:
  - S → A C
  - C → S B

**Step 5: Terminals in RHS with non-terminals**
- S → A C (both non-terminals, OK)
- C → S B (both non-terminals, OK)
- A → a (terminal production, OK)
- B → b (terminal production, OK)

**Final CNF:**
- S → A C
- C → S B
- A → a
- B → b

### Example 3: CYK Algorithm Application

Given grammar in CNF:
S → AB | BC
A → a
B → b
C → S

To check if string "ab" can be derived:

The CYK algorithm would construct a parsing table:
- Position 1: a ∈ L(A) → A ∈ R(1,1)
- Position 2: b ∈ L(B) → B ∈ R(2,2)

For substring (1,2):
- Check S → AB: A ∈ R(1,1) and B ∈ R(2,2) → S ∈ R(1,2)

Since S ∈ R(1,2), the string "ab" is derivable.

## Exam Tips

1. **Remember the definition**: In CNF, productions are either A → BC (two non-terminals) or A → a (single terminal). This is the most frequently tested concept.

2. **Order of elimination matters**: Always eliminate ε-productions first, then unit productions, then convert to binary form. This order prevents complications.

3. **Nullable non-terminals**: When eliminating ε-productions, make sure to add all possible combinations of productions by replacing nullable non-terminals with ε.

4. **Unit production elimination**: Use the closure algorithm to find all reachable non-terminals through unit productions before replacing them.

5. **Use of new non-terminals**: When converting to binary form, introduce new non-terminals with distinct names (like A₁, A₂, etc.) to avoid conflicts.

6. **CNF and parsing**: Remember that CNF is essential for the CYK algorithm, which is a bottom-up parsing algorithm with O(n³) time complexity.

7. **Start symbol special case**: If the start symbol is nullable, you may keep S → ε in some definitions of CNF, but this is often excluded in strict CNF.

8. **Practice conversion**: The best way to master CNF conversion is through practice. Work through at least 5-10 examples before the exam.
