### Purpose
Understanding registers and memory (RAM / ROM) is essential for any computer system, as they form the core of data storage and quick access needed for program execution and hardware control. Mastery of their organization, operation, and interaction enables efficient low‑level programming, system‑level design, and performance optimization in real‑world CS applications.

### Learning Objectives
- **Identify** the purpose, types, and size of CPU registers within the data path.  
- **Differentiate** between RAM (volatile, read‑write) and ROM (non‑volatile, read‑only) in terms of usage, speed, and technology.  
- **Explain** the internal organization of memory cells into words, banks, and addressable units.  
- **Analyze** addressing modes and memory‑mapping techniques used in microprocessor systems.  
- **Design** a basic memory interface circuit that connects RAM/ROM to a CPU bus.  
- **Evaluate** the impact of memory hierarchy (registers → RAM → cache) on overall system performance.  
- **Implement** simple read/write operations for registers and memory using assembly language.
