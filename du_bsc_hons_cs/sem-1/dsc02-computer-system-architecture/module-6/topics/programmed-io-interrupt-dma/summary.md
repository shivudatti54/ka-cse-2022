# Programmed I/O, Interrupt, and DMA - Summary

## Key Definitions and Concepts

*   **Programmed I/O**: A technique where the CPU directly controls the data transfer process.
*   **Interrupt**: A signal sent by a device to the CPU, indicating that it requires attention.
*   **DMA (Direct Memory Access)**: A technique that allows devices to transfer data directly to and from memory without CPU intervention.
*   **Polling**: A technique used in programmed I/O where the CPU continuously checks the status of a device.
*   **Busy-waiting**: A technique used in programmed I/O where the CPU waits in a loop until a device is ready for data transfer.
*   **Interrupt Service Routine (ISR)**: A routine that handles interrupts and returns control to the CPU.
*   **DMA Controller**: A hardware component that manages DMA operations.

## Important Formulas and Theorems

*   **CPU Utilization**: The percentage of time the CPU spends executing tasks.
*   **Data Transfer Rate**: The rate at which data is transferred between devices.

## Key Points

*   Programmed I/O is a technique where the CPU directly controls the data transfer process.
*   Interrupts allow devices to interrupt the CPU when they require attention.
*   DMA enables devices to transfer data directly to and from memory without CPU intervention.
*   Polling and busy-waiting are techniques used in programmed I/O.
*   ISRs handle interrupts and return control to the CPU.
*   DMA controllers manage DMA operations.
*   CPU utilization and data transfer rates are important metrics for evaluating system performance.
*   Programmed I/O, interrupt-driven I/O, and DMA have trade-offs in terms of system performance and CPU utilization.

## Common Mistakes to Avoid

*   Confusing programmed I/O with interrupt-driven I/O.
*   Failing to understand the role of DMA controllers in managing DMA operations.
*   Not considering the trade-offs between programmed I/O, interrupt-driven I/O, and DMA.
*   Incorrectly implementing programmed I/O, interrupt-driven I/O, and DMA in code examples.
