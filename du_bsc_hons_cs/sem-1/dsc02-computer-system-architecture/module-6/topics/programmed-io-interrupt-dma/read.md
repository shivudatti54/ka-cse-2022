# Programmed I/O, Interrupt, and DMA

## Introduction

Computer systems use various techniques to manage data transfer between devices. Programmed I/O, Interrupt, and DMA (Direct Memory Access) are three fundamental methods used to facilitate communication between the CPU, memory, and input/output devices. Understanding these concepts is crucial for designing and developing efficient computer systems.

Programmed I/O involves the CPU directly controlling the data transfer process, which can be time-consuming and inefficient. Interrupts allow devices to interrupt the CPU when they require attention, improving system responsiveness. DMA enables devices to transfer data directly to and from memory without CPU intervention, significantly enhancing system performance.

## Key Concepts

### Programmed I/O

*   The CPU is responsible for managing the data transfer process.
*   The CPU executes a program that controls the transfer of data between devices.
*   Programmed I/O can be performed using two techniques:
    *   **Polling**: The CPU continuously checks the status of a device to determine if it is ready for data transfer.
    *   **Busy-waiting**: The CPU waits in a loop until the device is ready for data transfer.

### Interrupt

*   An interrupt is a signal sent by a device to the CPU, indicating that it requires attention.
*   When an interrupt occurs, the CPU temporarily suspends its current activity and executes an interrupt service routine (ISR).
*   The ISR handles the interrupt and returns control to the CPU.
*   Interrupts can be classified into two types:
    *   **Hardware interrupts**: Generated by hardware devices, such as keyboard presses or disk completion.
    *   **Software interrupts**: Generated by software, such as system calls or exceptions.

### DMA

*   DMA allows devices to transfer data directly to and from memory without CPU intervention.
*   DMA improves system performance by reducing CPU utilization and increasing data transfer rates.
*   DMA controllers are used to manage DMA operations and ensure that data is transferred correctly.

## Examples

### Example 1: Programmed I/O using Polling

Suppose we want to transfer data from a keyboard to a computer using programmed I/O with polling. The CPU continuously checks the keyboard status to determine if a key has been pressed.

```c
while (1) {
    if (keyboard_status == KEY_PRESSED) {
        // Read data from keyboard
        data = keyboard_read();
        // Process data
        process_data(data);
    }
}
```

### Example 2: Interrupt-Driven I/O

Consider a scenario where a disk completion interrupt occurs. The CPU executes an ISR to handle the interrupt.

```c
void disk_completion_isr() {
    // Read data from disk
    data = disk_read();
    // Process data
    process_data(data);
    // Return control to CPU
    return;
}
```

### Example 3: DMA Transfer

Suppose we want to transfer data from a network interface card (NIC) to memory using DMA. The DMA controller manages the data transfer process.

```c
// Initialize DMA controller
dma_init();

// Set DMA transfer parameters
dma_set_source(NIC_ADDRESS);
dma_set_destination(MEMORY_ADDRESS);
dma_set_transfer_size(TRANSFER_SIZE);

// Start DMA transfer
dma_start();
```
