# Program Interrupt Cycle

## Introduction

The Program Interrupt Cycle is a fundamental concept in computer system architecture that enables the processor to respond to asynchronous events efficiently. In the Delhi University BSc (Hons) Computer Science NEP 2024 UGCF syllabus, this topic covers how the CPU suspends the current program execution to handle urgent requests from peripheral devices, ensuring effective CPU utilization and real-time response capabilities.

## Key Concepts

### What is an Interrupt?
- An interrupt is a signal to the CPU indicating an event that requires immediate attention
- Allows the CPU to switch from the currently executing program to a special routine called Interrupt Service Routine (ISR)
- Enables efficient handling of I/O operations without continuous polling

### Types of Interrupts

**Based on Source:**
- **Hardware Interrupts**: Generated by external devices (keyboard, mouse, printer, disk drive)
- **Software Interrupts**: Generated by programs executing (system calls, divide by zero, overflow)

**Based on Maskability:**
- **Maskable Interrupts**: Can be temporarily ignored by the CPU (using Interrupt Flag)
- **Non-Maskable Interrupts (NMI)**: Cannot be ignored; used for critical events like power failure

### Interrupt Cycle Phases

The processor executes the following steps during an interrupt cycle:

1. **Interrupt Detection**: CPU finishes current instruction and checks for interrupt signal
2. **Interrupt Acknowledge**: CPU sends acknowledgment to the interrupting device
3. **Save Current State**: Processor saves the current program counter (PC) and status flags onto the stack
4. **Fetch ISR Address**: CPU obtains the address of the Interrupt Service Routine from interrupt vector table
5. **Execute ISR**: Control transfers to the ISR to handle the interrupting event
6. **Restore State**: After ISR completion, CPU restores the saved PC and status from the stack
7. **Resume Execution**: CPU returns to the interrupted program from the exact point of interruption

### Important Terminology

- **Interrupt Service Routine (ISR)**: Special program executed in response to an interrupt
- **Interrupt Vector Table**: Memory structure containing addresses of ISRs for different interrupt sources
- **Interrupt Latency**: Time between interrupt request and start of ISR execution
- **Interrupt Priority**: Mechanism to handle multiple simultaneous interrupts
- **Interrupt Nesting**: Allowing higher priority interrupts to preempt currently executing ISR

### Interrupt-Driven vs. Polling

| Aspect | Interrupt-Driven | Polling |
|--------|------------------|---------|
| CPU Utilization | High | Low |
| Response Time | Fast | Slow |
| Hardware Complexity | More | Less |

## Conclusion

The Program Interrupt Cycle is essential for modern computer systems as it enables efficient CPU utilization, real-time event handling, and seamless I/O operations. Understanding interrupt mechanisms, types, and handling procedures is crucial for computer architecture examinations and practical system design. Mastery of this topic helps students comprehend how CPUs manage concurrent operations and respond to asynchronous events effectively.
