### Purpose  
Understanding Memory, Register, and I/O instructions is fundamental to low‑level software development, because these instructions define how the CPU moves data among registers, main memory, and peripheral devices. Mastery of this topic enables students to write efficient system code, develop device drivers, design embedded systems, and appreciate the performance trade‑offs that arise from different I/O addressing strategies.

### Learning Objectives  
- **Explain** the role of memory, register, and I/O instructions within the CPU’s instruction‑execution cycle.  
- **Describe** the various addressing modes used for memory and I/O operations.  
- **Differentiate** between memory‑mapped I/O and isolated (port‑mapped) I/O architectures.  
- **Analyze** the impact of different I/O instructions on code density, execution speed, and hardware complexity.  
- **Design** simple assembly language sequences that transfer data between registers, memory, and I/O ports.  
- **Implement** basic interrupt‑driven I/O handling using appropriate instructions.  
- **Evaluate** the trade‑offs of employing specific I/O instructions in terms of performance and resource usage.  
- **Compare** instruction encodings for memory and I/O operations across typical processor families.
