# NIST Architecture Case Study — Cloud Computing

## Introduction

The National Institute of Standards and Technology (NIST) provides the definitive reference architecture for cloud computing, widely adopted in academic curricula including the Delhi University BSc (Hons) Computer Science NEP 2024 UGCF syllabus. This case study establishes standardized definitions and frameworks essential for understanding cloud computing deployment and service models.

## Key Concepts

### Essential Characteristics
- **On-demand self-service**: Users can provision computing resources automatically without human intervention
- **Broad network access**: Services accessible via standard mechanisms across multiple platforms
- **Resource pooling**: Multi-tenant model with dynamic resource allocation
- **Rapid elasticity**: Capabilities that can be quickly scaled out/in
- **Measured service**: Pay-per-use model with resource monitoring

### Service Models (NIST Defined)
- **Infrastructure as a Service (IaaS)**: Virtualized computing resources (servers, storage, networks); Example: AWS EC2, Azure VMs
- **Platform as a Service (PaaS**: Development platforms with built-in tools; Example: Google App Engine, Heroku
- **Software as a Service (SaaS)**: Complete applications delivered over the internet; Example: Google Workspace, Salesforce

### Deployment Models
- **Public Cloud**: Services offered to the general public over the internet
- **Private Cloud**: Dedicated infrastructure for a single organization
- **Hybrid Cloud**: Combines public and private cloud with data portability
- **Community Cloud**: Shared infrastructure for specific communities

### NIST Reference Architecture Components
- **Cloud Consumer**: End-users and organizations using cloud services
- **Cloud Provider**: Entities offering cloud services (storage, processing, applications)
- **Cloud Broker**: Intermediaries managing cloud service usage and performance
- **Cloud Carrier**: Telecommunications providers enabling network access
- **Cloud Auditor**: Independent assessors of security, privacy, and compliance

### Key Security Considerations
- Data protection and privacy
- Identity and access management
- Compliance with regulatory frameworks
- Shared responsibility model across service layers

## Conclusion

The NIST cloud computing architecture provides a standardized vocabulary and framework essential for understanding modern cloud systems. For Delhi University students, mastering these concepts is crucial as cloud computing forms a core component of the NEP 2024 syllabus, with direct applications in enterprise computing and technology deployment.
