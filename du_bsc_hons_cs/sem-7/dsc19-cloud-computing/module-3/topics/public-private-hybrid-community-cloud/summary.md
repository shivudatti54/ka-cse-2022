## Introduction

Cloud computing delivers computing resources (servers, storage, databases, networking, software) over the internet. The National Education Policy (NEP) 2024 under UGCF emphasizes understanding deployment models as fundamental to cloud literacy. The four primary cloud deployment models—Public, Private, Hybrid, and Community—each address distinct organizational requirements regarding control, security, scalability, and cost.

---

## Deployment Models

### Public Cloud
- **Definition:** Cloud services owned and operated by third-party providers (e.g., AWS, Microsoft Azure, Google Cloud)
- **Characteristics:** 
  - Resources shared across multiple organizations (multi-tenant)
  - Pay-as-you-go pricing
  - Minimal infrastructure management for users
- **Advantages:** Scalability, cost-efficiency, low maintenance
- **Disadvantages:** Less control, security concerns for sensitive data

### Private Cloud
- **Definition:** Cloud infrastructure dedicated to a single organization
- **Characteristics:**
  - Can be hosted on-premises or by a third party
  - Enhanced security and privacy
  - Customizable resources
- **Advantages:** Greater control, data sovereignty, compliance readiness
- **Disadvantages:** Higher costs, requires IT expertise

### Hybrid Cloud
- **Definition:** Combines public and private cloud environments
- **Characteristics:**
  - Data and applications shared between models
  - Orchestrated via automation and orchestration tools
- **Advantages:** 
  - Flexibility — sensitive workloads in private cloud, scalable workloads in public
  - Cost optimization
  - Business continuity and disaster recovery
- **Examples:** AWS Outposts, Azure Arc, Google Anthos

### Community Cloud
- **Definition:** Shared infrastructure for a specific community (e.g., healthcare, government, education)
- **Characteristics:**
  - Managed by community members or third-party provider
  - Addresses common regulatory and compliance needs
- **Advantages:** Cost-sharing, tailored compliance, collaborative environment
- **Disadvantages:** Limited scalability, governance complexity

---

## Key Differences at a Glance

| Feature | Public | Private | Hybrid | Community |
|---------|--------|---------|--------|-----------|
| Ownership | Third-party | Single organization | Multiple | Specific community |
| Security | Moderate | High | Variable | High (domain-specific) |
| Scalability | High | Limited | High | Moderate |
| Cost | Low | High | Medium | Shared |

## Conclusion

Understanding these deployment models is essential for cloud architects and developers. Delhi University's NEP 2024 syllabus emphasizes selecting the appropriate model based on business requirements—balancing security, scalability, cost, and compliance. Hybrid cloud is increasingly favored in enterprise environments for its flexibility, while community cloud supports collaborative domains like education and healthcare.
