## Purpose  
In today’s hyper‑connected world, every software system is a potential target for cyber‑threats, making security literacy essential for any computer scientist. This topic equips you to recognize, analyze, and respond to common attacks, thereby protecting information assets and enabling the design of resilient, trustworthy systems.

## Learning Objectives
- **Identify** the core security concepts (confidentiality, integrity, availability) and their significance in system design.  
- **Explain** the taxonomy of security threats and attack vectors such as malware, phishing, ransomware, and DDoS.  
- **Analyze** the lifecycle of a typical cyber‑attack, including reconnaissance, exploitation, persistence, and data exfiltration.  
- **Differentiate** between vulnerabilities, threats, and risks, and illustrate how they interrelate in risk assessment.  
- **Evaluate** the effectiveness of common security controls (e.g., firewalls, IDS/IPS, encryption) against specific attacks.  
- **Design** basic security policies and countermeasures for a given system or application.  
- **Apply** risk assessment frameworks (e.g., CVSS, STRIDE) to prioritize security mitigation actions.
