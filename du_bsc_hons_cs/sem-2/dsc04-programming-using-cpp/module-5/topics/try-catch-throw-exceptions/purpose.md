# Learning Objectives

After studying this topic, you should be able to:

1. Explain the concept of exception handling in C++ and distinguish it from traditional error handling techniques like return codes and error flags.

2. Identify the three essential components of C++ exception handling—try, catch, and throw—and describe their roles in the exception handling mechanism.

3. Write C++ programs that use try-catch blocks to handle runtime errors gracefully, including division by zero, out-of-range access, and invalid input.

4. Implement custom exception classes by deriving from standard exception classes and overriding the `what()` method.

5. Analyze the C++ standard exception hierarchy and select appropriate exception types for different error conditions.

6. Understand stack unwinding and explain how local objects are destroyed when exceptions propagate through the call stack.

7. Apply best practices for exception handling including throwing by value, catching by reference, and proper ordering of catch blocks.

8. Evaluate the use of the `noexcept` specifier and explain its implications on function behavior and program termination.
