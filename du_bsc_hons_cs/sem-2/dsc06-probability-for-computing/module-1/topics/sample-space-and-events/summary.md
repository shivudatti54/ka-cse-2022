# Sample Space and Events - Summary

## Key Definitions and Concepts

- **Random Experiment**: A procedure with multiple possible outcomes that cannot be predicted with certainty before execution.
- **Sample Space (S)**: The complete set of all possible outcomes of a random experiment.
- **Event**: A subset of the sample space; a collection of outcomes we are interested in.
- **Simple Event**: Contains exactly one outcome.
- **Compound Event**: Contains more than one outcome.
- **Impossible Event**: The empty set ∅.
- **Certain Event**: The entire sample space S.

## Important Formulas and Theorems

- **Classical Probability**: P(A) = |A| / |S|, where |A| and |S| are the number of outcomes in event A and sample space S (equally likely outcomes only)

- **Complement Rule**: P(A') = 1 - P(A)

- **Addition Rule**: P(A ∪ B) = P(A) + P(B) - P(A ∩ B)

- **Mutually Exclusive**: P(A ∪ B) = P(A) + P(B) when A ∩ B = ∅

- **De Morgan's Laws**:
  - (A ∪ B)' = A' ∩ B'
  - (A ∩ B)' = A' ∪ B'

## Key Points

- Always define the sample space first before solving any probability problem
- Sample spaces can be finite, countably infinite, or uncountable (continuous)
- Events are sets; use proper set notation (∈, ⊂, ∪, ∩, ∅)
- For continuous sample spaces (intervals), probability equals the ratio of interval lengths
- "At least one" = 1 - "none" — use complements for easier calculation
- Venn diagrams are essential tools for visualizing relationships between events
- Events A and B are mutually exclusive if A ∩ B = ∅

## Common Mistakes to Avoid

1. **Using P(A) = |A|/|S] when outcomes are not equally likely** — always verify equal likelihood first.

2. **Forgetting to subtract the intersection** when calculating P(A ∪ B) for non-mutually exclusive events.

3. **Confusing "at least" with "exactly"** — read questions carefully.

4. **Ignoring the sample space** when events are defined implicitly; always write S explicitly.

## Revision Tips

1. Practice constructing sample spaces for at least 5 different computing scenarios (random numbers, network packets, algorithm outcomes).

2. Draw Venn diagrams for all problems involving two or more events.

3. Memorize the complement rule — it simplifies many "at least" problems significantly.

4. Solve previous year DU exam questions on this topic to understand the question patterns and marking scheme.
