## Purpose & Learning Objectives

### Purpose

Functional dependencies form the theoretical foundation for **relational database design**, enabling practitioners to systematically eliminate data redundancy, prevent update anomalies, and ensure **data integrity** in real-world software systems. Mastery of this topic equips students with the analytical skills required to design robust, scalable databases that meet industry standards for efficiency and reliability.

### Learning Objectives

- **Define** the concept of functional dependencies and explain their significance in relational database theory
- **Identify** functional dependencies from given attributes and real-world scenarios
- **Apply** Armstrong's axioms (reflexivity, augmentation, transitivity) to derive closure sets
- **Compute** attribute closures and determine candidate keys using functional dependencies
- **Distinguish** between trivial and non-trivial dependencies, and recognize transitive dependencies
- **Normalize** relational schemas to Third Normal Form (3NF) and Boyce-Codd Normal Form (BCNF) using dependency analysis
- **Evaluate** database designs for dependency preservation and lossless-join properties
- **Design** normalized database schemas that minimize redundancy while maintaining data integrity
