**Purpose**  
Understanding how to define and apply constraints when creating tables is fundamental to building reliable, maintainable database systems. Mastery of DDL constraints ensures data integrity, enforces business rules, and supports robust relationships between tables—skills essential for any real‑world software development or data‑administration role.

**Learning Objectives**  
- **Define** the core integrity constraints (PRIMARY KEY, FOREIGN KEY, UNIQUE, NOT NULL, CHECK, DEFAULT).  
- **Create** tables using SQL DDL statements that incorporate each type of constraint.  
- **Apply** referential integrity actions (CASCADE, SET NULL, RESTRICT) to manage related data.  
- **Identify** the appropriate constraint for a given business rule and justify the choice.  
- **Enforce** domain integrity by specifying data‑type restrictions and custom CHECK conditions.  
- **Modify** existing table constraints through ALTER TABLE statements.  
- **Design** normalized table structures that leverage constraints to minimize redundancy.  
- **Demonstrate** error handling for constraint violations and explain how they preserve data consistency.
