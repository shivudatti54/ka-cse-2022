## Purpose
Sorting is a fundamental operation in computer science, underpinning data organization, search efficiency, and numerous algorithmic applications. Understanding **Heapsort** and **linear‑time sorting** techniques equips students with the knowledge to choose the most appropriate algorithm based on input size, key range, and stability requirements, thereby optimizing real‑world software performance.

## Learning Objectives
- **Explain** the heap data structure and how it enables the Heapsort algorithm.  
- **Implement** Heapsort in a programming language, ensuring correct heapify and build‑heap operations.  
- **Analyze** the time‑ and space‑complexity of Heapsort and compare it with O(n log n) comparison‑based sorts.  
- **Describe** linear‑time sorting methods (counting sort, radix sort, bucket sort) and their assumptions about input data.  
- **Compare** the performance, stability, and memory usage of Heapsort versus linear‑time sorts under various constraints.  
- **Apply** the appropriate sorting algorithm based on data characteristics (e.g., integer keys, bounded range, distribution).  
- **Evaluate** the practical implications of algorithm choice in real‑world scenarios such as database indexing and competitive programming.  
- **Design** a hybrid sorting strategy that leverages Heapsort and linear‑time sorts to achieve optimal average‑case performance.
