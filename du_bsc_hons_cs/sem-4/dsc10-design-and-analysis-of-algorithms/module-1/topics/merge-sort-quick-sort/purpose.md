Sorting is a cornerstone operation in virtually every software system, enabling efficient data retrieval, organization, and analysis. Understanding merge sort and quick sort equips students with the skills to choose, implement, and optimise sorting solutions that meet real‑world performance constraints.

**Learning Objectives**

- Explain the divide‑and‑conquer paradigm underlying merge sort and quick sort.  
- Analyse the best, average, and worst‑case time and space complexities of both algorithms.  
- Compare the stability, memory usage, and performance characteristics of merge sort and quick sort.  
- Implement both sorting algorithms in a high‑level programming language.  
- Select the appropriate sorting algorithm based on input size, data distribution, and system constraints.  
- Evaluate the impact of partitioning strategies and optimisation techniques on quick‑sort’s performance.
