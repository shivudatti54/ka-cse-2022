# Learning Objectives

After studying this topic, you should be able to:

1. Define the single-source shortest path problem and explain its significance in graph theory applications.

2. Describe Dijkstra's algorithm's greedy approach and explain why the greedy choice property holds for non-negative edge weights.

3. Trace through Dijkstra's algorithm step-by-step on a given weighted graph and determine shortest path distances from source to all vertices.

4. Implement Dijkstra's algorithm using a binary min-heap priority queue and analyze its time and space complexity.

5. Perform the relaxation operation correctly and explain how it updates tentative distances.

6. Compare Dijkstra's algorithm with Bellman-Ford algorithm in terms of correctness, time complexity, and handling of negative edge weights.

7. Identify the limitations of Dijkstra's algorithm and explain scenarios where alternative algorithms like Bellman-Ford must be used.

8. Apply Dijkstra's algorithm to solve real-world problems such as network routing and transportation networks.
