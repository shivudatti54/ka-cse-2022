### Purpose
Understanding UDP, TCP, and IP flow control is essential for designing reliable, efficient network applications and for troubleshooting performance issues in real‑world computer systems. Mastery of these protocols equips students with the skills to develop robust client–server solutions and to optimise data transmission over the Internet.

### Learning Objectives
- **Explain** the differences between UDP and TCP in terms of communication models and reliability.
- **Describe** the mechanisms of IP flow control and how they prevent network congestion.
- **Compare** the sliding‑window approach of TCP with the congestion‑control algorithms (e.g., slow start, congestion avoidance).
- **Analyze** the impact of packet loss, latency, and throughput on UDP and TCP streams.
- **Differentiate** between flow control (receiver‑driven) and congestion control (network‑driven) in TCP.
- **Identify** appropriate use cases for UDP versus TCP in distributed applications.
- **Implement** a simple client–server program using both UDP and TCP to demonstrate flow‑control behavior.
