# Learning Objectives

After studying this topic, you should be able to:

1. Explain the major services provided by an operating system to users and application programs.

2. Describe the purpose and functionality of process management, memory management, file management, and device management services.

3. Define system calls and explain their role as the interface between user programs and the operating system kernel.

4. Distinguish between user mode and kernel mode, and explain how system calls achieve the transition between these modes.

5. Categorize system calls into their functional types: process control, file management, device management, information maintenance, communication, and protection.

6. Demonstrate practical usage of common system calls like fork(), exec(), open(), read(), write(), close(), and pipe() with working code examples.

7. Compare and contrast Application Programming Interfaces (APIs) with system calls, explaining the layered approach to operating system services.

8. Explain the different methods of parameter passing to system calls and analyze their advantages and limitations.
