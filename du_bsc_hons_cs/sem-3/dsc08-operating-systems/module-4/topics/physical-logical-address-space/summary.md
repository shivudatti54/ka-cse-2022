# Physical and Logical Address Space - Summary

## Key Definitions and Concepts

- **Logical Address Space**: The range of memory addresses that a program can use, generated by the CPU. Each process has its own independent logical address space.

- **Physical Address Space**: The actual memory locations in RAM hardware. Limited by the amount of installed RAM and shared by all processes in the system.

- **Address Binding**: The process of mapping logical addresses to physical addresses. Can occur at compile time, load time, or execution time.

- **Memory Management Unit (MMU)**: Hardware component that translates logical addresses to physical addresses and enforces memory protection.

- **Base Register**: Holds the starting physical address of a process in memory.

- **Limit Register**: Specifies the size of a process's allocated memory region for protection.

- **Paging**: Memory management technique using fixed-size pages and frames, eliminating external fragmentation.

- **Segmentation**: Memory management technique dividing logical space into variable-sized segments based on program structure.

## Important Formulas and Theorems

- **Base-Limit Translation**: Physical Address = Base Register + Logical Address
- **Paging**: Page Number = Logical Address / Page Size; Offset = Logical Address % Page Size
- **Paging Translation**: Physical Address = (Frame Number × Page Size) + Offset
- **Address Validity**: Logical Address must be < Limit Register for valid access
- **32-bit addressing**: Maximum logical address space = 2³² = 4 GB
- **64-bit addressing**: Maximum logical address space = 2⁶⁴ (practically unlimited)

## Key Points

- CPU always generates logical addresses; physical addresses are what RAM hardware uses
- Address binding can be static (compile/load time) or dynamic (execution time)
- MMU performs real-time translation of every memory reference made by CPU
- Base and limit registers provide simple but effective memory protection
- Paging uses fixed-size pages (typically 4KB) and eliminates external fragmentation
- Segmentation reflects logical program structure (code, data, stack segments)
- Page tables map virtual pages to physical frames; each entry stores frame number and control bits
- Translation Lookaside Buffer (TLB) caches recent page table entries for faster translation
- Each memory management technique has trade-offs between fragmentation, overhead, and flexibility

## Common Mistakes to Avoid

1. **Confusing logical and physical addresses**: Remember logical addresses are generated by CPU, physical addresses are actual RAM locations

2. **Incorrect page offset calculation**: Make sure to use the correct number of bits for offset based on page size (e.g., 4KB page = 12 bits for offset)

3. **Forgetting protection checks**: Always verify offset < limit before performing address translation in segmentation

4. **Ignoring unit conversions**: Be consistent with units (bytes vs. words) when solving numerical problems

5. **Not showing work in exam problems**: Step-by-step solutions earn partial credit even if final answer is incorrect
