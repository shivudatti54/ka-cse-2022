# Physical and Logical Address Space

## Introduction

Memory management is one of the most critical functions of any operating system, and understanding the distinction between physical and logical address spaces forms the foundation of this domain. In modern computer systems, the CPU always generates logical addresses (also called virtual addresses), while the actual memory locations in RAM are referred to as physical addresses. This separation is fundamental to achieving process isolation, memory protection, and efficient utilization of available memory through techniques like virtual memory.

The concept of address spaces becomes particularly important when multiple processes run concurrently on a system. Each process believes it has access to the entire range of memory addresses, from zero up to some maximum, but in reality, the operating system maps these logical addresses to physical memory locations that may be scattered throughout the RAM. This mapping is handled by a hardware component called the Memory Management Unit (MMU), which transparently translates every memory reference made by a running program.

For students preparing for DU semester examinations, a thorough understanding of address binding, the role of MMU, and the various memory allocation techniques is essential. This topic regularly appears in both theoretical questions and problem-solving sections of the Operating Systems paper, carrying significant weightage.

## Key Concepts

### Logical Address Space

A logical address space represents the range of addresses that a program can use to reference memory locations. When a program is written in a high-level language like C or Java, it references variables, arrays, and function calls using symbolic names. The compiler and linker convert these symbolic references into numerical addresses, which constitute the logical address space of that program.

The logical address space is process-specific — each running process has its own logical address space, completely independent of other processes. This isolation is crucial for system stability and security, as it prevents one process from accidentally or maliciously accessing the memory of another process. The size of the logical address space depends on the system's architecture; for example, a 32-bit system typically allows each process to address up to 4 GB of memory (2³² addresses), while a 64-bit system supports a virtually unlimited address space.

The logical address generated by the CPU is also called a virtual address because it does not necessarily correspond to an actual physical location in RAM. The CPU includes a segment selector or page table base register that helps the MMU in translating these virtual addresses to physical addresses.

### Physical Address Space

Physical addresses refer to the actual hardware memory locations in the computer's RAM chips. These are the real addresses where data is actually stored and retrieved. The physical address space is limited by the amount of installed RAM in the system; for instance, a computer with 8 GB of RAM has a physical address space of approximately 8 billion addressable locations.

Unlike logical addresses which are process-relative, physical addresses are system-wide. All processes share the same physical memory, and the operating system must carefully manage this shared resource to ensure fair allocation and prevent conflicts. The physical address space is typically smaller than the logical address space, especially in systems that support virtual memory, where part of the logical address space may be mapped to secondary storage (like hard disk) when RAM is full.

Physical addresses are what the memory hardware (RAM chips) actually understands. When the CPU issues a memory read or write instruction with a physical address, the data is transferred directly from or to the specified location in RAM.

### Address Binding

Address binding is the process of mapping logical addresses to physical addresses. This binding can occur at three different stages, each with its own implications:

**Compile Time Binding**: If the memory location where a process will be loaded is known in advance, the compiler can generate absolute code with physical addresses. This method is simple but inflexible — if the chosen memory location is occupied, the process cannot be loaded.

**Load Time Binding**: The compiler generates relocatable code, and the loader (part of the OS) binds logical addresses to physical addresses when the process is loaded into memory. Once loaded, the process cannot be moved to another location without reloading.

**Execution Time Binding**: Also called dynamic address translation, this allows a process to be moved between different memory locations during execution. This is the most flexible approach and is used in modern operating systems. The binding is performed dynamically by the MMU as each memory reference is made.

### Memory Management Unit (MMU)

The Memory Management Unit is a specialized hardware component that translates logical addresses to physical addresses on every memory access. It works in conjunction with the operating system to provide memory protection and efficient allocation.

In a simple implementation using base and limit registers, the MMU adds the base register value (which holds the starting physical address of the current process) to the logical address generated by the CPU. The limit register ensures that the resulting address does not exceed the allocated memory region for that process. If an attempt is made to access memory outside the allowed range, the MMU triggers a memory protection fault.

Modern MMUs support more sophisticated techniques like paging and segmentation. They maintain translation lookaside buffers (TLBs) to cache recent address translations, significantly speeding up the translation process. The MMU also enforces memory protection by checking access permissions (read, write, execute) for each memory page.

### Base and Limit Registers

Base and limit registers are the simplest form of memory management hardware. The base register holds the starting physical address of the current process in memory, while the limit register specifies the size of the process's allocated memory region.

When the CPU generates a logical address, the MMU performs the translation by adding the base register value to the logical address. It then checks whether this physical address is within the bounds specified by the limit register. If the address is valid, the memory access proceeds; otherwise, a trap to the operating system is generated.

This scheme provides two key benefits: it enables memory protection (processes cannot access memory outside their allocated region) and it allows multiple processes to be loaded at different physical locations while each believes it is loaded at address zero. The operating system updates the base register when switching between processes, making this scheme transparent to user programs.

### Paging

Paging is a memory management technique that eliminates the need for contiguous physical memory allocation. The logical address space is divided into fixed-size blocks called pages, while the physical memory is divided into frames of the same size. The operating system maintains a page table that maps each page to its corresponding frame in physical memory.

When the CPU generates a logical address, it is divided into two parts: the page number (which indexes into the page table) and the offset (the position within that page). The MMU uses the page number to look up the corresponding frame number in the page table, then combines this frame number with the offset to produce the physical address.

Paging offers several advantages: it eliminates external fragmentation (holes between allocated memory blocks), allows memory to be allocated in non-contiguous blocks, and simplifies memory allocation for the operating system. However, it introduces internal fragmentation (unused space within allocated pages) and requires additional memory for storing page tables.

### Segmentation

Segmentation is a memory management technique that divides a program's logical address space into variable-sized segments based on logical divisions like code, data, stack, and heap. Each segment has a name (or number) and a length, and logical addresses are specified as a combination of segment name and offset within that segment.

Unlike paging, which is invisible to the programmer, segmentation reflects the program's logical structure. A programmer can think in terms of distinct code segment, data segment, and stack segment, rather than a flat array of bytes. This makes programs easier to understand, debug, and modify.

The MMU uses a segment table to translate segment:offset pairs to physical addresses. Each entry in the segment table contains a base address (starting physical location) and a limit (segment length). The MMU adds the offset to the base address after verifying that the offset is within the limit. This provides both translation and protection — each segment can have its own access permissions.

## Examples

### Example 1: Base and Limit Register Translation

Consider a system where a process is loaded at physical address 10000 with a limit of 4000 bytes. If the CPU generates the logical address 1500, calculate the physical address and determine if the access is valid.

**Solution:**

Physical Address = Base Register + Logical Address
= 10000 + 1500
= 11500

To check if access is valid:
Logical Address < Limit Register?
1500 < 4000? YES

Therefore, the physical address is 11500, and the memory access is valid.

If the CPU had generated logical address 4500:
Physical Address = 10000 + 4500 = 14500
Valid? 4500 < 4000? NO

This would trigger a memory protection fault, and the operating system would terminate the process.

### Example 2: Paging Address Translation

A system uses paging with page size of 1024 bytes. The page table for a process is as follows:

| Page Number | Frame Number | Valid |
|-------------|--------------|-------|
| 0           | 3            | 1     |
| 1           | 5            | 1     |
| 2           | 1            | 1     |
| 3           | 7            | 1     |

For the logical address 2700, determine:
(a) The page number and offset
(b) The physical address

**Solution:**

Given: Page size = 1024 bytes = 2¹⁰, so offset requires 10 bits

(a) Logical Address = 2700
Page Number = Logical Address / Page Size = 2700 / 1024 = 2 (integer division)
Offset = Logical Address % Page Size = 2700 % 1024 = 652

So, Page Number = 2, Offset = 652

(b) From the page table, Page 2 maps to Frame 1

Physical Address = (Frame Number × Page Size) + Offset
= (1 × 1024) + 652
= 1024 + 652
= 1676

Therefore, logical address 2700 maps to physical address 1676.

### Example 3: Segmentation Translation

A process has the following segment table:

| Segment | Base     | Limit    | Permissions |
|---------|----------|----------|-------------|
| 0       | 20000    | 1000     | Read/Execute|
| 1       | 15000    | 2000     | Read/Write  |
| 2       | 35000    | 1500     | Read/Write  |

For each of the following segment:offset pairs, find the physical address and determine if access is permitted:
(a) (1, 500)
(b) (2, 1600)
(c) (0, 500)

**Solution:**

(a) Segment 1, Offset 500
Check: Offset < Limit? 500 < 2000? YES
Physical Address = Base + Offset = 15000 + 500 = 15500
Permissions: Read/Write — access permitted for read/write operation

(b) Segment 2, Offset 1600
Check: Offset < Limit? 1600 < 1500? NO
This is an invalid access — offset exceeds segment limit.
The MMU would generate a fault, and the OS would handle it.

(c) Segment 0, Offset 500
Check: Offset < Limit? 500 < 1000? YES
Physical Address = Base + Offset = 20000 + 500 = 20500
Permissions: Read/Execute — access permitted for read or execute, but not write

## Exam Tips

1. **Understand the fundamental difference**: Remember that the CPU always generates logical addresses, while physical addresses refer to actual RAM locations. This distinction is crucial for exams.

2. **Know all three stages of address binding**: Compile time, load time, and execution time binding — understand when each is used and their trade-offs.

3. **MMU function is essential**: The Memory Management Unit is the hardware component that performs address translation. Be clear about its role in memory protection and address translation.

4. **Base and limit registers**: This is the simplest form of address translation. Practice problems involving base and limit register calculations as they frequently appear in exams.

5. **Paging calculations**: Be comfortable calculating page numbers and offsets from logical addresses, and translating them to physical addresses using page tables. Remember that page size determines how bits are split.

6. **Segmentation vs. paging**: Understand the key differences — segmentation reflects logical program structure with variable-sized segments, while paging uses fixed-size pages and is transparent to programmers.

7. **Address space sizes**: Know the address space sizes for common architectures (32-bit = 4GB, 64-bit = practically unlimited) and understand why this matters for virtual memory.

8. **Protection mechanisms**: Understand how MMU enforces memory protection through limit checks (segmentation) and access rights (pages/segments).

9. **Internal vs. external fragmentation**: Paging suffers from internal fragmentation, while contiguous allocation suffers from external fragmentation — this is a common exam question.

10. **Write neat step-by-step solutions**: In exam problems, always show your calculation steps clearly, including how you split logical addresses into page number and offset.
