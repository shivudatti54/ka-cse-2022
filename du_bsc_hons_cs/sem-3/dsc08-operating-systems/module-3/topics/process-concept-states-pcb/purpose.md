# Learning Objectives

After studying this topic, you should be able to:

1. **Define and distinguish** between a program and a process, explaining why a process is considered an active entity while a program is passive.

2. **Identify and explain** the five fundamental process states (New, Ready, Running, Blocked, Terminated) and the conditions that cause transitions between them.

3. **Draw and interpret** the process state transition diagram, clearly showing all possible state changes and their triggering events.

4. **Describe the complete structure** of a Process Control Block (PCB), including all its components and the purpose of each field.

5. **Explain the context switching mechanism**, including what information is saved/restored and why context switching incurs overhead.

6. **Analyze process scheduling scenarios** by computing turnaround time, waiting time, and other performance metrics using Gantt charts.

7. **Differentiate between ready queue and blocked queue**, understanding which processes reside in each and how the OS manages these queues.

8. **Apply theoretical concepts** to real-world operating system behavior, explaining observed phenomena like program hangs, slow response, and multitasking.
