# Learning Objectives

After studying this topic, you should be able to:

1. Define a queue as an abstract data type and explain the FIFO (First-In-First-Out) principle that governs its behavior.

2. List and describe the primary operations performed on a queue: enqueue, dequeue, front, isEmpty, and size.

3. Differentiate between various types of queues including linear queue, circular queue, priority queue, and double-ended queue (deque).

4. Implement a queue using array-based (both linear and circular) and linked list approaches, and analyze the time and space complexity of each operation.

5. Identify and explain the problems of "queue overflow" and "queue underflow" and describe conditions to detect these states in different implementations.

6. Apply the circular queue concept to efficiently utilize fixed-size array space through modular arithmetic.

7. Demonstrate the practical applications of queues in computer science, such as CPU scheduling, breadth-first search (BFS), and print job spooling.

8. Compare and contrast the different queue implementations, selecting the appropriate implementation based on given requirements.
