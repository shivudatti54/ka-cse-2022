# Learning Objectives

After studying this topic, you should be able to:

1. Define the properties of a Binary Search Tree (BST) and explain how it maintains ordered data.

2. Analyze the time complexity of search, insert, and delete operations in a BST for best, average, and worst-case scenarios.

3. Explain why the height of the tree (h) is the determining factor for BST operation complexity and express this relationship mathematically.

4. Compare and contrast balanced versus unbalanced BSTs and their respective performance characteristics.

5. Trace through the deletion operation for all three cases: leaf node, single child, and two children.

6. Identify scenarios where BST performance degrades to O(n) and explain how this impacts practical applications.

7. Calculate the exact number of comparisons needed for specific search operations in trees of given sizes.
