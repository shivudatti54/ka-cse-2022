### Learning Purpose: Continuous Internal Evaluation

**1. Why is this topic important?**
Continuous Internal Evaluation (CIE) is crucial because it shifts assessment from a single high-stakes exam to an ongoing process. This provides real-time feedback on student comprehension, identifies learning gaps early, and allows for timely corrective action. It fosters a more engaged and responsive learning environment, mirroring the iterative feedback loops central to modern software development practices like Agile and DevOps.

**2. What will students learn?**
Students will learn the principles and methodologies behind designing and implementing a CIE framework. This includes formulating various assessment instruments (e.g., quizzes, assignments, presentations, mini-projects), establishing rubrics for objective grading, and analyzing results to measure learning outcomes effectively. They will understand how to use CIE data to improve both teaching and learning strategies.

**3. How does it connect to other concepts?**
CIE is directly connected to core software engineering concepts. It embodies the iterative feedback principle of **Agile methodology** and is a foundational practice in **DevOps** for continuous improvement. Effectively managing CIE processes also requires solid **project management** skills to schedule, track, and evaluate multiple assessments throughout the semester, ensuring alignment with overall course objectives.

**4. Real-world applications**
The principles of CIE are directly applied in the professional world through **continuous integration/continuous deployment (CI/CD)** pipelines, where code is constantly tested and evaluated. Furthermore, project managers use similar ongoing evaluation techniques to monitor team performance, assess project health, and make data-driven decisions to keep projects on track, on budget, and within scope.
