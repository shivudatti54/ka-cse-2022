# Learning Objectives
After studying this topic, you should be able to:
1.  Define the Waterfall Model and describe its sequential, non-iterative nature.
2.  List and describe the purpose of each of the six distinct phases in the Waterfall Model.
3.  Identify the key advantages of using the Waterfall Model for software development.
4.  Analyze and explain the major disadvantages and risks associated with the Waterfall Model.
5.  Evaluate a given project scenario to determine if the Waterfall Model is an appropriate choice.
6.  Compare and contrast the Waterfall Model with iterative and Agile methodologies based on flexibility, customer involvement, and risk.
