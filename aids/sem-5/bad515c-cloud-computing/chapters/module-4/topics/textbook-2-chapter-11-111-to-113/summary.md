# **Cloud Security: Revision Notes**

## **Chapter 11: Cloud Security**

### 11.1: Cloud Security Concerns

- Key concerns for cloud users:
  - Data privacy and security
  - Data ownership and control
  - Compliance and governance
  - Risk management
- Cloud security risks:
  - Insider threats
  - Third-party risks
  - Malicious insiders
  - Physical security threats

### 11.2: Risks in Cloud Computing

- Types of risks:
  - Confidentiality risk
  - Integrity risk
  - Availability risk
  - Compliance risk
- Risk assessment methods:
  - Risk scoring models
  - Fault tree analysis
  - Decision trees
  - Probabilistic risk assessments

### 11.3: Privacy Impact Assessment

- Definition: The process of evaluating the potential privacy impacts of cloud computing
- Steps:
  1.  Identify potential data collection points
  2.  Determine the data types and purposes
  3.  Evaluate the risks of data breaches and unauthorized access
  4.  Assess the impact on data subjects
  5.  Develop mitigation strategies
- Important formula:
  - Risk = (Threat x Vulnerability x Exposure) / Protective Controls

**Definitions**

- **Cloud Security**: The practice of protecting cloud computing resources and data from unauthorized access, use, disclosure, disruption, modification, or destruction.
- **Cloud Security Threat**: A potential threat to cloud security, including insider threats, third-party risks, and physical security threats.

**Theorems**

- **Cloud Security Principle**: The principle that cloud security should be designed and implemented to protect data and applications from unauthorized access, use, disclosure, disruption, modification, or destruction.
