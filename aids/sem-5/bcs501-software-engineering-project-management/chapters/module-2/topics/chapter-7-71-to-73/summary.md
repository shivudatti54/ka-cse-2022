# **Software Engineering & Project Management Revision Notes**

## **Chapter 7: 7.1 to 7.3**

### 7.1: Requirements Engineering

- **Definition:** The process of identifying, analyzing, documenting, and managing functional and non-functional requirements
- **Key Concepts:**
  - Functional requirements
  - Non-functional requirements (e.g., performance, security)
  - Requirements management
  - Requirements validation and verification
- **Important Formulas/Definitions:**
  - Requirements Engineering Matrix
- **Key Theorems:**
  - The law of diminishing returns in requirements engineering

### 7.2: Software Design

- **Definition:** The process of creating a detailed plan for the structure and behavior of a software system
- **Key Concepts:**
  - System design patterns
  - Architectural styles
  - Design trade-offs
- **Important Formulas/Definitions:**
  - KISS principle (Keep it Simple, Stupid)
  - YAGNI principle (You Ain't Gonna Need It)
- **Key Theorems:**
  - The Law of Demeter

### 7.3: Software Construction

- **Definition:** The process of translating the software design into a working software product
- **Key Concepts:**
  - Programming paradigm
  - Object-Oriented Programming (OOP) principles
  - Software development lifecycle
- **Important Formulas/Definitions:**
  - Waterfall model
  - Agile methodology
- **Key Theorems:**
  - The proof of correctness for finite automata
