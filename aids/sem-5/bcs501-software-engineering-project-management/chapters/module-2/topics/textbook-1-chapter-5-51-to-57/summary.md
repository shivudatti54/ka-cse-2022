# **Textbook 1: Chapter 5: 5.1 to 5.7 Revision Notes**

## **5.1 Introduction to Software Project Management**

- Definition: Software project management is the application of knowledge, skills, and techniques to execute a software project successfully.
- Key concepts:
  - Project scope, schedule, budget, quality, and resources
  - Project management lifecycle (initiation, planning, execution, monitoring and control, closure)

## **5.2 Project Scope Statement**

- Definition: A formal document that defines the project scope, including what is included and excluded.
- Key elements:
  - Project goals and objectives
  - Deliverables and outcomes
  - Constraints and assumptions

## **5.3 Work Breakdown Structure (WBS)**

- Definition: A hierarchical decomposition of the project scope into smaller, manageable tasks.
- Key elements:
  - Level 1: Project scope statement
  - Level 2: Major deliverables
  - Level 3: Tasks and subtasks

## **5.4 Project Schedule**

- Definition: A plan that outlines the sequence of tasks and their dependencies.
- Key elements:
  - Task duration and dependencies
  - Start and finish dates
  - Milestones and deadlines

## **5.5 Resource Allocation**

- Definition: The process of assigning resources (people, materials, equipment) to tasks.
- Key elements:
  - Resource types (human, material, equipment)
  - Resource allocation models (fixed-price, time-and-materials)

## **5.6 Project Cost Management**

- Definition: The process of estimating, authorizing, and controlling project costs.
- Key elements:
  - Cost estimation techniques (bottom-up, top-down)
  - Cost control techniques (cost accounting, budgeting)

## **5.7 Project Quality Management**

- Definition: The process of ensuring that the project meets the required quality standards.
- Key elements:
  - Quality planning and assurance
  - Quality control and monitoring
  - Quality metrics and measurement
