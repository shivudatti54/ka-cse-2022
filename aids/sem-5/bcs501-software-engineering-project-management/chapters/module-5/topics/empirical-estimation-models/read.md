# **Empirical Estimation Models**

## **Introduction**

Empirical estimation models are a type of estimation technique that uses historical data to estimate the effort required for a software project. These models are widely used in software engineering and project management due to their ability to provide accurate estimates and reduce uncertainty.

## **Definition**

Empirical estimation models are based on the idea that past experience can be used to predict future project outcomes. These models take into account various factors such as project characteristics, team experience, and historical data to estimate the effort required for a project.

## **Types of Empirical Estimation Models**

### 1. **Delta Model**

The Delta Model is a simple empirical estimation technique that uses a historical baseline effort to estimate the effort required for a new project. The model calculates the delta (Δ) between the baseline effort and the new project effort.

**Formula:**

Δ = (New Project Effort - Baseline Effort) / Baseline Effort

**Example:**

Let's say we have a historical baseline effort for a project estimated at 1000 man-hours. We want to estimate the effort required for a new project with 1200 man-hours.

Δ = (1200 - 1000) / 1000 = 0.2

This means that the new project effort is 20% higher than the baseline effort.

### 2. **I-Factor Model**

The I-Factor Model is a more advanced empirical estimation technique that takes into account the number of people working on the project (I) and the amount of time available for the project (F). The model estimates the effort required for the project as follows:

Effort = (I x F) / (Team Efficiency x Baseline Effort)

**Example:**

Let's say we have a team of 10 people with an available time of 120 days. The team efficiency is 80% (based on historical data). The baseline effort for the project is 800 man-hours.

Effort = (10 x 120) / (0.8 x 800) = 15 man-hours

### 3. **Three-Point Estimation**

Three-Point Estimation is a simple empirical estimation technique that uses a range of effort estimates to account for uncertainty. The model estimates the effort required for the project as follows:

Effort = (Low Estimate x 0.6 + High Estimate x 0.4) / 2

**Example:**

Let's say we have a low estimate of 1000 man-hours and a high estimate of 1200 man-hours.

Effort = (1000 x 0.6 + 1200 x 0.4) / 2 = 1100 man-hours

## **Key Concepts**

- **Historical data**: Data collected from past projects to estimate future project outcomes.
- **Team characteristics**: Factors such as team size, experience, and skills that affect project effort.
- **Project characteristics**: Factors such as project size, complexity, and scope that affect project effort.
- **Effort estimation**: The process of estimating the amount of work required to complete a project.
- **Uncertainty**: The degree of uncertainty associated with an effort estimate.

## **Best Practices**

- **Use historical data**: Use data from past projects to estimate future project outcomes.
- **Consider team characteristics**: Take into account team size, experience, and skills when estimating project effort.
- **Use multiple models**: Use multiple empirical estimation models to account for uncertainty and improve accuracy.
- **Regularly review and update estimates**: Regularly review and update effort estimates as new information becomes available.

## **Conclusion**

Empirical estimation models are a powerful tool for estimating effort in software projects. By using historical data and considering various factors, these models can provide accurate estimates and reduce uncertainty. By following best practices and using multiple models, software engineers and project managers can improve the accuracy of effort estimates and make more informed decisions.
