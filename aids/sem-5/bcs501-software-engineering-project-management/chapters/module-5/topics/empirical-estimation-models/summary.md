# **Empirical Estimation Models**

**Definition:** Empirical Estimation Models are statistical methods used to estimate the effort and duration required for a software development project.

**Key Points:**

- **Types of Empirical Estimation Models:**
  - Historical Data Analysis
  - Expert Judgment
  - Analogous Task Estimation
- **Historical Data Analysis:**
  - Uses past project data to estimate future project effort
  - Formula: Effort Estimation = (Sum of Effort of Similar Tasks) / (Number of Similar Tasks)
- **Expert Judgment:**
  - Involves estimating effort based on experts' opinions and experience
  - Formula: Effort Estimation = Expert's Judgment
- **Analogous Task Estimation:**
  - Estimates effort based on the similarity between two tasks
  - Formula: Effort Estimation = (Effort of Analogous Task) / (Similarity Index)

**Important Formulas:**

- **Markowitz' Formula:** (Sum of Effort of Similar Tasks) / (Number of Similar Tasks)
- **Pareto Law:** 80% of effort comes from 20% of tasks

**Important Definitions:**

- **Similarity Index:** A measure of the similarity between two tasks
- **Effort:** The amount of work required to complete a task

**Important Theorems:**

- **Law of Large Numbers:** The average of a large number of independent and identically distributed random variables will be close to the population mean.

**Revision Tips:**

- Understand the strengths and weaknesses of each empirical estimation model
- Be able to apply the formulas and definitions to real-world scenarios
- Practice estimating effort using historical data, expert judgment, and analogous task estimation
