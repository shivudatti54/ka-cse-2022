# **Project Planning and Quality Management in Software Development**

### Key Concepts

- **Project Planning:**
  - Definition: The process of defining project scope, goals, and timelines.
  - Phases: Feasibility study, requirement gathering, design, development, testing, deployment, and maintenance.
  - Techniques: Gantt charts, PERT diagrams, SWOT analysis.
- **Quality Management:**
  - Definition: The process of ensuring software meets the required standards and specifications.
  - Quality Models: ISO 9001, CMMI, Six Sigma.
- **Software Quality Metrics:**
  - Definition: Quantifiable measures of software quality.
  - Examples: Defect density, test coverage, code complexity.

### Quality Management Theories

- **ISO 9001:**
  - Definition: International standard for quality management.
  - Key Principles: Customer focus, leadership, engagement, process approach, improvement.
- **CMMI:**
  - Definition: Capability Maturity Model Integration.
  - Key Stages: Initiation, Planning, Execution, Monitoring and Control, Continuous Process Improvement.

### Quality Management Techniques

- **Agile Methodologies:**
  - Definition: Iterative and incremental software development approach.
  - Key Principles: Flexibility, collaboration, continuous improvement.
- **Test-Driven Development (TDD):**
  - Definition: Software development process that relies on the repetitive cycle of writing automated tests before writing the actual code.
  - Benefits: Improved code quality, reduced bugs, faster development.

### Quality Metrics and Models

- **Defect Density:**
  - Definition: The number of defects per unit of code.
  - Formula: Defect density = (number of defects / total lines of code).
- **Test Coverage:**
  - Definition: The percentage of code covered by tests.
  - Formula: Test coverage = (number of tested lines / total lines of code).

### Important Formulas and Theorems

- **Buckingham-Osgood Theorem:**
  - Definition: Theorem that relates the project duration to the critical path.
  - Formula: Project duration = (maximum path length) x (number of activities).
- **PMBOK (Project Management Body of Knowledge):**
  - Definition: Guide to the project management body of knowledge.
  - Key Processes: Initiating, Planning, Executing, Monitoring and Controlling, Closing.
