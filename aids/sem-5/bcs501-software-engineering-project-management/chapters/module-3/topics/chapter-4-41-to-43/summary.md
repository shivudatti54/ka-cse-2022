# **Revision Notes: Chapter 4.1-4.3**

## **Software Engineering & Project Management**

### 4.1: Project Management Fundamentals

- **Definition:** Project management is the application of knowledge, skills, and techniques to execute projects effectively.
- **Project Management Life Cycle:** Initiation, Planning, Execution, Monitoring and Control, Closing.
- **Key Concepts:**
  - Project scope, schedule, budget, quality
  - Project stakeholders, risks, issues
  - Project management frameworks (e.g. PMBOK, Agile)

### 4.2: Project Scope and Scheduling

- **Definition:** Project scope refers to the range of tasks, products, and services that a project should deliver.
- **Scope Statement:** A written document that defines project scope.
- **Scheduling:**
  - Gantt charts
  - Critical Path Method (CPM)
  - Programme Evaluation and Review Technique (PERT)
- **Key Concepts:**
  - Task duration, dependencies, slack
  - Resource allocation, leveling

### 4.3: Cost Management

- **Definition:** Cost management is the process of planning, organizing, and controlling project costs.
- **Cost Management Plan:** A document that outlines project cost management responsibilities.
- **Cost Types:**
  - Fixed, variable, semi-variable
  - Direct, indirect, overhead
- **Key Concepts:**
  - Cost estimation, budgeting, cost control
  - Earned Value Management (EVM)

### Important Formulas and Definitions

- **Earned Value:** EV = (AC - BC) / CV
- **Cost Variance:** CV = (AC - EC)
- **Schedule Variance:** SV = (AC - PS)
- **Definition of Scope:** The set of all tasks, products, and services that a project should deliver.

### Important Theorems

- **Project Management Triangle:** Scope, Schedule, Budget
- **Law of Diminishing Returns:** As the size of the project increases, the cost of adding more resources decreases.
