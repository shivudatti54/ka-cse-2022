# **Revision Notes: Textbook 1, Chapter 3**

## **3.1 Software Development Life Cycle (SDLC)**

- SDLC: A systematic approach to software development
- Phases: Requirements, Analysis, Design, Implementation, Testing, Deployment, Maintenance
- Benefits: Improved quality, increased efficiency, and reduced costs

## **3.2 Phases of SDLC**

- **Requirements**: Gathering and documenting user needs
- **Analysis**: Identifying and evaluating alternatives
- **Design**: Creating a detailed design document
- **Implementation**: Writing the code
- **Testing**: Verifying the software meets requirements
- **Deployment**: Releasing the software to users
- **Maintenance**: Updating and modifying the software

## **3.3 Types of SDLC Models**

- **Waterfall**: Sequential and linear approach
- **Agile**: Iterative and flexible approach
- **V-Model**: Combines traditional and agile approaches

## **3.4 Software Requirements**

- **Functional Requirements**: What the software should do
- **Non-Functional Requirements**: How the software should behave
- **User Requirements**: Requirements from the end-users

## **3.5 Software Design**

- **Design Patterns**: Reusable solutions to common problems
- **Object-Oriented Design**: Organizing code into objects and classes

## **3.6 Software Testing**

- **Types of Testing**: Unit testing, Integration testing, System testing
- **Testing Techniques**: Black box testing, White box testing, Gray box testing

## **Important Formulas and Definitions**

- **Definition of Software Engineering**: The application of engineering principles to the development of software systems.
- **Definition of Project Management**: The planning, organization, and control of resources to achieve a specific goal.
