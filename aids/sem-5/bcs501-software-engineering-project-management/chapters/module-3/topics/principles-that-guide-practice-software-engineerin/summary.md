# Software Engineering Knowledge

### Overview

This module covers the principles that guide practice in software engineering. Key concepts include design patterns, software metrics, quality metrics, testing, and software reuse.

### Key Concepts

- **Design Patterns**: Reusable solutions to common software design problems
  - Creational patterns (e.g. Singleton, Factory)
  - Structural patterns (e.g. Adapter, Bridge)
  - Behavioral patterns (e.g. Observer, Strategy)
- **Software Metrics**: Quantitative measures of software quality
  - Coupling metrics (e.g. Cohesion, Coupling)
  - Complexity metrics (e.g. Halstead's metrics)
  - Maintainability metrics (e.g. WMC, DIT)
- **Quality Metrics**: Measures of software quality that support decision-making
  - Functionality metrics (e.g. FMM, FMO)
  - Usability metrics (e.g. User Satisfaction, Effort)
- **Testing**: Validation of software through experimentation
  - Black box testing (e.g. Equivalence partitioning)
  - White box testing (e.g. Statement coverage)
  - Grey box testing (e.g. Path coverage)
- **Software Reuse**: Reducing redundant work through modular design
  - Extract method
  - Extract class
  - Inheritance

### Important Formulas and Theorems

- **Halstead's metrics**: Measures of software complexity
  - N = Number of operators
  - K = Number of operands
  - M = Number of statements
  - df = Difficulty
  - Nf = Effort
- **WMC (Wieser-Milgram-Coleman) metric**: Measures of maintainability
  - M = Number of operations
  - e = Effort
  - C = Complexity
  - N = Number of nodes

### Definitions

- **Coupling**: The degree to which modules depend on each other
- **Complexity**: The degree to which a module is difficult to understand or modify
- **Maintainability**: The ease with which a module can be modified or updated

### Important Theorems

- **Duff's Law**: "It always takes longer than you expect, even when you take into account Duff's Law"
- **The Law of Demeter**: "Every method should have at most two arguments"
