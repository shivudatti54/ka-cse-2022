# Principles that Guide Each Framework Activity

## **Introduction**

In software engineering and project management, frameworks provide a structured approach to delivering projects efficiently. Each framework activity is guided by a set of principles that ensure the project is completed successfully, on time, and within budget. In this module, we will delve into the principles that guide each framework activity, exploring their historical context, modern developments, and applications.

## **Agile Framework Principles**

Agile is a popular framework that emphasizes flexibility, collaboration, and iterative development. The principles that guide Agile framework activities include:

### 1. **Individuals and Interactions**

- **Why:** In Agile, individuals and interactions are prioritized over processes and tools.
- **Example:** A team works together to complete a task, rather than relying on a single individual to complete it.
- **Diagrams:** The Agile manifesto highlights the importance of individuals and interactions.

  |                              |                                         |
  | ---------------------------- | --------------------------------------- |
  | Individuals and Interactions | Prioritize individuals and interactions |
  |                              |                                         |

### 2. **Working Software**

- **Why:** The primary goal of Agile is to deliver working software, rather than following a rigid process.
- **Example:** A team delivers a working prototype of a new feature, which is then tested and refined.
- **Diagrams:** The Agile manifesto emphasizes the importance of working software.

  |                  |                          |
  | ---------------- | ------------------------ |
  | Working Software | Deliver working software |
  |                  |                          |

### 3. **Customer Collaboration**

- **Why:** Agile emphasizes collaboration between customers and developers to ensure that the software meets the customer's needs.
- **Example:** A customer provides feedback on a new feature, which is then incorporated into the software.
- **Diagrams:** The Agile manifesto highlights the importance of customer collaboration.

  |                        |                            |
  | ---------------------- | -------------------------- |
  | Customer Collaboration | Collaborate with customers |
  |                        |                            |

### 4. **Responding to Change**

- **Why:** Agile is adaptable to change, allowing teams to respond quickly to new requirements or priorities.
- **Example:** A new requirement is added to the project, which is then incorporated into the software.
- **Diagrams:** The Agile manifesto emphasizes the importance of responding to change.

  |                      |                           |
  | -------------------- | ------------------------- |
  | Responding to Change | Respond to change quickly |
  |                      |                           |

### 5. **Simplicity**

- **Why:** Agile emphasizes simplicity, allowing teams to deliver software quickly and efficiently.
- **Example:** A team uses a simple, iterative approach to deliver a new feature.
- **Diagrams:** The Agile manifesto highlights the importance of simplicity.

  |            |                         |
  | ---------- | ----------------------- |
  | Simplicity | Deliver software simply |
  |            |                         |

## **Kanban Framework Principles**

Kanban is a popular framework that emphasizes visual management, continuous improvement, and flow. The principles that guide Kanban framework activities include:

### 1. **Visualization**

- **Why:** Kanban emphasizes visualization, allowing team members to see the entire project workflow and track progress.
- **Example:** A team uses a Kanban board to display the project workflow and track progress.
- **Diagrams:** The Kanban framework emphasizes visualization.

  |               |                        |
  | ------------- | ---------------------- |
  | Visualization | Visualize the workflow |
  |               |                        |

### 2. **Flow**

- **Why:** Kanban emphasizes flow, allowing team members to work on tasks in a continuous and smooth manner.
- **Example:** A team works on tasks in a continuous flow, rather than relying on a single individual to complete tasks.
- **Diagrams:** The Kanban framework emphasizes flow.

  |      |                           |
  | ---- | ------------------------- |
  | Flow | Work in a continuous flow |
  |      |                           |

### 3. **Pull**

- **Why:** Kanban emphasizes the pull principle, allowing team members to pull tasks into the workflow when they are ready.
- **Example:** A team pulls a task into the workflow when they have the necessary resources and capacity.
- **Diagrams:** The Kanban framework emphasizes the pull principle.

  |      |                              |
  | ---- | ---------------------------- |
  | Pull | Pull tasks into the workflow |
  |      |                              |

### 4. **Limit WIP**

- **Why:** Kanban emphasizes limiting the number of tasks in the workflow (WIP), allowing team members to focus on a manageable number of tasks.
- **Example:** A team limits the number of tasks in the workflow to five, allowing them to focus on a manageable number of tasks.
- **Diagrams:** The Kanban framework emphasizes limiting WIP.

  |           |                           |
  | --------- | ------------------------- |
  | Limit WIP | Limit the number of tasks |
  |           |                           |

## **Waterfall Framework Principles**

Waterfall is a popular framework that emphasizes linear development, phase-by-phase completion, and a rigid process. The principles that guide Waterfall framework activities include:

### 1. **Sequential Development**

- **Why:** Waterfall emphasizes sequential development, allowing teams to complete each phase before moving on to the next one.
- **Example:** A team completes each phase of the project in a linear sequence, rather than working on multiple phases simultaneously.
- **Diagrams:** The Waterfall framework emphasizes sequential development.

  |                        |                              |
  | ---------------------- | ---------------------------- |
  | Sequential Development | Complete phases sequentially |
  |                        |                              |

### 2. **Phased Completion**

- **Why:** Waterfall emphasizes phased completion, allowing teams to complete each phase before moving on to the next one.
- **Example:** A team completes each phase of the project in a linear sequence, without working on multiple phases simultaneously.
- **Diagrams:** The Waterfall framework emphasizes phased completion.

  |                   |                            |
  | ----------------- | -------------------------- |
  | Phased Completion | Complete phases one by one |
  |                   |                            |

### 3. **Predictive Planning**

- **Why:** Waterfall emphasizes predictive planning, allowing teams to plan and schedule the project with a high degree of accuracy.
- **Example:** A team creates a detailed project schedule and plan, using historical data and industry benchmarks to estimate the project duration and costs.
- **Diagrams:** The Waterfall framework emphasizes predictive planning.

  |                     |                               |
  | ------------------- | ----------------------------- |
  | Predictive Planning | Plan and schedule the project |
  |                     |                               |
