# Learning Objectives
After studying this topic, you should be able to:
1. Define distributed transactions and explain how they differ from local transactions
2. Explain the ACID properties in the context of distributed systems
3. Compare and contrast flat and nested distributed transaction models
4. Describe the Two-Phase Commit protocol and its operation sequence
5. Identify different concurrency control methods used in distributed transactions
6. Explain how distributed deadlocks occur and methods to prevent or detect them
7. Describe recovery mechanisms for distributed transactions after failures
