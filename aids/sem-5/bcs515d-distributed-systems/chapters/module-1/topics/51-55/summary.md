# **5.1-5.5: CHARACTERIZATION OF DISTRIBUTED SYSTEMS**

## **Definition and Overview**

- A distributed system is a collection of interconnected nodes (computers, servers, etc.) that work together to achieve a common goal.
- Distributed systems can be classified into two main categories:
  - **Homogeneous**: All nodes are identical.
  - **Heterogeneous**: Nodes have different characteristics.

## **Types of Distributed Systems**

- **Homogeneous Distributed Systems**
  - Advantages:
    - Easy to manage and maintain
    - Simple communication
  - Disadvantages:
    - Limited scalability
    - Limited flexibility
- **Heterogeneous Distributed Systems**
  - Advantages:
    - Scalable and flexible
    - Can handle diverse workloads
  - Disadvantages:
    - Complex management
    - Difficult communication

## **Characteristics of Distributed Systems**

- **Autonomy**: Each node operates independently.
- **Organizability**: Nodes can be added or removed as needed.
- **Scalability**: Distributed systems can handle increased workload.
- **Fault Tolerance**: Distributed systems can continue to operate even if one node fails.

## **Communication in Distributed Systems**

- **Synchronous Communication**: Nodes communicate in real-time.
- **Asynchronous Communication**: Nodes communicate at different times.
- **Distributed Communication Protocols**: Standard protocols for communication, such as TCP/IP.

## **Important Formulas and Theorems**

- **Petersen's Theorem**: A necessary and sufficient condition for a system to be connected.
- **Kruskal's Theorem**: A necessary and sufficient condition for a system to be strongly connected.

## **Important Definitions**

- **Cluster**: A group of nodes that work together to achieve a common goal.
- **Peer-to-Peer**: Nodes act as both clients and servers.
- **Client-Server**: Nodes have a master node (server) and nodes that request services (clients).
