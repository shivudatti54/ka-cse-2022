# Revision Notes for Chapter 1.1: Introduction to Distributed Systems

===========================================================

### Definitions and Key Concepts

- **Distributed System**: A collection of independent computers that appear to be a single cohesive system to the user.
- **Distributed System Architecture**: The structure and organization of a distributed system.
- **Network**: A communication infrastructure that connects multiple computers.
- **Client-Server Model**: A common architecture for distributed systems, where a client requests services from a server.

### Important Formulas and Theorems

- **Paxos Algorithm**: A protocol for achieving consensus in a distributed system.
- **Raft Algorithm**: A consensus protocol for distributed systems.
- **Availability**: The ability of a system to provide services to users at any time.

### Key Theorems

- **Turing Test**: A theoretical limit on the ability of a computer to simulate human thought.
- **Chernoff Bound**: A mathematical bound on the probability of error in distributed systems.

### Important Definitions

- **Load Balancing**: The process of distributing workload across multiple servers to improve responsiveness.
- **Scalability**: The ability of a system to handle increased workload without a decrease in performance.

### Key Concepts

- **Distributed File System (DFS)**: A file system that stores and manages files across multiple computers.
- **Distributed Database**: A database that stores and manages data across multiple computers.
- **Cloud Computing**: A model for delivering computing resources over the internet.

### Important Concepts in Distributed Systems

- **Fault Tolerance**: The ability of a system to continue operating even if one or more components fail.
- **Concurrency**: The ability of a system to handle multiple requests simultaneously.
- **Asynchronous Communication**: A communication method where messages are exchanged without a specific order.
