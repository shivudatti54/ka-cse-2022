# Learning Objectives
After studying this topic, you should be able to:
1.  Identify and describe the three fundamental components of a file service architecture (Flat File Service, Directory Service, Client Module) and their respective responsibilities.
2.  Explain the purpose and advantages of using a Unique File Identifier (UFID) instead of a pathname for low-level file operations.
3.  Trace the sequence of interactions between components for a common file operation, such as opening and reading a file specified by a pathname.
4.  Compare and contrast the stateful and stateless server design approaches, listing the advantages and disadvantages of each.
5.  Differentiate between the Upload/Download and Remote Access models for file services.
6.  Describe the problem of cache coherence in distributed file systems and identify common strategies to address it.
