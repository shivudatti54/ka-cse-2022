# **13.1-13.3: Distributed Systems - Revision Notes**

## **13.1: Distributed File Systems**

- **Definition:** A distributed file system (DFS) is a system that stores and manages files across a network of nodes.
- **File Service Architecture:**
  - **Client-Server Architecture:** Client requests access to files, server provides access.
  - **Distributed File System (DFS) Architecture:** Multiple servers work together to provide access to files.
- **Key Concepts:**
  - **File System Hierarchical Structure (FSHS):** Hierarchical structure of files and directories.
  - **File System Name (FSN):** Unique identifier for each file system.

## **13.2: Distributed File System Architectures**

- **Key Architectures:**
  - **Client-Server Architecture**
  - **Peer-to-Peer Architecture**
  - **Hierarchical Architecture**
  - **Distributed File System (DFS) Architecture**
- **Characteristics:**
  - **Scalability:** Ability to handle large amounts of data.
  - **Availability:** Ability to access files from multiple locations.
  - **Fault Tolerance:** Ability to recover from node failures.

## **13.3: Distributed File System Protocols**

- **Key Protocols:**
  - **Network File System (NFS)**
  - **Server Message Block (SMB)**
  - **Open Distributed File System (ODFS)**
- **Characteristics:**
  - **Connectionless vs. Connection-Oriented:** Connectionless protocols allow multiple clients to access files simultaneously.
  - **Caching:** Ability to cache frequently accessed files to improve performance.

## **Important Formulas and Theorems:**

- **GPS (Global Positioning System) Formula:** Not applicable to DFS.
- **Theorem of Distributed Systems:** Not applicable to DFS.
