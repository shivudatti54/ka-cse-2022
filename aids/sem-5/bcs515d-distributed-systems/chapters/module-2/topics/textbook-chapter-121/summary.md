# **Distributed Systems: Chapter 12.1 Revision Notes**

### Introduction

- A distributed system is a collection of interconnected computers that appear as a single system to the user.
- Each computer in the system is called a node or a processor.

### File Service Architecture

- File System Architecture:
  - Client-Server Model
  - Peer-to-Peer Model
  - Distributed File System (DFS)
- Characteristics of DFS:
  - Data is distributed across multiple nodes
  - Each node is a file system client and server
  - Data is replicated for reliability and availability

### Key Concepts

- **File Service Architecture**: A framework for organizing and managing files in a distributed system.
- **Distributed File System (DFS)**: A type of file system that stores data across multiple nodes.
- **Client-Server Model**: A type of file service architecture where clients request services from servers.
- **Peer-to-Peer Model**: A type of file service architecture where each node acts as both client and server.

### Important Formulas and Definitions

- **Availability**: The probability that a system is functioning and can be accessed.
- **Throughput**: The rate at which data is transferred between nodes.
- **File System Hierarchy Standard (FHS)**: A standard for organizing files on a Unix-like system.

### Theorems and Concepts

- **Cap theorem**: A theorem that states that there are three conflicting demands for a distributed system: availability, throughput, and reliability.
- **Replication**: A technique for ensuring data availability by copying data across multiple nodes.

### Revision Tips

- Understand the client-server and peer-to-peer models of file service architecture.
- Familiarize yourself with the characteristics of a distributed file system.
- Review the key concepts of availability, throughput, and replication.
