# **Text Book: Chapter 1 Revision Notes**

### Introduction to Data Warehousing

- Definition: A centralized repository for storing data from various sources, used to support business decision-making.
- Need for Data Warehousing:
  - Growing amounts of data from various sources
  - Increasing complexity of business decision-making
  - Need for accurate and timely information

### Failures of Past Decision-Support Systems

- Common problems:
  - Inadequate data quality
  - Insufficient data integration
  - Lack of business support
  - Inability to adapt to changing business needs

### Key Concepts and Definitions

- **Data Warehouse**: A repository that stores data in a structured and integrated manner.
- **ETL (Extract, Transform, Load)**: The process of extracting data from various sources, transforming it into a consistent format, and loading it into the data warehouse.
- **OLAP (Online Analytical Processing)**: A technology that enables fast and efficient analysis of data.
- **Data Mining**: The process of discovering patterns, relationships, and insights from large datasets.

### Important Formulas and Theorems

- **Data Warehouse Size Formula**: N = (S × T) / E, where N = number of users, S = total storage capacity, T = time, and E = error rate.
- **Data Integration Theorem**: If data from multiple sources is integrated correctly, the resulting data warehouse will provide accurate and timely information.

### Key Theorems

- **Data Quality Theorem**: If data is of poor quality, the resulting analysis and decision-making will also be of poor quality.
- **Data Security Theorem**: If data is not properly secured, it can be compromised, leading to data breaches and other security threats.

### Quick Revision Tips

- Understand the definition and need for data warehousing
- Identify the common problems of past decision-support systems
- Familiarize yourself with key concepts, definitions, formulas, and theorems
- Practice analyzing data and making decisions using data warehousing techniques.
