# **Data Warehouse Defined**

### Overview

A data warehouse is a centralized repository that stores data from various sources in a structured and integrated way, making it easier to access and analyze.

### Key Points

- **Definition:** A data warehouse is a centralized repository that stores data in a structured and integrated way, supporting business intelligence and decision-making.
- **Characteristics:**
  - Centralized repository
  - Integrated data from various sources
  - Structured data
  - Supports business intelligence and decision-making
- **Data Warehouse Model:** Three-tier model:
  - **Data Source Layer:** Raw data from various sources
  - **Metadata Layer:** Descriptive information about the data
  - **Application Layer:** Interfaces for data access and analysis

### Important Formulas and Definitions

- **Data Warehouse Model Architecture (DWM):**
  - DWH = ETL + OLAP
    - ETL: Extract, Transform, Load
    - OLAP: Online Analytical Processing
- **ETL Process:**
  - ETL = Extract (raw data), Transform (data cleansing and formatting), Load (data into the warehouse)
- **Star and Snowflake Schemas:**
  - Star schema: Simplified structure with a central fact table and dimension tables
  - Snowflake schema: More complex structure with multiple fact tables and dimension tables

### Theorems and Concepts

- **Data Warehousing Theorem:** A well-designed data warehouse can improve decision-making and business performance.
- **Data Integration Theorem:** A data warehouse can integrate data from various sources, improving data consistency and reducing data redundancy.

### Revision Tips

- Understand the definition and characteristics of a data warehouse
- Familiarize yourself with the three-tier model and ETL process
- Review the star and snowflake schemas and their applications
- Be able to explain the data warehousing theorem and data integration theorem
