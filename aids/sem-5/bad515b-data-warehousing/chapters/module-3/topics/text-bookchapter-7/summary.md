# **Text Book: Chapter 7 Summary**

## **Data Warehouse Architecture**

- A data warehouse is a centralized repository for storing data from various sources.
- It provides a single, unified view of an organization's data.
- The main purpose of a data warehouse is to support business intelligence and decision-making.

## **Types of Data Warehouses**

- **Physical Data Warehouse (PDW)**: A physical storage of data in a warehouse.
- **Virtual Data Warehouse (VDW)**: A virtual representation of a physical data warehouse.
- **Hybrid Data Warehouse**: A combination of physical and virtual data warehouses.

## **Data Warehouse Architectures**

- **Star Schema**: A schema that uses a central fact table surrounded by dimension tables.
- **Snowflake Schema**: A schema that uses a central fact table surrounded by dimension tables, with each dimension table further divided into sub-tables.
- **Fact-Related Dimension (FRD)**: A schema that uses a central fact table surrounded by dimension tables, with some dimension tables related to the fact table.

## **Data Warehouse Design**

- **Entity-Relationship Model**: A model that represents data as entities and relationships between them.
- **Dimensional Modeling**: A model that represents data as facts and dimensions.

## **Key Formulas and Definitions**

- **Third-Party Data Warehousing**: A data warehouse that is hosted and managed by a third party.
- **ETL (Extract, Transform, Load)**: A process of extracting data from multiple sources, transforming it into a consistent format, and loading it into a data warehouse.
- **Data Warehousing Model**: A conceptual model that represents the structure and organization of a data warehouse.

## **Important Theorems**

- **Data Warehousing Law**: The law states that "Information is the lifeblood of an organization."
- **Data Warehousing Principle**: The principle states that "Data should be stored in a centralized repository to support business intelligence and decision-making."
