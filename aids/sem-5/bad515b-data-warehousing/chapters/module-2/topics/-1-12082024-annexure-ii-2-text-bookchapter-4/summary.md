# Revision Notes: Planning Your Data Warehouse (Chapter 4)

## Introduction

- Planning is a critical phase of the data warehouse project
- Involves defining the project scope, objectives, and requirements

## Data Warehouse Requirements

- Define the business requirements and objectives
- Identify the data sources and data volume
- Determine the data warehouse size and complexity
- Establish the data warehouse architecture

## Data Warehouse Architecture

- Define the data warehouse layers:
  - Physical layer: database management system
  - Logical layer: schema and data model
  - Physical layer: data storage

## Data Warehouse Design

- Define the data warehouse design:
  - Star schema
  - snowflake schema
  - Data mart

## Data Warehousing Concepts

- **Data warehouse**: a repository of historical data for analysis and reporting
- **ETL (Extract, Transform, Load)**: the process of extracting data from sources, transforming it into a format suitable for analysis, and loading it into the data warehouse
- **Data mart**: a subset of the data warehouse that is optimized for a specific business area
- **OLAP (Online Analytical Processing)**: a technology that enables fast and efficient analysis of data

## Important Formulas and Theorems

- **Data volume**: the amount of data stored in the data warehouse
- **Data variety**: the variety of data types and sources
- **Data velocity**: the speed at which data is generated and processed
- **Data value**: the value of the data to the organization

## Key Terms

- **Data warehousing**: the process of designing, building, and maintaining a data warehouse
- **ETL tool**: a software tool used to extract, transform, and load data into the data warehouse
