# Learning Objectives
After studying this topic, you should be able to:
1. Define software architecture and explain its importance in software development
2. Compare and contrast at least four different architectural patterns
3. Analyze how architectural decisions impact quality attributes like performance, scalability, and maintainability
4. Create basic architectural diagrams using appropriate UML notations
5. Evaluate architectural choices based on specific system requirements and constraints
6. Apply architectural principles such as separation of concerns and low coupling in design scenarios
