# Revision Notes: Chapter 7 (pg: 210-227) - Blockchain Technology

=============================================

## Key Concepts

### Definitions

- **Hash**: A fixed-length string of characters created by hashing a variable-length input string.
- **Blockchain**: A distributed digital ledger technology.
- **Consensus Mechanism**: A protocol used to achieve consensus among nodes in a decentralized network.

### Theorems

- **Byzantine Generals' Theorem**: States that in a network of `n` nodes, if `f(n) > n/3`, then there is a malicious node that can determine the majority vote.
- **Gödel's Incompleteness Theorems**: Show that any formal system powerful enough to describe basic arithmetic is either incomplete or inconsistent.

### Consensus Mechanisms

- **Proof of Work (PoW)**: Requires nodes to solve a complex mathematical puzzle to validate transactions and create new blocks.
- **Proof of Stake (PoS)**: Requires nodes to "stake" their own cryptocurrency to validate transactions and create new blocks.

### Hash Functions

- **SHA-256**: A cryptographic hash function used to secure blockchain transactions.
- **Keccak-256**: A cryptographic hash function used in Ethereum's proof-of-work consensus mechanism.

### Important Formulas

- **Hash function**: `hash(x) = f(x)`
- **Blockchain validation**: `valid = (transactions \* blocks)`

### Other Key Points

- **Distributed Ledger Technology**: A decentralized, distributed ledger that records transactions across a network.
- **Smart Contracts**: Self-executing contracts with the terms of the agreement written directly into lines of code.
- **Ethereum's Consensus Mechanism**: A hybrid consensus mechanism that combines proof-of-work and proof-of-stake.
