# **Mobile Application Development Revision Notes (Chapters 3 & 4)**

## **Directory Structure**

- Java package structure:
  - `com.example.app` (package name)
  - `MainActivity.java` (main activity)
  - `MainActivity.xml` (activity layout)
  - `activity_main.xml` (activity declaration)
- Project structure:
  - `app` (main package)
  - `res` (resource folder)
    - `layout` (folder for layout files)
    - `values` (folder for values files)

## **Android User Interface**

- UI components:
  - `TextView` (displays text)
  - `Button` (perform action when clicked)
  - `EditText` (user input field)
  - `ImageView` (displays image)
- Layout files:
  - `activity_main.xml` (activity layout)
  - `fragment_main.xml` (fragment layout)
- Layout parameters:
  - `android:layout_width` (set width)
  - `android:layout_height` (set height)
  - `android:textSize` (set text size)

## **Key Concepts and Formulas**

- **Intent**: a message passed between Android components
- **Activity**: a single screen in an Android app
- **Service**: a long-running operation in an Android app
- **BroadcastReceiver**: a component that receives Intent broadcasts
- **Threading**: managing threads for background operations

## **Important Definitions and Theorems**

- **Android API**: a set of libraries and tools for developing Android apps
- **Java Virtual Machine (JVM)**: an environment for running Java bytecode
- **Activity lifecycle**: the state of an activity (e.g., created, started, stopped)

## **Revision Tips**

- Review Java package structure and Android UI components
- Understand the importance of layout files and parameters
- Familiarize yourself with key concepts and formulas (Intent, Activity, Service, BroadcastReceiver, Threading)
- Review activity lifecycle and Android API
