### 6.1 to 6.3 Revision Notes: Introduction to Data Structures

#### 6.1 Stacks and Queues

- A **stack** is a Last-In-First-Out (LIFO) data structure, where elements are added and removed from the top.
- A **queue** is a First-In-First-Out (FIFO) data structure, where elements are added to the end and removed from the front.
- Operations:
  - Stack: `push()`, `pop()`, `peek()`
  - Queue: `enqueue()`, `dequeue()`, `peek()`

#### 6.2 Lists

- A **list** is a ordered sequence of elements, where elements can be added or removed from any position.
- Types of lists:
  - **Dynamic list**: can be of any size, where elements can be added or removed dynamically.
  - **Static list**: has a fixed size, where elements cannot be added or removed.

#### 6.3 Trees

- A **tree** is a non-linear data structure, where each element has a value and zero or more child nodes.
- Types of trees:
  - **Binary tree**: each node has at most two child nodes.
  - **N-ary tree**: each node can have any number of child nodes.

#### Important Formulas/Definitions/Theorems

- **Big O notation**: a measure of an algorithm's efficiency, where O(1) is constant time, O(log n) is logarithmic time, O(n) is linear time, and O(n log n) is linearithmic time.
- **Time complexity**: the amount of time an algorithm takes to complete, measured in terms of the size of the input.
- **Space complexity**: the amount of memory an algorithm uses, measured in terms of the size of the input.

#### Key Concepts

- **Search algorithms**: algorithms that find a specific element in a data structure.
- **Insertion and deletion algorithms**: algorithms that modify a data structure by adding or removing elements.
- **Sorting algorithms**: algorithms that arrange elements in a data structure in a specific order.
