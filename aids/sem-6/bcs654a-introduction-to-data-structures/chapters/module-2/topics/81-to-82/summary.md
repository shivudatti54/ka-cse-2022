# **8.1 to 8.2: Quick Revision Notes**

## **Introduction to Data Structures**

- Data structure: A collection of data elements, each of which has a specific format and organization.

## **Types of Data Structures**

- **Array**: A collection of elements of the same data type stored in contiguous memory locations.
  - Formula: `T[i]`, where `T` is the type of element and `i` is the index.
- **Linked List**: A dynamic collection of elements, where each element points to the next element.
  - Definition: A node consists of a data field and a reference (i.e., "link") to the next node in the sequence.

## **Other Important Data Structures**

- **Stack**: A Last-In-First-Out (LIFO) data structure that follows the principle of Last In, First Out.
  - Formula: `S.push(element)`, where `S` is the stack and `element` is the element to be pushed.
- **Queue**: A First-In-First-Out (FIFO) data structure that follows the principle of First In, First Out.
  - Formula: `Q.enqueue(element)`, where `Q` is the queue and `element` is the element to be enqueued.

## **Important Theorems and Formulas**

- **Big O Notation**: A measure of the complexity of an algorithm, typically expressed as a function of the size of the input.
- **Time Complexity**: The amount of time an algorithm takes to complete as a function of the size of the input.
- **Space Complexity**: The amount of memory an algorithm uses as a function of the size of the input.

## **Key Concepts**

- **Data Structure**: A way of organizing data in a computer so that it can be efficiently accessed and manipulated.
- **Algorithms**: A set of instructions that can be used to solve a problem or perform a task.
