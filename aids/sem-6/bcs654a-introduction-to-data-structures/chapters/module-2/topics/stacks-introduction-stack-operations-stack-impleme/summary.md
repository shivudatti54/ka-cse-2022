# **Stacks: Introduction, Stack Operations, Stack Implementation using Arrays, Applications of Stacks**

## **I. Introduction to Stacks**

- A stack is a linear data structure that follows the Last-In-First-Out (LIFO) principle
- A stack is a region of memory that can be accessed through two ends: the top and the bottom
- The top element is the most recently added and also the first one to be removed

## **II. Stack Operations**

- **Push**: add an element to the top of the stack
- **Pop**: remove the top element from the stack
- **Peek**: return the top element from the stack without removing it
- **IsEmpty**: check if the stack is empty
- **Size**: return the number of elements in the stack

## **III. Stack Implementation using Arrays**

- Create a stack using an array and two pointers (top and bottom)
- Top pointer points to the top element of the stack
- Bottom pointer points to the first element of the stack
- When an element is pushed, increment the top pointer
- When an element is popped, decrement the top pointer

## **IV. Applications of Stacks**

- Evaluating postfix expressions
- Parsing expressions in programming languages
- Implementing recursive algorithms iteratively
- Implementing Backtracking algorithms
- Data compression using Run-Length Encoding

## **Important Formulas and Definitions**

- **Stack size**: the maximum number of elements that can be stored in the stack
- **Stack overflow**: occurs when the stack size is exceeded
- **Stack underflow**: occurs when the stack is empty

## **Important Theorems**

- **Stack Analysis**: a method of analyzing the time complexity of algorithms using stacks.
