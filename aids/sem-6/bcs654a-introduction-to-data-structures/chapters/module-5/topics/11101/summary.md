# 11.10.1 Summary

### Introduction to Sorting Data Structures

- **Definition:** Sorting is the process of arranging data in a specific order, such as ascending or descending.
- **Importance:** Efficient sorting is crucial for many applications, including data analysis, database management, and algorithm design.

### Sorting Algorithms

- **Bubble Sort:**
  - Time complexity: O(n^2)
  - Description: Repeatedly iterates through the array, swapping adjacent elements if they are in the wrong order.
- **Selection Sort:**
  - Time complexity: O(n^2)
  - Description: Selects the smallest (or largest) element from the unsorted portion of the array, swapping it with the first element of the unsorted portion.
- **Insertion Sort:**
  - Time complexity: O(n^2)
  - Description: Iterates through the array, inserting each element into its correct position within the previously sorted portion.

### Important Formulas and Theorems

- **Big O Notation:** A measure of an algorithm's time or space complexity, often expressed as O(n), O(n log n), etc.
- **Stability:** A property of sorting algorithms that preserves the order of equal elements.
- **Comparison-based sorting:** A type of sorting algorithm that relies on comparisons between elements to determine their order.
- **In-place sorting:** A type of sorting algorithm that sorts the array without using additional storage space.

### Key Concepts

- **Time complexity:** A measure of an algorithm's performance, often expressed in terms of the number of operations it performs.
- **Space complexity:** A measure of an algorithm's memory usage, often expressed in terms of the amount of additional memory it requires.
- **Scalability:** The ability of an algorithm to handle large datasets efficiently.
