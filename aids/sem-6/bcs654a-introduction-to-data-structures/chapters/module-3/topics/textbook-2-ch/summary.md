### Introduction to Linked Lists

- A linked list is a linear collection of elements whose elements are not stored in contiguous memory locations.
- Each element is called a node, and each node contains two items:
  - Data: The actual data stored in the node.
  - Reference (or link): A reference (i.e., a "link") to the next node in the sequence.
- Linked lists can be either singly linked or doubly linked.

### Singly Linked List

- Singly linked lists have only one reference (or "link") to the next node in the sequence.
- Examples:
  - A basic implementation of a singly linked list.
  - A common data structure used in many applications, including databases, email clients, and web browsers.

### Self-Referential Structures

- A self-referential structure is a data structure that contains references to itself.
- Examples:
  - A singly linked list with a self-referential structure.
  - A doubly linked list with a self-referential structure.

### Operations on Linked Lists

- **Insertion:** Inserting a new node at a specific position in the linked list.
- **Deletion:** Removing a node from the linked list.
- **Search:** Searching for a specific node in the linked list.
- **Traversal:** Displaying the elements of the linked list in a specific order.

### Important Formulas and Definitions

- **Node:** An element in a linked list that contains data and a reference to the next node.
- **Head:** The first node in a linked list.
- **Tail:** The last node in a linked list.
- **Length:** The number of nodes in a linked list.

### Important Theorems

- **Beat the Clock Theorem:** If the nodes in the linked list are accessed in a random order, the time complexity of the operations is O(n) in the worst case.
- **Amortized Analysis:** The average-case time complexity of a sequence of operations is a more accurate measure of the performance of an algorithm than the worst-case time complexity.
