# Learning Objectives

After studying this topic, you should be able to:

1. Define what a process is and distinguish it from a program in the context of operating systems
2. Explain the structure and components of a Process Control Block (PCB) and its role in process management
3. Describe the various process states and the transitions between them in the process lifecycle
4. Illustrate how processes are created, execute, and terminate using system calls
5. Differentiate between independent and cooperative processes
6. Analyze real-world scenarios involving multiple processes from the same program
7. Understand the relationship between process concepts and CPU scheduling, which is covered in subsequent topics
