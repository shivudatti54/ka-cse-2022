# **5.1 Introduction to Operating Systems**

### Overview

An operating system (OS) is a software that manages computer hardware resources and provides common services to computer programs. It acts as an intermediary between the user and the computer hardware, allowing users to interact with the computer in a meaningful way.

### Key Functions of an Operating System

- Manages computer hardware resources, such as CPU, memory, and storage devices
- Provides a platform for running applications and services
- Controls input/output operations
- Provides a interface for users to interact with the computer
- Offers security and protection features
- Offers file management and storage services

### Types of Operating Systems

- **Single-user, single-tasking**: Allows only one user to access the computer at a time and only one program to run at a time (e.g., MS-DOS)
- **Multi-user, multi-tasking**: Allows multiple users to access the computer simultaneously and multiple programs to run at the same time (e.g., Windows, Linux)
- **Real-time operating system**: Requires the operating system to meet strict timing constraints, often used in embedded systems (e.g., traffic lights, medical devices)

### Key Concepts

- **Process**: A program in execution, with its own memory space and resources
- **Thread**: A lightweight process that shares the same memory space and resources
- **Memory Management**: The ability of the operating system to manage and allocate memory to processes
- **File System**: A collection of files and directories that are stored on a storage device and managed by the operating system

### Example

Suppose we have a computer running Windows 10, a multi-user, multi-tasking operating system. The user logs in to the computer and opens two programs: a word processor and a web browser. The operating system allocates memory and resources to each program, allowing the user to interact with both programs simultaneously. The operating system also manages input/output operations, providing the user with a graphical user interface to interact with the computer.

### Key Terms

- **Bootloader**: A program that loads the operating system into memory when the computer is powered on
- **Kernel**: The core part of the operating system that manages computer hardware resources
- **Device Driver**: A program that manages interaction between the operating system and a specific hardware device

### Key Concepts

- **Device Driver**: A program that manages interaction between the operating system and a specific hardware device
- **Interrupt**: A signal sent by the hardware to the operating system to request attention
- **Context Switching**: The process of switching from one process to another, often involving saving and restoring process state
