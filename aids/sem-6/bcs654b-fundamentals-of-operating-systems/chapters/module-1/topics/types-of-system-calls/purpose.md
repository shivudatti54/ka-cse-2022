# Learning Objectives

After studying this topic, you should be able to:

1.  Explain the purpose of dual-mode operation (user and kernel mode) and its relationship to system calls.
2.  Define a system call and describe the step-by-step process of how it is executed.
3.  Differentiate between an Application Programming Interface (API) and a system call.
4.  Categorize common system calls (e.g., from Linux or Windows) into process control, file management, device management, and other types.
5.  Articulate the critical role system calls play in providing security, abstraction, and portability.
6.  Trace the flow of a high-level API call (like `printf`) down to the underlying system call(s).
