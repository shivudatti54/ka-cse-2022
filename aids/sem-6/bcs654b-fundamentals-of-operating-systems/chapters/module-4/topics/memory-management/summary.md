# Memory Management - Summary

## Key Definitions and Concepts

- **Memory Management Unit (MMU)**: Hardware device that translates logical addresses to physical addresses dynamically during execution

- **Logical Address**: Address generated by the CPU, used by programs; also called virtual address

- **Physical Address**: Actual hardware address in RAM where data is stored

- **Paging**: Memory management scheme dividing logical and physical memory into fixed-size pages and frames

- **Page Table**: Data structure mapping virtual page numbers to physical frame numbers

- **Translation Lookaside Buffer (TLB)**: Hardware cache storing recent page table translations for fast access

- **Virtual Memory**: Technique allowing programs to use more memory than physically available by using disk as backing store

- **Page Fault**: Occurs when a process accesses a page not currently in physical memory

## Important Formulas and Theorems

- **Logical address format**: [Page Number][Offset]
- **Page Number**: Logical Address ÷ Page Size (integer division)
- **Offset**: Logical Address mod Page Size
- **Physical Address**: (Frame Number × Page Size) + Offset
- **Number of Pages**: 2^(logical address bits - offset bits)
- **Number of Frames**: Physical Memory Size ÷ Page Size
- **Effective Access Time**: TLB Hit Rate × (TLB Access + Memory Access) + (1 - TLB Hit Rate) × (TLB Access + 2 × Memory Access)

## Key Points

- Memory management enables multiple processes to share limited RAM safely and efficiently

- Address binding can occur at compile time, load time, or execution time; execution time binding is essential for virtual memory

- Contiguous allocation suffers from external fragmentation; paging eliminates external fragmentation but introduces internal fragmentation

- Page tables can be optimized using hierarchical structure, hashing, or inverted tables to save memory

- TLB significantly improves performance with hit rates typically above 95%

- Segmentation reflects logical program structure; paging provides physical memory management; they can be combined

- Virtual memory extends available memory using disk, enabling programs larger than physical RAM to execute

- Page replacement algorithms (FIFO, LRU, Optimal) determine which page to evict when memory is full

## Common Mistakes to Avoid

- Confusing page number with frame number in address translation calculations

- Forgetting that offsets are identical in both logical and physical addresses within a page

- Not considering TLB access time when calculating effective memory access time with paging

- Mixing up internal fragmentation (within allocated blocks) with external fragmentation (between free blocks)

- Assuming that larger page sizes always improve performance; they increase internal fragmentation

## Revision Tips

- Practice converting between logical and physical addresses with different page sizes

- Draw memory layouts for both paging and segmentation to visualize the concepts

- Memorize the formula for effective memory access time and practice numerical problems

- Understand why page faults are expensive (disk I/O takes millions of CPU cycles)

- Review how protection bits in page table entries enforce memory access restrictions between processes
