# File System Interface: File Concept

### Definitions

- **File**: A collection of related data stored in a physicalmedium (e.g., disk) and managed by an operating system.
- **File System Interface**: The set of rules and protocols that allow applications to interact with files.

### File Characteristics

- **File Name**: A unique identifier for a file.
- **File Type**: The type of data stored in the file (e.g., text, image, executable).
- **File Size**: The amount of data stored in the file.
- **File Location**: The physical location of the file on the storage medium.

### File System Concepts

- **File System Hierarchy Standard (FHS)**: A standard for organizing files on a file system.
- **Root Directory**: The topmost directory in a file system.
- **Subdirectories**: Directories contained within a parent directory.
- **Files and Directories**: The basic units of organization in a file system.

### Important Formulas and Theorems

- **Inode**: A data structure that stores metadata about a file (e.g., file type, permissions).
- **Block Size**: The size of a contiguous block of data in a file system.
- **Fragmentation**: The process of breaking files into smaller, non-contiguous blocks.

### Key Points

- Files are collected data stored on a physical medium, managed by an operating system.
- The file system interface allows applications to interact with files.
- Files have characteristics such as name, type, size, and location.
- File systems use concepts like directories, subdirectories, and inode to organize files.
- Important formulas and theorems include inode, block size, and fragmentation.
