# What Operating Systems Do

## Overview

The Operating System performs four core functions to manage computer resources: Process Management, Memory Management, File System Management, and Device Management. It acts as both an Extended Machine (hiding hardware complexity) and a Resource Manager (allocating resources fairly and efficiently).

## Key Points

- **Process Management**: CPU scheduling, process creation/termination, and inter-process communication (IPC)
- **Memory Management**: Allocation/deallocation, memory protection, and virtual memory implementation
- **File System Management**: File operations, directory organization, access control, and disk scheduling
- **Device Management**: Driver interface, buffering/caching, and spooling for devices like printers
- **Extended Machine**: Provides simple abstractions hiding complex hardware details
- **Resource Manager**: Allocates CPU cycles, memory, storage, and I/O devices fairly among processes
- **System Services**: User Interface, Program Execution, I/O Operations, Error Detection, Resource Allocation, Accounting, and Protection

## Important Concepts

- OS provides uniform, logical view of information storage abstracting physical media details
- System calls are the gateway for application programs to request OS services
- Dual role as both extended machine and resource manager is fundamental to OS design
- Memory hierarchy and virtual memory allow running programs larger than physical RAM

## Notes

- Focus on the dual role concept - frequently asked in exams
- Be able to explain each core function with practical examples
- Understand the relationship between system calls and OS services
