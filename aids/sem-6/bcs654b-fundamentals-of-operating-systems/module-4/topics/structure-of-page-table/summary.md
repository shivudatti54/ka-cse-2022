# Structure of Page Table - Summary

## Key Definitions and Concepts

- **Page Table**: Data structure mapping virtual addresses to physical addresses in paged memory systems
- **Virtual Address**: Logical address generated by CPU (divided into _page number_ and _page offset_)
- **Physical Address**: Actual location in RAM (divided into _frame number_ and _offset_)
- **Page Frame**: Fixed-size block of physical memory
- **TLB (Translation Lookaside Buffer)**: Hardware cache storing recent page table entries

## Important Formulas and Theorems

```markdown
1. Virtual Address Translation:
   Physical Address = (Frame Number × Page Size) + Offset

2. Effective Access Time (EAT) with TLB:
   EAT = (TLB_hit_time + Memory_access) × hit_ratio + (TLB_miss_time + 2 × Memory_access) × (1 - hit_ratio)

3. Page Table Size:
   Total Size = Number_of_pages × Entry_size
```

## Key Points

- Hierarchical (Multi-level) Paging: Divides page table into smaller tables (e.g., 2-level: outer + inner tables)
- Hashed Page Tables: Uses hash function to map virtual page numbers to chains of entries
- Inverted Page Tables: Single table with one entry per physical frame (saves memory)
- TLB accelerates address translation by caching frequent page table entries
- Hardware support crucial for efficient page table traversal (MMU handles address translation)
- Hierarchical structure reduces memory overhead but increases access time
- Hashed tables handle sparse address spaces efficiently
- Inverted tables ideal for 64-bit architectures with large address spaces
- Page table base register (PTBR) points to active page table in memory
- Modern OS use combination of techniques (e.g., Linux: 4-level paging for 64-bit systems)

## Common Mistakes to Avoid

1. Confusing page table structures (e.g., using hierarchical for 64-bit systems instead of inverted)
2. Forgetting TLB in EAT calculations (leading to incorrect memory access counts)
3. Miscalculating page table size (not considering entry size vs page count)
4. Ignoring hardware aspects (assuming software handles all translation steps)
