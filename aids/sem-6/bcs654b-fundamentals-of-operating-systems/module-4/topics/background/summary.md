# Background (Memory Management and Deadlocks)

## Overview

This module covers memory management (allocating memory to processes efficiently) and deadlocks (circular waiting for resources). Memory management includes contiguous allocation, paging, and virtual memory, while deadlock handling involves prevention, avoidance, detection, and recovery strategies.

## Key Points

- **Memory Management**: Allocates memory to processes, ensures protection, implements virtual memory for efficient resource utilization
- **Address Binding**: Compile time (absolute addresses), load time (relocatable addresses), execution time (logical to physical mapping)
- **Logical vs Physical Address**: Logical generated by CPU, physical used by memory unit, mapping by MMU
- **Deadlock**: Set of processes each waiting for resource held by another process in the set
- **Resource Allocation Graph**: Visual representation of processes, resources, and allocation/request edges
- **Memory Protection**: Base and limit registers ensure process accesses only its allocated memory region

## Important Concepts

- Memory management unit (MMU) translates logical to physical addresses at runtime
- Swapping moves entire process between memory and backing store
- Deadlock requires four necessary conditions: mutual exclusion, hold and wait, no preemption, circular wait
- Memory management techniques: contiguous allocation, paging, segmentation, virtual memory

## Notes

- Understand difference between logical and physical address spaces
- Know address binding times and their implications
- Remember four conditions necessary for deadlock
- Practice drawing resource allocation graphs
