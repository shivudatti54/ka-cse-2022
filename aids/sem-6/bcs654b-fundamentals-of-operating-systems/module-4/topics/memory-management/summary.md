# Memory Management

## Overview

Memory management allocates memory to processes efficiently, ensuring protection and supporting multiprogramming. Techniques include contiguous allocation (fixed/variable partitions), paging (fixed-size pages/frames), segmentation (variable-size logical units), and virtual memory (demand paging, page replacement).

## Key Points

- **Address Binding**: Compile time (absolute), load time (relocatable), execution time (dynamic with MMU)
- **Logical Address**: Generated by CPU during program execution
- **Physical Address**: Actual address in memory unit
- **MMU (Memory Management Unit)**: Hardware device mapping logical to physical addresses using relocation register
- **Swapping**: Moving process between memory and backing store to manage multiprogramming level
- **Contiguous Allocation**: Fixed partitions (internal fragmentation) or variable partitions (external fragmentation)
- **Fragmentation**: External (free memory fragmented into small holes), internal (allocated memory larger than needed)
- **Compaction**: Shuffle memory contents to place all free memory together

## Important Concepts

- Dynamic loading loads routine only when called, reducing memory usage
- Dynamic linking postpones linking until execution time, shared libraries
- Overlays allow program larger than physical memory by manual memory management
- First-fit, best-fit, worst-fit allocation strategies have different fragmentation characteristics

## Notes

- Understand difference between logical and physical addresses
- Know when address binding occurs for each type
- Practice calculating internal vs external fragmentation
- Remember swapping time includes transfer time and disk seek/latency
