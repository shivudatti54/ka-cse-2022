## Introduction to Memory Management

Memory management is a fundamental function of an operating system that handles or manages primary memory (RAM) and moves processes back and forth between main memory and disk during execution. It keeps track of each memory location, regardless of whether it is allocated to some process or free. It checks how much memory is to be allocated to processes and decides which process will get memory at what time. It also tracks whenever some memory gets freed or unallocated and updates the status accordingly.

The main goals of memory management are:

- **Efficient utilization**: Maximize the use of available memory
- **Protection**: Prevent processes from interfering with each other's memory
- **Sharing**: Allow controlled sharing of memory between processes
- **Transparency**: Provide an abstraction that hides physical memory details from processes

## Memory Hierarchy

```
+---------------------+

```

## Basic Memory Concepts

### Address Spaces

Every process operates in its own **logical address space**, which is separate from the physical memory addresses. The memory management unit (MMU) translates logical addresses to physical addresses.

- **Logical Address**: Generated by the CPU; also called virtual address
- **Physical Address**: Actual address in memory hardware
- **Address Binding**: The process of mapping logical addresses to physical addresses

### Memory Allocation Methods

The operating system uses various methods to allocate memory to processes:

#### 1. Single Contiguous Allocation

The entire memory is allocated to a single process, with the operating system typically residing in low memory.

```
+---------------------+
| Operating System |
| (Fixed Partition) |
+---------------------+
| |
| User Process |
| (Single Process) |
| |
+---------------------+
| |
| Unused Memory |
| |
+---------------------+
```

**Advantages**: Simple implementation
**Disadvantages**: Severe memory underutilization, cannot support multiprogramming

#### 2. Fixed Partitioning (Multiple Fixed Partitions)

Memory is divided into fixed-size partitions, each of which can hold exactly one process.

```
+---------------------+
| Operating System |
+---------------------+
| Partition 1 | → Process 1
+---------------------+
| Partition 2 | → Process 2
+---------------------+
| Partition 3 | → Process 3
+---------------------+
| ... |
+---------------------+
```

**Advantages**: Supports multiprogramming, relatively simple
**Disadvantages**: Internal fragmentation, limits process size

#### 3. Dynamic Partitioning (Variable Partitions)

Partitions are created dynamically according to process size requirements.

```
+---------------------+
| Operating System |
+---------------------+
| Process 1 (Size: 50K)|
+---------------------+
| Process 3 (Size: 100K)|
+---------------------+
| Free Space (150K) |
+---------------------+
| Process 2 (Size: 75K)|
+---------------------+
| Free Space (125K) |
+---------------------+
```

**Advantages**: No internal fragmentation, efficient memory utilization
**Disadvantages**: External fragmentation, requires compaction

### Fragmentation

Fragmentation occurs when memory space is used inefficiently, leading to reduced capacity and performance.

#### Internal Fragmentation

Occurs when allocated memory blocks are larger than what the process requested. The unused space within a partition is wasted.

Example: If a partition is 100KB and a process uses only 80KB, the remaining 20KB is internal fragmentation.

#### External Fragmentation

Occurs when free memory is scattered throughout the system in small, non-contiguous blocks that cannot be used for new processes despite their cumulative size being sufficient.

```
Before Compaction:
+-------+ +-------+ +-------+
| Proc A| | Free | | Proc B|
| 100K | | 50K | | 150K |
+-------+ +-------+ +-------+
| Proc C| | Free | | Proc D|
| 75K | | 25K | | 200K |
+-------+ +-------+ +-------+

Total Free: 75K, but fragmented

After Compaction:
+-------+ +-------+ +-------+
| Proc A| | Proc B| | Proc C|
| 100K | | 150K | | 75K |
+-------+ +-------+ +-------+
| Proc D| | Free |
| 200K | | 75K |
+-------+ +-------+
```

**Compaction** is a technique to reduce external fragmentation by moving processes to consolidate free memory. However, it's computationally expensive and requires processes to be relocatable.

## Memory Protection

Memory protection ensures that a process cannot access memory that doesn't belong to it, preventing malicious or accidental interference.

### Base and Limit Registers

The most common protection mechanism uses two special CPU registers:

- **Base Register**: Contains the smallest legal physical memory address
- **Limit Register**: Contains the size of the range

For each process, the operating system loads these registers with the appropriate values. The CPU checks every memory reference generated in user mode to ensure it's between the base and base+limit.

```
Logical address: 0x500
Base register: 0x2000
Limit register: 0x1000

Physical address = 0x2000 + 0x500 = 0x2500
Check: 0x500 < 0x1000 → Valid access
```

If the logical address exceeds the limit, a trap occurs, and the operating system handles the memory violation (usually by terminating the process).

## Memory Allocation Algorithms

When allocating memory to processes, the operating system must decide which free block to use. Common algorithms include:

### First-Fit

Allocate the first hole that is big enough.

**Advantages**: Fast, minimal searching
**Disadvantages**: May leave large free fragments at beginning

### Best-Fit

Allocate the smallest hole that is big enough.

**Advantages**: Minimizes wasted memory in each allocation
**Disadvantages**: Requires searching entire list, leaves many small fragments

### Worst-Fit

Allocate the largest available hole.

**Advantages**: Leaves large free fragments
**Disadvantages**: Requires searching entire list, may not perform well in practice

### Next-Fit

Similar to first-fit, but starts searching from the last allocation point.

**Advantages**: More uniform distribution of allocations
**Disadvantages**: May not find the best possible fit

Comparison of Allocation Algorithms:

| Algorithm | Speed  | Fragment Size        | Memory Utilization |
| --------- | ------ | -------------------- | ------------------ |
| First-Fit | Fast   | Medium fragments     | Good               |
| Best-Fit  | Slow   | Many small fragments | Moderate           |
| Worst-Fit | Slow   | Large fragments      | Poor to moderate   |
| Next-Fit  | Medium | Medium fragments     | Good               |

## Swapping

Swapping is a technique where a process can be swapped temporarily out of memory to a backing store (disk) and then brought back into memory for continued execution.

```
Memory State:
+---------------------+
| Operating System |
+---------------------+
| Process A |
+---------------------+
| Process B |
+---------------------+
| Process C |
+---------------------+
| Free Space |
+---------------------+

Swap Out Process B:
+---------------------+
| Operating System |
+---------------------+
| Process A |
+---------------------+
| Free Space | ← Process B swapped to disk
+---------------------+
| Process C |
+---------------------+
| Free Space |
+---------------------+
```

**Swap Time**: The time required to transfer a process between memory and disk, which is primarily determined by the transfer rate of the backing store.

**Context Switch Overhead**: Additional time needed to save and restore process state during swapping.

## Memory Management Unit (MMU)

The MMU is a hardware component that handles all memory accesses requested by the CPU. Its main functions include:

- Translating virtual addresses to physical addresses
- Memory protection
- Cache control
- Bus arbitration

The MMU uses a relocation register (base register) to convert logical addresses into physical addresses:

```
Logical Address → MMU → Physical Address
 0x500 + Base: 0x2000 → 0x2500
```

## Exam Tips

1. **Understand the difference between logical and physical addresses**: Logical addresses are process-specific, while physical addresses refer to actual memory locations.

2. **Remember fragmentation types**: Internal (within allocated blocks) vs. External (between allocated blocks).

3. **Know the allocation algorithms**: Be able to compare First-Fit, Best-Fit, Worst-Fit, and Next-Fit in terms of performance and fragmentation.

4. **Base and Limit registers**: Understand how they provide memory protection and relocation.

5. **Swapping concepts**: Know what swapping is, when it's used, and its performance implications.

6. **Memory hierarchy**: Be familiar with the different levels and their characteristics (speed, size, cost).

7. **Practice diagram drawing**: Being able to draw and interpret memory allocation diagrams is crucial for exam questions.
