### Learning Purpose: Terms used in RDBMS and MongoDB

**1. Why is this topic important?**
Understanding the terminology of both relational (RDBMS) and non-relational (MongoDB) databases is crucial. It forms the foundational language required to design, query, and manage modern data systems. As Big Data often involves polyglot persistence (using multiple database types), fluency in these terms is essential for effective data modeling and extraction.

**2. What will students learn?**
Students will learn to define and contrast core concepts. From RDBMS: tables, rows, columns, schemas, SQL, and primary/foreign keys. From MongoDB: collections, documents, BSON, embedding, referencing, and the MongoDB Query Language (MQL). They will map analogous terms (e.g., table → collection, row → document) to understand structural differences.

**3. How does it connect to other concepts?**
This knowledge directly connects to prior modules on data structures and subsequent topics on querying, data migration, and performance optimization. It provides the lexical toolkit to compare relational and non-relational models, which is vital for choosing the right database for a specific analytics task.

**4. Real-world applications**
This applies directly to roles like Data Engineer or Analyst. For example, understanding MongoDB's flexible schema is key for ingesting unstructured social media data, while knowing RDBMS terms is mandatory for working with structured transactional data in systems like data warehouses, enabling informed tool selection.
