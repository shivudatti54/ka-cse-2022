# **TB1: Ch 6: 6.1-6.5 - Big Data Analytics: Introduction to MongoDB**

## **6.1: What is MongoDB?**

MongoDB is a NoSQL, document-based database management system. It was created by Hammer and co-founders in 2009 and is now a part of the MongoDB, Inc. company.

### Key Features of MongoDB:

- **Schema-less**: MongoDB does not require a predefined schema for data storage.
- **Document-based**: MongoDB stores data in JSON-like documents.
- **Scalable**: MongoDB is designed to handle large amounts of data and scale horizontally.
- **Flexible**: MongoDB supports various data types, including strings, integers, dates, and more.

## **6.2: Why MongoDB?**

MongoDB is chosen for several reasons:

- **Handling Large Amounts of Data**: MongoDB is designed to handle large amounts of data and scale horizontally.
- **Fast Data Retrieval**: MongoDB provides fast data retrieval and querying capabilities.
- **Flexible Data Model**: MongoDB's flexible data model allows for easy adaptation to changing data requirements.

### Benefits of Using MongoDB:

- **Improved Data Flexibility**: MongoDB's flexible data model allows for easy adaptation to changing data requirements.
- **Increased Scalability**: MongoDB's horizontal scaling capability allows for easy handling of large amounts of data.
- **Reduced Development Time**: MongoDB's document-based data model reduces the need for complex schema definitions.

## **6.3: Terms used in RDBMS and MongoDB**

Here are some key terms used in relational database management systems (RDBMS) and MongoDB:

### RDBMS Terms:

- **Schema**: A predefined structure for data storage.
- **Table**: A collection of related data stored in rows and columns.
- **Primary Key**: A unique identifier for each record in a table.
- **Foreign Key**: A field in a table that references the primary key of another table.

### MongoDB Terms:

- **Collection**: Similar to a table in RDBMS, but MongoDB uses the term "collection" instead.
- **Document**: A single unit of data in MongoDB, which can contain multiple fields.
- **Field**: A single piece of data within a document.
- **Index**: A data structure that improves query performance by providing a quick way to locate data.

## **6.4: Data Types in MongoDB**

MongoDB supports various data types, including:

### Basic Data Types:

- **String**: A sequence of characters.
- **Integer**: A whole number.
- **Float**: A decimal number.
- **Boolean**: A true or false value.

### Embedded Data Types:

- **Array**: A collection of values of the same data type.
- **Object**: A collection of key-value pairs.
- **Date**: A date and time value.

### Special Data Types:

- **OID**: A unique identifier for a document.
- **Regex**: A regular expression pattern.

## **6.5: Data Storage in MongoDB**

MongoDB stores data in a variety of formats, including:

### Document Format:

- **JSON-like**: MongoDB documents are similar to JSON objects.
- **BSON**: MongoDB's binary serialization format.

### Collection Format:

- **Documents**: A collection of documents.
- **Fields**: A collection of fields within a document.

### Index Format:

- **Hash Index**: A simple index that uses a hash function to map keys to locations in the data.
- **Range Index**: A index that allows for range queries.
- **MultiKey Index**: An index that allows for queries on multiple fields.
- **Text Index**: An index that allows for full-text queries.
