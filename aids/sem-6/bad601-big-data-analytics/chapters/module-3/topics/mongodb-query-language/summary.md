# **MongoDB Query Language Revision Notes**

## **What is MongoDB Query Language?**

- MongoDB Query Language is a query language used to retrieve data from a MongoDB database.
- It is a flexible and powerful language that allows for complex querying and data manipulation.

## **Key Features of MongoDB Query Language**

- **Operations**: `$eq`, `$ne`, `$gt`, `$lt`, `$gte`, `$lte` for comparison operations
- **Pattern Matching**: `$regex` for regular expression matching
- **Array Operations**: `$all`, `$in`, `$nin` for array operations
- **Aggregation**: `$group`, `$sum`, `$avg`, `$max`, `$min` for aggregation operations

## **Important Formulas and Definitions**

- **QBE (Query By Example)**: A query language that allows users to specify data by example rather than by criteria.
- **LQL (Linguistic Query Language)**: A query language that uses natural language to specify queries.
- **FLDL (Flexible and Lightweight Data Labeling)**: A data labeling system that allows for flexible and lightweight data labeling.

## **Important Theorems and Concepts**

- **Document-Oriented Database**: A database that stores data in documents, rather than tables.
- **Schema-Less**: A database that does not require a predefined schema.
- **NoSQL**: A database that does not follow the traditional relational model.

## **Common MongoDB Query Methods**

- **Find**: Retrieves a list of documents that match the query.
- **Update**: Updates one or more documents that match the query.
- **Insert**: Inserts one or more new documents into the database.
- **Delete**: Deletes one or more documents that match the query.

## **Important MongoDB Query Language Concepts**

- **Filter**: A query that filters data based on specific criteria.
- **Projection**: A query that selects specific fields from a document.
- **Sort**: A query that sorts data in a specific order.
- **Limit**: A query that limits the number of results returned.
