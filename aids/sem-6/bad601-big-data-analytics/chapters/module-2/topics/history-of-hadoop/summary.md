# History of Hadoop - Summary

## Key Definitions and Concepts

- **Nutch**: An open-source web search engine project started in 2002 by Doug Cutting and Mike Cafarella, which became the birthplace of Hadoop when the founders needed scalable distributed computing solutions for web indexing.

- **Hadoop**: An open-source distributed computing framework named after Doug Cutting's son's toy elephant, designed for processing large datasets across clusters of commodity computers.

- **GFS (Google File System)**: Google's 2003 paper that introduced the architecture for a distributed file system, which directly influenced HDFS (Hadoop Distributed File System).

- **MapReduce**: Google's 2004 programming model for processing large datasets in parallel, which became Hadoop's core processing framework.

## Key Points

- Hadoop originated in 2002 within the Nutch project when Doug Cutting and Mike Cafarella needed to build a scalable web crawler

- Google's publication of the GFS (2003) and MapReduce (2004) papers provided the architectural foundation for Hadoop

- Doug Cutting joined Yahoo in 2006 and separated Hadoop from Nutch to create an independent project

- Yahoo played a critical role in Hadoop's early development, providing development resources and proving the technology at scale

- Hadoop was released under Apache Software Foundation in 2006, enabling worldwide collaboration and adoption

- In 2008, Hadoop became an Apache Top-Level Project, signaling production-readiness

- The Hadoop ecosystem expanded to include HBase, Hive, Pig, Flume, Sqoop, and other tools

- Hadoop was designed for horizontal scalability on commodity hardware, making Big Data processing economically viable

## Common Mistakes to Avoid

- Confusing Hadoop's origin—some students mistakenly believe Hadoop originated entirely from Google; it was actually adapted from Google's published research by Doug Cutting

- Misattributing the naming—Hadoop is NOT an acronym; it is named after a toy elephant

- Assuming Hadoop replaced RDBMS—Hadoop and RDBMS serve different purposes; Hadoop is designed for unstructured and semi-structured data at massive scale, not for transactional systems
