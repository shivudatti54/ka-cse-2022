# **TB1: Ch 5: 5.1- Big Data Analytics Revision Notes**

## **Introduction to Hadoop**

- Hadoop is a distributed computing framework used for processing large data sets.
- It was created by Doug Cutting and Mike Cafarella in 2005.
- Hadoop is why: it's open-source, scalable, fault-tolerant, and cost-effective.

## **Why Not RDBMS?**

- Relational databases (RDBMS) are not suitable for big data due to:
  - Limited scalability.
  - Inefficient data storage and retrieval.
  - High cost and maintenance.

## **RDBMS vs Hadoop**

- Key differences:
  - **Scalability**: Hadoop is designed to handle large data sets, while RDBMS is limited to a single database.
  - **Data storage**: Hadoop uses a distributed file system (HDFS), while RDBMS uses a centralized database.
  - **Cost**: Hadoop is cost-effective, while RDBMS can be expensive to maintain and scale.

## **Key Concepts**

- **Big Data**: large, complex data sets that require new processing techniques.
- **Hadoop Ecosystem**: Hadoop, HDFS, MapReduce, YARN, and other related components.
- **MapReduce**: a programming model and software framework used for processing large data sets.

## **Important Formulas and Definitions**

- **Hadoop Formula**: Hadoop = (N \* M) / (N \* M) + 1
  - Where N is the number of nodes, and M is the number of data sources.
- **Scalability**: the ability of a system to handle increased load.
- **Data Distribution**: the process of dividing data across multiple nodes.

## **Important Theorems**

- **Hadoop Theorem**: Hadoop is scalable because it can process large data sets in parallel.
- **MapReduce Theorem**: MapReduce is efficient because it allows for parallel processing of large data sets.
