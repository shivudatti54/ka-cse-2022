**5.10-5.12: Introduction to Hadoop**

**5.10: What is Big Data?**

### Definition

Big data refers to the vast amounts of structured and unstructured data that are generated by various sources, such as social media, sensors, and IoT devices. This data is characterized by its:

- **Volume**: Large amounts of data
- **Velocity**: High-speed generation and processing of data
- **Variety**: Different types of data, including structured, semi-structured, and unstructured data
- **Veracity**: Uncertainty and variability in data quality

### Characteristics

- Scalability: Ability to handle large amounts of data
- Flexibility: Ability to handle different types of data
- Speed: Ability to process data quickly

### Real-World Examples

- **Social Media**: Twitter, Facebook, Instagram
- **Sensors**: Temperature sensors, GPS trackers, weather stations
- **IoT Devices**: Smartphones, smart homes, smart cities

**5.11: What is Hadoop?**

### Definition

Hadoop is an open-source, distributed computing framework that is designed to process large amounts of data across a cluster of computers. It was created by Doug Cutting and Mike Cafarella in 2005.

### Key Features

- **Distributed Computing**: Breaks down large datasets into smaller chunks and processes them in parallel
- **Scalability**: Can handle large amounts of data and scale horizontally
- **Flexibility**: Supports various data formats and processing algorithms
- **Cost-Effective**: Open-source and allows for customization

### Components

- **Hadoop Distributed File System (HDFS)**: Stores data in a distributed manner
- **MapReduce**: Process data in parallel using a programming model
- **YARN (Yet Another Resource Negotiator)**: Manages resources and schedules jobs

### Real-World Examples

- **Netflix**: Uses Hadoop to process streaming data
- **Twitter**: Uses Hadoop to process tweet data
- **Amazon**: Uses Hadoop to process customer data

**5.12: Why Hadoop?**

### Advantages

- **Scalability**: Can handle large amounts of data
- **Flexibility**: Supports various data formats and processing algorithms
- **Cost-Effective**: Open-source and allows for customization
- **High Performance**: Can process large datasets quickly

### Disadvantages

- **Complexity**: Requires expertise in Hadoop and related technologies
- **Steep Learning Curve**: Can be challenging to learn and implement
- **Limited Support**: Not as widely supported as other technologies

### Real-World Examples

- **Data Integration**: Hadoop can integrate data from various sources
- **Data Analysis**: Hadoop can perform complex data analysis and machine learning tasks
- **Data Storage**: Hadoop can store large amounts of data in a scalable manner
