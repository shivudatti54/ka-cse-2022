### Revision Notes: 5.10-5.12 - Big Data Analytics

#### Introduction to Hadoop

##### Key Points

- **Definition of Hadoop**: Hadoop is an open-source, distributed computing framework used for processing and analyzing large data sets.
- **Why Hadoop**: Designed to handle massive amounts of data, provides scalability, fault-tolerance, and high performance.
- **Why Not RDBMS**: RDBMS is not designed for Big Data, lacks scalability, and is not suitable for handling large amounts of unstructured data.

##### RDBMS vs Hadoop

##### Key Points

- **RDBMS**: Relational Database Management System, designed for structured data, provides ACID compliance, and supports joins and transactions.
- **Hadoop**: Designed for unstructured and semi-structured data, provides high scalability, and supports MapReduce and HDFS.
- **Comparison**:

* RDBMS: ACID compliance, joins, transactions, structured data.
* Hadoop: High scalability, MapReduce, HDFS, unstructured data.

##### History of Hadoop

##### Key Points

- **History**: Developed in 2005 by Doug Cutting and Mike Cafarella, initially called "Nutch", later renamed to Hadoop.
- **Evolution**: Hadoop 1.0 (2006), Hadoop 2.0 (2012), Hadoop 3.0 (2016), and Hadoop 3.2 (2020).

### Important Formulas and Definitions

- **MapReduce**: A programming model used for processing large data sets, consists of Map and Reduce phases.
- **HDFS (Hadoop Distributed File System)**: A distributed file system used for storing and retrieving large data sets.
- **YARN (Yet Another Resource Negotiator)**: A resource management layer used for managing resources in Hadoop.

### Important Theorems

- **Gottschalk's Theorem**: States that the number of nodes in a network determines its total power consumption.
- **Amdahl's Law**: States that the maximum theoretical speedup that can be achieved by parallel processing is limited by the fraction of the program that cannot be parallelized.
