# **Ch 8: 8.1 - 8.8 - BIG DATA ANALYTICS**

### Introduction to Hadoop

- **What is Hadoop?**: A distributed computing framework for processing large data sets.
- **Why Hadoop?**: Scalable, fault-tolerant, and cost-effective for big data processing.
- **Why not RDBMS?**: Limited scalability and data model flexibility for big data.

### RDBMS vs Hadoop

- **RDBMS (Relational Database Management System)**:
  - Advantages:
    - Well-established data model
  - Disadvantages:
    - Limited scalability
    - Not designed for big data
- **Hadoop**:
  - Advantages:
    - Scalable
    - Fault-tolerant
    - Cost-effective
  - Disadvantages:
    - Complex architecture
    - Steeper learning curve

### History of Hadoop

- **Hadoop 0.9**: Released in 2004, the first version of Hadoop.
- **Hadoop 1.0**: Released in 2006, the first stable version of Hadoop.
- ** Hortonworks Data Platform**: Released in 2011, a commercial distribution of Hadoop.

### Key Concepts

- **Hadoop Distributed File System (HDFS)**: A distributed storage system for storing large data sets.
- **MapReduce**: A programming model for processing large data sets in parallel.
- **YARN (Yet Another Resource Negotiator)**: A resource management layer for Hadoop.

### Important Formulas and Definitions

- **NoSQL Data Model**: A data model that does not use the traditional table-based relational model.
- **Big Data**: A term used to describe large, complex data sets.
- **Scalability**: The ability of a system to handle increased load without a decrease in performance.

### Theorems and Concepts

- **MapReduce Theory**: A theoretical framework for processing large data sets in parallel.
- **Hadoop Architecture**: A software framework for processing large data sets in a distributed manner.
