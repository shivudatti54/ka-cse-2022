# Chapter 1 (Sections 1.1) Analysis & Design of Algorithms

### 1.1 Introduction to Algorithmic Problem Solving

- Algorithmic problem solving is the process of solving computational problems using a systematic approach.
- Algorithms provide a step-by-step procedure for solving problems.

### 1.1.1 Key Concepts

- **Algorithm**: A well-defined procedure that takes some input and produces a corresponding output.
- **Problem**: A question or task that requires a computational solution.
- **Solving Algorithm**: The process of finding a solution to a problem using an algorithm.

### 1.1.2 Importance of Algorithm Analysis

- Algorithm analysis is concerned with measuring the efficiency and effectiveness of an algorithm.
- Key metrics include:
  - **Big O notation**: A measure of an algorithm's time or space complexity.
  - **Time complexity**: The amount of time an algorithm takes to complete.
  - **Space complexity**: The amount of memory an algorithm uses.

### 1.1.3 Types of Algorithm Complexity

- **O(1)**: Constant time complexity (i.e., the time taken does not depend on the size of the input).
- **O(log n)**: Logarithmic time complexity (i.e., the time taken is proportional to the logarithm of the input size).
- **O(n)**: Linear time complexity (i.e., the time taken is proportional to the input size).
- **O(n log n)**: Linearithmic time complexity (i.e., the time taken is proportional to the product of the input size and its logarithm).
- **O(n^2)**: Quadratic time complexity (i.e., the time taken is proportional to the square of the input size).
- **O(2^n)**: Exponential time complexity (i.e., the time taken is proportional to 2 raised to the power of the input size).

### 1.1.4 Notations and Formulas

- **Big O notation**: O(f(n)) = f(n)
- **Little o notation**: o(f(n)) = f(n) + c for some constant c

### 1.1.5 Theorems

- **Master Theorem**: A tool for solving recurrence relations.
- **Recurrence relation**: An equation that describes the relationship between the size of an input and the time or space complexity of an algorithm.
