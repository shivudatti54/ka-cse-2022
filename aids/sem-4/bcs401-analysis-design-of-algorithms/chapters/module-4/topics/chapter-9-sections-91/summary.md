# **Revision Notes: Chapter 9 (Sections 9.1) Analysis & Design of Algorithms**

### Key Concepts

- **Dynamic Programming**: A method for solving complex problems by breaking them down into smaller subproblems, solving each subproblem only once, and storing the solutions to subproblems to avoid redundant computation.
- **Memoization**: A technique used in dynamic programming to store the solutions to subproblems in a memory-based data structure, such as an array or a hash table.
- **Bottom-up Approach**: A technique used in dynamic programming to build the solution to the original problem by solving the subproblems iteratively.

### Important Formulas and Definitions

- **Optimal Substructure**: A property of problems that allows them to be broken down into smaller subproblems, and the optimal solution to the original problem can be constructed from the optimal solutions of the subproblems.
- **Overlapping Subproblems**: A problem that has subproblems that have some overlap, meaning that some subproblems may be identical or have similar solutions.
- **Tabulation**: A technique used in dynamic programming to fill up a table in a bottom-up manner.

### Theorem: Optimal Substructure Theorem

- If a problem has optimal substructure, then it can be solved using dynamic programming.

### Key Points

- The Knapsack Problem is a classic example of a problem that can be solved using dynamic programming.
- The problem of finding the shortest path in a graph with non-negative weights is another example of a problem that can be solved using dynamic programming.
- The use of memoization can significantly improve the time complexity of dynamic programming algorithms.

### Important Theorems and Definitions

- **Knapsack Problem**: A problem in combinatorial optimization that involves finding the optimal way to pack a set of items of different weights and values into a knapsack of limited capacity.
- **Bellman-Ford Algorithm**: An algorithm used to find the shortest path from a source vertex to all other vertices in a weighted graph.

### Revision Tips

- Practice solving problems using dynamic programming techniques.
- Familiarize yourself with the concept of optimal substructure and overlapping subproblems.
- Learn to use memoization and tabulation to improve the time complexity of dynamic programming algorithms.
