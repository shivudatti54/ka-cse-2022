# **Chapter 6 (Sections 6.3) Revision Notes: Analysis & Design of Algorithms**

### Introduction

- Analysis of algorithms: measuring the amount of resources used by an algorithm
- Design of algorithms: creating efficient algorithms
- This chapter focuses on the analysis of balanced search trees, heaps, and heapsort

### Key Concepts

- **Time Complexity**:
  - Big O notation: upper bound on the number of steps
  - Upper and lower bounds: tightest possible bounds
  - Big Ω notation: lower bound on the number of steps
- **Space Complexity**:
  - Upper bound: maximum amount of space used by an algorithm
- **Balanced Search Trees**:
  - Definition: a search tree with a balance factor of -1 to 1
  - Properties: self-balancing, search, insert, delete operations
- **Heaps**:
  - Definition: a specialized tree-based data structure
  - Properties: complete binary tree, heap property: parent node is greater than or equal to children
- **Heapsort**:
  - Definition: an in-place comparison sort algorithm
  - Properties: O(n log n) time complexity, O(1) space complexity

### Important Formulas and Definitions

- **Binary Search Tree (BST) Properties**:
  - Height: h = log(n) + 1
  - Balance factor: |h_left - h_right| ≤ 1
- **Heap Properties**:
  - Heap size: n
  - Heap factor: n/2
- **Heapsort Algorithm**:
  - Time complexity: O(n log n)
  - Space complexity: O(1)

### Important Theorems

- **Heapsort Theorem**: heapsort is a comparison sort algorithm
- **Balanced BST Theorem**: a balanced BST satisfies the BST property

### Quick Revision Tips

- Focus on time and space complexity
- Understand the properties of balanced search trees, heaps, and heapsort
