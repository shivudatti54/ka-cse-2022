# **Chapter 7: Analysis & Design of Algorithms - Sections 7.1**

## **Key Concepts**

- **Big O Notation**: Upper and lower bounds for the time complexity of an algorithm.
  - Big O (Upper bound): O(n) = O(1), O(n), O(n log n), O(n^2), etc.
  - Big Ω (Lower bound): Ω(n) = Ω(1), Ω(n), Ω(n log n), Ω(n^2), etc.
- **Time Complexity Analysis**: Measuring the time taken by an algorithm to complete.
  - Measuring time complexity: counting the number of operations performed.
- **Space Complexity Analysis**: Measuring the space used by an algorithm to complete.
  - Measuring space complexity: counting the number of data structures used.

## **Important Formulas and Definitions**

- **Big O Notation Rules**:
  - Constant multiple rule: O(a + b) = O(max(a, b))
  - Sum rule: O(a + b) = O(a) + O(b)
  - Product rule: O(ab) = O(a) × O(b)
- **Time Complexity Notations**:
  - O(1) - Constant time complexity
  - O(log n) - Logarithmic time complexity
  - O(n) - Linear time complexity
  - O(n log n) - Linearithmic time complexity
  - O(n^2) - Quadratic time complexity
  - O(2^n) - Exponential time complexity
- **Space Complexity Notations**:
  - O(1) - Constant space complexity
  - O(log n) - Logarithmic space complexity
  - O(n) - Linear space complexity
  - O(n log n) - Linearithmic space complexity
  - O(n^2) - Quadratic space complexity

## **Important Theorems**

- **Master Theorem**: A general formula for solving recurrence relations.
- **Recurrence Relation Solving Techniques**: Substitution, iteration, and recursive relations.
