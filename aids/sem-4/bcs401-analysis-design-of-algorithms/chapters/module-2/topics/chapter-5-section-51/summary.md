# **Revision Notes: Chapter 5 (Section 5.1) - Exhaustive Search**

## **Key Concepts**

- **Exhaustive Search**: A brute-force approach that evaluates all possible solutions to a problem.
- **Travelling Salesman Problem (TSP)**: A classic example of an NP-hard problem.
- **Backtracking**: A technique used in exhaustive search to explore all possible solutions.

## **Formulas and Definitions**

- **TSP Formula**: The TSP can be represented as an integer linear programming problem (ILP) or a constraint satisfaction problem (CSP).
- **Greedy Algorithm Formula**: The greedy algorithm for TSP involves making locally optimal choices to find a near-optimal solution.
- **Hamiltonian Cycle**: A cycle in a graph that visits each vertex exactly once.

## **Theorems**

- **Haitz Theorem**: The TSP is NP-hard, meaning that the running time of algorithms for solving TSP increases rapidly as the size of the input increases.
- **Christofides Algorithm**: A polynomial-time algorithm for approximating the TSP solution.

## **Important Results**

- **NP-Completeness**: The TSP is NP-complete, meaning that there is no known efficient algorithm for solving TSP exactly for large instances.
- **Approximation Algorithms**: There are several approximation algorithms for TSP, including the greedy algorithm and the Christofides algorithm.

## **Key Terms**

- **NP-Hardness**: A problem is NP-hard if all known algorithms for solving it have an exponential time complexity.
- **Approximation Algorithm**: An algorithm that finds a solution that is close to the optimal solution but does not guarantee an optimal solution.
