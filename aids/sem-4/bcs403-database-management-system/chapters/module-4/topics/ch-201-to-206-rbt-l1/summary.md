### DATABASE MANAGEMENT SYSTEM (RBT L1)

#### Module: 8 hours

#### Ch 20.1 to 20.6

**Key Concepts:**

- **Database Management System (DBMS):** A software system that manages and controls access to and manipulation of data in a database.
- **Database:** A collection of organized data that is stored in a way that allows for efficient retrieval and manipulation.
- **Database Model:** A conceptual representation of a database, including entities, attributes, and relationships.
- **Database Schema:** A visual representation of the database structure, including entities, attributes, and relationships.

**Database Management Systems (DBMS):**

- **Types:**
  - Relational DBMS (RDBMS)
  - Object-Oriented DBMS (OODB)
  - Hierarchical DBMS
  - Network DBMS
- **Characteristics:**
  - Data independence
  - Controlled access
  - Data integrity
  - Scalability

**Database Design:**

- **Entity-Relationship Model (ERM):** A conceptual representation of a database, including entities, attributes, and relationships.
- **Normalization:** The process of organizing data in a database to minimize data redundancy and improve data integrity.
- **Denormalization:** The process of organizing data in a database to improve query performance.

**Database Control Systems:**

- **Access Control:** The process of controlling who can access the database and what actions they can perform.
- **Security:** The process of protecting the database from unauthorized access or tampering.
- **Backup and Recovery:** The process of backing up the database and recovering it in case of data loss or failure.

**Important Formulas and Theorems:**

- **BCNF (Boyce-Codd Normal Form):** A higher level of normalization than 3NF.
- **3NF (Third Normal Form):** A normalization form that minimizes data redundancy and improves data integrity.
- **Join Operation:** The process of combining data from multiple tables in a database.

**Important Definitions:**

- **Primary Key:** A unique identifier for each record in a table.
- **Foreign Key:** A field in a table that references the primary key of another table.
- **Normalization Factor:** A measure of the degree of normalization in a database.
