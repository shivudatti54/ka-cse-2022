# **Textbook 1: Ch 7.1 to 7.3 Database Management System Revision Notes**

## **Key Concepts**

- **Database Management System (DBMS)**:
  - Definition: Software that manages and supports the use of databases
  - Role: Provides a way to interact with databases using standardized commands
- **Database**:
  - Definition: Collection of organized data
  - Characteristics: Structure, data integrity, and data security
- **DBMS Features**:
  - Data definition, control, and manipulation
  - Data independence and platform independence
  - Data integrity and security

## **Important Formulas and Theorems**

- **BCNF (Boyce-Codd Normal Form)**:
  - Definition: A relational schema is in BCNF if and only if each non-key attribute depends on the entire primary key
- **3NF (Third Normal Form)**:
  - Definition: A relational schema is in 3NF if and only if each non-key attribute depends on the entire primary key and there are no transitive dependencies
- **2NF (Second Normal Form)**:
  - Definition: A relational schema is in 2NF if and only if each non-key attribute depends on the entire primary key and there are no partial dependencies
- **Normal Form Theorems**:
  - Definition: A set of rules for ensuring data consistency and reducing redundancy in relational databases

## **Important Definitions**

- **Primary Key**: A unique identifier for each record in a database
- **Foreign Key**: A field in a table that references the primary key in another table
- **Normalization**: The process of organizing data in a database to minimize data redundancy and improve data integrity
- **Denormalization**: The process of deliberately allowing redundant data in a database for performance reasons

## **Revision Tips**

- Understand the role of a DBMS and its features
- Learn the different types of normalization (BCNF, 3NF, 2NF)
- Familiarize yourself with normal form theorems and their applications
- Practice applying these concepts to real-world database scenarios.
