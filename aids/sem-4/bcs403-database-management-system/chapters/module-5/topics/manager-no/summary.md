## Database Management System

### Key Points

- **Manager No**: A unique identifier for each record in a database, used to manage and retrieve data efficiently.
- **Indexing**: The process of creating a data structure that allows for faster data retrieval and manipulation.
- **Key**: A column or set of columns that uniquely identifies each record in a database.
- **Primary Key**: A unique identifier for each record, ensuring data integrity and preventing duplicate records.
- **Foreign Key**: A column or set of columns that references the primary key of another table.

### Important Formulas and Definitions

- **Indexing Formula**: `I = (R \* C) / (N \* B)`, where I = number of indexes, R = number of records, C = number of columns, N = number of tuples, and B = block size.
- **Database Normalization**: A process of organizing data in a database to minimize data redundancy and improve data integrity.
- **Data Consistency**: The property of a database that ensures all data is consistent and accurate.

### Important Theorems

- **First Normal Form (1NF)**: Each table cell must contain a single value, eliminating data redundancy.
- **Second Normal Form (2NF)**: Each non-key attribute in a table must depend on the entire primary key.
- **Third Normal Form (3NF)**: If a table is in 2NF, and a non-key attribute depends on another non-key attribute, then it should be moved to a separate table.
