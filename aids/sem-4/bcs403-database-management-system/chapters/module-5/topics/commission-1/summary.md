# **COMMISSION) 1 Revision Notes**

## **Database Management System**

### Definitions

- **Commission**: A payment or fee paid to an individual or organization for their work or services.
- **Database**: A collection of organized data that is stored in a way that allows for efficient retrieval and manipulation.
- **Database Management System (DBMS)**: A software system that manages and controls the creation, maintenance, and modification of databases.

### Formulas and Theorems

- **Commission Formula**: Commission = (Total Sales x Commission Rate) / 100
- **Discount Formula**: Discount = (Original Price x Discount Rate) / 100
- **Theorem of Total Variation**: Δy = Δx + Δf + Δg, where Δy is the total change in y, Δx is the change in x, Δf is the change in the function f, and Δg is the change in the function g.

### Key Points

- **Types of Databases**: Relational databases, NoSQL databases, Graph databases, etc.
- **Database Design**: Entity-relationship model, normalization, denormalization, etc.
- **Database Management**: Data storage, data retrieval, data manipulation, etc.
- **Database Security**: Authentication, authorization, encryption, etc.
- **Database Performance**: Query optimization, indexing, caching, etc.

### Important Terms

- **Normalization**: The process of organizing data into a structured format to minimize data redundancy.
- **Denormalization**: The process of adding redundant data to improve query performance.
- **Indexing**: The process of creating a data structure that allows for efficient querying of data.
- **Caching**: The process of storing frequently accessed data in a faster, more accessible location.

### Key Concepts

- **Data Modeling**: The process of creating a conceptual representation of a database.
- **Data Warehousing**: The process of creating a centralized repository of data for analysis and reporting.
- **Business Intelligence**: The process of using data to support business decision-making.
