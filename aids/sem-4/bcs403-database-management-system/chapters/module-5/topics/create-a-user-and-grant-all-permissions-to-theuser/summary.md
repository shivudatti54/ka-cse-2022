# **Database Management System: Create a User and Grant All Permissions**

### Overview

- Create a user in a database management system to assign permissions
- Grant all permissions to the user to ensure they have full access to the database

### Key Points

- **CREATE USER Statement**: Used to create a new user in the database
- **GRANT Statement**: Used to assign permissions to a user
- **ALL PRIVILEGES**: Grants all privileges to a user
- **Privilege Levels**:
  - SELECT: view data
  - INSERT: add new data
  - UPDATE: modify existing data
  - DELETE: delete data
  - CREATE: create new tables, indexes, etc.
  - DROP: drop tables, indexes, etc.
- **Granting All Permissions**: Use `GRANT ALL PRIVILEGES ON DATABASE TO 'username';`

### Important Formulas and Definitions

- **Privilege**: A right or authority granted to a user or role
- **Role**: A set of privileges assigned to a user or group
- **Database**: A collection of related data stored in a database management system

### Theorems

- **Data Integrity**: The principle of ensuring data consistency and accuracy
- **Data Security**: The principle of protecting data from unauthorized access or modification

### Revision Tips

- Practice creating users and granting permissions
- Review privilege levels and their corresponding actions
- Understand the importance of data integrity and security in database management
