# **Database Management System Revision Notes**

### **Definitions and Key Concepts**

- **Database**: A collection of organized data stored in a way that allows for efficient retrieval and manipulation.
- **Database Management System (DBMS)**: A software system that manages and controls access to a database.
- **Schema**: The overall structure of a database, including the relationships between different tables.

### **Database Models**

- **Relation Model**: A model that represents a database as a collection of interconnected tables.
- **Entity-Relationship Model**: A model that represents a database as a collection of entities and their relationships.

### **Database Normalization**

- **First Normal Form (1NF)**: A normalized form where each table cell contains a single value.
- **Second Normal Form (2NF)**: A normalized form where each non-key attribute depends on the entire primary key.
- **Third Normal Form (3NF)**: A normalized form where each non-key attribute depends on the entire primary key and has no transitive dependencies.

### **Database Design**

- **Entity**: A real-world concept that is represented in a database as a table.
- **Attribute**: A characteristic of an entity that is stored in a table.
- **Key**: A unique identifier for an entity or attribute.

### **SQL and Querying**

- **SQL (Structured Query Language)**: A language used to manage and manipulate databases.
- **SELECT Statement**: Retrieves data from a database table.
- **WHERE Clause**: Filters data based on conditions.

### **Important Formulas and Theorems**

- **Data Integrity**: The property of a database that ensures data is consistent and accurate.
- **Consistency**: The property of a database that ensures data remains consistent over time.
- **Concurrency Control**: A mechanism that ensures multiple users can access a database simultaneously without conflicts.

### **Important Theorems**

- **ACID (Atomicity, Consistency, Isolation, Durability)**: A set of principles that ensure database transactions are processed reliably.

This summary provides a concise overview of the key concepts and formulas in Database Management System. It is designed to be a quick revision guide before exams.
