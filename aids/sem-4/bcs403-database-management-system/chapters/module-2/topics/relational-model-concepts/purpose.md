# Learning Objectives

After studying this topic, you should be able to:

1. Define and explain the fundamental components of the relational model including relations, tuples, attributes, and domains.

2. List and describe the five essential properties that characterize a valid relation in the relational model.

3. Differentiate between various types of keys: super key, candidate key, primary key, foreign key, and composite key with appropriate examples.

4. Understand the concepts of relational schema and relation instance, and explain how they differ from each other.

5. Explain the purpose and handling of NULL values in relational databases and discuss three-valued logic implications.

6. Identify and describe the different integrity constraints supported by the relational model, including domain, key, entity integrity, and referential integrity constraints.

7. Apply the concepts of degree and cardinality to analyze given relations and determine their characteristics.

8. Evaluate whether a given table structure qualifies as a valid relation based on the relational model's defining properties.
