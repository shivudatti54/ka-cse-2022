## **Introduction**

Database Management System (DBMS) is a software system that allows you to define, create, maintain, and manipulate databases. A DBMS provides a way to store, organize, and retrieve data efficiently.

## **8.1 Database Entities**

- A real-world entity is a concept, object, or thing that has a distinct identity.
- A database entity is a table in a database that represents a real-world entity.
- Examples of database entities:
  - Customers
  - Orders
  - Products

## **8.2 Database Attributes**

- A database attribute is a characteristic or feature of a database entity.
- Attributes define the properties or features of an entity.
- Examples of database attributes:
  - Customer Name
  - Customer Address
  - Order Date

## **8.3 Relational Model**

- The relational model is a data model that represents data as relationships between entities.
- It uses tables and fields to store data.
- The relational model is based on the concept of normalization.

## **8.4 Normalization**

- Normalization is the process of organizing data in a database to minimize data redundancy and improve data integrity.
- It involves dividing large tables into smaller tables and linking them through relationships.
- Normalization benefits:
  - Reduces data redundancy
  - Improves data integrity
  - Simplifies data manipulation

## **8.5 Data Modeling**

- Data modeling is the process of creating a conceptual representation of a database.
- It involves defining the relationships between entities and attributes.
- Data modeling benefits:
  - Improves data understanding
  - Enhances data integrity
  - Facilitates database design

## **Key Concepts**

- Database entities
- Database attributes
- Relational model
- Normalization
- Data modeling

## **Example**

Suppose we have a database that represents a university's student database. We can define the following entities:

- Students
- Courses
- Enrollments

The attributes for each entity can be:

- Students:
  - Student ID
  - Name
  - Email
- Courses:
  - Course ID
  - Course Name
  - Credits
- Enrollments:
  - Enrollment ID
  - Student ID
  - Course ID
  - Grade

We can create a relational model that represents the relationships between these entities:

- Each student can enroll in multiple courses.
- Each course can have multiple students enrolled.
- Each enrollment is associated with one student and one course.

We can normalize this database to minimize data redundancy and improve data integrity.

This is a basic example of a database management system. Database management systems are complex and involve many more concepts, but this study material covers the basics of database entities, attributes, relational model, normalization, and data modeling.
