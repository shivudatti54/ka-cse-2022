# **Database Management System Revision Notes**

## **Definition and Objectives**

- Definition: A database management system (DBMS) is a software that allows you to define, create, maintain, and manipulate databases.
- Objectives:
  - Store and retrieve data efficiently
  - Ensure data security and integrity
  - Provide data backup and recovery

## **Database Management System Components**

- **Hardware**: Computer hardware that supports the DBMS
- **Software**: DBMS itself
- **Database**: Collection of organized data
- **User**: Person who interacts with the DBMS

## **Database Management System Types**

- **Relational Database Management System (RDBMS)**: Organizes data into tables with defined relationships
- **Object-Oriented Database Management System (OODBMS)**: Organizes data into objects with defined relationships
- **Hierarchical Database Management System (HDBMS)**: Organizes data into a hierarchical structure
- **Network Database Management System (NDDBMS)**: Organizes data into a network structure

## **Database Schema**

- **Entity Relationship Diagram (ERD)**: Visual representation of data relationships
- **Database Schema**: Overall structure of the database

## **Database Normalization**

- **First Normal Form (1NF)**: Each table cell should contain a single value
- **Second Normal Form (2NF)**: Each non-key attribute should depend on the entire primary key
- **Third Normal Form (3NF)**: Each non-key attribute should depend on the entire primary key

## **Database Performance and Optimization**

- **Indexing**: Improves query performance by allowing faster data retrieval
- **Normalization**: Improves data integrity and reduces data redundancy
- **Data Distribution**: Optimizes data storage and retrieval

## **Important Formulas and Definitions**

- **Data Integrity**: Ensures data consistency and accuracy
- **Data Security**: Protects data from unauthorized access
- **Data Backup**: Creates copies of data to ensure recovery in case of data loss
- **Data Recovery**: Restores data from backup or redundant data

## **Theorems and Concepts**

- **Database Theorem**: A set of rules that govern the design and implementation of DBMS
- **Data Relationships**: The connections between different data entities
