# Database Management System

### Overview

- Module: 8 hours
- Topics: Ch 9.1 to 9.2 Textbook 2: 3.5 RBT: L1

### Key Points

#### 1. Types of Databases

- Relational databases (RDBMS)
- Non-relational databases (NoSQL)
- Time-series databases
- Graph databases
- XML databases

#### 2. Database Management Systems (DBMS)

- Definition: software that manages and controls access to a database.
- Features:
  - Data definition
  - Data manipulation
  - Data control
  - Data retrieval

#### 3. Relational Database Management System (RDBMS)

- Definition: stores data in tables with defined relationships between them.
- Characteristics:
  - Schema-based
  - SQL (Structured Query Language) based
  - ACID (Atomicity, Consistency, Isolation, Durability) properties

#### 4. Normal Forms

- Definition: a set of rules to minimize data redundancy and improve data integrity.
- Normal forms:
  - First normal form (1NF)
  - Second normal form (2NF)
  - Third normal form (3NF)
  - Boyce-Codd normal form (BCNF)

#### 5. Database Design

- Entity-relationship modeling
- Entity, attribute, and relationship diagrams
- Data normalization

#### 6. SQL

- Definition: a programming language for managing relational databases.
- Syntax:
  - SELECT
  - INSERT
  - UPDATE
  - DELETE
  - JOIN

#### 7. Database Deployment

- Types:
  - Client-server
  - Peer-to-peer
  - Distributed
- Deployment models:
  - On-premises
  - Cloud-based
  - Hybrid

#### 8. Database Security

- Types:
  - Authentication
  - Authorization
- Security measures:
  - Encryption
  - Access control
  - Backup and recovery

### Important Formulas and Definitions

#### Database Size Formulas

- Total data size = (Number of records) \* (Average record size)
- Average record size = (Sum of all record sizes) / (Number of records)

#### Data Normalization Formulas

- Number of attributes in a table = Number of rows in a table
- Number of rows in a table = Number of columns in a table
