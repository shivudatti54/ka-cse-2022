### Three Schema Architecture and Data Independence, Database Languages, and Interfaces, The Database System Environment

#### I. Three Schema Architecture

- **Schema**: A schema is a conceptual representation of the database structure.
- **Entity-Relationship Model (ERM)**: A technique for modeling relationships between entities in a database.
- **Data Independence**: The ability of a database to adapt to changes in the schema without affecting the application.

#### II. Data Independence

- **Vertical Data Independence**: The ability of a database to change the schema without affecting the application.
- **Horizontal Data Independence**: The ability of a database to change the schema without affecting the application or other databases.
- **Local Data Independence**: The ability of a database to change the schema without affecting the application or other databases, but with an impact on the local data.

#### III. Database Languages

- **Query Language**: A language used to specify queries to the database.
- **Data Definition Language (DDL)**: A language used to define the structure of the database.
- **Data Manipulation Language (DML)**: A language used to manipulate data in the database.
- **Data Control Language (DCL)**: A language used to control access to the database.

#### IV. Interfaces

- **Application Programming Interface (API)**: A set of rules and protocols used to access the database from an application.
- **Database Interface**: A set of rules and protocols used to access the database from another database.

#### V. The Database System Environment

- **Database Management System (DBMS)**: A software system that manages a database.
- **Database System Call (DSC)**: A set of functions provided by the DBMS to access the database.
- **Database Interface Call (DIC)**: A set of functions provided by the DBMS to access the database through an interface.

#### Key Formulas and Definitions

- **First Normal Form (1NF)**: A normalized form of a database where each table has a unique primary key.
- **Second Normal Form (2NF)**: A normalized form of a database where each non-key attribute depends on the entire primary key.
- **Third Normal Form (3NF)**: A normalized form of a database where each non-key attribute depends on the entire primary key, and there are no transitive dependencies.
- **Entity**: A person, place, or thing in the real world.
- **Attribute**: A characteristic or feature of an entity.
- **Relationship**: A connection between two or more entities.

#### Theorems

- **Database Integrity**: The quality of a database that ensures the accuracy, consistency, and reliability of the data.
- **Data Security**: The ability of a database to ensure the confidentiality, integrity, and availability of the data.
