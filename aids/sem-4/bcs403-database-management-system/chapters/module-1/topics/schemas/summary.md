# Schemas

## Revision Notes

### Database Management System

#### Module: No. of Hours: 8

### Definition

- A schema is a conceptual representation of a database, defining the relationships between entities and attributes.

### Key Points

- A schema defines the structure of a database, including tables, fields, and relationships.
- It serves as a blueprint for database design and development.
- Schemas can be physical (DBMS-specific) or conceptual (independent of DBMS).

### Important Formulas and Theorems

- **First Normal Form (1NF)**: Each table cell must contain a single value.
- **Second Normal Form (2NF)**: Each non-key attribute must depend on the entire primary key.
- **Third Normal Form (3NF)**: If a table is in 2NF, and a non-key attribute depends on another non-key attribute, then it should be moved to a separate table.

### Key Concepts

- Entity-Attribute-Value (EAV) model: Describes a schema that stores data in key-value pairs.
- Entity-Relationship (ER) model: Represents relationships between entities and attributes.
- Normalization: A process of organizing data to minimize data redundancy and improve data integrity.

### Key Terms

- Entity: A real-world object or concept that has a unique identity.
- Attribute: A characteristic or property of an entity.
- Primary key: A unique identifier for an entity or table.
- Foreign key: A field that references the primary key of another table.
