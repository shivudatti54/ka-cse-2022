# Database Management System

### Chapter 1: Introduction to Database Management System (10 hours)

### 1.1: Definition of Database Management System

- Definition: A software system that allows an organization to store, manage, and retrieve data in a database.
- Characteristics: Centralized data storage, data manipulation, and data retrieval.

### 1.2: Types of Database Management Systems

- Relational Database Management Systems (RDBMS)
- Object-Oriented Database Management Systems (OODBMS)
- Time-Series Database Management Systems (TSDBMS)

### 1.3: Advantages and Disadvantages of DBMS

- Advantages:
  - Data organization and retrieval
  - Data security and access control
  - Data backup and recovery
- Disadvantages:
  - Complex implementation
  - High cost of maintenance

### 1.4: Database Organizations

- Hierarchical
- Network
- Relational

### 1.5: Database Types

- Relational databases
- Object-oriented databases
- Time-series databases

### 1.6: Database Design

- Entity-relationship model
- Entity-relationship diagrams (ERDs)
- Normalization

### 1.7: Database Normalization

- Rules of normalization:
  - First normal form (1NF)
  - Second normal form (2NF)
  - Third normal form (3NF)
- Normalization process

### 1.8: Database Testing

- Types of database testing:
  - Unit testing
  - Integration testing
  - System testing

**Important Formulas and Definitions:**

- Normalization formula: 1NF = ∑(R) ∩ ∫(R) = ∅
- Entity-relationship diagram (ERD) notation: E(R) = (Entity, Attributes, Primary Key, Foreign Key)
- Data integrity notation: DC (Data Consistency)

**Theorems:**

- Relational algebra theorem: RA = {σ, π, ∪, ∩, ∘}
- Database query language theorem: SQL (Structured Query Language)
