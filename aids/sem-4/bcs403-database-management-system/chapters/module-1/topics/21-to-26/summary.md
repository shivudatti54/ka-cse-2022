# **Database Management System Revision Notes (2.1 to 2.6)**

## **2.1 Database Systems**

- Definition: A database is a collection of organized data.
- Characteristics:
  - Data integration
  - Data storage
  - Data retrieval
  - Data manipulation
- Types of databases:
  - Relational databases
  - NoSQL databases
  - Graph databases

## **2.2 Database Management System (DBMS)**

- Definition: A DBMS is a software system that manages a database.
- Features:
  - Data definition language (DDL)
  - Data manipulation language (DML)
  - Query language (SQL)
  - Transaction management
- DBMS types:
  - Relational DBMS (RDBMS)
  - Object-oriented DBMS (OODBMS)
  - Extensible markup language (XML) DBMS

## **2.3 Data Storage**

- Data storage devices:
  - Hard disk drives (HDD)
  - Solid-state drives (SSD)
  - Flash storage
- Data storage concepts:
  - Relational data
  - Non-relational data
  - Hierarchical data

## **2.4 Data Retrieval**

- Data retrieval techniques:
  - SQL queries
  - Full-text search
  - Data mining
- Data retrieval systems:
  - Relational databases
  - NoSQL databases

## **2.5 Data Manipulation**

- Data manipulation techniques:
  - Data insertion
  - Data deletion
  - Data updating
- Data manipulation systems:
  - Relational databases
  - NoSQL databases

## **2.6 Database Security**

- Database security concepts:
  - Authentication
  - Authorization
  - Access control
- Database security techniques:
  - Encryption
  - Firewalls
  - Intrusion detection systems

## **Important Formulas and Definitions**

- **Normalization**: The process of removing data redundancy.
- **Denormalization**: The process of adding data redundancy for performance.
- **Referential integrity**: The relationship between tables in a relational database.

## **Theorems and Concepts**

- **Entity-relationship model (ERM)**: A conceptual model for designing databases.
- **Data modeling**: The process of creating a conceptual model for a database.
