## **Introduction**

In this module, we will explore the fundamental concepts of Database Management System (DBMS). DBMS is a crucial component of modern computing systems, enabling organizations to store, manage, and retrieve large amounts of data efficiently.

## **2.1: Introduction to DBMS**

- **Definition:** DBMS is a software system that allows users to define, create, maintain, and manipulate databases.
- **Characteristics:**
  - Data independence
  - Logical independence
  - Physical independence
  - Autonomy
- **Example:** A bank's customer database management system.

## **2.2: Types of DBMS**

- **Relational DBMS (RDBMS):** Organizes data into tables with well-defined relationships.
- **Object-Oriented DBMS (OODB):** Stores data as objects that contain data and methods to manipulate it.
- **Hierarchical DBMS:** Organizes data into a hierarchical structure.
- **Network DBMS:** Organizes data into a network of interconnected records.
- **Object-Relational DBMS (ORDBMS):** Combines the features of RDBMS and OODB.

## **2.3: DBMS Components**

- **External Data Subsystem:** Contains the database and provides interfaces for data input and output.
- **Internal Data Subsystem:** Manages the database, including data storage, retrieval, and manipulation.
- **Control Subsystem:** Provides interfaces for users to interact with the DBMS.
- **Application Subsystem:** Uses the DBMS to perform specific tasks and operations.

## **2.4: DBMS Applications**

- **Data Warehousing:** Stores historical data for analysis and reporting.
- **Business Intelligence:** Uses DBMS to analyze and report on business data.
- **Decision Support Systems:** Uses DBMS to support decision-making by analyzing and reporting on data.
- **Expert Systems:** Uses DBMS to mimic the decision-making abilities of a human expert.

## **2.5: DBMS Hardware Requirements**

- **CPU:** Handles data processing and execution of database operations.
- **Memory:** Stores data and program instructions.
- **Storage:** Holds the database and its related data.
- **Input/Output Devices:** Provides interfaces for data input and output.

## **2.6: DBMS Software Requirements**

- **Database Management System Software:** Manages the database and provides interfaces for data input and output.
- **Programming Languages:** Used to develop applications that interact with the DBMS.
- **Database Languages:** Used to define and manipulate database structures and data.

## **Conclusion**

In this module, we have explored the fundamental concepts of DBMS, including its introduction, types, components, applications, hardware requirements, and software requirements. Understanding these concepts is crucial for anyone interested in designing, implementing, and managing databases.
