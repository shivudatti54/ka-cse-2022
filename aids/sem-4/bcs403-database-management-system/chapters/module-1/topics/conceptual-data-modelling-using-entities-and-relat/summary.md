# **Conceptual Data Modelling using Entities and Relationships**

## **Module: Database Management System, 8 hours**

### Key Concepts

- **Entity Types**: A set of instances that share common attributes and behaviors, represented by a single entity name.
- **Entity Sets**: A collection of related entity types, represented by a plural form of the entity type name.
- **Structural Constraints**: Rules that govern the relationships between entities, such as cardinality and multiplicity.
- **Weak Entity Types**: Entity types that have a partial or dependent identity, relying on a related entity type for their existence.

### Entity-Relationship Diagrams (ER Diagrams)

- **Entities**: Represented by rectangles with entity names and attributes.
- **Relationships**: Represented by lines connecting entities, indicating the type of relationship.
- **Attributes**: Represented by the attributes of each entity.
- **Primary Keys**: Unique identifiers for each entity, represented by a single attribute.

### Theories and Formulas

- **First Normal Form (1NF)**: A table is in 1NF if each cell contains a single value, and each column contains only atomic values.
- **Second Normal Form (2NF)**: A table is in 2NF if it is in 1NF, and each non-key attribute depends on the entire primary key.
- **Third Normal Form (3NF)**: A table is in 3NF if it is in 2NF, and there are no transitive dependencies.

### Important Definitions

- **Entity Relationship**: A relationship between two or more entities.
- **Attribute**: A characteristic or property of an entity.
- **Key**: A unique identifier for an entity.

### Revision Tips

- Identify the main entities and relationships in a given problem.
- Apply structural constraints to ensure data consistency.
- Use ER diagrams to visualize the relationships between entities.
- Practice applying 1NF, 2NF, and 3NF to optimize database design.
