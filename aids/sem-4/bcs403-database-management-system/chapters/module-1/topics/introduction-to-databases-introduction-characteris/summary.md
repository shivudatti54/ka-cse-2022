# Introduction to Databases: Revision Notes

### Introduction

- Definition: A collection of interrelated data that are stored in a way that allows for efficient retrieval and manipulation.
- Purpose: To manage and manipulate data in a structured and controlled manner.

### Characteristics of Database Approach

- **Data Independence**: Data is organized and managed as a whole, not as individual pieces.
- **Control Independence**: Access control is centralized, ensuring that data is protected and consistent.
- **Functional Independence**: Each user has a unique view of the data, but the underlying data remains the same.
- **Logical Independence**: Data is organized in a way that makes it easy to manipulate and analyze.

### Advantages of Using the DBMS Approach

- **Improved Data Security**: Centralized access control ensures that data is protected.
- **Increased Data Integrity**: Data is consistently managed and enforced.
- **Enhanced Data Flexibility**: Data can be easily modified and updated.
- **Efficient Data Retrieval**: Data can be quickly and accurately retrieved.

### History of Database Applications

- **Pre-Computer Era**: Data was stored in paper-based records.
- **First Database Systems (1950s-1960s)**: Early database systems were developed, but they were limited in scope.
- **Relational Database Management System (RDBMS) (1970s)**: RDBMS revolutionized database management with its relational model.
- **Modern Database Systems (1980s-Present)**: The development of modern database systems, including object-oriented and NoSQL databases.

### Important Formulas, Definitions, and Theorems

- **Data Independence Theorem**: A data independence theorem states that the data is independent of the database management system.
- **Data Integrity Theorem**: A data integrity theorem states that the database management system ensures the consistency of the data.
- **ACID (Atomicity, Consistency, Isolation, Durability) Properties**: A set of properties that ensure database transactions are processed reliably.
