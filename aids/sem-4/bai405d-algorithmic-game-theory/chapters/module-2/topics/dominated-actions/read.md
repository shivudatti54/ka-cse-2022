# Interaction in Graphics

## Introduction to Interaction

Interaction in computer graphics refers to the communication between a user and a computer system through various input devices, enabling real-time manipulation and control of graphical elements. Unlike static images, interactive graphics allow users to dynamically change what is displayed, creating a two-way flow of information between the user and the system.

The fundamental concept of interaction transforms computer graphics from a passive display medium into an active tool for tasks such as design, simulation, gaming, and data visualization. This bidirectional communication is essential for modern graphical applications.

## Input Devices

Input devices can be categorized based on their functionality and the type of data they provide to the system.

### Logical Device Classification

From a programmer's perspective, input devices are often classified logically rather than physically:

| Logical Device Type | Description                     | Physical Examples                    |
| ------------------- | ------------------------------- | ------------------------------------ |
| **String**          | Inputs text data                | Keyboard, touchscreen keyboard       |
| **Locator**         | Provides coordinate positions   | Mouse, touchpad, graphics tablet     |
| **Pick**            | Selects graphical objects       | Mouse click, touch input             |
| **Choice**          | Selects from predefined options | Function keys, menu selections       |
| **Valuator**        | Inputs scalar values            | Dial, slider, pressure-sensitive pen |

### Physical Input Devices

**Keyboards:** The most common text input device. In graphics, special keys are often used for specific functions (arrow keys for navigation, function keys for tools).

**Mouse:** A relative positioning device that moves a cursor on screen. It typically has buttons for selection:

```
         ┌─────────────┐
         │   Mouse     │
         │             │
         │  Left       │  Movement detected
         │  Button ┌───┼──> ΔX, ΔY to computer
         │         │   │
         │  Right  │   │
         │  Button │   │
         └─────────┴───┘
```

**Touchscreens:** Absolute positioning devices that directly map touch points to screen coordinates. They support multi-touch gestures (pinch, swipe, rotate).

**Graphics Tablets:** High-precision absolute pointing devices used by artists and designers. The tablet surface maps directly to the screen area.

**3D Input Devices:** Include space balls, data gloves, and VR controllers that provide six degrees of freedom (position and orientation).

## Clients and Servers in Graphics Systems

In networked graphics environments, the concept of clients and servers is crucial:

**Server:** The program that controls the display(s) and input devices. It maintains the display and handles input events.

**Client:** The application program that requests services from the server. It contains the specific graphics code.

This separation allows:

- Multiple clients to share display resources
- Graphics applications to run across networks
- Different computers to handle computation and display

```
Client-Server Architecture:
┌─────────────────┐    Network     ┌─────────────────┐
│                 │  Connection    │                 │
│    Client       │◄──────────────►│    Server       │
│  (Application)  │                │  (Display/Input)│
│                 │                │                 │
└─────────────────┘                └─────────────────┘
```

## Display Lists

Display lists are stored collections of graphics commands that can be executed repeatedly. They serve as a fundamental tool for efficient rendering and interaction.

### Creating and Using Display Lists

In OpenGL, display lists are created and used as follows:

```c
// Create a display list
GLuint listID = glGenLists(1);
glNewList(listID, GL_COMPILE);
    // Graphics commands
    glColor3f(1.0, 0.0, 0.0);
    glBegin(GL_POLYGON);
        glVertex2f(0.0, 0.0);
        glVertex2f(1.0, 0.0);
        glVertex2f(1.0, 1.0);
        glVertex2f(0.0, 1.0);
    glEnd();
glEndList();

// Execute the display list
glCallList(listID);
```

### Advantages of Display Lists

1. **Performance Optimization:** Commands are stored in compiled form for faster execution
2. **Network Efficiency:** Reduced data transmission in client-server environments
3. **Instance Modeling:** Multiple instances of complex objects without retransmitting data
4. **State Management:** Encapsulation of state changes and transformations

## Event-Driven Input Programming

Modern graphical applications use event-driven programming, where the program responds to events generated by input devices.

### Event Types

Common event types in graphics programming include:

- Mouse events (move, click, drag, scroll)
- Keyboard events (key press, key release)
- Window events (resize, expose, focus)
- Timer events (periodic updates)

### Event Loop Structure

The typical event handling loop:

```
Initialize application
Create window
Register event callbacks
Enter event loop:
    Wait for event
    Dispatch event to appropriate handler
    Update display if needed
```

### Example: OpenGL with GLUT

```c
void mouse(int button, int state, int x, int y) {
    if (button == GLUT_LEFT_BUTTON && state == GLUT_DOWN) {
        // Handle left mouse click
        printf("Click at (%d, %d)\n", x, y);
    }
}

void keyboard(unsigned char key, int x, int y) {
    switch(key) {
        case 'q': exit(0); break;
        case 'r': changeColor(RED); break;
        // ... other cases
    }
    glutPostRedisplay(); // Request display update
}

int main(int argc, char** argv) {
    glutInit(&argc, argv);
    glutCreateWindow("Interactive Graphics");
    glutDisplayFunc(display);
    glutMouseFunc(mouse);
    glutKeyboardFunc(keyboard);
    glutMainLoop();
    return 0;
}
```

## Interactive Techniques

### Rubber-Banding

A visual feedback technique where a line or shape stretches from a fixed point to the current cursor position:

```
Initial click at (x1,y1)      Drag to (x2,y2)        Release at (x3,y3)
     │                            │                      │
     ▼                            ▼                      ▼
    ┌───┐                       ┌───┐                   ┌───┐
    │   │                       │   │                   │   │
    │   │                       │   │                   │   │
    └───┘                       │   │                   │   │
                                │   │                   │   │
                                └───┘                   └───┘
```

### Dragging and Dropping

Allowing users to select objects and move them to new locations:

1. User clicks on object (pick operation)
2. System highlights selected object
3. User moves mouse while holding button
4. Object moves with cursor (feedback)
5. User releases button to drop object

### Double Buffering

Technique to eliminate flickering during screen updates:

- Draw to an off-screen buffer
- Swap buffers when drawing is complete
- Provides smooth animation and interaction

## Menus in Interactive Graphics

Menus provide a structured way for users to make choices without memorizing commands.

### Types of Menus

**Popup Menus:** Appear at cursor position when requested

```
         ┌────────────────────────┐
         │  Option 1              │
Cursor → │  Option 2              │
         │  Option 3              │
         │  Separator             │
         │  Option 4              │
         └────────────────────────┘
```

**Pull-Down Menus:** Appear from menu bars at the top of windows
**Pie Menus:** Circular menus arranged around the cursor position

### Menu Implementation

```c
void createMenu() {
    int menu_id = glutCreateMenu(menuHandler);
    glutAddMenuEntry("Red", RED);
    glutAddMenuEntry("Green", GREEN);
    glutAddMenuEntry("Blue", BLUE);
    glutAttachMenu(GLUT_RIGHT_BUTTON); // Right-click menu
}

void menuHandler(int value) {
    switch(value) {
        case RED: setColor(1.0, 0.0, 0.0); break;
        case GREEN: setColor(0.0, 1.0, 0.0); break;
        case BLUE: setColor(0.0, 0.0, 1.0); break;
    }
    glutPostRedisplay();
}
```

## Interaction Models

Different models define how input devices are used together:

### Request Mode

The program requests input when needed. The program pauses until input is received.

```c
printf("Enter coordinates: ");
scanf("%d %d", &x, &y);
drawPoint(x, y);
```

### Sample Mode

The program samples the current state of input devices without waiting.

```c
x = getMouseX();
y = getMouseY();
// Use current position regardless of whether it changed
```

### Event Mode

The most common model for modern interactive applications. Input devices generate events that are queued and processed by event handlers.
