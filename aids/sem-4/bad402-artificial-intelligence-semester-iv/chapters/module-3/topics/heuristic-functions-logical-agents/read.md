# Module 3: Heuristic Functions for Logical Agents

## Introduction

In the realm of Artificial Intelligence, logical agents use a knowledge base to reason about the world and make rational decisions. However, as problems become more complex (e.g., planning a sequence of actions), the search space explodes. Uninformed search strategies like BFS or DFS become computationally infeasible. This is where **heuristic functions** become critical. They are the "intelligent guesswork" that guides a logical agent towards a goal more efficiently, making informed search algorithms like **A\*** possible.

## Core Concepts

### 1. What is a Heuristic Function?

A heuristic function, denoted as **h(n)**, is a function that estimates the cost of the cheapest path from a node `n` to a goal state. It is problem-specific and leverages domain knowledge to evaluate the "promise" of a node.

*   **Key Idea:** It is an *estimate*, not an exact calculation. A good heuristic is one that is computationally cheap to evaluate yet provides a useful, informative estimate of the actual cost.
*   **Purpose:** To guide the search algorithm (e.g., Greedy Best-First Search, A\*) by ranking which nodes are most promising to expand next, thereby pruning large parts of the search tree.

### 2. Heuristics in Informed Search Algorithms

Heuristics are the core of several key algorithms:

*   **Greedy Best-First Search:** This algorithm expands the node that appears to be closest to the goal, based solely on the heuristic function `h(n)`. It is efficient but not optimal, as it ignores the cost already incurred to reach the current node (`g(n)`).
*   **A\* Search:** This is the most famous heuristic search algorithm. It combines the cost to reach the node `g(n)` with the heuristic estimate to the goal `h(n)`. Its evaluation function is **f(n) = g(n) + h(n)**.
    *   `g(n)`: Actual cost from the start node to node `n`.
    *   `h(n)`: Estimated cost from node `n` to the goal.
    *   `f(n)`: Estimated total cost of the path through node `n`.

### 3. Admissible and Consistent Heuristics

For A\* to be **optimal** (i.e., to find the cheapest path), the heuristic function `h(n)` must possess certain properties.

*   **Admissible Heuristic:** A heuristic is admissible if it **never overestimates** the true cost to reach the goal. Formally, for every node `n`, `h(n) ≤ h*(n)`, where `h*(n)` is the true cost from `n` to the goal.
    *   *Example: In a pathfinding problem on a grid with obstacles, the straight-line (Euclidean) distance to the goal is admissible because it is the shortest possible path, so the actual path can only be longer or equal.*

*   **Consistent (or Monotonic) Heuristic:** A heuristic is consistent if for every node `n` and every successor `n'` generated by an action `a`, the estimated cost of reaching the goal from `n` is no greater than the step cost of getting to `n'` plus the estimated cost from `n'` to the goal:
    `h(n) ≤ c(n, a, n') + h(n')`
    This is a form of the triangle inequality. All consistent heuristics are admissible, but not all admissible heuristics are consistent. Consistency guarantees that A\* is optimal and that it will find the optimal path without having to re-expand nodes (i.e., it is more efficient).

### 4. Example: The 8-Puzzle Problem

Consider the 8-puzzle, a sliding tile game on a 3x3 grid. The goal is to rearrange the tiles from a random initial state to a target state.

Two common heuristic functions are:

1.  **Number of Misplaced Tiles (h₁):** Counts how many tiles are not in their goal position. This is admissible because each misplaced tile requires at least one move to correct it (often more, hence it doesn't overestimate).
2.  **Manhattan Distance (h₂):** For each tile, calculate the sum of the horizontal and vertical distances from its current position to its goal position. Sum this for all tiles (excluding the blank). This is also admissible and more informative than `h₁` as it provides a better estimate of the actual number of moves required.

*   **Comparison:** For a given state, `h₂` will always be greater than or equal to `h₁`. Since it gives a tighter (closer to the true cost) estimate, it is a **more dominant heuristic** and will guide the A\* algorithm more efficiently, expanding far fewer nodes to find the optimal solution.

## Key Points & Summary

| Key Point | Description |
| :--- | :--- |
| **Heuristic Function h(n)** | An estimate of the cost from node `n` to the goal. It uses domain knowledge to make search efficient. |
| **Informs Search** | Drives algorithms like Greedy BFS and A\* Search by prioritizing promising nodes. |
| **A\* Search** | Optimal algorithm using `f(n) = g(n) + h(n)`. It finds the cheapest path if `h(n)` is admissible. |
| **Admissible** | A heuristic that **never overestimates** the true cost to the goal (`h(n) ≤ h*(n)`). Required for A\*'s optimality. |
| **Consistent** | A stronger property implying the heuristic follows the triangle inequality. Guarantees optimality and efficiency in A\*. |
| **More Informed** | A heuristic that is closer to the true cost (higher value, without overestimating) is better. It dominates weaker heuristics, leading to fewer node expansions. |

**In essence, for a logical agent, heuristic functions transform an intractable problem into a manageable one by providing a smart, estimated guide through the vast space of possible logical conclusions or action sequences.** Choosing the right heuristic is the key to building an efficient AI agent.
