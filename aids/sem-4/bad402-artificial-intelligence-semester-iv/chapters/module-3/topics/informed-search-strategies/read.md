# Informed Search Strategies

## Introduction to Informed Search

Informed search strategies, also known as heuristic search, use problem-specific knowledge beyond the problem definition itself to find solutions more efficiently than uninformed search methods. Unlike uninformed search algorithms (like breadth-first or depth-first search) that explore possibilities blindly, informed search algorithms use **heuristic functions** to guide the search toward the goal.

The key advantage of informed search is that it can significantly reduce the search space and computational time by making intelligent choices about which paths to explore. This makes informed search essential for solving complex real-world problems where the search space is too large for uninformed methods.

## Core Concepts

### Heuristic Function

A heuristic function, denoted as **h(n)**, provides an estimate of the cost from node `n` to the nearest goal node. It's a function that takes a state as input and returns a numerical value representing the estimated cost to reach the goal from that state.

**Properties of good heuristics:**

- **Admissible:** Never overestimates the true cost to reach the goal
- **Consistent (Monotonic):** For every node `n` and successor `n'` generated by action `a`, the estimated cost of reaching the goal from `n` is no greater than the step cost plus the estimated cost from `n'`

### Evaluation Function

The evaluation function **f(n)** is used to order nodes in the search queue. Different informed search algorithms use different evaluation functions:

- **Greedy Best-First Search:** f(n) = h(n)
- **A\* Search:** f(n) = g(n) + h(n)

Where:

- **g(n)** = actual cost from start node to node `n`
- **h(n)** = estimated cost from node `n` to goal

## Best-First Search

Best-first search is a general search strategy that expands the node that appears most promising based on an evaluation function. It uses a priority queue where nodes are ordered by their evaluation function value.

```
Algorithm: Best-First Search
1. Start with the initial node in the priority queue
2. While the queue is not empty:
   a. Remove the node with the best f(n) value
   b. If this node is the goal, return the solution
   c. Expand the node and add its children to the queue
```

## Greedy Best-First Search

Greedy best-first search tries to expand the node that is closest to the goal, using only the heuristic function h(n) as the evaluation function (f(n) = h(n)).

**Characteristics:**

- Often finds solutions quickly
- Not optimal (may not find the best solution)
- Not complete (may get stuck in infinite loops)
- Time and space complexity: O(b^m) in worst case

**Example: Route Finding**

```
Cities: A -- B -- C -- D (Goal)
        |         |
        E -- F -- G

Heuristic estimates: h(B)=30, h(C)=20, h(D)=0, h(E)=25, h(F)=15, h(G)=10

From A, we expand to B (h=30) and E (h=25). We choose E (lower h), then F (h=15),
then G (h=10), then C (h=20), then finally D (h=0). Path: A-E-F-G-C-D
```

## A\* Search Algorithm

A\* search is the most popular informed search algorithm. It combines the cost to reach the node (g(n)) and the estimated cost to the goal (h(n)): f(n) = g(n) + h(n).

**Properties of A\*:**

- Complete (will find a solution if one exists)
- Optimal (will find the best solution) if h(n) is admissible
- Time and space complexity: Exponential in worst case, but often much better than uninformed search

```
Algorithm: A* Search
1. Initialize open list with start node (f = g + h)
2. While open list is not empty:
   a. Remove node with lowest f-value
   b. If goal node, return solution
   c. Expand node and for each successor:
      i. Calculate g(successor) = g(current) + cost(current, successor)
      ii. Calculate f(successor) = g(successor) + h(successor)
      iii. If successor is new or has better f-value, add to open list
```

**Example: A\* in Action**

```
Start at A, goal at D
Edges: A-B (cost=1), A-C (cost=3), B-D (cost=5), C-D (cost=2)
Heuristics: h(A)=5, h(B)=4, h(C)=2, h(D)=0

Step 1: Expand A → f(B)=g(B)+h(B)=1+4=5, f(C)=3+2=5
Step 2: Expand B → f(D)=1+5+0=6
Step 3: Expand C → f(D)=3+2+0=5 (better)
Optimal path: A-C-D with cost=5
```

## Memory-Bounded Search Algorithms

### Iterative Deepening A* (IDA*)

IDA* combines the space efficiency of iterative deepening with the heuristic guidance of A*. It performs a series of depth-first searches with increasing f-cost limits.

**Advantages:**

- Complete and optimal (with admissible heuristic)
- Linear space complexity: O(bd)
- Avoids the memory problems of standard A\*

### Recursive Best-First Search (RBFS)

RBFS is a recursive algorithm that keeps track of the f-value of the best alternative path available from any ancestor of the current node.

**Characteristics:**

- More efficient memory usage than A\*
- More node re-expansion than IDA\*
- Complete and optimal with admissible heuristic

## Heuristic Functions

### Designing Effective Heuristics

Good heuristics can dramatically improve search efficiency. Common approaches:

1. **Relaxed Problems:** Remove constraints to create easier problems
2. **Pattern Databases:** Store exact solution costs for subproblems
3. **Learning from Experience:** Use machine learning to develop heuristics

### Example: 8-Puzzle Problem

For the 8-puzzle (sliding tile puzzle), common heuristics include:

- **h₁:** Number of misplaced tiles
- **h₂:** Sum of Manhattan distances of tiles from goal positions

```
8-Puzzle Example:
Current:    Goal:
2 8 3       1 2 3
1 6 4       8   4
7   5       7 6 5

h₁ = 6 (tiles 2,8,3,6,4,5 are misplaced)
h₂ = 2+2+0+3+1+1+2+1 = 12 (Manhattan distances)
```

## Comparison of Search Algorithms

| Algorithm         | Optimal? | Complete? | Time Complexity  | Space Complexity | Uses Heuristic? |

| Breadth-First     | Yes      | Yes       | O(b^d)           | O(b^d)           | No              |
| Depth-First       | No       | No        | O(b^m)           | O(bm)            | No              |
| Uniform Cost      | Yes      | Yes       | O(b^{1+⌊C\*/ε⌋}) | O(b^{1+⌊C\*/ε⌋}) | No              |

| RBFS              | Yes\*    | Yes       | Exponential      | O(bd)            | Yes             |

\*With admissible heuristic

## Real-World Applications

1. **Pathfinding:** GPS navigation, game AI
2. **Robotics:** Motion planning, manipulation
3. **Natural Language Processing:** Speech recognition, machine translation
4. **Bioinformatics:** Protein folding, sequence alignment
5. **Scheduling:** Resource allocation, timetabling

## Exam Tips

1. **Understand heuristic properties:** Be able to explain admissibility and consistency with examples
2. **Trace algorithms:** Practice step-by-step execution of A\* and greedy search
3. **Compare algorithms:** Know when to use which algorithm based on problem constraints
4. **Design heuristics:** For common problems like pathfinding or puzzles, be able to propose effective heuristics
5. **Memory considerations:** Understand the tradeoffs between standard A\* and memory-bounded variants
6. **Focus on A\*:** This is the most important algorithm - know its properties, optimality conditions, and implementation details
