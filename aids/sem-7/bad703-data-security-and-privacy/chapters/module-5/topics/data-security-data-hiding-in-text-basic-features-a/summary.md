# **Data Security: Data Hiding**

## **I. Introduction**

- Data hiding is a technique used to embed secret data within a normal-looking message or file.
- It has various applications in data security and privacy.

## **II. Data Hiding in Text-Basic Features**

- **Lempel-Ziv-Welch (LZW) compression**: can be used to hide data within compressed files.
- **Run-Length Encoding (RLE)**: can be used to hide data within images and text files.

## **III. Applications of Data Hiding**

- **Digital watermarking**: to identify the source of a digital file.
- **Stealth communication**: to hide data within seemingly innocuous messages.
- **Secure data storage**: to protect sensitive data from unauthorized access.

## **IV. Watermarking**

- Definition: embedding a hidden message or signature within a file.
- Applications:
  - Digital watermarking (e.g., logo embedding)
  - Image and audio watermarking
  - Text watermarking (e.g., hidden messages in documents)

## **V. Intuitive Methods**

- **Steganography**: hiding data within images, audio, or text files.
- **Caches and secret messages**: hiding data within seemingly innocuous files.

## **VI. Simple Digital Methods**

- **Frequency Hiding**: hiding data within the frequency domain of a signal.
- **Spread Spectrum**: hiding data within a spread-spectrum signal.

## **VII. Data Hiding Techniques**

- **Least Significant Bit (LSB) replacement**: replacing least significant bits with hidden data.
- **Bit-level hiding**: hiding data at the bit level.

**Important Formulas and Definitions:**

- **Steganography**: Sten- + graphy (hiding in plain sight)
- **Watermarking**: embedding a hidden message or signature within a file
- **Least Significant Bit (LSB)**: the least significant bit of a binary number

**Theorems:**

- **Lempel-Ziv-Welch (LZW) compression theorem**: describes the LZW compression algorithm.
- **Run-Length Encoding (RLE) theorem**: describes the RLE encoding algorithm.
