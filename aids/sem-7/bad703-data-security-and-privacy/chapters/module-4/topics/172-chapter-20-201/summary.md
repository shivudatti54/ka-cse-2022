# **Data Security and Privacy Revision Notes**

## **Chapter 20: 20.1**

### Key Concepts

- **Data Security**: Protecting data from unauthorized access, use, disclosure, disruption, modification, or destruction.
- **Data Privacy**: Respect for individuals' right to control their personal information and prevent its unauthorized disclosure.

### Important Formulas and Theorems

- **Data Encryption**: Converting plaintext to ciphertext using an algorithm, ensuring confidentiality.
  - Formula: `E(x) = C`, where `E` is encryption, `x` is plaintext, and `C` is ciphertext.
- **Hash Function**: One-way function that maps input data to a fixed-size output (hash value).
  - Formula: `h(x) = h(x')`, where `h` is hash function, `x` is input, and `x'` is output.

### Definitions

- **Confidentiality**: Insuring the secrecy of data.
- **Integrity**: Ensuring data is not modified or altered without authorization.
- **Authentication**: Verifying the identity of a user or system.
- **Authorization**: Granting or denying access to data or system resources.

### Key Terms

- **Secure Sockets Layer (SSL)**: Encryption protocol used to secure online communication.
- **Transport Layer Security (TLS)**: Encryption protocol used to secure online communication.
- **Firewall**: Network security system that monitors and controls incoming and outgoing network traffic.

### Quick Revision Tips

- Review data encryption and decryption concepts.
- Understand the importance of data confidentiality, integrity, and authenticity.
- Familiarize yourself with hash functions and their applications.
- Brush up on authentication and authorization concepts.
