# **Chapter 8: Inheritance Basics**

### Key Points

- **Inheritance**: A mechanism in OOP where a class can inherit properties and behavior from a parent class.
- **Super**: A keyword used to access parent class members in a child class.
- **Multilevel Inheritance**: A type of inheritance where a child class inherits from a parent class, and the parent class itself inherits from another parent class.
- **Hierarchical Inheritance**: A type of inheritance where multiple child classes inherit from a single parent class.
- **Diamond Problem**: A problem that arises when two classes inherit from the same parent class, and each class has a method with the same name.

### Important Formulas and Definitions

- **Inheritance Formula**: `Child` inherits from `Parent` using `extends` or `implements`.
- **Super Keyword**: Used to access parent class members in a child class.
- **Multilevel Inheritance Formula**: `Child1` inherits from `Parent1`, and `Child1` inherits from `Parent2`.
- **Polymorphism**: The ability of an object to take on multiple forms.

### Key Theorems

- **Diamond Problem Theorem**: When two classes inherit from the same parent class, and each class has a method with the same name.
- **Method Overriding Theorem**: When a child class provides a different implementation of a method that is already defined in its parent class.

### Revision Notes

- Inheritance allows for code reuse and modularity.
- The `super` keyword is used to access parent class members in a child class.
- Multilevel inheritance allows for a hierarchical structure of classes.
- The diamond problem can be solved using the `final` keyword.
- Polymorphism is a key concept in object-oriented programming.
