# **Inheritance Basics and Object Oriented Programming in Java**

### Key Concepts

- **Executed**: The order in which methods are called in a method overriding scenario
- **Method Overriding**: Hiding a method in a subclass with the same method name, return type, and parameter list as the method in its superclass
- **Dynamic Method Dispatch**: The process of determining the method to be called at runtime, rather than compile time
- **Using Abstract Classes**: Creating a class that cannot be instantiated and provides a blueprint for other classes to follow
- **Using final with Inheritance**: Modifying the behavior of a final class in subclasses
- **Local Variable Type Inference (LVTI)**: The compiler determines the type of a local variable based on the assigned value
- **Inherita**: A keyword not present in Java; likely a typo for "inheritance"

### Important Formulas and Definitions

- **Method Overriding Formula**: `super.methodName();` to call the method from the superclass
- **Method Overloading Formula**: `super.methodName(parameterList);` to call a method from the superclass with different parameters
- **Dynamic Method Dispatch Formula**: `super.methodName();` to determine the method to be called at runtime

### Key Theorems and Principles

- **Inheritance Theorem**: A subclass inherits all the properties and behavior of its superclass.
- **Method Overriding Principle**: A subclass can provide a different implementation of a method already defined in its superclass.
- **Polymorphism Principle**: Objects of different classes can be treated as objects of a common superclass.
