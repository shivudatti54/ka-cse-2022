## **Important Formulas and Definitions**

- **Abstraction**: A concept in OOP that hides internal implementation details and shows only necessary information to the outside world.
- **Encapsulation**: The idea of bundling data and methods that operate on that data within a single unit (class).
- **Inheritance**: The mechanism by which one class can inherit properties and behavior from another class.

## **Theorems and Principles**

- **Liskov Substitution Principle (LSP)**: A fundamental principle of object-oriented programming that states that subtypes should be substitutable for their base types.
- **Open-Closed Principle (OCP)**: A principle that states that software entities (classes, modules, functions, etc.) should be open for extension but closed for modification.

## **Key Concepts to Remember**

- **Constructors**: Special methods used to initialize objects when they are created.
- **this keyword**: A keyword used to refer to the current object.
- **super keyword**: A keyword used to access members of a superclass from a subclass.
