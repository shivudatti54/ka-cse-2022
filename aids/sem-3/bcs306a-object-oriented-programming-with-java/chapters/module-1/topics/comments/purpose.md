### **Learning Purpose: Comments**

**Why is this topic important?**  
Comments are fundamental to writing clean, maintainable, and collaborative code. They explain the 'why' behind complex logic, making code easier for others (and your future self) to understand. Ignoring comments leads to unreadable, error-prone software that is difficult to modify.

**What will students learn?**  
Students will learn the two primary types of Java comments: single-line (`//`) and multi-line (`/* ... */`). They will understand the purpose of Javadoc comments (`/** ... */`) for automatic documentation generation and learn best practices for writing meaningful, non-redundant comments.

**How does it connect to other concepts?**  
Comments are a core part of coding style and documentation, connecting directly to **code readability**, **debugging**, **team collaboration**, and **software maintenance**. They are the first step in professional programming practices.

**Real-world applications:**  
Comments are used everywhere in professional software development to document code purpose, parameters, return values, and author information. They are crucial for API documentation (e.g., the Java Standard Library documentation is generated from Javadoc comments) and for onboarding new developers onto a project.
