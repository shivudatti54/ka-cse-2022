# Chapter 2 Summary and Revision Notes

=====================================

### Object-Oriented Programming (OOP) Basics

- **Definition**: A programming paradigm that organizes software design around data, or objects, rather than data structures.
- **Key Characteristics**:
  - Encapsulation
  - Abstraction
  - Inheritance
  - Polymorphism

### Two Paradigms of OOP

- **Imperative Programming**: Focuses on the steps to be executed to achieve a result.
- **Object-Oriented Programming**: Focuses on the data and the interactions between objects.

### Abstraction

- **Definition**: A way to show only the necessary information to the outside world while hiding the internal details.
- **Benefits**:
  - Improved readability
  - Reduced complexity
  - Easier maintenance

### The Three OOP Principles

- **Abstraction**: Hide internal implementation details and show only the necessary information.
- **Encapsulation**: Bundles data and methods that operate on that data within a single unit.
- **Inheritance**: Creates a new class based on an existing class.

### Other Important Concepts

- **Polymorphism**: Ability of an object to take on multiple forms.
- **Method Overriding**: When a subclass provides a different implementation of a method that is already defined in its superclass.
- **Method Overloading**: When multiple methods with the same name can be defined, but with different parameters.

### Important Formulas and Definitions

- **Abstraction Formula**: A = x ~ y (Abstraction is to the internal implementation as x is to the external interface)
- **Encapsulation Formula**: E = x + y (Encapsulation is the combination of data and methods that operate on that data)

### Important Theorems

- **Theorem of Inheritance**: If A is a subclass of B, then A inherits all the properties and methods of B.
- **Theorem of Polymorphism**: If A is a subclass of B, then A can be treated as B in certain situations.
