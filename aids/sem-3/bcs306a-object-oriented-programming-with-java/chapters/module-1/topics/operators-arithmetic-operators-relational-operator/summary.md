# **Operators in Java**

### Arithmetic Operators

- **Arithmetic Operators**: Used for numerical operations
- **Order of Operations**:
  - Parentheses
  - Exponentiation
  - Multiplication and Division
  - Addition and Subtraction
- **Arithmetic Operators in Java**:
  - `+` (Addition)
  - `-` (Subtraction)
  - `*` (Multiplication)
  - `/` (Division)
  - `%` (Modulus)
  - `**` (Exponentiation)

### Relational Operators

- **Relational Operators**: Used for comparing values
- **Order of Operations**:
  - Parentheses
  - Relational Operators
  - Exponentiation
  - Multiplication and Division
  - Addition and Subtraction
- **Relational Operators in Java**:
  - `==` (Equal)
  - `!=` (Not Equal)
  - `>` (Greater Than)
  - `<` (Less Than)
  - `>=` (Greater Than or Equal)
  - `<=` (Less Than or Equal)

### Boolean Logical Operators

- **Boolean Logical Operators**: Used for logical operations
- **Order of Operations**:
  - Parentheses
  - Boolean Logical Operators
  - Relational Operators
  - Exponentiation
  - Multiplication and Division
  - Addition and Subtraction
- **Boolean Logical Operators in Java**:
  - `&&` (Logical And)
  - `||` (Logical Or)
  - `!` (Logical Not)

### Assignment Operator

- **Assignment Operator**: Used to assign a value to a variable
- **Syntax**: `variable = value;`
- **Examples**:
  - `x = 5;` (Assigns 5 to variable x)
  - `y += 3;` (Adds 3 to variable y)

### The ? Operator

- **Null Coalescing Operator**: Used to provide a default value if a variable is null
- **Syntax**: `variable ? value : default_value;`
- **Examples**:
  - `name = null ? "John" : "Jane";` (Assigns "John" if name is null, otherwise assigns "Jane")

### Operator Precedence

- **Order of Operations**: Follows the order specified in the Java Language Specification
- **Examples**:
  - `1 + 2 * 3` (Evaluates as 7)
  - `(1 + 2) * 3` (Evaluates as 9)

### Using Operators

- **Use operators to simplify code and improve readability**
- **Use parentheses to clarify the order of operations**
- **Use the `?` operator to provide default values in conditional statements**
