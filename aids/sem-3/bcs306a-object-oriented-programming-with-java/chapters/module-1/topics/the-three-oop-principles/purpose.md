### Learning Purpose: The Three OOP Principles

**1. Why is this topic important?**
The three core principles of Object-Oriented Programming (OOP)—Encapsulation, Inheritance, and Polymorphism—are the fundamental pillars upon which modern, scalable, and maintainable software is built. Mastering these concepts is essential for writing professional Java code that is robust, reusable, and easy to debug and modify. They are not just Java concepts; they are universal patterns used across almost all major programming languages.

**2. What will students learn?**
Students will learn to define and implement each principle:
*   **Encapsulation:** Bundling data (variables) and methods that operate on that data into a single unit (a class), while restricting direct access to some components.
*   **Inheritance:** Allowing a new class (subclass) to adopt the properties and methods of an existing class (superclass), promoting code reuse.
*   **Polymorphism:** Allowing objects of different classes to be treated as objects of a common superclass, primarily through method overriding, enabling flexible and dynamic code.

**3. How does it connect to other concepts?**
This topic is the direct application of the core "class and object" concept. It provides the necessary tools and structure to build upon foundational knowledge. These principles are prerequisites for understanding more advanced topics like abstract classes, interfaces, design patterns, and complex software architecture.

**4. Real-world applications**
These principles are used everywhere: **Encapsulation** protects user data in a `BankAccount` class. **Inheritance** creates a hierarchy like `Vehicle` -> `Car` -> `ElectricCar`. **Polymorphism** allows a graphics program to render different `Shape` objects (e.g., `Circle`, `Rectangle`) with a single `draw()` method call.
