# Direct Memory Access (DMA) is a hardware mechanism that enables I/O devices to transfer data directly to/from main memory without continuous CPU intervention. The DMA Controller (DMAC) acts as a co-processor, handling bulk data transfers after CPU initialization. Key transfer modes include Burst Mode (continuous bus control), Cycle Stealing Mode (alternating CPU/DMAC access), and Transparent Mode (CPU idle time only). The bus arbitration protocol involves sequential signals: DREQ, BR, BG, DACK, and completion interrupt. Quantitative analysis demonstrates significant CPU overhead reduction—approximately 4x improvement for typical transfers compared to programmed I/O. Assessment questions require calculating CPU time consumed in cycle stealing mode, analyzing priority-based channel scheduling, and comparing throughput across DMA modes.
