# **Data Flow and Behavioural Model**

### Overview

- **Definition**: A modelling technique that focuses on the input, output, and processing of data in a digital system.
- **Key Characteristics**: Describe the system's behavior using a set of inputs, outputs, and transformations.

### Models

#### Data Flow Model

- **Definition**: A model that describes the flow of data through a system.
- **Key Features**:
  - Inputs
  - Outputs
  - Data paths
  - Data transformations
- **Example**: A simple calculator using arithmetic logic units (ALUs) and registers.

#### Behavioural Model

- **Definition**: A model that describes the system's behavior using a set of inputs, outputs, and program control.
- **Key Features**:
  - Inputs
  - Outputs
  - Program control (sequences of instructions)
  - State variables
- **Example**: A computer's instruction decoding process.

### Important Formulas and Definitions

- **Data flow equation**: `DF = (I \* W) + (S \* R)`, where `DF` is the data flow rate, `I` is the input rate, `W` is the write rate, `S` is the shift rate, and `R` is the read rate.
- **Behavioural model theorem**: "A behavioural model is equivalent to a data flow model if and only if it can be simulated by a data flow model."

### Theorems

- **Ladberg's theorem**: "A behavioural model is equivalent to a data flow model if and only if it can be simulated by a data flow model."

### Key Terms

- **State variables**: Variables that represent the internal state of a system.
- **Program control**: The sequence of instructions that controls the flow of a program.
- **Data transformations**: Operations performed on data to produce a result.

### Revision Tips

- Understand the key differences between data flow and behavioural models.
- Focus on the key features of each model, including inputs, outputs, and transformations.
- Practice identifying and modelling digital systems using both data flow and behavioural models.
