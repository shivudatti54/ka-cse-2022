# **Revision Notes for Chapter 4.5: Data Structures and Applications**

## **Key Concepts**

- **Stack**: A Last-In-First-Out (LIFO) data structure where elements are added and removed from the top.
- **Queue**: A First-In-First-Out (FIFO) data structure where elements are added to the end and removed from the front.
- **Linked List**: A dynamic collection of elements, where each element points to the next element.
- **Tree**: A hierarchical data structure consisting of nodes with a value and references to child nodes.
- **Graph**: A non-linear data structure consisting of nodes and edges that connect them.

## **Definitions**

- **Top**: The top element of a stack or queue.
- **Tail**: The last element of a queue.
- **Head**: The first element of a queue.
- **Node**: An element in a linked list, graph, or tree.
- **Edge**: A connection between two nodes in a graph.

## **Formulas/Equations**

- **Time Complexity**:
  - Best Case: O(1)
  - Average Case: O(n)
  - Worst Case: O(n^2)
- **Space Complexity**:
  - Best Case: O(1)
  - Average Case: O(n)
  - Worst Case: O(n^2)

## **Theorems**

- **De Morgan's Law**: (A ∩ B)' = A' ∪ B'
- **Dijkstra's Algorithm**: A method for finding the shortest path between nodes in a graph.

## **Important Points**

- Stacks are used for evaluating postfix expressions and parsing syntax.
- Queues are used for job scheduling and print queues.
- Linked lists are used for dynamic memory allocation and implementing stacks and queues.
- Trees are used for organizing data in a hierarchical manner and for searching and traversing data.
- Graphs are used for modeling relationships between objects and for solving complex problems.
