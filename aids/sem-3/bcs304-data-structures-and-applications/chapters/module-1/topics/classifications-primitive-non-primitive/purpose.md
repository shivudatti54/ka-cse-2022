### Learning Purpose: Classifications (Primitive & Non-Primitive)

**1. Why is this topic important?**
Understanding data structure classification is the foundational step in learning how to organize, manage, and store data efficiently. It provides the essential vocabulary and conceptual framework for the entire course, distinguishing between basic building blocks and complex, user-defined structures. This knowledge is critical for selecting the appropriate data structure for any given computational problem.

**2. What will students learn?**
Students will learn to categorize data structures into two primary types: **Primitive** (basic types like integers, floats, and characters supported natively by a programming language) and **Non-Primitive** (complex structures like arrays, lists, stacks, queues, trees, and graphs). They will understand that non-primitive structures are built from primitive ones and can be further classified as linear (sequential) or non-linear (hierarchical or networked).

**3. How does it connect to other concepts?**
This classification is the gateway to all subsequent modules. Primitive structures are the atoms, while non-primitive structures are the molecules used to build algorithms. This hierarchy directly introduces the implementation of more advanced concepts like linked lists (linear) and binary trees (non-linear), which are built using pointers and dynamic memory allocation.

**4. Real-world applications**
*   **Primitive:** Used in every single program for basic calculations and value storage (e.g., a `float` for storing a temperature reading).
*   **Non-Primitive:**
    *   **Linear:** A **queue** manages print jobs or customer service requests; a **stack** handles function calls and undo operations.
    *   **Non-Linear:** A **tree** structures a company's organizational hierarchy or the DOM of a web page; a **graph** maps social network connections or navigation routes.
