# Dynamic Hashing enables hash tables to grow incrementally without costly full-table rehashing. Extendible Hashing uses a directory (array of $2^d$ pointers) indexed by the first $d$ bits of hash values, with buckets storing actual records. When a bucket overflows, splitting occurs: if local depth < global depth, the bucket splits without directory expansion; if local depth = global depth, the directory must double first. This ensures O(1) expected search time with space $O(2^d + n/b)$. Linear Hashing uses sequential bucket splitting via a split pointer without directory overhead, employing hash function family $h_i(x) = x \bmod 2^i$. Both methods maintain expected constant-time operations while supporting dynamic growth, with extendible hashing offering directory-based O(1) access and linear hashing providing simpler implementation with slightly higher search costs.
