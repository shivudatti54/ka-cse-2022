# The Graph Abstract Data Types
A graph G = (V, E) consists of vertices V and edges E connecting vertex pairs. Key terminology includes adjacency (neighboring vertices), incidence (edge-vertex relationship), degree (edge count per vertex), path (vertex sequence), and cycle (closed path). The Handshaking Lemma proves Σdeg(v) = 2|E| in undirected graphs. Graphs are classified by direction (directed/undirected), weighting (weighted/unweighted), connectivity (connected/strongly connected), and structure (bipartite, DAG, tree). Two primary representations exist: adjacency matrix (O(n²) space, O(1) edge lookup) and adjacency list (O(n+m) space, O(deg(v)) lookup). The Graph ADT defines operations including addVertex, removeVertex, addEdge, removeEdge, adjacent, and getNeighbors with formal specifications and complexity analysis. Five MCQs test application of the Handshaking Theorem, in-degree calculations, bipartite graph properties, minimum edges in connected graphs, and space complexity of adjacency matrices.
