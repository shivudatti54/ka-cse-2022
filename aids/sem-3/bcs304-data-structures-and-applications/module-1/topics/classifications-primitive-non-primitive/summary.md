# Classifications Primitive Non Primitive
Data structures are classified into primitive (int, float, char, bool, pointer) and non-primitive (linear: arrays, linked lists, stacks, queues; non-linear: trees, graphs) categories. Primitive data types provide fundamental building blocks with fixed memory sizes and direct hardware support, characterized by their value domains and IEEE 754 representations for floating-point. Non-primitive structures enable complex data organization: arrays provide O(1) random access with contiguous memory; linked lists offer dynamic sizing with O(1) insertion/deletion; stacks implement LIFO operations critical for recursion and expression evaluation; queues manage FIFO scheduling. Trees provide hierarchical organization with height bounds of ⌊log₂ n⌋ + 1 for binary trees; graphs represent arbitrary relationships using adjacency matrices (O(V²) space) or lists (O(V+E) space). Selection depends on time/space requirements, access patterns, and operational frequencies. Hard-level assessment requires analyzing pointer arithmetic, circular queue indices, tree height calculations, IEEE 754 bit representation, memory addressing, and pointer update operations.
