# Learning Objectives

After studying this topic, you should be able to:

1.  Define what constitutes a security incident and differentiate it from a security event.
2.  Compare and contrast reactive (alert-driven) and proactive (threat hunting) detection methods.
3.  Describe the role of a SIEM and correlation rules in the automated detection of incidents.
4.  Explain the purpose and steps of the triage process in incident handling.
5.  Evaluate an alert to distinguish between false positives and true positives.
6.  Assign appropriate severity and priority levels to a security incident based on a given scenario.
7.  Identify the tools and data sources used for contextual enrichment during the triage process.
