### Learning Purpose: Continuous Internal Evaluation

#### 1. Why is this topic important?
Continuous Internal Evaluation (CIE) is crucial as it shifts the focus from high-stakes, end-of-module exams to a consistent process of assessing understanding and skill acquisition. It provides regular feedback, allowing both students and instructors to identify knowledge gaps early and adjust the learning path accordingly. This fosters a more supportive and effective learning environment.

#### 2. What will students learn?
Students will learn to design and implement a series of small, practical coding assessments (e.g., quizzes, mini-projects, code reviews) that test their proficiency in core Module 5 concepts like data manipulation with Pandas, data visualization with Matplotlib/Seaborn, and introductory statistical analysis. The emphasis is on applying concepts incrementally rather than memorizing syntax.

#### 3. How does it connect to other concepts?
CIE directly builds upon the foundational programming skills from earlier modules (data types, functions, control flow) and applies them to data-specific tasks. Success in these continuous assessments ensures students have the practical competency needed to tackle the final capstone project, which integrates all modules into a complete data science workflow.

#### 4. Real-world applications
This process mirrors the agile, iterative nature of real-world data science projects. Professionals continuously develop, test, and refine their code and models, receiving feedback from peers and stakeholders. CIE prepares students for this environment by emphasizing incremental progress, code quality, and the practical application of theory over one-time achievement.
