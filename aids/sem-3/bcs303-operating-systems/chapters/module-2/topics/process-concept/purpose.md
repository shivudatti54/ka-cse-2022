# Learning Objectives

After studying this topic, you should be able to:

1. Define a process and explain how it differs from a program, with clear examples

2. Describe the structure and components of the Process Control Block (PCB), and explain its role in process management

3. Identify and explain all five states in the basic process state model and the transitions between them

4. Distinguish between different types of processes (foreground vs background, CPU-bound vs I/O-bound)

5. Explain the process creation mechanism in Unix-like systems using fork() and exec() system calls

6. Analyze the process termination sequence and understand zombie and orphan processes

7. Describe what happens during a context switch and how the PCB is utilized in this process

8. Apply process concepts to analyze real-world scenarios and operating system behaviors
