# **System Structures: What Operating Systems Do**

## **Introduction**

Operating systems (OS) are complex software that manage computer hardware resources and provide common services to computer programs. In this module, we will explore the various system structures that operating systems use to manage and utilize system resources.

## **System Structures**

A system structure refers to the organization and design of an operating system's components and how they interact with each other. The following are some common system structures used by operating systems:

### 1. Monolithic Operating System

- Definition: A monolithic operating system is a type of operating system where the kernel and the applications are combined into a single executable file.
- Characteristics:
  - The kernel and applications are tightly coupled.
  - The operating system is a self-contained unit.
  - It is difficult to modify or extend the operating system.
- Example: MS-DOS (old version)

### 2. Microkernel Operating System

- Definition: A microkernel operating system is a type of operating system where the kernel is minimal and the applications are separate from the kernel.
- Characteristics:
  - The kernel provides basic services such as process scheduling and memory management.
  - The applications are separate from the kernel.
  - It is easier to modify or extend the operating system.
- Example: Linux (kernel version 1.x)

### 3. Hybrid Operating System

- Definition: A hybrid operating system is a type of operating system that combines elements of both monolithic and microkernel operating systems.
- Characteristics:
  - The kernel provides basic services such as process scheduling and memory management.
  - The applications are separate from the kernel.
  - It is easier to modify or extend the operating system.
- Example: Windows NT (kernel version 3.x)

### 4. Real-Time Operating System

- Definition: A real-time operating system is a type of operating system that ensures predictable and fast response times for real-time applications.
- Characteristics:
  - The operating system provides priority scheduling and time slicing.
  - The operating system is designed for applications that require predictable timing.
  - It is suitable for applications such as control systems and embedded systems.
- Example: VxWorks

## **Key Concepts**

- **Process scheduling**: The process of allocating and managing CPU time to different processes.
- **Memory management**: The process of managing memory to prevent conflicts between processes and applications.
- **Interrupt handling**: The process of handling interrupts generated by hardware devices.
- **Device management**: The process of managing hardware devices such as printers and disk drives.

## **Conclusion**

In conclusion, system structures play a critical role in determining the characteristics and behavior of an operating system. Understanding the different system structures and their characteristics is essential for designing and implementing efficient operating systems. By examining the characteristics of monolithic, microkernel, hybrid, and real-time operating systems, we can gain a deeper understanding of the complexities of operating system design.

## **Key Terms**

- **Kernel**: The core component of an operating system that manages system resources.
- **Process**: A program in execution that can be running, waiting, or sleeping.
- **Device driver**: Software that manages communication between the operating system and hardware devices.
- **Interrupt**: A signal generated by hardware devices to indicate an event or request.
