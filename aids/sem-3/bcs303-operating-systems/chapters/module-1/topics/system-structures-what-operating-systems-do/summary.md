# **System Structures: What Operating Systems Do**

## **Key Concepts**

- **Monolithic Operating System**: A single, self-contained program that manages the entire system.
- **Microkernel Operating System**: A small kernel that coordinates the activities of multiple smaller software modules.
- **Hybrid Operating System**: A combination of monolithic and microkernel design.

## **Operating System Structures**

### 1. Process Structure

- **Process**: A program in execution.
- **Process Management**: Creation, synchronization, and termination of processes.
- **Process Scheduling**: Allocation of CPU time to processes.

### 2. Memory Structure

- **Virtual Memory**: A combination of physical and secondary storage.
- **Page Replacement Algorithms**: Policies for replacing pages in virtual memory.
- **Memory Protection**: Isolation of processes from each other's memory.

### 3. File System Structure

- **File System**: A collection of files and directories.
- **File System Hierarchy**: A tree-like structure for organizing files.
- **File System Management**: Creation, deletion, and retrieval of files.

### 4. Input/Output (I/O) Structure

- **I/O Devices**: Devices that interact with the user or external devices.
- **I/O Management**: Scheduling and handling of I/O requests.
- **I/O Buffering**: Storing data in a buffer to optimize I/O performance.

## **Important Formulas and Definitions**

- **CPU Utilization Formula**: $CPU Utilization = (System Time / Total System Time) \* 100$
- **Throughput Formula**: $Throughput = (Total Output / Total Input) \* 100$
- **Definition of Operating System**: "A software system that manages computer hardware resources and provides common services to programs."

**Theorem:**
Operating System is a program that provides a platform for other programs to run on.
