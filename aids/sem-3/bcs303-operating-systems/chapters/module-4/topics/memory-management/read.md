# Memory Management

## Introduction

Memory Management is one of the most critical functions of any operating system, serving as the foundation upon which modern computing systems efficiently utilize the computer's primary memory (RAM). In simple terms, memory management refers to the process of controlling and coordinating computer memory, assigning portions called blocks to various running programs to optimize overall system performance. Without effective memory management, a computer system would be unable to function efficiently, leading to memory fragmentation, inefficient process execution, and potential system crashes.

The importance of memory management in operating systems cannot be overstated. When multiple processes execute concurrently on a computer system, they all require memory space to store their instructions, data, and stack information. The operating system must allocate this memory fairly and efficiently while ensuring that processes do not interfere with each other's memory spaces—a concept known as memory protection. Furthermore, modern computer systems often run more processes than can fit simultaneously in physical memory, requiring sophisticated techniques like virtual memory to create the illusion of a larger memory space than actually exists.

From the perspective of the University of Delhi's Computer Science curriculum, memory management represents a fundamental topic that bridges theoretical concepts with practical system design. Students must understand how operating systems abstract physical memory into logical address spaces, enabling process isolation, efficient memory utilization, and seamless execution of programs larger than available physical memory. This knowledge forms the backbone of system programming, database management, and advanced operating system design.

## Key Concepts

### 1. Memory Hierarchy and Organization

Computer memory is organized in a hierarchical structure with different levels offering varying speeds and capacities. At the top of the hierarchy lies the CPU registers, which provide the fastest access but have very limited capacity (typically 8-128 bytes). Cache memory (L1, L2, L3) follows, offering faster access than main memory but at higher cost and lower capacity. Main memory (RAM) forms the primary working area, while secondary storage (hard disks, SSDs) provides the largest capacity at the slowest speeds.

Memory management primarily deals with main memory, which is a large array of bytes, each with its unique address. The operating system must efficiently manage this finite resource among competing processes, ensuring that each process has adequate memory to execute while maximizing overall system throughput.

### 2. Address Binding

Address binding is the process of mapping logical addresses (generated by the CPU) to physical addresses (addresses in main memory). This binding can occur at different times during program execution:

- **Compile Time**: If the memory location of a process is known beforehand, the compiler generates absolute code with fixed physical addresses. This method is inflexible and rarely used in modern systems.
- **Load Time**: The compiler generates relocatable code, and the loader (at load time) binds these logical addresses to physical addresses. The process must remain in one location throughout execution.
- **Execution Time (Runtime)**: Binding occurs during execution, allowing processes to be moved between different memory locations. This approach requires hardware support (like base and limit registers) and is used in modern operating systems.

### 3. Logical vs Physical Address Space

A **logical address** (also called virtual address) is the address generated by the CPU for a process—this is what the program sees. A **physical address** is the actual address in main memory where data is stored. The operating system, with hardware assistance, translates logical addresses to physical addresses through a component called the Memory Management Unit (MMU).

The logical address space represents the range of addresses that a process can use, while the physical address space represents the actual available memory in the system. Virtual memory techniques create the illusion that each process has access to a large, contiguous address space, even when physical memory is fragmented or insufficient.

### 4. Swapping

Swapping is a memory management technique where entire processes are moved between main memory and secondary storage (typically a swap file or partition). When physical memory becomes full, the operating system may swap out inactive processes to secondary storage to make room for other processes. When needed again, the swapped-out process is brought back into memory.

While swapping enables the system to run more processes than physical memory would otherwise allow, it introduces significant overhead due to the time required to transfer data between memory and disk. Modern systems use more sophisticated techniques like paging, but swapping remains a valuable mechanism in certain scenarios.

### 5. Contiguous Memory Allocation

In contiguous memory allocation, each process is allocated a single continuous block of memory. This approach is simple to implement but leads to two major problems:

- **External Fragmentation**: When free memory is divided into small non-contiguous blocks, making it impossible to allocate a large process even though total free memory is sufficient.
- **Internal Fragmentation**: Memory allocated to a process may be larger than requested, leaving unused space within the allocated block.

The degree of external fragmentation depends on the method used to allocate memory:
- **First Fit**: Allocate the first hole (free block) large enough for the process. Fast but may lead to more fragmentation.
- **Best Fit**: Allocate the smallest hole large enough. Minimizes wasted space but requires searching all holes.
- **Worst Fit**: Allocate the largest available hole. Tries to leave large leftover holes but is generally inefficient.

### 6. Paging

Paging is a memory management scheme that eliminates external fragmentation by dividing both physical memory and logical address space into fixed-sized blocks called **frames** and **pages** respectively. The size of pages and frames is typically a power of 2 (commonly 4KB), which simplifies address translation.

In a paging system, the logical address consists of two parts: a page number and a page offset. The page number indexes into a **page table**, which stores the frame number corresponding to each page. The physical address is then constructed using the frame number and the page offset.

**Example of Paging Address Translation:**
If a system uses 16-bit logical addresses with 4KB (2^12 bytes) pages:
- Page size = 4KB = 2^12 bytes
- Number of bits for offset = 12
- Number of bits for page number = 16 - 12 = 4
- Number of pages per process = 2^4 = 16

For a logical address 0x3A7 (binary: 0011 1010 0111):
- Page number = 3 (0011 binary)
- Page offset = 0xA7 (1010 0111 binary)
- If page 3 maps to frame 5, physical address = (5 × 4096) + 0xA7 = 20480 + 247 = 20727

### 7. Structure of Page Table

The page table is a data structure maintained by the operating system that maps virtual page numbers to physical frame numbers. Several optimizations and structures exist for page tables:

- **Hierarchical Paging**: Divides the page table into multiple levels to reduce memory consumption. For example, a two-level page table splits the page number into two parts: a page directory index and a page table index.

- **Hashed Page Tables**: Uses a hash table where the virtual page number is hashed to an entry in the page table. This approach handles sparse address spaces efficiently.

- **Inverted Page Table**: Maintains one entry for each physical frame, reducing memory overhead for large address spaces but increasing lookup time.

- **Translation Lookaside Buffer (TLB)**: A special high-speed cache that stores recently used page table entries, dramatically speeding up address translation.

### 8. Segmentation

Segmentation is a memory management technique that divides a program's logical address space into variable-sized segments based on logical divisions like functions, arrays, or data structures. Each segment has a name (or number) and a length, providing a more natural view of memory from the programmer's perspective.

A logical address in a segmented system consists of a segment number and an offset within that segment. The segment table maps each segment number to a base address (starting location) and limit (segment length). The hardware ensures that the offset does not exceed the segment limit, providing memory protection.

Segmentation offers several advantages: it provides logical transparency (programs can be written in terms of meaningful segments), it enables code and data protection (different permissions for different segments), and it allows sharing of specific segments between processes.

### 9. Virtual Memory Management

Virtual memory is a technique that allows the execution of processes that may not be completely in memory. It creates an illusion of a large, contiguous address space by using both main memory and secondary storage. Virtual memory enables:

- **Running programs larger than physical memory**: Only the active portions of a program need to be in memory.
- **Increased degree of multiprogramming**: More processes can be loaded simultaneously.
- **Memory protection**: Each process has its own virtual address space.

The two primary implementations of virtual memory are **paging** and **segmentation**, often combined as **segmented paging** or **paged segmentation**.

### 10. Demand Paging

Demand paging is a virtual memory management technique where pages are loaded into memory only when they are needed—that is, when a page fault occurs. A page fault happens when the CPU tries to access a page that is not currently in physical memory.

The demand paging process involves:
1. The CPU generates a reference to a page not in memory
2. The Memory Management Unit (MMU) triggers a page fault
3. The OS identifies the required page on secondary storage
4. If memory is full, a victim page is selected for replacement
5. The required page is loaded into a free frame
6. The page table is updated
7. The instruction is restarted

This lazy loading approach minimizes I/O operations and memory usage, as only actively used pages consume physical memory.

### 11. Copy-on-Write

Copy-on-write (COW) is an optimization technique primarily used in fork() system calls. When a parent process forks a child process, instead of immediately copying all pages, the pages are shared between parent and child in read-only mode. If either process attempts to modify a page, a copy is made for that specific page—hence "copy on write."

This technique significantly reduces memory overhead and speeds up process creation, as many pages may never need to be copied (if the child simply executes a new program).

## Examples

### Example 1: Calculating Effective Access Time with TLB

Consider a system with the following characteristics:
- Memory access time (MA) = 100 nanoseconds
- TLB lookup time = 20 nanoseconds
- TLB hit ratio = 80% (0.8)

Calculate the effective memory access time.

**Solution:**

The effective access time depends on whether the TLB lookup finds the page mapping:

**When TLB hits (80% of the time):**
Time = TLB lookup + Memory access
Time = 20 ns + 100 ns = 120 ns

**When TLB misses (20% of the time):**
Time = TLB lookup + Page table access + Memory access
Time = 20 ns + 100 ns + 100 ns = 220 ns

**Effective Access Time:**
EAT = (0.8 × 120) + (0.2 × 220)
EAT = 96 + 44 = 140 nanoseconds

Without TLB, the access time would be 200 nanoseconds (100 ns for page table + 100 ns for memory). The TLB provides a significant speedup, reducing effective access time by 30%.

### Example 3: Segmented Memory Address Translation

A system uses segmentation with the following segment table:

| Segment | Base    | Limit   |
|---------|---------|---------|
| 0       | 200     | 1000    |
| 1       | 1900    | 500     |
| 2       | 1200    | 800     |
| 3       | 3500    | 600     |

For each logical address, determine if it is valid and find the physical address:
- (0, 400)
- (1, 600)
- (2, 900)

**Solution:**

**Address (0, 400):**
- Segment 0: Base = 200, Limit = 1000
- Offset 400 < Limit 1000: VALID
- Physical address = Base + Offset = 200 + 400 = 600

**Address (1, 600):**
- Segment 1: Base = 1900, Limit = 500
- Offset 600 > Limit 500: INVALID
- This would cause a segmentation fault

**Address (2, 900):**
- Segment 2: Base = 1200, Limit = 800
- Offset 900 > Limit 800: INVALID
- This would cause a segmentation fault

## Exam Tips

1. **Understand the difference between logical and physical addresses thoroughly**: This is fundamental to memory management and frequently appears in exams. Remember that logical addresses are generated by the CPU, while physical addresses exist in actual memory.

2. **Master page table calculations**: Given a logical address and page size, you should be able to extract the page number and offset, then use the page table to find the physical address. Practice multiple numerical problems.

3. **Know all page replacement algorithms**: FIFO, Optimal, and LRU are the most important. Understand how each works and their relative advantages. The optimal algorithm serves as a theoretical benchmark.

4. **Understand thrashing**: This important concept describes excessive paging activity that degrades system performance. Know the working set model and how to prevent thrashing.

5. **Remember the TLB role**: The Translation Lookaside Buffer is a hardware cache for page table entries. Understand how it improves performance and calculate effective access times.

6. **Differentiate between internal and external fragmentation**: These are common exam questions. Internal fragmentation occurs within allocated blocks; external fragmentation occurs between blocks.

7. **Know when page faults occur**: A page fault happens when a requested page is not in memory. Understand the complete page fault handling process—this is frequently tested.

8. **Be clear on swapping vs paging**: Swapping moves entire processes in and out of memory, while paging moves individual pages. This distinction is important.

9. **Understand the purpose of copy-on-write**: Know how it optimizes fork() operations by deferring page copying until modification is needed.

10. **Remember memory management functions**: The key functions include allocation, deallocation, protection, sharing, and logical organization of memory.
