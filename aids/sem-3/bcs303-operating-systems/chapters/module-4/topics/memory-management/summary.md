# Memory Management - Summary

## Key Definitions and Concepts

- **Memory Management**: The OS function of controlling and coordinating computer memory, allocating portions to processes for optimal system performance
- **Logical Address**: Address generated by the CPU (virtual address seen by programs)
- **Physical Address**: Actual address in main memory hardware
- **MMU (Memory Management Unit)**: Hardware device that translates logical addresses to physical addresses
- **Swapping**: Moving entire processes between memory and secondary storage
- **Paging**: Memory management scheme dividing memory into fixed-size pages and frames
- **Page Table**: Data structure mapping virtual pages to physical frames
- **TLB (Translation Lookaside Buffer)**: Hardware cache for page table entries
- **Virtual Memory**: Technique allowing execution of processes not completely in memory
- **Page Fault**: Exception when a referenced page is not in memory
- **Thrashing**: Excessive paging activity degrading system performance

## Important Formulas and Theorems

- **Effective Access Time with TLB**: EAT = p × (TLB + MA) + (1-p) × (TLB + 2×MA), where p = TLB hit ratio
- **Logical Address Structure**: Page number (p) + Page offset (d), where p = logical address / page size
- **Physical Address**: Frame number × frame size + page offset
- **Page Fault Rate**: Number of page faults / total memory references

## Key Points

1. Memory management enables multiple processes to share limited physical memory while maintaining isolation and protection

2. Address binding translates logical addresses to physical addresses at compile time, load time, or execution time

3. Paging eliminates external fragmentation by using fixed-size pages and frames but may cause internal fragmentation

4. Segmentation provides logical transparency with variable-sized segments but requires both internal and external management

5. Demand paging loads pages only when needed, reducing I/O and memory usage

6. The TLB significantly improves address translation speed by caching recent page table entries

7. Page replacement algorithms (FIFO, LRU, Optimal) determine which page to evict when memory is full

8. Thrashing occurs when the system spends more time paging than executing processes

## Common Mistakes to Avoid

- Confusing logical addresses with physical addresses—remember they are different; translation is required
- Misidentifying page number and offset in address calculations—use the correct number of bits based on page size
- Forgetting that page fault handling involves multiple memory accesses (page table lookup + actual access)
- Confusing internal and external fragmentation—internal occurs within allocated blocks; external occurs between blocks
