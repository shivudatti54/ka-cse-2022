# System Calls
**Why this topic matters:**
System calls are the fundamental mechanism by which application programs request services from the operating system kernel. Understanding system calls is essential for grasping how software interacts with hardware through the OS layer.

**Real-world applications:**
Every program you run -- from web browsers to text editors -- uses system calls to perform file I/O, create processes, allocate memory, and communicate over networks. Developers writing system-level software, device drivers, or performance-critical applications must understand how system calls work.

**Connection to other concepts:**
System calls connect directly to OS services (each service is accessed via system calls), process management (fork, exec, wait), file management (open, read, write, close), and the user-OS interface (CLI and GUI ultimately invoke system calls to perform actions).
