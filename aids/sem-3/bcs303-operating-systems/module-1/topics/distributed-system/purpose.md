# Learning Objectives

After studying this topic, you should students able to:

1. Describe the master-slave architecture of HDFS and explain the distinct roles of the NameNode and DataNodes.
2. Explain the concepts of data blocks and replication in HDFS, including the standard replication strategy and its benefits for fault tolerance.
3. Trace the sequence of steps involved in reading a file from and writing a file to HDFS.
4. Analyze the design trade-offs of HDFS, such as the use of large block sizes and the write-once-read-many model, and justify their purpose.
5. Differentiate between the basic HDFS setup and advanced configurations like Federation and High Availability (HA).
6. Recall basic HDFS shell commands for file manipulation and cluster administration.
