## 1. Introduction

In the digital world, data is constantly transmitted between devices across various communication channels. These channels—包括无线网络、光纤电缆、和卫星链接—are prone to noise, interference, and signal attenuation. When data travels from sender to receiver, it may get corrupted, leading to errors in the received bits. This is where **Error Detection and Correction Protocols** become essential.

### Real-World Relevance

Imagine you transfer money through online banking, and a single bit error changes your transfer amount from ₹1000 to ₹10000. Or consider a spacecraft sending back images from Mars—cosmic radiation can flip bits, corrupting scientific data. These scenarios demonstrate why error handling is critical in:

- **Telecommunications**: Mobile calls, video streaming
- **Data Storage**: Hard drives, SSDs, cloud storage
- **Financial Transactions**: ATM, online payments
- **Scientific Communication**: Space missions, weather satellites
- **Internet Protocol**: TCP/IP ensures reliable data delivery

This topic carries significant weight in Delhi University's BSc CS examinations, with questions frequently appearing on parity checks, CRC, Hamming codes, and checksum calculations.

## 2. Types of Errors

### 2.1 Single-Bit Error

- Only one bit in the data unit changes (0→1 or 1→0)
- Rare in serial transmission but common in parallel communication
- Example: `10110010` → `10110011`

### 2.2 Burst Error

- Two or more bits in the data unit are corrupted
- More common in serial transmission
- Length of burst = from first error to last error
- Example: `10110010` → `11101011` (burst length: 5 bits)

## 3. Error Detection Methods

### 3.1 Parity Check

#### Single Parity Check (Vertical Redundancy Check - VRC)

The simplest error detection method. Add one parity bit to make the total number of 1s either **even** (even parity) or **odd** (odd parity).

**Rules:**
- **Even Parity**: Total 1s including parity bit must be even
- **Odd Parity**: Total 1s including parity bit must be odd

**Example 1: Even Parity**
```
Data: 1011001 (4 ones)
Parity Bit: 0 (to make total ones = 4, which is even)
Transmitted: 10110010

Received: 10110011 (5 ones) → Error Detected!
```

**Example 2: Odd Parity**
```
Data: 1100110 (4 ones)
Parity Bit: 1 (to make total ones = 5, which is odd)
Transmitted: 11001101
```

#### Two-Dimensional Parity Check (Longitudinal Redundancy Check - LRC)

Data is arranged in a matrix. Parity bits are added both horizontally (for each row) and vertically (for each column).

**Example:**

```
Data Bytes:    11100101
               11001011
               00111001
               10101010
               
Row Parity:    10000101 (even parity for each row)
Column Parity: 10111001
```

This method can detect burst errors and locate the position of single-bit errors.

**Limitation**: If two bits in the same column are corrupted, errors may go undetected.

### 3.2 Checksum

A checksum is the arithmetic sum of fixed-size data segments, with overflow bits wrapped around (模运算). The receiver recalculates the checksum and compares it with the transmitted value.

**Algorithm:**
1. Divide data into k segments of n bits
2. Sum all segments
3. Add carry bits back to result (end-around carry)
4. Take complement (ones complement) to get checksum

**Worked Example (8-bit segments):**

```
Data: 01101001 11001101 11100111 10101010

Step 1: Sum all segments
  01101001  = 105
  11001101  = 205
  11100111  = 231
  10101010  = 170
  ──────────────────
  Sum       = 711

Step 2: Convert to 8-bit (drop overflow)
  711 = 1011000111
  Lower 8 bits: 11000111
  Carry (2): 10 → Add back: 11000111 + 10 = 11001001
  
Step 3: One's Complement
  Checksum = 00110110
```

**Verification at Receiver:**
- Sum received data + checksum
- Result should be all 1s (11111111)

### 3.3 Cyclic Redundancy Check (CRC)

CRC is the most powerful error detection method, widely used in digital networks and storage devices. It uses polynomial division.

#### Key Concepts:
- **Generator Polynomial (G(x))**: A predetermined polynomial of degree ≥ 1
- **Data Word (D)**: The message to be transmitted
- **Codeword**: The transmitted message (data + CRC bits)
- **CRC Bits**: Generated by dividing D × n (n = degree of G) by G

#### CRC-8, CRC-16, CRC-32

Common standards:
- **CRC-8**: Used in ATM headers, SMBus
- **CRC-16**: Used in Bluetooth, USB, SD cards
- **CRC-32**: Used in Ethernet, ZIP files, PNG

#### Worked Example:

**Given:**
- Data: `11010011101100`
- Generator: `1011` (degree 3)

**Step 1**: Append (degree of generator - 1) = 3 zeros to data
- Data becomes: `11010011101100` + `000` = `11010011101100000`

**Step 2**: Divide using binary division (XOR operation)

```
11010011101100000 ÷ 1011
```

Let's perform the division:

```
           1100001010
        _______________
1011 ) 11010011101100000
        1011
        ─────
         1100
         1011
         ─────
          0110
          0000
          ─────
           1101
           1011
           ─────
            1101
            1011
            ─────
             1100
             1011
             ─────
              1110
              1011
              ─────
               1010
               1011
               ─────
                0010
                0000
                ─────
                 0100
                 0000
                 ─────
                  1000
                  1011
                  ─────
                   0110
                   0000
                   ─────
                    1100
                    1011
                    ─────
                     1010
                     1011
                     ─────
                      0010 (Remainder)
```

**Remainder = 010**

**Step 3**: Append remainder to original data (without appended zeros)
- Transmitted message: `11010011101100` + `010` = `11010011101100010`

**Verification at Receiver:**
Divide received codeword by same generator. If remainder is 0, no error detected.

### 3.4 Internet Checksum (Used in IP, TCP, UDP)

The Internet checksum uses 16-bit words and one's complement arithmetic.

**Example:**

```
Data: 0x4A2F 0x9B1E 0x3C5D 0x7E2A

Step 1: Sum all 16-bit words
  0x4A2F = 18991
  0x9B1E = 39678
  0x3C5D = 15453
  0x7E2A = 32314
  ────────────────
  Sum     = 106436

Step 2: Wrap around carry (carry = 1, since 106436 > 65535)
  106436 - 65536 = 40900
  Add carry: 40900 + 1 = 40901 = 0x9FA5

Step 3: One's Complement
  Checksum = 0x605A (flip all bits: FFFF - 9FA5)
```

## 4. Error Correction Methods

### 4.1 Hamming Code

Hamming code, developed by Richard Hamming in 1950, can correct single-bit errors using redundant bits called **parity bits**.

#### Key Properties:

For a data word of length m bits, we need r redundant bits where:

```
2^r ≥ m + r + 1
```

| m (data bits) | r (parity bits) | Codeword length (m+r) |
|---------------|-----------------|----------------------|
| 1             | 2               | 3                    |
| 2-4           | 3               | 6-7                  |
| 5-11          | 4               | 9-15                 |
| 12-26         | 5               | 17-31                |
| 27-57         | 6               | 33-63                |

#### Positioning Parity Bits

Parity bits are placed at positions that are powers of 2: 1, 2, 4, 8, 16...

#### Worked Example 1: Encoding (7,4) Hamming Code

**Given:** Data word = `1101` (4 bits)
**Find:** Codeword with parity bits

**Step 1**: Calculate number of parity bits needed
```
2^r ≥ m + r + 1
2^r ≥ 4 + r + 1

r = 3: 2^3 = 8 ≥ 5 + 3 = 8 ✓
```
So we need 3 parity bits, codeword length = 7 bits.

**Step 2**: Position the bits
```
Position:  7  6  5  4  3  2  1
          ─────────────────────
          d4 d3 d2 p4 d1 p2 p1
Data:     1  1  0  -  1  -  -
```

Where:
- p1, p2, p4 are parity bits at positions 1, 2, 4
- d1, d2, d3, d4 are data bits

**Step 3**: Calculate parity bits (Even Parity)

**p1 (position 1)**: Check bits at positions 1,3,5,7
- Positions: 1(p1), 3(d1), 5(d2), 7(d4)
- Values: p1, 1, 0, 1
- p1 + 1 + 0 + 1 = even → p1 = 0

**p2 (position 2)**: Check bits at positions 2,3,6,7
- Positions: 2(p2), 3(d1), 6(d3), 7(d4)
- Values: p2, 1, 1, 1
- p2 + 1 + 1 + 1 = even → p2 + 3 = even → p2 = 1 (since 1+3=4, even)

**p4 (position 4)**: Check bits at positions 4,5,6,7
- Positions: 4(p4), 5(d2), 6(d3), 7(d4)
- Values: p4, 0, 1, 1
- p4 + 0 + 1 + 1 = even → p4 + 2 = even → p4 = 0

**Step 4**: Final codeword
```
Position:  7  6  5  4  3  2  1
          ─────────────────────
Codeword: 1  1  0  0  1  1  0
          d4 d3 d2 p4 d1 p2 p1
```

**Transmitted:** `1100101` (MSB to LSB)

---

#### Worked Example 2: Error Detection and Correction

**Scenario:** Received codeword = `1100111` (one bit corrupted)

**Step 1**: Arrange received bits
```
Position:  7  6  5  4  3  2  1
          ─────────────────────
Codeword: 1  1  0  0  1  1  1
```

**Step 2**: Check each parity group (Even Parity)

**Check p1 (positions 1,3,5,7)**: 1 + 1 + 0 + 1 = 3 (odd) → Error in group!
- Binary position indicators: positions 1, 3, 5, 7

**Check p2 (positions 2,3,6,7)**: 1 + 1 + 1 + 1 = 4 (even) → OK
- Binary position indicators: positions 2, 3, 6, 7

**Check p4 (positions 4,5,6,7)**: 0 + 0 + 1 + 1 = 2 (even) → OK
- Binary position indicators: positions 4, 5, 6, 7

**Step 3**: Calculate error position
```
Error position = p4 p2 p1 = 0 1 1 = binary 3
```
Error is at position 3!

**Step 4**: Correct the error
- Flip bit at position 3: 1 → 0

**Corrected codeword:** `1100101`

**Extract data:** d4=1, d3=1, d2=0, d1=1 → `1101` ✓

---

### 4.2 Single Parity Bit vs Hamming Code Comparison

| Feature | Single Parity | Hamming Code |
|---------|---------------|--------------|
| Error Detection | Can detect single-bit errors | Can detect single & double-bit errors |
| Error Correction | Not possible | Can correct single-bit errors |
| Redundancy | 1 bit per data unit | Log₂(data bits + 1) bits |
| Efficiency | Higher | Lower |

## 5. Python Implementation Examples

### 5.1 CRC-32 Implementation

```python
def crc32(data: bytes, poly: int = 0x04C11DB7) -> int:
    """
    Calculate CRC-32 checksum
    """
    crc = 0xFFFFFFFF
    for byte in data:
        crc ^= byte << 24
        for _ in range(8):
            if crc & 0x80000000:
                crc = (crc << 1) ^ poly
            else:
                crc <<= 1
            crc &= 0xFFFFFFFF
    return crc ^ 0xFFFFFFFF

# Example usage
data = b"Hello, World!"
checksum = crc32(data)
print(f"CRC-32 of '{data.decode()}': {checksum:08X}")
# Output: CRC-32 of 'Hello, World!': 9F5A1E07
```

### 5.2 Hamming Code (7,4) Encoder and Decoder

```python
def hamming_encode_7_4(data: str) -> str:
    """
    Encode 4-bit data into 7-bit Hamming code
    data: 4-character string of '0' or '1'
    """
    d = [int(b) for b in data]
    
    # Calculate parity bits (even parity)
    p1 = d[0] ^ d[1] ^ d[3]  # positions 1,3,5,7
    p2 = d[0] ^ d[2] ^ d[3]  # positions 2,3,6,7
    p4 = d[1] ^ d[2] ^ d[3]  # positions 4,5,6,7
    
    # Construct codeword: p1, p2, d1, p4, d2, d3, d4
    codeword = f"{p1}{p2}{d[0]}{p4}{d[1]}{d[2]}{d[3]}"
    return codeword

def hamming_decode_7_4(codeword: str) -> tuple:
    """
    Decode 7-bit Hamming code and correct if needed
    Returns: (corrected_codeword, was_error, data_bits)
    """
    c = [int(b) for b in codeword]
    
    # Check parity groups
    p1 = c[0] ^ c[2] ^ c[4] ^ c[6]
    p2 = c[1] ^ c[2] ^ c[5] ^ c[6]
    p4 = c[3] ^ c[4] ^ c[5] ^ c[6]
    
    # Error position (binary)
    error_pos = p4 * 4 + p2 * 2 + p1 * 1
    
    if error_pos > 0:
        # Correct the error
        c[error_pos - 1] ^= 1
        was_error = True
    else:
        was_error = False
    
    # Extract data bits: positions 3,5,6,7 → indices 2,4,5,6
    data = f"{c[2]}{c[4]}{c[5]}{c[6]}"
    
    return (''.join(map(str, c)), was_error, data)

# Example usage
original_data = "1101"
encoded = hamming_encode_7_4(original_data)
print(f"Original Data: {original_data}")
print(f"Encoded (Hamming 7,4): {encoded}")

# Simulate error at position 3
received = list(encoded)
received[2] = str(int(received[2]) ^ 1)  # Flip bit at position 3
received = ''.join(received)

print(f"Received (with error): {received}")
corrected, was_error, decoded = hamming_decode_7_4(received)
print(f"Corrected: {corrected}, Error detected: {was_error}, Decoded: {decoded}")
```

**Output:**
```
Original Data: 1101
Encoded (Hamming 7,4): 1100101
Received (with error): 1110101
Corrected: 1100101, Error detected: True, Decoded: 1101
```

## 6. Important Formulas for Delhi University Exams

### For Hamming Code:

1. **Number of parity bits (r):** `2^r ≥ m + r + 1`
2. **Parity bit positions:** Powers of 2 (1, 2, 4, 8, 16, ...)
3. **Error position calculation:** `P = P4 P2 P1` (binary to decimal)
4. **Minimum Hamming distance:** `d_min = 3` for single-bit error correction

### For CRC:

1. **Number of zeros appended:** Equal to degree of generator polynomial
2. **Codeword:** `Data bits + CRC remainder bits`
3. **Error detection:** If remainder = 0 at receiver, no error

## 9. Key Takeaways

1. **Error Detection vs. Correction**: Detection only identifies if errors occurred; correction locates and fixes them.

2. **Parity Check Methods**: Simple but limited—single parity can only detect odd-numbered errors; two-dimensional parity can locate errors.

3. **CRC**: Most powerful detection method, widely used in networking (Ethernet, WiFi) and storage (ZIP, PNG). Can detect burst errors up to the degree of generator polynomial.

4. **Checksum**: Common in Internet protocols (IP, TCP, UDP). Good for detecting random errors but weaker than CRC.

5. **Hamming Code**: Can correct single-bit errors and detect double-bit errors. Minimum distance d=3 for single-bit error correction.

6. **Trade-offs**: More redundancy means better error handling but lower efficiency. The choice depends on channel characteristics and application requirements.

7. **Exam Focus**: Delhi University questions frequently ask for:
   - Calculating parity bits for Hamming code
   - CRC polynomial division
   - Checksum calculation
   - Error detection vs correction capabilities

8. **Real-World Applications**: Error control is essential in 5G networks, satellite communication, deep-space transmission, and modern storage systems.
