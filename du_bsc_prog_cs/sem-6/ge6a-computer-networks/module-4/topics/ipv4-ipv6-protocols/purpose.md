# Learning Objectives

After studying this topic, you should be able to:

1. Explain the structure and purpose of each field in the IPv4 header, including version, TTL, protocol, and checksum fields.

2. Differentiate between IPv4 classful addressing, CIDR notation, and subnetting concepts for efficient address allocation.

3. Analyze IPv4 private address ranges and special IP addresses (loopback, broadcast, default route) with their appropriate use cases.

4. Describe the IPv6 header structure, highlighting improvements over IPv4 including simplified processing and enhanced QoS support.

5. Evaluate IPv6 address types (unicast, multicast, anycast) and their specific applications in network communication.

6. Apply IPv6 address compression rules to simplify address representation and expand compressed addresses to full form.

7. Compare IPv4 and IPv6 protocols across dimensions including address space, security, auto-configuration, and fragmentation handling.

8. Analyze IPv6 transition mechanisms (dual stack, tunneling, translation) and explain scenarios where each would be appropriate.
