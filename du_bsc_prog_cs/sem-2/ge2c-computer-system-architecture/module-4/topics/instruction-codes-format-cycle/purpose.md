# Learning Objectives

After studying this topic, you should be able to:

1. Define instruction code, instruction format, and instruction cycle, and explain their interrelationships in CPU operation.

2. Distinguish between the different types of instruction codes (zero-address, one-address, two-address, three-address) and explain their applications in various computer architectures.

3. Analyze the structure of common instruction formats and determine appropriate field sizes based on architectural requirements.

4. Trace the execution of instructions through the fetch-decode-execute cycle, identifying the role of each CPU register (PC, MAR, MBR, IR, AC).

5. Compare different addressing modes (direct, indirect, immediate, register) and explain how each affects instruction execution and effective address calculation.

6. Design simple instruction formats given specific constraints on word size, opcode length, and address field requirements.

7. Evaluate the trade-offs between different instruction set architectures in terms of code density, execution speed, and hardware complexity.

8. Apply the knowledge of instruction cycle to understand how higher-level programs are translated and executed at the machine level.
