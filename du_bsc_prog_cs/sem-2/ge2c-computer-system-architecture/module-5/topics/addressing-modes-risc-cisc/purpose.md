# Learning Objectives

After studying this topic, you should be able to:

1. Define and explain all major addressing modes used in computer architecture, including immediate, register, direct, indirect, register indirect, indexed, base-register, relative, and stack addressing.

2. Identify the advantages and disadvantages of each addressing mode and determine appropriate scenarios for their use in assembly language programming.

3. Differentiate between RISC and CISC architectural philosophies and explain the rationale behind each approach.

4. Compare and contrast RISC and CISC architectures across multiple dimensions including instruction set size, instruction format, addressing modes, register count, and execution model.

5. Analyze real-world processor families (Intel x86, ARM, MIPS) and classify them according to RISC or CISC architecture.

6. Explain the load-store architecture characteristic of RISC processors and understand why this design choice improves performance.

7. Evaluate the modern convergence of RISC and CISC architectures in contemporary processor design.

8. Apply understanding of addressing modes and architectural concepts to solve problems in assembly language programming and computer system design.
