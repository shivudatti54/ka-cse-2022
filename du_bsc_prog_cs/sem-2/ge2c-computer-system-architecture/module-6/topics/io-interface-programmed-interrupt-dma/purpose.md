### IO Interface: Programmed I/O, Interrupt, and DMA

**Purpose:** In modern computer systems, efficient I/O is crucial for achieving high performance and low latency. This module examines programmed I/O, interrupt‑driven I/O, and Direct Memory Access (DMA), the three fundamental mechanisms for CPU–peripheral communication. Understanding these techniques enables students to design efficient I/O subsystems, write device drivers, and optimize system resource utilization in embedded and general‑purpose computing environments.

**Learning Objectives:**
- Explain the operation of programmed I/O, interrupt‑driven I/O, and DMA in terms of control flow and data movement.
- Compare the performance characteristics (throughput, CPU overhead, latency) of each I/O method.
- Identify the key hardware components of a typical I/O interface and their functions.
- Design a simple DMA transfer scheme for a block device, specifying control signals and memory addressing.
- Analyze the impact of interrupt handling on system responsiveness and the role of interrupt service routines.
- Implement a basic interrupt‑driven I/O handler in pseudo‑code or a high‑level language.
- Evaluate trade‑offs between hardware complexity, cost, and I/O efficiency in system design.
