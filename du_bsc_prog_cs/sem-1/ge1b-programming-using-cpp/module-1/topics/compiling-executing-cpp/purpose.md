### Purpose  
Understanding how C++ source code is compiled and executed is fundamental to turning high‑level programs into machine‑ready binaries, a skill that underpins all real‑world software development, from embedded systems to large‑scale applications. Mastery of the compilation pipeline equips students with the ability to debug, optimize, and deploy cross‑platform C++ solutions effectively.

### Learning Objectives
- **Explain** the stages of the C++ compilation pipeline (preprocessing, compilation, assembly, and linking).  
- **Demonstrate** how to compile a simple C++ program using a command‑line compiler such as `g++`.  
- **Execute** compiled binaries and interpret runtime output and error messages.  
- **Identify** and resolve common compilation errors and warnings.  
- **Compare** the effects of different compiler flags (e.g., `-O2`, `-g`, `-Wall`) on performance and debugging.  
- **Analyze** the differences among source code, object code, and executable files.  
- **Utilize** basic build tools or IDEs to manage multi‑file C++ projects.  
- **Discuss** the role of the loader and operating system in running compiled executables.
