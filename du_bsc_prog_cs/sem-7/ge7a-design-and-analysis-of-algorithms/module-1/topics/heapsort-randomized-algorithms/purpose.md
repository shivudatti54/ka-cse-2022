## Purpose  
Heapsort is a fundamental comparison‑based sorting algorithm that provides guaranteed *O(n log n)* worst‑case performance and serves as the backbone of priority‑queue data structures used in scheduling, event simulation, and graph algorithms such as Dijkstra’s. Introducing randomization to heapsort (or using heaps as a building block in randomized contexts) equips students to handle adversarial inputs, analyse expected running times with probability theory, and apply these concepts to real‑world problems like load balancing and efficient resource management.

## Learning Objectives
- **Explain** the binary‑heap property, the array‑based representation of a heap, and the heapify operation.  
- **Implement** the classic heapsort algorithm and a randomized variant that shuffles the input before building the heap.  
- **Analyze** the worst‑case, average‑case, and expected time complexities of heapsort using recurrence relations and probability.  
- **Compare** heapsort’s performance and memory usage with other *O(n log n)* sorts such as mergesort and quicksort.  
- **Evaluate** the effect of randomization on avoiding pathological inputs and on the variance of running time.  
- **Apply** heap‑based priority queues to practical scenarios like job scheduling, event‑driven simulation, and selection algorithms.  
- **Design** a randomized heapsort algorithm and prove its expected *O(n log n)* bound.  
- **Demonstrate** how heap properties translate to efficient data‑structure implementations in code.
