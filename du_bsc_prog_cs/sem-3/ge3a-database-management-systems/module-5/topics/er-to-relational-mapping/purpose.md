# Learning Objectives

After studying this topic, you should be able to:

1. Explain the need for converting ER models to relational schemas in database design

2. Apply the mapping rules for regular (strong) entity types to create relational tables with appropriate primary keys

3. Map weak entity types by incorporating the owner's primary key and forming composite primary keys

4. Convert 1:1, 1:N, and M:N binary relationships to relational schema using appropriate strategies (foreign keys or junction tables)

5. Handle multivalued attributes by creating separate relations to store multiple values

6. Map n-ary (ternary and higher) relationships to relational schema using junction relations with composite primary keys

7. Distinguish between attributes that should be stored versus those that should be derived, and apply this knowledge in mapping

8. Design complete relational schemas from complex ER diagrams by systematically applying all mapping rules
