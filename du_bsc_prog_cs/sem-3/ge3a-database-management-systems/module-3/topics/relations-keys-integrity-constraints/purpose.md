# Learning Objectives

After studying this topic, you should be able to:

1. Define the relational model and explain the structure of relations (tables) including tuples, attributes, and domains
2. Distinguish between different types of keys: Super Key, Candidate Key, Primary Key, Alternate Key, Foreign Key, Composite Key, and Unique Key
3. Apply the concept of minimal super keys to identify candidate keys in a given relation schema
4. Explain and implement entity integrity, referential integrity, domain integrity, and user-defined integrity constraints
5. Design database schemas with appropriate primary keys and foreign keys to establish relationships between tables
6. Write SQL statements to create tables with various integrity constraints (PRIMARY KEY, FOREIGN KEY, UNIQUE, NOT NULL, CHECK)
7. Analyze scenarios to determine which integrity constraint is being violated and suggest corrections
8. Understand the practical importance of keys and constraints in maintaining data quality and consistency in real-world database applications
