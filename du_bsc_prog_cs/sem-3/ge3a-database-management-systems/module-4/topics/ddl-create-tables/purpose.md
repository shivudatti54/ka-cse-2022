# Learning Objectives

After studying this topic, you should be able to:

1. Understand the basic syntax of the CREATE TABLE statement in SQL and identify its essential components.

2. Explain and differentiate between various SQL data types including numeric, string, and date/time types, and select appropriate data types for given scenarios.

3. Apply different types of constraints (NOT NULL, UNIQUE, PRIMARY KEY, FOREIGN KEY, CHECK, DEFAULT) in CREATE TABLE statements to enforce data integrity.

4. Distinguish between column-level and table-level constraint definitions and determine when to use each appropriately.

5. Design and create tables with composite primary keys using table-level constraint syntax.

6. Create related tables with proper foreign key relationships to establish referential integrity between database tables.

7. Analyze given requirements and translate them into complete CREATE TABLE statements with all necessary constraints.

8. Demonstrate practical knowledge of auto-increment/identity columns for generating unique sequential identifiers.
