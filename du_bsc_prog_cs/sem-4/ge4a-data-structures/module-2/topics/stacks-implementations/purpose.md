# Learning Objectives

After studying this topic, you should be able to:

1. Define a stack data structure and explain the Last In, First Out (LIFO) principle that governs its behavior.

2. Identify and describe all stack operations including push, pop, peek, isEmpty, and isFull, along with their purposes and return types.

3. Implement a stack using an array (static implementation), including initialization, push, pop, peek, and status checking operations.

4. Implement a stack using a linked list (dynamic implementation), demonstrating how nodes are created, linked, and freed during operations.

5. Analyze the time and space complexity of stack operations for both array-based and linked list-based implementations.

6. Apply stacks to solve practical problems such as checking balanced parentheses and understanding function call management.

7. Compare and contrast array-based and linked list-based stack implementations in terms of memory usage, flexibility, and performance characteristics.
