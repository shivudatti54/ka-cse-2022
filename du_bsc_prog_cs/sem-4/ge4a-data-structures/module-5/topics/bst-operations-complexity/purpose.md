# Learning Objectives

After studying this topic, you should be able to:

1. Define the properties of a Binary Search Tree and explain how it differs from a regular binary tree.

2. Implement and analyze the search operation in a BST, explaining why it achieves O(log n) average time complexity.

3. Trace through the insertion algorithm step-by-step and identify how insertion order affects tree structure.

4. Handle all three cases of deletion (leaf node, one child, two children) and explain the use of in-order successor/predecessor.

5. Compare best, average, and worst-case time complexities for all major BST operations.

6. Explain the significance of tree height and balance in determining operational efficiency.

7. Apply BST traversal methods (in-order, pre-order, post-order) and describe their practical applications.

8. Evaluate when BSTs are preferred over other data structures like arrays, linked lists, or hash tables.
