# Learning Objectives

After studying this topic, you should be able to:

1. Analyze and select appropriate architectural patterns for given business requirements
2. Design fault-tolerant systems using circuit breakers and bulkheads
3. Evaluate trade-offs between consistency, availability, and partition tolerance
4. Implement sharding strategies for relational and NoSQL databases
5. Create capacity plans considering RPS, data storage, and network bandwidth
6. Optimize system performance using caching layers and CDN configurations
7. Design secure authentication flows using OAuth 2.0 and JWT
8. Document system designs using UML diagrams and C4 model
