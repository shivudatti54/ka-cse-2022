# Learning Objectives

After studying this topic, you should be able to:

1. Explain the three fundamental properties of cryptographic hash functions with real-world consequences of their failure
2. Describe the complete process of digital signature generation and verification using RSA and DSA algorithms
3. Compare and contrast HMAC with digital signatures in terms of security guarantees and implementation requirements
4. Analyze the impact of hash length (e.g., SHA-256 vs SHA-512) on collision resistance and system performance
5. Evaluate the suitability of specific hash/signature combinations for given application scenarios (e.g., IoT vs banking)
6. Demonstrate how chosen-prefix collisions in MD5 can compromise document integrity despite digital signatures
7. Implement a simplified version of SHA-256's message scheduling and compression functions
8. Critically assess NIST recommendations for post-quantum digital signature algorithms like CRYSTALS-Dilithium
