# Major Cloud Platforms - Summary

## Key Definitions and Concepts

- **Cloud Platform:** On-demand computing resources and services delivered over the internet
- **Region:** Geographic area containing multiple data centers (Availability Zones)
- **Availability Zone (AZ):** Physically isolated data center within a region
- **IaaS/PaaS/SaaS:** Infrastructure as a Service, Platform as a Service, Software as a Service delivery models
- **Serverless Computing:** Event-driven code execution without server management (AWS Lambda, Azure Functions, Cloud Functions)

## Important Services by Platform

**AWS:** EC2, S3, Lambda, RDS, DynamoDB, EMR, Redshift, Athena, Kinesis
**Azure:** Virtual Machines, Blob Storage, Azure SQL, Cosmos DB, Synapse, HDInsight, Databricks
**GCP:** Compute Engine, Cloud Storage, BigQuery, Dataflow, Dataproc, Pub/Sub, GKE
**IBM Cloud:** Watson AI, Cloud Functions, Cloud Pak, Autonomous Database (Oracle)

## Key Points

1. AWS is the market leader with 200+ services, pioneered cloud computing in 2006
2. Azure offers best hybrid cloud integration for Microsoft-centric enterprises
3. GCP leads in data analytics, machine learning, and Kubernetes (GKE)
4. All three major providers offer compute, storage, database, analytics, and AI/ML services
5. Big data services: AWS EMR/Redshift, Azure Synapse/HDInsight, GCP BigQuery/Dataflow
6. Global infrastructure includes Regions and Availability Zones for high availability
7. Pricing models include pay-as-you-go, reserved instances, and savings plans
8. IBM Cloud focuses on enterprise AI and hybrid deployments
9. Oracle Cloud specializes in database workloads with Autonomous Database

## Common Mistakes to Avoid

- Confusing similar services across platforms (e.g., AWS S3 bucket vs. Azure Blob container)
- Ignoring data residency and compliance requirements when selecting regions
- Underestimating egress data transfer costs in cloud pricing
- Selecting the wrong instance type for specific workload requirements

## Revision Tips

1. Create a comparison table of equivalent services across AWS, Azure, and GCP
2. Practice designing solutions on free tiers of each platform
3. Review recent feature announcements and service updates
4. Focus on big data and analytics services as they are crucial for the MCA curriculum
