# Cloud Service Deployment Models - Summary

## Key Definitions and Concepts
- Public Cloud: Shared infrastructure via third-party providers (AWS, Azure)
- Private Cloud: Dedicated infrastructure with granular control
- Hybrid Cloud: Integration of public/private with workload portability
- Community Cloud: Shared infrastructure for specific industry verticals
- Multi-Cloud: Strategic use of multiple cloud providers

## Important Formulas and Theorems
- Cloud TCO Formula: Total Cost = (Infrastructure + Operations + Security) - (Reserved Instances Savings + Spot Instance Savings)
- CAPEX vs OPEX Model: Private cloud → High CAPEX, Public cloud → OPEX dominant
- NIST Cloud Reference Architecture: Five essential characteristics, three service models

## Key Points
- Public clouds offer scalability but require strong IAM policies
- Private clouds need virtualization management expertise
- Hybrid clouds require API standardization (OpenAPI, gRPC)
- Community clouds enforce industry-specific compliance (PCI DSS, HIPAA)
- Multi-cloud architectures need service mesh (Istio, Linkerd)
- Edge computing extends hybrid models to network periphery
- Cloud security posture management (CSPM) is critical across models

## Common Mistakes to Avoid
- Confusing hybrid cloud with multi-cloud architectures
- Ignoring data gravity in hybrid deployments
- Underestimating egress costs in multi-cloud setups
- Assuming private clouds are inherently more secure
