# Learning Objectives

After studying this topic, you should be able to:

1. Differentiate between actors, use cases, and system boundaries in UML
2. Model <<include>> and <<extend>> relationships with proper notation
3. Construct Use Case diagrams for complex systems like UPI payment apps
4. Analyze real-world scenarios to identify extension points
5. Apply generalization to both actors and use cases
6. Validate Use Case diagrams against functional requirements
7. Translate Use Case models into test scenarios
8. Critique existing diagrams for common errors in actor identification
