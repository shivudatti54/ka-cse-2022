# Software Process Improvement - Summary

## Key Definitions and Concepts
- SPI: Systematic approach to improve software development effectiveness
- CMMI: Staged/continuous model for process maturity assessment
- DPMO: Defects Per Million Opportunities (Six Sigma metric)
- Process Asset Library: Repository of organizational process knowledge
- Baselining: Capturing current process performance metrics

## Important Formulas and Theorems
- Sigma Level = NORMSINV(1 - DPMO/1,000,000) + 1.5
- ROI = (Cost Savings - SPI Investment) / SPI Investment × 100
- Process Capability Index (Cpk) = min[(USL - μ)/3σ, (μ - LSL)/3σ]
- COCOMO II: Effort = A × (Size)^B × ∏EM_i

## Key Points
- CMMI Level 3 requires institutionalized standard processes
- ISO 15504 uses 6-point capability scale (0-Incomplete to 5-Optimizing)
- Six Sigma's 1.5σ shift accounts for long-term process variation
- Agile SPI emphasizes continuous improvement through retrospectives
- PSP focuses on personal measurement and defect prevention
- Statistical Process Control uses X-bar/R charts for variance analysis
- SPI success factors: management commitment, measurement program, pilot projects

## Common Mistakes to Avoid
- Confusing CMMI maturity levels with ISO 15504 capability levels
- Neglecting cultural factors in SPI implementation
- Overlooking measurement system analysis in Six Sigma
- Applying Agile SPI to safety-critical systems without modifications
- Treating SPI as one-time activity instead of continuous process
