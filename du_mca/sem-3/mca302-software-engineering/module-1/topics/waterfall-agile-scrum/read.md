# Waterfall, Agile, and Scrum Methodologies

## Introduction
Software development methodologies provide structured approaches to managing complex projects. The Waterfall model, introduced in 1970 by Winston Royce, represents a linear sequential approach where each phase must be completed before the next begins. Its rigid structure made it popular for government and aerospace projects with fixed requirements.

Agile methodologies emerged in 2001 through the Agile Manifesto as a response to changing market demands. This iterative approach emphasizes customer collaboration, working software, and adaptability. Scrum, a specific Agile framework, introduced time-boxed iterations (sprints) and defined roles to manage evolving requirements effectively.

The shift from Waterfall to Agile/Scrum reflects modern software engineering needs where 45% of projects face changing requirements (Standish Group, 2020). Today, 85% of organizations use Agile methods (Digital.ai, 2022), with Scrum being implemented by 66% of them, making this comparison critical for software engineers.

## Key Concepts

1. **Waterfall Model**
   - Phases: Requirements → Design → Implementation → Verification → Maintenance
   - V-Model: Verification & Validation parallel to development phases
   - Critical Path Method (CPM) for scheduling
   - Change Control Boards for requirement modifications

2. **Agile Principles**
   - 4 Values: Individuals/interactions over processes, Working software over documentation
   - 12 Principles including daily collaboration, sustainable development pace
   - Empirical Process Control (Transparency, Inspection, Adaptation)

3. **Scrum Framework**
   - Roles: Product Owner, Scrum Master, Development Team
   - Artifacts: Product Backlog, Sprint Backlog, Increment
   - Events: Sprint Planning, Daily Scrum, Sprint Review, Retrospective
   - Metrics: Velocity, Burn-down Charts, Cycle Time

4. **Hybrid Models**
   - Wagile (Waterfall + Agile)
   - SAFe (Scaled Agile Framework)
   - Scrumban (Scrum + Kanban)

## Examples

**Example 1: Waterfall in Aerospace Software**
Problem: Develop flight control software with strict certification requirements
Solution:
1. 6-month requirements gathering with FAA
2. Architectural design review with NASA
3. Implementation using DO-178C standards
4. 3-month testing phase with hardware-in-loop
5. Final delivery after 22 months

**Example 2: Scrum in Fintech SaaS**
Problem: Build AI-driven fraud detection with evolving regulations
Solution:
- 2-week sprints
- Sprint 1: Basic transaction monitoring (Story Points: 13)
- Sprint 2: Integration with RBI APIs (Velocity: 15)
- Sprint 3: Machine learning model training (Burn-down shows 20% scope creep)
- Daily standups resolve cloud deployment blockers

**Example 3: Agile Transition in Healthcare**
Problem: Hospital CMS migration with HIPAA compliance
Solution:
- Phase 1: Waterfall for infrastructure setup (6 months)
- Phase 2: Scrum for patient portal development
- Used Jira for backlog refinement
- Achieved 40% faster user acceptance testing

## Exam Tips
1. Always contrast Waterfall's "Big Design Up Front" with Agile's emergent design
2. Memorize Scrum events with their timeboxes: Sprint Planning (8hrs/4wk sprint), Daily Scrum (15min)
3. For 10-mark questions, use the Iron Triangle comparison (Waterfall: Fixed scope vs Agile: Fixed time/cost)
4. Remember V-Model as Waterfall extension - testing phases map to development stages
5. When asked about limitations, mention Waterfall's difficulty with changes after 30% completion
6. For case studies, recommend Agile when requirements are volatile (e.g., AI projects)
7. Cite real-world examples: NASA uses Waterfall, Spotify uses Scrum@Scale
