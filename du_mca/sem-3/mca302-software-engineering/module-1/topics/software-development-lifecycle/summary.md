# Software Development Lifecycle (SDLC) - Summary

## Key Definitions and Concepts

- **SDLC (Software Development Lifecycle)**: A systematic, structured approach defining the complete process of creating, maintaining, and updating software systems from conception to retirement.

- **Requirements Engineering**: The first phase involving gathering, analyzing, documenting, and validating software requirements.

- **SRS (Software Requirements Specification)**: The primary output document of the requirements phase containing all functional and non-functional requirements.

- **Verification vs. Validation**: Verification ensures the product is built correctly (are we building the product right?), while validation ensures the correct product is built (are we building the right product?).

## Important Formulas and Models

- **Waterfall Model**: Sequential approach with linear progression through phases—Requirements → Design → Implementation → Testing → Deployment → Maintenance

- **V-Model**: Extension of Waterfall with corresponding testing phases for each development phase—unit testing corresponds to detailed design, integration testing to high-level design, system testing to requirements

- **Spiral Model**: Cyclical process with four activities—Planning, Risk Analysis, Engineering, Evaluation—allowing repeated iterations

- **Agile Model**: Iterative approach emphasizing flexibility, collaboration, customer feedback, and adaptability through short development cycles (sprints)

## Key Points

1. SDLC provides a structured framework ensuring systematic development, quality assurance, and stakeholder satisfaction.

2. Requirements Engineering is critical as errors in this phase account for approximately 40% of software project failures.

3. The Waterfall model is suitable for projects with stable, well-defined requirements; Agile suits projects with evolving requirements.

4. Testing is integral throughout SDLC, not just a final phase—the V-Model explicitly demonstrates this parallel relationship.

5. Maintenance consumes 40-60% of total software lifecycle costs and includes corrective, adaptive, perfective, and preventive maintenance.

6. The Spiral model specifically addresses risk management through iterative risk analysis and prototype development.

7. Agile emphasizes working software over documentation, responding to change over following a plan, and customer collaboration.

8. Modern DevOps and CI/CD practices integrate with SDLC to enable continuous delivery and automation.

## Common Mistakes to Avoid

1. **Confusing verification with validation**: Remember verification checks conformance to specifications; validation checks fitness for use.

2. **Skipping documentation**: While Agile values working software over comprehensive documentation, some documentation is essential for maintenance.

3. **Ignoring maintenance phase**: Many students focus only on development phases, forgetting maintenance is the longest and costliest phase.

4. **Wrong model selection**: Applying Agile to projects with fixed requirements or Waterfall to innovative projects with unclear requirements leads to failure.

5. **Neglecting risk analysis**: The Spiral model's key differentiator is risk management—ignoring this defeats its purpose.
