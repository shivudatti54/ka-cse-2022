# Software Development Lifecycle (SDLC)

## Introduction
The Software Development Lifecycle (SDLC) is a structured framework defining the processes for building high-quality software systems. It provides a systematic approach to software development through defined phases, ensuring alignment with user requirements while optimizing resources. In DU's MCA program, SDLC forms the backbone of software engineering practices, critical for managing complex projects in banking, healthcare, and enterprise systems.

Modern software systems like UPI payment gateways (e.g., BHIM) and Aadhaar infrastructure rely on rigorous SDLC implementation. The lifecycle mitigates risks through phased development, with industry reports showing 60% reduction in post-deployment defects when following structured SDLC models. With India's software exports reaching $200 billion in 2023, SDLC expertise remains vital for DU graduates targeting roles in tech giants like TCS and Infosys.

## Key Concepts
1. **SDLC Models**:
   - **Waterfall**: Linear sequential approach (Requirements → Design → Implementation → Testing → Deployment). Ideal for projects with fixed requirements (e.g., government systems).
   - **Agile**: Iterative development using sprints. Used by 70% of Indian IT companies for web/mobile apps.
   - **Spiral Model**: Risk-driven approach combining Waterfall and prototyping. Common in aerospace and defense projects.
   - **V-Model**: Verification & validation parallel to development phases. Critical for medical software compliance.

2. **Core Phases**:
   - Requirements Analysis (SRS documentation)
   - System Design (Architecture diagrams, ER models)
   - Implementation (Coding standards, version control)
   - Testing (Unit, Integration, System, UAT)
   - Maintenance (Bug tracking, CI/CD pipelines)

3. **Supporting Processes**:
   - Configuration Management (Git, SVN)
   - Quality Assurance (ISO 9001 standards)
   - Risk Management (FMEA techniques)

## Examples
**Example 1: Waterfall Model for Banking Software**
1. Requirements: RBI compliance for transaction logging
2. Design: UML sequence diagrams for fund transfer
3. Implementation: Java backend with J2EE
4. Testing: JMeter for load testing 1M transactions/hr
5. Maintenance: Quarterly security patches

**Example 2: Agile for E-Commerce App**
- Sprint 1: User authentication module (Spring Security)
- Sprint 2: Product search API (Elasticsearch integration)
- Sprint 3: Payment gateway (Razorpay SDK)
- Daily standups and Jira for task tracking

**Example 3: Spiral Model for Healthcare System**
1. Risk Analysis: HIPAA compliance for patient data
2. Prototype: FHIR-based EHR interface
3. Develop: Python/Django backend with AES-256 encryption
4. Evaluate: Third-party penetration testing

## Exam Tips
1. Compare SDLC models using parameters like flexibility, risk handling, and cost.
2. Memorize IEEE 12207 standards for lifecycle processes.
3. Practice drawing UML activity diagrams for each SDLC phase.
4. Understand how DevOps extends SDLC through CI/CD (Jenkins, Docker).
5. Case study focus: Relate models to project types (e.g., Agile for startups vs Waterfall for regulatory projects).
6. Know the difference between Verification (design compliance) and Validation (user needs).
7. Always mention real-world tools: Jira for Agile, HP ALM for Waterfall testing.
