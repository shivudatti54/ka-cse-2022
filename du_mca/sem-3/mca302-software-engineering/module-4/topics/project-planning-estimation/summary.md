# Project Planning & Estimation - Summary

## Key Definitions and Concepts
- **KLOC**: Thousand Lines of Code (measure of project size)
- **Velocity**: Agile metric for work completed per sprint
- **Slack Time**: CPM parameter showing task delay tolerance
- **TCF**: Technical Complexity Factor in Function Point Analysis

## Important Formulas and Theorems
- **Basic COCOMO**: Effort = a×(KLOC)^b
- **Function Points**: FP = UFP × TCF
- **PERT Time**: (O + 4M + P)/6
- **Risk Exposure**: RE = Probability × Monetary Impact

## Key Points
- WBS decomposition should follow 8/80 rule (tasks 8-80 hours)
- COCOMO II adds scale factors for modern development practices
- Critical path determines minimum project duration
- Function Points measure functionality independent of technology
- Planning Poker combines expert judgment in Agile teams
- Resource leveling avoids over-allocation in Gantt charts
- Risk register must be updated throughout project lifecycle

## Common Mistakes to Avoid
- Confusing effort (person-months) with duration (calendar time)
- Ignoring complexity multipliers in Function Point calculations
- Missing task dependencies in CPM network diagrams
- Using story points as absolute time measures in Agile

## Revision Tips
1. Practice COCOMO calculations with different project modes
2. Create WBS for familiar systems like DU's exam portal
3. Solve past DU papers on CPM (2022 question had 7-task problem)
4. Use flashcards for Function Point complexity weights
5. Join study groups to simulate Agile planning poker sessions
