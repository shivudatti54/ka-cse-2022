# Learning Objectives

After studying this topic, you should be able to:

1. Define risk exposure and calculate it using EMV (Expected Monetary Value)
2. Apply FMEA techniques to critical systems like medical software
3. Develop risk mitigation plans using STRIDE (Spoofing, Tampering, Repudiation, Info Disclosure, DoS, Elevation of Privilege) framework
4. Formulate risk transfer strategies for cloud compliance (GDPR, HIPAA)
5. Evaluate risk management frameworks (CMMI-DEV v2.0, ISO 31000) for enterprise adoption
6. Analyze real-world case studies using OCTAVE (Operationally Critical Threat, Asset, and Vulnerability Evaluation) methodology
7. Design risk monitoring dashboards with leading indicators (requirements volatility index, defect density)
8. Integrate risk management with agile methodologies through Risk Burndown Charts
