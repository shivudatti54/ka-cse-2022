# Learning Objectives

After studying this topic, you should be able to:

1. Explain the architecture of a query optimizer and its role in DBMS
2. Transform logical query plans using equivalence rules
3. Analyze and compare physical execution plan costs
4. Apply predicate pushdown and join reordering techniques
5. Design effective indexes for given query patterns
6. Interpret EXPLAIN plan outputs from real DBMS
7. Evaluate tradeoffs between different join algorithms
8. Utilize database statistics for cardinality estimation
