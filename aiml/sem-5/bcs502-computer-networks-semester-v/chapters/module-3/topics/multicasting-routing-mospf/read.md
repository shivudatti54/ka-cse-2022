# **Module 3: Multicast Routing - MOSPF (Multicast Open Shortest Path First)**

### **1. Introduction**

In computer networks, multicast is the efficient one-to-many or many-to-many distribution of data, essential for applications like video conferencing, online gaming, and live stock tickers. While unicast routing protocols like OSPF find the shortest path between two points, multicast requires a different approach: building optimal **distribution trees** from a source to all interested receivers.

MOSPF, or Multicast Open Shortest Path First, is an extension of the widely used OSPF (Open Shortest Path First) unicast routing protocol. It was designed to provide multicast routing within a single OSPF routing domain (an Autonomous System). Unlike other protocols that use a separate mechanism, MOSPF integrates multicast routing directly into the OSPF link-state framework.

---

### **2. Core Concepts of MOSPF**

#### **2.1. The Foundation: OSPF Link-State Advertisements (LSAs)**
OSPF works by having every router maintain an identical **Link-State Database (LSDB)**, a map of the entire network topology. Routers exchange information using Link-State Advertisements (LSAs). MOSPF introduces a new type of LSA: **Group-Membership LSA (Type 6 LSA)**. This LSA is generated by a router to announce that it has active members (hosts) for a specific multicast group on one of its connected networks.

#### **2.2. Key Mechanism: Source-Based Trees**
MOSPF uses **Shortest Path Tree (SPT)** routing. This means that for each unique combination of a **source (S)** and a **multicast group (G)**, MOSPF calculates a separate delivery tree. This tree is rooted at the source and branches out through the network to reach all group members. The paths in this tree are the shortest paths, as determined by the OSPF algorithm.

#### **2.3. How MOSPF Works: A Step-by-Step Process**

1.  **Membership Registration:** When a host on a local network (e.g., connected to Router R1) wants to join a multicast group G (using IGMP), it informs its local router (R1).
2.  **Flooding Group Information:** Router R1 generates a **Group-Membership LSA** for group G and floods it throughout the entire OSPF area. This informs every other OSPF router that R1 has members for G.
3.  **Building the Source-Based Tree:** When a source (e.g., Server S) sends a packet to group G, the first MOSPF router (the one closest to the source) receives it. This router, knowing the entire network topology from the LSDB and the location of all group members from the Group-Membership LSAs, performs an on-demand **Dijkstra's algorithm** calculation.
4.  **The Dijkstra's Pruned Calculation:** The router doesn't calculate paths to every node. Instead, it calculates the shortest path from the source to *only* those routers that have advertised membership for group G (via Group-Membership LSAs). This creates a "pruned" SPT that includes only the necessary branches to reach all receivers.
5.  **Packet Forwarding:** The multicast packet is then forwarded along the branches of this computed tree. Routers that are not on the SPT for (S, G) simply discard the packet.

**Example:** Imagine a network with five routers (R1 to R5). A video server (Source S) is connected to R1. Hosts connected to R3 and R5 want to receive the stream (Group G).
*   R3 and R5 flood Group-Membership LSAs for G.
*   When S sends a packet, R1 performs the Dijkstra calculation. It finds the shortest path to R3 (e.g., R1->R2->R3) and to R5 (e.g., R1->R4->R5).
*   The resulting SPT has branches at R1 going to R2 and R4. R2 forwards to R3, and R4 forwards to R5.
*   Router R2 will not forward the packet towards R4, as it's not on the path to any member. This prevents unnecessary traffic.

### **3. Advantages and Limitations**

| Advantage | Limitation |
| :--- | :--- |
| **Efficiency:** Creates optimal (shortest path) trees, minimizing network latency and resource usage. | **Scalability:** The (S, G) state and on-demand Dijkstra calculations consume significant CPU and memory for large networks with many sources and groups. |
| **No Flooding:** Data packets are not flooded; they follow the pre-computed SPT. | **Limited Scope:** Designed for a single OSPF area. Inter-area and inter-AS multicast require additional, complex mechanisms. |
| **Integration:** Leverages existing OSPF infrastructure and topology knowledge. | **Overhead:** Group-Membership LSAs are flooded to every router in the area, creating control overhead. |

---

### **4. Summary: Key Points**

*   **What it is:** MOSPF is a **multicast routing protocol** for **single OSPF domains**.
*   **Mechanism:** It uses **source-based shortest path trees (SPT)** for each (Source, Group) pair.
*   **How it works:**
    1.  Routers announce group membership by flooding **Group-Membership LSAs (Type 6)**.
    2.  Upon receiving the first packet from a source, the router calculates the delivery tree using a **pruned Dijkstra's algorithm** on the OSPF LSDB.
    3.  Packets are forwarded only along the branches of this tree.
*   **Strength:** Provides efficient, optimal paths within a confined network.
*   **Weakness:** Poor scalability due to computational overhead, making it less suitable for large, complex networks like the public internet. Protocols like **PIM (Protocol Independent Multicast)** are more commonly used in such scenarios.
