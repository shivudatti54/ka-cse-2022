# Controlled Access

## Introduction

Controlled access is a technique used in computer networks to manage access to shared resources, such as networks, devices, or data. It ensures that only authorized users or devices can access the resources, preventing unauthorized access and potential security breaches. In this section, we will explore the concept of controlled access, its historical context, and modern developments.

## History

The concept of controlled access dates back to the 1970s, when the first computer networks were developed. Initially, networks were open and uncontrolled, allowing anyone to access them. However, as networks grew in size and complexity, the need for controlled access became apparent.

One of the earliest examples of controlled access was the development of the Ethernet network in the late 1970s. The Ethernet network was designed to provide a shared medium for devices to communicate with each other. However, it lacked a mechanism to control access to the network, making it vulnerable to hacking and unauthorized access.

In the 1980s, the development of the Token Ring network introduced a token-based access control mechanism. The token was a logical unit of access that was passed from device to device, allowing only the device holding the token to access the network.

## Modern Developments

Fast forward to the present day, controlled access has become an essential component of computer networks. The proliferation of wireless networks, such as Wi-Fi and Bluetooth, has made it increasingly important to control access to these networks.

There are several modern techniques used to control access to computer networks, including:

- **Access Control Lists (ACLs)**: ACLs are a list of rules that define which devices or users are allowed to access a network or device.
- **Port Control Protocol (PCP)**: PCP is a protocol used to control access to networks based on the type of traffic being sent.
- **Wi-Fi Protected Access (WPA)**: WPA is a security protocol used to control access to Wi-Fi networks.
- **Virtual Private Networks (VPNs)**: VPNs are networks that use encryption and other security measures to control access to remote networks.

## Techniques

There are several techniques used to control access to computer networks, including:

- **Single Sign-On (SSO)**: SSO is a technique used to authenticate users before allowing access to a network or device.
- **Two-Factor Authentication (2FA)**: 2FA is a technique used to authenticate users using two factors, such as a username and password, and a physical token or biometric scan.
- **Encryption**: Encryption is a technique used to scramble data, making it unreadable to unauthorized users.
- **Access Control Lists (ACLs)**: ACLs are a list of rules that define which devices or users are allowed to access a network or device.

## Applications

Controlled access has numerous applications in computer networks, including:

- **Enterprise Networks**: Controlled access is essential in enterprise networks, where sensitive data and resources are stored.
- **Wireless Networks**: Controlled access is critical in wireless networks, where unauthorized access can compromise security.
- **Cloud Computing**: Controlled access is necessary in cloud computing environments, where sensitive data is stored and processed remotely.
- **IoT Devices**: Controlled access is essential in IoT devices, where unauthorized access can compromise security and data integrity.

## Case Studies

- **AT&T's Wi-Fi Network**: AT&T's Wi-Fi network uses a combination of PCP and WPA to control access to its wireless network.
- **Google's Cloud Network**: Google's cloud network uses a combination of ACLs and encryption to control access to its cloud infrastructure.
- **Verizon's Enterprise Network**: Verizon's enterprise network uses a combination of SSO and 2FA to control access to its network.

## Controlled Access in Action

Here is an example of controlled access in action:

### Example Scenario

A company has a network that contains sensitive data, including employee information and financial records. The company wants to control access to this network, ensuring that only authorized employees can access the data.

The company implements a combination of ACLs and encryption to control access to the network. The ACLs define which employees are allowed to access the network, and the encryption scrambles the data, making it unreadable to unauthorized users.

When an employee tries to access the network, they are prompted to authenticate using their username and password. If the authentication is successful, the employee is granted access to the network, and the data is decrypted for display.
