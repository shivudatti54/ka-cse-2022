# **And Find the Number of Packets Dropped**

### Definition

- The number of packets dropped refers to the number of packets that are not delivered to their intended destination due to various network errors or congestion.

### Concepts

- **Packet Loss**: The phenomenon where packets are lost or dropped during transmission.
- **Error Rate**: The probability of packet loss or corruption.
- **Packet Dropped Rate**: The number of packets dropped per unit time.

### Formulas

- **Packet Loss Rate (PLR)**: PLR = (Number of packets dropped / Total number of packets transmitted) x 100
- **Error Rate (ER)**: ER = (Number of errors / Total number of packets transmitted) x 100

### Important Theorems

- **Theorems of packet loss**:
  - Theorems of packet loss show that the probability of packet loss is directly proportional to the packet loss rate.

### Key Points

- Factors affecting packet loss:
  - Network congestion
  - Error-prone areas (e.g., wireless links)
  - packet size and complexity
- Types of packet loss:
  - Physical layer packet loss
  - Data link layer packet loss
- Mitigation techniques:
  - Forward error correction (FEC)
  - Automatic repeat request (ARQ)
  - Equalization and noise reduction

### Important Terms

- **Interframe Gap (IFG)**: The gap between frames transmitted by a station in the same data link.
- **Collision Detection (CD)**: The technique used to detect collisions at the data link layer.
- **Collision Resolution (CR)**: The technique used to resolve collisions at the data link layer.

### Notes

- Packet loss can be analyzed using tools such as packet sniffers and network analyzers.
- Understanding packet loss is crucial for designing efficient network protocols and optimizing network performance.
