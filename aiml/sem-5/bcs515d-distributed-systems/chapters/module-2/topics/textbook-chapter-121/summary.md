# **Distributed Systems: Chapter 12.1 Revision Notes**

**Important Definitions and Formulas:**

- **Distributed System:** A collection of interconnected computers that appear as a single system to the user.
- **File System:** A set of programs and procedures for managing files on a computer.
- **File Service:** A distributed system that provides file sharing and access across multiple computers.
- **File Service Architecture:** A framework for designing and implementing a file service.

**Key Components of a File Service Architecture:**

- **Client:** Application or user that requests file access.
- **Server:** Computer that stores and manages files.
- **File System:** Stores and organizes files.
- **Network:** Interconnects clients and servers.

**Key Concepts:**

- **Distributed File System (DFS):** A file system that spans multiple computers.
- **File Sharing:** The ability to access and share files across multiple computers.
- **File Replication:** The process of duplicating files on multiple servers.
- **File Fragmentation:** The process of dividing large files into smaller chunks.

**Theorems:**

- **Nash Equilibrium:** A concept in game theory that describes a stable state where no single player can improve their payoff by unilaterally changing their strategy.
- **File System Consistency:** The property of a file system that ensures data consistency across all servers.

**Key Technologies:**

- **Coda:** A file system that uses a client-server architecture to provide file sharing and access.
- **NFS (Network File System):** A protocol that allows clients to access files on servers over a network.
- **Ceph:** A distributed file system that uses a peer-to-peer architecture to provide file sharing and access.
