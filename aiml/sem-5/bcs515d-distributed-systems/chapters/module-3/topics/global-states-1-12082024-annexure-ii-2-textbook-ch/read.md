# **Global States**

## **Introduction**

In distributed systems, a global state is a shared state that is accessible to all processes or threads in the system. It represents the current status of the system, including the state of each resource, the status of each process, and any other relevant information.

## **Definition**

A global state is defined as the set of all variables that are shared among all processes in a distributed system. It is a snapshot of the system's current state, taken at a particular point in time.

## **Types of Global States**

There are two types of global states:

- **Local State**: A local state is the state of a process or thread, which is unique to that process.
- **Global State**: A global state is the shared state of all processes in a distributed system.

## **Characteristics of Global States**

Here are some key characteristics of global states:

- **Immutable**: Global states are immutable, meaning that once they are set, they cannot be changed.
- **Uniform**: Global states are uniform, meaning that the state of every process in the system is the same.
- **Accessible**: Global states are accessible to all processes in the system.

## **Example**

Suppose we have a distributed system with three processes, P1, P2, and P3. Each process has a shared variable `x` that represents the current status of the system. The initial value of `x` is 0. After some time, P1 sets `x` to 1, P2 sets `x` to 2, and P3 sets `x` to 3. The global state of the system is now:

- `x` = 3

## **Clocks and Global States**

A clock is a mechanism that generates a sequence of timestamps, which are used to synchronize processes in a distributed system. Clocks are used to determine the order in which processes execute and to ensure that processes access shared resources in a consistent manner.

Here are some key concepts related to clocks and global states:

- **Timestamps**: Timestamps are the timestamps generated by a clock.
- **Clock Order**: Clock order is the order in which processes execute, determined by their timestamps.
- **Consistency**: Consistency is ensured by using timestamps to synchronize processes and ensure that they access shared resources in a consistent manner.

## **Events and Process States**

Events are the actions that occur in a distributed system, such as process creation, process termination, and resource updates. Process states are the states of processes in a distributed system, which can be in one of several states, such as running, waiting, or terminated.

Here are some key concepts related to events and process states:

- **Process States**: Process states are the states of processes in a distributed system.
- **Event-Process States**: Event-process states are the states of processes triggered by events.
- **Event-Process State Transition**: Event-process state transition is the transition of a process from one state to another based on an event.
