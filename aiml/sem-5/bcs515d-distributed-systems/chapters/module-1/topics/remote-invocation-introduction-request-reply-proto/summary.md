# REMOTE INVOCATION

### Introduction

- Remote invocation is a key concept in distributed systems, enabling communication between processes running on different machines.
- It allows one process to invoke methods or procedures on another process, located in a different physical location.

### Request-Reply Protocols

- Request-reply protocols involve a client process sending a request to a server process, which then responds with the result.
- Examples include:
  - RPC (Remote Procedure Call)
  - Remote Method Invocation (RMI)
  - Distributed Transaction Protocol (DTP)

### Remote Procedure Call (RPC)

- RPC is a request-reply protocol that enables a client process to invoke procedures on a server process.
- Characteristics:
  - Client-server architecture
  - Request-response mechanism
  - Authentication and authorization mechanisms

### Remote Method Invocation (RMI)

- RMI is a programming technique that allows an object to invoke methods on another object, located in a different physical location.
- Characteristics:
  - Object-oriented approach
  - Uses RMI protocol to invoke methods
  - Enables communication between objects in different processes or machines

### Definitions and Formulas

- **Remote Method Invocation (RMI)**: A programming technique that allows an object to invoke methods on another object, located in a different physical location.
- **RPC (Remote Procedure Call)**: A request-reply protocol that enables a client process to invoke procedures on a server process.
- **Client**: The process that sends requests to the server.
- **Server**: The process that receives requests and responds with results.
