# **Revision Notes for Chapter 1.1: Introduction to Distributed Systems**

### Definitions and Key Concepts

- **Distributed System**: A collection of interconnected computers that appear to be a single system to the user.
- **Client-Server Architecture**: A model where a client requests services from a server.
- **Network**: A collection of interconnected computers that can communicate with each other.

### Key Characteristics of Distributed Systems

- **Autonomy**: Each node in the system operates independently.
- **Organizational Structure**: A distributed system can be organized in various ways, such as a hierarchical or flat structure.
- **Communication**: Distributed systems communicate through a network.
- **Fault Tolerance**: Distributed systems can continue to function even if one or more nodes fail.

### Important Formulas and Theorems

- **P = NP** (Polytime vs. NP-Complete): A theoretical result that shows the complexity of problems in distributed systems.
- **Amdahl's Law**: A theoretical result that shows the impact of parallelism on the performance of a system.
- **Gaussian Distribution**: A mathematical distribution used to model the behavior of packets in a network.

### Important Theorems

- **Dijkstra's Algorithm**: An algorithm for finding the shortest path in a graph.
- **Bellman-Ford Algorithm**: An algorithm for finding the shortest path in a graph with negative weight edges.

### Key Objectives

- Understand the characteristics of distributed systems.
- Understand the importance of communication in distributed systems.
- Understand the impact of fault tolerance on distributed systems.
