# Deriving Functional and Nonfunctional Requirements

### Introduction

In software engineering, requirements are the specifications that define what the software should do and how it should behave. Deriving requirements is a critical step in the software development process, as it ensures that the software meets the needs and expectations of its users. In this study material, we will explore the concepts of functional and nonfunctional requirements, and provide a case study to derive both types of requirements.

### Functional Requirements

Functional requirements describe what the software should do, including its features, behaviors, and interactions. They are typically described in natural language and are used to define the software's functional specifications.

#### Definition:

Functional requirements are the specifications that define what the software should do, including its features, behaviors, and interactions.

#### Types of Functional Requirements:

- **Primary functional requirements**: These are the key features and behaviors that the software should have.
- **Secondary functional requirements**: These are the supporting features and behaviors that enable the primary functional requirements.
- **Enhancement functional requirements**: These are the features and behaviors that can be added to the software after its initial release.

#### Examples of Functional Requirements:

- The software should allow users to create and manage their own profiles.
- The software should provide a search functionality to find specific products.
- The software should allow users to compare prices and features of different products.

#### Deriving Functional Requirements:

To derive functional requirements, you can follow these steps:

1. **Conduct stakeholder interviews**: Talk to the users, customers, and stakeholders to understand their needs and expectations.
2. **Analyze the problem statement**: Break down the problem statement into its key components and identify the functional requirements.
3. **Use functional modeling techniques**: Use techniques such as use cases, use scenarios, and activity diagrams to model the software's behavior.

### Nonfunctional Requirements

Nonfunctional requirements describe the constraints and limitations of the software, including its performance, security, and usability.

#### Definition:

Nonfunctional requirements are the constraints and limitations of the software, including its performance, security, and usability.

#### Types of Nonfunctional Requirements:

- **Performance requirements**: These define the software's response time, throughput, and scalability.
- **Security requirements**: These define the software's ability to protect user data and prevent unauthorized access.
- **Usability requirements**: These define the software's ease of use, navigation, and user experience.

#### Examples of Nonfunctional Requirements:

- The software should respond within 2 seconds to a user's login request.
- The software should encrypt all user data using AES-256 encryption.
- The software should have a simple and intuitive interface that is accessible to users with disabilities.

#### Deriving Nonfunctional Requirements:

To derive nonfunctional requirements, you can follow these steps:

1. **Conduct performance testing**: Test the software's performance under various loads and scenarios to define its performance requirements.
2. **Analyze security threats**: Identify potential security threats and define the software's security requirements.
3. **Use usability testing**: Test the software's usability and define its usability requirements.

Deriving functional and nonfunctional requirements is a critical step in the software development process.
