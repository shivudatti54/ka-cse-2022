# Derive Both Functional and Nonfunctional Requirements from the Case Study

### Introduction

- The case study provides a real-world scenario for deriving functional and nonfunctional requirements.

### Functional Requirements

- **Definition:** Requirements that describe what the software should do.
- **Types:**
  - **Functional Requirements:** Describe the functionality of the software.
  - **Performance Requirements:** Describe the responsiveness and speed of the software.
  - **Security Requirements:** Describe the measures to protect the software from unauthorized access.
  - **Usability Requirements:** Describe the ease of use and user experience of the software.
- **Example:**
  - The case study requires a software to process orders for an e-commerce platform.
  - Functional requirements:
    - The software should be able to process orders in real-time.
    - The software should be able to handle multiple orders simultaneously.
    - The software should display order status in real-time.

### Nonfunctional Requirements

- **Definition:** Requirements that describe the characteristics of the software that are not directly related to its functionality.
- **Types:**
  - **Performance Requirements:** Describe the responsiveness and speed of the software.
  - **Security Requirements:** Describe the measures to protect the software from unauthorized access.
  - **Usability Requirements:** Describe the ease of use and user experience of the software.
  - **Scalability Requirements:** Describe the ability of the software to handle increased load.
- **Example:**
  - The case study requires a software to process orders for an e-commerce platform.
  - Nonfunctional requirements:
    - Performance requirement: The software should process orders within 2 seconds.
    - Security requirement: The software should use HTTPS to encrypt data.
    - Usability requirement: The software should be accessible on all devices.

### Important Formulas and Definitions

- **Definition of Functional Requirement:** A functional requirement is a requirement that describes the functionality of the software.
- **Definition of Nonfunctional Requirement:** A nonfunctional requirement is a requirement that describes the characteristics of the software that are not directly related to its functionality.
- **Formula for Performance Requirements:** Performance = Response Time + Throughput

### Key Theorems

- **Theorem of Functional Requirements:** A software that meets its functional requirements is more likely to meet its nonfunctional requirements.
- **Theorem of Nonfunctional Requirements:** A software that meets its nonfunctional requirements is more likely to meet its functional requirements.
