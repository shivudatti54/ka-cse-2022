# Agile Process Summary

### What is an Agile Process?

- Defined as an iterative and adaptive approach to managing projects and software development.
- Emphasizes flexibility, collaboration, and rapid delivery.
- Key characteristics:
  - Incremental development
  - Continuous improvement
  - Flexibility and adaptability
  - Customer involvement

### Extreme Programming (XP)

- Agile software development methodology
- Focuses on delivering high-quality software in short iterations
- Key principles:
  - Pair programming
  - Continuous integration
  - Test-driven development (TDD)
  - Refactoring
  - Collective code ownership
  - Customer involvement

### Other Agile Process Models

- Scrum: framework for managing and completing complex projects using iterative and incremental practices.
- Kanban: visual system for managing work, emphasizing continuous flow and limiting work in progress.
- Lean software development: methodology for improving the flow of value to customers by eliminating waste and maximizing value.

### Tool Set for Agile Process

- Version control systems: Git, SVN
- Issue tracking systems: JIRA, Trello
- Project management tools: Asana, Microsoft Teams
- Agile planning and estimation tools: Kanban, Scrum
- Continuous integration and deployment tools: Jenkins, Docker

## Key Formulas and Definitions

- **Agile Manifesto**: Four core values:
  1.  Individuals and interactions
  2.  Working software
  3.  Customer collaboration
  4.  Responding to change
- **Scrum Framework**: Three roles:
  1.  Product Owner
  2.  Scrum Master
  3.  Development Team
- **Kanban**: Three pillars:
  1.  Visualize work
  2.  Limit work in progress
  3.  Flow
