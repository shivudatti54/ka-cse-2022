# Textbook 1: Chapter 3: 3.1 to 3.6 Revision Notes

### 3.1: Introduction to Project Management

- Project management is the application of knowledge, skills, and methods to execute projects effectively.
- Project management is a discipline that uses processes, people, and technology to manage projects.
- Key aspects of project management: scope, time, cost, quality, resources, and stakeholders.

### 3.2: Project Management Frameworks

- Five traditional project management frameworks:
  - Waterfall
  - Agile
  - Hybrid
  - V-Model
  - Prototyping
- Key differences between frameworks: iterative vs. sequential, flexibility, and risk management.

### 3.3: Project Life Cycle

- Project life cycle phases:
  - Initiation
  - Planning
  - Execution
  - Monitoring and Control
  - Closure
- Each phase has specific tasks, deliverables, and outcomes.

### 3.4: Project Management Processes

- Nine Knowledge Areas of the Project Management Body of Knowledge (PMBOK):
  - Integration Management
  - Scope Management
  - Time Management
  - Cost Management
  - Quality Management
  - Resource Management
  - Communications Management
  - Risk Management
  - Procurement Management
- Each process group has specific tasks, deliverables, and outcomes.

### 3.5: Project Management Tools and Techniques

- Types of tools:
  - Technological tools (e.g., project management software)
  - Business tools (e.g., Gantt charts, resource allocation matrices)
  - Communication tools (e.g., meetings, emails, reports)
- Types of techniques:
  - Planning techniques (e.g., critical path method, program evaluation and review technique)
  - Control techniques (e.g., earned value management, budgeting and cost control)

### 3.6: Project Management Best Practices

- Key best practices:
  - Define project scope clearly
  - Set realistic project goals and objectives
  - Use project management frameworks and tools
  - Implement risk management and quality assurance
  - Foster effective communication among stakeholders
- Importance of continuous improvement and learning in project management.

## Important Formulas and Definitions:

- **Project Management Maturity Model (PMM)**: a framework for assessing and improving project management capabilities.
- **Critical Path Method (CPM)**: a planning technique that identifies the critical tasks in a project and estimates the minimum time required to complete them.
- **Earned Value Management (EVM)**: a technique that measures project performance by comparing actual earned value to planned value.
