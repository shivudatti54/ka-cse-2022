# **Principles that Guide Each Framework Activity**

### Overview

This module covers the key principles that guide each framework activity in software engineering and project management. The following are the main points:

### Framework Activities

- **Requirements Management**

* Define and collect requirements
* Analyze and prioritize requirements
* Validate and verify requirements

- **Design and Planning**

* Identify project scope and goals
* Develop a project schedule and budget
* Define project resources and stakeholders

- **Development and Implementation**

* Develop a software product or system
* Implement quality assurance and testing
* Manage project risks and issues

- **Monitoring and Control**

* Track project progress and performance
* Identify and mitigate project risks
* Take corrective action when necessary

- **Closure and Review**

* Formalize project closure and acceptance
* Evaluate project success and lessons learned
* Document project experience and best practices

### Important Formulas and Definitions

- **Kerwin's Law**: The time spent performing a task is directly proportional to the complexity of the task and the time spent planning the task.
- **Pareto Principle**: 80% of project failures are caused by 20% of project risks.
- **Definition of Done**: A set of criteria used to determine when a software product or system is complete.

### Theorems and Concepts

- **Project Management Body of Knowledge (PMBOK)**: A framework for project management that includes best practices and guidelines for project planning, execution, and monitoring and control.
- **Agile Methodologies**: An approach to project management that emphasizes flexibility, collaboration, and rapid delivery.
- **Waterfall Model**: A linear approach to project management that involves phase-by-phase development and testing.
