### Software and Software Engineering: Revision Notes

#### Module: Software Engineering & Project Management

#### 10 hours

**The Nature of Software**

- Definition: A set of instructions or programs that tell a computer what to do.
- Characteristics:
  - Intangible
  - Programmable
  - Modifiable
  - Duplicable
  - Reusable

**The Unique Nature of Web Apps**

- Definition: Software applications that run on the web, using web technologies such as HTML, CSS, and JavaScript.
- Characteristics:
  - Client-server architecture
  - Browser-based
  - Dynamic content
  - Collaborative

**Software Engineering**

- Definition: The application of engineering principles and techniques to the development of software.
- Key principles:
  - Abstraction
  - Modularity
  - Reusability
  - Testability
  - Maintainability

**The Software Process**

- Definition: A series of stages and activities used to develop software.
- Stages:
  - Requirements gathering
  - Analysis
  - Design
  - Implementation
  - Testing
  - Deployment
  - Maintenance

**Software Engineer**

- Definition: A professional who designs, develops, tests, and maintains software systems.
- Key skills:
  - Programming languages
  - Software development methodologies
  - System analysis and design
  - Testing and quality assurance
  - Project management

### Important Formulas, Definitions, and Theorems

- **Software Development Life Cycle (SDLC)**: A framework for developing software, consisting of phases and activities.
- **Waterfall Model**: A linear approach to software development, where each phase is completed before moving to the next one.
- **Agile Model**: An iterative and incremental approach to software development, emphasizing flexibility and collaboration.
- **Defect Density**: A measure of the number of defects per unit of code.
- **Kanban**: A visual system for managing work, emphasizing continuous flow and limiting work in progress.
