# **Chapter 2: 2.1 to 2.5 Revision Notes**

## **2.1 Requirements Elicitation**

- Requirements elicitation: The process of gathering and documenting requirements from stakeholders
- Types of requirements: functional, performance, security, and usability
- Techniques: interviews, surveys, focus groups, use cases

## **2.2 Use Cases**

- Use cases: a way to describe the interactions between a system and its users
- Format: actor, goal, preconditions, postconditions, and extensions
- Example: User wants to create an account

## **2.3 Use Case Diagrams**

- Use case diagrams: a graphical representation of use cases
- Components: actors, use cases, and generalization
- Example: User interface with login and registration use cases

## **2.4 Classes and Objects**

- Classes and objects: fundamental concepts in object-oriented programming
- Definition: a class is a blueprint for creating objects
- Example: A `Vehicle` class with attributes `color` and `model`

## **2.5 Object-Oriented Analysis (OOA)**

- OOA: a method for analyzing and designing object-oriented systems
- Steps: identify classes, attributes, and operations
- Example: Analyzing a `BankingSystem` using OOA
