# **Chapter 7: Requirements Management (7.1-7.3)**

## **7.1: Introduction to Requirements Management**

### Overview

Requirements management is an essential process in software engineering that involves collecting, documenting, analyzing, and maintaining requirements for a software project. It ensures that the software meets the needs and expectations of the stakeholders.

### Why Requirements Management is Important

- Ensures that the software meets the stakeholder's needs and expectations
- Helps to identify and mitigate risks
- Facilitates communication among stakeholders
- Ensures that the software is developed to meet the required specifications

### Definition

Requirements management is the process of collecting, documenting, analyzing, and maintaining requirements for a software project.

### Types of Requirements

- Functional requirements: describe what the software should do
- Non-functional requirements: describe how the software should behave (e.g., performance, security)
- Performance requirements: describe how the software should perform (e.g., response time, throughput)
- Security requirements: describe how the software should protect data and prevent unauthorized access

### Examples of Requirements Management

- Collecting user stories and acceptance criteria
- Creating use cases and use scenario diagrams
- Developing a requirements specification document

## **7.2: Requirements Analysis and Specification**

### Overview

Requirements analysis and specification is the process of analyzing and documenting the requirements of a software project.

### Steps in Requirements Analysis and Specification

- Identify and document requirements
- Analyze requirements for completeness and accuracy
- Develop a requirements specification document
- Review and revise the requirements specification document

### Techniques for Requirements Analysis and Specification

- Interviews with stakeholders
- User surveys and questionnaires
- Observations of users interacting with the software
- Review of existing documentation and data
- Use of tools such as use case diagrams and entity-relationship diagrams

### Examples of Requirements Analysis and Specification

- Developing a requirements specification document for a new e-commerce website
- Creating a use case diagram for a new banking system
- Conducting user interviews for a new mobile app

## **7.3: Requirements Management Processes**

### Overview

Requirements management processes are the activities involved in managing requirements throughout the software development life cycle.

### Steps in Requirements Management Processes

- Identify and document requirements
- Analyze requirements for completeness and accuracy
- Develop a requirements specification document
- Review and revise the requirements specification document
- Implement requirements in software development

### Techniques for Requirements Management Processes

- Agile methodologies such as Scrum and Kanban
- Waterfall methodologies
- Use of tools such as version control systems and requirements management tools

### Examples of Requirements Management Processes

- Conducting regular sprint reviews for an agile development team
- Developing a requirements management plan for a waterfall development project
- Using a requirements management tool to track and manage requirements throughout the development life cycle
