# Software Engineering & Project Management

### 7.1: Requirements Engineering

- Definition: The process of defining, analyzing, documenting, and maintaining all the requirements of a software system.
- Key activities:
  - Requirements gathering
  - Requirements analysis
  - Requirements definition
  - Requirements validation
- Important formulas:
  - Requirements engineering life cycle (RELC)
- Important definitions:
  - Functional requirements
  - Non-functional requirements
- Important theorems:
  - None

### 7.2: Software Design

- Definition: The process of creating a detailed plan for the software system, including its architecture, components, and interfaces.
- Key activities:
  - Requirements traceability
  - Design patterns
  - Object-Oriented Design (OOD)
- Important formulas:
  - Object-oriented design principles
- Important definitions:
  - Architecture
  - Component-based design
- Important theorems:
  - None

### 7.3: Software Construction

- Definition: The process of building the software system, including coding, testing, and deployment.
- Key activities:
  - Development life cycle
  - Coding standards
  - Testing strategies
- Important formulas:
  - Development life cycle models (e.g., Waterfall, Agile)
- Important definitions:
  - Coding standards
  - Testing methodologies
- Important theorems:
  - None
