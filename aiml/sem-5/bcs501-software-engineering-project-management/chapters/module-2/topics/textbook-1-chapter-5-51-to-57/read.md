# **Software Engineering & Project Management Study Material: Chapter 5**

## **5.1: Requirements Engineering**

### Definition

Requirements engineering is the process of identifying, analyzing, and documenting the requirements of a software development project. It involves understanding the needs and expectations of the stakeholders and translating them into a set of requirements that can be used to guide the development process.

### Steps in Requirements Engineering

- Identify stakeholders and their interests
- Gather requirements through interviews, surveys, and reviews
- Analyze and prioritize requirements
- Document requirements in a clear and unambiguous manner
- Validate requirements with stakeholders

### Example

A company wants to develop an e-commerce platform. The requirements engineer needs to gather information from stakeholders, including the business analyst, users, and technical leads. The requirements engineer may ask questions such as:

- What features do you need for payment processing?
- How will users be able to create accounts?
- What are the technical requirements for integrating with third-party payment gateways?

### Key Concepts:

- Stakeholders: individuals or groups who have an interest in the project
- Requirements: specific needs or wants that must be met
- Documentation: written descriptions of requirements
- Validation: verifying that requirements are understood and met

## **5.2: Software Requirements Specification (SRS)**

### Definition

A Software Requirements Specification (SRS) is a document that outlines the requirements for a software system. It provides a detailed description of the software's functionality, performance, and other characteristics.

### Structure of an SRS

- Introduction
- System Overview
- Functional Requirements
- Performance Requirements
- Environmental Requirements
- Security Requirements
- Non-Functional Requirements
- Appendices

### Example

A company wants to develop a mobile app for tracking personal expenses. The SRS for the app might include:

- Introduction: overview of the app's purpose and functionality
- System Overview: description of the app's architecture and components
- Functional Requirements:
  - User registration and login
  - Expense tracking and categorization
  - Budgeting and financial reporting
- Performance Requirements:
  - Response time for user interactions
  - Data storage and retrieval capabilities
- Environmental Requirements:
  - Compatibility with iOS and Android devices
  - Requirements for network connectivity

### Key Concepts:

- SRS: a document that outlines software requirements
- Functional requirements: specific needs or wants for the software
- Non-functional requirements: characteristics of the software that are not directly related to its functionality

## **5.3: Software Design Patterns**

### Definition

A software design pattern is a reusable solution to a common problem in software design. It provides a proven approach to solving a particular design problem, and can be applied to a wide range of software systems.

### Types of Design Patterns

- Creational patterns: provide a way to create objects
- Structural patterns: describe the structure of an object or system
- Behavioral patterns: describe the behavior of an object or system

### Example

A company wants to develop an e-commerce platform that allows users to create and manage their own online stores. A design pattern that might be used is the "Factory Pattern", which provides a way to create objects without specifying the exact class of object that will be created.

### Key Concepts:

- Design pattern: a reusable solution to a common problem in software design
- Creational pattern: a pattern that provides a way to create objects
- Structural pattern: a pattern that describes the structure of an object or system

## **5.4: Object-Oriented Analysis and Design (OOAD)**

### Definition

Object-Oriented Analysis and Design (OOAD) is a software design approach that uses objects and classes to model real-world systems. It provides a way to break down complex systems into smaller, more manageable components.

### Steps in OOAD

- Identify the problem domain
- Define the classes and objects
- Create a class diagram
- Define the relationships between classes
- Create an object diagram

### Example

A company wants to develop a software system for managing inventory. Using OOAD, the developer might:

- Identify the problem domain: inventory management
- Define the classes and objects: inventory item, warehouse, and order
- Create a class diagram: showing the relationships between classes
- Define the relationships between classes: a warehouse can hold many inventory items, and an inventory item can be part of many orders

### Key Concepts:

- OOAD: a software design approach that uses objects and classes
- Classes: abstract representations of objects
- Objects: instances of classes
- Relationships: connections between classes

## **5.5: Design Principles**

### Definition

Design principles are guidelines that provide a framework for designing software systems. They provide a way to ensure that software systems are designed with consideration for the needs and goals of users.

### Types of Design Principles

- Separation of Concerns (SoC): separates different concerns or functions into separate modules
- Don't Repeat Yourself (DRY): avoids duplicating code or functionality
- KISS (Keep it Simple, Stupid): favors simplicity over complexity

### Example

A company wants to develop a software system for managing customer relationships. A design principle that might be used is the "Separation of Concerns" principle, which might lead the developer to:

- Design a separate module for managing customer data
- Design a separate module for managing customer interactions
- Design a separate module for managing customer feedback

### Key Concepts:

- Design principle: a guideline for designing software systems
- Separation of Concerns: separates different concerns or functions into separate modules
- Don't Repeat Yourself: avoids duplicating code or functionality

## **5.6: Software Engineering Environments**

### Definition

A software engineering environment is a set of tools and technologies that support the development, testing, and maintenance of software systems. It includes programming languages, compilers, debuggers, and other software development tools.

### Types of Software Engineering Environments

- Integrated Development Environment (IDE): provides a comprehensive set of tools for software development
- Development Environment: provides a set of tools for software development and testing
- Software Configuration Management (SCM) system: manages changes to software code and documentation

### Example

A company wants to develop a software system for managing inventory. The software engineering environment used might include:

- A programming language such as Java or C++
- A compiler or interpreter for the language
- A debugger for testing and debugging the code
- A SCM system for managing changes to the code

### Key Concepts:

- Software engineering environment: a set of tools and technologies for software development
- Integrated Development Environment: provides a comprehensive set of tools for software development
- Development Environment: provides a set of tools for software development and testing

## **5.7: Project Management**

### Definition

Project management is the process of planning, organizing, and controlling resources to achieve specific goals and objectives. It involves coordinating and directing the efforts of team members to deliver a project on time, within budget, and to the required quality standards.

### Steps in Project Management

- Define project scope and objectives
- Develop a project plan and schedule
- Identify and allocate resources
- Monitor and control project progress
- Evaluate project success

### Example

A company wants to develop a software system for managing customer relationships. The project manager might:

- Define the project scope and objectives: develop a software system for managing customer relationships
- Develop a project plan and schedule: create a timeline and milestones for the project
- Identify and allocate resources: assign team members and resources to tasks
- Monitor and control project progress: track progress and make adjustments as needed
- Evaluate project success: evaluate the final product and assess its success

### Key Concepts:

- Project management: the process of planning, organizing, and controlling resources to achieve specific goals and objectives
- Project scope: the defining characteristics of a project
- Project plan: a detailed plan for completing a project
- Resource allocation: assigning team members and resources to tasks
