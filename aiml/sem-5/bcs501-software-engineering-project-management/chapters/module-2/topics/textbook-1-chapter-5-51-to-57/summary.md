# Textbook 1: Chapter 5: 5.1 to 5.7

## Software Engineering & Project Management

### Key Points

#### 5.1: Software Engineering Process

- The software engineering process involves:
  - Requirements definition
  - Analysis
  - Design
  - Implementation
  - Testing
  - Maintenance
- The Agile and V-Model approaches to software engineering process

#### 5.2: Software Engineering Methodologies

- Agile Methodology:
  - Emphasizes flexibility and rapid delivery
  - Iterative and incremental development
  - Focus on customer collaboration
- Waterfall Methodology:
  - Linear and sequential development
  - Top-down approach
  - Emphasizes reliability and maintainability

#### 5.3: Project Management

- Project Management Principles:
  - Define project scope
  - Establish project schedule
  - Determine project budget
  - Identify project risks
  - Develop project communication plan
- Project Management Life Cycle:
  - Initiation
  - Planning
  - Execution
  - Monitoring and Control
  - Closure

#### 5.4: Project Management Techniques

- Planning Techniques:
  - Critical Path Method (CPM)
  - Program Evaluation and Review Technique (PERT)
- Scheduling Techniques:
  - Gantt Chart
  - Bar Chart
- Cost Estimation Techniques:
  - Bottom-up Estimation
  - Top-down Estimation

#### 5.5: Project Scheduling

- Project Schedule:
  - Tasks and dependencies
  - Start and finish dates
  - Duration and slack
- Resource Allocation:
  - Resource type (human, material, equipment)
  - Resource utilization

#### 5.6: Cost Management

- Cost Estimation:
  - Direct Costs
  - Indirect Costs
  - Total Cost
- Cost Control:
  - Budgeting
  - Cost Accounting
  - Variance Analysis

#### 5.7: Project Monitoring and Control

- Project Monitoring:
  - Performance measurement
  - Progress tracking
  - Issue identification
- Project Control:
  - Decision-making
  - Corrective action
  - Preventive action

### Important Formulas and Definitions

#### 5.1

- **Requirements Definition**: Definition of the software requirements specification (SRS)
- **Analysis Phase**: Identification of the problem statement, requirements, and constraints

#### 5.3

- **Project Scope**: The set of activities that need to be performed to meet the project objectives
- **Project Schedule**: A plan for the sequence of tasks and activities to be performed during the project

#### 5.4

- **Critical Path Method (CPM)**: A method for determining the critical path in a project
- **Program Evaluation and Review Technique (PERT)**: A method for estimating the time and cost required to complete a project

#### 5.5

- **Gantt Chart**: A type of bar chart used to display a project schedule
- **Bar Chart**: A chart used to display the progress of a project

#### 5.6

- **Direct Costs**: Costs associated with the direct activities of a project
- **Indirect Costs**: Costs associated with the indirect activities of a project

#### 5.7

- **Performance Measurement**: The process of measuring the performance of a project
- **Progress Tracking**: The process of monitoring the progress of a project
