# **Chapter 6: 6.1 to 6.5 - Quick Revision Notes**

### 6.1: Requirements Definition

- Requirements definition: The process of identifying, analyzing, and documenting the functional and non-functional needs of a project
- Key activities:
  - Elicitation: Gathering requirements from stakeholders
  - Analysis: Identifying and prioritizing requirements
  - Definition: Documenting requirements in a clear and unambiguous manner
- Important concepts:
  - Functional requirements: Describing what the system should do
  - Non-functional requirements: Describing how the system should behave

### 6.2: Requirements Analysis

- Requirements analysis: The process of examining and evaluating the requirements to ensure they are feasible and meet stakeholder needs
- Key activities:
  - Review: Checking requirements for completeness and accuracy
  - Validation: Verifying that the requirements meet stakeholder needs
  - Verification: Ensuring that the requirements are feasible and meet technical constraints
- Important concepts:
  - Requirements decomposition: Breaking down complex requirements into smaller, more manageable parts
  - Use case analysis: Identifying and documenting the interactions between users and the system

### 6.3: Requirements Specification

- Requirements specification: The document that formally defines the requirements for a project
- Key activities:
  - Writing: Creating a clear and concise document that describes the requirements
  - Review: Ensuring that the document is accurate and complete
  - Approval: Obtaining approval from stakeholders
- Important concepts:
  - Requirements specification language (RSL): A standard language used to describe requirements
  - Relational database management system (RDBMS): A type of database that uses tables and relationships to store data

### 6.4: Requirements Management

- Requirements management: The process of controlling and maintaining the requirements throughout a project
- Key activities:
  - Requirements management plan: Creating a plan that outlines the requirements management process
  - Requirements tracking: Monitoring and updating the requirements throughout the project
  - Requirements verification: Verifying that the requirements are met during testing and deployment
- Important concepts:
  - Requirements management life cycle (RMLC): A framework for managing requirements throughout a project
  - Requirements management best practices (RMBP): A set of guidelines for managing requirements effectively

### 6.5: Requirements Analysis Techniques

- Requirements analysis techniques: Methods used to analyze and evaluate requirements
- Key techniques:
  - Use case diagramming: Creating diagrams to represent the interactions between users and the system
  - Entity-relationship diagramming: Creating diagrams to represent the relationships between entities and attributes
  - Finite state machine (FSM) modeling: Creating models to represent the states and transitions of a system
- Important concepts:
  - Requirements analysis tools: Software tools used to support requirements analysis
  - Requirements analysis methodologies: Frameworks and approaches used to analyze and evaluate requirements
