# Project Management Life Cycle

## Project Life Cycle Overview

A project life cycle defines the phases from project start to completion.

### Generic Project Phases

```
Initiation → Planning → Execution → Monitoring & Control → Closure
```

## Phase Details

### Phase 1: Initiation

**Purpose:** Define and authorize the project

**Key Activities:**
- Develop project charter
- Identify stakeholders
- Conduct feasibility study
- Define high-level scope
- Assign project manager

**Deliverables:**
- Project Charter
- Stakeholder Register
- Feasibility Report

### Phase 2: Planning

**Purpose:** Establish project roadmap

**Key Activities:**
- Define scope in detail (WBS)
- Develop schedule
- Estimate costs and budget
- Plan resources
- Identify risks
- Plan quality
- Plan communications

**Deliverables:**
- Project Management Plan
- Schedule (Gantt chart)
- Budget
- Risk Register
- Communication Plan

### Phase 3: Execution

**Purpose:** Perform the work to achieve objectives

**Key Activities:**
- Direct and manage work
- Acquire and develop team
- Manage stakeholder expectations
- Implement changes
- Assure quality
- Conduct procurements

**Deliverables:**
- Work packages completed
- Status reports
- Change requests
- Team development

### Phase 4: Monitoring & Control

**Purpose:** Track, review, and regulate progress

**Key Activities:**
- Monitor project performance
- Control scope changes
- Control schedule
- Control costs
- Control quality
- Report performance

**Tools:**
- Earned Value Management (EVM)
- Variance analysis
- Trend analysis
- Status meetings

### Phase 5: Closure

**Purpose:** Formally complete the project

**Key Activities:**
- Verify deliverables accepted
- Document lessons learned
- Archive project documents
- Release resources
- Celebrate success

**Deliverables:**
- Final product/service
- Lessons learned report
- Project closure report
- Archived documentation

## Traditional vs Modern PM Practices

### Traditional (Predictive)

| Characteristic | Description |
|---------------|-------------|
| Planning | Detailed upfront |
| Requirements | Fixed at start |
| Change | Controlled strictly |
| Delivery | At project end |
| Documentation | Comprehensive |
| Teams | Specialized roles |

### Modern (Adaptive/Agile)

| Characteristic | Description |
|---------------|-------------|
| Planning | Iterative, just-in-time |
| Requirements | Evolving |
| Change | Embraced |
| Delivery | Incremental |
| Documentation | Just enough |
| Teams | Cross-functional |

### Hybrid Approaches

Combining traditional and agile:
- Plan-driven for overall project structure
- Agile for development iterations
- Choose based on project characteristics

## Project Management Frameworks

### PMBOK (Project Management Body of Knowledge)

**10 Knowledge Areas:**
1. Integration Management
2. Scope Management
3. Schedule Management
4. Cost Management
5. Quality Management
6. Resource Management
7. Communications Management
8. Risk Management
9. Procurement Management
10. Stakeholder Management

### PRINCE2 (Projects IN Controlled Environments)

**7 Principles:**
1. Continued business justification
2. Learn from experience
3. Defined roles and responsibilities
4. Manage by stages
5. Manage by exception
6. Focus on products
7. Tailor to suit environment

**7 Themes:**
- Business Case, Organization, Quality, Plans, Risk, Change, Progress

### Comparison

| Aspect | PMBOK | PRINCE2 |
|--------|-------|---------|
| Origin | USA (PMI) | UK (OGC) |
| Focus | Knowledge areas | Process-based |
| Certification | PMP | PRINCE2 |
| Approach | Guidelines | Method |

## Project Governance

### Governance Elements

1. **Decision-making authority** - Who can decide what
2. **Escalation procedures** - How issues are raised
3. **Review gates** - Checkpoints for approval
4. **Reporting requirements** - What information, when

### Stage Gates

```
Gate 0     Gate 1     Gate 2     Gate 3     Gate 4
   │          │          │          │          │
   ▼          ▼          ▼          ▼          ▼
[Idea] → [Planning] → [Design] → [Build] → [Launch]
```

At each gate:
- Review progress
- Assess viability
- Decide: Proceed/Modify/Stop

## Key Takeaways

1. Project lifecycle has **five phases**: Initiation, Planning, Execution, M&C, Closure
2. Each phase has **specific activities and deliverables**
3. **Traditional** PM is predictive; **Modern** is adaptive
4. **PMBOK** and **PRINCE2** are leading frameworks
5. **Stage gates** provide control checkpoints
