# Project Management Fundamentals

## What is a Project?

A project is a temporary endeavor undertaken to create a unique product, service, or result.

### Project Characteristics

- **Temporary** - Has defined start and end
- **Unique** - Creates something new
- **Progressive elaboration** - Details emerge over time
- **Resources constrained** - Limited budget, people, time

### Project vs Operations

| Project | Operations |
|---------|------------|
| Temporary | Ongoing |
| Unique | Repetitive |
| Creates change | Maintains status quo |
| Has end date | Continuous |

## Why Project Management?

### Importance of Project Management

- **Achieve objectives** within constraints
- **Coordinate resources** effectively
- **Manage risks** proactively
- **Ensure quality** of deliverables
- **Satisfy stakeholders** expectations

### Software Project Management Challenges

1. **Intangibility** - Software cannot be seen or touched
2. **Complexity** - Many interrelated components
3. **Conformity** - Must adapt to existing systems
4. **Changeability** - Requirements evolve
5. **Invisibility** - Progress hard to visualize

## Contract Management

### Contract Types

| Type | Description | Risk |
|------|-------------|------|
| **Fixed Price** | Set price for defined scope | Buyer low, Seller high |
| **Time & Materials** | Pay for actual time/materials | Buyer high, Seller low |
| **Cost Plus** | Cost plus agreed profit | Buyer high, Seller low |
| **Cost Plus Fixed Fee** | Cost plus fixed fee | Shared risk |

### Contract Selection Factors

- Clarity of requirements
- Project duration
- Risk tolerance
- Competitive environment
- Trust between parties

## Activities Covered by Software Project Management

### Key Management Activities

1. **Proposal Writing** - Define project and win contract
2. **Project Planning** - Establish schedule, resources
3. **Project Costing** - Estimate and control costs
4. **Project Monitoring** - Track progress
5. **Personnel Selection** - Build the team
6. **Technical Reviews** - Ensure quality
7. **Reporting** - Communicate status
8. **Risk Management** - Identify and mitigate risks

### The W5HH Principle

Questions for project initiation:
- **Why** is the system being developed?
- **What** will be done?
- **When** will it be accomplished?
- **Who** is responsible for each function?
- **Where** are they organizationally located?
- **How** will the job be done technically and managerially?
- **How much** of each resource is needed?

## Plans, Methods, and Methodologies

### Project Plan Components

1. **Introduction** - Purpose, scope, objectives
2. **Project Organization** - Team structure
3. **Risk Analysis** - Identified risks
4. **Hardware/Software Requirements** - Resources needed
5. **Work Breakdown Structure** - Task decomposition
6. **Schedule** - Timeline and milestones
7. **Monitoring Mechanisms** - Progress tracking

### Methods vs Methodologies

| Method | Methodology |
|--------|-------------|
| Specific technique | Collection of methods |
| Single purpose | Comprehensive framework |
| Example: Prototyping | Example: Agile, Waterfall |

## Categorizing Software Projects

### By Size

| Category | Team Size | Duration |
|----------|-----------|----------|
| Small | 1-3 | < 6 months |
| Medium | 4-20 | 6-24 months |
| Large | 20-100 | 1-3 years |
| Very Large | 100+ | 3+ years |

### By Type

- **Embedded systems**
- **Information systems**
- **Real-time systems**
- **Scientific/engineering**
- **Web applications**

### By Objectives

- **Product development**
- **Custom development**
- **Enhancement/maintenance**
- **Migration**
- **Integration**

## Management and Management Control

### Management Functions

**POMC:**
- **P**lanning - Define goals, strategies
- **O**rganizing - Arrange resources
- **M**otivating (Leading) - Direct and inspire
- **C**ontrolling - Monitor and correct

### Control Mechanisms

```
Plan → Execute → Monitor → Compare → Adjust
                    ↓
               Actual vs Plan
```

### Project Success Factors

1. Clear objectives
2. Stakeholder engagement
3. Skilled team
4. Realistic planning
5. Effective communication
6. Risk management
7. Quality focus

## Key Takeaways

1. A **project** is temporary with unique deliverables
2. PM addresses **intangibility and complexity** of software
3. **Contract types** distribute risk differently
4. **W5HH principle** guides project initiation
5. Projects are categorized by **size, type, and objectives**
6. **POMC** defines core management functions
