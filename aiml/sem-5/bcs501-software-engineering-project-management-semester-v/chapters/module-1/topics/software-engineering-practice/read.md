# Software Engineering Practice

## What is Software Engineering?

Software Engineering is the application of a systematic, disciplined, quantifiable approach to the development, operation, and maintenance of software.

### IEEE Definition
> "The application of a systematic, disciplined, quantifiable approach to the development, operation, and maintenance of software; that is, the application of engineering to software."

## The Software Process

A **software process** is a framework for activities required to build high-quality software. It defines:
- **Framework Activities** - Major phases of work
- **Umbrella Activities** - Supporting activities throughout
- **Tasks** - Work products and quality points

### Generic Framework Activities

```
Communication → Planning → Modeling → Construction → Deployment
      │             │           │            │            │
   Stakeholder   Estimates    Analysis    Code/Test    Delivery
   collaboration  Schedule    Design                   Support
```

1. **Communication** - Understand stakeholder objectives
2. **Planning** - Define work, risks, resources, schedule
3. **Modeling** - Create models (analysis and design)
4. **Construction** - Code generation and testing
5. **Deployment** - Delivery, feedback, support

### Umbrella Activities

These span the entire software process:
- **Software Project Tracking and Control**
- **Risk Management**
- **Software Quality Assurance**
- **Technical Reviews**
- **Measurement**
- **Configuration Management**
- **Reusability Management**
- **Work Product Preparation**

## Software Engineering Practice

### The Essence of Practice

George Polya's problem-solving approach:
1. **Understand the problem** (communication, analysis)
2. **Plan a solution** (modeling, design)
3. **Carry out the plan** (code generation)
4. **Examine the result** (testing, quality assurance)

### Core Principles of Software Engineering

| Principle | Description |
|-----------|-------------|
| **Reason it exists** | Software should provide value to users |
| **KISS** | Keep It Simple, Stupid - avoid complexity |
| **Maintain vision** | Clear vision ensures consistent system |
| **What you produce, others consume** | Design for maintainability |
| **Be open to future** | Design for extension |
| **Plan for reuse** | Reduces cost and improves quality |
| **Think!** | Clear thinking before action |

## Software Engineering Ethics

### ACM/IEEE Code of Ethics

Software engineers shall:

1. **PUBLIC** - Act in public interest
2. **CLIENT AND EMPLOYER** - Act in best interest of client/employer consistent with public interest
3. **PRODUCT** - Ensure products meet highest standards
4. **JUDGMENT** - Maintain integrity and independence
5. **MANAGEMENT** - Promote ethical approach
6. **PROFESSION** - Advance integrity and reputation
7. **COLLEAGUES** - Be fair and supportive
8. **SELF** - Participate in lifelong learning

### Ethical Dilemmas

- Disagreement with company policies
- Conflicts of interest
- Intellectual property concerns
- Computer misuse

## Process Assessment and Improvement

### CMMI (Capability Maturity Model Integration)

| Level | Name | Characteristics |
|-------|------|-----------------|
| 1 | Initial | Ad hoc, unpredictable |
| 2 | Managed | Project management, requirements management |
| 3 | Defined | Organizational standards, processes |
| 4 | Quantitatively Managed | Quantitative process control |
| 5 | Optimizing | Continuous process improvement |

### SPICE (ISO/IEC 15504)

- International standard for process assessment
- Six capability levels (0-5)
- Focuses on process improvement

## Key Takeaways

1. Software engineering applies **systematic, disciplined** approach
2. The process framework includes **five generic activities**
3. **Umbrella activities** span the entire project
4. **Ethics** are fundamental to professional practice
5. **Process improvement** (CMMI, SPICE) ensures quality
