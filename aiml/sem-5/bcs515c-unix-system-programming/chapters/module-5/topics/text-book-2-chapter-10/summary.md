### Revision Notes

#### 1. Introduction to System Calls

- System calls are API functions that interact with the operating system.
- They provide an interface between the user-level applications and the kernel.
- Examples of system calls include process creation, file I/O, and memory management.

#### 2. Types of System Calls

- **System Primitive Calls**: These are the basic system calls that perform core system functions, such as process creation and termination.
- **System Service Calls**: These are system calls that provide additional functionality, such as file I/O and memory management.

#### 3. System Call Types

- **System Calls with Arguments**: These system calls take arguments and return values.
- **System Calls without Arguments**: These system calls do not take arguments and return values.

#### 4. System Call Examples

- `fork()` system call: Creates a new process.
- `exec()` system call: Executes a new program.
- `read()` system call: Reads data from a file.

#### 5. System Call Interface

- System calls typically have the following interface:
  - **System Call Number**: A unique number that identifies the system call.
  - **Argument List**: A list of arguments that are passed to the system call.
  - **Return Value**: The return value of the system call.

#### 6. Error Handling

- System calls can return errors if they fail.
- Errors can be checked using the `errno` variable.

#### 7. System Call Optimization

- System calls can be optimized using various techniques, such as caching and buffering.

#### 8. System Call Performance

- System calls can impact system performance.
- Optimizing system calls can improve system performance.

#### 9. System Call Security

- System calls can be used to exploit security vulnerabilities.
- System calls should be used securely to prevent exploitation.

#### 10. System Call Best Practices

- System calls should be used judiciously to avoid impacting system performance.
- System calls should be optimized to improve system performance.
- System calls should be used securely to prevent exploitation.
