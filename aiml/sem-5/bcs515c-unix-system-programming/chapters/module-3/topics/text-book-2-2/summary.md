# **UNIX SYSTEM PROGRAMMING**

## **Text Book 2: 2**

## **Key Points**

- **Introduction**:
  - UNIX operating system
  - Multi-user, multi-tasking, multi-processing
  - Open-source, free software
- **Unix Standardization**:
  - ISO/IEC 9945-1 (2002)
  - Standardized features and APIs
  - Ensured compatibility across systems
- **UNIX**:
  - Unix philosophy: simplicity, portability, and reusability
  - Core utilities: `ls`, `cd`, `mkdir`, etc.
  - File system: hierarchical, rooted at `/`

## **Important Formulas and Definitions**

- **File System Hierarchy Standard (FHS)**:
  - /bin
  - /boot
  - /dev
  - /etc
  - /home
  - /lib
  - /media
  - /mnt
  - /opt
  - /proc
  - /root
  - /run
  - /sbin
  - /srv
  - /sys
  - /tmp
  - /usr
  - /var
- **Unix Permissions**:
  - u (owner): read, write, execute
  - g (group): read, write, execute
  - o (other): read, write, execute
  - r (read): 4 (owner), 2 (group), 1 (other)
  - w (write): 2 (owner), 1 (group), 1 (other)
  - x (execute): 1 (owner), 1 (group), 1 (other)

## **Important Theorems**

- **Unix Philosophy**: "The best thing about a Unix is that you can use it twice as hard as any other machine if you know how."
- **Portability**: "A portable program is one that can be transferred from one machine to another without modification."
- **Reusability**: "A reusable program is one that can be reused on different machines without modification."
