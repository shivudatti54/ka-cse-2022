# Basic File Types/Categories

### Overview

- A file is a collection of data stored on a storage device
- Files are categorized into different types based on their characteristics and usage

### File Types

- **Regular Files**:
  - Contain data in a structured format (e.g., text, images, executables)
  - Can be accessed and modified by users
- **Directory Files**:
  - Contain metadata about other files (e.g., file names, permissions)
  - Used to organize and store information about files
- **Special Files**:
  - Represent device files, socket files, and named pipe files
  - Used to represent hardware devices or communication channels
- **Symbolic Links**:
  - Point to a file or directory on the system
  - Provide a shortcut to a file or directory
- **Block Special Files**:
  - Represent block devices (e.g., hard drives, SSDs)
  - Used to provide direct access to block devices
- **Character Special Files**:
  - Represent character devices (e.g., keyboards, printers)
  - Used to provide direct access to character devices

### File Categories

- **Executable Files**:
  - Contain machine code that can be executed by the CPU
  - Can be executed by users with execute permissions
- **Binary Files**:
  - Contain data in binary format (e.g., images, audio files)
  - May require additional software to interpret
- **Text Files**:
  - Contain data in plain text format (e.g., documents, configuration files)
  - Can be edited and modified using text editors
- **Archive Files**:
  - Contain multiple files compressed or stored in a single file
  - Used to store and transfer files efficiently

### Important Formulas and Definitions

- **File Association**:
  - A mapping between a file type and its corresponding file extension
  - Determines how a file is displayed and executed
- **File Permissions**:
  - A set of permissions that control access to a file or directory
  - Determined by the owner, group, and others

### Theorems

- **Unix File System (UFS) Theorem**:
  - States that the UFS is a hierarchical file system
  - Organizes files into a tree-like structure
