# **Becoming the Super User: su Command**

### Key Points

- **What is su?**: The `su` command allows a user to assume the identity of another user, becoming the superuser.
- **Syntax**: `su [options] [username]`
- **Options**:
  - `-s` specifies the shell to use
  - `-l` logs in as a login shell
  - `-p` preserves the current environment
- **Privileges**: Becoming the superuser grants all privileges, including root access.

### Important Formulas, Definitions, and Theorems

- None specific to this topic.

### Important Concepts

- **Root user**: The superuser, with full privileges and access to all files and systems.
- **User privilege**: The level of access and permissions a user has to a system or file.

### Key Functions of su

- **Switching user**: Allows a user to temporarily become the superuser.
- **Changing shell**: Allows a user to change the shell to use when logging in as the superuser.

### Key Applications of su

- **System administration**: The superuser is often used for system administration tasks, such as installing software or editing system configuration files.
- **Troubleshooting**: The superuser can be used to troubleshoot issues that require elevated privileges.
