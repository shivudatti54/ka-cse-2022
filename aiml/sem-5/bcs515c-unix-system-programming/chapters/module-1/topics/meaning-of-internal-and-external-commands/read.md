# **Meaning of Internal and External Commands**

## **Introduction**

In the UNIX operating system, commands are executed to perform various tasks. These commands can be categorized into two types: internal and external commands. In this study material, we will explore the meaning, definition, and differences between internal and external commands.

## **Internal Commands**

### Definition

Internal commands are programs that are built into the UNIX operating system. They are executed directly by the operating system without the need to invoke an external program.

### Characteristics

- Built into the operating system
- Executed directly by the operating system
- Not stored on the user's disk
- Typically used for simple tasks
- Examples: `cd`, `pwd`, `date`

### Examples

- `cd` (change directory): changes the current working directory
- `pwd` (print working directory): displays the current working directory
- `date`: displays the current date and time

### Advantages

- Faster execution time
- Reduce memory usage
- Simplify system administration

### Disadvantages

- Limited functionality
- Not customizable

## **External Commands**

### Definition

External commands are programs that are stored on the user's disk and executed by the operating system.

### Characteristics

- Stored on the user's disk
- Executed by the operating system
- Can be customized
- Typically used for complex tasks
- Examples: `grep`, `sed`, `bash`

### Examples

- `grep` (global regular expression print): searches for a pattern in a file
- `sed` (stream editor): edits and manipulates text in a file
- `bash` (Bourne-Again SHell): a command-line interface and scripting language

### Advantages

- Flexible and customizable
- Can perform complex tasks
- Reusable code

### Disadvantages

- Slower execution time
- Increase memory usage
- More complex to use

## **Key Concepts**

- Internal commands are built into the operating system and executed directly.
- External commands are stored on the user's disk and executed by the operating system.
- Internal commands are typically used for simple tasks, while external commands are used for complex tasks.
- Internal commands have faster execution time and reduce memory usage, but are limited in functionality.
- External commands have flexible and customizable functionality, but are slower and more memory-intensive.
