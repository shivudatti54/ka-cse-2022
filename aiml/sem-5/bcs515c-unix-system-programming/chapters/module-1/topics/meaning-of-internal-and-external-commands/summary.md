# **Meaning of Internal and External Commands**

## **Definition**

- **Internal Command**: A command that is built into the operating system and is executed by the kernel itself. It is not stored in the shell's command history and does not require the use of the shell to execute.
- **External Command**: A command that is not built into the operating system and is executed by the shell. It is stored in the user's command history and requires the use of the shell to execute.

## **Key Points**

- Internal Commands:
  - Examples: `cd`, `pwd`, `date`, `uname`
  - Executed by the kernel directly
  - Not stored in the shell's command history
- External Commands:
  - Examples: `ls`, `grep`, `sort`
  - Executed by the shell
  - Stored in the user's command history

## **Importance**

- Internal commands are faster and more efficient, as they do not require the shell to interpret the command.
- External commands are more flexible, as they can be modified and extended by the user.
