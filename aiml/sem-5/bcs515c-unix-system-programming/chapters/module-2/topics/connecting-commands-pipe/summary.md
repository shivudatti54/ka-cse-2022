# **Connecting Commands: Pipe**

### Overview

The pipe (`|`) is a powerful command connector that allows you to redirect the output of one command as the input for another.

### Key Points

- **Pipe syntax**: `command1 | command2`
- **Redirecting output**: Output of `command1` becomes input for `command2`
- **Multiple pipes**: Multiple commands can be connected using pipes to create complex workflows

### Benefits

- **Streamlines workflows**: Piping allows you to automate tasks and create efficient workflows
- **Simplifies complex tasks**: Breaking down complex tasks into smaller, manageable parts using pipes
- **Improves productivity**: By automating repetitive tasks and simplifying workflows

### Examples

- `ls -l | grep keyword` - Search for a keyword in the output of a list of files
- `cat file.txt | mail user@example.com` - Send the contents of a file to a user via email
