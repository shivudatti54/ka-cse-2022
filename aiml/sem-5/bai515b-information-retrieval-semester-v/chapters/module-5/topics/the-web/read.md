# The Web as an Information Retrieval Corpus

## Introduction

The World Wide Web is the largest, most diverse, and dynamic information repository ever created. Unlike controlled collections like a library database, the web is decentralized, unstructured, and constantly evolving. This presents unique and monumental challenges for Information Retrieval (IR) systems. Traditional IR techniques must be adapted and scaled to effectively find, organize, and rank information in this vast, ungoverned space. This module explores the core structures and mechanisms that enable IR on the web.

## Core Concepts

### 1. Web Structure: The Graph Model

The web is best modeled as a **directed graph**. In this model:
*   **Nodes:** Represent web pages (vertices).
*   **Edges:** Represent hyperlinks (directed edges) from one page to another.

This graph structure is not random; it exhibits specific properties like the **"bow-tie" structure**, which divides the web into several components:
*   **Giant Strongly Connected Core (SCC):** A large set of pages where you can navigate from any page to any other via a series of links.
*   **IN pages:** Pages that link into the SCC but cannot be reached from it.
*   **OUT pages:** Pages that are linked from the SCC but do not link back into it.
*   **Tendrils and Tubes:** Other isolated clusters.

This structure is fundamental for understanding how web crawlers traverse the web and how algorithms like PageRank determine importance.

### 2. Web Crawling

A **web crawler** (or spider) is a software program that systematically browses the web to download pages for a search engine's index. The crawling process is a graph traversal problem.

**How it works:**
1.  **Seed URLs:** The crawler starts with a pre-defined list of seed URLs.
2.  **Download:** It fetches the page content from a URL.
3.  **Parse & Extract:** It parses the downloaded page to extract all outgoing hyperlinks.
4.  **URL Frontier:** These new URLs are added to a queue-like data structure called the **URL frontier**.
5.  **Politeness:** The crawler must obey rules like `robots.txt` and introduce delays between requests to the same server to avoid overloading it.
6.  **Recursion:** The process repeats, taking the next URL from the frontier.

*Example:* Imagine a crawler starting at `.ac.in`. It downloads the page, finds links to `results..ac.in` and `notifications..ac.in`, adds them to the frontier, and then proceeds to crawl those pages next.

**Challenges:** The web's scale (billions of pages), its dynamic nature (pages change constantly), and the presence of "spider traps" (infinite loops generated by calendars) make crawling a complex engineering task.

### 3. The PageRank Algorithm

Traditional IR ranked results based solely on the content of a document (tf-idf). For the web, this is insufficient, as it's easy to create content-rich pages that are spammy or irrelevant. **PageRank**, developed by Google founders, introduced the concept of **link analysis** to measure the **importance** or **authority** of a web page.

The core idea is democratic: **A page is important if it is linked to by other important pages.**

**Simplified Explanation:**
PageRank models a "random surfer" who clicks on links at random. The probability that the surfer is on a given page at any time is that page's PageRank. To avoid dead-ends, the model includes a **damping factor `d`** (typically ~0.85), which is the probability that the surfer will click a link. There's a probability `(1-d)` that the surfer will get bored and jump to any random page on the web.

The PageRank `PR(P)` of a page `P` is calculated as:
`PR(P) = (1-d) + d * ( PR(T₁)/C(T₁) + ... + PR(Tₙ)/C(Tₙ) )`
Where:
*   `T₁` to `Tₙ` are pages that link to `P` (incoming links).
*   `C(Tₙ)` is the total number of outgoing links on page `Tₙ`.
*   `d` is the damping factor.

*Example:* If a highly-ranked page like Wikipedia links to your project page, it passes more "authority" (PageRank) than if a low-ranked, unknown blog links to you. This helps search engines rank genuinely valuable resources higher than spam.

### 4. The Hubs and Authorities Algorithm (HITS)

Another link analysis algorithm, **HITS** (Hyperlink-Induced Topic Search), identifies two types of pages for a specific query:
*   **Hubs:** Pages that are useful directories or link collections pointing to many good, authoritative pages on a topic. (e.g., a curated list of "Best Machine Learning Tutorials").
*   **Authorities:** Pages that contain high-quality, definitive information on the topic. (e.g., the official TensorFlow documentation).

HITS operates in a query-dependent fashion. It first retrieves a **root set** of pages relevant to the query, then expands it to a **base set** by including pages that link to or are linked from the root set. An iterative algorithm then assigns two scores to each page:
*   An **authority score** is the sum of the hub scores of pages pointing to it.
*   A **hub score** is the sum of the authority scores of pages it points to.

Good hubs point to good authorities, and good authorities are pointed to by good hubs.

## Key Points & Summary

| Concept | Key Idea | Purpose |
| :--- | :--- | :--- |
| **Web as a Graph** | Models pages as nodes and links as edges. | Understands the web's interconnected structure for traversal and analysis. |
| **Web Crawling** | Systematic, automated browsing and downloading of web pages. | Populates the search engine's index with raw data. |
| **PageRank** | Measures page importance based on the quality and quantity of incoming links. | Provides a query-independent, global importance score for ranking. |
| **HITS Algorithm** | Identifies hubs (good link lists) and authorities (definitive content) for a query. | Provides query-dependent scores to rank results on a specific topic. |

In conclusion, IR on the web extends beyond simple keyword matching. It leverages the web's inherent link structure to combat spam, infer quality, and ultimately deliver more relevant and authoritative results to users. Understanding these graph-based algorithms is crucial for any computer engineer working on large-scale data systems.
