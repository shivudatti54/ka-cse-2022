### Learning Purpose: Pie Charts

**1. Why is this topic important?**
Understanding pie charts is foundational for data visualization. While often criticized for potential misinterpretation, they remain a common method for communicating proportional data in business and media. Learning to create and, more importantly, critique them is crucial for any data analyst to both use them effectively and avoid their pitfalls.

**2. What will students learn?**
Students will learn the syntax to create pie charts in R using base R functions (`pie()`) and the `ggplot2` package (`coord_polar()`). They will master how to label segments, adjust colors, and add a legend. Crucially, they will also learn the best practices for when to use a pie chart and when to choose a superior alternative like a bar chart to represent part-to-whole relationships accurately.

**3. How does it connect to other concepts?**
This topic builds directly on Module 3’s introduction to `ggplot2` and core visualization principles. It reinforces the grammar of graphics (aesthetics, geometries) and serves as a practical application of data manipulation skills (e.g., aggregating data into counts or percentages) before creating a visual. It also provides a comparative basis for evaluating other chart types.

**4. Real-world applications**
Pie charts are used to quickly communicate market share, budget allocation, survey response distributions (e.g., percentage of customers satisfied), or demographic breakdowns. An analyst must be able to generate them for common stakeholder requests and, when necessary, advocate for a more effective visualization to ensure clear and truthful data storytelling.
