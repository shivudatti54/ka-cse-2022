# Storing A Word In Memory
The store operation transfers data from a CPU register to memory, requiring coordinated action of MAR, MDR, address/data buses, and control signals. Key concepts include: (1) RTL notation M[EA] ← R for describing store operations formally; (2) step-by-step execution involving address computation, data loading into MDR, write signal activation, and MFC acknowledgment; (3) timing analysis showing 2-3 clock cycles for main memory stores; (4) word alignment affecting performance—aligned stores access single memory locations while unaligned may require multiple; (5) endianness determining byte ordering in multi-byte storage (big-endian: MSB at lowest address; little-endian: LSB at lowest address); (6) cache write policies—write-through writes both cache and memory simultaneously while write-back defers memory update until cache line eviction; (7) pipeline considerations including store-to-load forwarding and memory ordering constraints. Store operations execute in the MEM stage of the instruction cycle and do not require the Write-Back stage.
