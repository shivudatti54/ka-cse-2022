# Encoder

## Introduction

An encoder is a fundamental combinational logic circuit that performs the reverse operation of a decoder. While a decoder converts n binary inputs into 2^n outputs, an encoder performs the opposite function: it converts 2^n input lines into n binary output lines. In digital systems, encoders are essential for reducing the number of transmission lines, compressing data, and converting parallel data into serialized format. They are extensively used in keyboards, where multiple keys need to be represented by a smaller set of binary codes, in analog-to-digital converters for position encoding, and in various communication systems for data compression.

The encoder accepts 2^n input lines, of which only ONE is activated at any given time, and produces an n-bit code corresponding to that active input. This conversion from multiple input lines to a compact binary representation is crucial in digital electronics where efficiency and resource optimization are paramount. Understanding encoders is essential for computer science students as they form the building blocks of more complex digital systems including multiplexers, memory addressing systems, and data routing circuits.

## Key Concepts

### Basic Encoder (8-to-3 Encoder)

The 8-to-3 encoder, also known as octal-to-binary encoder, has eight input lines (D0 through D7) and three output lines (A, B, C representing 3-bit binary code). The fundamental principle is that when exactly one input line is HIGH (or logic 1), the outputs produce the binary equivalent of that input's position. The output is essentially the binary representation of which input line is activated.

The truth table for an 8-to-3 encoder shows that when D0 is HIGH, output is 000; when D1 is HIGH, output is 001; when D2 is HIGH, output is 010, and so on up to D7 producing output 111. This one-hot to binary conversion is the core operation of all basic encoders.

### Priority Encoder

A critical limitation of the basic encoder arises when multiple input lines are HIGH simultaneously. In practical systems, multiple signals might be active at the same time, and the basic encoder cannot handle this situation. The priority encoder solves this problem by assigning priority levels to input lines. Typically, the input with the highest priority (usually the highest-numbered input line) determines the output code.

The 4-to-2 priority encoder demonstrates this concept effectively. When multiple inputs are active, the encoder examines them in a specific order (priority) and outputs the binary code for the highest-priority active input. Additionally, priority encoders often include a valid output indicator (V) that signals when at least one input is active, making the output meaningful.

### Decimal to BCD Encoder

This special-purpose encoder converts decimal digits (0-9) into Binary Coded Decimal representation. It has ten input lines corresponding to digits 0 through 9, and four output lines producing the BCD code. For instance, pressing key 5 would activate input D5, producing output 0101 on the four output lines. This type of encoder is fundamental in keyboard matrix circuits where numeric keypads interface with digital systems.

### Boolean Expressions for Encoder Outputs

For an 8-to-3 encoder, the Boolean expressions for outputs can be derived from the truth table:

A = D4 + D5 + D6 + D7 (outputs 1 for inputs 4-7)
B = D2 + D3 + D6 + D7 (outputs 1 for inputs 2,3,6,7)
C = D1 + D3 + D5 + D7 (outputs 1 for inputs 1,3,5,7)

These expressions can be implemented using OR gates, making the hardware implementation straightforward and cost-effective.

### Encoder Implementation Using OR Gates

The hardware realization of an 8-to-3 encoder requires three OR gates, each implementing one of the output Boolean functions. The first OR gate produces output A by ORing inputs D4, D5, D6, and D7. Similarly, output B is generated by ORing D2, D3, D6, and D7, while output C is the OR of D1, D3, D5, and D7. This implementation assumes that only one input is HIGH at any time, which is a fundamental assumption for basic encoders.

## Examples

### Example 1: Analyze the output of an 8-to-3 encoder when input D6 is HIGH

Solution: When input D6 is activated (logic 1), the encoder must produce the binary representation of decimal 6. Looking at the output lines (A, B, C), we determine each bit:

- Output A = D4 + D5 + D6 + D7 = 0 + 0 + 1 + 0 = 1
- Output B = D2 + D3 + D6 + D7 = 0 + 0 + 1 + 0 = 1
- Output C = D1 + D3 + D5 + D7 = 0 + 0 + 0 + 0 = 0

Therefore, the output is A=1, B=1, C=0, which represents binary 110 (decimal 6). This confirms that the encoder correctly converts the active input line D6 to its corresponding binary code.

### Example 2: Design a 4-to-2 priority encoder with input priority D3 > D2 > D1 > D0

Solution: For a priority encoder, we must handle all possible input combinations. The truth table shows inputs D3, D2, D1, D0 and outputs A, B along with a valid output V.

When D3=1 (highest priority), output is 11 regardless of other inputs. When D3=0 and D2=1, output is 10. When D3=0, D2=0, and D1=1, output is 01. When only D0=1 (all others 0), output is 00. When all inputs are 0, V=0 indicating no valid input.

The Boolean expressions are:
A = D3 + D2'·D1
B = D3 + D2
V = D3 + D2 + D1 + D0

This priority encoder can be implemented using OR gates and AND gates with inverters, providing correct output even when multiple inputs are HIGH.

### Example 3: Determine why a basic encoder cannot handle simultaneous inputs, and explain the solution

Solution: Consider an 8-to-3 encoder with inputs D3=1 and D5=1 simultaneously active. Using the Boolean expressions:

Output A = D4 + D5 + D6 + D7 = 0 + 1 + 0 + 0 = 1
Output B = D2 + D3 + D6 + D7 = 0 + 1 + 0 + 0 = 1
Output C = D1 + D3 + D5 + D7 = 0 + 1 + 1 + 0 = 1

The output would be 111, which incorrectly indicates input D7 is active. This is completely wrong because neither D7 is actually HIGH. The basic encoder produces erroneous results when multiple inputs are active simultaneously.

The solution is to use a priority encoder that establishes a hierarchy among inputs. When D3 and D5 are both HIGH, the priority encoder recognizes that D5 has higher priority (in a typical 7>6>5>4>3>2>1>0 scheme) and outputs the code for D5, which is 101. This demonstrates why priority encoders are preferred in practical applications where simultaneous inputs are possible.

## Exam Tips

For DU semester examinations, keep the following points in mind when tackling encoder questions:

The most frequently tested concept is deriving Boolean expressions from encoder truth tables. Practice converting truth tables to simplified expressions using Boolean algebra or Karnaugh maps. Remember that encoder outputs are expressed as OR operations of specific input lines.

Understanding the difference between basic encoders and priority encoders is crucial. Basic encoders assume only ONE input is active, while priority encoders can handle multiple simultaneous inputs by assigning priority. This distinction often appears in exam questions asking you to explain limitations or justify the use of priority encoders.

The 8-to-3 encoder and 4-to-2 priority encoder are the most commonly asked configurations. Memorize their truth tables and the corresponding Boolean expressions. You should be able to draw the gate-level implementation from either the truth table or the Boolean expressions.

When asked to design an encoder, always start by creating the truth table, then derive minterms for each output, and finally simplify using Boolean algebra. Show all steps as partial credit is awarded for correct procedure even if final answer has errors.

Remember that encoders have 2^n inputs and n outputs, while decoders have n inputs and 2^n outputs. This inverse relationship is a common multiple-choice question topic.

For numerical problems, verify your answers by checking if the output binary code correctly represents the activated input line number in decimal. This self-check method helps catch calculation errors.

The valid output (V) in priority encoders indicates when at least one input is active. This feature is important for system design and is often tested in context of practical applications.
