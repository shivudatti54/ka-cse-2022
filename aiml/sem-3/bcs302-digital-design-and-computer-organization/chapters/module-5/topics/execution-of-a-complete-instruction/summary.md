# **Execution of a Complete Instruction**

### Overview

- Execution of a complete instruction refers to the process of carrying out a single instruction in a computer.
- It involves fetching, decoding, and executing the instruction.

### Key Points

#### Fetch

- Fetch instruction: Retrieves the instruction from memory.
- Address calculation: Calculates the memory address of the instruction.
- Instruction fetch unit: Fetches the instruction from memory.

#### Decode

- Decode instruction: Converts the instruction into a machine code.
- Instruction decoding unit: Decodes the instruction and generates control signals.

#### Execute

- Execute instruction: Carries out the operation specified by the instruction.
- Arithmetic logic unit (ALU): Performs arithmetic and logical operations.
- Register file: Stores data temporarily.

#### Write Back

- Write back instruction: Stores the result of the instruction in memory or registers.
- Write back unit: Stores the result of the instruction.

### Important Formulas and Definitions

- **Instruction Length (IL)**: The number of bits required to represent an instruction.
- **Instruction Set Architecture (ISA)**: The set of instructions that a processor can execute.
- **Fetch Delay Cycle**: The time it takes to fetch an instruction from memory.
- **Decode Delay Cycle**: The time it takes to decode an instruction.
- **Execute Delay Cycle**: The time it takes to execute an instruction.
- **Write Back Delay Cycle**: The time it takes to write back the result of an instruction.

### Important Theorems

- **Amdahl's Law**: Limits the maximum theoretical speedup of a parallel program.
- **Gale-Shapley Algorithm**: A fairness-aware scheduling algorithm for multiple tasks.

### Revision Tips

- Understand the fetch-decode-execute cycle.
- Familiarize yourself with the instruction set architecture (ISA).
- Practice calculating fetch, decode, and execute delay cycles.
