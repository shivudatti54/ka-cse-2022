# Enabling And Disabling Interrupts
Interrupts are asynchronous events requiring CPU attention, classified into hardware interrupts (I/O, timer), software interrupts (system calls, exceptions), and categorized as maskable (can be disabled) or non-maskable (always serviced). Processors implement an Interrupt Flag (IF) in the status register—setting IF=1 enables recognition of maskable interrupts while IF=0 disables them. Different architectures provide instructions (CLI/STI on x86, CPSID/CPSIE on ARM, DI/EI on 8085, CSR manipulation on RISC-V) to control this flag. Critical sections requiring atomic execution can be protected by disabling interrupts, guaranteeing no ISR interleaves with the protected code. However, prolonged interrupt disablement degrades system responsiveness, increases interrupt latency, and may cause missed events or watchdog timeouts. Best practices include minimizing critical section duration, saving/restoring interrupt state for nested sections, using finer-grained locking when possible, and measuring maximum disable intervals in real-time systems.
