# Process Management
A process is an active entity representing a program in execution, distinguished from the passive program file by its execution context including program counter, CPU registers, and allocated resources. Processes are organized in memory with distinct segments: text (code), data (globals), heap (dynamic allocation), and stack (function calls). The process lifecycle consists of states—New, Ready, Running, Waiting, and Terminated—with transitions managed by the OS. Each process is represented by a Process Control Block (PCB) containing state, PID, registers, scheduling information, and resource allocations. Context switching saves and restores process state between CPU allocations. Process scheduling employs queues (Job, Ready, Device) and various algorithms (FCFS, SJF, Priority, Round Robin, Multilevel) to optimize CPU utilization, throughput, turnaround time, waiting time, and response time. The OS creates processes via system calls like fork() and exec(), manages their termination, and facilitates inter-process communication through shared memory, message passing, and other mechanisms.
