# Learning Objectives

After studying this topic, you should students able to:

1. **Define** the system model in operating systems and explain its importance in analyzing process synchronization and deadlock problems.

2. **Differentiate** between uniprocessor, multiprocessor, and distributed system models, recognizing how each affects synchronization and deadlock handling.

3. **Identify** the components of the process model including program counter, registers, stack, and address space.

4. **Classify** resources as preemptible or non-preemptible and explain why non-preemptible resources are central to deadlock scenarios.

5. **State and explain** the four Coffman conditions necessary for deadlock to occur: Mutual Exclusion, Hold and Wait, No Preemption, and Circular Wait.

6. **Construct and interpret** a Resource Allocation Graph (RAG) for a given system state, identifying request edges and assignment edges.

7. **Analyze** when a cycle in the Resource Allocation Graph indicates deadlock versus potential deadlock.

8. **Apply** the system model assumptions to evaluate whether deadlock is possible in given scenarios.
