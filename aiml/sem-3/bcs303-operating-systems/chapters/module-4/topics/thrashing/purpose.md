# Learning Objectives

After studying this topic, you should be able to:

1. Define thrashing and explain the circumstances under which it occurs in virtual memory systems.

2. Describe the mechanism of the thrashing cycle and explain why it creates a self-reinforcing pathological state.

3. Apply the working set model to determine whether a given memory allocation scenario will result in thrashing.

4. Explain the page fault frequency approach to thrashing prevention and describe how it dynamically allocates frames.

5. Identify the causes of thrashing and explain how each cause contributes to excessive paging activity.

6. Analyze system performance metrics to detect thrashing and propose appropriate resolution strategies.

7. Evaluate the relationship between degree of multiprogramming and system efficiency, including the thrashing point.

8. Compare different approaches to thrashing prevention and select appropriate strategies for various scenarios.
