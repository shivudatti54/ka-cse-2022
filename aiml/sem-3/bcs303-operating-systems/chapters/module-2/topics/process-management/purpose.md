# Learning Objectives

After studying this topic, you should be able to:

1. Define a process and explain how it differs from a program, identifying the key components that constitute process execution context

2. Describe the structure and contents of the Process Control Block (PCB), explaining its critical role in process management and context switching

3. Illustrate and interpret the process state transition diagram, identifying valid state transitions and the events that trigger each transition

4. Compare and contrast long-term, short-term, and medium-term schedulers, explaining their roles in managing process execution and system resources

5. Analyze different process scheduling algorithms (FCFS, SJF, Round Robin, Priority) by calculating waiting time, turnaround time, and throughput for given process sets

6. Explain process creation and termination operations, including the use of fork() and exec() system calls in Unix-like systems

7. Compare interprocess communication models (shared memory and message passing), identifying appropriate use cases and synchronization requirements for each
