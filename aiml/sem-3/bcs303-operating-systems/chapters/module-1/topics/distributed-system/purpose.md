# Learning Objectives

After studying this topic, you should be able to:

1. Define a distributed system and explain how it differs from centralized computing systems in terms of architecture and resource management.

2. Identify and explain the seven types of transparency in distributed systems, with practical examples of each.

3. Distinguish between distributed operating systems and network operating systems, highlighting the key differences in system image, process migration, and user perception.

4. Describe the three main categories of distributed systems and provide examples of each type.

5. Explain the communication mechanisms in distributed systems, including message passing, remote procedure call (RPC), and distributed shared memory.

6. Understand the role of middleware in distributed systems and how it facilitates application development.

7. Analyze the advantages and disadvantages of distributed systems, including scalability, reliability, complexity, and security considerations.

8. Apply the concepts of transparency and distribution to real-world distributed system examples like NFS, DNS, and web-based applications.
