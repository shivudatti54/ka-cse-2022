# What Operating Systems Do

## Introduction

An Operating System (OS) serves as the fundamental layer of software that manages computer hardware resources and provides services to application programs. It acts as an intermediary between the user and the computer hardware, creating an environment where users can execute programs without needing to understand the complex details of hardware manipulation. The operating system is perhaps the most critical component of any computer system, as it coordinates all activities and ensures that the diverse components of a computer work together harmoniously.

The concept of operating systems evolved significantly over time. In the early days of computing, there were no operating systems; users had direct access to hardware and had to manually manage all system resources. The introduction of batch processing systems in the 1950s and 1960s marked the beginning of operating system development, followed by time-sharing systems, personal computer operating systems, and modern distributed and mobile operating systems. Today, operating systems range from simple embedded systems running on IoT devices to complex multi-server systems powering cloud computing infrastructure. Understanding what operating systems do is fundamental to grasping computer science concepts and developing efficient software applications.

## Key Concepts

### 1. Resource Management

The primary function of an operating system is to manage computer resources efficiently. Resources include the central processing unit (CPU), main memory (RAM), secondary storage (hard drives, SSDs), and input/output devices such as keyboards, mice, printers, and network cards. The OS must allocate these limited resources among multiple programs and users, ensuring fair access while maximizing overall system performance. Resource management involves tracking which resources are in use, which are available, and implementing scheduling algorithms to determine which process gets access to which resource and for how long.

### 2. Process Management

A process is a program in execution, and process management is one of the most crucial functions of an operating system. The OS creates, schedules, and terminates processes. It maintains a process control block (PCB) for each process, which stores important information such as process state, program counter, CPU registers, memory management information, and accounting data. Process management also involves inter-process communication (IPC), allowing processes to share data and synchronize their activities safely. The OS implements process scheduling algorithms like First-Come-First-Served (FCFS), Shortest Job First (SJF), Round Robin, and Priority Scheduling to determine the order in which processes access the CPU.

### 3. Memory Management

Memory management refers to the OS's ability to control how physical memory is allocated to processes and how virtual memory is managed. The OS must track which parts of memory are in use and which are free, allocate memory to processes when they need it, and deallocate when processes terminate. Modern operating systems implement virtual memory, which allows programs to use more memory than physically available by swapping data between RAM and secondary storage. Memory management techniques include paging, segmentation, and demand paging, each with its own advantages in terms of memory efficiency and access speed.

### 4. File Management

Operating systems provide file management capabilities through a file system, which organizes data stored on secondary storage devices. The OS manages files by creating, deleting, opening, closing, reading, and writing files. It maintains file metadata including file name, creation date, permissions, and size. File management also involves organizing files into directories (folders) and managing the allocation of disk space. Different file systems like FAT32, NTFS, ext4, and APFS implement various algorithms for file storage and retrieval, affecting performance, reliability, and features.

### 5. Device Management

Operating systems manage input and output devices through device drivers, which are specialized programs that allow the OS to communicate with hardware peripherals. The OS handles device scheduling to manage multiple requests to the same device, implements buffering to improve I/O performance, and provides abstraction layers so that applications can interact with devices without knowing their specific hardware details. Device management includes handling interrupts generated by devices, managing device queues, and providing error recovery mechanisms.

### 6. Security and Protection

Operating systems implement security mechanisms to protect system resources from unauthorized access. This includes user authentication through passwords and biometrics, file permissions controlling who can read, write, or execute files, and access control lists defining privileges for different users. The OS also provides protection against malicious software by integrating antivirus capabilities and implementing process isolation to prevent one program from interfering with another. Modern operating systems include features like sandboxing, address space layout randomization (ASLR), and secure boot mechanisms.

### 7. User Interface

Operating systems provide interfaces through which users interact with the computer system. These include command-line interfaces (CLI) where users type text commands, and graphical user interfaces (GUI) with windows, icons, menus, and pointers. Some operating systems provide both options, while others focus primarily on one type. The user interface also includes utility programs like file managers, text editors, and system monitors that help users perform common tasks. Mobile operating systems like Android and iOS emphasize touch-based interfaces optimized for smartphones and tablets.

### 8. Platform for Application Execution

The operating system provides a platform or environment where application software runs. It offers system calls and APIs that programmers use to interact with hardware resources. By providing this abstraction layer, the OS enables application programs to run on different hardware configurations without modification. This portability is essential for software development, as programmers can write applications that work across multiple computer systems as long as they adhere to the OS's programming interfaces.

## Examples

### Example 1: Process Scheduling Demonstration

Consider a system with three processes arriving at different times: Process A (arrival time 0, burst time 5), Process B (arrival time 1, burst time 3), and Process C (arrival time 2, burst time 4). Using FCFS scheduling, the OS would execute Process A first from time 0 to 5, then Process B from time 5 to 8, and finally Process C from time 8 to 12. The average waiting time would be: Process A waits 0 units, Process B waits 4 units (5-1), and Process C waits 6 units (8-2), giving an average of (0+4+6)/3 = 3.33 time units.

If we apply Shortest Job First (non-preemptive), the OS would still execute in the same order because Process A has the shortest burst time among processes available at time 0. However, with pre-emptive SJF (Shortest Remaining Time First), at time 1 when Process B arrives, the OS compares remaining time of A (4) with B (3) and pre-empts A to run B first, demonstrating how modern OSes dynamically manage process execution.

### Example 2: Memory Allocation Scenario

Suppose a system has 1000 KB of memory and needs to allocate space to four processes: P1 requires 200 KB, P2 requires 300 KB, P3 requires 150 KB, and P4 requires 250 KB. Using contiguous memory allocation with First-Fit, the OS would find the first hole large enough for each process in sequence. If memory holes are [300 KB at address 0], [200 KB at address 400], [500 KB at address 700], P1 gets 200 KB at 0 (remaining hole 100 KB), P2 gets 300 KB at 700 (remaining hole 200 KB), P3 gets 150 KB at 400 (remaining hole 50 KB), and P4 gets 250 KB at the remaining space. This demonstrates how memory management algorithms work in practice.

### Example 3: File System Operation

When a user creates a new file named "notes.txt" containing the text "Operating Systems are important," the operating system's file management performs several operations: it searches the directory structure for the appropriate folder, creates a new directory entry with the filename and metadata, allocates disk blocks for the file content, writes the data to those blocks, and updates the file system's allocation tables. The file system maintains an inode (in UNIX-like systems) or MFT entry (in NTFS) that stores this information, including pointers to the actual data blocks on disk. When the file is later opened, the OS retrieves this metadata and sets up the appropriate data structures for reading and writing operations.

## Exam Tips

1. Understand the distinction between OS as a resource manager versus OS as a user interface - both perspectives are important for exams.

2. Memorize the five major functions of an operating system: Process Management, Memory Management, File Management, Device Management, and Security/Protection.

3. Be prepared to explain how system calls work - they provide the interface between user programs and the OS kernel.

4. Know the differences between various types of operating systems: batch, time-sharing, real-time, distributed, and network operating systems.

5. Understand the concept of virtualization - how operating systems create the illusion of multiple virtual machines.

6. Remember that the OS must balance efficiency (maximum resource utilization) with user convenience (ease of use).

7. Know the basic components of process control blocks (PCB) and their purposes.

8. Understand the difference between primary memory and secondary storage management techniques.

9. Be familiar with examples of popular operating systems (Windows, Linux, macOS, Android) and their distinguishing features.

10. Know the evolution of operating systems from batch processing to modern distributed systems.
