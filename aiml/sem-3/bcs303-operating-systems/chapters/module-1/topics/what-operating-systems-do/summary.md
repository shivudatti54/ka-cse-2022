# What Operating Systems Do - Summary

## Key Definitions and Concepts

- **Operating System**: System software that acts as an intermediary between users and computer hardware, managing resources and providing services to applications

- **Process**: A program in execution, consisting of program code, current activity, and system resources

- **Process Control Block (PCB)**: Data structure maintained by the OS containing information about each process (state, PC, registers, memory info)

- **System Calls**: Programming interface allowing user programs to request services from the OS kernel

- **Resource Management**: OS function of allocating and managing hardware resources among competing processes

- **Virtual Memory**: Memory management technique allowing programs to use more memory than physically available

## Important Functions of Operating Systems

The five core functions are: Process Management (creating, scheduling, terminating processes), Memory Management (allocating/deallocating memory space), File Management (organizing, storing, retrieving data), Device Management (controlling I/O devices through drivers), and Security/Protection (protecting system resources from unauthorized access)

## Key Points

- Operating systems evolved from simple batch processors to complex multi-purpose systems

- The OS must balance efficiency (resource utilization) with user convenience (ease of use)

- Device drivers provide abstraction between hardware and software layers

- File systems organize data hierarchically in directories on secondary storage

- Process scheduling algorithms determine CPU allocation order (FCFS, SJF, Round Robin)

- Modern OSes implement virtual memory using paging and swapping

- User interfaces range from command-line (CLI) to graphical (GUI) to touch-based

- Operating systems provide security through authentication, permissions, and access control

## Common Mistakes to Avoid

- Confusing the OS with application software - OS is system software, not application software

- Thinking that all operating systems are the same - they differ significantly in design and purpose

- Overlooking the importance of device drivers in system functionality

- Neglecting security aspects when discussing OS functions
