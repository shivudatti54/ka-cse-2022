# **System Structures: What Operating Systems Do**

### Key Concepts

- **Operating Systems (OS)**: Software that manages computer hardware resources and provides common services to computer programs.
- **System Structures**: Components that make up an operating system.

### System Structures and Their Functions

- **Process Structure**:
  - Process: An independent program in execution.
  - Process Scheduling: Allocation of time to each process (FCFS, SJF, Priority Scheduling).
- **Memory Management**:
  - Virtual Memory: Combination of RAM and hard disk storage.
  - Paging and Segmenting: Memory allocation techniques.
- **File System**:
  - File Organization: Storage and retrieval of files.
  - File Protection: Access control and security.
- **Input/Output Management**:
  - Input Devices: Keyboard, mouse, scanner.
  - Output Devices: Printer, display, speakers.
  - I/O Scheduling: Allocation of I/O resources.
- **Network Management**:
  - Network Interface Cards (NICs).
  - Network Protocols: TCP/IP, HTTP, FTP.

### Important Formulas and Definitions

- **CPU Utilization Formula**: CPU Utilization = (Number of processes in ready queue) / (Total number of processors)
- **Page Fault Formula**: Page Fault = (Number of page faults) / (Total number of page faults + Number of page faults due to page faults)
- **Segment Fault Formula**: Segment Fault = (Number of segment faults) / (Total number of segment faults + Number of segment faults due to segment faults)

### Theorems

- **Hadamard's Theorem**: CPU utilization is between 0 and 1.
- **AM-HM Inequality**: The average waiting time in a queue is less than or equal to the harmonic mean of the waiting times.
