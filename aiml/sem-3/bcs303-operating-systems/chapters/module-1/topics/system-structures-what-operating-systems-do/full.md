# **System Structures: What Operating Systems Do**

## **Introduction**

Operating systems are software that manage computer hardware resources and provide common services to computer programs. The primary function of an operating system is to act as an intermediary between computer hardware and user-level applications. In this module, we will explore the different system structures that operating systems use to manage these resources and provide services to applications.

## **System Structures**

A system structure refers to the organization and interconnection of hardware and software components in a computer system. Operating systems use various system structures to manage resources and provide services to applications. The following are some common system structures used by operating systems:

### 1. **Hardware Abstraction Layer (HAL)**

The Hardware Abstraction Layer (HAL) is a layer of software that abstracts the underlying hardware platform from the operating system. The HAL provides a standardized interface for the operating system to interact with the hardware, regardless of the specific hardware platform.

### 2. **Interrupt Handling**

Interrupt handling is a system structure used by operating systems to manage interrupts generated by hardware components. When a hardware component generates an interrupt, the operating system must interrupt the current task and execute a specialized routine to handle the interrupt.

## **Case Studies**

### 1. **Windows Operating System**

Windows operating system uses a combination of hardware abstraction layer (HAL), interrupt handling, process management, memory management, file system management, interrupt-driven I/O, threading, and concurrency to manage resources and provide services to applications.

**Example Code (C++):**

```c
#include <windows.h>

int main() {
    // Create a new process
    HANDLE hProcess = CreateProcess(NULL, "notepad.exe", NULL, NULL, FALSE, 0, NULL, NULL);

    // Wait for the process to terminate
    WaitForSingleObject(hProcess, INFINITE);

    // Close the process handle
    CloseHandle(hProcess);

    return 0;
}
```

### 2. **Linux Operating System**

Linux operating system uses a combination of hardware abstraction layer (HAL), interrupt handling, process management, memory management, file system management, interrupt-driven I/O, threading, and concurrency to manage resources and provide services to applications.

**Example Code (C):**

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

int main() {
    // Create a new process
    pid_t pid = fork();

    if (pid == 0) {
        // Child process
        execl("/bin/notepad", "notepad", NULL);
    } else {
        // Parent process
        wait(NULL);
    }

    return 0;
}
```

## **Applications**

Operating systems provide various applications that use system structures to manage resources and provide services to applications. Some examples of applications include:

### 1. **Text Editors**

Text editors use file system management and memory management to manage files and buffers.

### 2. **Web Browsers**

Web browsers use file system management, memory management, and interrupt-driven I/O to manage web pages and graphics.

### 3. **Database Systems**

Database systems use file system management, memory management, and concurrency to manage data storage and retrieval.
