# Learning Objectives

After studying this topic, you should be able to:

1. Define and distinguish between uniprocessor, multiprocessor, and distributed system architectures
2. Explain the system components and the boundary between kernel mode and user mode
3. Classify resources as preemptible or non-preemptible, shareable or exclusive
4. Identify and explain the four necessary conditions for deadlock occurrence
5. Analyze system states to determine whether a system is in a safe or unsafe state
6. Apply the Banker's Algorithm to determine system safety
7. Understand how the system model influences synchronization and deadlock handling approaches
8. Design resource allocation strategies that prevent circular wait conditions
