# **Implementation of File System: File System: File Concept**

## **Definitions**

- **File**: A collection of related data stored in a specific location on a storage device.
- **File System**: A collection of files and the structure used to organize, manage, and retrieve them.
- **File Organization**: The process of organizing files into a hierarchical structure.

## **Key Concepts**

- **File Attributes**: File properties such as file name, file type, file size, and creation date.
- **File Types**: Types of files such as text files, image files, and executable files.
- **File Allocation**: The process of assigning storage space to a file.
- **File Allocation Table (FAT)**: A data structure used to store file information.
- **Inode**: A data structure that contains file metadata.
- **File System Hierarchy Standard (FSHFS)**: A standard for organizing files and directories.

## **Important Formulas and Theorems**

- **Bus Number Formula**: `bus_number = 2^i`, where `i` is the number of available bus.
- **Page Replacement Algorithm**: `LRU (Least Recently Used)` and `FIFO (First-In-First-Out)`.

## **Revision Notes**

- Files can be classified into two types: **regular files** and **special files**.
- A **file system** typically consists of the following components: **inodes, buffers, and file allocation tables**.
- The **inode** is a data structure that contains metadata about a file.
- The **file allocation table** is a data structure used to store file information.

## **Quick Revision Tips**

- Understand the different types of files and their properties.
- Know the structure and components of a file system.
- Familiarize yourself with file organization and file allocation concepts.
- Review the formulas and theorems related to file systems.
