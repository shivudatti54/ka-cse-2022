# Learning Objectives

After studying this topic, you should be able to:

1. Define a queue data structure and explain the FIFO (First In, First Out) principle that governs its behavior.

2. Identify and describe all queue operations including enqueue, dequeue, front, rear, isEmpty, and isFull with their correct functionality.

3. Implement a queue using array representation, including handling overflow and underflow conditions appropriately.

4. Implement a queue using linked list representation, demonstrating dynamic memory allocation and pointer manipulation.

5. Compare array-based and linked list-based queue implementations with respect to time complexity, space utilization, and advantages/disadvantages.

6. Analyze queue behavior by tracing operations and determining the state of front, rear, and queue contents after a sequence of operations.

7. Recognize and explain real-world applications of queues in operating systems, computer networks, and algorithms.

8. Apply queue data structure to solve problems like breadth-first search traversal and understand its role in problem-solving contexts.
