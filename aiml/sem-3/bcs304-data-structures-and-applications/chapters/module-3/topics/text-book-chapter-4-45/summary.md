# Revision Notes: Chapter-4.5: Data Structures and Applications

## Introduction

- Definition of Data Structures: A collection of data elements, each of which represents a value or objects.
- Importance of Data Structures: Efficient storage and retrieval of data.

## Types of Data Structures

- **Sequential Data Structures:**
  - Array
  - Linked List
- **Non-Sequential Data Structures:**
  - Stack
  - Queue
  - Tree
  - Graph

## Array

- **Formula:** `T[A][n] = {a1, a2, ..., an)`
- **Definition:** A collection of elements of the same data type stored in contiguous memory locations.
- **Operations:** Insertion, Deletion, Searching

## Linked List

- **Formula:** `L = Node (data, next)`
- **Definition:** A dynamic collection of elements, each of which is a separate object.
- **Operations:** Insertion, Deletion, Searching

## Stack

- **Formula:** `S = {a1, a2, ..., an}`
- **Definition:** A Last-In-First-Out (LIFO) data structure.
- **Operations:** Push, Pop, Peek

## Queue

- **Formula:** `Q = {a1, a2, ..., an}`
- **Definition:** A First-In-First-Out (FIFO) data structure.
- **Operations:** Enqueue, Dequeue, Peek

## Tree

- **Definition:** A non-linear data structure consisting of nodes.
- **Types:** Binary Tree, N-ary Tree
- **Operations:** Insertion, Deletion, Traversal

## Graph

- **Definition:** A non-linear data structure consisting of nodes and edges.
- **Types:** Directed Graph, Undirected Graph
- **Operations:** Insertion, Deletion, Traversal

## Important Formulas and Theorems

- **Big O Notation:** A measure of an algorithm's complexity.
- **Space Complexity:** The amount of space an algorithm uses.
- **Time Complexity:** The amount of time an algorithm takes.

## Key Concepts

- **Data Structure:** A collection of data elements.
- **Algorithm:** A set of instructions to solve a problem.
- **Complexity Analysis:** Analyzing the time and space complexity of an algorithm.
