# Introduction to Data Structures

## Overview

A data structure is a way to organize, store, and manage data in a computer for efficient access and modification. It is crucial for writing efficient programs, as the choice of data structure determines the time and space complexity of algorithms. Data structures provide a foundation for algorithms, allowing programmers to think at a higher level without worrying about low-level memory details.

## Key Points

- A data structure is a collection of data values, relationships, and operations that can be applied to the data.
- Data structures are essential for efficient programming, as they reduce time and space complexity.
- Abstract Data Types (ADTs) define the behavior of data structures from the user's perspective, without specifying implementation details.
- Data structures can be classified as primitive (int, float, char) or non-primitive (arrays, linked lists, stacks, queues, trees, graphs).
- The choice of data structure depends on the required operations, time and space constraints, and data size.

## Important Definitions

- **Data Structure**: A way to organize data for efficient access and modification.
- **Abstract Data Type (ADT)**: A logical description of data and operations, without specifying implementation details.
- **Primitive Data Types**: Basic data types provided by the programming language (int, float, char, double).
- **Non-Primitive Data Types**: Derived from primitive types, can hold collections of values (arrays, linked lists, stacks, queues, trees, graphs).

## Key Formulas / Syntax

- Big-O notation for time complexity (e.g., O(1), O(n), O(log n)).
- Common data structure operations (e.g., push, pop, peek, enqueue, dequeue).

## Comparisons

| Data Structure | Time Complexity      | Space Complexity |
| -------------- | -------------------- | ---------------- |
| Array          | O(1) access          | Fixed size       |
| Linked List    | O(n) access          | Dynamic size     |
| Stack          | O(1) push/pop        | LIFO order       |
| Queue          | O(1) enqueue/dequeue | FIFO order       |
| Tree           | O(log n) search      | Hierarchical     |
| Graph          | O(n) search          | Non-hierarchical |

## Exam Tips

- Be prepared to define key terms (data structure, ADT, primitive and non-primitive types).
- Practice drawing the classification diagram for data structures.
- Understand the difference between ADTs and data structures, with examples.
- Compare and contrast different data structures (arrays vs linked lists, stacks vs queues).
- Mention real-world examples and time complexity when discussing data structures.
