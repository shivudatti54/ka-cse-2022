# Sparse Matrices
A sparse matrix contains predominantly zero elements, defined by density $\delta = t/(mn) < 0.5$. Dense representations waste memory ($O(mn)$) and computation ($O(mn)$ operations). The triplet format stores only non-zero elements with indices, requiring $O(t)$ space and achieving $O(t)$ access time. Compressed Sparse Row (CSR) and Compressed Sparse Column (CSC) representations optimize row and column access patterns respectively, using $O(t+m)$ and $O(t+n)$ space. Sparse matrix transpose can be performed in $O(cols \times t)$ time (simple) or $O(t + cols)$ time (fast). Addition requires $O(t_A + t_B)$ time, while multiplication achieves $O(t_A \cdot t_B / p)$ complexity versus $O(mnp)$ for dense approaches. The choice of representation depends on access patterns: CSR for row-wise operations, CSC for column-wise operations.
