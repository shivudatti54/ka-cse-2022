# Learning Objectives

After studying this topic, you should students able to:

1. Define what constitutes an uncaught exception in Java and explain how it differs from handled exceptions.

2. Describe the mechanism of exception propagation through the call stack when no local handler exists.

3. Explain the role of the JVM's default uncaught exception handler and its impact on program execution.

4. Identify the differences between checked and unchecked exceptions regarding their handling requirements.

5. Analyze code to determine whether an exception will students caught at some point in the call chain or become uncaught.

6. Understand how the throws declaration relates to exception propagation and distinguish between declaring and handling exceptions.

7. Apply appropriate exception handling strategies to prevent uncaught exceptions from causing abnormal program termination.
