# Introducing Methods

## Overview

Methods in Java are blocks of code that perform a specific task and can be called multiple times from different parts of a program. They are used to organize and reuse code, making it more efficient and easier to maintain. Methods have various components that define their behavior and accessibility.

## Key Points

- Methods are declared using the `access_modifier`, `non_access_modifier`, `return_type`, `method_name`, and `parameter_list`.
- Access modifiers (`public`, `private`, `protected`) control the visibility of a method.
- Non-access modifiers (`static`, `final`, `abstract`) provide additional information about a method's behavior.
- The return type specifies the data type of the value returned by a method.
- The method name is a unique identifier for the method.
- The parameter list defines the input parameters of a method.

## Important Definitions

- **Method**: A block of code that performs a specific task and can be called multiple times.
- **Access Modifier**: A keyword that controls the visibility of a method.
- **Non-Access Modifier**: A keyword that provides additional information about a method's behavior.

## Key Syntax

- `access_modifier non_access_modifier return_type method_name (parameter_list) { // method body }`

## Exam Tips

- Be prepared to identify and explain the different components of a method declaration.
- Understand the purpose and behavior of access and non-access modifiers in method declarations.
