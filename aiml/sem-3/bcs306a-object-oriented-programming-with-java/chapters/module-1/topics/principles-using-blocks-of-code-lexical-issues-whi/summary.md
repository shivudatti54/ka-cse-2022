# **Principles), Using Blocks of Code, Lexical Issues (Whitespace, Identifiers, Literals, Comments, Separators, The Java Keywords**

## **Principles**

- **Principle of Encapsulation**: Data hiding and accessing through methods.
- **Single Responsibility Principle (SRP)**: A class should have only one reason to change.
- **Open-Closed Principle (OCP)**: A class should be open for extension but closed for modification.
- **Liskov Substitution Principle (LSP)**: Derived classes should be substitutable for their base classes.

## **Using Blocks of Code**

- **Blocks of Code**: Used for grouping statements together.
- **Code Blocks**: Can be nested inside other blocks.
- **Switch Statement**: Used for executing different blocks of code based on conditions.

## **Lexical Issues**

- **Whitespace**: Spaces, tabs, and newline characters are ignored by the compiler.
- **Identifiers**: Names of variables, methods, and classes.
  - **Keyword Identifiers**: `public`, `private`, `class`, `void`, etc.
  - **Namespace Identifiers**: `java.lang`, `com.example`, etc.
- **Literals**: Constants that represent values.
  - **Integer Literals**: `int x = 10;`
  - **String Literals**: `String name = "John";`
- **Comments**: Used for adding notes to code.
  - **Single-Line Comment**: `// comment`
  - **Multi-Line Comment**: `/* comment */`
- **Separators**: Used to separate tokens in a program.
  - **Space**: Ignored by the compiler.
  - **Special Characters**: `;`, `}`, `{`, `,`, etc.

## **Java Keywords**

- **Reserved Words**: `public`, `private`, `class`, `void`, etc.
- **Primary Keywords**: `if`, `else`, `while`, `for`, etc.
- **Secondary Keywords**: `try`, `catch`, `finally`, etc.
