# **Chapter 2: Object-Oriented Programming with JAVA**

## **2.1 Introduction to Object-Oriented Programming**

Object-Oriented Programming (OOP) is a programming paradigm that revolves around the concept of objects and classes. OOP is based on the idea that real-world objects can be modeled as software entities, and that these objects can be manipulated using a set of well-defined rules and principles.

## **2.2 Historical Context of Object-Oriented Programming**

The concept of OOP has its roots in the 1960s, when the term "object-oriented" was first coined by Alan Kay, a computer scientist who worked at Xerox PARC. Kay's vision was to create a programming language that would allow developers to create reusable, modular, and flexible code.

In the 1970s, the first OOP languages were developed, including Simula and Smalltalk. These languages introduced key concepts such as classes, objects, inheritance, and polymorphism.

## **2.3 Two Paradigms of Object-Oriented Programming**

There are two main paradigms of OOP: **Imperative** and **Object-Oriented**.

### 2.3.1 Imperative Paradigm

The Imperative paradigm is based on the idea of specifying what the program should do, rather than how it should do it. This paradigm is based on the concept of procedures and functions, which are used to perform specific tasks.

### 2.3.2 Object-Oriented Paradigm

The Object-Oriented paradigm is based on the idea of modeling real-world objects and their interactions using classes and objects. This paradigm introduces key concepts such as inheritance, polymorphism, and encapsulation.

## **2.4 Key Concepts of Object-Oriented Programming**

### 2.4.1 Classes

A class is a blueprint for creating objects. A class defines the properties and behavior of an object, and is used to create multiple objects that have the same characteristics.

### 2.4.2 Objects

An object is an instance of a class. An object has its own set of attributes (data) and methods (functions) that are defined in the class.

### 2.4.3 Inheritance

Inheritance is the process of creating a new class based on an existing class. The new class inherits the properties and behavior of the existing class, and can also add new properties and behavior.

### 2.4.4 Polymorphism

Polymorphism is the ability of an object to take on multiple forms. This can be achieved through method overriding or method overloading.

### 2.4.5 Encapsulation

Encapsulation is the concept of hiding the internal details of an object from the outside world, while still exposing the necessary information through public methods.

## **2.5 Abstraction**

Abstraction is the process of showing only the necessary information to the outside world, while hiding the internal details. Abstraction is a key concept in OOP, as it allows developers to create complex systems that are easy to understand and use.

## **2.6 The Three OOP Principles**

The Three OOP Principles are:

### 2.6.1 Single Responsibility Principle

The Single Responsibility Principle states that a class should have only one reason to change. This means that a class should have a single purpose or responsibility, and should not be responsible for multiple, unrelated tasks.

### 2.6.2 Open-Closed Principle

The Open-Closed Principle states that a class should be open for extension, but closed for modification. This means that a class should be able to be extended or modified without changing its underlying structure.

### 2.6.3 Liskov Substitution Principle

The Liskov Substitution Principle states that derived classes should be substitutable for their base classes. This means that a derived class should be able to replace its base class without affecting the correctness of the program.

## **2.7 Case Study: Bank Account System**

A bank account system is a classic example of OOP in action. The system can be modeled using classes and objects, with each class representing a different entity in the system.

- **Account Class**: This class represents a bank account, and has attributes such as account number, balance, and account holder.
- **Customer Class**: This class represents a customer, and has attributes such as customer ID, name, and address.
- **Transaction Class**: This class represents a transaction, and has attributes such as transaction ID, account number, and transaction type.

The system can be modeled using the following relationships:

- A customer can have multiple accounts.
- An account is owned by one customer.
- A transaction is performed on one account.

## **2.8 Application of OOP in Real-World Systems**

OOP has numerous applications in real-world systems, including:

- **Operating Systems**: OOP is used to model the behavior of operating systems, including processes, threads, and memory management.
- **Database Systems**: OOP is used to model the behavior of database systems, including tables, rows, and queries.
- **Web Applications**: OOP is used to model the behavior of web applications, including user interfaces, business logic, and data models.

## **2.9 Modern Developments in OOP**

Modern developments in OOP include:

- **Aspect-Oriented Programming**: This paradigm focuses on separating cross-cutting concerns, such as logging and security, from the main program logic.
- **Functional Programming**: This paradigm focuses on the evaluation of expressions and the use of pure functions to avoid side effects.
- **Type Checking**: This technique is used to ensure that the types of variables and expressions are correctly specified at compile-time, rather than at runtime.
