# Chapter 6 Summary

### Class Fundamentals

- A **class** is a blueprint or template for creating objects.
- A class defines the properties (data) and methods (functions) of an object.
- A class can have multiple objects.

### Declaring Classes

- A **class declaration** includes the access modifier, class name, and parentheses.
- Access modifiers: `public`, `private`, `protected`
- Class name: a unique identifier for the class.
- Parentheses: used to specify the parameters of the constructor.

### Constructors

- A **constructor** is a special method that is used to initialize objects.
- A constructor has the same name as the class and no return type.
- A constructor is called when an object is created.

### Class Variables and Methods

- **Class variables** are shared by all objects of the class.
- **Instance variables** are unique to each object of the class.
- **Static methods** belong to the class itself, not to instances of the class.
- **Instance methods** belong to instances of the class.

### Object Reference Variables

- An **object reference variable** is a variable that holds a reference to an object.
- An object reference variable can hold a reference to any object of the same class.

### Important Formulas and Definitions

- **Abstraction**: hiding the implementation details of an object from the outside world.
- **Encapsulation**: bundling data and methods that operate on that data within a single unit.
- **Inheritance**: creating a new class based on an existing class.

### Key Theorems

- **Single Responsibility Principle (SRP)**: a class should have only one reason to change.
- **Open-Closed Principle (OCP)**: a class should be open for extension but closed for modification.
