# Lists and Tuples Revision Notes

=====================================

## Lists

---

### Definition

- A list is a collection of items, which can be of any data type, including strings, integers, floats, and other lists.
- Lists are ordered and can have duplicate values.

### Operations on Lists

- **Indexing**: Accessing an element by its index (e.g., `my_list[0]`).
- **Slicing**: Extracting a subset of elements from the list (e.g., `my_list[1:3]`).
- **Append**: Adding an element to the end of the list (e.g., `my_list.append(10)`).
- **Insert**: Inserting an element at a specific position in the list (e.g., `my_list.insert(2, 20)`).
- **Remove**: Removing the first occurrence of an element from the list (e.g., `my_list.remove(10)`).
- **Sort**: Sorting the list in ascending or descending order (e.g., `my_list.sort()`).
- **Reverse**: Reversing the order of the list (e.g., `my_list.reverse()`).

### Important Formulas/Definitions/Theorems

- None specific to lists, but understanding of indexing, slicing, and operations is crucial.

## Tuples

---

### Definition

- A tuple is a collection of items, which can be of any data type, including strings, integers, floats, and other tuples.
- Tuples are ordered and immutable, meaning they cannot be changed after creation.

### Creating Tuples

- Using parentheses `()` (e.g., `(1, 2, 3)`).
- Using the `tuple()` function (e.g., `tuple([1, 2, 3])`).

### Indexing and Slicing Tuples

- Indexing: Accessing an element by its index (e.g., `my_tuple[0]`).
- Slicing: Extracting a subset of elements from the tuple (e.g., `my_tuple[1:3]`).

### Operations on Tuples

- **Indexing**: Accessing an element by its index (e.g., `my_tuple[0]`).
- **Slicing**: Extracting a subset of elements from the tuple (e.g., `my_tuple[1:3]`).
- **Concatenation**: Combining two or more tuples into a single tuple (e.g., `my_tuple1 + my_tuple2`).

### Important Formulas/Definitions/Theorems

- None specific to tuples, but understanding of indexing, slicing, and operations is crucial.
