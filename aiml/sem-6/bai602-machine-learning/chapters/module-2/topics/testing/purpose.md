# Learning Objectives
After studying this topic, you should be able to:
1.  Differentiate between Static (SAST), Dynamic (DAST), and Interactive (IAST) Application Security Testing methodologies in the context of mobile apps.
2.  Identify and describe the purpose of at least three common tools used for mobile app security testing (e.g., MobSF, Burp Suite, Frida).
3.  Explain the importance of using both emulators/simulators and rooted/jailbroken physical devices in a testing strategy.
4.  Map specific mobile security testing tools to the vulnerabilities listed in the OWASP Mobile Top 10.
5.  Describe the process of using a proxy tool to intercept and analyze mobile app network traffic.
6.  Propose an integrated tooling strategy for incorporating security testing into a mobile application CI/CD pipeline.
