# Learning Objectives
After studying this topic, you should be able to:
1.  Explain the limitations of traditional centralized Intrusion Detection Systems (IDS) in a cloud computing environment.
2.  Describe the architecture and key components (HIDS, NIDS, Hypervisor-based, GIDM) of a Distributed Intrusion Detection System (DIDS).
3.  Differentiate between signature-based (misuse) and anomaly-based detection techniques, including their advantages and disadvantages.
4.  Identify and discuss the major challenges associated with implementing a DIDS in the cloud (e.g., data overhead, VM mobility, correlation).
5.  Compare and contrast cooperative and cloud provider-centric deployment models for DIDS.
6.  Evaluate a real-world cloud security service (e.g., AWS GuardDuty) and map its features to DIDS concepts.
