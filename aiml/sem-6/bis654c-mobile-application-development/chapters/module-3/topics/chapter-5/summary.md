## Chapter 5 Revision Notes

### Mobile Application Development

#### Introduction

- Definition: A mobile application (app) is a computer program designed to run on mobile devices such as smartphones and tablets.

#### Android App Development

- **Intent**: A message sent by an app to request an action from another app or system.
- **Activity**: A single screen in an Android app that performs a specific task.
- **Service**: A long-running process in an Android app that performs a specific task.

#### iOS App Development

- **ViewController**: The central class in an iOS app that handles user input and updates the app's UI.
- **Delegate**: An object that conforms to a specific protocol and receives messages from a class.
- **Storyboard**: A visual representation of an app's UI that defines the layout and behavior of views.

#### Cross-Platform App Development

- **Hybrid Apps**: Apps built using a combination of native and web technologies.
- **React Native**: A framework for building cross-platform apps using JavaScript and React.

#### App Testing and Debugging

- **Unit Testing**: Testing individual components or units of code.
- **Integration Testing**: Testing the interaction between multiple components or units of code.
- **Debugging Techniques**: Using tools such as debuggers and print statements to identify and fix errors.

#### App Security

- **Encryption**: Protecting data by converting it into an unreadable format.
- **Authentication**: Verifying the identity of users or devices.
- **Authorization**: Controlling access to app resources and data.

#### App Publishing and Maintenance

- **App Store**: A digital distribution platform for apps.
- **App Store Optimization**: Optimizing an app's visibility and ranking in the app store.
- **App Updates**: Updating an app to fix bugs, add features, and improve performance.

Formulas, Definitions, and Theorems:

- **Mobile App Development Life Cycle**: A series of stages that an app goes through from development to deployment.
- **Software Development Methodology**: A framework for planning, designing, and delivering software applications.
