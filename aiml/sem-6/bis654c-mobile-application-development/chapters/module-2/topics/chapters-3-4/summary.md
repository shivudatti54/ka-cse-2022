# **MOBILE APPLICATION DEVELOPMENT**

### Directory Structure

---

- **Project Structure:**
  - `app`: Android Application Package
  - `src`: Source Code Directory
  - `res`: Resource Directory
  - `android`: Android-Specific Code
- **Package Declaration:**
  - `package com.example.appname;` (Replace with your app's package name)

### Android User Interface

---

- **Layout Files:**
  - `.xml` files (e.g., `activity_main.xml`)
  - ` inflate()` method to load layout files
- **View Hierarchy:**
  - `View` class (e.g., `TextView`, `Button`)
  - `(FrameLayout`, `LinearLayout`, `RelativeLayout`) for layout management
- **User Interface Components:**
  - `EditText`, `CheckBox`, `RadioButton`, etc.

### Important Formulas, Definitions, Theorems

---

- **AndroidManifest.xml**: Definition of an application's metadata and permissions.
- **Intent**: Definition of an application's request to perform a specific action.
- **Activity**: Definition of an application's main interface.
- **Fragment**: Definition of a reusable UI component.

## **Revision Key Points**

- Create a project structure with `app`, `src`, and `res` directories.
- Declare a package name for your Android application.
- Use layout files to design the user interface.
- Understand the concept of view hierarchy and user interface components.
- Familiarize yourself with AndroidManifest.xml, Intent, Activity, and Fragment.
