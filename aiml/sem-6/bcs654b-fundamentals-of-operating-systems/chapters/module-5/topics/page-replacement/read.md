# Page Replacement Algorithms

## Introduction to Page Replacement

In a virtual memory system using demand paging, processes execute with only some of their pages in physical memory. When a process references a page that isn't currently in memory (a **page fault**), the operating system must bring that page from secondary storage (disk) into a free frame in physical memory.

However, if **no free frames are available** when a page fault occurs, the OS must select a page currently in memory to **evict** (remove) to make space for the new page. The algorithm used to select which page to remove is called a **page replacement algorithm**.

The goal of page replacement algorithms is to minimize the **page fault rate** - the frequency at which page faults occur. A good algorithm should avoid replacing pages that will be needed again soon.

## Basic Page Replacement Process

When a page fault occurs and no free frame exists:

1. Find the location of the desired page on disk
2. Find a free frame:
   - If there's a free frame, use it
   - If no free frame exists, use a page replacement algorithm to select a **victim frame**
3. Write the victim page to disk if it has been modified (set the **dirty bit**)
4. Bring the desired page into the newly freed frame
5. Update the page table and frame table
6. Restart the instruction that caused the page fault

```
+---------------------+     Page Fault     +-----------------------+
| Process References  | -----------------> | Check if Page is in   |
| Page Not in Memory  |                    | Memory (Page Table)   |
+---------------------+                    +-----------------------+
                                                  |
                                                  | Not Found
                                                  v
                                    +-------------------------------+
                                    | Is there a Free Frame?        |
                                    +-------------------------------+
                                                  |
                                      +-----------+-----------+
                                      | Yes                   | No
                                      v                       v
                            +-------------------+   +-----------------------+
                            | Use Free Frame   |   | Select Victim Frame   |
                            +-------------------+   | Using Page Replacement|
                                                    | Algorithm            |
                                                    +-----------------------+
                                                                 |
                                                                 v
                                                +---------------------------------+
                                                | Write Victim to Disk if Modified|
                                                | (Check Dirty Bit)               |
                                                +---------------------------------+
                                                                 |
                                                                 v
                                                +---------------------------------+
                                                | Bring Requested Page into Frame|
                                                +---------------------------------+
                                                                 |
                                                                 v
                                                +---------------------------------+
                                                | Update Page Table and Restart   |
                                                | Instruction                     |
                                                +---------------------------------+
```

## Key Concepts in Page Replacement

### Reference String

A **reference string** is the sequence of page numbers generated by a process during execution. It represents the memory access pattern of the process and is used to analyze page replacement algorithms.

Example: For a process accessing pages: 1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5
The reference string would be: 1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5

### Page Fault Calculation

For each algorithm, we calculate:
- **Page faults**: Number of times a requested page isn't in memory
- **Page fault rate**: Page faults ÷ Total references

### Dirty Bit

The **dirty bit** (or modify bit) is a flag associated with each page that indicates whether the page has been modified since it was loaded into memory. If a page has been modified (dirty bit = 1), it must be written back to disk when replaced. If not modified (dirty bit = 0), it can simply be overwritten.

## Page Replacement Algorithms

### 1. First-In-First-Out (FIFO)

**Concept**: Replace the page that has been in memory the longest (the first page that came in).

**Implementation**: Maintain a queue of pages in the order they were loaded. When replacement is needed, remove the page at the front of the queue.

**Example**: Consider reference string: 7, 0, 1, 2, 0, 3, 0, 4, 2, 3, 0, 3, 2, 1, 2, 0, 1, 7, 0, 1
With 3 frames:

```
Reference: 7, 0, 1, 2, 0, 3, 0, 4, 2, 3, 0, 3, 2, 1, 2, 0, 1, 7, 0, 1
Frames: 
7: 7         (Fault)
0: 7, 0      (Fault)
1: 7, 0, 1   (Fault)
2: 0, 1, 2   (Fault - replace 7)
0: 0, 1, 2   (Hit)
3: 1, 2, 3   (Fault - replace 0)
0: 2, 3, 0   (Fault - replace 1)
4: 3, 0, 4   (Fault - replace 2)
2: 0, 4, 2   (Fault - replace 3)
3: 4, 2, 3   (Fault - replace 0)
0: 2, 3, 0   (Fault - replace 4)
3: 2, 3, 0   (Hit)
2: 2, 3, 0   (Hit)
1: 3, 0, 1   (Fault - replace 2)
2: 0, 1, 2   (Fault - replace 3)
0: 0, 1, 2   (Hit)
1: 0, 1, 2   (Hit)
7: 1, 2, 7   (Fault - replace 0)
0: 2, 7, 0   (Fault - replace 1)
1: 7, 0, 1   (Fault - replace 2)

Total page faults: 15
```

**Advantages**: Simple to implement
**Disadvantages**: Poor performance - doesn't consider frequency of use (may replace frequently used pages)

**Belady's Anomaly**: With FIFO, increasing the number of frames can sometimes increase the page fault rate.

### 2. Optimal Page Replacement (OPT)

**Concept**: Replace the page that will not be used for the longest period in the future.

**Implementation**: Requires knowledge of the future reference string. For each page in memory, determine when it will be referenced next. Replace the page with the longest time until its next reference.

**Example**: Same reference string with 3 frames:

```
Reference: 7, 0, 1, 2, 0, 3, 0, 4, 2, 3, 0, 3, 2, 1, 2, 0, 1, 7, 0, 1
Frames:
7: 7         (Fault)
0: 7, 0      (Fault)
1: 7, 0, 1   (Fault)
2: 0, 1, 2   (Fault - replace 7, which won't be used again soon)
0: 0, 1, 2   (Hit)
3: 0, 1, 3   (Fault - replace 2, which will be used later than 0 or 1)
0: 0, 1, 3   (Hit)
4: 0, 1, 4   (Fault - replace 3, which will be used later than 0 or 1)
2: 0, 1, 2   (Fault - replace 4, which won't be used again)
3: 0, 1, 3   (Fault - replace 2, which will be used later than 0 or 1)
0: 0, 1, 3   (Hit)
3: 0, 1, 3   (Hit)
2: 0, 3, 2   (Fault - replace 1, which won't be used as soon as 0 or 3)
1: 0, 3, 1   (Fault - replace 2, which won't be used as soon as 0 or 3)
2: 0, 1, 2   (Fault - replace 3, which won't be used as soon as 0 or 1)
0: 0, 1, 2   (Hit)
1: 0, 1, 2   (Hit)
7: 0, 1, 7   (Fault - replace 2, which won't be used as soon as 0 or 1)
0: 0, 1, 7   (Hit)
1: 0, 1, 7   (Hit)

Total page faults: 9
```

**Advantages**: Lowest possible page fault rate for any fixed number of frames
**Disadvantages**: Impossible to implement in practice (requires future knowledge) - used as a benchmark for comparing other algorithms

### 3. Least Recently Used (LRU)

**Concept**: Replace the page that has not been used for the longest time.

**Implementation**: Track the time of last use for each page. Replace the page with the oldest timestamp.

**Example**: Same reference string with 3 frames:

```
Reference: 7, 0, 1, 2, 0, 3, 0, 4, 2, 3, 0, 3, 2, 1, 2, 0, 1, 7, 0, 1
Frames:
7: 7         (Fault)     [7]
0: 7, 0      (Fault)     [7, 0]
1: 7, 0, 1   (Fault)     [7, 0, 1]
2: 0, 1, 2   (Fault - replace 7, LRU)     [0, 1, 2]
0: 0, 1, 2   (Hit)       [1, 2, 0]
3: 1, 2, 3   (Fault - replace 0, LRU)     [2, 3, 1]
0: 2, 3, 0   (Fault - replace 1, LRU)     [3, 0, 2]
4: 3, 0, 4   (Fault - replace 2, LRU)     [0, 4, 3]
2: 0, 4, 2   (Fault - replace 3, LRU)     [4, 2, 0]
3: 4, 2, 3   (Fault - replace 0, LRU)     [2, 3, 4]
0: 2, 3, 0   (Fault - replace 4, LRU)     [3, 0, 2]
3: 3, 0, 2   (Hit)       [0, 2, 3]
2: 0, 2, 3   (Hit)       [0, 3, 2]
1: 0, 3, 1   (Fault - replace 2, LRU)     [3, 1, 0]
2: 3, 1, 2   (Fault - replace 0, LRU)     [1, 2, 3]
0: 1, 2, 0   (Fault - replace 3, LRU)     [2, 0, 1]
1: 2, 0, 1   (Hit)       [2, 0, 1]
7: 2, 0, 7   (Fault - replace 1, LRU)     [0, 7, 2]
0: 0, 7, 2   (Hit)       [7, 2, 0]
1: 7, 2, 1   (Fault - replace 0, LRU)     [2, 1, 7]

Total page faults: 12
```

**Implementation Approaches**:
- **Counters**: Each page has a time-of-use field updated on every reference
- **Stack**: Maintain a stack of page numbers - most recently used at top, LRU at bottom

**Advantages**: Good performance, approximates OPT
**Disadvantages**: Requires hardware support for accurate implementation, overhead of tracking usage

### 4. LRU Approximation Algorithms

Since true LRU requires significant hardware support, several approximations are used:

#### Additional Reference Bits Algorithm
Periodically shift the reference bit into an 8-bit byte for each page. The page with the smallest value is the LRU approximation.

#### Second-Chance Algorithm (Clock Algorithm)
**Concept**: A circular list of pages with a reference bit. When a page is referenced, set its reference bit to 1. When replacement is needed:
1. Check the next page in the circular list
2. If reference bit = 0, select for replacement
3. If reference bit = 1, set it to 0 and move to next page

```
     +-------------------+
     | Start at current  |
     | pointer position |
     +-------------------+
              |
              v
+------------------------------+
| Check reference bit of page |
| at current position         |
+------------------------------+
              |
    +---------+---------+
    | 0                | 1
    v                  v
+-----------+    +-----------------+
| Replace   |    | Set reference   |
| this page |    | bit to 0        |
+-----------+    +-----------------+
    |                  |
    v                  v
+-----------+    +-----------------+
| Update    |    | Move to next    |
| pointer   |    | page in circle   |
+-----------+    +-----------------+
```

#### Enhanced Second-Chance Algorithm
Uses both reference bit and modify (dirty) bit to classify pages into four categories:
1. (0, 0) - not recently used, not modified (best to replace)
2. (0, 1) - not recently used, modified (need to write to disk)
3. (1, 0) - recently used, not modified (might be used again)
4. (1, 1) - recently used, modified (worst to replace)

### 5. Counting Algorithms

**Least Frequently Used (LFU)**: Replace the page with the smallest reference count
**Most Frequently Used (MFU)**: Replace the page with the largest reference count (based on theory that page with small count was probably just brought in)

## Comparison of Algorithms

| Algorithm | Implementation Complexity | Requires Future Knowledge | Performance | Practical Use |
|-----------|---------------------------|---------------------------|-------------|---------------|
| FIFO      | Simple                    | No                        | Poor        | Rare          |
| OPT       | Simple but impossible     | Yes                       | Optimal     | Benchmark only |
| LRU       | Complex                   | No                        | Very Good   | Common        |
| Clock     | Moderate                  | No                        | Good        | Very Common   |
| LFU/MFU   | Moderate                  | No                        | Fair        | Occasionally  |

## Performance Considerations

The effectiveness of a page replacement algorithm depends on the **memory access pattern** of processes. Some patterns to consider:

- **Locality of reference**: Processes tend to access a small set of pages frequently for a period of time
- **Sequential access**: Processes accessing pages in sequence
- **Random access**: Processes accessing pages with no predictable pattern

## Exam Tips

1. **Practice with examples**: Be able to trace through reference strings for each algorithm
2. **Understand Belady's Anomaly**: Only FIFO exhibits this behavior where more frames can lead to more page faults
3. **Know the dirty bit**: Remember that modified pages must be written to disk when replaced
4. **Compare algorithms**: Be prepared to discuss trade-offs between different algorithms
5. **Real-world implementation**: Clock algorithm is most commonly used in practice due to good performance and reasonable overhead
6. **Calculate page fault rates**: Practice calculating page faults for different algorithms and frame counts
