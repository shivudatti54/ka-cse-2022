### Revision Notes for Chapter 5.1: Fundamentals of Operating Systems

#### Key Concepts

- **Definition:** An operating system (OS) is a software that manages computer hardware resources and provides common services to computer programs.
- **Functions of an Operating System:**
  - Process management
  - Memory management
  - File management
  - Input/Output management
  - Security and protection
  - Networking
- **Types of Operating Systems:**
  - Single-user, single-tasking
  - Single-user, multi-tasking
  - Multi-user, single-tasking
  - Multi-user, multi-tasking

#### Important Formulas and Theorems

- **Bus Timing Theorem:** The minimum time required to access memory is equal to the time it takes to fetch an instruction from memory.
- **Memory Hierarchy:** Memory is organized in a hierarchical structure, with smaller, faster memories (e.g. registers, cache) serving as a buffer for larger, slower memories (e.g. main memory).

#### Key Terms

- **Bootloader:** A small program that loads the operating system into memory.
- **Boot sector:** A sector of a disk that contains the loader code.
- **Interrupt:** A signal sent by hardware to the operating system to request attention.
- **Paging:** A memory management technique that divides memory into fixed-size blocks (frames) and maps them to memory locations.

#### Key Definitions

- **Process:** A program in execution.
- **Thread:** A flow of instructions within a process.
- **Kernel:** The core part of the operating system, responsible for managing hardware resources.
- **Device driver:** Software that manages interaction between the operating system and hardware devices.
