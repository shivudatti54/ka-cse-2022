# Continuous Internal Evaluation - Summary

## Key Definitions

- **Continuous Internal Evaluation (CIE)**: A systematic assessment method that evaluates student performance throughout an academic semester using multiple evaluation components rather than relying solely on final examinations.

- **Formative Assessment**: Ongoing evaluation aimed at providing feedback to improve learning, typically including quizzes, assignments, and lab work.

- **Summative Assessment**: Evaluation at the end of an instructional unit, usually through formal examinations.

## Important Formulas

- **Average Waiting Time**: Sum of waiting times of all processes divided by number of processes

- **Turnaround Time**: Completion time minus arrival time

- **Throughput**: Number of processes completed per unit time

- **CPU Utilization**: (Busy Time / Total Time) × 100%

## Key Points

- CIE in OS courses typically includes written tests (40-50%), practical assignments (30-40%), and project work (10-20%).

- Core OS topics assessed include process management, CPU scheduling, memory management, deadlock handling, and file systems.

- Practical assessments require implementation of OS concepts like semaphores, page replacement algorithms, and process synchronization.

- Evaluation rubrics assess both conceptual understanding and practical application skills.

- Regular self-assessment helps identify knowledge gaps before formal evaluations.

- Understanding OS terminology is essential for performing well in written assessments.

- Drawing diagrams (Gantt charts, state diagrams) strengthens answers in examinations.

## Common Mistakes

1. **Neglecting numerical practice**: Many students focus only on theoretical concepts and fail to practice scheduling and memory management calculations.

2. **Ignoring practical components**: Underestimating the importance of lab assignments and programming tasks leads to poor overall performance.

3. **Inconsistent study habits**: Treating CIE as a single end-of-semester event rather than ongoing assessment creates revision pressure.

4. **Poor time management**: Failing to allocate time properly during examinations results in incomplete answer sheets.
