# **6.1 to 6.3: Introduction to Data Structures**

## **6.1: Types of Data Structures**

- **Sequential Data Structures**: Linear data structures, such as arrays and linked lists.
- **Non-Linear Data Structures**: Trees, graphs, and heaps.
- **Composite Data Structures**: Arrays of structures, structures of arrays.

### 6.1.1: Arrays

---

- **Definition**: A collection of elements of the same data type stored in contiguous memory locations.
- **Example**: `int[] array = new int[5];`
- **Operations**: Indexing, assignment, and iteration.

### 6.1.2: Linked Lists

---

- **Definition**: A sequence of nodes, each containing a value and a reference (link) to the next node.
- **Example**: `Node node = new Node(5, nodeNext);`
- **Operations**: Insertion, deletion, and traversal.

### 6.1.3: Stacks and Queues

---

- **Stacks**: Last-In-First-Out (LIFO) data structure.
- **Queues**: First-In-First-Out (FIFO) data structure.
- **Operations**: Push, pop, peek, and enqueue/dequeue.

## **6.2: Trees and Graphs**

### 6.2.1: Trees

---

- **Definition**: A non-linear data structure consisting of nodes with a value and references to children.
- **Types**: Binary trees, AVL trees, and BSTs.
- **Operations**: Insertion, deletion, and traversal.

### 6.2.2: Graphs

---

- **Definition**: A non-linear data structure consisting of nodes and edges.
- **Types**: Directed and undirected graphs, weighted and unweighted graphs.
- **Operations**: Traversal and shortest path finding.

### 6.2.3: Heaps

---

- **Definition**: A specialized tree-based data structure that satisfies the heap property.
- **Operations**: Insertion, deletion, and heapify.

## **6.3: Data Structure Operations**

- **Insertion**: Adding an element to a data structure.
- **Deletion**: Removing an element from a data structure.
- **Traversal**: Visiting each element in a data structure.

## **Important Formulas and Definitions**

- **Big O Notation**: A measure of the time or space complexity of an algorithm.
- **Space Complexity**: The amount of memory an algorithm requires.

## **Theorems**

- **Master Theorem**: A tool for solving recurrence relations.

## **Quick Revision Tips**

- Review the definitions and properties of each data structure.
- Practice solving problems using different data structures.
- Focus on understanding the trade-offs between different data structures.
