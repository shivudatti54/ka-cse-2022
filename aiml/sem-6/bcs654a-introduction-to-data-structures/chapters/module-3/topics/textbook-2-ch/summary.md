### Textbook 2: Ch

#### Linked Lists: Introduction, Singly Linked List, Self-Referential Structures, Operations on

### Key Concepts

- **Definition**: A linked list is a data structure in which elements are linked together through pointers.
- **Types of Linked Lists**:
  - Singly Linked List: Each element points to the next element.
  - Doubly Linked List: Each element points to both the previous and next element.
  - Self-Referential Structures: A linked list that contains references to itself.
- **Linked List Operations**:
  - Insertion: Adding a new element to the list.
  - Deletion: Removing an element from the list.
  - Search: Finding an element in the list.
  - Traversal: Visiting each element in the list.
- **Common Operations**:
  - Adding a new node to the end of the list: `addNode(head, node)`.
  - Removing the first occurrence of a node with a given key: `removeNode(head, key)`.
  - Finding the node with a given key: `findNode(head, key)`.
- **Important Formulas and Theorems**:
  - Time complexity for inserting a new node: O(1) amortized.
  - Time complexity for deleting a node: O(1) amortized.
  - Space complexity for storing a linked list: O(n).

### Key Formulas and Definitions

- **Node**: A single element in a linked list, containing a value and a reference to the next node.
- **Head**: The first node in a linked list.
- **Tail**: The last node in a linked list.
- **Next**: The reference to the next node in a linked list.
- **Previous**: The reference to the previous node in a doubly linked list.
- **Length**: The number of nodes in a linked list.

### Theorems

- **Worst-case Time Complexity**: The maximum time required to complete an operation on a linked list.
- **Best-case Time Complexity**: The minimum time required to complete an operation on a linked list.
