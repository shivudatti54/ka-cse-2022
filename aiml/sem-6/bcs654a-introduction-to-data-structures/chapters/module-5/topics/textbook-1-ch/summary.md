### Introduction to Data Structures

- Data structure: A collection of data elements whose elements are stored in a way that allows for efficient access, modification, and retrieval.
- Types of data structures: Arrays, Linked Lists, Stacks, Queues, Trees, Graphs.

### Sorting: Introduction

- Sorting: The process of arranging data in a specific order, usually in ascending or descending order.
- Types of Sorting Algorithms:
  - Bubble Sort
  - Selection Sort
  - Insertion Sort

### Sorting Algorithms

#### Bubble Sort

- Algorithm: Swap adjacent elements if they are in the wrong order.
- Time complexity: O(n^2)
- Space complexity: O(1)
- Example:
  - `arr = [5, 2, 8, 1, 9]`
  - `arr = [1, 2, 5, 8, 9]`

#### Selection Sort

- Algorithm: Select the smallest (or largest) element and swap it with the first element.
- Time complexity: O(n^2)
- Space complexity: O(1)
- Example:
  - `arr = [5, 2, 8, 1, 9]`
  - `arr = [1, 2, 5, 8, 9]`

#### Insertion Sort

- Algorithm: Insert each element into its proper position in the sorted portion of the array.
- Time complexity: O(n^2)
- Space complexity: O(1)
- Example:
  - `arr = [5, 2, 8, 1, 9]`
  - `arr = [1, 2, 5, 8, 9]`

### Important Formulas and Definitions

- **Time complexity**: A measure of the amount of time an algorithm takes to complete, usually expressed as a function of the size of the input.
- **Space complexity**: A measure of the amount of memory an algorithm uses, usually expressed as a function of the size of the input.
- **Big O notation**: A way to describe the upper bound of an algorithm's time or space complexity.

### Theorems

- **Stirling's Accumulation**: A measure of the number of comparisons required by an algorithm.
- **Cocktail Sort**: A variation of Bubble Sort that sorts in both directions.
