# **Cloud Computing & Security Revision Notes: 11.5 to 11.8**

### Key Concepts

- **Cloud Data Security**: The process of protecting cloud data from unauthorized access, use, disclosure, modification, or destruction.
- **Cloud Security Controls**: A set of measures implemented to prevent, detect, and respond to security threats in the cloud.
- **Risk Management**: The process of identifying, assessing, and mitigating risks to the cloud infrastructure and data.

### Key Formulas and Definitions

- **Cloud Security Threat Modeling**: A method for identifying potential security threats to the cloud infrastructure using a structured approach.
  - Threat Model: A representation of the potential security threats to the cloud infrastructure.
  - Threat Description: A detailed description of the potential security threat.
- **Cloud Security Architecture**: A design for the security controls and measures implemented to protect the cloud infrastructure and data.
  - Security Requirements: A set of requirements for the security controls and measures implemented.
  - Security Interoperability: The ability of different security controls and measures to work together seamlessly.

### Theorems and Concepts

- **Zero Trust Architecture**: A security model that assumes that all users and devices are untrusted and verifies their identity and permissions before granting access to the cloud infrastructure.
- **Least Privilege Principle**: A security principle that states that users and devices should be granted the minimum level of access and privileges necessary to perform their tasks.
- **Cloud Security Governance**: A set of policies, procedures, and standards that govern the security of the cloud infrastructure and data.

### Important Theorems and Concepts (continued)

- **Cloud Security Framework**: A set of guidelines and standards for securing the cloud infrastructure and data.
  - NIST Cloud Security Framework: A framework for securing cloud infrastructure and data developed by the National Institute of Standards and Technology (NIST).
  - ISO 27017: A standard for securing cloud services developed by the International Organization for Standardization (ISO).
- **Cloud Security Metrics and Monitoring**: The process of measuring and monitoring the security of the cloud infrastructure and data.
  - Security Metrics: A set of measurements used to evaluate the effectiveness of the security controls and measures implemented.
  - Security Monitoring: The process of actively monitoring the security controls and measures implemented to detect and respond to security threats.
