# **Cloud Computing & Security Revision Notes: Chapter 6**

## **6.1: Cloud Security Architecture**

- Cloud security architecture refers to the design and organization of security controls within a cloud computing environment.
- Key components:
  - Cloud security gateway
  - Firewall
  - Access control lists (ACLs)
  - Identity and access management (IAM)
- Security principles:
  - Least privilege
  - Defense in depth
  - Segregation of duties

## **6.2: Cloud Security Challenges**

- Cloud security challenges include:
  - Lack of visibility and control
  - Data sovereignty and compliance
  - Identity and authentication
  - Data encryption and key management
- Mitigation strategies:
  - Monitoring and logging
  - Security information and event management (SIEM)
  - Cloud security gateways
  - Data loss prevention (DLP)

## **6.3: Cloud Security Governance**

- Cloud security governance refers to the processes and policies that ensure cloud security.
- Key components:
  - Cloud security policies
  - Cloud security procedures
  - Cloud security training
  - Cloud security awareness
- Best practices:
  - Establish a cloud security governance framework
  - Define roles and responsibilities
  - Conduct regular risk assessments
  - Monitor and review cloud security policies and procedures

## **6.4: Cloud Security Risks**

- Cloud security risks include:
  - Data breaches
  - Unauthorized access
  - Malware and ransomware
  - Denial of service (DoS) and distributed denial of service (DDoS)
- Mitigation strategies:
  - Data encryption and key management
  - Identity and access management (IAM)
  - Security information and event management (SIEM)
  - Incident response and disaster recovery

## **6.5: Cloud Security Measurement**

- Cloud security measurement refers to the processes and tools used to measure and assess cloud security.
- Key components:
  - Cloud security metrics
  - Cloud security dashboards
  - Cloud security monitoring
  - Cloud security reporting
- Best practices:
  - Establish cloud security metrics and targets
  - Use cloud security dashboards and monitoring tools
  - Conduct regular cloud security reporting and analysis
  - Use cloud security metrics to inform security decisions
