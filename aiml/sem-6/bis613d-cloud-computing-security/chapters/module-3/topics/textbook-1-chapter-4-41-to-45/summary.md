# **Cloud Computing & Security Revision Notes: Chapter 4**

### 4.1: Cloud Computing Benefits

- Advantages of cloud computing:
  - Scalability
  - Cost-effectiveness
  - Increased flexibility
  - Improved reliability
  - Enhanced collaboration
- Key benefits:
  - Reduced capital expenditures
  - Increased agility
  - Simplified management

### 4.2: Cloud Service Models

- Three main service models:
  - Infrastructure as a Service (IaaS)
  - Platform as a Service (PaaS)
  - Software as a Service (SaaS)
- Characteristics of each model:
  - IaaS: customer controls infrastructure
  - PaaS: provider manages infrastructure and tools
  - SaaS: provider manages infrastructure, tools, and software

### 4.3: Cloud Deployment Models

- Two main deployment models:
  - Public Cloud: shared resources, multi-tenancy
  - Private Cloud: dedicated resources, single-tenancy
- Characteristics of each model:
  - Public Cloud: scalability, high availability, security
  - Private Cloud: control, security, customization

### 4.4: Cloud Security

- Key security concerns:
  - Data security
  - Authentication and authorization
  - Network security
  - Compliance and governance
- Security measures:
  - Data encryption
  - Access controls
  - Monitoring and logging
  - Incident response

### 4.5: Cloud Security Theories and Models

- Overview of various security theories and models:
  - Penetration testing
  - Vulnerability management
  - Risk management
  - Threat modeling
- Key concepts:
  - Zero trust architecture
  - Defense in depth
  - Incident response planning

## Important Formulas and Definitions:

- **Cloud Security Architect**: a professional responsible for designing and implementing secure cloud computing systems.
- **Cloud Security Framework**: a structured approach to managing and mitigating cloud security risks.
- **Zero Trust Architecture**: an approach to security that assumes all users and devices are untrusted.
