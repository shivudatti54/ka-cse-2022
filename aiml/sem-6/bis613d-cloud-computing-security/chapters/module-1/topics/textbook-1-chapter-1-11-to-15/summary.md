# **Revision Notes: Textbook 1, Chapter 1: 1.1 to 1.5**

## **Distributed System Models and Enabling Technologies**

### 1.1: Introduction to Distributed Systems

- Definition: A distributed system is a collection of independent computers that appear to be a single, cohesive system.
- Characteristics:
  - Autonomy: Each node operates independently.
  - Organization: Nodes interact through communication.
  - Scalability: Distributed systems can be expanded or contracted as needed.

### 1.2: Types of Distributed Systems

- **Homogeneous Systems**: All nodes have the same characteristics and are identical.
- **Heterogeneous Systems**: Nodes have different characteristics and may be from different vendors.

### 1.3: Distributed System Models

- **Homogeneous Distributed System Model**: Assumes all nodes are identical and have the same capabilities.
- **Heterogeneous Distributed System Model**: Accounts for the differences in node capabilities and characteristics.

### 1.4: Scalable Computing

- Definition: The ability of a system to increase or decrease its capacity to handle workload without sacrificing performance.
- **Scalability**: Can be achieved through horizontal scaling (adding more nodes) or vertical scaling (upgrading existing nodes).

### 1.5: Cloud Computing

- Definition: A model for delivering computing services over the internet, where resources are provided as a service.
- Key characteristics:
  - On-demand self-service
  - Broad network access
  - Resource pooling
  - Rapid elasticity

## **Important Formulas, Definitions, and Theorems**

- **Scalability Formula**: $S = \frac{d}{t}$, where S is scalability, d is degree of parallelism, and t is time.
