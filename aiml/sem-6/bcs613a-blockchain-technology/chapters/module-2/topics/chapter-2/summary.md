# **Blockchain Technology Revision Notes - Chapter 2**

### Introduction

- Blockchain is a decentralized, distributed ledger technology
- Enabling secure, transparent, and tamper-proof data storage and transmission

### Decentralization using Blockchain

- **Decentralization**: A system where no single entity has control over the data or network
- **Blockchain Decentralization**: A network of nodes that validate and record transactions, without a central authority
- **Consensus Mechanisms**: Methods for nodes to agree on the state of the blockchain, e.g., Proof of Work (PoW), Proof of Stake (PoS)

### Methods of Decentralization

- **Distributed Ledger Technology (DLT)**: A decentralized ledger that records transactions across a network of nodes
- **Cryptocurrencies**: Digital currencies that use blockchain technology for secure and transparent transactions
- **Smart Contracts**: Self-executing contracts with the terms of the agreement written directly into lines of code

### Blockchain and Full Ecosystem

- **Blockchain Network**: A network of nodes that validate and record transactions
- **Node**: A computer that participates in the blockchain network
- **Consensus Algorithm**: A method for nodes to agree on the state of the blockchain
- **Hash Function**: A one-way mathematical function that maps input data to a fixed-size string of characters

### Important Formulas and Definitions

- **Hash Function**: `h(x) = f(x) = x's unique digital fingerprint`
- **Public-Private Key Cryptography**: A method for secure data transmission using pairs of keys (public and private)
- **Consensus Theorem**: "If a network of nodes agrees on the state of the blockchain, then the state is consensual"

### Important Theorems

- **Gödel's Incompleteness Theorem**: "Any formal system is either incomplete or inconsistent"
- **Shannon's Channel Capacity Theorem**: "The maximum rate at which information can be transmitted through a channel is limited by the channel's bandwidth"
