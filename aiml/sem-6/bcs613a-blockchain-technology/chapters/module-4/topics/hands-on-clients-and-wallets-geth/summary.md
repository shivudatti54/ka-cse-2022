# Hands-on: Clients and Wallets – Geth

### Introduction

- Geth is a popular Ethereum client used for development and testing purposes.
- It is an open-source, standalone Ethereum node that can be run on a local machine.

### Key Features

- **Node Mode**: Runs a full Ethereum node, connecting to the Ethereum network.
- **Light Mode**: Runs a lightweight Ethereum client, connecting to a local Ethereum instance.
- **Web3.py**: A Python library that provides an interface to interact with the Ethereum network using Geth.

### Important Concepts

- **Network ID**: A unique identifier for the Ethereum network.
- **Chain ID**: A unique identifier for the Ethereum chain.
- **Geth Configuration**: A file that configures the Geth client, including network settings and wallet configuration.

### Geth Configuration Files

- **config.json**: Stores network settings, such as network ID and chain ID.
- **wallets.json**: Stores wallet configuration, including wallet addresses and private keys.

### Important Formulas and Definitions

- **Blockchain Hash**: A unique digital fingerprint of each block in the Ethereum blockchain.
- **Block Height**: The number of blocks in the Ethereum blockchain, used to track the state of the network.
- **Block Time**: The average time it takes for a block to be mined on the Ethereum network (approximately 15 seconds).

### Important Theorems

- **Sharding**: A theoretical concept that proposes to split the Ethereum network into smaller, parallel chains to improve scalability.
- **Proof of Stake (PoS)**: A consensus algorithm that allows validators to participate in the validation process based on the amount of tokens they hold.

### Revision Notes

- Familiarize yourself with the Geth client and its configuration files.
- Understand the difference between Node Mode and Light Mode.
- Learn about Network ID, Chain ID, and Geth configuration files.
- Study the formulas and definitions related to blockchain hashes, block height, and block time.
- Review the concepts of sharding and Proof of Stake (PoS).
