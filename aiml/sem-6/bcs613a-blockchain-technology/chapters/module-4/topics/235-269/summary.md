# Blockchain Technology

**235-269: Ethereum Stack and Blockchain**

### Components of the Ethereum Stack

- **Frontend:**
  - Web3.js: library for interacting with the Ethereum blockchain
  - Ethers.js: library for interacting with the Ethereum blockchain
- **Backend:**
  - Ethereum nodes: full nodes that store a copy of the blockchain
  - Full clients: nodes that store a copy of the blockchain and can execute Smart Contracts
  - Light clients: nodes that only store the current state of the blockchain
- **Smart Contracts:**
  - Self-executing contracts with the terms of the agreement written directly into code
  - Utilize the Ethereum Virtual Machine (EVM)

### Ethereum Blockchain

- **Blockchain Structure:**
  - Blocks: collections of transactions
  - Hash: unique digital fingerprint of a block
  - Merkle tree: data structure used to efficiently verify the integrity of the blockchain
- **Consensus Mechanisms:**
  - Proof of Work (PoW): energy-intensive method of validating transactions
  - Proof of Stake (PoS): energy-efficient method of validating transactions

### Key Concepts

- **Gas:** unit of measurement for transaction fees
- **Transaction Fee:** fee paid to miners for processing transactions
- **Block Reward:** reward paid to miners for creating a new block
- **Turing Complete:** ability of a system to simulate the behavior of any other system

### Formulas and Definitions

- **Hash Function:** f(x) → h(x): takes input x and produces output h(x)
- **Merkle Tree:** a data structure used to efficiently verify the integrity of the blockchain
- **Ethereum Virtual Machine (EVM):** virtual machine that executes Smart Contracts

### Important Theorems

- **Shannon's Source Coding Theorem:** explains the concept of entropy and its relation to data compression
- **Fermat's Last Theorem:** states that there are no integer solutions to the equation a^n + b^n = c^n for n > 2
