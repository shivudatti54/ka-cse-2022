# **20.3 20.4 Data Security and Privacy Revision Notes**

### Data Security

- **Definition**: The practice of protecting data from unauthorized access, use, disclosure, modification, or destruction.
- **Types of Threats**:
  - Insider threats
  - Physical threats
  - Technical threats
- **Key Concepts**:
  - Confidentiality
  - Integrity
  - Availability
- **Importance of Data Security**:
  - Protects sensitive information
  - Ensures data is accurate and complete
  - Ensures data is accessible when needed

### Data Privacy

- **Definition**: The control over personal information and the ability to make informed decisions about its use.
- **Types of Privacy Risks**:
  - Disclosing personal information to unauthorized parties
  - Unauthorized use of personal information
  - Unauthorized modification or destruction of personal information
- **Key Concepts**:
  - Data minimization
  - Data anonymization
  - Data protection by design
- **Importance of Data Privacy**:
  - Protects individuals' personal information
  - Ensures respect for individuals' autonomy
  - Enhances trust in organizations

### Important Formulas and Definitions

- **Bloom Filter**: A probabilistic data structure used to test whether an element is a member of a set.
- **Hash Function**: A function that takes input data of any type and size and produces a fixed-length string of characters, known as a hash value or digest.
- **Data Encryption**: A process of transforming plaintext into unreadable ciphertext to protect data from unauthorized access.
- **Data Masking**: A process of hiding sensitive data by replacing it with fictional data or obfuscating it to prevent unauthorized access.

### Important Theorems

- **Shannon's Theorem**: A fundamental theorem in information theory that describes the relationship between the uncertainty in a message and the amount of information in the message.
- **Data Privacy Theorem**: A theorem that describes the relationship between data privacy and the use of personal information.
