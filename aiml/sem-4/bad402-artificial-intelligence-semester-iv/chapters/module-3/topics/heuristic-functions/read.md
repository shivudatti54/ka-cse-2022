# Heuristic Functions in Artificial Intelligence

## Introduction to Heuristic Functions

Heuristic functions are a fundamental concept in artificial intelligence that enable informed search algorithms to make smarter decisions about which paths to explore. Unlike uninformed search strategies that blindly explore all possibilities, heuristic functions provide a way to estimate the cost from a given state to the goal state, guiding the search process more efficiently.

A heuristic function, denoted as h(n), is a function that estimates the cost of the cheapest path from node n to a goal node. The term "heuristic" comes from the Greek word "heuriskein" meaning "to discover," reflecting its purpose of helping discover solutions more quickly.

## Key Properties of Heuristic Functions

### Admissibility

A heuristic function is admissible if it never overestimates the actual cost to reach the goal. Formally, for all nodes n, h(n) ≤ h*(n), where h*(n) is the true cost from n to the nearest goal.

```
Example: In pathfinding on a grid, straight-line distance is admissible
because it's the shortest possible path, never overestimating actual distance.
```

### Consistency (Monotonicity)

A heuristic is consistent if for every node n and every successor n' of n generated by action a, the estimated cost of reaching the goal from n is no greater than the step cost of getting to n' plus the estimated cost of reaching the goal from n':

h(n) ≤ c(n, a, n') + h(n')

This property ensures that the heuristic follows the triangle inequality.

## Types of Heuristic Functions

### Domain-Specific Heuristics

These are tailored to specific problem domains:

**8-Puzzle Problem:**

- Misplaced tiles heuristic: Counts how many tiles are not in their goal position
- Manhattan distance heuristic: Sums the horizontal and vertical distances each tile must move

```
8-Puzzle Example:
Current state:   Goal state:
1 2 3           1 2 3
4   5           4 5 6
7 8 6           7 8

Misplaced tiles: 2 (tiles 5 and 6)
Manhattan distance:
Tile 5: |1-0| + |1-2| = 1 + 1 = 2
Tile 6: |2-2| + |2-1| = 0 + 1 = 1
Total: 2 + 1 = 3
```

### Domain-Independent Heuristics

These can be applied across various problems:

- Relaxation heuristics: Simplify the problem by removing constraints
- Pattern database heuristics: Precompute solutions to subproblems
- Learning heuristics: Improve through experience

## Designing Effective Heuristics

### Trade-offs in Heuristic Design

| Heuristic Quality | Search Efficiency | Computation Time   |
| ----------------- | ----------------- | ------------------ |
| More informed     | More efficient    | Potentially slower |
| Less informed     | Less efficient    | Faster to compute  |

### Dominance of Heuristics

Heuristic h₂ dominates heuristic h₁ if for all nodes n, h₂(n) ≥ h₁(n). A dominating heuristic typically leads to more efficient search as it provides better guidance.

## Heuristic Functions in Search Algorithms

### A\* Search Algorithm

A\* search uses the evaluation function f(n) = g(n) + h(n), where:

- g(n) = cost from start to node n
- h(n) = estimated cost from n to goal

A\* is optimal if h(n) is admissible and consistent.

```
A* Search Example:
Start node S with g(S)=0, h(S)=10 → f(S)=10
Expand S to nodes A and B:
- A: g(A)=3, h(A)=6 → f(A)=9
- B: g(B)=1, h(B)=12 → f(B)=13
Next, expand A (lowest f-value)
```

### Greedy Best-First Search

Uses only the heuristic function: f(n) = h(n). Tends to be efficient but not optimal.

## Evaluation of Heuristic Performance

### Effective Branching Factor

Measures how effectively the heuristic reduces the search space. A lower effective branching factor indicates better heuristic performance.

### Completeness and Optimality

| Algorithm  | Complete? | Optimal? | Requirements         |
| ---------- | --------- | -------- | -------------------- |
| A\*        | Yes       | Yes      | Admissible heuristic |
| Greedy BFS | No        | No       | None                 |
| IDA\*      | Yes       | Yes      | Admissible heuristic |

## Practical Applications

### Pathfinding and Navigation

- GPS navigation systems use heuristic functions based on straight-line distance
- Game AI for character movement

### Puzzle Solving

- Rubik's cube solvers
- Sliding tile puzzles
- Sokoban and other puzzle games

### Robotics and Planning

- Motion planning with configuration space heuristics
- Task planning with relaxed plan heuristics

## Advanced Concepts

### Pattern Databases

Precomputed databases storing exact solution costs for subproblems, providing highly informed heuristics for problems like the 15-puzzle.

### Learning Heuristics

Machine learning techniques can be used to improve heuristic functions over time through experience.
