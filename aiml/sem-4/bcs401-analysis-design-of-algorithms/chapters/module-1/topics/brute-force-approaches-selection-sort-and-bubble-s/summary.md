# **BRUTE FORCE APPROACHES: Selection Sort and Bubble Sort**

### Overview

- Brute force approaches: simple, yet inefficient algorithms for sorting data
- Selection Sort and Bubble Sort: two classic examples of brute force sorting algorithms

### Definitions

- **Selection Sort**: a sorting algorithm that sorts an array by repeatedly finding the minimum element from the unsorted part and putting it at the beginning
- **Bubble Sort**: a sorting algorithm that repeatedly steps through the list, compares adjacent elements and swaps them if they are in the wrong order
- **Brute Force**: an approach that uses the simplest possible solution to a problem, often resulting in inefficient algorithms

### Algorithms

#### Selection Sort

- Steps:
  1.  Find the minimum element in the unsorted part of the array
  2.  Swap it with the first element of the unsorted part
  3.  Repeat steps 1-2 until the entire array is sorted
- Time complexity: O(n^2)
- Space complexity: O(1)

#### Bubble Sort

- Steps:
  1.  Compare adjacent elements in the array
  2.  Swap them if they are in the wrong order
  3.  Repeat steps 1-2 until no more swaps are needed
- Time complexity: O(n^2)
- Space complexity: O(1)

### Formulas

- **Time Complexity Formula**: T(n) = O(n^2) for both Selection Sort and Bubble Sort
- **Space Complexity Formula**: S(n) = O(1) for both Selection Sort and Bubble Sort

### Important Theorems

- **Stirling's Accurate Formula**: used to estimate the number of comparisons required for Selection Sort and Bubble Sort
- **Big O Notation**: used to describe the upper bound of an algorithm's time complexity

### Key Points

- Brute force approaches are simple to understand but slow for large datasets
- Selection Sort and Bubble Sort are examples of brute force algorithms
- Both have a time complexity of O(n^2) and space complexity of O(1)
- Stirling's Accurate Formula and Big O Notation are used to analyze and describe the performance of Selection Sort and Bubble Sort.
