# **Chapter 2: Introduction to Algorithms**

### 2.1 Fundamentals of Algorithmic Problem Solving

#### Definitions and Notations

- **Algorithm**: A well-defined procedure that takes some input and produces a corresponding output.
- **Pseudocode**: A high-level representation of an algorithm using plain text.
- **Time Complexity**: The amount of time an algorithm takes to complete as a function of input size.
- **Space Complexity**: The amount of memory an algorithm uses as a function of input size.

#### Important Formulas and Theorems

- **Big O Notation**:
  - O(1) - Constant time complexity
  - O(log n) - Logarithmic time complexity
  - O(n) - Linear time complexity
  - O(n log n) - Linearithmic time complexity
  - O(n^2) - Quadratic time complexity
  - O(2^n) - Exponential time complexity
- **Divide and Conquer**: A problem-solving strategy that breaks down a problem into smaller sub-problems.

#### Key Concepts

- **Problem Solving Strategies**:
  - Brute Force: Trying all possible solutions.
  - Dynamic Programming: Breaking down a problem into smaller sub-problems.
  - Greedy: Making the locally optimal choice at each step.
  - Divide and Conquer: Breaking down a problem into smaller sub-problems.
- **Input/Output**: The input to an algorithm and the output it produces.

#### Important Theorems

- **Master Theorem**: A theorem used to solve recurrence relations.

#### Key Terms

- **Recurrence Relation**: An equation that defines a sequence of values.
- **Recursive Function**: A function that calls itself in its own definition.
- **Dynamic Programming**: A method for solving complex problems by breaking them down into smaller sub-problems.
