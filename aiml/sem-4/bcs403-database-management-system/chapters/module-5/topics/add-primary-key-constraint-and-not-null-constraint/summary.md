# **Add Primary Key Constraint and Not Null Constraint to Employee Table**

**Definition:**

- Primary Key: A unique identifier for each record in a table, ensuring data integrity and allowing for efficient data retrieval.
- Not Null: A constraint that requires a field to contain data, preventing empty values.

**Key Points:**

- **Primary Key Constraint:**
  - Ensures uniqueness of each record.
  - Allows for efficient data retrieval and storage.
  - Typically assigned to a unique column or combination of columns.
- **Not Null Constraint:**
  - Ensures that a field cannot contain empty or null values.
  - Prevents data inconsistencies and errors.
  - Can be applied to any field.
- **Importance:**
  - Ensures data consistency and integrity.
  - Prevents errors and inconsistencies.
  - Improves data security and reliability.

**Theoretical Background:**

- **Database Normalization:** Ensures that each table contains only the necessary data, reducing data redundancy and improving data integrity.
- **Data Integrity:** The concept of ensuring that data is consistent and accurate throughout the database.

**Revision Tips:**

- Understand the importance of primary keys and not null constraints in maintaining data integrity.
- Apply primary key and not null constraints to individual fields or combinations of fields.
- Verify that data is consistent and accurate throughout the database.
