# **DATABASE MANAGEMENT SYSTEM - QUICK REVISION NOTES**

### Definitions

- **Database Management System (DBMS)**: Software that manages and controls data in a database.
- **Database**: A collection of organized data stored in a way that allows for efficient access and manipulation.

### Key Concepts

- **Data Structure**: Organizes data in a way that facilitates efficient access and manipulation.
  - Types:
    - Relational
    - Hierarchical
    - Network
    - Document
- **Database Model**: A conceptual representation of a database.
  - Types:
    - First Normal Form (1NF)
    - Second Normal Form (2NF)
    - Third Normal Form (3NF)
- **Database Design**: The process of creating a database, including data modeling, normalization, and physical design.

### Important Formulas and Theorems

- **Data Integrity**: Ensures that data is consistent and accurate.
  - Constraints: Primary Key, Foreign Key, Unique, and Check Constraints
- **Database Security**: Protects data from unauthorized access and modification.
  - Types:
    - Authentication
    - Authorization
    - Encryption

### Key Terms

- **SQL (Structured Query Language)**: A language used to manage and manipulate databases.
- **SQL Queries**:
  - SELECT
  - INSERT
  - UPDATE
  - DELETE
- **Database Systems**: Include DBMS, database design, and database management.

### Important Theorems

- **Data Independence Theorem**: States that a database system should provide independence between:
  - Physical and logical design
  - External and internal data definitions

### Quick Revision Tips

- Focus on key concepts and definitions
- Practice SQL queries and database design
- Review database security and data integrity concepts
