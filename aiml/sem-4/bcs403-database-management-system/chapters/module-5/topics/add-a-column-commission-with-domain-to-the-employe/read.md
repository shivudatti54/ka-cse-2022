# **Add a Column Commission with Domain to the Employee Table**

## **Introduction**

In this topic, we will learn how to add a new column to the Employee table with a specific data type and constraints. We will use SQL (Structured Query Language) to achieve this.

## **What is Commission?**

Commission is a form of payment where an employee receives a percentage of the total sales or revenue generated by their work. It is a common practice in many industries, including sales, marketing, and finance.

## **Why Domain?**

A domain is a set of values that a column can take. It is used to restrict the values that can be entered into a column, ensuring data consistency and accuracy. In this topic, we will learn how to create a domain for the Commission column.

## **Key Concepts**

- **Column**: A column is a single field or attribute in a table.
- **Data Type**: A data type is the type of value that a column can hold.
- **Domain**: A domain is a set of valid values that a column can take.
- **SQL**: SQL (Structured Query Language) is a language used to manage relational databases.

## **Adding a Column Commission with Domain to the Employee Table**

### Step 1: Design the Commission Column

First, we need to design the Commission column with the correct data type and domain. We will use the `DECIMAL` data type to store decimal values, and create a domain with the following values:

| Commission Rate |
| --------------- |
| 10%             |
| 15%             |
| 20%             |

```sql
CREATE DOMAIN commission_domain AS
VALUES ('10%', '15%', '20%');
```

### Step 2: Add the Commission Column to the Employee Table

Next, we will add the Commission column to the Employee table with the correct data type and domain:

```sql
ALTER TABLE employees
ADD COLUMN commission DECIMAL(5, 2) DOMAIN commission_domain;
```

In this example, we are adding a new column named `commission` with a data type of `DECIMAL(5, 2)`, which means it can hold decimal values with a maximum of 5 digits and 2 decimal places. We are also specifying the `commission_domain` domain, which restricts the values to the ones we defined earlier.

### Example Use Case

Suppose we want to insert a new employee with a commission rate of 15%. We can use the following SQL query to insert the new employee:

```sql
INSERT INTO employees (name, department, commission)
VALUES ('John Doe', 'Sales', '15%');
```

In this example, the `commission` value is inserted as a string value, but we can also store it as a decimal value by removing the quotes:

```sql
INSERT INTO employees (name, department, commission)
VALUES ('John Doe', 'Sales', 15);
```

### Best Practices

- Always use meaningful column names and data types to ensure data consistency and accuracy.
- Use domains to restrict values and prevent data errors.
- Test your queries thoroughly to ensure data integrity.

By following these steps and best practices, you can successfully add a column Commission with domain to the Employee table in your database management system.
