# Learning Objectives
After studying this topic, you should be able to:
1.  Define Redis and explain its primary role as an in-memory key-value store.
2.  Identify and describe the purpose of Redis's core data structures: Strings, Hashes, Lists, Sets, and Sorted Sets.
3.  Compare and contrast the RDB and AOF persistence mechanisms, including their advantages and disadvantages.
4.  Explain the primary use cases for Redis in a big data architecture, specifically caching and publish/subscribe messaging.
5.  Diagram a basic caching pattern using Redis to reduce load on a primary database.
6.  Differentiate Redis from other types of NoSQL databases based on its data model, performance characteristics, and typical use cases.
