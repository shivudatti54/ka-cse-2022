# Database Management System

### Overview

- A database management system (DBMS) is a software system that allows organizations to store, manage, and retrieve data in a controlled and efficient manner.
- DBMS provides a layer of abstraction between the user and the physical database.

### Key Concepts

- **Database**: A collection of organized data in a specific format.
- **Data**: Information in any form, such as text, numbers, or images.
- **Schema**: A description of the database structure, including tables, fields, and relationships.
- **Normalization**: The process of organizing data in a database to minimize data redundancy and improve data integrity.

### Important Formulas and Definitions

- **ACID (Atomicity, Consistency, Isolation, Durability)**: A set of properties that ensure database transactions are processed reliably.
- **Normalization Rule 1 (NR1)**: Each non-key attribute in a table must depend on the entire primary key.
- **Normalization Rule 2 (NR2)**: Each non-key attribute in a table must have a unique value for each unique combination of values in the primary key.
- **Normalization Rule 3 (NR3)**: A table should have the least possible number of columns to minimize data redundancy.

### Important Theorems

- **Gödel's Incompleteness Theorem**: A fundamental result in mathematics that states that any formal system that is powerful enough to describe basic arithmetic is either incomplete or inconsistent.
- **Database Theorem**: A theorem that states that a database can be normalized to at most 3NF.
