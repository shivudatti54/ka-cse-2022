# **Specifying Constraints as Assertions and Action Triggers**

## **Overview**

Constraints in a database management system (DBMS) are rules that define the relationships between data entities. Specifying constraints as assertions and action triggers helps ensure data integrity and consistency.

## **Key Concepts**

- **Assertions**: A statement that describes a condition or relationship between data entities.
- **Action Triggers**: A mechanism that enforces assertions by enforcing constraints.
- **Constraint Types**:
  - **Referential Integrity Constraint (RI)**: Ensures that a foreign key references an existing primary key.
  - **Unique Constraint (UC)**: Ensures that all values in a column or set of columns are unique.
  - **Primary Key Constraint (PK)**: Ensures that a column or set of columns uniquely identifies each record.
  - **Default Constraint**: Assigns a default value to a column if no value is provided.
  - **Check Constraint**: Ensures that a column or expression meets a specific condition.
- **Data Integrity Theorem**: States that a DBMS can ensure that a database is consistent if the constraints are correctly specified and enforced.

## **Important Formulas and Definitions**

- **Primary Key (PK)**: A column or set of columns that uniquely identifies each record.
- **Foreign Key (FK)**: A column or set of columns that references a primary key.
- **Referential Integrity (RI)**: Ensures that a foreign key references an existing primary key.

## **Key Points for Revision**

- Constraints are rules that define relationships between data entities.
- Assertions describe conditions or relationships between data entities.
- Action triggers enforce assertions by enforcing constraints.
- Constraint types include RI, UC, PK, default, and check constraints.
- The Data Integrity Theorem ensures consistency in a DBMS.
