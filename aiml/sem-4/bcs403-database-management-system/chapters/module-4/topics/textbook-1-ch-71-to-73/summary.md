# **Database Management System (DBMS) Revision Notes**

**Module:** No. of Hours: 8
**Topic:** Textbook 1: Ch 7.1 to 7.3

**Key Points:**

- **Definition:** A database management system (DBMS) is a software that enables the creation, modification, and manipulation of data in a database.
- **Types of DBMS:**
  - Relational DBMS (RDBMS): uses tables to store data.
  - File DBMS: stores data in a single file.
  - Object-Oriented DBMS: stores data as objects.
- **Database Model:**
  - Data model: defines the structure of the database.
  - View: a virtual table that displays data from multiple tables.
- **Data Types:**
  - Atomic: cannot be divided into smaller parts.
  - Composed: composed of smaller parts.
  - Composite: contains multiple atomic values.
- **Database Normalization:**
  - First normal form (1NF): eliminates repeating groups.
  - Second normal form (2NF): eliminates partial dependencies.
  - Third normal form (3NF): eliminates transitive dependencies.
- **Database Design:**
  - Entity-Relationship Model (ERM): defines the structure of the database.
  - Entity-Attribute-Value (EAV) Model: stores data in a separate table.
- **Database Management:**
  - Database administration: responsible for database maintenance.
  - Query optimization: optimizes database queries for performance.

**Important Formulas and Theorems:**

- **1NF:** No repeating groups
- **2NF:** No partial dependencies
- **3NF:** No transitive dependencies
- **BCNF:** No transitive dependencies
- **Third Normal Form Theorem:** If a table is in 2NF and there is a partial dependency between two non-key attributes, then the table cannot be in 2NF.

**Definitions:**

- **Primary Key (PK):** uniquely identifies a record.
- **Foreign Key (FK):** uniquely identifies a record in a related table.
- **Primary Index (PI):** an index on the primary key.
- **Composite Primary Key (C-PK):** multiple columns form the primary key.
