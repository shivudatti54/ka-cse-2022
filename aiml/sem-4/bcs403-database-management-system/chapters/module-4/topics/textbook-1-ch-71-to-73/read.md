# **Database Management System Study Material**

**Module:** No. of Hours: 8
**Topic:** Textbook 1: Ch 7.1 to 7.3

### Introduction

**Definition of Database Management System (DBMS):**
A DBMS is a software system that enables organizations to store, manage, and retrieve data in a structured and controlled manner.

**Importance of DBMS:**
A DBMS provides a powerful tool for managing data, enabling organizations to make informed decisions, and improving operational efficiency.

### Chapter 7.1: Database Management System Components

#### 7.1.1 Database Management System (DBMS) Components

- **Hardware Components:**
  - Central Processing Unit (CPU)
  - Memory (RAM)
  - Storage Devices (HDD, SSD)
  - Input/Output Devices (Keyboard, Mouse, Printer)
- **Software Components:**
  - Operating System
  - Database Management System (DBMS)
  - Application Software
- **Database Components:**
  - **Database Schema:** A conceptual representation of the database structure.
  - **Database Files:** Physical storage of data, consisting of multiple files.

#### 7.1.2 Database Management System (DBMS) Functions

- **Data Definition:** Defining the structure of the database.
- **Data Control:** Controlling access to the data.
- **Data Retrieval:** Providing access to the data.
- **Data Manipulation:** Enabling users to update and delete data.

### Chapter 7.2: Database Model

#### 7.2.1 Entity-Relationship Model (ERM)

- **Definition:** A conceptual representation of the database structure using entities and relationships.
- **Components:**
  - **Entities:** Independent objects with unique attributes.
  - **Attributes:** Characteristics of the entities.
  - **Relationships:** Links between entities.

#### 7.2.2 Normalization

- **Definition:** Removing redundant data from the database.
- **Benefits:**
  - Reduces data redundancy.
  - Improves data integrity.
  - Increases data scalability.

### Chapter 7.3: Database Design

#### 7.3.1 Data Modeling

- **Definition:** Creating a conceptual representation of the database structure.
- **Techniques:**
  - Entity-Relationship Model (ERM)
  - Relational Model
  - Object-Oriented Model

#### 7.3.2 Database Design

- **Definition:** Creating a physical representation of the database structure.
- **Components:**
  - **Tables:** Relational database design.
  - **Classes:** Object-Oriented database design.

### Key Concepts

- **Database Management System (DBMS):** A software system that enables organizations to store, manage, and retrieve data.
- **Hardware Components:** CPU, Memory, Storage Devices, Input/Output Devices.
- **Software Components:** Operating System, Database Management System (DBMS), Application Software.
- **Database Schema:** A conceptual representation of the database structure.
- **Database Files:** Physical storage of data, consisting of multiple files.
- **Entity-Relationship Model (ERM):** A conceptual representation of the database structure using entities and relationships.
- **Normalization:** Removing redundant data from the database to improve data integrity and scalability.
- **Data Modeling:** Creating a conceptual representation of the database structure.
- **Database Design:** Creating a physical representation of the database structure.
