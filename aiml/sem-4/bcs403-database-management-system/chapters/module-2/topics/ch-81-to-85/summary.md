### Database Management System (DBMS) Revision Notes (Ch 8.1 to 8.5)

#### Overview

- Definition: A database management system (DBMS) is a software that enables the creation, modification, and management of databases.
- Purpose: To provide a controlled environment for data storage, retrieval, and manipulation.

#### Key Concepts

- **Database**: A collection of organized data that is stored in a way that allows for efficient retrieval and manipulation.
- **Database Management System (DBMS)**: A software that manages a database, providing a interface to interact with the data.
- **Normal Form (NF)**: A set of rules that ensures a database is in a third-normal form (3NF) to minimize data redundancy and dependency.

#### Normal Forms

- **First Normal Form (1NF)**: Each table cell must contain a single value.
- **Second Normal Form (2NF)**: Each non-key attribute in a table must depend on the entire primary key.
- **Third Normal Form (3NF)**: If a table is in 2NF, and a non-key attribute depends on another non-key attribute, then it should be moved to a separate table.

#### Relational Model

- **Entity-Relationship Diagram (ERD)**: A visual representation of a database's structure, showing the relationships between entities.
- **Entity**: A thing or concept that is represented in the database.
- **Attribute**: A characteristic or property of an entity.
- **Primary Key (PK)**: A unique identifier for each entity.

#### Data Control and Security

- **ACID Properties**: Atomicity, Consistency, Isolation, Durability (ensures database transactions are processed reliably).
- **Data Encryption**: The process of converting plaintext data into unreadable ciphertext to protect it from unauthorized access.

#### SQL and Querying

- **SQL (Structured Query Language)**: A standard language for managing relational databases.
- **SELECT Statement**: Retrieves data from a database table.
- **WHERE Clause**: Filters data based on conditions.

#### Indexing and Optimization

- **Indexing**: Improves query performance by creating a data structure that facilitates fast lookup and retrieval of data.
- **Optimization**: Techniques used to improve the performance of a database system.

#### Import and Backup

- **Import**: The process of copying data from one database to another.
- **Backup**: The process of creating a copy of a database to ensure data recovery in case of loss or corruption.

#### Important Formulas and Theorems

- **Relational Algebra**: A mathematical framework for manipulating relational databases.
- **Query Optimization**: Techniques used to improve the performance of queries.
