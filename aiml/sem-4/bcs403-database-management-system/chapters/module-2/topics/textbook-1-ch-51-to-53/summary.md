### Overview

- Database management system (DBMS) is a software that allows organizations to store, manage, and retrieve data in a structured and controlled manner.

### Key Concepts

- **Database**: A collection of organized data, represented in a way that makes it easy for users to access, modify, and manage.
- **Database Management System (DBMS)**: A software that enables the creation, modification, and manipulation of databases.
- **Schema**: A blueprint or structure of a database, defining the relationships between data entities.

### Important Definitions

- **Entity**: A thing or concept that has identity and existence, represented in a database.
- **Attribute**: A characteristic or feature of an entity, represented in a database.
- **Relationship**: A connection between two or more entities, represented in a database.

### Theories and Formulas

- **First Normal Form (1NF)**: A database is in 1NF if each table cell contains a single value.
- **Second Normal Form (2NF)**: A database is in 2NF if it is in 1NF and all non-key attributes are fully dependent on the primary key.
- **Third Normal Form (3NF)**: A database is in 3NF if it is in 2NF and there are no transitive dependencies.

### Important Theorems

- **Entity-Relationship Model (ERM)**: A model that represents the relationships between entities in a database.
- **Database Design Principles**: A set of guidelines for designing databases that ensure data integrity, consistency, and efficiency.

### Key Points

- DBMS plays a crucial role in managing data in various organizations.
- Database design principles and entity-relationship model are essential in designing databases.
- Normalization techniques (1NF, 2NF, 3NF) are used to ensure data integrity and minimize data redundancy.
