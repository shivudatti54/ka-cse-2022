### Database Management System Revision Notes

#### Ch 9.1 to 9.2 Textbook 2: 3.5 RBT: L1

**Definitions:**

- **Database Management System (DBMS):** Software that manages a database, including creating, modifying, and retrieving data.
- **Database:** A collection of organized data that is stored in a way that allows for efficient retrieval and manipulation.

**Key Concepts:**

- **Database Models:** Relational, Hierarchical, Network, and Entity-Relationship models.
- **Database Design:** Normalization, Primary Keys, Foreign Keys, and Data Integrity.
- **Database Management Techniques:** Normalization, Data Warehousing, and Data Mining.

**Formulas and Theorems:**

- **Normalization Theorem:** A database is in 3NF (third normal form) if it is in 2NF (second normal form) and all non-key attributes are fully functionally dependent on the primary key.
- **Relational Algebra:** A mathematical language used to manipulate and query relational databases.

**Important Topics:**

- **Database Normalization:** 1NF, 2NF, 3NF, Boyce-Codd Normalization.
- **Database Design Principles:** Data Consistency, Data Integrity, and Data Security.
- **Database Management Systems:** Features, Advantages, and Disadvantages.

**Key Terms:**

- **Relational Database:** A database that stores data in tables and uses relationships to link data.
- **Data Modeling:** The process of creating a conceptual representation of a database.
- **Data Warehousing:** The process of creating a centralized repository for historical data.
