# Introduction to Databases: Revision Notes

### Introduction

- A database is a collection of organized data that can be accessed, managed, and manipulated using a database management system (DBMS).
- Databases are used to store, manage, and retrieve data efficiently.
- A database can be thought of as a digital filing system.

### Characteristics of Database Approach

- **Data Integration**: Combines data from multiple sources into a single, unified view.
- **Data Normalization**: Organizes data into a structured format to reduce data redundancy and improve data integrity.
- **Data Consistency**: Ensures that data is accurate and up-to-date.
- **Data Security**: Protects data from unauthorized access and tampering.
- **Data Backup and Recovery**: Ensures that data is recoverable in case of loss or corruption.

### Advantages of Using the DBMS Approach

- **Improved Data Management**: Automates data management tasks, reducing manual errors.
- **Increased Data Security**: Provides robust security features to protect data.
- **Data Sharing and Collaboration**: Enables multiple users to access and share data.
- **Improved Data Integrity**: Ensures data accuracy and consistency.

### History of Database Applications

- **1950s**: First database management systems (DBMS) developed, including CODASYL and Information Management System (IMS).
- **1970s**: Relational databases developed, including IBM's System R and Digital Equipment Corporation's (DEC) Interbase.
- **1980s**: SQL (Structured Query Language) developed, enabling database querying and manipulation.
- **1990s**: Object-oriented databases developed, including IBM's DB2 and Oracle's Oracle8.
- **2000s**: Cloud-based databases developed, including Amazon Relational Database Service (RDS) and Google Cloud SQL.

Important Formulas and Definitions:

- **Data Inconsistency**: A situation where data is inaccurate or inconsistent.
- **Data redundancy**: Duplicate data that can be avoided through normalization.
- **Data normalization**: A process that reduces data redundancy and improves data integrity.
- **Data dependency**: A relationship between data elements that affect each other.

Important Theorems:

- **Entity-Relationship Model (ERM)**: A theoretical framework for designing databases.
- **Database Normalization Theorem**: A theorem that proves the benefits of normalization.
- **Database Consistency Theorem**: A theorem that proves the benefits of consistency.
