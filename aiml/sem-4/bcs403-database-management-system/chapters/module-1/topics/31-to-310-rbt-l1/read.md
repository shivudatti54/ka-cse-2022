### 3.1: Database Management Systems

**Definition:** A Database Management System (DBMS) is a software system that helps in managing and controlling access to a database.

**Explanation:** A DBMS provides a way to store, modify, and retrieve data in a database. It acts as an intermediary between the user and the database, providing a simple and consistent interface to access the data.

**Example:** Microsoft Access, MySQL, Oracle Database

### 3.2: Types of Database Management Systems

#### 3.2.1: Relational Database Management System (RDBMS)

**Definition:** A RDBMS is a type of DBMS that stores data in tables and relationships between tables.

**Explanation:** RDBMS uses the concept of tables, rows, and columns to store data. Each table has a primary key, which uniquely identifies each row.

**Example:** MySQL, Microsoft SQL Server, Oracle Database

#### 3.2.2: Non-Relational Database Management System (NoSQL)

**Definition:** A NoSQL DBMS is a type of DBMS that stores data in a variety of formats, such as key-value pairs, documents, or graphs.

**Explanation:** NoSQL DBMS does not use the traditional table-based structure. Instead, it uses a flexible schema to store data.

**Example:** MongoDB, Cassandra, Redis

### 3.3: Characteristics of a Database Management System

- **Data Independence:** The ability to access data without knowing the details of the underlying database.
- **Control:** The ability to control access to the database.
- **Autonomy:** The ability to operate independently of other systems.
- **Integrated:** The ability to integrate data from multiple sources.
- **Management:** The ability to manage data efficiently.

### 3.4: Features of a Database Management System

- **Data Constraints:** Rules that restrict the data in the database.
- **Data Validation:** Checks to ensure data is correct and consistent.
- **Data Encryption:** Techniques to protect data from unauthorized access.
- **Data Backup:** Copies of the data to ensure data recovery in case of failure.
- **Data Recovery:** Techniques to restore data in case of failure.

### 3.5: Advantages and Disadvantages of a Database Management System

**Advantages:**

- **Improved Data Management:** DBMS provides a way to manage and control access to data.
- **Increased Productivity:** DBMS automates many tasks, freeing up time for more important tasks.
- **Scalability:** DBMS can scale to meet the needs of large organizations.

**Disadvantages:**

- **Steep Learning Curve:** DBMS requires a significant amount of knowledge and expertise.
- **High Cost:** DBMS can be expensive to purchase and maintain.
- **Data Complexity:** DBMS can make data more complex to manage.

### 3.6: Normalization

**Definition:** Normalization is the process of organizing data in a database to minimize data redundancy and dependency.

**Explanation:** Normalization involves creating multiple tables to store data and establishing relationships between tables to ensure data consistency.

**Example:** Third Normal Form (3NF), Boyce-Codd Normal Form (BCNF)

### 3.7: Data Denormalization

**Definition:** Data denormalization is the process of storing data in a database in a non-normalized format to improve query performance.

**Explanation:** Denormalization involves creating a single table with multiple columns to store data, which can improve query performance.

**Example:** Data warehouse, Data mart

### 3.8: Data Integrity

**Definition:** Data integrity refers to the accuracy and consistency of data in a database.

**Explanation:** Data integrity ensures that data is correct and consistent, and that it can be trusted.

**Example:** Primary keys, foreign keys, Check constraints

### 3.9: Data Security

**Definition:** Data security refers to the protection of data from unauthorized access or tampering.

**Explanation:** Data security involves implementing techniques to protect data from unauthorized access or tampering.

**Example:** Authentication, Authorization, Encryption

### 3.10: Database Transaction

**Definition:** A database transaction is a sequence of operations that are executed as a single, all-or-nothing unit.

**Explanation:** A database transaction ensures that data is consistent and accurate by ensuring that all operations are executed correctly.

**Example:** Atomicity, Consistency, Isolation, Durability (ACID)
