### Schemas

#### Database Management System

#### Module: 8 hours

#### Topic Summary

- **Definition:** A schema is a conceptual representation of the structure and organization of a database.
- **Types of Schemas:**
  - Physical schema: defines the physical storage of data in the database.
  - Logical schema: defines the relationships between data entities in the database.
  - User schema: defines the views and relationships between data entities for a specific user.
- **Importance of Schemas:**
  - Defines data organization and relationships.
  - Helps in data consistency and integrity.
  - Facilitates data retrieval and manipulation.
- **Key Concepts:**
  - Entity-relationship model (ERM).
  - Relational model.
  - Object-oriented model (OOM).
- **Important Formulas and Theorems:**
  - Normal form (NF1-NF3) for relational databases.
  - Data normalization rules.
  - Entity-relationship diagram (ERD) construction rules.
- **Key Terms:**
  - Entity: a thing or concept that has attributes.
  - Relationship: a connection between entities.
  - Att tribute: a characteristic of an entity.
  - Primary key: a unique identifier for an entity.
  - Foreign key: a field that references the primary key of another entity.
- **Key Concepts and Definitions:**
  - Data integrity: the guarantee that the data is consistent and accurate.
  - Data consistency: the guarantee that the data remains in a consistent state.
  - Data normalization: the process of organizing data in a database to minimize data redundancy and improve data integrity.
  - Denormalization: the process of intentionally violating data normalization rules to improve query performance.

### Revision Key Points

- Schemas define the structure and organization of a database.
- Schemas can be physical, logical, or user-defined.
- Schemas help in data consistency and integrity.
- Schemas facilitate data retrieval and manipulation.
- Entity-relationship model (ERM) is a key concept in schema design.
- Normal form (NF1-NF3) is an important concept in relational database design.

### Important Formulas and Theorems

- Normal form (NF1-NF3):
  - NF1: each non-key attribute must be in a separate row.
  - NF2: each non-key attribute must be in a separate column.
  - NF3: each non-key attribute must be in a separate row and column.
- Data normalization rules:
  - First normal form (1NF): eliminate repeating groups.
  - Second normal form (2NF): eliminate partial dependencies.
  - Third normal form (3NF): eliminate transitive dependencies.
