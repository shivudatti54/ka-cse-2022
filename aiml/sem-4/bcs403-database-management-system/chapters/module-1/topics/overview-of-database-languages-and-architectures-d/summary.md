# Database Management System

### Overview of Database Languages and Architectures: Data Models

#### Key Concepts

- **Data Model**: A representation of the data and its relationships using abstract concepts.
- **Entities**: Real-world objects or concepts that are represented in the database.
- **Attributes**: Characteristics or properties of an entity.
- **Relationships**: Connections between entities.
- **Data Types**: Classification of data into categories such as integer, string, date, etc.

#### Data Models

- **Entity-Relationship Model (ERM)**: A conceptual model that represents data using entities, attributes, and relationships.
- **Object-Oriented Model (OOM)**: A model that represents data using objects and their interactions.
- **Relational Model**: A model that represents data using tables and relationships between them.

#### Notations and Symbols

- **Entity Set**: A set of entities.
- **Relationship Set**: A set of relationships between entities.
- **Attribute Set**: A set of attributes for an entity.
- **Primary Key (PK)**: A unique identifier for an entity.
- **Foreign Key (FK)**: A key that references another entity.

#### Important Formulas and Theorems

- **Data Integrity Theorem**: Ensures that the data in a database is consistent and accurate.
- **Consistency Theorem**: Ensures that the database maintains its consistency over time.
- **Normalization Theorem**: Ensures that the database is organized in a way that minimizes data redundancy.

#### Data Model Design

- **First Normal Form (1NF)**: Ensures that each table has a unique primary key.
- **Second Normal Form (2NF)**: Ensures that each non-key attribute depends on the entire primary key.
- **Third Normal Form (3NF)**: Ensures that each non-key attribute depends on the primary key and not on another non-key attribute.
