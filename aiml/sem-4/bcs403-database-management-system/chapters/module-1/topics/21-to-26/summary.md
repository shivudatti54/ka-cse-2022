# **DATABASE MANAGEMENT SYSTEM: 2.1 to 2.6 Revision Notes**

## **2.1 Database Types**

- Definition: A database is a collection of organized data.
- Types:
  - Relational databases
  - NoSQL databases
  - Graph databases
- Advantages and Disadvantages of each type

## **2.2 Database Management Systems**

- Definition: A database management system (DBMS) is a software that manages a database.
- Characteristics of a DBMS:
  - Data independence
  - Control over data definition
  - Control over data modification
- Examples of popular DBMS: MySQL, Oracle, Microsoft SQL Server

## **2.3 Database Normalization**

- Definition: Normalization is the process of organizing data in a database to minimize data redundancy and dependency.
- Normalization rules:
  - First normal form (1NF)
  - Second normal form (2NF)
  - Third normal form (3NF)
- Importance of normalization

## **2.4 Database Denormalization**

- Definition: Denormalization is the process of organizing data in a database to improve query performance.
- Techniques of denormalization:
  - Data replication
  - Data aggregation
- Importance of denormalization

## **2.5 Data Modeling**

- Definition: Data modeling is the process of creating a visual representation of a database schema.
- Types of data models:
  - Entity-relationship model (ERM)
  - Object-oriented model (OOM)
- Importance of data modeling

## **2.6 Database Design**

- Definition: Database design is the process of creating a comprehensive and well-structured database schema.
- Steps involved in database design:
  - Identify entities and attributes
  - Create entities and relationships
  - Apply normalization rules
- Importance of database design

## **Important Formulas and Definitions**

- **Entity-Relationship Model (ERM)**: A representation of a database schema using entities and relationships.
- **First Normal Form (1NF)**: A normalized database schema where each table has a unique primary key.
- **Second Normal Form (2NF)**: A normalized database schema where each non-key attribute depends on the entire primary key.
- **Third Normal Form (3NF)**: A normalized database schema where each non-key attribute depends on the primary key and not on another non-key attribute.

## **Important Theorems**

- **Gödel's Incompleteness Theorem**: A mathematical theorem that states that any formal system that is powerful enough to describe basic arithmetic is either incomplete or inconsistent.
- **Church-Turing Thesis**: A mathematical theorem that states that any effectively calculable function can be computed by a Turing machine.
