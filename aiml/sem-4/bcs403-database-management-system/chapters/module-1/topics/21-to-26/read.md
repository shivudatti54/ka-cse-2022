# **2.1 to 2.6: Database Management System Study Material**

## **2.1: Introduction to Database Management System**

### Definition

A Database Management System (DBMS) is a software system that allows you to define, create, maintain, and manipulate databases. It provides a way to store, manage, and retrieve data in a logical and efficient manner.

### Characteristics of a DBMS

- **Data Independence**: The DBMS provides a way to store and manage data independently of the underlying hardware and software.
- **Security**: The DBMS provides a way to control access to the data and ensure that it is protected from unauthorized access.
- **Concurrency Control**: The DBMS provides a way to manage multiple transactions that occur concurrently.
- **Data Integrity**: The DBMS provides a way to ensure that the data is consistent and accurate.

### Advantages of DBMS

- **Improved Data Management**: DBMS provides a centralized way to manage data, making it easier to access and maintain.
- **Increased Productivity**: DBMS automates many tasks, such as data backup and recovery, making it easier to focus on business activities.
- **Better Data Security**: DBMS provides a secure way to store and manage data, reducing the risk of data breaches.

### Disadvantages of DBMS

- **Complexity**: DBMS can be complex to implement and manage.
- **High Cost**: DBMS can be expensive to purchase and maintain.
- **Learning Curve**: DBMS requires specialized knowledge and skills.

## **2.2: Types of DBMS**

### Classification of DBMS

There are two main types of DBMS:

- **Relational DBMS**: Relational DBMS stores data in tables with well-defined relationships between them.
- **Object-Oriented DBMS**: Object-Oriented DBMS stores data in objects that contain both data and methods for manipulating that data.

### Example of Relational DBMS

- SQL Server
- Oracle
- MySQL

### Example of Object-Oriented DBMS

- ObjectDB
- Matisse
- Gemstone

## **2.3: Database Models**

### Definition

A database model is a conceptual representation of a database. It provides a way to define the structure and organization of data.

### Types of Database Models

- **Physical Model**: Physical model represents the actual storage of data.
- **Logical Model**: Logical model represents the conceptual structure of data.
- **Functional Model**: Functional model represents the functions that a database performs.

## **2.4: Database Design**

### Definition

Database design is the process of creating a database that meets the needs of an organization.

### Steps in Database Design

- **Define the Requirements**: Define the requirements of the database, including the types of data that will be stored and the functions that the database will perform.
- **Choose a DBMS**: Choose a DBMS that meets the requirements of the database.
- **Design the Database Structure**: Design the database structure, including the tables, fields, and relationships.
- **Implement the Database**: Implement the database, including the creation of tables, fields, and relationships.

## **2.5: Database Administration**

### Definition

Database administration is the process of managing and maintaining a database.

### Tasks Involved in Database Administration

- **Backup and Recovery**: Back up the database regularly and implement a recovery plan in case of data loss.
- **Security**: Ensure that the database is secure and that access is restricted to authorized users.
- **Performance Tuning**: Optimize the performance of the database to ensure that it can handle a high volume of transactions.
- **Troubleshooting**: Identify and resolve problems with the database.

## **2.6: Database Applications**

### Definition

Database applications are software applications that use a database to store and manage data.

### Examples of Database Applications

- **E-commerce Websites**: E-commerce websites use databases to store and manage customer information, orders, and inventory.
- **Banking Systems**: Banking systems use databases to store and manage customer information, transactions, and accounts.
- **Healthcare Systems**: Healthcare systems use databases to store and manage patient information, medical records, and billing information.
