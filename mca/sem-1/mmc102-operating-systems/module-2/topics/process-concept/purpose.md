# Learning Objectives

After studying this topic, you should be able to:

1. Define what a process is and explain how it differs from a program, demonstrating understanding of the process as an active execution entity versus a passive code entity.

2. Identify and describe the five process states in the process state model, and accurately draw and explain the state transition diagram showing how processes move between states.

3. Explain the structure and components of the Process Control Block (PCB), understanding its role as the central data structure for process management and context switching.

4. Analyze the context switching mechanism, including the steps involved in saving and restoring process state, and evaluate the performance implications of context switching overhead.

5. Distinguish between CPU-bound and I/O-bound processes and explain how their characteristics affect system performance and scheduling decisions.

6. Compare and contrast the functions of long-term, short-term, and medium-term schedulers in the process management hierarchy.

7. Apply process concept knowledge to solve numerical problems involving context switching overhead calculations and process state transitions.
