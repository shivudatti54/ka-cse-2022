# Learning Objectives

After studying this topic, you should be able to:

1. Define system calls and explain their role as the interface between user applications and the operating system kernel

2. Describe the mechanism by which system calls transition from user mode to kernel mode, including the use of software interrupts or specialized instructions

3. Identify and categorize the six major types of system calls: Process Control, File Management, Device Management, Information Maintenance, Communication, and Protection

4. Explain the purpose and usage of key process control system calls including fork(), exec(), wait(), and exit()

5. Demonstrate understanding of file management system calls by explaining open(), read(), write(), close(), and their parameters

6. Compare and contrast different inter-process communication mechanisms including pipes, named pipes (FIFOs), and sockets

7. Analyze how protection system calls implement security policies through permissions, ownership, and privilege management

8. Apply knowledge of system calls by writing programs that create processes, perform file I/O, and implement inter-process communication
