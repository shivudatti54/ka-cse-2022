# Learning Objectives

After studying this topic, you should be able to:

1. Define system calls and explain their role as the interface between user applications and the operating system kernel

2. Differentiate between user mode and kernel mode execution and describe how system calls facilitate the transition between these modes

3. Categorize system calls into their functional types: process control, file management, device management, information maintenance, communication, and protection

4. Explain the mechanism by which system calls are invoked, including parameter passing and the role of the system call interface

5. Analyze and write C programs that use common system calls such as fork(), exec(), open(), read(), write(), close(), pipe(), and wait()

6. Demonstrate understanding of inter-process communication using pipe() and related system calls

7. Interpret return values from system calls, including error handling through return values and the errno variable

8. Apply knowledge of system calls to solve practical system programming problems and understand operating system internals
