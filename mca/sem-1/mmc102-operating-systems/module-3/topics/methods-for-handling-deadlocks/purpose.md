# Learning Objectives

After studying this topic, you should be able to:

1. EXPLAIN the four necessary conditions for deadlock and their role in deadlock handling strategies.

2. COMPARE and contrast deadlock prevention, deadlock avoidance, deadlock detection with recovery, and deadlock ignorance with respect to overhead, system restrictiveness, and applicability.

3. APPLY the Banker's Algorithm to determine whether a given system state is safe or unsafe.

4. DESIGN a resource ordering scheme to prevent circular wait in a given system scenario.

5. ANALYZE the wait-for graph to detect the presence of deadlock in a system of processes.

6. EVALUATE the trade-offs involved in choosing among different deadlock handling methods for specific system environments.

7. EXPLAIN why most general-purpose operating systems use the ostrich algorithm rather than more sophisticated deadlock handling techniques.
