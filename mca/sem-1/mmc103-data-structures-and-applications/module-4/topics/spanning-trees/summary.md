# Spanning Trees
A spanning tree of a connected graph G(V,E) is a connected, acyclic subgraph containing all |V| vertices with exactly |V|-1 edges. Key properties include: unique path between any two vertices, fundamental cycles created by adding non-tree edges, and fundamental cutsets formed by removing tree edges. DFS and BFS traversals produce spanning trees with distinct characteristics—DFS yielding deep, narrow structures with back edges, while BFS produces short, wide trees with cross edges and guarantees shortest paths in unweighted graphs. Cayley's formula states that K_n has n^(n-2) spanning trees, computable for any graph via Kirchhoff's Matrix-Tree Theorem using the Laplacian matrix. The Minimum Spanning Tree problem is solved optimally by Kruskal's algorithm (O(E log V) using union-find) or Prim's algorithm (O(E log V) with binary heap), both justified by the cut property: the lightest edge crossing any cut belongs to some MST. For graphs with distinct edge weights, the MST is unique.
