# Forests
A forest is an acyclic graph with multiple connected components, each component being a tree. Key properties include: (1) a forest with $n$ vertices and $c$ components has exactly $n-c$ edges, (2) forests are bipartite, and (3) the edge-vertex relationship determines tree vs. forest classification. Representation methods include parent pointer, left-child-right-sibling (transforming general trees to binary trees), and sequential array representation. Fundamental operations include MakeTree, FindRoot, Merge, and Split with corresponding time complexities. Critical applications include: Union-Find data structure with $O(\alpha(n))$ amortized complexity, spanning forests in BFS/DFS graph traversal, and Kruskal's MST algorithm construction. The LCRS transformation enables binary tree algorithms on general tree structures.
