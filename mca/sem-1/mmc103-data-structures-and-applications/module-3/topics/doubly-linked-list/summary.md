# Doubly Linked List
A Doubly Linked List extends singly linked list functionality by maintaining bidirectional links between nodes through prev and next pointers. This enables O(1) deletion given a node pointer, bidirectional traversal, and O(1) insertion/deletion at both ends (with tail pointer). The key theorems establish O(1) deletion complexity compared to O(n) in singly linked lists. Core operations include insertion (beginning, end, position), deletion (beginning, end, position, given node), search, and traversal (forward and reverse). Practical applications span browser history, music playlists, undo mechanisms, and LRU caches. Circular variants eliminate boundary conditions. Comprehensive time-space complexity analysis and assessment items verify understanding at the application-analysis level.
