# Sparse Matrices
A sparse matrix contains predominantly zero values, and storing it efficiently is crucial for memory-constrained applications. The linked list representation (multilist or orthogonal list) addresses limitations of array-based triplet representation by providing dynamic memory allocation, efficient insertion/deletion operations, and direct row/column access through header nodes. Each non-zero element is stored as a node with five fields: row index, column index, value, next_row pointer (same column), and next_col pointer (same row). The matrix employs row and column header arrays to enable O(1) access to any row or column. Key operations include insertion O(k_i + k_j), deletion O(k_i + k_j), traversal O(k), and search O(k_i). The transpose operation requires O(k × (k_i + k_j)) time, while matrix addition runs in O(k_A × k_B) worst case. The linked representation trades implementation complexity and cache performance for dynamic sizing and efficient pointer-based operations.
