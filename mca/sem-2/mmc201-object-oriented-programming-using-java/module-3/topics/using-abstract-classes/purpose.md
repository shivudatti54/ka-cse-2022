# Learning Objectives

After studying this topic, you should students able to:

1. Define what an abstract class is and explain its purpose in Java programming.
2. Identify the key characteristics of an abstract class, including the use of the `abstract` keyword and the presence of abstract and concrete methods.
3. List the rules for declaring and using abstract classes, including the inability to instantiate them directly.
4. Explain how to extend an abstract class and implement its abstract methods in a concrete subclass.
5. Describe the scenarios in which abstract classes are preferred over interfaces, and vice versa.
6. Compare and contrast abstract classes with interfaces, highlighting their similarities and differences.
7. Justify the use of abstract classes in real-world applications, such as the Java Collections framework.
8. Design a simple class hierarchy using abstract classes to demonstrate their practical application.
