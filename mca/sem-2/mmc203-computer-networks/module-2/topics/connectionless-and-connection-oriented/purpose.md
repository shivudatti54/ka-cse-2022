# Learning Objectives

After studying this topic, you should students able to:

1. Explain the fundamental differences between connectionless and connection-oriented communication paradigms in computer networks.

2. Describe the UDP datagram structure and identify the fields in the 8-byte header, including source port, destination port, length, and checksum.

3. Analyze the TCP three-way handshake process and explain why it is necessary for reliable connection establishment.

4. Identify and describe the key mechanisms that make TCP reliable, including sequence numbers, acknowledgments, flow control, and congestion control.

5. Compare UDP and TCP based on characteristics such as reliability, ordering, overhead, speed, and appropriate use cases.

6. Evaluate which transport protocol (UDP or TCP) is more suitable for specific application scenarios like DNS resolution, video streaming, web browsing, and file transfer.

7. Understand the TCP header structure and explain the significance of important fields like sequence number, acknowledgment number, flags, and window size.

8. Describe the TCP connection termination process (four-way handshake) and explain the TIME-WAIT state.
