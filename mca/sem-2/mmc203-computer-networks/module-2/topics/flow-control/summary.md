# Flow control at the Data Link Layer regulates data transmission rates between communicating nodes to prevent receiver overload. Stop-and-Wait, the simplest mechanism, transmits single frames awaiting acknowledgment, achieving utilization $U = T_t/(T_t + 2T_p)$ but suffering low throughput in high-delay networks. Sliding Window protocols permit multiple outstanding frames through a transmission window, with Go-Back-N retransmitting all subsequent frames upon loss and Selective Repeat retransmitting only lost frames. Protocol implementations include HDLC's RR/RNR frames, PPP's LCP negotiation, and Ethernet's PAUSE frames. Optimal window sizing requires consideration of the bandwidth-delay product for full channel utilization.
