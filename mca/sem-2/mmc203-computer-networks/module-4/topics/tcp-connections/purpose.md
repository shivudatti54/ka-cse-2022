# Learning Objectives

After studying this topic, you should students able to:

1. Explain the TCP three-way handshake process and describe the significance of each step in connection establishment.

2. Identify and describe all TCP control flags (SYN, ACK, FIN, RST, PSH, URG) and their roles in TCP communication.

3. Trace the four-way handshake process for graceful TCP connection termination and explain the TIME_WAIT state.

4. Differentiate between TCP and UDP, highlighting why TCP is preferred for reliable data transfer applications.

5. Write socket programs for both client and server using the BSD socket API in C.

6. Describe the various TCP connection states and explain state transitions in the TCP state diagram.

7. Analyze TCP segments using Wireshark or similar tools by interpreting header fields and flags.

8. Understand TCP options like MSS, window scaling, and SACK, and explain their role in optimizing TCP performance.
